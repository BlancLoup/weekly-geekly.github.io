Who's what about shifttstas about distributed networks It has been almost a year since the last review of the ZeroNet network, during this time a lot of interesting things happened on the network.

What changes were in the core


Since then, interesting changes have been made to the core:

- Added support for optional files (files that the client will download to himself only with direct access)
- Accelerated network operation and especially startup time (Significantly)
- Related Sites (Merger Sites)
- Ability to subscribe to the resource of interest and receive changes to your feed (for example, new posts or comments)
- Search function for sites you are subscribed to
- Improvement of the transfer / change tracking protocol (now less data is transmitted within the network since only changes are transmitted)

But, of course, the most important thing is the resources and content that appeared during this time.

ZeroWiki - distributed wikipedia





ZeroWiki - the engine that allows you to create sites like Wikipedia (or similar to the wiki engine), supports the main functions - versioning, creating files.

It would seem that there is such a thing - in 2016 to make a wiki-like engine, but do not forget - the network is distributed and, accordingly, you need to ensure the integrity and authenticity of the data.

What is missing network?


During this year, the network made a big breakthrough and overgrown with content, users, but a couple of technical implementations are not enough for what could be said - yes, this network meets all needs 100%.

Namely:

- Work through DHT
- IPV6
- Mobile customers
- Implementing a network kernel in a compiled language
