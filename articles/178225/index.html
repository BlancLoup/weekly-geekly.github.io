Q. What versions of UC Browser are available for the Android platform?
A. The lightweight mobile browser UCWEB and the more functional UC Mobile on the WebKit engine were initially released, but then these projects were merged and the product was renamed UC Browser. In spite of this, at the moment there are three different browsers that, besides the interface language and a variety of additional functions, differ primarily in the browser engine:
1. UC Browser mini on the U2 engine (classic compression engine, compression can be disabled only for wap / wml) - the heir to the classic UCWEB. There are only three types of page adaptation - screen compression, compression with scaling, scaling without compression.
2. UC Browser on the U3 engine (modern engine, supports work with both compressed and uncompressed html pages) - the result of combining UCWEB and UC Mobile;
3. UC Browser HD (for tablets) on the engine of the system (stock) browser Android, traffic compression does not occur.

Q. What is the difference between these versions?
A. It is worth noting that the versions can be divided into official and unofficial.
Official:
1. Chinese - in the version there are a lot of things, all sorts of goodies in the form of a Chinese market (for example, in it you can download initially paid programs for free). New features are first received by this version, but the obvious disadvantage is the Chinese interface language;
2. Russian - a version based on the Chinese, but rather severely curtailed in functional terms;
3. The mini version is an even shorter browser version. Nothing extra. Uses a minimum of memory with minimal functionality. The main difference between the mini version is its uhtm page format;
4. HD version - for tablets.
Informal:
1. Chinese - the functionality is almost the same as that of the official Chinese version, but it is translated into Russian and corrected by our craftsmen. Minus - some places in the interface remain Chinese (if they are sent by the server) or require root-user intervention (replacing the home page, search engines, etc.).
Starting from version 9.0, the official Russian version will have the same functionality as the Chinese one.

Q. How to view browser saved pages?
A. You just need to open the saved html-file with the browser's built-in file manager. If you open a program through the file manager and the program is not assigned to this file type by default, the system will ask you to open this file. Pages saved in the mini version should be opened only in the browser.

Q. Flash does not work on the latest versions. What to do?
A. Try the following: there is an Adobe icon in the phone menu, click on it, click on UC Browser, if there is a puzzle instead of the settings menu - it means that UC Browser does not see your flash player. It is noted that for normal operation Adobe flash should be non-system, or there should be an update without integration into ROM, if the player is installed in the system.
Plus, it may be the performance of the phones, because flash is very hard, especially youtube videos. For this, it is better to use special applications.
All of the above applies to translated Chinese versions. On the official Russian versions of flash, it works acceptably starting from version 9.0.1.275, on earlier ones it does not.

Q. The home page flies off the translated versions.
A. To prevent the homepage from flying, we follow the path /data/data/com.UCMobile/UCMobile/usdata4 [or 5, depending on the version] and for the files:
lp_favor, lp_navi, lp_navi_css, nlp_navi_us_popularsites, nlp_navi_us_quicklinks, nlp_navi_us_topsites remove the write permission (using any file manager with root).
The search files sl_start_search, sl_start_search2 are first replaced, then we put the read-only attributes. Ad advertisement files (of type sl_ad_10) - delete everything and put a ban on writing to the usdata4 folder [5].



In general, it is necessary to take into account the fact that everything changes from version to version and the files can be different.

Q. Can I make sure that UC Browser does not close when minimized to the background?
A. The Android operating system itself determines when and which programs to close, the more RAM, the more programs remain in it. The browser does not have any specific settings for working in the background, everything is determined solely by the operating system. Plus, UC Browser remembers its last state, open pages and even the position on the page itself.

Q. Spoilers that open, then no, why is this happening?
A. To allow spoilers to open without reloading the page, it is enough to turn off page compression in the browser settings. When compression is enabled, the spoiler content is not loaded and becomes available only after your request, which causes the content to load along with the page reload.

Q. How can I put another theme on the official Russian version?
A. Everything is simple here: download the theme, then open the built-in file manager (Menu - Downloads - Fileman) - tap on the downloaded theme. Themes intended for versions below 8.8 are not compatible with themes for newer versions.

Q. Why are some of my websites without icons (logos) in navigation?
A. Icons are displayed only if they are in the UC Mobile database. The base itself is quite small, so the icons of most sites will not be displayed.

Q. Sites do not correctly determine my time zone. How to fix?
A. Quite often sites put the time zone automatically, according to your IP (browser servers are located all over the world, including Russia, China, America, Canada and many others). Sometimes the browser may not work through your national server, hence the wrong definition. To solve the problem, it is enough to remove the automatic setting of the time zone in the settings of your profile.

Q. When the page compression mode is enabled, it will not let me on some sites. Why so?
A. There are several options:
1. The site administrator banned the IP of the compression servers of the UC Browser;
2. Some sites impose restrictions: on the use of a proxy, on a location, on a User-agent, because when you turn off compression, sites see your real location, and in compression mode there will be a proxy server location;
3. Also, the provider, the operator can limit the visit and block sites, and nothing can be done about it.

Q. Can I import bookmarks from the FireFox browser?
A. Yes, you can.
1. On Firefox install the Gmarks plugin
2. In the main Firefox menu, select: Gmarks - Arrange bookmarks - File - Import.
3. Bookmarks are imported into Google Bookmarks (since you are using the Android OS, it is assumed that you have a google account).
4. Install GBookmark or GoMarks for Google Bookmarks on your device to sync Google Bookmarks bookmarks and stock browser, synchronize them.
5. In UC Browser: Bookmarks - Manage - Import - Import system bookmarks.

In a similar way, you can import bookmarks from other browsers if they allow you to save them in html-format or synchronize with Google Bookmarks.

Q. Can I import bookmarks from Opera mini?
A. You can use bookmarks from any Opera browser without synchronizing them. For this:
1. In Opera settings, turn on the Opera Link bookmark synchronization service;
2. Synchronize with the service;
3. In UC Browser, go to link.opera.com , log in, and see our bookmarks there;
4. Save the bookmark with this page in the UC Browser. Now all bookmarks with Opera are always at hand, up to date.
