Three ways to update a query in Jira from ScriptRunner using the Jira Java API

This article will look at three ways to update a query in Jira using the Jira Java API.
I will use the following methods of the Jira Java API:

  • Issue.setCustomFieldValue (CustomField customField, Object value)
  • CustomField.updateValue (FieldLayoutItem fieldLayoutItem, Issue issue, ModifiedValue modifiedValue, IssueChangeHolder issueChangeHolder)
  • IssueService.update (ApplicationUser user, IssueService.UpdateValidationResult updateValidationResult)

There will be examples of scripts for updating all types of custom fields available in Jira from the ‚Äúbox‚Äù with a table that shows the differences between the work of the considered methods from each other.


I tested all the scripts on Jira 7.7.0 in the script console of the Adaptivist ScriptRunner 5.3.7 plugin. The purpose of the article was not to write a ‚Äúclean code‚Äù, but a statement of principles for working with queries.

Script texts can be downloaded here .
