Writing trading robots


One of the most dynamically developing areas in the financial industry in recent years has been algorithmic trading and the creation of trading robots. Let's talk about which programming languages ‚Äã‚Äãand why are used in this case.

First of all, as in the case of any software products, the developers of trading robots somehow have to make various compromises that allow you to take into account the main factors affecting the success of work in the market:

  • Trade speed, i.e. the execution of trade operations themselves - the speed of issuing an order, the speed of receiving a response from the exchange, the speed of processing by the robot itself
  • Versatility and customizability of the robot for working with new markets, financial instruments, variants of algorithms
  • The speed of development and making significant changes.

Accordingly, in those cases where speed is important (for example, in the case of HFT trading), effective low-level languages ‚Äã‚Äãare used - C ++ and even pure C.

If it is planned to program a trading system that will work according to a strategy that does not involve making a large number of transactions, then at first glance it‚Äôs not the speed that works, but the convenience and speed of the development itself - in this case, not the fastest languages ‚Äã‚Äãare used, for example, on the platform .NET (the same C # or Visal Basic).

Another option is a situation in which significant computations are required, but not necessarily in real time. In such cases, the use of interpreted Python, R, or TradeScript languages ‚Äã‚Äãcan be justified.

In other words, the corresponding programming languages ‚Äã‚Äãare used for different modules of the trading robot:

  • A trading engine is a simple, fastest robot that performs a simple set of operations, as a rule, it is created in C, C ++ and assemblers of different levels.
  • Parameters management system - such a module combines the functions of controlling the operation of algorithms and the user interface, it can include control parameters and mechanisms for presenting trading results. C ++, C #, Java, etc. can be used here.
  • The system of back-testing and selection of trading parameters - the module is used to test new algorithms on historical data; it selects new parameters for current algorithms. To implement this module, you can use Python and other scripting languages. (In our blog, we published a story about creating an event-oriented backtesting system in Python ).
