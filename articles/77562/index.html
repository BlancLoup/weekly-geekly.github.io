RSS / HTML Grabber 1.0 for Livestreet

Why do I need it?
More and more information is required for a modern website. Frequent updates allow you to increase the searchability of the site in search engines and, consequently, increase attendance.
But what to do if you want the site itself filled with content?
You can put a cunning engine and hire someone to write articles to the site.
You can put a special module that will periodically fill your site with information.
And what to do if you already have a working site on CMS Livestreet , which you don‚Äôt want to change a lot, but would you like some fresh content?

What is an RSS / HTML grabber?
A module for CMS Livestreet that collects content from various RSS feeds and thus attracts a large number of visitors from search engines to the site.
Each page of this site is well indexed by search engines.
Hourly update information on the site makes it a favorite of search engines, and their bots are constantly plowing its pages. Such a love of search engines for a site translates into thousands of visitors to the pages of this site.
This site can be used to sell advertising space, here you can place contextual advertising AdSence, advertise their commercial projects.
Test sites show attendance of more than 2000 visitors per day in the first 3-4 months of the site.

The main features of the grabber:

- Supports RSS 1 & 2, ATOM feeds.
- Set the update interval and the maximum number of imported messages in the tape.
- Select the blog and the author of publications for imported messages.
- Setting a mask for searching and importing full text from an HTML page using [[get]] & [[set]] tags.
- Set valid HTML tags.
- Indication of a forced withdrawal to the main page and the possibility of commenting on new topics.
- Importing the date of the tape or indicating the current date of the new topics.
- Import images to the server.
- Synonymizer with a base of 1,300,000 words . (It is recommended to use if you have a VDS or VPS or your hosting provider do not care for a small load and consumption of 256+ virtual memory)
- Preliminary ajax check of the tape before adding (It is recommended to use to check the correctness of the search mask for the full text of the topic).
- Manage tapes (Create / Edit / Delete).
- Manual or automatic update of each tape or all tapes (manual - pens, automatic - crown =))
- Automatic generation of tags based on the entire content of the topic
- Work with encodings WINDOWS-1251, KOI8-R, ISO-8859-5, UTF-8.
- Ability to easily translate the interface into another language.

Server system requirements:

- PHP 5.x +
- MySQL 4+
- PHP libraries xml, mb

Required PHP settings:

- safe_mode is off
- allow_url_fopen is on
