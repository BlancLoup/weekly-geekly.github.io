An overview of the online service for working with accounts Tradeshift.com

Introduction


When we tried to briefly describe our software, we ran into the problem of how to classify it. Began to look for analogues in the Russian market, but did not find them. We conducted an analysis of similar products in the west and found the Tradeshift.com service. This service has a lot in common with our Agora system. Below is a brief overview of the Tradeshift.com service.


Idea


The creators of Tradeshift give a simple definition to their brainchild - ‚Äúa free invoicing service‚Äù.
‚ÄúTradeshift is a global startup that aims to change the way you do business. This will allow organizations to exchange accounts for free. The service unites enterprises with all suppliers and customers, which helps not only to save time, but also to get paid on bills in a shorter period. ‚Äù

The main idea of ‚Äã‚Äãthis service is that sellers upload accounts to the Tradeshift server in various ways: manually, via the tradeshift-ERP connector, via the SFTP / REST service, or via the Tradeshift API. Ways to work and monetize


Currently, Tradeshift provides customers with several modes of operation:
1) Manually uploading bills to the Tradeshift web server
A user in his personal account can upload accounts to the tradeshift server. In this case, the service works as a storage and format converter.
This mode is free.

2) Download bills via ERP connector
The client can generate invoices in the format of his ERP and automatically send them to the Tradeshift server through a connector.

3) Download bills via REST web service or SFTP
Tradeshift provides developers with an API for self-development of loaders. Data can be loaded via a REST web service or SFTP server.

Additional features
