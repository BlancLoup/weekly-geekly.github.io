Repetition - the mother of learning


I will try to rephrase and explain the above written about styles. Why it was necessary to make styles of their layout? It's necessary:
  • so that styles can be conveniently applied repeatedly;
  • so that when changing the design / element does not interfere with the layout;
  • that the code is semantic and clean (the two points above are included in this paragraph);
  • to reduce the HTML code / page (well, yes, this is a funny point nowadays);
  • something else.


CSS3 came to the aid of all of the above with its capabilities. But more on that later.

And what do frameworks do? Of course the main thing - it simplifies life. But for the rest:
  • Styles are in a separate file, but how many of these styles?
  • to change the layout, you need to climb not in styles, but in layout;
  • code is semantic, but redundant most often;
  • all sites are of the same type (well, this is the trouble of frameworks in general)
  • something else.

To make it clear - I will show by example:

90% of site frameworks look something like this on HTML5
