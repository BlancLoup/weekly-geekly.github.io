Testing the monitoring system of virtual servers

As part of a set of measures to improve the quality of service for our clients, we conduct a test of the monitoring system for custom virtual servers. In order for us to add your virtual server to monitoring, you need to install zabbix-agent on your virtual server. This is done by means of the package manager of the system (yum, apt, emerge, yast, and so on).

Installation on CentOS / Fedora:
yum install zabbix-agent
Installing on Ubuntu / Debian:
apt-get install zabbix-agent ( , apt-get update)
Installation on Gentoo:
USE="agent" emerge -v1 zabbix

After installing the agent, you need to edit its configuration
file and bring it to mind:

Hostname=xxxx-xx.clodo.ru # - - id
Server=monitor.clodo.ru
ServerPort=10051
StartAgents=3

After that, you need to add an agent to the system autorun and start it.

After setting up the virtual server, please notify the technical support ID of the virtual server for which you have completed the preparatory work for monitoring connection.

After the end of testing, the agent will be added to the base system and its additional configuration will not be required. There will also be an increase in the number of services to be monitored and will
the opportunity to choose the method of notification and, possibly, access to the web monitoring interface.
