Hard disk 3TB - connect to Windows XP

In connection with the crisis of the hard disk market, it has increasingly begun to think that now is the time to buy a 3TB disk. By chance, my choice fell on the model ST3000DM001. The cost of this miracle turned out to be rather big, but I decided for myself in advance that I would chase the volume, closing my eyes to the price.

At work, I stuck my purchase in Windows 7 and without sound, formatted it in a new-fashioned GPT, which, unlike the MBR, supports disks larger than 2TB. But there are problems on the home Windows XP, the disk is visible, but there is no access to it - a problem with GPT. Outdated OSes refuse to accept a newbie.

Began to google, found the discussion , which boils down to what you need to put Paragon GPT Loader. But this version is only suitable for Hitachi, for other models dancing with a tambourine, judging by the forum, exceeds all reasonable limits.

As a result, on the advice of Dip, created the install.reg file: 
Windows Registry Editor Version 5.00 [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\gpt_loader] "VIDS"=hex:0E,DA

Next, using Universal Extractor , I extracted from the Paragon GPT Loader distribution kit the gpt_loader files with note 2 located in the MSI directory: \ SourceDir \ program files \ Hitachi \ GPT Disk Manager \ gpt_loader: 
 21.12.2010 19:18 7 464 gpt_loader,2.cat 21.12.2010 19:18 2 590 gpt_loader,2.inf 21.12.2010 19:18 51 792 gpt_loader,2.sys

And renamed them, removing the deuce from the name, it turned out like this (MD5 checksums): 
 7eb7121fe3bf4d30e319533a6e0116c5 *gpt_loader.cat 44275f00bdd459a816e1633846542943 *gpt_loader.inf 2234a9ba20e6a5d4d6404c499e8438a2 *gpt_loader.sys 22aa73696c1d1e4fff650fc606a2192f *install.reg

Further applied to the REG file: regedit install.reg
And on the gpt_loader.inf file I right-clicked in the explorer and selected Install.

After the reboot, the hard drive was determined and earned.
