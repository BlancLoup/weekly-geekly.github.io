Google code hosting - as I saw it


I have had a small project project on my homepage long ago, but since I began to use it in some general education schools, interest in it has grown and I decided not only to significantly improve it, but also to choose some proper open source hosting .



What we have


Before telling about the real state of affairs, I will briefly repeat everything that can be read without me on Google.

Qualitative restrictions


  • It is forbidden to host not opensource (as well as child porn and everything else,
    which is contrary to humanistic ideals)
  • There is no localization (yet), but you can write in Russian for everything in UTF.
  • You can post only wiki-pages. (To hack your site with blackjack and whores will not work.)

Opportunities


  • The project has a title page.
  • There is a dump for distributions (download).
  • There is a wiki with a somewhat reduced set of features.
  • There are "Issues" where you can write something, so as not to forget.
  • SVN with external access via HTTPS and the ability to view via the web-interface.

How it all works


First, what did you like?


The nice thing is that everything is very simple and very friendly.

Everywhere hangsaminki, links to various explanatory and other help. I never had the desire to "google" - everything is at hand.

The project is created in five minutes. You don't have to worry about anything. It is captivating and the first impression is just very pleasant.

A lot has not been completed


For example, there is a web-interface for editing a wiki, but it has no possibilities but to correct the text in <textarea>. To elementary upload a picture - use SVN.

By the way, the pictures are a separate topic. You can use the <img> tag, but where to place files with a picture is not clear. Most people (and me too) lump them together with wiki files.
