Gnome 2.26 Development Overview

At the moment, if we take a look at the Gnome Roadmap of the project, we will see absolutely no significant plans for the new release (unless of course you are not impressed by the dark theme of the design right in the kit). However, this does not mean that the project is absolutely stopped in development. Based on the developer blogs and news from Commit-Digest , I prepared a small selection of existing changes:
- The program continues from gnome-vfs to gio and gvfs.
- Virtually all projects clean up dependencies on libgnome and libgnomeui.
- Message manager Empathy has learned to transfer files (but not in Jabber)
- GEdit got a plugin manager written in Python.
- The basket has got a backend on gvfs.
- Nautilus file manager has learned to access PackageKit to install applications required to open files with unknown extensions.
- In the program for viewing Evince documents, the print dialog has been redesigned.
- Completely rewritten System Log
- Brasero, a disc burning software that integrates seamlessly with Nautilus, Totem and Rhytmbox.
- In Rhytmbox - small changes in order to reduce the load on the processor.
