Java / Python SDKs 1.3.6 released

  • Cancellation of the 1000-limit request to the storage for count and offset. Such unlimited requests will now be executed before the result is obtained, or throw out an error by the timeout of the repository. I myself slightly tested it - there are small brakes on an average base of a couple of million records with a shift of a million (about a couple of seconds delay), but it is quite suitable. Although, judging by the SDK code, the maximum limit is still equal to 2 ^ 31 - 1.
  • New Namespaces API for Java and Python , which allows you to create "utility" (multi-tenancy) applications without special modifications to the already existing code.
  • Replacing the image service with a high-performance picasa-based solution that allows you to do thumbnails (crop and / or resize) without actually using your resources (traffic only) in addition to the previous skills of the Image API.
  • Custom pages for system errors - over quota, DoS, timeout and others.
  • In Java, the Capabilities API has been added, by analogy with Python, as well as java-applications have learned to understand configs in the yaml-format.
  • Many fixed quotas for free applications have increased to the level of paid, more .
  • Blobstore API now supports Content-Range header.
