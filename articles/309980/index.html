Automatic deployment of the ElasticBeanstalk application using Bitbucket Pipelines

Bitbucket has a service (Bitbucket Pipelines), which among other things makes it extremely easy to organize automatic deployment of applications in the Amazon cloud, in particular, using ElasticBeanstallk. Who cares, I ask under the cat.



Prerequirements


You should already have an ElasticBeanstalk application created and deployed environments (to do this, refer to the AWS official documentation, like this, for example, Django is deployed ). Since the service is still being run-in, you need to request access to it here .

Project Setup


Suppose we have two branches, and master we want to deploy to the production environment, and development - to the development environment. Create a bitbucket-pipelines.yml file in the project root with the following contents: 

image: python:2.7.11 pipelines: branches: master: - step: script: - apt-get update # required to install zip - apt-get install -y zip # required for packaging up the application - pip install boto3==1.3.0 # required for beanstalk_deploy.py - zip -rv /tmp/artifact.zip ./* ./.ebextensions ./.elasticbeanstalk # package up the application for deployment - python beanstalk_deploy.py # run the deployment script development: - step: script: - apt-get update # required to install zip - apt-get install -y zip # required for packaging up the application - pip install boto3==1.3.0 # required for beanstalk_deploy.py - zip -rv /tmp/artifact.zip ./* ./.ebextensions ./.elasticbeanstalk # package up the application for deployment - python beanstalk_deploy.py # run the deployment script

To manage the deployment process, I took the script from the example provided by Bitiucket and modified it in order to change the environment depending on the branch.
