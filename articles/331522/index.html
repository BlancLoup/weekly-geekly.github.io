Zextras implementation results for SaaS provider

In this article we want to talk about the example of the actual implementation of Zimbra Open Source Edition and Zextras Suite for the European SaaS service provider, which specializes in providing services to government, educational and medical institutions. They provide their own cloud solutions and serve more than 21,000 mailboxes for more than 200 different domains. In order to modernize the IT infrastructure, wishing to provide end-user high-tech and secure products, ensure reliable mobile synchronization, manage file storage and provide multi-rent, it was decided to implement Zimbra Open Source Edition and a complete set of Zextras Suite simlet.


Zimbra Open Source Edition is a reliable mail server with high-performance PIM / collaboration features.

Zextras Suite - modular extensions with which you can perform backups in real time, manage volumes, configure mobile sync, instant messaging and administrative delegation.

The implementation of Zimbra and Zextras took place in several stages:

  • formation of a modernization plan (data backup, new storage configuration);
  • software installation;
  • create backup storage, new life storage configuration.

We want to show that in the end we succeeded after the implementation of all stages of implementation.

Infrastructure Design: Platform


To improve performance, the partner installed the Zimbra Collaboration multi-server from 15 servers. All of them are located in Amazon's EC2:

Mailbox

  • 1 Mailboxd + Master LDAP
  • 4 mailboxd

Proxy

  • 2 Proxy + LDAP Replica + MTA (Specialized)

MTA

  • 3 Inbound MTA + AS / AV
  • 4 Outbound MTA
  • 1 Outbound MTA (‚Äúon domain‚Äù for batch sending)


Use Storage


Due to the compression and deduplication of Zextras Backup and Zextras Powerstore, extreme volume optimization is possible. The total ‚Äúuser‚Äù quota of maiboxes was 15.5TB, while the actual total size of the main volumes is 1.8 TB, and the actual total size of the secondary volumes is 9 TB. This means saving 30% of the disk space. Secondary volumes contain emails that are stored for more than 90 days and saved as data on Amazon S3 blocks, with huge savings in storage costs. Backup takes a total of 9TB, with 32 days remaining after deletion (maintaining non-linear growth - so in this particular case it could be increased to 90 days without even doubling the backup size).

Infrastructure Design: Storage


Data is stored on Amazon EBS drives. Amazon EBS allows you to create storage volumes and connect them to your Amazon EC2 instances. Once the volumes are mounted on them, you can create the file system, run the databases, or use them in any other way. Amazon EBS volumes are located in specific availability zones within which they are automatically replicated to protect information from the failure of individual pieces of equipment. All EBS volume types have equally reliable snapshot capabilities and provide 99.999% availability.

Managed by storage through Zimbra Store and Zextras Powerstore, which provide separation into storage levels and EC2 compatibility of Amazon EC2 blocks. Zextras Powerstore module allows you to manage multiple volumes and HSM policies through the Zextras Management Zimlet, which is fully integrated into the Zimbra Control Panel, which provides efficient storage management.

  • Primary volume on local drivers (GP2 General Purpouse SSD - Amazon EBS)
  • Secondary volume on Amazon S3 blocks
  • Backup on local drivers (GP2 General Purpouse SSD - Amazon EBS)
  • Compression: Primary: NO | Secondary: Auto | Backup: Yes
  • Deduplication: Primary: Periodic | Secondary: Auto | Backup: Yes
