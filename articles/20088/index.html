What is in the opera, which is not in Fox

In advance I will make a reservation - it is about Linukh and KDE, the specified behavior was observed there. Check in another OS or DM can not for lack thereof.

I've been using Opera since vinosa times. Unfortunately, recently, when more and more flash and javascript, Opera does not perform all the functions assigned to it. Therefore, it is necessary to keep parallel with it and Firefox. I watch a flash in it, in the Opera I do everything else. I hope the release of release 9.5 will correct the situation at least a little.
There are many reasons that do not allow me to switch to Fox completely (except for the fact that I simply subjectively like Opera more :)). Below I will list what causes discomfort after using the Opera.
  1. Loading addresses from the buffer
    There is some non-clickable URL. In the Opera I select it (at the same time it is automatically buffered) and I make an average click on the tabbara. A background tab opens in which the link in the buffer is loaded.
    In Fox, plug-ins like TabMix, SuperT, etc. do the same thing but!
    - loading occurs in the active tab
    - download occurs regardless of whether the tabbara was clicked or on some tab.
  2. Download url from third-party programs in the background
    Comes url, for example, in pidgin. In the settings set - open url in firefox, new tab. I click on the url. It really opens in a new tab, but at the same time Fox stubbornly pops out on top of all windows. And if you need to open several links? He clicked, alltab, clicked, alttab ... Not at all interesting.
    Of course, you can use the KDE tools to prevent the application from intercepting focus, but this is not a solution to the problem.
  3. Reopen closed tabs. Is it possible to somehow force the browser to use the usual Ctrl + Z to cancel the closure to an unlimited depth? There is a great vimperator plugin that provides a vim-like control, including canceling a close on the u button. Only here it exists only for the second version of Fox, and I prefer the third one.

Question to the community - are there any ways to solve these problems?
