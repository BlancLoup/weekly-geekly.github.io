The basic idea of ‚Äã‚Äãunobtrusive javascript



Microsoft introduced the first validation module in the first version of MVC. The module was improved with each version, and turned into unobtrusive validation in ASP.Net MVC 3.
In short, even in modern JavaScript, which we write mainly in a separate js file, we sometimes need data related to Html, and therefore we write the metadata of JavaScript objects inside the Html page. We call the function from the js file to which we pass the object metadata on the page.

One of the goals of unobtrusive javascript is to separate javascript from html markup.
You can read more about this here (the author refers us to an article about unobtrusive JavaScript in English-language Wikipedia, approx. Translator)

Understanding Unobtrusive Validation


1. The ‚Äúinput‚Äù elements, their validation rules and how they work

So let's find out what happens with the ‚Äúdata- *‚Äù attributes.

MVC creates data- * attributes based on validation rules.
jquery.validate.unobtrusive reads the rules and converts them to a path.
After the unobtrusive validation has transformed these rules, it will call the ‚Äúvalidate ()‚Äù method and pass an array of rules with their settings to validate them.

There are three parts to any unobtrusive rule for any Html element:

  • data-val = ‚Äùtrue‚Äù is just a flag denoting that an unobtrusive validation is attached to the element.
  • data-val-rulename = ‚Äùmessage‚Äù - does two things. The ‚Äúdata-val-rulename‚Äù part indicates the name of the rule, and the message text contains the validation message.
  • data-val-rulename-additional values ‚Äã‚Äã= ‚Äùvalue‚Äù to add additional values ‚Äã‚Äãneeded by the validation rule to complete validation. (For example, validation of the "range" of a range of values ‚Äã‚Äãwhen the value should be less than some value and more than another)


Let us consider in more detail the generated element ‚Äúinput‚Äù and explain the attributes ‚Äúdata- *‚Äù. 

 <input id="Name" class="ignore" type="text" name="Name" value="" data-val="true" data-val-required="*" data-val-atleastonerequired="Enter at least Name, Email or Lastname" data-val-atleastonerequired-attributes="Email,LName" />


Two rules apply to the item. One is the built-in attribute ‚Äúrequired‚Äù, and the other is its own, called ‚Äúatleastonerequired‚Äù.
When the user does not enter any data and clicks "Submit Form", "*" appears as a validation message that indicates that the element is required.
If the user violates the ‚Äúatleastonerequired‚Äù rule, his validation message will appear.

2. Validation messages and how they work in unobtrusive validation.

Let's start with an example. 

 <!-- Required input using unobtrusive validation --> <input id="FirstName" class="text-box single-line" type="text" name="FirstName" value="" data-val="true" data-val-required="The FirstName field is required." /> <!-- The input's validation message placeholder --> <span data-valmsg-replace="true" data-valmsg-for="Name" class="field-validation-valid help-inline" />

As we can see, there are two ‚Äúdata- *‚Äù elements in the container for the error message.

  • ‚ÄúData-valmsg-for‚Äù is the value of the ‚Äúname‚Äù attribute of the ‚Äúinput‚Äù element to which the container belongs. Both the input element and the container must be in the same form tag.
  • ‚ÄúData-valmsg-replace‚Äù is set to true by default. The difference between true and false values ‚Äã‚Äãis the following: if set to true, the error message in the container will change depending on which validation rule was violated, if false, it will be constant and the content will not change, only the class with ‚Äú valid to invalid.


If one of the validation rules is violated, the value of the ‚Äúclass‚Äù attribute will contain ‚Äúfield-validation-error‚Äù, and if not, then ‚Äúfield-validation-valid‚Äù.
