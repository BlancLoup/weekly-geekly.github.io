Multilingual trees in Yii2 on the example of creating a menu module

Introduction


Many novice web developers are faced with the need to create menus, directories or rubrics for their project on Yii2, which would have a hierarchical structure, but at the same time support multilingualism. The task is quite simple, but not quite obvious within the framework of this framework. There are a large number of ready-made extensions for creating tree structures (menus, catalogs, etc.), but it is rather difficult to find a solution that would support full-fledged work with several languages. And here we are talking not about the interface translation by the regular means of the framework, but about storing data in the database in several languages. It is also quite difficult to find a convenient and fully functional widget for managing the tree, which could also work with multilingual content without complicated code manipulations.


I would like to share a recipe on how to create such modules on the example of the implementation of the menu module. For example, I will use the Yii2 App Basic application template, but you can adapt everything to your template if it is different from the basic one.


Training


To accomplish the task, we will need several wonderful extensions, namely:


  • Adjacency List - for storing a tree structure
    menu in the database;
  • Yii2 Bootstrap Treeview widget
    for convenient display of the menu in the form of a tree;
  • Translateable Behavior - behavior for
    multi-language support in models;

Install these extensions via composer: 


composer require paulzi/yii2-adjacency-list composer require execut/yii2-widget-bootstraptreeview composer require creocoder/yii2-translateable

To implement the menu as a module, use the Gii generator (or manually) to create a new menu module and connect it in the application settings.


The project should also be configured mechanism for switching languages. I prefer to use this extension for Yii2 .


Creating models


To store the menu (or another entity that is multilingual) in the database we need to create two tables. In fact, different methods can be used to store multilingual data, but the variant with two tables, one of which stores the essence itself, and the second - its language variations, I like more than the others. To create tables it is convenient to use migrations. Here is an example of such a migration:

After we created the necessary tables and added a new menu to them using migration, we need to create models. Since the project is multilingual and trees can be found not only in the menu, but also in other entities (for example, pages of the site), I propose to bring out methods that implement the multilingual mechanism and organization of the tree into separate traits so that in the future we can easily use them in other models without duplicating the code. Create a traits daddy in the application root (if it isn‚Äôt there yet) and put two files there:


LangTrait.php 
 <?php namespace app\traits; use Yii; use yii\behaviors\SluggableBehavior; use creocoder\translateable\TranslateableBehavior; trait LangTrait { public static function langClass() { return self::class . 'Lang'; } public static function langTableName() { return self::tableName() . '_lang'; } public function langBehaviors($translationAttributes) { return [ 'translateable' => [ 'class' => TranslateableBehavior::class, 'translationAttributes' => $translationAttributes, 'translationRelation' => 'translations', 'translationLanguageAttribute' => 'language', ], ]; } public function transactions() { return [ self::SCENARIO_DEFAULT => self::OP_INSERT | self::OP_UPDATE, ]; } public function getLang() { return $this->hasOne(self::langClass(), ['owner_id' => 'id'])->where([self::langTableName() . '.language' => Yii::$app->language]); } public function getTranslations() { return $this->hasMany(self::langClass(), ['owner_id' => 'id']); } }

TreeTrait.php

Now in the module we will create the file /modules/menu/controllers/AdminController.php. This will be the main controller for managing the menu, and since it implements the tree and multilingualism, it will inherit from the base one, which we have already created in the previous step:


AdminController.php 
 <?php namespace app\modules\menu\controllers; use app\base\controllers\AdminLangTreeController as BaseController; class AdminController extends BaseController { public $modelClass = \app\modules\menu\models\Menu::class; public $modelClassSearch = \app\modules\menu\models\MenuSearch::class; public $modelName = 'Menu'; public $modelNameLang = 'MenuLang'; }

As you can see, the code of this controller contains only the names of models and their classes. Ie, to create CRUD controllers of other modules (catalog, rubricator, etc.), which will also use the tree and multilingualism, you can do the same thing - expand the base controller.
