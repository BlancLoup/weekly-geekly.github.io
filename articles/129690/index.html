BitBucket Service Update - Git Support with Mercurial and More



The service for hosting projects and their joint development BitBucket has undergone significant changes.

Changes:
  • Git version control system support. Previously, only Mercurial was supported. Improved ability to import projects from other services (for example, CodePlex, GitHub, Google Code, SourceForge);
  • Simplified copying operations (forks) of repositories, making changes and merging branches of project versions. Added the ability to discuss change packets between project participants;
  • Integration with email, Twitter, Jenkins, JIRA, Pivotal, Lighthouse. REST API for the possibility of developing your own integration with the service;
  • Improvements in user interface.

The main advantage of the service remains: it is possible to create an unlimited number of public repositories and an unlimited number of private repositories (for your commercial projects). There are no disk space limitations!

The main limitation also remains in force: no more than 5 people can work on each specific private project. For more people in the project there are paid rates .
