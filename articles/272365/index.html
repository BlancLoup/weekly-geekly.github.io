DreemChest: Chain Reaction

About transfer


Hello to all. I decided to do the translation of the documentation for the game engine DreemChest, I don’t know if someone did it to me, but I hope not.
Why did I decide to start with the translation of "Chain Reaction"? The answer is simple: this is the most interesting lesson at the same time as the practical part of all the documentation.
Well, let's start.

Chain reaction


In this lesson, you will see an example of a simple chain reaction game (The name corresponds to the genre). The meaning of the game is that the player needs to destroy constantly appearing objects by clicking on them with the mouse, as quickly as possible.
The foundation

Run the engine and create a new project. When it is launched, you will see a dialog box that prompts you to create a new project or to load an already created one. Enter the project name and click on the “Greate” button.


By standard, the editor has only one game screen resolution (320x480), but you can always add your own. Let's give our project a new resolution, for example 640x480, to do this, open the pull-down menu (Well, honestly, did not know how to translate, correct, if that), which is shown in the image below in the main toolbar and select the option "Edit".

Now you can see the dialog box that has just appeared. To add a new resolution, enter the width and height, edit in the first and second window, and then click on the “Add” button. To close the dialog box, click on the “Close” button, to remove the permission, click on it, thereby selecting it and click on the “Remove” button.

Now that the new resolution has been added, you can select it in the pull-down menu, if you select a different resolution, the borders of the scene will be changed.

Adding resources

A standard project contains only one htcehc with the name “root”, which is the only game level (scene). Let's now add some sprites, which we will use in the future.
To import a resource, click the RMB (Right mouse button) on the “Assest” panel and in the menu that appears just select “Import Assest”. We see a window appeared through which we will select and add our sprites. Let's add 2 sprites "Enemy.png" ("Enemy") and "Bullet.png" ("Bullet").
The “Assest” panel is located in the lower right corner, if you did not find it because it is not there, you can always add it through “View” -> “Assests”

The Assest panel allows you to create resources (script, shape, stage object, etc.), as well as import ready resources (Sprites, flash menus and sounds / music). To make a prototype of the future game, we need to register the game logic, let's start by creating a script. Right-click on the Asssest panel and in the menu that appears just select “Create” -> “Script”.

Before writing the script, let's select the “bullet” and “enemy” resources. The image loaded by us can be dragged onto our scene, but it cannot be called through the script, unless this script is the main class of the object. So that we can create an object from a script, let's set a unique name for our object class, for “bullet” it will be “bmpBullet”, and for “enemy” “bmpEnemy”, we can always change the name of the object class to another or write it for the first time object in the “Assest” panel, and then in the “Properties” panel in the “Class” line, enter the name of the main object class.
The class of our resource can be called in any other resource, or attached to another object, but at the moment we are interested in working with scripts.

Let's start writing our script. The prototype of our script will contain 1 class and 8 methods.
Main - Method called at the start of the game. Each main class must contain this method.
spawnEnemies - Creates a certain number of Enemy objects.
spawnEnemy - Creates a single "Enemy" object on the stage with a built-in sprite.
shoot - Creates 4 bullets at specific coordinates.
spawnBullet - Creates a single “Bullet” object on the stage with an embedded sprite.
onUpdate - Updates objects (Coordinates, status, etc.).
updateBullet - Allows Bullet objects to move around the scene and checks for collisions with Enemy objects.
updateEnemy - Allows Enemy objects to move around the scene.

The script of the prototype of our game is ready. But why, when the game starts, nothing happens (to start the game you need to click the "Start" button on the main toolbar)? This is because our script is not standard when you start the game. To fix this, open the drop-down menu next to the “Start” button and select our script.

Testing and exporting

The launch function of the project was done in order to save the time we spend on compilation in order to test what happened with us. To pause the game launched in the editor, you need to click on the Stop button, which replaces the Start button when the game starts. If you edited the project and started it, but nothing has changed, then you forgot to save it (Ctrl + S), only after saving the changes they will be visible when the game starts.

Let's run our game.

To separate our project from the engine and export it to a specific platform, select the “Export” item in the “File” drop-down menu (“File” -> “Export”). There you can also configure many options for exporting. (PS Exporting takes longer than just launching a project and much longer).
