Printing on any paper size in Linux

Some time ago I needed to print a document on a non-standard size paper. I did not even expect this to be so difficult. Now I would like to tell you how I solved this problem. I found a workaround, but in a good way, I need to fix the print module kde so that no crutches are needed. The purpose of this article is to share the information that I have gathered. At least it will be useful to someone, but as a maximum - there will be a person who will correct this situation.

Anyone interested, please under the cat.

So, I need to print a document that is smaller than standard A4 paper. Suppose the paper size on which I want to print is 10 by 20 cm. Let me create a document with such a paper size in the LibreOffice Writer. Go to the menu Format ‚Üí Page and set the desired size. Save the document. After I prepared the document, I saved it and put the paper of the same size in the printer tray. But, as it turned out when sending to print, I can not set an arbitrary paper size. LibreOffice has its own dialog interface, which I will discuss at the end.

Now we just export to pdf and print it with Okular - this is the standard pdf viewer surrounded by kde.

screenshots okular



As you can see, from those sizes that are offered in the list, nothing suits me. And to set your own arbitrary size is not possible (the option ‚Äúspecial‚Äù / ‚Äúcustom‚Äù is present, although for some reason it is not selected).

I began to google how to be in such a situation. I found a video of how such a problem is solved on windows. There is a very obscure way, through a lot of clicks in the windows you need to get to a certain field, which without this video you just can not find. But, nevertheless, there is such an opportunity on Windows.

What about Linux? As long as I did not search, I did not find anything like it.

Of course, such a crutch immediately comes to mind as the re-creation of a document in such a way that it is located on the top of a page of usual size A4. And when printing it just would get to the right place. But first, for this, you have to tinker with formatting the document again, and secondly, it‚Äôs interesting to figure out how to do it in a normal way. I began to understand, and came to the conclusion that I need to rewrite the printer driver.

Each printer has its own so-called driver - ppd file. This is essentially just a text file with a specific syntax that describes the capabilities of the printer, including which paper sizes it supports.

Let's find the file that describes my printer. I have a printer from the company hp and I installed a special package (hplip) in the system, which contains drivers for their printers. In my case, the drivers are in / usr / share / ppd / HP.

But before we enter the value of the width and height of our paper, we must specify the features associated with the units of measurement that are used in the ppd files. Unfortunately, you can not just enter the size in millimeters.

These files use their own units of measurement - postscript points. Each such point is 1/72 of an inch. We are also interested in millimeters. Let's calculate the coefficient by which we will multiply our millimeters for conversion to postscript points. In inch 25.4 mm. Expressing millimeters, we get a coefficient equal to 72 / 25.4, i.e. approximately 2,834. We translate the length and width of our paper size from mm to points:

100 * 2.834 = 283.4 points,
200 * 2,834 = 566.8 points.

Thus, in the language PostScript paper size will be described by the numbers 283 and 566.

Great, now we need to add our paper description to this file. This should be done in four places. All by analogy with existing formats, just append your own.
PageSize, PageRegion, ImageableArea, PaperDimension.

I haven‚Äôt studied in detail what these parameters describe, but, as you can see, everywhere except for the ImageableArea parameter, the same sizes are indicated. And in the rows of ImageableArea two times in a row goes the number 12, and then the paper size, reduced by 12 points. I guess this parameter is responsible for the zone in which the printer is capable of printing. If you translate 12 points in mm, you get about 4.2 mm. Most likely - this space is overlapped by wheels with which the printer pulls in the paper. Let's enter our values, which we also reduce by 12. Now save the file.
