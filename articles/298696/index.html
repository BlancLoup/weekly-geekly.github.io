And where, if not on the sidelines of events get a "dose" of useful content? I received all the answers to my questions precisely in the intervals between presentations from Sergey Halyapin . Sergey gave me useful comments on pumping my service.

- How to improve SaaS (browser services)? Does it make sense to improve - such services are available from any form factor from the public cloud?

The problem is actually very simple and many people have come across it - under one browser the site opens and works, under another we see only part of the content, and the third does not open the site at all. In the corporate environment, a number of web applications from the point of view of technical support are generally very tightly tied to a certain type of Internet browser (including the version). With the help of the new feature, when publishing links to internal or external web sites, it will be possible to specify which browser they should open.

As a result, on the access portal, the end user sees the application icons and when they start up, the administrator-defined browser will open and the corresponding website will open in it.

- We conditionally build our public cloud on a) hardware b) in the provider's virtual environment - what products will help me and in short how?

Depending on what is going to be deployed in the cloud. If you need virtual applications or desktops, then XenApp and XenDesktop can be integrated with CloudPlatform , AWS and Azure out of the box. In the case of other cloud infrastructures, you will need to install within virtual machines, but cloud services (portals, etc.) will not be available.
You can use XenServer including the available as a free product (but there will be no possibility to purchase technical support).

You can use the Citrix Cloud Bridge to connect the clouds to each other or to connect the cloud with the corporate environment to optimize the transfer of data over the WAN channels, as well as to form the IPSec VPN tunnel.

If web servers are deployed in the public cloud, then Citrix NetScaler is recommended to use a virtual or physical device for balancing and protecting traffic (SSL VPN and / or Web Application Firewall), or now you can also use container technologies - NetScaler CPX containers.
