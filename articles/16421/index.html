Denwer 3 alpha

Yesterday there was a joyful event for me. Dmitry Koterov, the developer and founding father of the Denver project, announced the release of an alpha version of the next, third, Denver project. I already thought that the project had safely died down, however, as it turned out, no.

Currently, the differences from Denver 2 are as follows:
  1. SSL support (apache 2.2).
  2. PHP5 by default (+ sqlite, php_mysql, php_pdo, php_gd2 modules).
  3. MySQL5 is default.
  4. Perl is no longer included in the base package (it will go in the full package
    extensions in the / usr / local / perl directory).
  5. Compatible with Windows Vista (including troubleshooting
    read-only file hosts).
  6. Explicit separation of OpenSource components and Denver files.
  7. Extended debug information output in case of installation error.

The author points out that the Denver-2 extensions packs WILL NOT HAPPEN for Denver-3!
