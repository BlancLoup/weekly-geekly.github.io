Troubleshooting xdebug with php version 5.3

Who uses the Netbeans editor for PHP development and the installed version of the interpreter at it> 5.3, probably noticed that during the debugging sessions the variables in the ‚ÄúVariables‚Äù editor area are not displayed. As already written by the user @ gray-one here , the beta version of debugger 2.1.0 has been released (the second one is already). For verification, I installed all of this with svn (since up to now macports have only version 2.0.5 without php5.3 support):

# cd /tmp
# svn co svn://svn.xdebug.org/svn/xdebug/xdebug/trunk xdebug
# cd xdebug
# phpize
# ./configure --enable-xdebug
# cp modules/xdebug.so ////


and add a line to your ini-file:
zend_extension="/////xdebug.so"


Well, restart the server:
# /opt/local/apache2/bin/apachectl graceful


While beta, sometimes I noticed ‚ÄúSocket exception occured‚Äù errors at the end of a debugging session. We use at your own risk :)

I give an example of a typical netbeans + xdebug setup
