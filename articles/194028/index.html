Reverse Thread Bolt for .slip Embarcadero RAD Studuo XE

Advice on how not to exceed the number of licenses allowed for one registered copy of Embarcsdero RAD Studio XE. It may be useful for system administrators and project managers who are developing on Embarcadero RAD Studio XE to read.
Tested on Delphi XE3.


Who is guilty?


Your organization is a happy owner of Embsrcsdero RAD Studui XE licenses for each developer. Of course, if the strict rules for identifying the workstation and development users are followed, then there is no reason to fear that the license will fall off from Embarcadero. It's simple: one machine - one license with a certain limit of the number of registrations for one key.
But life is a slightly different thing. As they say, if the gun is hanging on the wall, then someday it will definitely shoot. Who is to blame that the station name and / or developer account can change? Probably, the one who quits, or the one who forgot about the rules and out of habit began to consider that his computer is his possession with his charter and rules.
In short, there was a failure at the developer station. For the first time, RAD Stidio will forgive you and confirm re-registration. But, there will be no concession for the second time. Calls and other will be required (hereinafter - medical vocabulary).

What to do?


Briefly by points:
1. After installing and registering Embarcadero RAD Studio XE, save the key and the .slip file in a separate storage. For example, identify the folder on the server as the name of the workstation and the user. I leave the identification method at your discretion. The main thing is to copy the .slip file to another location.
The file location for Windows 7 and Windows 8 is the ProgramData directory. For example, C: \ ProgramData \ Embarcadero . You will see a file starting with a dot and having the extension .slip . Something like .2 *** _ *. 13 ********. Slip
Copy it somewhere in the repository.
2. After the next launch of the Embarcader RAD Studio on behalf of the new user informs the creative partner that the number of registrations has exceeded the allowed number, do:
2.1. Close Studio.
2.2. Copy the saved .slip file to the C: \ ProgramData \ Embarcadero directory.
2.3. Delete in this ( C: \ ProgramData \ Embarcadero ) directory the .licenses folder and all files matching the .cgb_license template *
2.4. Copy the saved .slip file to the folder with the installed RAD Studio XE in the folder License . For example, C: \ Program Files \ Embarcadero \ RAD Studio \ 1 * .0 \ License . For amd64, the folder may look like C: \ Program Files (x86) \ Embarcadero \ RAD Studio \ 1 * .0 \ License
2.5. Delete the file RADStudioXE * .slip in this folder
2.6. Rename the copied .slip file to RADStudioXE * .slip
2.7. Launch RAD Studio XE and confirm the license.

Work on what you paid for.

Good luck! :)
