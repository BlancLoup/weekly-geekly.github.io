Project management for a blogger

Michael Martine gives practical advice on how to plan projects for a blogger. By projects he understands: creating a new blog, developing a theme for Wordpress or moving from blogspot.com to your own domain and Wordpress.

Stages of the project

Typically, projects consist of basic steps:

1. Start
2. Planning
3. Implementation
4. Completion


Stage One: Start

1. Determine which problem the project should solve.
2. Define project objectives
3. To determine the main tasks that must be performed for the project
4. Set a project end date
5. Determine the scope of the project (what to include and what not)

Stage Two: Planning

1. What to do (task list)
2. How much money can and should be spent (budget)
3. What people and means are needed to complete the project (resources)

Stage Three: Implementation

At this stage, it is necessary to do the tasks and mark the completed ones. In your head you need to keep at least the following:

1. Do you meet the time and budget
2. Is everything taken into account
3. What has been done and what remains to be done
4. Any problems that have suddenly arisen and spoiled your plans.

Stage Four: Completion

If you are involved in blogging projects, then completing a project may include:

1. Plugin Testing
2. Testing various types of content in the frame of the chosen topic or template
3. Subscribe to feeds to check their work
4. Search Testing
5. Testing of non-existent pages, i.e. 404 error
6. Testing comments from an unregistered user
7. Testing software integrated into a blog, for example, a forum
8. Beer with pistachios, champagne! You deserve this holiday!

From each project it is necessary to learn the lessons and take into account all the accumulated experience - this will not be useful in the following projects.
