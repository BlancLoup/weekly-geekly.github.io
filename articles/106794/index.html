Drupal 7.0 Beta 2 has been released. There are only eight bugs left until RC

Congratulations to all with the release of Drupal 7.0 Beta 2!

New in this release:



  • Improved "catching" errors during installation and updates in various incompatible systems
  • Improved support for exported items in a new type of machine_name form in the API. Format IDs are now stored as strings instead of as integers.
  • Links can now participate in the AJAX Drupal 7 framework. Previously, this was allowed only by forms.
  • Minimum PHP version requirements increased to PHP 5.2.4 due to security issues with streaming handlers.
  • A large number of changes in the default profile navigation in order to overload the user information as little as possible
  • Fixed regression in the database schema API, due to which modules like Date could not add new data types to the supported kernel
  • The profile module is no longer shown in the list of modules on new sites. It is replaced by custom user properties.
  • Fixed numerous bugs in Bartik and Seven themes, as well as RTL bugs. Although there are still a lot of them there.
