Monitoring of servers from Infobox has become more convenient: notifications, monitoring of arbitrary ports, correct work with firewalls

We recently launched a server availability monitoring tool. It allows you to perform periodic requests to the server and assess the availability of servers for the required period of time.

The tool is available to users of the cloud infrastructure InfoboxCloud and VPS from Infobox on the main control panel page:



We are pleased to announce the release of the update tool, taking into account user feedback.

Below we take a closer look at all the innovations and the process of using the monitoring tool.

What's new


  • Added the ability to monitor an arbitrary port.
  • Added the ability to enable notification of unavailability by mail
  • Added information about which address is being monitored.
  • When entering the IP address, the system now offers to select one of the addresses used in the current account.
  • Added filter by status in monitoring history
  • Default display interval changed to 1 day

Overview


Let's look at the monitoring tool in action and see innovations in the process.

Add the server to the monitoring. When adding, ip-addresses of servers on the current account ( new ) are suggested. Of course, you can add a third-party address.



Now you can immediately specify the email-address for sending notifications about the server status change ( new ).



Add the necessary monitoring rules.



Available rules:
  • ICMP (ping)
  • FTP (21)
  • SMTP (25)
  • Web server (80)
  • HTTPS Web Server (443)
  • MySQL server (3306)
  • The ability to specify the port manually ( new )
