IE11: Compatibility List vs. X-UA-Compatible

Yesterday I published a topic in which I described some amazing decisions that Microsoft made when releasing new versions of its Internet Explorer browser. In short, these are:
  • the introduction of compatibility modes with all previously released versions of the browser up to 5.5;
  • the decision to emulate this compatibility by carefully porting old bugs;
  • Refusing to specify the browser name in the user agent against the background of claims that IE11 should not be detected by the user agent;
  • scrapping backward compatibility between the latest Developer Preview and RTM release;
  • different behavior, up to and including the fall in some situations, of the same browser under different versions of the operating system;
  • maintaining a global compatibility list, which includes those sites that, according to Microsoft, should be shown in one of the IE compatibility modes;
  • the formation of this list on the basis of statistics on user clicks of preview versions of the browser in the ‚ÄúCompatibility View‚Äù button;
  • the priority of this list over the webmaster's specified X-UA-Compatible meta in RTM build IE11.
