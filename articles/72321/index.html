apt get --torrent

Idea for debian and ubuntu developers. Why not cross apt-get with a torrent client. This will result in partial independence from the central server with repositories. Thus, the load on the channel of the server with repositories will be significantly reduced at the time of release of releases.

Let the server send a torrent file with all packages from the repository on request to apt-get. apt-get in torrent mode the client downloads only the requested ones, installs and starts distributing packets from its cache. When cleaning the cache distribution stops.
