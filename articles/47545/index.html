Recursive patterns in XSLT

Hello to all!
I want to talk about the use of recursive templates in XSLT, as many beginners working with XSLT are faced with tasks that require their use and do not know how to solve such problems.
Take a couple of common examples:
1. There is a node with a string, it is necessary to split it into parts according to a certain character (in our case, take a space character) and decorate each part in different colors.
2. Make a page number (pager) output based on the fact that we know the total number of objects (for example, forum topics), the number of objects on a page and the page number on which we are currently located.

It is necessary to break the line inside the string node by spaces and each odd element to display in green, and even every element in red.
XSLT / XPATH 2.0 has a great tokenize function that can break a line into parts and, accordingly, with xsl: for-each, we can run over it and do everything we want with each part. But XSLT 2.0, as far as I know, is well supported only by the Saxon processor. The embedded XSLT processors of browsers and libxslt in PHP do not have its support, so we will use a recursive template.
  1. <? xml version = '1.0' ? >
  2. < xsl: stylesheet version = "1.0" xmlns: xsl = "http://www.w3.org/1999/XSL/Transform" >
  4. <! - you can match and just string,
  5. then the contents of the node will be converted to text
  6. but if there are more nodes inside the string (for example, <br />),
  7. we can get an error
  8. Do not forget that if we are going to work with the text,
  9. you need to match the text,
  10. make your code work obvious ->
  11. < xsl: template match = "string / text ()" >
  12. < xsl: call-template name = "colorer" >
  13. < xsl: with-param name = "text" select = "." />
  14. <! - the remaining values ‚Äã‚Äãwill be taken from the default values ‚Äã‚Äã->
  15. </ xsl: call-template >
  16. </ xsl: template >
  18. < xsl: template name = "colorer" >
  19. < xsl: param name = "text" />
  20. <! - default delimiter - space ->
  21. < xsl: param name = "delimeter" select = "''" />
  22. <! - by default the element is colored as odd ->
  23. < xsl: param name = "even" select = "false" />
  24. < xsl: variable name = "color" >
  25. < xsl: choose >
  26. < xsl: when test = "$ even" >
  27. < xsl: text > red </ xsl: text >
  28. </ xsl: when >
  29. < xsl: otherwise >
  30. < xsl: text > green </ xsl: text >
  31. </ xsl: otherwise >
  32. </ xsl: choose >
  33. </ xsl: variable >
  34. < xsl: choose >
  35. <! - if the line contains a separator ->
  36. < xsl: when test = "contains ($ text, $ delimeter)" >
  37. <! - then print the string before the delimiter ->
  38. < span class = "{$ color}" > < xsl: value-of select = "substring-before ($ text, $ delimeter)" /> </ span >
  39. <! - and once again call the template for the remaining line ->
  40. < xsl: call-template name = "colorer" >
  41. < xsl: with-param name = "delimeter" select = "$ delimeter" />
  42. < xsl: with-param name = "even" select = "not ($ even)" />
  43. < xsl: with-param name = "text" select = "substring-after ($ text, $ delimeter)" />
  44. </ xsl: call-template >
  45. </ xsl: when >
  46. < xsl: otherwise >
  47. < span class = "{$ color}" > < xsl: value-of select = "$ text" /> </ span >
  48. </ xsl: otherwise >
  49. </ xsl: choose >
  50. </ xsl: template >
  52. </ xsl: stylesheet >

It is necessary to display page numbers (pager). Before and after the page numbers you need to display the transition to the previous and next page, provided that we are not on the first or the last page. We also set a condition that it is necessary to display only links to go to several nearby pages, and not to all at once, because then the pager can turn out very large.
We get this pretty big template:
  1. <? xml version = "1.0" ? >
  2. < xsl: stylesheet version = "1.0" xmlns: xsl = "http://www.w3.org/1999/XSL/Transform" >
  4. < xsl: template match = "pages" >
  5. < xsl: call-template name = "page-numbers" >
  6. < xsl: with-param name = "total-results" select = "topics / @ count" />
  7. < xsl: with-param name = "results-per-page" select = "topics-per-page / @ count" />
  8. < xsl: with-param name = "max-from-current-page" select = "3" />
  9. < xsl: with-param name = "current-page" select = "current-page / @ number" />
  10. < xsl: with-param name = "href" select = "link / @ href" />
  11. </ xsl: call-template >
  12. </ xsl: template >
  14. < xsl: template name = "page-numbers" >
  15. < xsl: param name = "total-results" />
  16. < xsl: param name = "results-per-page" />
  17. < xsl: param name = "max-from-current-page" />
  18. < xsl: param name = "current-page" />
  19. < xsl: param name = "href" />
  22. <! - How many pages we have ->
  23. < xsl: variable name = "max-page" select = "ceiling ($ total-results div $ results-per-page)" />
  25. <! - If there are more than one page, then print the page numbers ->
  26. < xsl: if test = "1 <$ max-page" >
  28. <! - From which page to start displaying page numbers ->
  29. < xsl: variable name = "from-page" >
  30. < xsl: choose >
  31. <! - If the number of the current page is greater than the maximum distance ->
  32. < xsl: when test = "$ current-page> $ max-from-current-page" >
  33. <! - This will be the first page that is deleted a specified number of pages from the current one ->
  34. < xsl: value-of select = "$ current-page - $ max-from-current-page" />
  35. </ xsl: when >
  36. < xsl: otherwise > 1 </ xsl: otherwise >
  37. </ xsl: choose >
  38. </ xsl: variable >
  40. <! - Which page to finish printing page numbers ->
  41. < xsl: variable name = "to-page" >
  42. < xsl: choose >
  43. <! - If the number of the current page is removed from the last page number more than the maximum distance ->
  44. < xsl: when test = "$ max-page - $ current-page> $ max-from-current-page" >
  45. <! - Then the last page will be deleted for a specified number of pages from the current one ->
  46. < xsl: value-of select = "$ current-page + $ max-from-current-page" />
  47. </ xsl: when >
  48. < xsl: otherwise >
  49. < xsl: value-of select = "$ max-page" />
  50. </ xsl: otherwise >
  51. </ xsl: choose >
  52. </ xsl: variable >
  54. <! - If the current page is not the first, then output arrows with a link to the previous page ->
  55. < xsl: if test = "1! = $ current-page" >
  56. <a href = "{$ href} {$ current-page - 1}"> <<</ a >
  57. < xsl: text > </ xsl: text >
  58. </ xsl: if >
  60. <! - Call the page numbering template with initial values ‚Äã‚Äã->
  61. < xsl: call-template name = "page-number" >
  62. < xsl: with-param name = "max-page-number" select = "$ to-page" />
  63. < xsl: with-param name = "current-number" select = "$ from-page" />
  64. < xsl: with-param name = "current-page" select = "$ current-page" />
  65. < xsl: with-param name = "href" select = "$ href" />
  66. </ xsl: call-template >
  68. <! - If the current page is not the last, then output arrows with a link to the next page ->
  69. < xsl: if test = "$ max-page! = $ current-page" >
  70. < xsl: text > </ xsl: text >
  71. <a href = "{$ href} {$ current-page + 1}"> >> </ a >
  72. </ xsl: if >
  73. </ xsl: if >
  74. </ xsl: template >
  76. < xsl: template name = "page-number" >
  77. < xsl: param name = "max-page-number" />
  78. < xsl: param name = "current-number" />
  79. < xsl: param name = "current-page" />
  80. < xsl: param name = "href" />
  82. < xsl: choose >
  83. <! - If we display the number of the current page, then without reference ->
  84. < xsl: when test = "$ current-number = $ current-page" >
  85. < xsl: value-of select = "$ current-number" />
  86. </ xsl: when >
  87. <! - Numbers of other pages with a link ->
  88. < xsl: otherwise >
  89. <a href = "{$ href} {$ current-number}">
  90. < xsl: value-of select = "$ current-number" />
  91. </ a >
  92. </ xsl: otherwise >
  93. </ xsl: choose >
  95. <! - If the current number is not the last one, then we call the template for the output of the next number ->
  96. < xsl: if test = "$ current-number <$ max-page-number" >
  97. < xsl: text > | </ xsl: text >
  98. < xsl: call-template name = "page-number" >
  99. < xsl: with-param name = "max-page-number" select = "$ max-page-number" />
  100. < xsl: with-param name = "current-number" select = "$ current-number + 1" />
  101. < xsl: with-param name = "current-page" select = "$ current-page" />
  102. < xsl: with-param name = "href" select = "$ href" />
  103. </ xsl: call-template >
  104. </ xsl: if >
  105. </ xsl: template >
  107. < xsl: template match = "@ * | node ()" >
  108. < xsl: copy >
  109. < xsl: apply-templates select = "@ * | node ()" />
  110. </ xsl: copy >
  111. </ xsl: template >
