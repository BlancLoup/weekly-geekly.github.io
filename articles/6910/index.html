Mobile ads 2.0

Mobile 2.0 application - classified for mobile users. The initial idea: a mobile network subscriber should be able to place an arbitrary advertisement on the network and receive responses to it. The possibility of placing an ad must be attendance - the subscriber should not be registered anywhere, enter into some kind of social network, a closed club, etc.

What have we done? As it should be in the dany tm era, the whole service is a big mashup. Everything is assembled from ready-made pieces, based on the APIs they have.


1. System users post ads by SMS. Announcements are received by the GSM modem (real GSM phone and data cable ) and are processed by the SMS gateway . Accordingly, the ads are sent as regular SMS to a regular number (non premium rate). The cost of placing an ad is the cost of sending an SMS at the rate of the author of the ad.

2. Received ads need to be placed somewhere. You can make your site, but you can, so long as we have a mashup, take an open API of the finished platform and place ads there. In our case, we used the API Livejournal.com and made the publication of ads in a journal. At the same time, the problem of viewing ads is solved - you can simply view them in the friends feed if you use LiveJournal or subscribe to the RSS feed of this magazine and view the ads in another convenient way.

3. To post an ad, no registration is required. You just need to send SMS to some dedicated number. For feedback, it is not appropriate to directly indicate the number of the subscriber who sent the advertisement - suddenly the author is against. Therefore, we used an anonymous chat system based on Voice SMS . Each published advertisement is automatically supplied with a code, using which you can send a voice message to the author. And the author's phone does not appear anywhere.

What does this look like in the end? It is here: http://mobnews.livejournal.com All instructions for use are given in the very first message of the journal. Announcement is placed in the journal by sending SMS, each advert being placed is automatically supplied with a code for communication with the author through the Voice SMS system.

Naturally, you can easily multiply the system into several magazines - by topic. As well as replacing the Livejournal API with something else. For example, if the administration (or users) of liveinternet.ru needs such a service, we can start it.

One more thing. Recently, many discussions have been devoted to the transformation of traditional media in the era of Web 2.0. The declarations with safe feedback described above are one of the possible examples of transformation. After all, one can also reprint them in the form of text from a network in a newspaper. And it is possible to add a voice communication channel to newspaper texts on similar principles.

Naturally, the critical point for the entire 2.0 story is the presence of open interfaces to the telecommunications environment. According to an extremist remark in one of the discussions at the Emerging Telecom conference, a modern telecom operator should be one big API. While this, of course, is not the case. In this project, the telecommunications component was implemented on the basis of CTI products of the company Audiotel .
