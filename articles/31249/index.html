My first Linux virus?

Translation notes My first Linux virus? for Linux in digg in Russian .

Wandering about the Linux file system on my computer, I found a strange directory in / home ...
drwxr-xr-x 2 root root 4096 2007-08-19 12:03 eb588afc0325b12eeb074fd6

I did not create this directory. If it is a virus, then it is the most dull virus in the universe, but anything can happen. Then I looked inside the directory and saw the following files:
$ l eb588afc0325b12eeb074fd6/

total 956

-rw-r--r-- 1 root root 865822 2007-08-02 21:41 mrt.exe._p

-rw-r--r-- 1 root root 96216 2007-08-02 21:34 mrtstub.exe

-rw-r--r-- 1 root root 45057 2007-08-19 12:03 $shtdwn$.req

My God, if this is really a virus, then it is even dumber than I thought: trying to inject .exe files into my Linux system ! In any case, someone must know the answer ... Search by mrtstub issued this page , directly from the enemy site. Pretty quickly, I found the answer:
The mrtstub is part of the Malicious Software Removal Tool. Responsible for copying and running mrt.exe.

In short: I have two systems installed on my computer: Linux and Microsoft Windows (my son sometimes uses for games), and my Linux home directory is mounted using an ext3 driver in Windows. The system asked me about installing this Malicious Software Removal Tool , which I refused 10 times, and asked every damn thing to NEVER INSTALL . On the eleventh time, my son also banned the installation of the Malicious Software Removal Tool (he always does this), but Windows silently, without permission, installed this piece of shit in my computer.

I'm over it! I will spend the top five and buy CrossOver to run games on Linux, once and for all getting rid of Windows.
