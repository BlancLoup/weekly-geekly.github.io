In this build, we present a small but important innovation in the interface.
In a few words: Kesterl received support for EV, the address field changed the way the page information was displayed, and now we have automatic renewal of certificates!

This latest build supports Extended Validation (EV) certificates .
This means that on EV sites you will see a lock in the address bar on a green background.
And remember: The site is not EV until it is all EV. Opera is stricter than other browsers and displays a ‚Äúgreen lock‚Äù on the EV site only if all page elements are on the EV sites. See Yngve for more details.

A few more changes to the address bar: the fraud protection button now works a little differently. On HTTP sites you will not see it anymore. On HTTPS sites there will be a lock on a yellow background (on a green - EV). If you entered via the HTTPS website, which reduces the overall security level of the page (for details on security levels, see the Yngve article ), you will see a gray "?" Button that will show you more detailed information about the security level. Fraudulent pages will still be signaled by the red field of the address bar.
Changes in this build has also undergone a database of certificates: now we can add a new Root certificate without the need to install it by the user.
Details you will find in the article Yngve .
