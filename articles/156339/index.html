How to get drivers for old Mobility Radeon cards on Windows 8

Relatively recently, AMD released a special driver package for Windows 8, AMD Catalyst Legacy 12.6 ( link to the news on the AMD website ), but it did not work on a laptop with the Mobility Radeon HD 3400 (this does not mean that it will not work for you - check it before proceeding with the procedure described in this topic).

Having visited the site, I discovered that in the section of old mobile maps, Windows 8 is not mentioned at all. You can start a driver for Windows 7. Do you need it at all, this driver is from the manufacturer, if the eight has a built-in one that works quite well - you decide. But I‚Äôll say right away that Catalyst Control Center (a graphical configuration utility) may not work, alas. How to get a driver for Windows 7, I will tell under the cut.

  1. Go to the AMD site and download the latest driver for Windows 7 of your architecture.
  2. Open the installation of components (windows + X -> Control Panel -> Programs and Components -> Install Windows Components), select the .NET Framework 3.5 component (now it can not be downloaded from the site, and by default it is disabled, it turns on that way. There is also quite An inconvenient way to install it offline, from the installation disk, if someone really needs it - I will try to remember and write in the comments).
  3. We are trying to install the driver downloaded from the AMD site. It should be installed without errors (in his words), but immediately after clicking on the final OK, we understand that in fact nothing has changed, the system still uses the driver from Microsoft. Important: although the driver did not work, install it (or at least unpack it somewhere on the hard disk) - it is necessary!
  4. Download and install the modder: http://www.hardwareheaven.com/modtool.php
  5. If you run the modder now, it will crash for an unknown reason, no matter what compatibility options we set for it. Therefore, download the file by reference, unpack, and replace the eponymous exe file with it in the directory where the modder was installed: http://www.driverheaven.net/driverheaven-tools-discussion/178628-possible-fix-9-2- later.html .
  6. Yeah, now you can run this executable, tell it the path to the folder where the official drivers are unpacked (I have C: \ AMD \ Support \ 12-6-legacy_vista_win7_32_dd_ccc_whql), click the modify button and wait a bit.
  7. Now we go to C: \ AMD \ Support \ 12-6-legacy_vista_win7_32_dd_ccc_whql \ Packages \ Drivers \ Display \ W7_INF, or where you unpacked the official drivers there. We see a bunch of files there. We are interested in those that end in * .inf, they can be installed. Installation is performed by clicking the right mouse button on the file and selecting the "install" item. The names of the files in this directory may be different, for example, my driver was called CL145534.inf. And not all the infs can be installed there, in my case CW145534.inf lying in the same place was not installed. Indicator of a successful installation can be considered the appearance of a dialog box that says "successfully installed." If your driver does not want to be installed due to non-signing, see clause 8, otherwise skip it.
  8. This item is needed if Windows refuses to install the driver due to non-signing. He will also reward us with the eternal inscription "Test Mode" in the right-bottom corner of the desktop (if someone can tell in the comments how to remove this inscription, it would be awesome)
    1. We open the command line in the mode of elevated privileges (open the new start menu, type cmd on the keyboard, right-click on the Command Line icon, select ‚Äúrun as administrator‚Äù in the pop-up menu at the bottom.
    2. We type the following commands there: 
      bcdedit /set {current} testsigning on bcdedit /set {current} nointegritychecks true
    3. Press on the keyboard (windows button) + R, enter gpedit.msc there and press Enter. In the window that opens, follow the path User Configuration -> Administrative Templates -> System -> Driver Installation, in the right pane, double-click on the item ‚ÄúDigital Signature of Device Drivers‚Äù, select ‚ÄúOff‚Äù and click OK
    4. Restarting the computer and returning to point 7. (Administrators and experienced Windows users here can remember that there is a gpupdate command and two spaces after it, which allows you to instantly update policies, so it will not work, you really need to reboot)

  9. We open properties of drivers of the video card. (windows + X -> Device Manager -> Video Adapters -> Your Video Card (ATI Mobility Radeon HD 3400 Series) -> right-click and select ‚Äúproperties‚Äù -> Go to the tab ‚ÄúDriver‚Äù). We need not this one, we need one that is called as your card, for example, I have the following: ATI Mobility Radeon HD 3400 Series. If there is nothing there, go back to point 7 and look for some other inf that you forgot to try to install. After selecting the driver, click next, then OK, until the installation is complete, and after the installation is completed, we reboot.
  10. After rebooting, we make sure that the driver is really the one (we open the properties of the video card drivers, as was done in the previous paragraph, in the ‚Äúvendor‚Äù field the name should change from Microsoft to Advanced Micro Devices).
