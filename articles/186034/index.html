Parallels Plesk Panel 11.5: what has appeared in the new version

Hello! The new version of the control panel for hosters Parallels Plesk Panel 11.5 on the air. Previous versions have been discussed here more than once, and we closely followed all comments. As a result, the new version of the wishes of users were taken into account to the maximum. So what has changed in 11.5?

Php


‚Ä¢ For each site, you can now choose any version of PHP installed on the server and registered in Plesk - your build or downloaded from the repository. We were asked to do this, because different websites often require different versions of PHP.
‚Ä¢ The ‚ÄúHybrid Web Server‚Äù mode appeared, when part of PHP scripts can be run using nginx while the rest are working on Apache. Hybridity is setting up a domain, not a server.


PHP configuration on the site settings page

Security


‚Ä¢ Secure FTP support - now all traffic (including passwords!) Is transmitted over encrypted protocols.
‚Ä¢ Now you can get additional administrators on Linux (previously it was possible only on Windows); for them, as well as for users, password generation works.

Migration


‚Ä¢ In previous versions, we made a mechanism for preliminary checking the conditions of migration, which identifies all possible problems and limitations. But now we inform the user about these bottlenecks in advance and give convenient handles so that many of them can be quickly corrected (include the necessary services, add disk space, etc.) BEFORE the migration starts, not after it stopped due to an error in the middle of the process a few hours after the start.


Here's what it looks like.

‚Ä¢ The user is no longer bombarded with numerous warnings about errors of the same type during the process - adequate error prioritization has appeared, and we ‚Äústick together‚Äù all the information into one message. Also, all the corresponding logs for each migration are collected in a separate folder with the date and time, for example, / usr / local / psa / PMM / logs / migration-2013-04-10-19-39-22-557 /.
‚Ä¢ You can migrate sites with remote (remote) databases (without the risk of errors like "The database already exists," which happened before). Just be careful: do not delete the old site along with its database, it is better to simply suspend its work.
‚Ä¢ To shorten the backup time, we began to use the multi-threaded version of gzip - pigz (= ‚Äúparallel implementation of gzip‚Äù), which uses all available processors and cores to the maximum during data compression. As a result, according to our tests, backup is done 3.5 times faster on real servers. Archive pigz can be deployed and gzip-ohm.
‚Ä¢ For Windows: deferred file transfer is introduced - this reduces the migration time and the required additional space - free space is needed for exactly one largest subscription, and not for everything together.
‚Ä¢ For Windows: mail data can be migrated in the native mail server format (IceWarp, Smartmail, MailEnable) - we have abandoned slow IMAP. Also, the backup tool built into MS SQL is used (because of which the database recovery speed doubled).

Heading "You are very requested"


Many users of older versions of Parallels Plesk Panel were frustrated because they were not given enough control in the system. You are welcome:
‚Ä¢ Database users now have access to multiple databases from the same account.
‚Ä¢ Passwords for users are automatically generated (in one click).
‚Ä¢ You can quickly give access to the panel to the mail user - when you create it, simply check the corresponding option in the panel. You can enable the ban on changing the username in the system (some use for FTP access).
‚Ä¢ Mail queue can be cleaned directly through the interface.
‚Ä¢ Now we give more data on disk usage, which removes earlier questions like ‚ÄúWhat did it take 10 gigs for ?!‚Äù

Interface


Parallels Panel users have always had many questions about the interface. He was overloaded, the user was often confused and forgot where the function he needed was. Now the user sees only the necessary (the unwanted / unpopular we have removed in the appropriate tabs). Therefore:
‚Ä¢ All important (but not daily needed) information (such as system data, Plesk versions, etc.) was taken out of the home page.
‚Ä¢ The ‚ÄúWebsites and Domains‚Äù tab with all the tools became a homepage. On it all your sites are immediately displayed, moreover in the form of an active list. Here is what it looks like now:


It looks like a rolled plate with a site in the active list, when the site is empty

‚Ä¢ In the active list on the plate of each site the most popular functions are displayed. If the site is empty, the panel itself will offer to create content using the Parallels Presence Builder designer, install web applications (the same Wordpress), download files and / or create a database. If the content is already there, a link will be displayed on it.

‚Ä¢ If there are many sites, the active list mode may be inconvenient. Then you can return the table view.
‚Ä¢ The settings are hidden, since changes in them are required no more than once a month.
‚Ä¢ Instead of a dozen incomprehensible system folders on the user's home page (this is exactly what caused genuine pain to those who worked on older versions), now only 3 are needed with a clear purpose.
‚Ä¢ We divided the settings for the site and the settings for PHP so that there is no confusion. At the same time most of the parameters of Apache, nginx and IIS can now be configured directly in the panel.
‚Ä¢ Changed the file manager interface - to a more familiar one: with a folder tree and without ripples. You can upload files via drag & drop, upload multiple files at once, preview, unpack archives, etc.


New file manager interface - with folder tree and tidier

SEO


We made it possible to manage the PHP settings and the web server for SEO optimization, namely:
‚Ä¢ Aliases and www-prefixes of domains can be configured according to SEO requirements, including a custom canonical name for the site, with or without www.
‚Ä¢ The same for domain forwarding - you can choose from two types of redirects (permanent 301 and temporary 302).
‚Ä¢ Made a correct suspension of the web server. If the site is suspended, the code 503 is issued, if closed - 200 (previously, in any case, it was 200 and the default page). For those who are not in the subject, - earlier the search bot could decide that the site disappeared (default page) and the search engine "lost" all the accumulated useful information about it.

Applications and Licenses


‚Ä¢ Installation of applications (now there are more than 200 of them - both free and commercial) can now be launched in one click: the user is no longer annoyed by numerous questions, and all parameters are configured by default.
‚Ä¢ Users are notified that an update for the application has appeared, and can enable automatic updating. Do hosters have the opportunity to put a forced update of all applications on the server in case they are ready to prefer the risk of application failure to the risk of hacking the server through the old version of the application.
‚Ä¢ Search for applications: you can filter both paid and free (as before), and also sort by popularity, see the number of applications not only in categories, but in each subcategory.


Pleasant change - the number of applications in each subcategory

‚Ä¢ You can install a specific version of the application yourself (for example, to avoid problems with plug-in compatibility).
‚Ä¢ Added support for APS Mail Aspect, which makes it possible to integrate some Email applications (such as Open-Xchange).

post office


‚Ä¢ The manufacturer of our old webmail client has discontinued support, so we switched to the popular Roundcube (in the Linux version), and Horde has been updated to IMP 5.0. The last, as we know, is important for you - for this, many have transferred the production of the server to the preview.

‚Ä¢ A universal utility has appeared that allows you to run all CLI utilities on Linux for access to tools, logs, configurations and administrative databases. It supports autocompletion.
‚Ä¢ Special utility cfgmon allows administrators to track all changes made in critical configs and roll back to the previous version - in fact, it is a version control system for some configs.
‚Ä¢ ‚ÄúTolerance‚Äù and support for connected third-party repositories, for example, Atomic and Remi.
