Substitutions for nginx (Web Sockets)

At one time, nginx became a breath of fresh air for web development.

Currently, unfortunately, this product hinders the development of the Web, as it is a dinosaur of the HTTP / 1.0 era.

If you are using the modern features of HTTP / 1.1, or partial updates, or Web Sockets, you need a working alternative.

At present, there are only two alternatives for proxying Web Sockets. Both are pure proxy servers. That is, for the return of statics, or caching, everything is also worth using nginx or squid in the second echelon.

Firstly, this is the HA Proxy , which, by the way, shows higher response time characteristics than nginx. Configuration example from y8.

The second is Pound .

GHS talked about Yaws , an Erlang Web server that supports WebSockets.

If you know other working options, please write in the comments.
