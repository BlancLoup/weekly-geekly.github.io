What's new


  • Separation into paid and free versions . Yes, the most discussed "improvement" :) Let me clarify the situation a little:
    • The authors have completed the development and decided to cut the dough? Far from it. The current monetization is an old and planned step (for we have no means of developing a product for several years, as I went to vkontakte before starting to turn banners) to improve the quality of the product. Look at the majority of free open source utilities: almost all of them are developing (if they are developing at all) extremely depressing. One new feature can be added for months. We do not want such an outcome, we want to make our product much better than it is now. And do it in a short time. But for this you need at least ten times more time than you already spent on it. And we will invest this time, but without financial support this cannot be done. Therefore, we focus on the market and are looking for support in it.
    • Why are there so few features in the free version? We analyzed thousands of real product installations and selected the most efficient parts (on the one hand) and the most difficult configurable and problematic (on the other). Naturally, we left all the first group and bring the second to the paid version. The free version is now the best of all its free counterparts (due to a carefully crafted file merge algorithm, compression logic, caching logic, and a set of rules for .htaccess ). In fact, we have secured most of the users from themselves and left the maximum acceleration for professionals.
    • No more innovations? We have very far-reaching plans for creating an integrated tool to speed up websites ( some of them have already been announced ): this includes competent server-side caching processing at all levels, maximum improvement of file processing algorithms, and a set of profiling tools ( webo.in functionality) and a set of tools for "manual" optimization, and using the advantages of quick layout , and much more. We are not going to rest on our laurels, we just laid some foundation for future productive work.
  • Okay, back to the new opportunities. Now Web Optimizer is available as a plugin for Wordpress, Joomla! 1.5, Joomla! 1.0. We are working on integration in Drupal and Bitrix. All plugins work the same way: they wrap the Web Optimizer functionality into system calls and interfaces.
  • Improved compression. The gzip, x-gzip, deflate and x-deflate methods are now fully supported. In addition, a correct separation was made at the HTML caching level for all these types, which significantly increased the reliability of the product.
  • Improved caching of static resources. Now CSS and JavaScript files can be written without additional GET parameters (necessary for correct flushing of the cache). Thus, the cache of proxying servers gets clogged up faster, and even less traffic leaves the server.
  • Many improvements in the logic of CSS Sprites. In particular, the algorithm for integrating images from SpriteMe was integrated , which resulted in a smaller size of the final files. Also improved algorithm for translucent images for compatibility with weak servers.
  • Improved style merging algorithm. Now they "break" (if at all this happens) in units of cases out of tens of thousands.
  • Added Server Configuration tab (on the main page). Now you can check all the features of the system without additional analysis of the settings and the result of their application.
  • Improved accessibility and ease of use of the graphical interface (a large number of small innovations, some of which delivered two sleepless nights in a row).
  • Improved unobtrusive JavaScript conversion. Now supports more counters and ads.
  • The number of external integration features of Web Optimizer technology has been increased. Now you can simply send the HTML code for optimization and get the result at the output. Read more in the wiki .
  • Improved compatibility with HTML / XHTML / XML documents (we don‚Äôt add xml:lang to the first, xml:lang to the second, and skip the third).
