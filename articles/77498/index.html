Ajax Control Toolkit becomes part of ASP.Net Ajax


A few weeks ago, a beta version of the ASP.Net Ajax library was released. I am glad to announce that the Ajax Control Toolkit is becoming part of the ASP.Net Ajax library!

The new ASP.Net Ajax library integrates ASP.Net Ajax and Ajax Control Toolkit into one open source project. The Ajax Control Toolkit and the Ajax library toolkit are components of many web applications today and make life easier for developers.

The project will be implemented under the BSD license and can be used in many technologies, not only in ASP.Net, but also in PHP and Ruby on Rails. The project under development will be released with the support of CodePlex Foudation.

Opportunities:
  • Efficient libraries and developer tools
  • Productivity - allows you to create high-performance web sites
  • Compatibility - the ability to work on any server platform and with jQuery
  • Extensibility - built on top of the library and inherited from controls like DataView

ACT has provided and will provide many controls for developers, allowing you not to write code on the client side. In the beta version, we converted 34 controls into clean client scripts, now developers writing client scripts (including ASP.Net MVC, PHP and Ruby on Rails) can use them in their applications too. As a bonus, we made it possible to connect ACT controls as jQuery plugins.

The ASP.Net Ajax library is compatible with most modern browsers such as Microsoft Internet Explorer, Opera, Apple Safari, Google Chrome, and Mozilla Firefox.

The ASP.Net Ajax library is the first project to be submitted to the CodePlex Foundation . When the ASP.Net Ajax library finally comes out, Microsoft will fully support it. We also provide full support for jQuery and will continue to move in the same direction.

More information about the possibilities can be found here . Download and view the beta version of the library ASP.Net Ajax can be on the site CodePlex , where there are many examples and documentation.
