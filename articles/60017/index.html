The Book of Total Commander

After almost three years of preparation and collection of material, a book about Total Commander has been published. As far as I know, this is the first book about this file manager in Russian, and, probably, the first such detailed publication in general.

The authors are Konstantin Vlasov ( CaptainFlint , he unfortunately lacks karma for the post) and Vadim Kazakov, long-time activists of the okolototalovsky community, the authors of the most complete and popular localization package, and in general are good people.

Authors: Vlasov K. A., Kazakov V. Yu.
Name: Total Commander: effective work with files and archives
Series: In the Original
Publisher: BHV-Petersburg

Format: 70 * 100 1/16, paperback
Volume: 752 pages
ISBN 978-5-9775-0022-7
Circulation: 2000 copies.
Price: from 400 rubles.

Online stores:
OZON.ru
Books.Ru
Although it is not for me to judge the real need for just the book version, it is quite obvious that there is a lack of information about TC. For many, this is just a convenient two-panel replacement for a conductor, and not everyone can imagine what Total Commander is really capable of. Existing electronic sources of information are quite accessible, but, for some reason, poorly used. So any version of reporting information is good.

The saturation of useful information from the book is impressive (as can be judged by the table of contents ), and the volume of 752 pages makes it almost an encyclopedia. In addition to practical things, a little talked about how TC developed, and why it became so, and not some other. Both Kostya and Vadim have long been collaborating directly with the author of the program, Christian Ghisler, and a lot of interesting information in the book is obtained, so to speak , firsthand and for the first time in general. I think that I, using the TC for the second decade, will find something new for myself. Why, at least a couple of excerpts:
... About undocumented features:
... You can do without the <unpacking> dialog, if you start the SFX archive created in TC with the command line parameters. If you specify any of them, unpacking starts automatically. Multiple options together can not be set.
* Drive: \ Directory Files will be unpacked at the specified path with a request (if necessary) to confirm the overwrite.
* / S Files will be unpacked into the current directory, if necessary, a request for confirmation of overwriting is issued.
* / O Files will be unpacked into the current directory, overwriting the files of the same name is performed automatically.

... about interesting plugins:
... Do not think that a plugin can work only within the scope of tasks dictated by its extension: sometimes it has the functionality of several types of plugins at the same time, and you can use one set of functions or another in different circumstances. For example, the archiver plugin CHMDir can by itself also work in the ‚Äúinformational‚Äù mode (this displays the current help language, the tool used to compile it, etc.).

... or how to get around the "eccentricities" of different versions of Windows:
... For some reason in Windows Vista, getting a list of computers has become extremely slow and takes about ten seconds. Neither the reasons for this behavior, nor the more correct ways to obtain the list could be found, so Christian Gisler implemented list caching. Now in Vista (and only in it!) The list of computers is not automatically updated when opening the Network Neighborhood, but instead the saved contents are simply displayed.

A CD is included with the book, including the latest distributive, at the time the book was released, a stable version of TC (7.04a) and a set of the most useful plug-ins and utilities. It is quite important to mention that the book describes the capabilities of Total Commander of the previous, seventh version. 7.5, in the end, is in the midst of testing, and did not want to postpone the book indefinitely. However, the authors do not deny the possibility of reprinting a book when it is necessary for it to mature.
