The development of hyper-convergent technologies: a new version of HyperFlex 3.0 from Cisco

Last week, Cisco announced the release of a new version of software for the HyperFlex hyperconvergent platform - HyperFlex 3.0. Among the innovations include support for Microsoft Hyper-V, stretched clusters and containers, and services for working with multi-cloud. In more detail about these and other updates we will tell under a cat.


/ photo by Robert CC

The Cisco Hyperflex Hyper-Converged Solution is built on the basis of the Cisco UCS factory and the HX Data Platform, which integrates cluster HDD and SSD drives into a single distributed multi-level object data storage. Thus, it is a unified system that solves the problem of data storage, processing and transmission. Management of computing resources is performed in a single management console.

As Cisco vice president Liz Centoni notes (Liz Centoni), the latest version of HyperFlex 3.0 software is based on the same infrastructure, but introduces new features that make it even easier to work with data center systems.

Hyperflex 3.0 Updates


In the Hyperflex 3.0 system, Cisco has combined the capabilities of its own hardware platform with the capabilities of the Springpath software, which the IT giant announced last year. The platform now supports working with the Microsoft Hyper-V hypervisor in addition to the VMware ESXi hypervisor.

HyperFlex 3.0 also got the opportunity to work with software containers. Cisco has added a new FlexVolume driver, which allows you to work with persistent volumes (Persistent Volume) in Kubernetes and run applications in the data centers that are optimized for use in the cloud.

In the new version, users were also provided with more options for scaling: the opportunity to configure clusters with 64 nodes, and the distance between data centers was increased to 100 km. Another addition - support for 3.5-inch disks, which allowed working with more capacious storage.

According to IDC CloudView Survey last year's report, 87% of companies using cloud technologies are starting to pay attention to the hybrid cloud, and 94% plan to use several clouds. Therefore, Cisco has implemented two more tools: AppDynamics and CloudCenter. The first will allow you to monitor the performance of hybrid applications running on HyperFlex, the second - will facilitate the transfer of workloads between HyperFlex and private and public clouds.

Cisco Develops Hyper-Converged Ecosystem


Cisco is also actively developing an ecosystem of hyperconvergent technologies. For example, in October last year, the company announced the beginning of a partnership with Google. IT giants will provide a hybrid solution to work with hybrid clouds, which will allow you to deploy and manage services in the on-premise environment and on the Google Cloud platform.

In addition, Cisco recently bought several companies that are developing hyperconvergent solutions: Springpath, which we talked about earlier, and Skyport Systems. According to Cisco Business Development Vice President Rob Salvagno, the deal with Skyport Systems will allow the company to continue active development in the field of hyper-converged infrastructure, which has recently been gaining popularity.

According to the ActualTech Media survey, among thousands of IT specialists and managers of IT organizations, the number of hyper-convergent solutions implemented by companies in 2016 increased by 54% compared to 2015.
