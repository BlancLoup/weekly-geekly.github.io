How to convert video from GoToMeeting (g2m) to wmv3

My job duties include conducting internal trainings and webinars for company employees.

Webinars are conducted through GoToMeeting.

However, videos from GoToMeeting are saved as a completely unusual wmv file, which you can open
  • only after installing a special codec (g2m),
  • only under Windows (or poppy).
Without the presence of the g2m codec, the file will open as follows: there is sound, no picture.

And converting similar files into other formats using conventional methods is a complete disappointment. Not every editor will cope with this, and the result will be the following: the video is played at high speed, while the sound does not hurry anywhere ...

The problem is solved

General logic of action

  1. First you need to convert the webinar recording using GoToMeeting native tools.
    As a result, we get the same wmv-file, but encoded with the wmv3 codec, not g2m. Such a file can already be opened by regular players even in Ubuntu (if all standard codecs are installed there), and in the usual ways.
  2. Then convert the file to another format, if necessary. For example, in avi.
  3. Avi files are already easy to edit ...

Environment preparation


1. Install the g2m codec.
* Start Windows and / dev / hands /
* Download codec .
* Install.
2. Check the installation result: C: \ Program Files \ Citrix \ GoToMeeting \ 799 \

The number in the directory path depends on the version of the installed codec; in this case, G2m version 5.0 build 799 is installed, so the resulting directory is called '799'.

Before you start broadcasting, you should set up a way to record video.

1. Make sure that you have an Internet connection. Without this, g2m does not start.
2. Launch G2m. An orange snowflake g2m will appear in the tray.
3. Call the snowflake properties (right click> Preferences)
4. Follow the path: Category> Recording> Video> 'Convert to Windows Media Player file'> Ok.

Conduct a webinar


Enable recording of shamanism on the monitor.

Prepare a video file for conversion


Copy the final video file to a separate directory on your hard disk.

Recommended

1. actually copy the file, more changes will be applied directly to the proposed file, and you should not risk the original.
2. give the directory a simple and short name like 'C: \ 123'.
3. Give the file a simple and short name like '456.wmv'.

The simplicity and shortness of naming allows us to avoid using quotes. Sometimes quotes unreasonably lead to the error "Transcoding initialization error", but we do not need it.

Recoding a file from g2m to wmv3


1. Start the console (or use Far).
2. In the console, go to the directory where the g2m codec is installed
> cd "c:\Program Files\Citrix\GoToMeeting\799"
3. The next command from the catalog with the codec is running the program g2mtranscoder.exe with the file that needs to be recoded, as the 'source' parameter:
> g2mtranscoder source=EXPECTED_FILE
4. Our example:
> g2mtranscoder source=C:\123\456.wmv
5. A separate g2m window will appear indicating the conversion process.

It is going slowly, but reliably, we are waiting.

Result


Outwardly, nothing will change.

The same file on the disk, the same landscape outside the window.

However, SUDDENLY it turns out that now the file C: \ 123 \ 456.wmv can be opened with anything, including under Ubuntu.

Accordingly, it can be easily converted to other formats ...
