How I stopped worrying and fell in love with Hyper-V Server - part of R2

Recently I wrote an article about Free Hyper-V Server and ways to improve working with it. As a result of the discussion, it turned out that some habrusers managed to get RSAT for Windows 7 to work with Hyper-V 2012, but with Windows 2012 R2 something got broken.

And the incompatibility of Linux Integration Services and Windows 2012 R2, which I noted, has completely cursed Hyper-V. However, they were undeserved.




It turned out that the problem was that the hv_kvp driver supporting configuration configuration data (in particular, he was responsible for forwarding the IP address of the virtual machine to the host) chose the wrong version of the protocol. The problem was discovered and corrected in September, but this change was included only in the kernel 3.12. If you want you can rebuild this patch on your core.



Interesting fact: KY Srinivasan - the developer who wrote the original version of the driver, worked at Novell at the time of writing the first version of the driver, and recent fixes were made by him, but already as an employee of Microsoft.
