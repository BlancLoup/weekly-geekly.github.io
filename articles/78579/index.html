Recovering data from a hard disk using testdisk

What happened to me is not a pleasant story. From the number of sections on the hard drive disappeared one that had family photos. This is a tragedy! But the data was recovered. If you are interested in how, then read.

First of all.


If you accidentally deleted a partition that had data on it, or it disappeared itself, and also if you deleted files, then the probability of their recovery is quite high. If the disk was formatted, especially in the BIOS or after deleting the files other information was recorded, then there is little or no chance of recovery (

Secondly.


The recovery will require a free and cross-platform testdisk utility.

Install testdisk. Go to the console and run.

The program will ask whether to create a log file. Select 'Create'.


We select the hard drive with which we will work


Select the type of partition table. For IBM compatible 'Intel'


The program displays the menu. If you just need to copy the lost data, then choose 'Analyze'. The rest of the sections I did not look, so here themselves) But for data recovery they are not needed. Attention!!! Do not experiment on the disc with valuable information. Not skillful use of other options, it can bury her forever.


Next, the disk status is displayed at the moment. Click on 'Quick Search'


Choose Y or N, depending on whether MS Vista is used on a disk.


and ... watch the current percentages.
If testdisk detects the missing partition before the scan reaches 100%, then press 'stop'.


Next, select the desired section and press 'p'.


Testdisk will show files on the remote partition. Files deleted from the section will be displayed in red.


Next, select the necessary files and press 'c' to copy them. Specify where we want to copy and wait for the process to complete.
