1. Nikolai Alimenkov, Senior Delivery Manager in EPAM / XPInjection with the report ‚ÄúDo we need JMS in the world of modern Java technologies?‚Äù


JMS is known in the Java world as a standard solution for implementing distributed work using messaging. There are many JMS providers, both free and commercial. A large percentage of developers use JMS wherever they need to send a message and process it at the other end. But there are many alternative solutions for organizing effective message queues: AMQP, Redis, ZooKeeper, Apache Kafka, or even self-signed queues on Cassandra. Why not use them instead of JMS?
In this report, we will discuss the underlying problems of any distributed messaging system, and with this knowledge we will take another look at JMS and alternatives from the point of view of practical real-life scenarios.

Nikolai is an expert in developing Java applications and managing teams. With more than 10 years of development experience, she specializes in the development of complex distributed scalable systems. 2. Yet another technology stack in web-app development


When developing web applications, Java developers are faced with the choice of technologies. Often, developers want to stay in the Java / XML world, choosing GWT or JSF. I will try to propose a solution that would go beyond the framework of the Java / XML world, retaining its advantages and eliminating some of its minuses. The solution from the server side is based on the Spring Framework and FreeMarker, and on the client side, AngularJS and TypeScript.
