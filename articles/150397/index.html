RAD Studio XE3 - changes to the EULA?

Today, a text appeared on one of the Asian sites, allegedly representing a confidential letter from Embarcadero Technology to its partners about the upcoming restrictions in the EULA. I cite below the translation of the letter, and from myself I will add that if what I said there proves to be true, then I will hardly be able to upgrade to XE3 - from the principle. So:

This letter is intended to notify Embarcadero Technology partners of some changes in the EULA of version XE3 products.

In particular, the use of client / server data access technologies in the Professional edition is no longer permitted. This limitation applies to solutions from both Embarcadero and third-party developers. Users of the Professional edition can legally provide access to local databases only in their applications.

Users wishing to use client / server database access can order the Client / Server add-on package for their Professional edition of the product or purchase the Enterprise, Ultimate or Architect edition.

This restriction applies only to new licenses. Users upgrading to XE3 will be allowed to continue using third-party client-server technologies in version XE3. Additionally, in the Starter edition, access to data is limited to MyBase technology (CDS and XML file formats).

What does this mean for our partners? If your company offers tools that enable client-server or multi-tier access to the database for Delphi and C ++ Builder, we ask you to specify the condition of use in the information about your products only with certain Delphi and C ++ Builder: Professional editions with the client / server add-on package , Enterprise, Ultimate and Architect. Also, please provide information on the prohibition of using these tools with Starter and Professional editions without the client / server add-on package.

Prior to the release of version XE3, this information is protected by the current partnership agreement on non-disclosure of information Embarcadero Technology. Please do not transmit the details of this letter to persons outside of your organization until the release of XE3, scheduled for XXX XXXX.

Questions about these changes, please address to ???? and ????.
