SAP HANA features: big data turbo mode




The SAP HANA (High Performance Analytic Appliance) is a platform for developing, deploying and productively operating business-critical applications. SAP HANA includes an in-memory relational database, a set of tools for developing application code in various programming languages, tools for processing unstructured information, a library of templates for solving machine learning problems in various business domains and many other components. SAP, as part of its platform, offers a unique opportunity to solve problems that handle different types of workloads - OLTP and OLAP.

The SAP HANA relational database engine supports data processing in column and inline data storage options. In the second case, the column data storage option allows not only speeding up the aggregation operations during data processing, but also seriously compressing the data and saving a lot on the disk space for storing the original data.



In-Memory technology allows not only to speed up the processing of analytical requests, but also to speed up transactional requests, which are primarily characterized by low response time of the DBMS and a large number of requests per unit of time.

The SAP HANA platform allows applications built on its basis to access data directly, which in turn offers developers the opportunity to implement the business logic of the interaction of numerous components of their application simply and efficiently, without the use of intermediate protocols and data processing layers.

However, in the case of the transfer of previously created applications to the SAP HANA platform, it is possible to use standard open data access protocols: JDBC, ODBC, oData, REST services, ADO. NET, Python ODBC, R language.

The SAP HANA platform is scaled both within a single server node and within several server nodes, thus forming an SAP HANA cluster, and it becomes possible not only to automatically balance the load when processing requests from customers, but also to increase the volume of requests processed by simple Add a new node to the SAP HANA cluster.

What tasks can be solved using the power of SAP HANA?

Below we will talk about the fact that you can solve any of the tasks of interest to you on the basis of the SAP HANA platform, but for now we will give several examples of SAP projects already implemented or under active development on the basis of SAP HANA.

Personalized medicine

Decoding the genome allows you to talk about personalized medicine. Suppose that in one gene there are approximately 200 megabytes of data available to us for analysis. If traditional systems decipher DNA in 2-3 days, then with the help of SAP HANA this can be done in 20 minutes: 261 times faster.

Formula 1

On one racing car, McLaren, participating in the "Formula 1", installed about 150 sensors. Indicators like speed, tire pressure, engine oil temperature are collected in real time. For one racing lap, the system collects two gigabytes of such information, which gives about three terabytes of information for the entire race (about half a day). SAP HANA processes similar information for each car over the past 10 years, specifically for the company McLaren.

Business forecasting

Do not forget that the main direction of ERP is primarily a business for which the use of SAP HANA opens up tremendous opportunities. For example, you can collect a huge amount of data on the business system in real time, and build a comprehensive simulator of company productivity. At the same time, business leaders are not faced with the problem of data sampling: the sample can be made really large, and the system will not lose in speed.

These are just a few examples of how SAP HANA can be used as a data processing system. It also finds wide application in marketing, management, logistics, and other industries, the quality of which can be significantly improved with timely analytics.
