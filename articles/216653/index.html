Get acquainted! Appnestic - a new generation hosting platform

Dear Habro users.

The era of VPS and unmanaged servers is in full swing, but at the same time there are new platforms that allow you to host applications in a couple of clicks. Appnestic platform is one of these.

Prehistory

For me, the developer often lacked a tool that would allow quickly deploying an application, so as not to mess with hosting, servers, linux, mail and other infrastructure, configure it and pass access to the client. There was also a lack of a convenient DNS zone manager, the ability to configure the application and transfer it to the client, manage the application without ssh access and configuration files.

For the resource unit we took the concept of nest (nest). This is a virtual parameter meaning 256 MB of RAM, 5 GB of disk space and 1 processor core.

What can the platform do?

Quick deployment of resources for the application. A few clicks and you have a ready system to install the application. A few more clicks - and the database server is available to the application.

The code version control system, application load monitoring, the ability to clone environments make the process of developing, testing and operating web applications as simple and convenient as possible.

Need more RAM? Is the load increasing?
No problem!
Just add more nestos or create a copy of the node.

PHP and Python support. You can post a project on Django or on Yii. The plans - support for Ruby and Java applications.

Updating code on a push event from a Git or Mercurial repository. A feature that is convenient during project development, useful for developers.

Change of ownership of the application. You are a web studio, developing a project for a customer, launching it, setting it up - and transferring it to the customer.

Collective tuning and resource management. After transferring the application to the customer, you can ask to add you to a group that has access to the application management interface.

Manage DNS zones. Applications and domains are different products, but we decided to merge. Domain management can also be passed to users or set up group access.

Hourly pay for resources. Yes, you only pay for how much resources your application needs right now.
