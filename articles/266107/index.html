3CX API - what are they and what to do with them? Third Party Plugins (Part 2)

In the last article I looked at the API for 3CX Phone System, what to do with it and what solutions are already available from the developer. But, of course, 3CX cannot keep up with all applications, especially those that are distributed locally, such as our 1C. In this case, local craftsmen are connected.



Integration with 1C


The brave guys from Orsk have already made 4 versions:
  • SIP softphone for 1C - a full-fledged softphone built into the 1C interface
  • 3CX WebAPI Client is a CTI application that implies the presence of some kind of dialer, soft or iron.
  • ifClient 3CX Free - a trimmed version of the 3CX WebAPI Client, can initiate a call and show a card. A definite plus is free.
  • 3CX CDR Client for 1C is a separate module for statistics. History is recorded in a fully automatic mode without the participation of users. Data is taken from 3CX CDR files.




All these modules do not require 3CXPhone and work directly with the PBX. Demos available for testing.

Call Records Management




3CX Exporter allows you to transfer of Call Detail Records (CDR) from your own 3CX base (PostgreSQL) to Microsoft SQL Server or MySQL. This is a small Windows service that synchronizes external database with 3CX.
Moving Call Detail Records from 3CX to SQL Server or MySQL reduces the burden on PBX and allows you to create your own reports.
www.voiptoys.com/3cx-exporter-for-call-detail-records
