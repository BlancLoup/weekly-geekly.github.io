Php


  • RFC: Exceptions in the engine ‚Äî The proposal to implement standard errors instead of fatal errors was previously rejected as too dramatic for a minor version of PHP. Now it is proposed to include this feature in the release of PHP 7. In contrast to the previous version, in addition to fatal errors, it will also be possible to catch parsing errors.
  • RFC: Objects as hash keys - It is proposed to add the magic method __hash() , which will allow the use of objects as keys of arrays.
  • RFC: Return Type Declarations - Updated type hinting clause for return values. The suggested syntax is: function getUser(): User { return new User(); } function getUser(): User { return new User(); }
  • RFC: Readonly Properties - It is proposed to add another readonly access modifier, which will designate properties that can be written within the class and only for reading outside of it.
  • RFC: UString - Offer to include the ustring extension in the kernel, thus getting a UString class that encapsulates work with unicode strings.
  • RFC: Safe Casting Functions ‚Äî It is proposed to add the to_int() , to_float() and to_string() , which will return false in case the transferred value cannot be cast to the appropriate type.
  • RFC: Remove deprecated functionality in PHP 7 - Offer to remove all deprecated features in PHP 7.
  • RFC: Anonymous Classes v2 - Second attempt to implement anonymous classes in PHP.
  • RFC: PHP 7.0 timeline - PHP 7 release plan. The final version is expected in October 2015.
  • PHP 5.6 constants - An interesting undocumented PHP 5.6 feature: arrays can be assigned to constants.
