Parse.com is closing

Hello friends!

According to information from the official blog , the service Parse.com, known and popular among developers of mobile (and not only) applications, will stop its work.

The service will be completely disabled January 28, 2017 - existing users have exactly one year to migrate data and transfer their applications to another location.

To this end, the Parse team has released a utility for migrating data from Parse to MongoDB and Parse Server - an open-source solution that allows (to some extent) transfer API functionality to its equipment.

Migration Guide offers to transfer existing applications in two stages:
  • data transfer to MongoDB own database (API will still be available via api.parse.com)
  • Installing and configuring your own Parse Server and then transferring the API to it.

The bad news is that Parse Server does not 100% implement the functionality of Parse.com. So, according to the documentation, not supported:
  • Cloud Code modules (it is proposed to use third-party libraries)
  • Analytics (it is suggested to use other services, for example Mixpanel or Google Analytics)
  • Config
  • Dashboard (no administration UI out of the box)
  • iOS in-app purchase verification
  • Jobs (suggested to use other solutions, for example kue)
  • Push notifications
  • Schema API
  • Webhooks

Those. simple applications that, for example, only store data or use authentication and sessions, will be transferred without any problems. However, if you use jobs, hooks, send push notifications or use analytics, you will have to do a serious reworking of the application.

Of course, there is a possibility that in a year the Parse team will expand the number of supported functions of the Parse Server, but hope only for this is not worth it.

As emphasized in the text, it is recommended to start work on migration as soon as possible .

It may be recalled that Parse.com was founded in 2011 (one of the founders is an emigrant from Ukraine, Ilya Sukhar), and in 2013 Facebook was bought for $ 85 million. At that time, Facebook had not yet had significant indicators in the monetization of mobile traffic and the company considered various options for development in other directions. Buying Parse allowed us to gain a foothold in the niche of paid services for mobile application developers, with an eye to future growth and revenues.
However, now that Facebook‚Äôs quarterly profit has exceeded $ 1 billion (and 80% of this is due to advertising on mobile devices), the company has less and less desire to engage in third-party projects that are not related to the advertising model. Especially, when serious bets were made on WhatsApp and Oculus, which are not bringing any profit at the moment.

As a result, the forces will be transferred to more promising areas, and Parse is closed.

All successful migration!
