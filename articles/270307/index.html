A cryptographic program encrypts user files offline.



There are a lot of different kinds of ransomware, cryptographer programs now. Some simply block the PC until the user pays. Other types of such software encrypt files by sending a key to a server controlled by scammers. But there are other types of cryptographers who act even more original.

Researchers at Check Point recently analyzed the work of one of the varieties of such programs, which uses an alternative method of encrypting files and providing the key to their creators. The program itself is not new, it was first noticed in June last year. Since then, the author has repeatedly updated his creation (about once every two months), the cryptographer is constantly evolving and improving. According to information security experts, this sample was created by Russian-speaking attackers, and this software works, as a rule, with users from Russia.

When downloading to the victim's PC, this software encrypts all user files, simultaneously renaming them. New names are given by mask:

email- [address to contact] .ver- [Ransomware internal version] .id- [Machine identifier] - [Date & Time] [Random digits] .randomname- [Random name given to the encrypted file] .cbf (example: email -Seven_Legion2@aol.com.ver-CL 1.0.0.0.id-NPEULAODSHUJYMAPESHVKYNBQETHWKZOBQFT-10 @ 6 @ 2015 9 @ 53 @ 19 AM5109895.randomname-EFWMERGVKYNBPETHVKZNBETCHKKHPHPHKKZBBHPKKHPKHPKKHPKHPKHPKHPKHPKHPHKKHPHKZOBQFT-10 @ 6

At the same time, no data is transmitted to the attackers' server, no software information is accepted (like an encryption key).

The beginning of each file (the first 30,000 bytes) of each file is encrypted using two sets of numbers and letters, which are generated in a random order already on the infected machine. The rest of the file is encrypted using the RSA public key, also randomly generated on the user's machine, along with the RSA private key, which is required to decrypt the data.

Randomly generated character sets and a local RSA key (secret) are added as metadata to each file to be encrypted, and then encrypted with the use of three complex RSA 768 public keys that have already been created remotely. The corresponding RSA secret key is required to decrypt the metadata on the attacker's side.

When an attacker contacts the victim via e-mail, he asks to send a sample of the encrypted file. Next, he extracts the encrypted metadata from the file, uses the RSA secret key to decrypt the data, and thus receives the character sets and the local RSA secret key that the victim needs to decrypt the files.

The contact address used by the fraudster is often a Gmail or AOL account. Researchers contacted the attacker, telling about themselves as the victim of a cryptographer. He asked for the decryption of data 20,000 rubles.

According to specialists who have studied software, to pay is the only way to get your data back in this case (if you do not have backup).

Decrypt files without the corresponding key is not possible. The required time for a brute force using a network of powerful PCs is about two years.
