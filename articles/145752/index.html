iOS 6 renders slider highlights based on device position



When the device changes its position in space, the drawn glare of the metal control is drawn in real time and corresponds to it in reality, as if the slider were really metal.

Here you have the attention to detail (I suspect that the addition of such logic has almost no effect on the operating time of the device) - Metalutionary !

It is worth noting that while this behavior is fixed only in the application Music. In other places (for example, the brightness and volume slider at the very beginning of the menu of running applications) such sliders remain static (it is possible that for the time being this is only a beta version).

Please write in the comments where you have seen before that the use of spatial position in the usual interface (except for games and special applications using a gyroscope, for example, in augmented reality). It is the use of skeomorphism at this level.
