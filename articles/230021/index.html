Data center


We chose the most reliable data center for failover tests. Our goal is not only to sell the service, but also to ensure the maximum quality and safety of its operation to solve business problems. Placing a public cloud in a data center of this level is a precedent. In InfoboxCloud, we have combined exceptional data center reliability, cloud server storage technology without a point of failure Parallels Cloud Storage, enterprise-level cloud management software for Parallels Automation for Cloud Infrastructure, while keeping prices affordable.

DataSpace data processing center became the first data center in Russia and Eastern Europe, which received Tier III Design and Tier III Facility certificates from the international organization Uptime Institute. The data center complies with international standards of reliability and fault tolerance, which guarantees the reliability and continuity of our clients' business processes.
DataSpace guarantees 100% availability of equipment, applications and data, which is much higher than the Tier III Facility certification requirements (99.98% = 1.6h per year).



In the Russian market of data centers, quite often there are statements of conformity of the data center to the Tier II or Tier III level that are not confirmed by certificates. Many have certified only the documentation for the data center (Tier III Design certificate).

DataSpace has been certified in two types: Tier III Design and Tier III Facility (checking the infrastructure of the data center). To obtain a Tier III Facility certificate, more than 1,000 data center tests were conducted by the Uptime Institute certification organization.

Compliance with the level of Tier III Facility guarantees the reservation of all engineering subsystems and the possibility of carrying out planned repairs and equipment replacement without stopping services. Users of data centers of this level are large banking and financial institutions, organizations that support internal and external customers around the clock, electronic market transaction systems.

Cloud infrastructure


Cloud Infrastructure (IaaS) InfoboxCloud allows you to create, operate and manage reliable, resilient, secure cloud servers connected by a private network. Cloud management is done through the control panel or through the REST API. At any time, you can create servers of the required configuration with the possibility of autoscaling and hourly payment for the amount of allocated resources to servers without subscription fees. For stopped servers, payment is made only for the occupied disk space, since other resources are not consumed at this time. From the control panel, you can configure firewall rules for each machine, delegate rights to manage the cloud to administrators.

The key difference from the Amazon Web Services, Windows Azure and Google Compute Engine services is the ability for each cloud server to set the optimal parameters for the amount of resources allocated for the task (CPU, Ram, disk, etc.) independently of each other.



There is no need to choose from dozens of tariff plans and pricing tricks. We tried to make a single and as simple as possible tariff and included 3 terabytes of traffic per month for free, so that you can roughly predict the price of the service for the month. InfoboxCloud cloud infrastructure supports two types of virtualization: containers and isolated virtual machines (‚ÄúI will manage the OS kernel‚Äù in the InfoboxCloud control panel). Container virtualization is faster than virtual machines, but virtual machines allow the user to make changes to the kernel of the OS if necessary.

The installation of InfoboxCloud in the data center consists of grouping new servers combined into a fault-tolerant cluster using the PACI enterprise-level cloud solution (Parallels Automation for Cloud Infrastructure). Cloud Infrastructure Control Panel Now it is possible to order from it not only the cloud, but also VPS in various locations.

The OS uses Parallels Cloud Server with activated distributed failover file system Parallels Cloud Storage for reliable storage of virtual machines. The technology distributes data between physical servers, maximizing the throughput of the disk subsystem, performs migration and replication of servers with zero downtime and works without a point of failure. In case of failure of a disk array or a physical server, automatic recovery of damaged, lost or inaccessible copies of data occurs without stopping the user's cloud servers. Background verification and error correction is provided on the fly to ensure data integrity and accuracy.
