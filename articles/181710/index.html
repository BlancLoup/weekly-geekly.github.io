Qt Creator 2.8.0 beta released

On May 30, Qt Creator 2.8 beta was released, in which many new features were implemented and a large number of bugs fixed.

All innovations and corrections in one post will be quite difficult to fit, so I will try to highlight only a few highlights.

A new feature has been added to the editor, available through the Window> Split New Window. This will create a new window, similar to the editing area in the main editor window. A new document will open in the window that was active. In the previous version of Qt Creator, the possibility of simultaneous work with several files was also present, but this was realized by dividing the main editing window into corresponding parts. Now (not excluding the previous possibility), files can be ‚Äúplaced‚Äù into separate windows, which will allow working with them on different monitors.


Ct ++ support in Qt Creator has also undergone many changes and fixes, as well as more refactoring features have been added:
  • Moving the implementation of a function from the header file to the implementation file
  • Assigning the return value of a function or expression "new" to local variables
  • Adding declarations and implementations of pure virtual functions from the superclass

You probably didn't notice, but in Qt Creator 2.7 there is an experimental possibility of pairwise file comparison (which must be explicitly included). In Qt Creator 2.8, this is probably enabled by default and is used for version control in git. You can compare two arbitrary files using Tools> Diff (Compare).


The display of many data types during debugging from CDB was also corrected, QNX and Android support received many fixes ... many ... very many ... For more details, I recommend reading the change log or just downloading it.
