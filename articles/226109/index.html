Firefox 30 release

I am very late to announce that Firefox has become available for desktop downloads. The main innovations are: blocking by default all plug-ins except those in the white list, support for GStreamer 1.0 for Linux, changes in tools for web developers, support for background-blend-mode. List of all changes in Firefox 30 under the cut.


  • Added a button to display side panels with an interface to provide simplified access to bookmarks, history of visits and exchange links in social networks;
  • Tightened restrictions on displaying plug-in-related content. The post-click activation function added to Firefox 26 , which works for Java and older NPAPI plug-ins, will now be switched to blocking all plug-ins by default, with the exception of those added to the special white list ;
  • OS X adds support for the command + E keyboard shortcut for substituting the found expression into the selected text;
  • Added support for GStreamer 1.0 for Linux (previously supported 0.10);
  • Expanded tools for web developers: more in the original article in English and its translation , made by spmbt ;
  • When using Web Audio for each window is available off / on and adjust the sound volume;
  • Web applications do not allow WebIDL constructors to be called as functions;
  • Elimination of elements beyond borders, including beyond the box-shadow area;
  • Support for the background-blend-mode CSS element is enabled by default;
  • Using line-height is now allowed for input type = ‚Äúreset | button | submit‚Äù ;
  • Implemented ECMAScript6 array and generator comprehensions;
  • The error stack now contains the column number;
  • Added support for the alpha option in the canvas context settings ( more );
  • When suggesting saving passwords with a password manager, autocomplete = "off" is now ignored ;
  • Fixed an issue where TypedArrays did not support new named properties;
  • NTLMv1 authentication disabled
