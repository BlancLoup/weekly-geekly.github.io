Opera Developer Bookmark Sync



In the recent build of Opera Developer 26, the synchronization of bookmarks, long awaited by you (and us), has appeared - so far performed by computers, but the picture with the phone seems to hint. This implementation replaced the one that worked for some time when turned on from flags. The new sync is still on the Developer channel and goes into beta closer to the release.

Answers on questions


Where is the synchronized data stored?

Data is stored on Opera's own software servers.

What data is synced?

At this stage, we still sync bookmarks only. In the future, we plan to add to them and other information. As you can see, the flags have a setting for synchronizing open tabs, which is currently disabled by default.

Are my synced bookmarks encrypted?

During the transfer of data between the client and the server - everything is securely encrypted. However, the data itself is stored on the server in an unencrypted form. When (and if) in the future we synchronize sensitive personal data, such as passwords, we will store it in encrypted form.

How are these bookmarks related to bookmarks from Opera Link?

All data synchronized from Opera 12 is still accessible from the Opera Links web interface . You can also import bookmarks to a new system from an existing Opera 12 installation.

Is it possible to remove the button with the accounts?

This button allows you to access new synchronization options. But that's not all - watch out for future releases.

What does this synchronization have to do with the previous implementation?

The previous experimental implementation (which appeared in Opera 19) was replaced by a new one.
