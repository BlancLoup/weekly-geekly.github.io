Jelastic PaaS - major update: horizontal scaling, Java 7, Jetty, HTTPS



I rush to share great news about a great update Jelastic - a cloud platform for Java applications. The latest release is really big. It was added the long-awaited
  • horizontal scaling
  • Java 7 support
  • Jetty 6
  • Https
  • topology editing

Horizontal scaling is one of the most commonly used solutions for high loads. In our implementation, you can very quickly pick up several parallel copies of a web server. Now you can add and remove servers on the fly. However, in any case, your application should be designed to work on multiple servers. Unlike vertical scaling, horizontal scaling requires the application to ‚Äúunderstand‚Äù that it can grow horizontally. Those. The developer must consider these features.

Now developers have a unique opportunity to combine two types of scaling to achieve maximum efficiency of their applications. The provision of similar functionality in cloud solutions is unique at the moment on a global scale. None of the current competitors have such capabilities.

JDK 7 support . Now you can experiment very much with the new version of the JDK, check how compatible your application is with the new version of Java. It is important that you can switch the JDK version in the control panel without having to re-create the environment or redeploy the application.

Jetty support . At the moment we have added support for Jetty 6. This is the second most popular, after Tomcat, "lightweight" java-server. In the next releases Jetty 7 and Jetty 8 will be added.

HTTPS support is one of the most requested features. HTTPS is enabled with one click of the mouse (like everything else). In the current release, we provide HTTPS for internal domains based on * .jelastic.com. In the next releases there will be an opportunity to upload personal certificates and enable HTTPS for external domains.

Editing the topologies of environments . From the current release Jelastic supports editing previously created environments. now you can add or remove databases on the fly, change the number of application servers, add or remove a balancer, in general, perform all basic operations on editing the environment topology.

+ Naturally, many other smaller, but useful features were added, such as: improvements to the configs editor, the ability to expand logs to full-screen mode, batch file downloads, and more.

The next release is planned

  • package manager image magic, ffmpeg, others
  • session replication
  • build node (maven, git-push-deploy)

Participation in JavaOne 2011


You might be interested to know that we will be participating in the largest event in the Java world - JavaOne 2011 , which will be held October 2-6 in San Francisco. We will have our demo booth. If anyone can attend this event - come to us, we will be happy to talk - we will tell you in more detail about current opportunities and plans for the future.

Run in the US data center


Another good news is that we have found a partner in the USA and we will soon launch Jelastic in one of the modern data centers in the USA. Details of cooperation have not been disclosed. This is the second hosting partner that will provide our solution to customers. The first is in Europe. Also, negotiations are under way with hosting providers in the CIS. We will try to provide you with maximum flexibility in choosing a data center.
