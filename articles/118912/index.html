We put clouds in one basket



Cloud data storage is undoubtedly one of the most interesting and relevant areas in IT. Suffice to say that the leadership here is an active struggle between such giants as Microsoft, Google, Amazon. Interest in this is actively fueled by third-party development, actively using these services. For example, the manager of Dropbox, which, as it now seems, has always been. I can not believe that some 2 years ago he was hardly known .

Over time, I was ‚Äúovergrown‚Äù with various repositories, which, in turn, were overgrown with services. And the recent googling on the name of Gladinet when reading comments here on Habr√© (I apologize, could not find the comment), led to the site developers of the Gladinet desktop application , quite worthy of close attention.

Having installed the free version of the application, I found out that I can connect almost all the clouds I used to the newly created device. All data at the first start are connected as links. The download occurs when the files are actually accessed.

The application is built into Explorer and other file managers (in my case, Total Comander) as an additional disk in which all virtual devices are assembled.





Google services

Personal Google account. Connection is made by GMail + Password.
GDocs and Picasa are great. At the same time they look a bit different. Picasa shows image sizes, but GDocs does not. What will happen in GDocs when one document is in several collections has not yet been verified. There is hope that will sync.

Google Storage for Developers . Finally got a full-fledged client for Google Storage for Developers !



Here the connection is not to the root of the repository, but to a separate baket. It is required to enter a pair of keys . If the bucket does not exist, then it is created. I tested the work of the storage for storing raster tiles - KML2KML tiles and Google Storage and simultaneous display in OSM, GMaps, GEarth from one set of tiles. Everything worked well as a system, but it took a small rite with a tambourine to include data in open access. If I get together, I will describe the subtleties of creating similar systems in another publication.

Google Apps. In addition to personal accounts, you can connect to GDocs for domains . In Gladinet, this is called a Converted Google Apps account . An additional device is created that is completely analogous to personal GDocs.

Amazon Services
Amazon Cloud Drive. Another free 5Gb of additional space. Amazon Cloud Drive connects in the same way as GDocs, but already via an AWS account.



Amazon S3. Connects like Google Storage for Developers. Connection is made to the baket by a pair of keys Access Key ID Secret Access Key . Sharing the console and the virtual device allows you to solve a lot of problems when managing file data on the web. On trial as well as in Google Storage for Developers posted a set of raster tiles for sharing access from several applications. The access to the data is controlled from the console , so there was no need for additional utilities, like in Google Storage for Developers. In this regard, Amazon S3 is more convenient than a competitor.



Microsoft Skydrive. No problem connected Skydrive . Connects to the entire login similar to GDocs. Another additional 25Gb data storage.

Box.Net. Another cloud storage with 5Gb free space. Connected similarly to GDocs.

I am not hooked to Dropbox yet, I keep both applications running.

The list left untested local and network devices, MS Azure, FTP, and a number of virtual devices that were not encountered. If you share your experience with these devices in the comments, I will be very grateful. I would like to add to the list.
