Veeam Cloud Connect in Microsoft Azure

Every time when it comes to backing up, a lot of questions arise, and concerns about reliability are seen as a red thread through all these questions. Reliability of recovery, reliability of storage, reliability of creating the backups themselves. A good backup product allows you to get out of situations where reliability is in question. Best in class product - allows you not to be in such a situation in principle.

It is not for nothing that the proverbial ‚Äúnot to keep all eggs in one basket‚Äù exists. There are many examples with not the best endings, when backup copies are written to a dying data storage system (DSS), and sometimes they are completely stored on the same DSS as productive data. Based on the experience of using Veeam products to 168,000 customers, and in order for users not to repeat the mistakes of their colleagues, Veeam architects promote the idea of ‚Äã‚Äãan ‚Äúideal backup system architecture‚Äù. Among other things, the ‚Äúideal architecture‚Äù implies separation of backup storage for business continuity purposes and for long-term data archiving.

There is a rule ‚Äú3-2-1‚Äù , which says that your infrastructure should have:

  • 3 copies of data
  • 2 types of backup storage media
  • 1 of the copies must be outside the main site



In small and medium-sized organizations, the main issues, as a rule, arise precisely with the last item - the need to have a backup site. Not everyone has built or rented a platform for disaster recovery. Sometimes, building or renting a place in a data center, with the purchase of equipment, service charges and associated costs, is economically impractical.

In this case, it's time to turn to cloud technologies, which can significantly reduce both the time of organizing the notorious reserve site, and significantly reduce the cost of its creation.

Veeam Cloud Connect




Back in version Veeam Backup & Replication 7.0, Backup Backup Job and WAN Acceleration technologies appeared. The first allows you to duplicate the main copy of backup files to the desired number of other disk storage systems, and it does not matter how these storage volumes are presented - local disks, shared CIFS / NFS folders or a virtual hard disk hooked to an IaaS virtual machine. The second uses a very efficient global deduplication, with a small block size and shared cache, in order to reduce the amount of transmitted traffic up to 50 times. This coefficient is obtained empirically, on the basis of the transfer of standard virtualized services to the Republic of Kazakhstan.

In Veeam Backup & Replication 8.0, based on these technologies, the functionality of Veeam Cloud Connect appeared . In its first version, it allows you to maximally simplify the process of archiving data to a service provider, or as part of a hybrid infrastructure, into the cloud part of your ‚Äúpossessions‚Äù. There are two typical options for using Veeam Cloud Connect:

  • You are a service provider, registered in the Veeam Cloud & Service Provider Program (VCSP) , want to receive backup copies from existing users of Veeam Backup & Replication. You can do this without even having your own infrastructure using Microsoft Azure.
  • You are an end user and you have a Microsoft Enterprise Agreement, or you have acquired 100 licenses of the Veeam Backup & Replication / Veeam Availability Suite

If you don‚Äôt recognize your organization in these two descriptions, there‚Äôs a third option ‚Äî it‚Äôs enough to have one IaaS virtual machine in Azure , and an Azure virtual network for network accessibility.

Setup process


In the Azure Marketplace there is a prepared virtual machine with Veeam Cloud Connect - all components are already pre-installed and the configuration process from the receiving side takes about 10 minutes. You need:

  1. Add a certificate for SSL protection of the created connection
  2. Set the role of Cloud Gateway
  3. Add repositories for storing backups
  4. Add user accounts for access



When adding Cloud Gateway, you will need to specify how you can reach the gateway from the outside - it is connected to the real IP directly, or is behind NAT.



After that, it is enough to add a user, assign him a password and specify a quota for disk space.



From the user's side, everything looks the same - you need to add a service provider, and then configure the regular Backup Copy Job, which all users of Veeam Backup & Replication are accustomed to.



After this setup, all traffic will go through one port in a secure SSL connection. All data can be transferred through a pair of WAN accelerators, and your backups will be in reliable Azure storage, your backup system will begin to comply with rule 3-2-1, and your life will be one less care.
