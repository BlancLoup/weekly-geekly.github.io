Transfer mail from Exchange to Zimbra

Why upgrade to Zimbra with Microsoft Exchange? Zimbra is Open Source, so you can be sure that you have full control over all the information that comes to your employees and is stored on your server. Microsoft Exchange can not be changed, and Zimbra can be configured up to a complete rebranding in the Open Source Edition and remarketing in the Network Edition. If you buy Exchange, you will most likely need to purchase other Microsoft software, and with Zimbra you do not need to do it. Also, Exchange is very difficult to integrate with other business-critical applications, while for Zimbra these tasks are solved very simply. In the end, there is the question of the cost of mail for your organization. In this article we want to tell how you can quickly move from Exchange to Zimbra.


If you are thinking about migrating from Microsoft Exchange 2003, 2007 or 2010 to Zimbra, then the ZCS Migration Wizard will help you in transferring all the data. It allows you to transfer accounts, messages, calendars and address books. And keywords in Microsoft Exchange are converted to tags Zimbra Collaboration. ZCS Migration Wizard imports all data from the Outlook server, except notes, tasks, and logs with errors and warnings.

During migration, ZCS performs the following tasks:

  • Creates a user account on the ZCS server, if it does not already exist.
  • Imports email, attachments, calendars and address books.
  • Creates a log file to resolve any errors or error warnings.

Migration planning


Before migrating from the Exchange server, create a plan to notify users of the new ZCS email program and provide the users with new information about access to mail (ZCS URL, username and password).

It is recommended to schedule migration at a time when server load is minimal. Any emails sent from Exchange after migration are not saved in the ZCS mailbox.

There are two options available for moving from Exchange to ZCS:

1. The ZCS Migration Wizard asks questions in a series of dialog boxes on how to set up new accounts. Before you begin, you should define:

  • Which domain will the user accounts be migrated to.
  • Whether to select individual accounts or select groups of accounts to import.

2. The One Step Migration option from the ZCS Migration Wizard allows you to use the generated XML file with user data that is required for the migration.

Migration. Brief instruction


Before starting the migration, change the mail relay information to the Zimbra server instead of the Exchange server.

Migrating accounts and importing email


1. Download the migration wizard.

2. Run the ZCS Migration Wizard and click Next to continue.

3. The Import Destination dialog box will appear. To connect and authenticate on the server, enter:

  • Hostname This is the hostname of the Zimbra server.
  • Admin Port. The default administration port is 7071.
  • Use secure connection (SSL) to establish secure
  • Connection to the server port.
  • Username and password. This is the name and password of the administrator account on the Zimbra server.

Click "Next".

4. In the "Destination Domain" dialog box, select the Zimbra domain name where the accounts will be transferred.

5. To continue the import, in the MAPI Logon dialog box, define the MAPI profile that you created for use with the ZCS Migration Wizard. Click "Launch Profile Chooser" to launch select an existing MAPI profile or create a new one.

6. In the Source and Destination Mailboxes window, define the users that will be transferred from Exchange to the Zimbra server.

7. Check the added accounts in Source & Destination Mailboxes and click Next.

8. Now the ZCS Migration Wizard will check if these accounts exist on
server Zimbra.

9. Select service classes (COS) for accounts.

10. Wait for the creation of accounts.

11. Choose which information to transfer.

12. Done.

Detailed information on the work of the ZCS Migration Wizard can be found on our website.
