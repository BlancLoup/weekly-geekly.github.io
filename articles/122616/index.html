Full IP-KVM for all deductible servers


The news is one paragraph: now, without exception, our dedicated servers (including Xeons and Atoms) have the ability to use IP-KVM built into IPMI. It:
  • the ability to watch the console (including graphics modes, BIOS, etc.)
  • Press any tricky buttons (Ctrl-Alt-Del, Alt-SysRq, etc.) from the virtual keyboard
  • the ability to connect the ISO as if it were a USB-CD connected to the server
  • Similarly - images of floppy disks (is someone else using this?)
  • Turn on / off the server power regardless of the opinion of this operating system.

Story

All servers that we rent are equipped with a built-in IPMI module. A very, very useful thing that allows you to install your favorite operating system while sitting in a comfortable chair, rather than standing near a cart with a monitor and keyboard in a noisy and cold machine room.

Until recently, the full range of possibilities was available only for the platform part. While the other part of the platforms was only able to "on / off / reload". Offensively? Yes. Probably the easiest solution would be to give these IPMI users "as is", but: firstly, IP addresses are less and less, secondly, the problem with user qwerty passwords.

So the only acceptable solution was to integrate the capabilities of IPMI into our control panel. But here we were waiting for trouble. The main problem was that for different platforms different implementations of the software part of IPMI boards were used (the hardware did not change so much). Some of them needed one controlling port, some two, some three + to each, each of the implementations had its own authorization method.

As a result, we picked the interaction format between the java-applet and the server part and we managed to integrate a full-fledged IPMI for all our configurations.
