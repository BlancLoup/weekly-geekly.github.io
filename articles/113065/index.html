What's going on here? Exactly what I would do in the absence of a mouse, in order to get the same result that will turn out. I would press the left windows {LWIN} key, that is, I went to the Start menu of my Windows XP, then three times the "UP" key {UP 3}, so I went to "Run ...", I would press the {ENTER key }, typed notepad and started the notepad, wrote a typical HelloWorld in it, closed it with alt-f4, which is written as% {F4}, a typical dialog about saving data ‚ÄúYes No Cancel‚Äù appeared, pressed the {RIGHT} key to select "No" and {ENTER}. Pauses are needed so that the necessary programs can be opened and the corresponding characters entered.

Simultaneously with the execution of this code, you should not stand under a tree to do something, because the active window will switch and the sequence of commands will be broken. Then, by the way, in my example, {LWIN} {UP} {ENTER} {RIGHT} {ENTER} may occur and the computer will turn off. Of course, this module can be used more reasonably, for the same simplification of ordinary event handling.
