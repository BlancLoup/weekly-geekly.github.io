PHP internals


  • [RFC] Deprecations for PHP 7.4 - All items from the proposal have crossed the threshold for voting and will be deprecated in PHP 7.4.
  • [RFC] Escape PDO "?" parameter placeholder - Suggestion to escape a character? in PDO it is accepted and adopted in PHP 7.4.
  • [RFC] Explicit call-site pass-by-reference - An old proposal from Nikita is again in discussion. At the moment, the fact that the function takes an argument by reference is indicated only in the definition of the function itself: 
    function byRef(&$ref) {...} byRef($var);
    It is proposed to make it possible to explicitly indicate the transfer by reference. Thus warning the developer that the passed variable will change: 
     function byRef(&$ref) {...} byRef(&$var);
  • [RFC] Namespace-scoped declares - Another old RFC is up for discussion. Initially, it was proposed to make it possible to set interpreter directives for entire namespaces, instead of declare(); in each file. This can be convenient, especially if strict_operators are accepted.
    Hidden text 
     namespace_declare('Vendor\Lib', [ 'strict_types' => 1, 'strict_operators' => 1, ... ]);
    Now discussing the possibility of declaring folders instead of namespaces - this solves implementation problems.

However, after it was adopted, indignation and debate rained down. This RFC offers a milder way: if the short_open_tag option is enabled in php.ini, then the first time you use <? Instruments


  • krakjoe / tombs - Extension for finding "dead" code. Connect on production without fear of performance problems, wait a day / week / month, then look at the statistics and see which functions / methods have never been performed. More details in the post of the author .
  • krakjoe / stat - Another cool concept from Joe Watkins. Extension for profiling PHP applications on production. Provides information in a parallel thread of execution without performance overhead. More details in the post .
  • jeremeamia / iter8 - Iterate and transform arrays using generators. An alternative to the classics from Nikita Popov: Nikic / Iter .
  • cpliakas / git-wrapper - PHP shell for working with Git.
  • exussum12 / coverageChecker - The tool allows you to incrementally implement good practices. Baseline is essentially for other tools. For example, you can require 90% test coverage, but not for the entire code, but for the one that was added to the commit.
