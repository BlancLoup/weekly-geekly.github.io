WordPress 2.8.1 released. Version contains important security fixes.

WordPress 2.8.1 fixes many bugs and tightens security bolts for plugin administration pages. Core Security Technologies has notified us that the admin pages added by some plugins can be viewed by users without appropriate access rights, as a result of which information may leak. Not all plugins are subject to this problem, but we strongly recommend upgrading to version 2.8.1 for your security - the official press release says.

What else is new from the release of 2.8? Below we have summarized the most important changes since 2.8:
  • Some topics called the get_categories () function in such a way that it caused an error. 2.8.1. corrects this defect.
  • Reduced memory consumption by the control panel. Some users did not have enough free memory, which resulted in incomplete loading of the control panel.
  • Automatic update no longer mistakenly deletes files during the cleaning of the installation files and after the failed update.
  • Resolved a problem related to problems with compression, which did not load the text editor.
  • Additional security measures have been taken to better protect you from plug-ins that are not undergoing an explicit security check.
  • Now wp_page_menu () is by default sorted by the order selected by the user in the menu, and not by the title.
  • Download error messages are now displayed correctly.
  • Fixed autosave error for some IE users.
  • Fixed glitch styles in the plugin editor.
  • Syntax highlighting is disabled due to browser incompatibility.
Although there is no Russian version on the site yet, the Russified WordPress is automatically updated automatically. Enjoy the update.
