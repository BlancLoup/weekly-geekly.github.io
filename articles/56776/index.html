JuffEd 0.6.0 release

The development began purely by chance, as the answer to one comrade in the dispute "yes, damn, it's just impossible to write a Guy in C ++, you will torture everything with your hands, but Delphi ....". As a response, a simple text editor on Qt with the functionality of a la Notepad from WinXR was thrown for half an hour-hour. Then I accidentally saw an example of how code highlighting is done in Qt and fastened purely for the sake of practice. Then I realized that they could already use it, which I started to do. Well, let's go: a feature there, a feature here, a publication on qt-apps.org, features from users ...


I will say in advance: I write mostly for myself, but, apparently, someone else uses it (judging by the number of downloads). And yes, I completely agree with you that with respect to the $ {something} feature (or the set of features $ {some_feature_list}) it loses $ {something to your favorite \ editor} 100%, which, however, does not stop me or the rest, who uses it :)

The main functions repeat the standard set, which is in any decent editor:
  • editing multiple documents simultaneously (tabs)
  • syntax highlighting
  • regular expression search / replace
  • auto indent
  • highlighting of indents, pair brackets, current line
  • line numbering
  • word wrap around window width
  • show invisible characters (spaces, tabs, line breaks)


There are also some features that exist in many places, but not everywhere:
  • autocompletion (according to the current document and according to the dictionary)
  • code folding
  • block text selection (for now only to delete / copy / paste, text input in several lines at the same time will be in the next version)
  • optional backup file when saving
  • file tracing with an external program
  • the possibility of replacing entered tabs with spaces
  • ability to save named sessions
  • specifying the file encoding (with the ability to remove extra encodings in the settings, leaving only the desired set)
  • markers for strings (special marks that allow you to quickly find the marked string through the menu or shortcuts)
  • plugin system


The editor uses the QScintilla component (the Scintilla port on Qt4), so many features have something in common with other editors using the same component.

Among other changes, version 0.6.0:
  • definition and change of line endings (Unix-style, Win-style, Mac-style)
  • the ability to "split" the document horizontally and to have 2 working areas one above the other (thereafter there will be an opportunity to do the same vertically)
  • autodetection of syntax highlighting not only by extension, but also by the first line of the file (for some file types)
  • ‚ÄúActive‚Äù statusbar (the ability to choose the style of highlighting, encoding, end of lines by clicking on the appropriate label)
  • completely redesigned API plugins
  • transition to CMake build system
  • a new application icon (thanks to the SoftIcon company, which announced the contest on Habr√© and drew icons for several selected applications)
