  • Adopted by PSR-14 Event Dispatcher - A new standard regulates the work with events in the application. The document defines three interfaces: EventDispatcherInterface , StoppableEventInterface and ListenerProviderInterface , and also describes the characteristics that all elements and implementation guidelines should follow. Read more in the series of posts from one of the participants of the working group of the standard Larry Garfield.
    It is noteworthy that the new dispatcher in Symfony 4.3 , in fact, is compatible with the standard, but at least due to the use of the type of object , which is available only in PHP 7.2+, it is not possible to use the standard directly in Symfony.
    Another example of the implementation of the yiisoft / event-dispatcher standard is likely to be used in Yii 3.

Php internals


  • [RFC] JIT - The proposal is accepted for PHP 8.0, but the idea to enable the feature in experimental mode in PHP 7.4 did not find support.
    The topic is a great article by Joe Watkins about what JIT is in general and what its implementation for PHP will give .
  • [RFC] Change the precedence of the concatenation operator - Currently, concatenation operators . and math + , - have the same priority and run from left to right. It is proposed to lower the priority of concatenation:
    Hidden text 
     echo "sum: " . $a + $b; //   echo ("sum: " . $a) + $b; //   echo "sum :" . ($a + $b);
  • [RFC] Nullable Casting - When casting into a primitive type, it is proposed to make the ability to cast to null, that is, the ability to use (?int)$x in addition to (int)$x .
  • [RFC] Spread Operator in Array Expression - Again, you are prompted to add an operator ... for arrays. The updated offer removed support for arrays with non-numeric keys.
    Hidden text 
     $parts = ['apple', 'pear']; $fruits = ['banana', 'orange', ...$parts, 'watermelon']; // ['banana', 'orange', 'apple', 'pear', 'watermelon'];
