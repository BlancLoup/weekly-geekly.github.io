X Neural Switcher - Cookbook (Part 3). Keylogger

Greetings.

Today, continuing the series of articles about setting up and using X Neural Switcher (programs for automatic switching layouts), I will tell you about one important and even controversial possibility of the program.

This feature is the ability to write typed text to a log file .

I gave a brief analysis of linux keyloggers in the previous article in the X Neural Switcher series - Cookbook (Part 1). Forerunners and analogues

In general, I‚Äôm lying in my heart if I say that I didn‚Äôt think of introducing such functionality into the program from the very beginning. Punto Switcher already had the functionality of a keylogger called "Diary", so why should xneur be deprived of such a function? And it's silly not to do such a simple thing in a program that is designed to intercept the keyboard, after all!

From personal experience I‚Äôll say that I never thought that the keylogger functionality could be useful to me personally. And this option was turned on in my config by forgetfulness - it remained after testing the program. However, in one far from wonderful moment, this function allowed to expose one bad person who used my computer for purposes discrediting me. So it happened that you never know where to lay straws.

Attention! X Neural Switcher is not a spyware program, and is not intended for unauthorized use (information theft, access to someone else's correspondence, password theft and other things)!



Purpose of using as a keyloger


In general, the purpose of the application does not shine with originality, and it is quite standard for keylogger class programs.
  • track the typing on the keyboard phrases or words, the transfer of which to third parties will lead to material or moral damage
  • recover information after a computer crash
  • have a journal of typed text for all programs, not just those that support the history of typing
  • control the use of your personal computer by unauthorized persons
Of course, this list is not complete, and you yourself can come up with a dozen more options for what to use the keylogger.

General description of logging features


At the moment, the program has the following features:
  • save the typed text in the html-file indicating the program in which it was typed, and the date and time of recording in the journal
  • determine the size of an uncompressed log file, above which the log will be archived
  • send the log archive to the specified email via SMTP server (the server must be without authorization)
Yes, xneur does not know how to take screenshots, intercept mail, record video from a camera and sound from a microphone, rob a cow and call a genie. Well, what to do, xneur is not intended for that!

Logging program setup options


Below I will give the options of the configuration file that are directly responsible for the keylogger function.
  • LogSave - can be Yes or No. If No, the other options do not matter.
  • LogSize - takes the size of the log file in bytes. After reaching this size, the program will archive the log and try to send it to the email.
  • LogMail - here you specify the address of the mail to which the archive of the journal will be sent. If the address is not specified, the sending attempt will not occur.
  • LogHostIP - you specify an outgoing smtp server. Through this server will be sent. The argument can be either the IP or the domain name of the server.
  • LogPort is a port on the smtp server.
In general, the list of options is short and should not cause confusion.

Next, I will provide a specific configuration file in which all options not related to the keylogger are disabled.

Automatic switching mode is disabled, all hot keys are disabled, all notifications about the program's operation are disabled, problem applications are listed in the exceptions list - in general, everything that is not related to the keylogger is disabled.

In the ideal case, the program will not show its presence in any way, and will only write a journal and send it to the post office. Significant options are highlighted in bold.

What where When


The user‚Äôs configuration file is located here - ~ / .xneur / xneurrc.
The log file is here - ~ / .xneur / xneurlog.html
Archives of the journal are here - ~ / .xneur /, and look like "xneurlog.html 06/23/2010 09: 08: 36.gz"

Logging occurs at the time of resetting the internal program buffer, usually when the active window is changed.

If you build the program from source yourself, you can completely disable the keylogger functionality using the build configuration option - without-keylogger.
