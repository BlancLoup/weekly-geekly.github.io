3 days before the release of Fedora 12

3 days left before the release of Fedora 12 Constantine.


The list of the most interesting innovations that will appear in Fedora 12:

* GNOME 2.28 (in the alpha version - 2.27.90 beta) and KDE 4.3 are used as the main desktop environments.

* Empathy comes in as the default messaging program instead of Pidgin, due to more complete integration with the GNOME desktop.

* Support for setting up mobile broadband operators in NetworkManager, for which a large base of telecom operators with a ready-made template of settings for each of them is included. Added support for new equipment, the ability to scan GSM-networks, simplified the configuration process.

* Expanded PackageKit features, added plugins to install packages by clicking from a web browser

* Improved performance of PulseAudio - the volume control applet now supports defining different sound profiles, changes the mixer operation logic, adds the ability to switch sound sources, adds support for UPnP media servers, simplifies the process of setting sound parameters, improves support for hot-plugging devices and Surround output mode sound

* Involving the Gnote note taking program, which is written in C ++, unlike TomBoy, which requires Mono for its work. Gnote will also be included in the base install LiveCD.

* Work is underway on the liblvm library, which will provide the ability to work with LVM sections from user applications

* Improvements for developers, the kit includes: NetBeans 6.7, PHP 5.3, Eclipse 3.5.0

* Improved SystemTap dynamic debugging system (analog DTrace for Solaris), support for new Linux kernel trace calls, debugging support via debuginfo ("dwarf") output in gcc, additions to Eclipse for visual interpretation of tracing

* Support for the ext4 file system in the GRUB bootloader - despite the fact that Ext4 was offered by default since Fedora 11, GRUB did not support this file system, which resulted in the need to create an additional bootable Ext3 or Ext2 partition

* In order to save resources, starting with Fedora 12, the background process of monitoring Bluetooth services is started only when necessary and suspends its work after 30 seconds since the last access to the device;
