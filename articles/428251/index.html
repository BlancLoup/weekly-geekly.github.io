Stupid vulnerability in the application "My Beeline"

This short story consists of two parts:


  1. Detection of a vulnerability, most likely due to the negligence of developers;
  2. Report on the find in Beeline.

But this post would not have appeared if everything was so simple.


Part 1. What happened?


Yesterday, debugging my application in Android Studio, I opened Logcat, and I saw that ‚ÄúMy Beeline‚Äù flashed there. So what? I really have this application installed. Many applications write something to the log.


But. This turned out to be a HTTP request and response log. Again, this is not so scary. API request link, successful response message.


But. In addition to the link, the header contains the header and the request body, including json content (carefully formatted) and token sessions.


Then you can just make yourself popcorn, filtered by the keyword "beeline" and watch the network life of the application.


Part 2. We beat the alarm.


Of course, this can not be left. Without hesitation, I go to the website of the company mentioned above, I run around diagonally and do not find anything better than an offer to start a chat with support directly on the site. Of course, stumble on the face control in the form of a chat bot.
