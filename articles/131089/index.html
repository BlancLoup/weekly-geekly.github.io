Search PDF, DOC, DOCX files using Sphinx and PHP

Good time of day.

I want to share my experience writing a script for PHP to search for files in PDF, DOC and DOCX formats using Sphinx . Everything stated below is written for people who already had experience with a bunch of Sphinx and PHP. If not, then in due time Martin Streacher 's article ‚ÄúCreating Your Own Search Engine Using PHP‚Äù , published on IBM, helped me a lot.

Most recently, on one of the projects, which is a web interface for a user database, I received a task to organize a search for documents. The project was written before me and has already used Sphinx.

The task seems to be trivial, but a little googling, I could not find a clear instruction with a specific example of searching through PDF, DOC and DOCX file formats, which was the reason for writing this article.

Sphinx can only process XML text files. To process data in this format, use the xmlpipe or xmlpipe2 driver . That is, to index information in PDF, DOC, and DOCX files, it must be given to sphinx in XML format.

For this you need to write a script that will go to the files, read the data and give them in the right format.
Below is an example configuration and code that will help us with indexing.

In the sphinx.conf configuration file, specify the following: 
#    source SourceParseXml { #  xmlpipe2,    type = xmlpipe2 # ,    ,   XML.  xml-parser.php  XML xmlpipe_command = /path/to/php /path/to/xml-parser.php } #   index IndexParseXml { # ,    source = SourceParseXml #      path = /path/where/to/store/index-data #      min_word_len = 1 #   charset_type = utf-8 } #   indexer { #       mem_limit = 32M }
