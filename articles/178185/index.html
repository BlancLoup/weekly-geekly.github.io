VDS lineup update!

Hello!

Since the first article about VDS on Habr√©, we successfully completed the launch of the service , gained tremendous experience in providing the service and presented several new products, which we want to tell about.




Processor cores


In our new pools, we started using Intel Xeon E5 processors. They are 50% more powerful than previous processors. This means that now you have one and a half times more productive VDS for the same price (pricing goes in processor cores)!

The transition to the new generation of processors allowed the use of 4-channel memory, the bandwidth of which is almost 2 times the previous memory type!

SSD drives


After the launch of VDS, it became obvious that the disk subsystem is the weakest link. Therefore, for new virtual servers, we began to use vaults completely on SSD disks. It is expensive, but much faster!

Prices for customers for the service with the transition to SSD do not change.

Appeared mail.


Users of our virtual hosting are used to the fact that the e-mail service is included in the main service and does not even think that it can be provided separately.

Those of our customers who started using VDS often ask us a question - how to post mail. They are ready to administer web services, they know how to configure NGINX and Apache for themselves, they know how to scale the load. But mail is another matter, which is why we offered our customers to use our powerful mail cluster.

As part of the VDS update, we added an email service. Now it is possible to connect mail for management through a single panel with a single click. And the creation and configuration of the boxes has become very simple.



As part of mailbox management, you can access various TIMEWEB mailbox functions. In addition, you can use our webmail, announced in December , to work with mail.

Card binding


We all know very well that payment by plastic card is one of the most convenient payment methods. We decided to simplify the payment for services by entering regular automatic payment, it is convenient.

If you are allowed to withdraw money from your card automatically, then upon reaching a certain threshold - the money will be debited as payment for the next month of work. You will receive SMS and email about the operation.

However, we do not store your credit card details. The payment system simply repeats the successfully completed payment with a new amount, which ensures the maximum degree of protection of your personal data.



VDS cloning


For our clients building high-load solutions on dozens of virtual servers, we have prepared the possibility of cloning VDS. This feature has reduced the time required to scale the cluster. Now you can copy several virtual machines and, after making the necessary changes to the server configuration, immediately include them in the cluster. Deploying scalable systems has become much easier.



Automatic snapshot creation


What can be more tiresome than manually creating snapshots everyday? =) We have automated the backup process. Now you can adjust the number of copies and the frequency of backups. You are protected from possible failures.

We have significantly reduced the price of snapshots. Now the storage of 1 Gigabyte will cost you only 4 rubles.



CD boot and single user boot


One of the first features that we introduced after the previous announcement. It allows you to perform a number of operations that are impossible in the normal boot mode. Here is a brief list of the main tasks that this feature solves:

  • change root password;
  • change the size of the main disk;
  • system recovery after failure;
  • root file system check.




Updated OS set


We analyzed the popularity of installing operating systems on VDS and conducted a number of replacements. Were removed rarely used operating systems such as Scientific linux and added new, more popular (Ubuntu 12.10, Fedora 18).

API


From the very start of the service, there have been many requests for an API. It was in our plans, so we were happy to engage in its development. The API works on the principle of POST requests, which makes it easy to write requests in any programming language.

API documentation is available at: help.timeweb.ru/api

User documentation for the service


We have expanded the service documentation to include the most frequent scenarios for using VDS, some new operations that have become possible in the updated panel (for example, changing the size of the main disk), as well as the traditional FAQ that will help answer frequent questions.

Development and plans


We are not satisfied with what has already been achieved and continue to work on improving the VDS service. Follow the announcements on our website and blog on Habr√©.
