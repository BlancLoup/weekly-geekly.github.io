Wordpress 2.1 Ella

The new version of the popular CMS for blogging - Wordpress - officially introduced on January 22. Wordpress 2.1, like Windows, was codenamed Ella - in honor of Ella Fitzgerald. The developers have fixed more than 500 errors and improved the administration panel interface. The new version contains a solid list of changes, which obviously will please everyone who uses Wordpress to manage their blog.

First of all, it is worth noting the autosave feature already familiar to other popular services. When writing a post, the text is periodically stored in ‚Äúdrafts‚Äù, so the probability of losing important information due to a sudden loss of communication with the Internet is minimized.

Switching between editing modes - WYSIWYG and HTML - can now be done "on the fly" - switching tabs next to the text editing field. This is much more convenient than the pop-ups that were used previously. In WYSIWYG mode, spell check is available. Compared to previous versions, the file upload interface has been significantly improved.

The owner of the blog can now prevent the search engines from indexing the site without directly editing the code: this function has been added to the administration panel.

Developers claim that Wordpress performance is improving: the system now works better with the database, and controls configured with AJAX work faster. At the same time, MySQL version 3 is no longer supported, which can bring problems not only to those whose blogs are hosted on "unrefined" hosting, but also to those who use old plugins.
