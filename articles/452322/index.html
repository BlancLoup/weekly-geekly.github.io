From the epicenter of Google Cloud Next '19: CEO field notes

If you want to know where the wind is blowing from, go to the heart of the hurricane.

Approximately the feeling came from attending the Google Cloud Next '19 conference, where for three days developers, product managers, data experts and other bright minds shared the hottest news in application development and architecture, security, cost management, data analysis, hybrid cloud technologies, ML and I, off-server data processing, etc.‚Ä¶ 30,000 people, flows in industry and technology, almost non-stop lectures. This is a stir!

Let me tell you some important news about Google Cloud in 2019-2020. and its 122 updates are not to be missed, based on the notes of Vlad Flax, CEO of OWOX , who attended the conference.

Products: Anthos (formerly Cloud Services Platform)


While the product is in a closed beta, but the developers are already waiting for an acquaintance with Anthos , which allows them to write and run hybrid applications regardless of the development environment. Locally or in the cloud, you are guaranteed the same level of security and ease of access.

Thanks to the system of containers and microservices, your virtual machines, servers and services can be launched from various points of the planet (and possibly the Moon)



In Anthos:

  • GKE On-Prem, the key element, is responsible for the local orchestration of container infrastructure.
  • For microservice management - Istio
  • For cloud capabilities - GCP

Although all the technical stuffing is based on open source, this luxury in combination is still worth the money: from $ 10,000 / month. See Anthos video presentation here .

Products: serverless, but still slightly cloudy Cloud Run (BETA)


Another joy for developers, which will finally answer the question of how to run applications from half a ping without hating the choice between local or server data processing.

Synthesizing a product that solves the problems of scaling, supporting and configuring servers, and the limitations of standard procedures, libraries, and local solution languages ‚Äã‚Äã(read, crossing bulldogs and rhinos) for web application developers, Cloud Run has appeared, so far beta.

The point is: to take everything useful from these two approaches. Place the HTTP container without saving the state, working in any language for which all the necessary libraries and dependencies will be automatically pulled, in a full-fledged ‚Äúcloud‚Äù, where the server's capacity will be automatically tightened if the traffic load changes. Sounds hard?

But it really brings more time-to-value in the development process: no more thinking, choosing-asking the price for server-side, container development is easily managed and adapted using the Cloud Code . Created, launched, got the result.

API Management: Hybridization


Apigee hybrid (beta) is a new feature of the API management system that provides the ability to manage the API in hybrid cross-cloud environments: in your data center or in a public cloud environment.

It also includes many updates to expand access to the basic functions of Google Cloud. As well as the necessary security alert APIs .

This again simplifies the execution of business tasks that want to go everywhere in the shortest and safest way.

Data Management: From Anywhere, From Anything.


Data management in Google Cloud is based on the principle: no matter how much data you have, let it be easy to manage, transfer and access this data. Therefore, this year the following service improvements were announced:

  • Cloud SQL under Microsoft SQL Server (in special display mode)
  • CloudSQL under PostgreSQL, now version 11 is supported
  • Multiregional copy of Cloud Bigtable is available.

And a lot of security and access issues got their updates. For example, improved management of the file system of the Google Filestore cloud , a unified system for coordinating access to the database is now more than convenient, and full access via AWS V4 keys will become fully available after beta testing.

For business, this is again good news, because it speeds up and simplifies access to managing your data from anywhere in the world and expands the operating and software and technical base for Google Cloud.

To start using this service you will not need to do anything superhuman soon. After all, another precept of Google - to simplify everything for a non-technical specialist, while maintaining an adequate level of security and privacy.

Networking


Depending on the type of organization, you have the option to choose one of four schemes that provides the fastest traffic to internal or external IP addresses, from local to global, to your Google Cloud, BigQuery services or third-party SaaS services via VPN or Cloud Interconnect. By the way, is 100 Gbps a lot or a little for Interconnect? Such a connection capacity will delight any giants of the data world!

In addition, there is already a Traffic Director in beta that fully covers the SLA and is designed to cope with hundreds of products and services at your disposal.

If you came up with what kind of application you would like to have, more than likely, Google has already released its beta.

Security and Identification


More than 17 updates designed to save you from even accidental data loss, information vandalism, phishing, etc. Additional protection of individual perimeters, the definition of malicious software, crypto-mining or DDoS attacks based on proprietary computational models.

Who knows more about security flaws than a company that professionally collects data?

Analytics: the most important and interesting


Data Fusion (Beta) is a fully managed, native cloud data aggregation service designed to accept and integrate data from various sources into BigQuery. All ETL processes and data structures, regardless of complexity and type, will be presented in a single interface with a high level of protection, ease of data visualization in real time and integration with other services.

But from the point of view of the OWOX practice, data lake is still a lot of work to build an analytical structure for the daily business needs.

BigQuery Data Transfer Service now supports more than 100+ SaaS applications, automating the collection of data on a schedule without a single line of code.

But if you are looking for personal solutions, you still have to sweat.

Google makes it easy to migrate databases of any size from platforms such as Teradata, Amazon Redshift, Amazon S3, and more ...
