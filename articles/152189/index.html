Problems of the updated App Store

Chris Newman, executive director of Lightwood Games, commented very critically on the changes made by Apple in the latest version of iOS 6. According to his assessment, App Store innovations are a disaster for indie developers. We consider it necessary to share the views of Chris Newman with the Russian developers and readers of Habr (with a full translation of the text about the dubious changes in iOS 6 can be found on the App2Top resource).



Problem 1. You cannot sort applications by release date

This is a serious problem for small independent teams. If earlier they had the opportunity to publish their application in the New Releases category and at least somehow appear in front of users, now they have no such opportunity. New items no longer stand out in a separate category. User choice is limited to the categories Paid, Free and Top Grossing, which, according to Newman, are hardly distinguishable in content.

Problem 2. New search and its bugs

Previously, a search query in the App Store on the iPhone screen displayed up to five results, now the app store is limited to only one, obviously, the most popular application. It turns out that even within the search (what can we say about the tops), if you are not the first, then there is practically no chance to declare yourself.



Another flaw in the search is the inconsistency of the results with the search queries by categories and by the name of the developer company. For example, in response to a request from Newman's Word Puzzle, the search engine issued a bird puzzle, to the request of Lightwood Games a Clintwood games is issued.

Problem 3. Vertical version of the application store

In the App Store before it was not possible to display the store data horizontally, but it was possible to draw screenshots horizontally. Now they can be post only vertically, which means that screenshots of ‚Äúhorizontal‚Äù applications will be small and illegible.

Problem 4. Minimalism of the application card

The application card in the app store of the new version of iOS is also far from ideal: there is practically no place for the name of the application itself. So if the name of your program consists of two words, then the second is unlikely to be squeezed into a line.

Problem 5. Limitations of direct links

Previously, when logging in from a mobile device, for example, at lightwoodgames.com, the user selectively entered either the Lightwood Games website or directly on the App Store with a full list of developer games. Now, when choosing the second, an error is issued.
