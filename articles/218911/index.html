The book is available in print and electronic form.

Annotation:



Write programs that rely directly on the Linux kernel services and on the main system libraries. The author of this universal book, Robert Love, participated in the development of the Linux kernel. The book is a system programming guide for Linux, a reference manual for Linux system calls, and an insider story on how to write faster and smarter code.

Love clearly distinguishes between standard POSIX functions and special services that are offered only on Linux. This updated and updated edition, in which a new chapter on multithreading has appeared, allows to take a closer look at Linux from both a theoretical and an applied point of view. The book covers a wide range of aspects of programming, in particular:

  • Overview of Linux kernel, C library, C compiler
  • Basic input / output operations, such as reading and writing files
  • Advanced I / O Interfaces, Memory Allocation, Optimization Techniques
  • System Call Family for Basic Process Management
  • Advanced process control techniques, in particular, in real time
  • Threads, Threading, Pthreads
  • File and directory management
  • Interfaces for memory allocation and optimization of access to it
  • Basic and advanced signaling interfaces, their role in the system
  • Clock management, in particular, POSIX clocks and high-resolution timers


Robert Love used Linux and participated in its development at the dawn of the history of this operating system. The author made a significant contribution to the creation of the Linux kernel and the GNOME desktop environment. Robert works as a department manager at Google, where he participated in the design and creation of the Android operating system.

The book "System Programming in Linux" should be in the library of every specialist who is seriously engaged in writing applications for Linux or who simply wants to improve their programming skills. Robert has created the most important resource for professional Linux programmers.
