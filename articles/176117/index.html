Features of logical comparison in PHP

On April 4th, a question appeared on stackoverflow regarding the operation of comparison operators in PHP. Almost immediately he received a detailed response. Surely for many it is an interesting topic.

Question


PHP is famous for its type conversion. I spent a lot of time looking for the basics of the comparison logic in it.

For example: if $a > $b is true and $b > $c is true, does that mean that $a > $c also true?

Guided by the simplest logic, I can assume that this expression is also true, but I do not really trust PHP in this question. Can someone give me an example in which this statement is false?

I am also interested in the work of the ‚Äúmore‚Äù and ‚Äúless‚Äù operators. Will the comparison result when turning the expression: 
 # precondition: if ($a === $b) { throw new Exception( 'both are strictly equal, can not compare strictly for greater or smaller' ); } ($a > $b) !== ($b > $a)

For most combinations of types, the work of the comparison operators more / less is not documented.

Answer


The comparison operator in PHP at some points differs from the canonical definition:

The equality relation must be reflexive, symmetric, and transitive:

  • The == operator is not reflexive in PHP, i.e. $a == $a not always true: 
     var_dump(NAN == NAN); // bool(false)
    Note: The fact that comparing with NAN is always false is not a feature of PHP. This behavior is defined in the IEEE 754 standard format for representing floating-point numbers ( explained on stackoverflow );

  • The operator == symmetric , that is, $a == $b and $b == $a always equal;

  • The operator == not transitive , that is, $a == $b and $b == $c does not mean that $a == $c : 
     var_dump(true == "a"); // bool(true) var_dump("a" == 0); // bool(true) var_dump(true == 0); // bool(false)

The relation <= / >= should be non-reflexive, antisymmetric and transitive:

  • The <= operator in PHP is not reflexive , that is, the expression $a <= $a not always true (see the example for == );

  • The operator <= not antisymmetric , that is, the truth of the expressions $a <= $b and $b <= $a does not mean that $a == $b : 
     var_dump(NAN <= "foo"); // bool(true) var_dump("foo" <= NAN); // bool(true) var_dump(NAN == "foo"); // bool(false)

  • The operator <= not transitive , that is, the truth of the expressions $a <= $b and $b <= $c does not mean that $a <= $c (the example is the same as for the == operator).

  • The <= operator <= not complete , that is, both $a <= $b and $b <= $a can be false: 
     var_dump(new stdClass <= new DateTime); // bool(false) var_dump(new DateTime <= new stdClass); // bool(false)

The relation of strict inequality < / > should be anti-reflective, asymmetric and transitive:

  • The < operator in PHP is anti-reflective , that is, $a < $a always false. This is relevant since PHP 5.4. In previous versions, INF < INF is true;

  • The operator < not asymmetric , that is, the truth of the expression $a < $b does not mean that !($b < $a) is true (see the example for <= );

  • The operator < not transitive , that is, it does not follow from the truth of $a < $b and $b < $c that $a < $c also true: 
     var_dump(-INF < 0); // bool(true) var_dump(0 < TRUE); // bool(true) var_dump(-INF < TRUE); // bool(false)

  • In addition: The operator < not trichotomous , that is, the expressions $a < $b , $b < $a and $a == $b can be false (the example is the same as for <= );

  • In addition: The operator < can be looped around , i.e. there are cases when $a < $b , $b < $c and $c < $a are true: 
     var_dump(INF < []); // bool(true) var_dump([] < new stdClass); // bool(true) var_dump(new stdClass < INF); // bool(true)
    Note: This example generates a warning of the "level of stdClass could not be converted to double" level notice.

You can find some amazing graphs in the PHP Sadness 52 article - Comparison Operators .

And in the end I want to note that two equalities in PHP are guaranteed (unlike almost everything else) because the interpreter leads them to the same form: 
 ($a > $b) == ($b < $a) ($a >= $b) == ($b <= $a)

UPD: Second Answer


In PHP (relevant for version 5.5.0beta2) there is no strict comparison> == or <== with type checking, but there are several ways to compare types before comparing more / less:
  1. Compare if ( gettype($a)===gettype($b) ) ... variable types if ( gettype($a)===gettype($b) ) ...
  2. Explicitly cast the variables to the required type if ( (string)$a===(string)$b ) ...
  3. Use manipulation with if ( ($a.'')===($b.'') ) ... types

Also note the following:
  • Floating-point numbers have limited accuracy;
  • The constants NAN and INF are of type float;
  • Comparing INF with INF is mathematically incorrect;
  • The numbers in the e-notation are of type float, even if they are small;
  • Integers above PHP_INT_MAX are automatically converted to floating point numbers;
  • Numbers with a floating point beyond the limits defined by the system contain INF;
  • Undeclared variables return NULL;
  • When assigning, integers starting with 0 are converted from octal to decimal;
  • When casting a string to an integer, it loses leading zeros.
