Prehistory


I'll start from far away: about two months ago, a wonderful box equipped with a GPS receiver and a GSM module that can send data via GPRS came to me from China. Initially I took the box for my own fun, but then I had the idea to throw it into the car, connecting it to the onboard circuit.

And the meaning of a simple box?


So I thought so ... Just a box in itself does not carry anything, because she just sends the data to where she will be told and how she will be told. It should be noted that there are ready-made services for such devices, in which you only need to register and for a small fee they will keep track of such a device, but it seemed to me interesting to implement such an application myself. This is where the Trackkr project came from , which is currently under development.

And here is Google.Latitude?


Given the way the GPS tracker and Trackkr described above interact, nothing prevents you from implementing a client for Android and iOS, and each user will be able to independently set up his own copy of Trackkr and not transfer his location data to third-party organizations (As feared in the Federation Council) .

Briefly about the project


The project itself is written in Python using Django 1.4. It is a daemon listening to the port for messages from GPS trackers and a web-interface showing the location of devices on the Yandex map. We plan to package it and prepare for storing large amounts of data, taking into account MySQL replication.
