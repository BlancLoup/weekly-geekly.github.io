Cloud or VPS? What to choose? Quick Start Guide

In this article, we briefly describe the difference between cloud VPS and classic VPS . This will help you in deciding what to use. If after reading you have any questions, we will be happy to answer in the comments. If you can not leave comments on Habr√© - you can ask a question in our community .



Difference between cloud and classic VPS


You buy VPS for a specific period (for example, a month or a year), and in the cloud you pay for the resources used by servers depending on their consumption. Cloud VPS are virtual servers of the new generation, which can solve equally well both the simple and the most important tasks thanks to scaling and the most modern technological platform.

Benefits of Cloud VPS

  • The ability at any time to create the required number of servers of the desired configuration
  • The ability to change the CPU speed, the amount of memory and disk if necessary (much can be changed even without rebooting the servers)
  • Support for autoscaling , allowing you to change the available resources of cloud servers, depending on the load
  • Hourly rate depending on the allocated resources from the balance in the control panel (no subscription fee, only a one-time replenishment of the balance is required at registration)
  • When the server is stopped, payment for CPU and RAM resources is not charged, since these resources are released.
  • Failover platform with triple replication: exceptional reliability of data storage. Data is stored in 3 copies on different physical servers.
  • Enterprise SSD ‚Äì caching: the highest performance of the disk subsystem on PCI ‚Äì Express SSD and SSD DataCenter drives. (especially important when working with loaded databases)
  • Built-in HTTP Load Balancer
  • Built-in firewall
  • Hardware virtualization support: the ability to control the operating system kernel if necessary
  • Ability to create server templates: once you set up a server, you can create its template and at any time create new servers from it
  • Optimized for use in the corporate sector of the market: we have already tested the cloud in the most important use cases in companies and recommend it to corporate users.
  • Rest API support: you can manage the cloud programmatically by sending requests to the cloud management server
  • Fast external network up to 1 Gbit / s, unlimited network inside the data center
  • 3 TB of external traffic per month as a gift

Advantages of classic VPS

  • One of the lowest prices on the market
  • Fixed monthly / annual payment
  • VPS is ideal for simple projects and sites.

How much is?

Tariffs for classic VPS are available on the official page . For cloud VPS, you can find out the approximate cost without using autoscaling (with autoscaling you can save a lot) on the official page . Fully tariffs are listed in the section "Tariffs and calculation of resources." Check for cloud VPS is actually predictable. You install in the control panel the amount of resources needed by the server. Even if you use autoscaling - set limits FROM and TO.

Conclusion

If the goal is maximum savings, the project will not grow and if you know in advance how many resources you need - use the classic VPS from Infobox. This is the easiest to use and proven technology for your projects.

If maximum performance is important (especially the disk subsystem), maximum fault tolerance, or if you plan to use the service to allocate corporate resources, you need to use cloud-based VPS from Infobox. The cloud can save you by reducing the amount of resources available to servers when the load is less. On the other hand, the cloud can allow your project to withstand the growing load, as you can simply add the necessary amount of resources to choose from (CPU, RAM, disk) server.

If you still do not know what to choose, use the cloud-based VPS from Infobox.
