The new version of the ECM-system eDocLib 2.5 has been released

In December, we released a new version of the eDocLib 2.5 corporate content management system. Changes affected the interface, search capabilities and integration with other applications, settings and administration.
Details - under the cut.

eDocLib is a universal platform for managing documents and information and staff interaction. Among the main functions of the solution are the management of documents, document images (Document Imaging), electronic records (ERM) and data, workflows (Workflow), web content (WCM) and multimedia content (DAM), as well as knowledge management. The solution also allows you to organize collective work (Cooperation) and create custom portals and specialized information systems.
Interface

Access to the documents of the system can now be done both through its own interface and through Windows Explorer, which is quite convenient, because everyone is familiar and everyone knows how to do it - it will not take long to learn. The necessary documents can be stored in Personal folders, and, access to them can be provided to other users - this is required when working together.
The program interface is unified with familiar Windows applications.


Search

The concept of a floating range has appeared, i.e. You can set not only the exact date to search, but also a range of several days around this date. You can find the document and the date of the last edit. You can also conduct a search for "everything" - and by the details, and by the contents of the file.
Another innovation is the ability to set the format of the print form in which the list of found documents will be displayed.
EDocLib 2.5 introduces a new mechanism for displaying data from document sections in search results. Section properties when creating a query allow you to customize the appearance of the data. You can customize the order of output details with the help of various formatting elements, as well as determine the maximum number of displayed section entries using various formatting elements, a table with which you can adjust the output order of individual details.
In addition, the search query used to add links can now be customized by each user at his own discretion.
Integration capabilities


Expanded integration with Microsoft Office products. For example, emails that are in Microsoft Outlook can be registered as documents in eDocLib, and eDocLib documents can be sent using Outlook. In this case, registration in eDocLib can be both manual and automatic (pre-set).



Integration with the in-line scanning system allows you to start the scanning process from the eDocLib system and attach a scanned electronic image of the document to the document RC.
Simplified configuration and administration
Improved mechanisms for entering information into the system. The export / import utility now has the ability to save user settings for later use. For example, this way you can save data about which directories and documents were exported and according to which rules.
Developers using eDocLib as a platform for creating their own applications will surely pay attention to some of the innovations. For example, in the new version, it became possible to execute the program code in C # while storing the document. Added the ability to handle form events (trigger mechanism), which allows to implement more complex business logic using eDocLib. More information about the possibilities of developing applications on the platform eDocLib - in one of the following posts.
We do not intend to rest on our laurels and are already working on the next version of the product. We plan that it will provide work through an SSL connection, as well as it will be possible to use EDS and encryption.
