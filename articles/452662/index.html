Work with databases


What interesting features does the DL380 Gen10 server have for working with SQL?

HPE DL380 Gen10 supports Persistent Memory technology, which provides bit-based memory access, which reduces latency and increases the number of transactions by up to 41%. The test results of such configurations are in the public domain .


NVDIMM technology allows you to work with a huge number of I / O queues - 64k, in contrast to SAS and SATA with 254 queues. Another important advantage - low latency - 3-8 times lower than that of SSD.

Similar test results are for Oracle and Microsoft Exchange systems.



In addition to NVDIMM technology, Intel Optane devices are actively used in the arsenal of tools to accelerate database performance, which can be tested on HPE hardware at Selectel. Preliminary test results were published on the Selectel blog.

Features and technologies


The HPE Proliant Gen10 server has several unique technologies that allow it to stand out among other servers.

First of all - safety. HPE introduced Run-Time Firmware Verification, a technology that allows the server to check the signature of the firmware for its origin before installing it on the server, this avoids its being changed or installed root-kit (malware).

Processor types


HPE ProLiant Gen10 is available with five CPU categories:

  • Platinum (8100, 8200 series) for ERP, in-memory analytics, OLAP, virtualization, containers;
  • Gold (6100/5100, 6200/5200 series) for OLTP, analytics, AI, Hadoop / SPARK, Java, VDI, HPC, virtualization and containers;
  • Silver (4100, 4200 series) for SMB, web front-end, network and storage applications;
  • Bronze (3100, 3200 series) for SMB loads.

For all customers, in addition to this, there are a number of innovations in Gen10 servers:



Workload Matching - automatic adjustment of all server parameters to a specific type of load, for example, SQL. Measured results show a difference in performance in comparison with the usual settings up to 9%, which is quite good for customers who want to get the most out of their server.

Jitter Smoothing - saving a given processor frequency without parasitic peaks after turning on Turbo Boost, ideal for customers who need to work at higher frequencies with minimal delays.

Core Boosting - allows you to get an increase in the frequency of the processor, reducing overhead. Ideal for customers using software licensed by number of cores, such as Oracle. The technology allows you to use less cores, but with a higher frequency.

Work with memory


  • Advanced ECC / SDDC : memory error checking and correction (ECC) combined with single device data correction (SDDC) ensures that the application continues to operate in the event of a DRAM failure. The server firmware removes the failed DRAM card of the entire memory and restores the data in the new address space.
  • Demand scrubbing : overwrites the corrected data into memory after recovering a correctable error.
  • Patrol scrubbing : proactive search and correction of correctable errors in memory. Patrol and demand scrubbing work together to prevent the accumulation of correctable errors and reduce the possibility of unplanned downtime.
  • Failure DIMM Isolation : Identifying a faulty DIMM allows the user to replace only the failed DIMM.

Learn more about the technologies used on the HPE website .
