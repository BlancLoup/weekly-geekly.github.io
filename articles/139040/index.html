Centralized syslog

Hey. In view of the growing number of servers that are admins, and all sorts of other devices (routers, IP phones, etc.) in their IT infrastructure, there is a need to collect logs in one place and have a more or less readable interface for their output. Maybe I‚Äôve been inventing a bicycle, but at that time I didn‚Äôt manage to google something suitable, and I wanted to create something myself.

A bunch of syslog-ng + PostgreSQL was chosen to collect logs, and a simple perl web-interface was written for fetching from the database and visualization. All this is successfully virtualized in FreeBSD Jails, but it does not matter, it will work on any UNIX-like OS. I will not cover here the installation and configuration of all software (there is a lot of material on the network), but only the necessary.
So, to make syslog-ng add remote logs to the database, enter the following destination in the syslog-ng configuration file:

destination d_pgsql {
sql (type (pgsql) host ("<your database server>") port ("5432") username ("<your username>") password ("<your password>") database ("<your database name>") table ("<table>") columns ("host", "facility", "priority", "level", "tag", "date", "time", "program", "msg") values ‚Äã‚Äã('$ HOST, $ FACILITY, $ PRIORITY, $ LEVEL, $ TAG, $ YEAR- $ MONTH- $ DAY, $ HOUR: $ MIN: $ SEC, $ PROGRAM, '$ MSG') indexes ("host", "level", "date", "time", "msg"));
};

We link source to a new destination, restart syslog-ng (didn't you forget to create a database and a table before this?). Everything, logs should pour in a DB now. One important point - be sure to create indices in the database, otherwise you will see something like ‚ÄúRequest time out‚Äù in the browser with a large database.
