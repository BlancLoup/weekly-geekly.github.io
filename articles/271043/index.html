Isilon.Next - how Big Data storage has become larger, cloudier and softer *

* play on words with English. "Software"



I share with you the news that, on the basis of the Isilon storage system, the integration of Big Data, clouds and software-defined storage continues. November 10 was officially announced the long awaited Data Lake 2.0, aka Isilon.Next.

From the main point, you should pay attention to the following features in the new version of the product:
  • cloud storage can be used as a storage level in Isilon
  • software storage nodes
  • geographically distributed core-edge cluster architecture
  • full continuity of work




Isilon is a storage system specifically designed to work with very large amounts of data (from tens of terabytes to tens of petabytes), at speeds of millions of operations per second .

And that's what's new in Isilon.Next:

CloudPools - Isilon distributed cloud storage extension


  • Allows you to work with cloud storage provided by service providers via the S3 protocol
  • Can connect EMC Elastic Cloud Storage object storage as a cloud resource.


IsilonSD - the first fully software version of the Isilon node


  • It can be used to join a 6-node Edge-cluster.
  • Can connect to the core (Core) of the cluster is transparent.
  • It can be both a source for data aggregation and a target system for their distribution from the core.


New OneFS microcode version 8.0 with improved features to ensure continuous operation


  • Supports SMB 3, with automatic client reconnection without stopping work.
  • Full continuity of operations of the whole complex, including during microcode updates (as well as in the case of rollback).


Why is this update important?
Firstly, Isilon, who came to IT from the media field, confidently occupies the corporate field of Big Data, where the requirements for continuity of access to information grow along with their volumes. The system containing Petabytes of data in itself can no longer be so easily stopped for routine prevention. And batch processing of huge amounts of data requires uninterrupted access, even in the case of individual hardware failures. Uninterrupted service of high-speed data access should be maximized.

Secondly, confident movement in the direction of the clouds, decoupling the storage functionality from hardware, and providing it in software form, gives customers more freedom. And it is especially good that this software solution is compatible with the Isilon hardware, which, I don‚Äôt say, has already won a very high level of trust and reliability. That is, Isilon customers get the flexibility of a software solution compatible with the hardware platform.

Thirdly, there is little doubt that the most economical storage is in the cloud. At the same time, few storage systems can use clouds as an economical level of storage, as a deep archive. Isilon.Next - now not only can store data in the cloud, but makes the clouds part of its automated storage functionality. Moreover, cloud storage technologies are not only savings, but also geographically distributed data protection and uninterrupted access to them . And this is a very serious plus for Petabyte-scale systems.
