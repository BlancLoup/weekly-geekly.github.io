Netbeans IDE 8.1 - Yet RC

A stable release of a wonderful new version and, most importantly, a free IDE for programming languages ‚Äã‚Äãsuch as PHP , Java , C / C ++ and others is about to be released. In the meantime, it is running around the Release Candidate (RC) run-in of this version, let's see what has changed in this IDE .

The main changes (not taking into account changes in the support of frameworks and other things) compared to version 8.0 are directly indicated by the developers themselves:
  • Direct work with Node.js.
    For me, as a WEB developer, this is a key function, which in turn draws many other good improvements, such as normal support for Gulp and Grunt .
  • Oracle WebLogic Server Remote Control Support
  • Support for server application WildFly 9 and 10 versions

Node.js

  • Node.js project support
  • Manage npm packages directly from project settings
  • Running and debugging Node.js applications


HTML5 / JavaScript

  • Gulp support



  • Mocha and Selenium support
  • Jade Template Support
  • Extended Grunt support
  • Expanded support for AngularJS and KnockoutJS
  • Projects now recognize package.json , bower.json and composer.json


Java

  • Improved auto code completion
  • More convenient and clear navigator

  • Improved control in the settings window

Java ee

  • Oracle WebLogic Server Remote Control Support

  • Support for server application WildFly 9 and 10 versions

Php

  • Phing support
  • Advanced Composer Integration
  • Code Coverage with Nette Tester
  • Integration with the symfony installer


C / C ++

  • Mixed development support in C ++ / Java
  • Improvements in the "Project Creation Wizard"
  • SVN , Git and Mercurial are supported in remote mode.

  • C ++ editor improvements
  • Improvements checks, tips and refactoring
  • Backward Development Support (Call Graph / Call Graph)

Profiler

  • Redesign

  • New features:
    • Real-time display of CPU status
    • Real-time memory status display
    • CPU usage monitoring
    • Dumps of threads of the profiled application
  • Engine improvements:
    • Faster connection to the profiled application
    • Restriction of outgoing calls to profiling methods
    • Memory accounting for selected classes
  • Simplified settings
  • Improved IDE integration

Other

  • Git support extended
  • And much more


More details about this version can be found on the official WIKI page of this version (in English) .
It may also interest you:
  • NetBeans 8.1 RC download page
  • NetBeans IDE 8.1 RC Release Notes (English)


I myself use version 8.1 for the JS project from the state of Beta , and my personal opinion is that the version can definitely be used.
The only nuance that has befallen me is that in Linux for direct work with npm from the project, sudo rights are necessary, but the IDE does not request them for the terminal itself.
