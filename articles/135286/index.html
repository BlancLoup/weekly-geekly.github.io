Microsoft Lync goes to mobile platforms, client review, server setup



Good time, dear Habroskoobschestvo.
In November, Microsoft released the long-awaited update for Lync Server 2010 Corporate Communication Server, which allows mobile clients to connect. However, by that time there was not a single mobile client, nor under any platform, even under the native Windows Phone. From early press releases, Microsoft made it clear that it would begin developing Lync clients for WP7, Android, iOS, Symbian and BlackBerry. Fortunately, grief had not so long.



Last week, a Lync client appeared on Windows Phone 7 (Mango) and Android, this week on iOS (iPhone, iPad), a Symbian client should come out the other day, and in January it is expected to be BlackBerry. The functionality of clients for different platforms is different, here is a comparative label:



Oddly enough, among all the competing platforms, only Android has the most truncated functionality.

The main functions of Lync Mobile for all platforms:
- information about the presence of colleagues, the choice of the optimal way to communicate - via instant messaging, e-mail or phone;
- a simple way to organize a conference;
- unified interface for making and receiving calls in corporate voice service races
- Protection of data transmission via TLS protocol, without the need for a VPN.

Call organization:
Voice communication between Lync pins is done through the PSTN gateway, using callback. The organization of calls by this method can be interpreted both from a positive and from a negative point of view:

+ The call is not through VoIP, therefore, you do not depend on the Internet coverage (Wi-Fi, 3G), the Internet in roaming is often more expensive than voice. With the use of callback you do not care about the price of the call, the company assumes all costs, subject to free incoming calls. When using this method, the main advantage for business is achieved - to be always in touch
- A more complex and expensive organization of calls using PSTN gateways. To reduce the cost of calls, you can use a SIP provider, but most of them are not compatible with Lync Server. To solve this problem, you can use Asterisk (if you are interested in how to do this, we can write an article)

Continuing the topic of criticism regarding mobile clients:
- lack of push technology on Android, unlike WP7 and iOS
- the color of the title in WP7 is always blue, does not change color when you change the theme in the phone
- poor client optimization for tablets
- the clipboard does not work in Android
- there is no possibility to make video calls and transfer files

Instructions for configuring Lync Server 2010 Mobility Service

Before installing the components, you need to install the latest Cumulative Update Lync Server 2010 Mobility Service . From the entire list, select LyncServerUpdateInstaller.exe and install on the front-end and edge server.
Also, before installing, you must make changes to the topology - add internal and external ports for the Mobility service. To do this, open the Lync Server Management Shell and enter the commands:

Using the Management Shell install the missing components for the service:

Import-Module ServerManager
Add-WindowsFeature Web-Server, Web-Dyn-Compression

Download and install Lync Server 2010 Mobility Service update
After the Bootstrapper.exe is completed, Lync Mobility services are ready to go. It remains only to update the certificates for internal and external use and add FQDN records to the DNS server.
For the internal zone, you need to create a new A record lyncdiscoverinternal.sipdomain pointing to the front-end server, and on the external DNS server A write lyncdiscover pointing to an external fqdn for the front-end.
To upgrade internal certificates, you can use the Lync Server Deployment Wizard in the Install or Update Lync Server System section and select Step 3. Using the Certificate Request wizard, issue a new certificate containing additional names lyncdiscoverInternal.sipdomain and lyncdiscover.sipdomain.
As an external certificate, you can use the existing wildcard (* .sipdomain) or order a new certificate with CN lyncdiscover.sipdomain or containing this name in the SAN field.

You can test the performance of a new service using the Test-CsMcxP2PIM cmdlet
