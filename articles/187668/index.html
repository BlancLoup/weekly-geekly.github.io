Golang daemon

About a year ago I needed to write a linux daemon that implements a small network service. At that time I was actively learning Go and I really liked this language, so after weighing all the pros and cons I decided to implement the task on it. In addition, Go was already stable and had version 1.0.1.

About what pitfalls I had to face, read under the cut, but at once I will make a reservation: I will describe only the subtleties of the demon implementation on Go. If you have little idea what a ‚Äúdemon‚Äù is or how the process is demonized, you should first read about it by searching in Google or on the ‚Äúlinux daemon‚Äù or browsing the list of links at the end of the article.

But back to the demons. At first I decided to act classically:
  • Spawning child process and terminating parent (fork system call);
  • Next in the child process:
    • Setting a mask for access rights to newly created files (system call umask);
    • Creating a new session, disconnecting from the terminal (setsid system call);
    • Change the working directory to the root (system call chdir);
    • Redirecting standard input / output stream descriptors to / dev / null.

The lack of a clean fork in the standard syscall package did not stop me and did not even arouse any suspicion. I just did something like this (simplified): 
ret, _, err := syscall.Syscall(syscall.SYS_FORK, 0, 0, 0) if err != 0 { os.Exit(2) } if ret > 0 { //   os.Exit(0) }
