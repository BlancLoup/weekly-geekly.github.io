Php


  • RFC: Null Coalesce Equal Operator - Is it proposed to implement the combined operator ??= in addition to the usual ?? added in PHP 7. 
     //     $this->request->data['comments']['user_id'] = $this->request->data['comments']['user_id'] ?? 'value'; $this->request->data['comments']['user_id'] ??= 'value';

  • RFC: Short ternary Equal Operator - It is also proposed to implement a combined ternary operator: 
     $x = $x ?: $y; $x ?:= $y;

  • RFC: "var" Deprecation - It is proposed to obsolete the keyword var . Interestingly, this keyword was already obsolete in versions 5.0.0 - 5.1.2, and at the moment is just a synonym for public .
  • RFC: Catching Multiple Exception Types - For consideration it is proposed to catch several types of exceptions in a single catch block: 
     try { // Some code... } catch (ExceptionType1 | ExceptionType2 $e) { // Code to handle the exception } catch (\Exception $e) { // ... }

  • RFC: Traits with interfaces - It is proposed to enable the use of interfaces in traits: 
     interface I { function foo(); } trait T implements I { function foo() { } }
