Localization RC (C ++, MFC, Win32)

Continuation of the publication "Localization of WPF pages . " The internationalization of C ++ projects is basically a translation of RC resources. As before, we consider the localization process using a simple example of an MFC project.

Windows 8, Visual Studio 2013

So, create a new MFC project.



Leave everything by default and click ‚ÄúFinish‚Äù.

Run and check that everything is in order and in English.



Add to our project Nuget package Elas.



As in the case of localization of the WPF project in a solution, a new Elas configuration file has appeared.

We will collect the project.

Please note that a new file ‚ÄúMFCApplication1.rc.xlf‚Äù has been added to the project.



This file contains resources that need to be localized.
This includes lines, dialogs, menus, accelerators, etc. all resources from "MFCApplication1.rc"
