What will happen if I compose a password for OS X from emoticons?

Inquisitive user Artyom Dashinsky (Artiom Dashisnky) wondered : is it possible to use emoji in the password for OS X Yosemie?

It turned out - you can.

After that, Artem had another problem. It turned out that on the authorization screen there are only native keyboards - and no emoji. How to enter your password? With such a question, the user turned to the StackExchange programming audience.

The boot disk is fully encrypted by FileVault.

Artyom sees three options here, but does not know how to implement any of them:

1. Find a way to enter emoji in the password field.
2. Some way to connect to another computer and enter the password from it.
3. Find a password recovery method despite using FileVault.

What to do in this situation?

The correct answer is from Tom Gewecke
In the upper right corner of the screen there is the ‚ÄúOther Input Sources‚Äù option, there you need to select Unicode Hex Input . With its help, you can enter Emoji or any other character, you only need to know its code. It can be found in the Character Viewer or found on the Internet. For example, the airplane U + 2708 ‚úà.

To enter a character from the Unicode Hex Input keyboard, you need to dial a four-digit code with the Option key pressed .

If the symbol corresponds to a five-digit code like U + 1F600, then you will have to find two four-digit surrogates: look in the same Character Viewer or on the Internet on sites like fileformat.info . For example, the D83D and DE00 substitutes correspond to code 1F600. In this case, with the Option key pressed, type D83DDE00.
