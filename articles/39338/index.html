Thus, the class that inherits this data - automatically gets the functionality
use get / set methods, while there is no need to redefine any methods in inheriting classes.

Of the benefits


  • Simplicity
  • Minimum redundancy
  • Hint: The get & set methods themselves can be made both public and protected - it will depend on this - whether they can be called directly (Ex. In the case of public - besides the possibility $ obj-> height = 444; it will be possible to call the method directly $ obj- > setHeight (444))


Of the minuses


  • Due to the features of the visibility of variables and methods (public | protected | private) - these constructions do not work with methods and variables of type private
