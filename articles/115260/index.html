New release of Jabber-server OpenFire 3.7.0

After two years of development, and six months after the release of the beta version of the program, the Ignite Realtime community finally presented the long-awaited release of the Jabber server OpenFire 3.7.0 to the public . The updated version, as usual, contains many new features, fixed bugs and improvements.

Here are the most significant ones:
  • Openfire is released under the Apache 2.0 license .
  • Improved OpenFire work with ‚Äúidle‚Äù connections.
  • OpenFire caching has become less voracious to memory, the cache size is calculated more correctly.
  • Fixed a bug in which it was impossible to log in as a newly installed OpenFire console under the administrator until the server was restarted.
  • Fixed a bug with displaying the statuses of users of a shared roster, if the server is configured to work with LDAP.
  • Fixed memory leak in PEP.
  • Changed logging interface.
  • Fixed a number of issues with self-signed SSL certificates.
  • Some improvements and fixes on the Multi-User Chat (MUC) page of the administrator console.
  • Some improvements have been made to existing plugins. In addition, a new plugin Jingle Nodes .
  • There are some fixes in French, Russian and Lithuanian translations of the program interface and some plugins.
A complete list of changes from the previous release of 3.6.4 can be found here . It also contains descriptions of 11 fixed bugs after the release of the beta version.

From myself I would add that I finally waited for the bug fix with the shared roster , which has existed since 2006, was repeatedly discussed in many forums, but had no solution.
