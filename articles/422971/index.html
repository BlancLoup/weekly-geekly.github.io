The algorithm works as follows. At the current iteration, the sum of the values ‚Äã‚Äãof rule * value does not exceed rule * n (since value has one of two values: 0 or 1). At the next iteration, the single value of rule exceeds this amount, that is, rule = rule * n + 1.

The number of the inertia with a unit subtracted from it (at the first iteration with a zero sum, the answer is unknown) is the answer number (for K. Neylor, the positive sum of the values ‚Äã‚Äãof the rules * value meant the first answer, the negative one - the second).

Obviously, at the first iteration (zero values ‚Äã‚Äãrule by default) the sum of values ‚Äã‚Äãwill also be zero. At the second iteration, it will be in the range [1, n] (see the formula for the rule above), on the third - in the range [n + 1, (n + 1) * n]; in the fourth range, it will be: [(n + 1) * n + 1, (((n + 1) * n + 1) * n] and so on.

Thus, the right border of the range is calculated by the formula sum = (sum + 1) * n. When all possible answers are checked, we reset rule.
