Software Defined Radio - how does it work? Part 8

Hi Habr.

In a previous article about Software Defined Radio, a question was asked how to decode RDS using GNU Radio. The RDS decoder is not so simple to create from scratch, but fortunately for us, it is already built into GNU Radio, so it‚Äôs easy to see how it works without writing a single line of code, just RTL-SDR receiver.



How it works, continued under the cut.

About GNU Radio and its installation, I will not repeat, it has already been described earlier . In short, the distribution under Windows can be found here . Let's go over to the RDS itself.

If someone suddenly does not know, RDS (Radio Data System) is a protocol used in ordinary FM broadcasting to transmit additional digital information (time, exchange rates, weather, etc.). The transmission goes at a speed of 1187.5bps at a frequency of 57KHz (3rd harmonic of the pilot tone), to minimize the frequency band, BPSK (Binary Phase Shift Keying) phase coding is used. The RDS bit level has already been considered in the Habr√© earlier , well, we move on to GNU Radio.

The decoder for GNU Radio was made about 10 years ago , but that version is already useless to watch, the names of many units have changed, and the examples cited there no longer work. Updated fork can be taken from here .

By the way, it didn‚Äôt work to compile gr-rds from source files for Windows - cmake produces errors for the absence of Boost, although it is installed. It is not clear by the way why over the years of cmake and boost, cmake has not been taught to find ways in Windows - like finding a folder on the disk is not a rocket science at all (if anyone knows the solution, write in the comments, although judging by Stack Overflow, the problem exists for years and all do not care). But it turned out that we don‚Äôt need it - the RDS decoder has already been added to GNU Radio, so from the project on github we need only examples that are in the apps folder.

The block diagram attached in the examples is rather monstrous, moreover, when opening it, errors are displayed (blocks are marked in red).



In real life, however, everything is simpler - 2/3 of the scheme is a stereo player for FM, where L + R and LR channels are extracted from the original signal, processed and fed to the sound card. It is irrelevant to us now, so these blocks can be removed (especially since out of the box for some reason it did not work, but it was too lazy to understand). Errors occur due to the Grid Position parameter, which apparently is not supported by the Windows version, but it can be removed without problems, it does not affect the functionality.

After removing ‚Äúall superfluous‚Äù, the working RDS decoder scheme looks like this:



Let's see what is there.

The source signal comes from RTL-SDR Source, the frequency is set by the freq parameter, which has the WX GUI Slider type (yes, you can create your own UI in GNU Radio, and there are basic controls). To avoid a peak at zero frequency in the center, the freq_offset parameter is used, the Frequency Xlating block shifts the frequency by this value. The WBFM Receive unit, as the name implies, performs FM demodulation, then the frequency is shifted again to allocate the RDS itself to 57KHz. The Root Cosined Filter block allocates a narrow frequency, and the MPSK decoder with parameter 2 performs BPSK decoding (RDS itself is transmitted using phase modulation with two states). RDS uses differential encoding , so the corresponding differential decoder is called, and finally the finished binary stream is fed to the RDS Decoder block (its source can be viewed on github ). After the decoder, an equally important part is the RDS parser ‚Äî there are quite a few types of packages in RDS, and the parser does all the work of decoding them.

Actually that's all. The results of the decoder on KDPV and screenshots.







If someone needs to use the program in no-UI mode, you can use the FR Tap block, a more detailed description here , I personally have not tried it. If the lower bit-rate level of RDS is interesting, I considered it earlier , it can also be useful for general interest.

As usual, all successful experiments.
