Node.js v0.2.4 and expected improvements in the v0.3.x branch

Last weekend can boast of two releases of Node.js: the next minor version in the v0.2.x branch and the official release of the v0.3.x branch.
And even though the branch being developed is still marked as ‚Äúunstable‚Äù, it expects significant improvements in performance, portability, and the addition of new features that you should know about right now. Moreover, this version is already fully operational.
Also at the end of the topic is a bonus in the form of links to the Node.js v0.2.x and NPM package repositories (Node.js package manager).

Changes in version 0.2.4:

  • Added option --eval, which allows to execute the javascript passed as a string (similar to perl -e or php -r).
  • Corrections in the module working with the network:
    • Added the ability to start writing to the queue before the connection is completed.
    • In case of an unsuccessful attempt to close the connection, the write queue is no longer cleared.
    • Fixed bug with frequent checking of released descriptors in case of reaching the limit of connections ( Do not spin on accept () with EMFILE ).
  • The option --max-stack-size has been added to limit the maximum stack size of the V8 (V8 itself also supports options for limiting the size of the memory used by the engine).
  • Fixed errors in the child_process.exec () function related to the termination of the child process (now it happens using the SIGTERM signal, not SIGKILL).
  • Fixed incorrect release of resources when shutting down Node.
Download distribution: nodejs.org/dist/node-v0.2.4.tar.gz
Documentation: nodejs.org/docs/v0.2.4/api.html

Changes in version 0.3.0 (unstable):

  • All the above fixes for version 0.2.4.
  • Multiple improvements to the readline module.
  • Fixed a memory corruption bug with incorrect constructor calls.
  • The process.nextTick () function has been fixed, now exceptions in the callback function will not affect its operation Node.
  • Buffers were rewritten again, much faster work with them was achieved.
  • When compiling modules, the -D_FILE_OFFSET_BITS = 64 and -D_LARGEFILE_SOURCE flags are added by default.
  • Simplified parsing HTTP requests strings.
  • Added support for HTTP 100 server response.
  • Simplified system of modules:
    • The require.async () and require.registerExtension () methods are removed, the require.extensions [] property is added.
    • Added public API for require.resolve () and require.cache [].
  • Changed the formatting of commas in the output in sys.inspect (), added the output of function names.
  • Constants are allocated from process to a separate require ('constants') module.
  • Fixed analysis of the amount of memory used in Linux for programs with spaces in the name.
  • The sys module is renamed util.
  • Fixed bug with non-returning non-blocking status to the stdio stream when the program was terminated by SIGTERM and SIGINT signals.
  • Improved decoding of base64 strings with additional characters (such as spaces and line breaks).
  • Added the ability to override and add commands to the REPL.
  • V8 is updated to version 2.5.1, Libev to version 4.00, libeio and http-parser are also updated.
  • Other fixes and improvements in the crypto, fs, http and net modules.
Download distribution: nodejs.org/dist/node-v0.3.0.tar.gz
Documentation: nodejs.org/docs/v0.3.0/api.html
