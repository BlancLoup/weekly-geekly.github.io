Django release 1.9

After 10 and a half months of work, the developers finally released the release version of Django 1.9.

Under the link you can familiarize with all changes in all details, but from the most tasty it is possible to note at once:

  • Support for executing actions (signals) after a transaction commit
  • Password validation support
  • Permission mixins for class-based views (in English, I think it‚Äôs clearer)
  • New styles for admin
  • Support for parallel execution of tests


Of the small but pleasant changes, I would personally note the improved customization options for the admin panel , for example, the empty_value_display parameter, with which you can set a string for output instead of an empty field value. Those. You can get rid of the inscriptions "(Nothing)" in the list without additional. code.

PostgreSQL users should like the improvements in `django.contrib.postgres`. For example, the JSONField field was added.

Full support for Django 1.9 will last until August 2016, and extended (fixes for security bugs and data loss) until April 2017. Now that version 1.9 is released, version 1.7 has finally ceased to be supported. 1.7.11 will remain the final release of this series and everyone is recommended to upgrade to 1.8 LTS, which will be supported until April 2018.
