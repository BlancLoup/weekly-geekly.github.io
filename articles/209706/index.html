System Update - ImageCMS 4.5.2

A new release of ImageCMS 4.5.2 has been released.

In the updated version, we did the following work:

1. Created a module for fast automatic translation of the site into other languages.

Now the administrator does not need to upload files with a list of words to POEdit, since the new module simulates the operation of this program.
The new module is also new options:
  • Search for untranslated words (also in the template);
  • selection of translations from previously translated similar words and automatic translation of words and phrases using the Yandex.Translate tools.

Thus, we have achieved comfortable work with the getText extension;

2. From scratch rewrote all the functionality of the basket

It was decided to completely change the functionality of the basket due to problems that had previously arisen due to its non-isolation from other components. Malfunctions occurring, for example, in Discounts, Delivery, Payments, also affected the work of the basket. Results of changes:
  • relative isolation of functional parts from each other is a big plus for ease of integration and scaling;
  • expansion of the basket;
  • Added ARI for developers.



The full developer documentation version can be downloaded here: www.imagecms.net/developers . It contains a description of all classes and variables.

Z. From scratch rewrote synchronization module with 1C

Previously, users who imported a large number of commodity items from 1C were forced to split the load. In the new version, we have significantly increased productivity - accelerated imports by 10 times.

4. In ImageCMS 4.5.2, pagination is configured separately for each template.

In this release, we have developed the ability to set the pagination style separately for each template. Recall that before setting the type of pagination was carried out only in the code of the site.


5. Introduced the ability to leave a fixed or non-fixed cost of shipping goods

In previous versions, some difficulties arose due to the presence of only two options for the specified cost of delivery - either a clear price, or free delivery. Since version 4.5.2. we have introduced the third possibility - the cost is being specified. This will help to avoid inconvenience due to the need to pay extra for delivery in some cases - if its cost depends on dimensions, for example.


In addition to all the above innovations, we also completed about 100 tasks to improve the available functionality.
