Google AdSense for Domains - now in Russia!

Last December, Google announced the expansion of AdSense for Domains , a way to monetize AdSense-based domain names. Prior to this, this service was available only to a narrow circle of partners with very high traffic rates. Since December 2008, the service was provided to everyone, but temporarily limited geographically (Web publishers from North America could only use it).

The other day, the AdSense for Domains service became available in Russia (I believe that in many other countries outside of North America).


In the Russian AdSense interface, you can start using the service like this: ‚ÄúSetting up AdSense -> Get Ads -> AdSense for Domains -> Add New Domains‚Äù. Domains can either be listed in the input field or downloaded to a CSV file. After adding domains, the system will show detailed instructions on how to configure DNS. This setting, by the way, is not very convenient compared to a number of other well-known parking systems, where it is enough to change the DNS server for a domain in the registrar interface. In order for domains to start working in AdSense for Domains, you need to be able to edit the zone (set one record of type CNAME and four records of type A).

Google domains check in two stages: the first one rejects the names that, from their point of view, violate the service provision policy (for details, see the Russian FAQ), and the second one checks the correct DNS setting, after which the domains are brought to working condition.

In order to see the statistics of AdSense for Domains by individual domains, you need to use the advanced report, specifying the grouping method "By domains".

Detailed information about the service is now available in Russian .
