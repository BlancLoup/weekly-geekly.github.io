Private mode and list of entered addresses in the new build Vivaldi 1.0.264.3

Hello!

In today's assembly, we present one of the most requested features that users have asked about with enviable consistency - the private viewing mode. In addition, we worked on improving the web panels, and also added some less noticeable, but useful features. All this is in addition to the many corrections, which list will traditionally be at the end of the article. And now a little more about everything.

Private browsing mode

This is the first implementation of this function. So far - only in the form of a private window (available through the menu File -> New Private Window). Private tabs are slightly more difficult to implement, so for now we have only added the ability to work in a separate window. In this mode, no data about your online activity using a private window will not be saved in the browser. For clarity, we added a special background to the address bar and the corresponding icon, so that you can always understand whether you are working in private mode or in normal mode:


Due to the fact that this is only the initial stage of the implementation of a function, there must inevitably be bugs and shortcomings in it. Please test the private mode and be sure to let us know about the errors found, especially in terms of privacy.

Web dashboard enhancements

We rewrote the function code for this assembly. At the same time, they fixed the bug with opening links from the web panel in the main browser window. We also reduced the size of the header of the web panel and even made it possible to turn it off. The Back and Update buttons are now merged into one Home button, which, when clicked, takes you back to the home page and simultaneously reloads it. In this context menu gives you the ability to perform other actions related to the web panel. Perhaps this is not very convenient - write what you think about this change.


We also slightly changed the + button to create a web panel: now when you click on this button, the address of the current page will be automatically placed in the address field.

Unfortunately, in my Linux version I noticed that the content of the web panel disappears after the restart of the browser. We will look for the cause and correct it.

List of entered addresses

To be honest - I do not know when this feature appeared in the browser. Perhaps - during my vacation. But, be that as it may, this function has been requested for a long time and now it is. If you place the cursor in the address bar and press the down arrow on a computer keyboard, a list of all addresses entered into the address bar will open. While the function, of course, is far from perfect and does not allow, for example, editing the list, but for a start it turned out quite well.


Known Issues

  • The private window in the Mac version looks like normal
  • Resetting web panels: if you created web panels in a previous build, you need to re-create them for a new build
