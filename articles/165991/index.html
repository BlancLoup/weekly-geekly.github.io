Magmi is the best way to import products into Magento

A few years ago I did a lot of Magento based websites. In the archive of my blog you will find several articles that allow you to solve some problems on Magento.

Recently, I again worked with Magento (v1.7.0.2). There is one thing that incredibly limits the use of Magento. This is import functionality. If you have ever imported goods through the profiles of Magento Dataflow - you know how crookedly everything is implemented there:

  • There is no possibility to import multiple images (image gallery, etc.);
  • there is no possibility to import categories;
  • there is no possibility of importing cross-selling and related products;
  • there is no possibility to import multiple prices (for different groups of customers);
  • in short, this scheme works extremely badly ...

Fortunately, about a year ago I found Magmi. If you want to import something into Magento, use Magmi. It works fine. The developer of Magmi is Seb Bracquemont .
Here is the Magmi home page: http://sourceforge.net/apps/mediawiki/magmi/index.php . I do not know why Magento does not buy the project and its developers. And probably never know. Magmi may seem a bit tricky at first, but believe me, it will make working with Magento a lot easier.
Need to import from a remote CSV? It's not a problem. Need to update your image options? Magmi can do it. Want to create all your categories on the fly and activate them in one fell swoop? Yes, it is also possible!

Magmi is a tool that few seem to use. I hope this post helps:

  1. raise awareness of Magmi, he deserves more recognition;
  2. facilitate the import of data in Magento;
  3. convince users to support the Magmi project - it‚Äôs really great
