As I did from the physical server virtual

Accountants had bit2000 and everything suited them, well, almost everything. I did not like the fact that it would be necessary to link more warehouses to the program, but this did not happen. And we moved to 1c, and the server was purchased and it was good, and the old server was not much needed, but you need hardware.

And I thought, and how would I make a physical machine virtual. He began to dig the Internet and even in the search engine - p2v, there is nothing worthwhile that could help.

How to make a mold of the system so that it starts without problems?

The Internet prompted several solutions, only I have the original windows server 2003 system, and even with crack, not everything works, you need a crack for the server. I have to say right away that Acronis will not work, Paragon Disk Server Manager can make an impression of the system, but uploads it not to local disks, but only to physical disks. Keep this in mind when you decide to use them.

Internet prompted. In Andrei Beshkov's blog there was a link to the free utility disk2vhd . Small, but remote, allows you to take a snapshot of a working system and upload it to a local disk.

A description of how the initial section of 300 gigabytes was divided into 3 sections can be skipped.



The utility is very simple, choose the disks on which the system you need stands, and choose the path - you will have to register it manually. I have not tried it, but, probably, the program can also transfer snapshot of the system to network drives and folders. I also ticked ‚ÄúPrepare for use in Virtual PC‚Äù, although instead of Virtual PC, I have Hyper-V. Help is not working.

After creating a virtual disk, create a virtual machine.



Choose the 1st generation.



We install the necessary amount of RAM - it will stand out from the physical, in my case, the virtual machine rotates at 1 gigabyte of RAM, if you set it up more, it simply does not start - there is not enough memory, not only it works on the server.
Consider this moment.



We do not select a network adapter yet.



We specify to the virtual machine to use the created virtual disk by us.



In this configuration, my virtual machine works.

We proceed to configure the virtual machine.



We delete the existing network adapter - it will not work on the server, the server that was zavrutalen simply will not see it.



We set the number of processor cores that we allocate for this virtual machine.



You can set the desired amount of RAM.



Boot from disk, installing a new system is not required.



We install new equipment - ‚ÄúOutdated Network Adapter‚Äù, our windows server 2003 will see it.



Select the desired virtual adapter. In my case, since another proxy server acts as a proxy server and a separate network card is allocated for the Internet and its network is not accessible from a normal local network, I chose a virtual adapter created for the second network card using our local network.

Everything is ready, you can run.



After the launch, there will most likely be some surprises waiting for you - the mouse may not work. Configure the network card will have to use the keyboard. After setting up the network card, we managed to connect using Radmin, directly to our new virtual machine.

By the way, when I tried to increase the screen resolution from 800x600, the system produced a blue screen of death, most likely this is due to the small amount of RAM I allocated for the virtual machine.

Everything is very simple done.
