Apple fixes an important iCloud vulnerability

Apple fixed a dangerous vulnerability in its iCloud service, which allowed users to sort through Apple ID account passwords, bypassing the limit imposed by the security system on the number of incorrect password entry attempts. The source code of the exploit for this vulnerability called iDict was previously posted on the GitHub service. According to the author, this tool could also bypass the Apple ID two-factor authentication system.

A 100% Working iCloud Apple ID.





A few months ago we wrote about another, but similar to this, vulnerability in iCloud, which allowed attackers to carry out a brute force attack on Apple ID accounts. The tool was called iBrute and its source texts, at one time, were also laid out in open access.
