qutIM 0.2 beta - now banana

After three months of anxious waiting, the development team of the anniversary 300th commit√© presented us with a beta version of the wonderful cross-platform messenger qutIM. During this time, many new and interesting things have been done, many have been rewritten, broken and corrected, new functionality has been added and work has been optimized. Since the release of the alpha version, it has been downloaded by more than 90 thousand people (excluding regular assemblies and main revisions), in the beta version we hope to increase this number :)

(alternative image link)


The most pleasant news, perhaps, is that the core of the program has been divided into so-called large layers (CL, chat, history, etc.), which can be replaced with their implementations as a plug-in. For example, this allows you to make your own implementation of the contact list, chat windows, notifications, and other things for Mac OS X, which will best fit into the environment of this system, without having to edit the program itself and use the latest update as the kernel itself, and plugins to qutIM. This feature can also be used to create lighter and alternative implementations of the ‚Äúparts‚Äù of the program.

In addition to the heap of corrected and added errors, the number of supported protocols has increased. Now the full assembly includes support for social. VKontakte networks.
Work has also begun on a plugin for twitter and a list of contacts is being compiled for developing an MSN plugin .

A list of simple plugins has been added, among which I would like to highlight PlugMan, which is likely to be included in the release. The plugin allows you to set content (styles, translations, plugins, etc.) in one click directly from qutIM. Currently preparing to transfer and create packages for existing content.

The remaining plugins can be viewed in the appropriate section of the forum .

From the improvements of the program itself, it can be noted:
  • multiple bugfixes (about 300)
