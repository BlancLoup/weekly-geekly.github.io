Php internals


  • RFC: Retry functionality - It is proposed to expand the try-catch-finally block and the keyword retry. If an appropriate exception is thrown if there is a retry, the try block will be repeated: 

    try { somethingSketchy(); } retry 3 (RecoverableException $e, $attempt) { echo "Failed doing sketchy thing on try #{$attempt}. Retrying..."; sleep(1); } catch (RecoverableException $e) { echo $e->getMessage(); } 

     try { somethingSketchy(); } catch (RecoverableException $e) retry; // Go to top of try block }

  • RFC: Unary null coalescing operator - Is it proposed to implement a unary version of the operator ?? added in PHP 7.0: 

     if ($_POST["action"]?? === "submit") { // Form submission logic } else { // Form display logic }

Instruments


  • amphp / amp 2.0.0 - A powerful asynchronous framework with a concise interface thanks to generators. A number of additional components are also available: asynchronous mysql and postgres clients, DNS resolver , HTTP / WebSocket server , and others.
  • wapmorgan / ServerAvailabilityMonitor - Utility monitors the servers for availability and sends error reports to the mail. Supports http, mysql, pgsql, memcache and redis. Sent wapmorgan .
  • paragonie / sapient - Library for ensuring the security of the API, even when TLS is broken. Post in support.
  • genkgo / mail - Library for sending mail. A suitable alternative to PHPMailer or Swift Mailer.
  • prooph / event-store - PHP 7.1 EventStore for implementing Event Sourcing pattern.
  • jonathantorres / construct - The tool generates the folder structure and files for the new PHP project.
  • wikimedia / composer-merge-plugin - Plugin for Composer, which combines several composer.json files on the fly. Convenient for the separation of the project into internal components with their dependencies.
