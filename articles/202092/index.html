Windows Virtual Servers: New Features



Over the past few months, we have done quite a lot of work to improve the rental services of a virtual server based on Windows. The following innovations are now available for users of our virtual Windows servers:
  • control panel has been significantly improved and supplemented with new functions;
  • all Windows-based virtual servers are now equipped with vhdx virtual disks;
  • A new edition of Windows Server 2012 R2 Standard is available for installation;
  • It became possible to select an OC template with preinstalled software focused on solving specific tasks.

In more detail about all the changes and new features we will discuss in this article.

Vhdx support


All virtual machines are now equipped with a virtual hard disk in vhdx format, which allows you to get a number of advantages.

The use of virtual hard disks is fraught with the risk of an unreadable file in the event of a power failure during active metadata updating. The vhdx format solves this problem because it is journaling: all changes in the metadata are recorded first in the static zone of the log, and only then in the metadata table. If a power failure occurs while updating the metadata table, information about pending transactions will be saved in the log, on the basis of which you can return the complete format of the disk.

The block size for dynamic and differential disks is now increased to 4 Kb, which allows for increased performance.

In addition, the vhdx format allows you to create virtual disks of up to 64 TB.

Windows Server 2012 R2 Standard


The edition of Windows Server 2012 R2 Standard was published on October 17, 2013. Compared with the previous edition, it was significantly expanded and supplemented. The main innovations include the following:

  • Second-generation virtual machines (Gen2) provide faster guest OS booting and faster boot times. Second-generation virtual machines can boot from virtual hard disks via SCSI, from a SCSI DVD, or over a network, which also provides fast booting;
  • the possibility of online export of virtual machines;
  • Hyper V R2 adds the ability to configure the minimum and maximum number of I / O operations (IOPS) allocated to a separate virtual machine;
  • The set of functions of a remote desktop is significantly expanded. Improved connection performance, which allows a 50% reduction in traffic compared to Windows Server 2012. This is achieved through the use of new codecs that provide better compression.
  • Also added the ability to remotely control or view the active session of another user.
  • In GPO, IPv6 support has been enhanced, event logging has been upgraded, and policy caching has been added;
  • The Powershell functionality has been expanded, new cmdlets have been added to administer DNS, DHCP, Hyper-V, File Services, etc .;
  • Now you can limit bandwidth for SMB traffic in more detail here .
