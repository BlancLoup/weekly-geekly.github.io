Hash function Stribog or in the city of the new sheriff

In 2012, the entire public, more or less involved in information security, closely watched the election of the new SHA-3 data hash standard. At Habr√©, this important event was widely covered: the results of each round of the competition were published ( one , two , three ), a description of the new standard was given, and even the reason why the new standard was so cool was explained .
However, behind all this excitement, another, no less significant event remained completely unnoticed: on January 1, 2013, the hash function standard also changed in the Russian Federation.
So, meet: a complete description of the new standard and its implementation in C #. As they say, better late than never.

The hash function called Stribog was adopted as the new standard, which received the number 34.11-2012. Stribog is a family of hash functions that includes only two functions. A function with an output value length of 256 bits and a function with an output value length of 512 bits. Both of these functions have the same structure and differ from each other only in the initial internal state. The input to both functions is a 512-bit data block. In case the length of the message is more than 512 bits, then the message is divided into blocks. In the case, if the length is less than 512 bits, the message is added.

Before proceeding to the description of the algorithm, we describe the transformations that are used in the calculation of the hash function.

Compression function


The main element of any hash function is the compression function. We describe the compression function g n used in the new standard as an algorithm.
Let h , N and m be 512-bit sequences. To calculate the function g (N, m, h) , the following steps must be done:
  1. Calculate the value of K = h ‚äï N;
  2. Assign the value K = S (K);
  3. Assign the value K = P (K);
  4. Assign the value K = L (K);
  5. Calculate t = E (K, m);
  6. Assign the value t = h ‚äï t;
  7. Calculate the value G = t ‚äï m;
  8. Return G as the result of calculating the function g (N, m, h).


You probably noticed that the description of the function g (N, m, h) uses an unfamiliar function E (K, m) , which performs the following actions:
  1. Calculate the value of state = K ‚äï m;
  2. For i = 0 to 11:
    • Assign the value state = S (state);
    • Assign the value state = P (state);
    • Assign the value state = L (state);
    • Calculate K = KeySchedule (K, i);
    • Assign the value state = state ‚äï K;
  3. Return state as a result.


And again an unfamiliar concept. This time it is the function KeySchedule (K, i) , which is responsible for the formation of a temporary key K at each round of the function E (K, m). Let's fill this gap in the description of the algorithm. The KeySchedule (K, i) function performs the following calculations:
  1. Assign the value K = K ‚äï C [i];
  2. Assign the value K = S (K);
  3. Assign the value K = P (K);
  4. Assign the value K = L (K);
  5. Return K as the result of a function.

Hash calculation


Now we describe the procedure for generating a hash value. For any input message M:
  1. Assign initial values ‚Äã‚Äãof internal variables:
    • For a hash function with an output length of 512 bits: h = iv = 0x00 64 . For a hash function with an output length of 256 bits: h = iv = 0x01 64 .
    • N = 0 512
    • Œ£ = 0 512
  2. Check the following condition: message length M <512. If the condition is met go to step 3. Otherwise, perform a sequence of calculations:
    • m - last 512 bits of message M
    • h = g (N, m, h)
    • N = (N + 512) mod 2 512
    • Œ£ = (Œ£ + m) mod 2 512
    • Trim M by removing the last 512 bits
    • Go to step 2
  3. Make the addition of the message M to a length of 512 bits according to the following rule:
    m = 0 511 - | M | || 1 || M, where | M | - the length of the message M in bits.
  4. Calculate h = g (N, m, h)
  5. Calculate N = (N + | M |) mod 2 512
  6. Calculate Œ£ = (Œ£ + m) mod 2 512
  7. Calculate h = g (0, h, N)
  8. Calculate h = g (0, h, Œ£)
  9. For a hash function with an output length of 512 bits, we return h as the result. For a function with an output length of 256 bits, we return MSB 256 (h).
