Dynamic code verification - minimizing fraud risk while maintaining the convenience of making purchases online

Dynamic Code Verification (DCV) technology is a breakthrough in the international struggle in card fraud, since it ensures the security of CNP transactions at the highest level. According to statistics, 70% of fraud in the market of EMV-cards are CNP fraud. How exactly does a DCV solution protect CPN operations?


Dynamic code verification from Gemalto is a transaction protection solution without a card (CNP), the parameters of which are set by the issuing bank and includes three main elements - a payment card with a display, a mobile device solution and a confirming server. This configuration allows banks to minimize the likelihood of fraud without compromising the convenience of making purchases on the Internet.

In 2013, 41.3% 1 of the total number of Internet users worldwide bought goods online. As expected, in 2017 this figure will increase to 46.4%. With the increase in online purchases and increased protection of transactions with the physical presentation of the card, fraud in making payments predictably shifts towards the next (most attractive for attackers and, at the same time, the weakest) chain link - transactions without presenting a card (CNP operations). The total amount of fraudulent transactions carried out using cards issued in the Single European Payment Space (SEPA) and acquired throughout the world amounted to 1.44 billion euros2 in 2013, which is 8% more than in 2012.

The main factor behind this growth was fraud in CNP operations, which in 2013 accounted for losses amounting to 958 million euros.

That is why the prevention of fraud in CNP operations and the prevention of such crimes becomes a prerequisite for preserving the universality and convenience of online shopping.

Gemalto Dynamic Code Verification (DCV) Solution Package


The Dynamic Code Verification Package offered by Gemalto is a comprehensive solution for securing CNP operations based on the use of a payment card with a display, a mobile application and a confirming server. This solution allows card issuers to replace the static protected cryptogram, traditionally used for making purchases online, with the dynamic code displayed on the customer‚Äôs EMV card or mobile application. This code, valid for a limited time, excludes the use of stolen static card data by criminals for fraudulent transactions.

In addition, Gemalto's DCV solution allows card issuers to reduce the risk of fraud without changing the online payment infrastructure of online stores and payment systems.

Optelio DCV is a classic contact or contactless payment card of the EMV standard, on the back side of which a dynamic code is displayed. This code automatically changes over time without the participation of the cardholder.

The Ezio DCV mobile solution allows your customers to store their cards and read dynamic codes on their mobile phone.
