Come study classical administration: regulations, tools, scripts Southbridge

For 10 years, Southbridge has created a work standard that allows one administrator to support 150 servers, quickly carry out initial configuration, easily transfer a project between administrators and groups, see what the night duty did, quickly get up to speed after the holidays, and, of course, provide customer reliability and security infrastructure.


From October 22 to 24 Southbridge conducts intensively for system administrators, where he will show his approaches, regulations, tools, instructions and scripts.



RedSlerm is essentially a collection of materials for training a new Southbridge employee.


Mastering an approach to administration based on unification and standardization is useful even for a novice administrator.


All that we can, we practice in practice.


RedSlerm (October 22 - 24) is a cure for red eyes for those who want to support dozens and hundreds of servers and sleep well at night.


RedSlerma website



The administrator will be able to:


- easy to manage the "zoo";
- build a database of solutions and use it;
- quickly navigate in the settings of projects;
- set up proactive monitoring to detect problems in advance.


Head unifies


working with updates, access, monitoring, data storage and hardware, and as a result will be able to prepare a specialist for 1 month internship, who will pick up any existing project.


Occupation Topics:


Topic 1: Work as Southbridge. Main regulations
- Management in the company. How workflows are organized
- Standards in the company. How to learn to quickly solve complex problems and reduce the influence of the human factor
- Work with Redmine. Our developments that improve the quality of service and security
- Rules and standards for working with configuration management systems (Ansible / Slack)


Topic # 2: Monitoring. A complex approach
- Monit local monitoring. Scope, device, setup and operation
- Remote monitoring: Zabbix. Project creation, automatic templates, monitoring features
- Our scripts. Overview of Southbridge automation and monitoring scripts, rules of operation


Topic 3: Work as Southbridge. Technical Regulations
- Basics of setting up dedicated servers and tuning for maximum performance
- Basics of working with OpenVZ / KVM. Installation, configuration, operation
- Work with hardware RAID controllers
- Practice. Setting up and tuning servers using Ansible


Theme number 4: Databases. Southbridge Experience and Practice
- Accepted customization standards
- MySQL. Installation, configuration, tuning, applying changes without downtime
- MySQL. Configure Master-Slave replication with minimal downtime
- PostgreSQL. Installation and architecture fundamentals
- PostgreSQL. Tuning, tuning, replication, backups
- Redis. Installation and architecture fundamentals
- Redis. Tuning, tuning, replication, backups
- Practice


Topic # 5: Web servers. Southbridge Experience and Practice
- Accepted standards, automation scripts and rules for writing configs
- Setting specific parameters for php
- Let's Encrypt. We get free production-ready certificates
- Obtain and configure SSL certificate (A +), features of application in production


Topic 6: PostgreSQL Database: Clustering and Recovery
- PostgreSQL: Pacemaker based clustering
- PostgreSQL: restore the state at a given point in time
- Practice


Topic 7: 1C-Bitrix web servers: configuration features
- Load testing tools and their application in practice
- Highly loaded projects on Bitrix from the point of view of the server administrator. Southbridge experience


Topic number 8: Server and project security. Southbridge experience
- SSH protection. Using OTP to access servers. Scripts and their use
- Protecting the site from DDoS attacks
- Scripts for protection against http flood, port scanning, packet integrity checking
- Software verification scripts for vulnerabilities and automatic updates


Topic 9: Version Control Systems. Work with Git and Gitlab
- Install and configure Gitlab. Operating experience
