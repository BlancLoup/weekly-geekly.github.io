How it works


Scenario number 1: you need to install Microsoft 365 on each workstation. Your list includes both ‚Äúancient‚Äù computers with CD-ROM drives, and newer ones with DVD and USB support, as well as brand new thin laptops without drives. What to do?

You can run the installation and configuration tool for Microsoft applications, which:

  • automatically determine the need for software updates;
  • install applications with required updates;
  • set up email and instant messenger.

There is an improved version of this scenario, it uses the technology of Azure AD Join, which appeared in the Windows 10 operating system. The technology uses the active directory Azure AD and allows you to apply a verified identity, signed by the user. After verifying the identity, Azure AD Join "vouches" for this user in front of all cloud applications from its catalog ‚Äî these are Microsoft Office 365 applications, as well as Salesforce, Concur, Workday, and thousands of other popular cloud services. For the user, it looks like all the applications he needs are already installed on any Windows 10 system on which he is authenticated.

Scenario number 2: you need to provide secure access to Microsoft 365 services from any device to mixed groups of users that include not only company employees. Depending on whether the user belongs to the organization or not, at the entrance he will receive one or another set of rights. Thanks to multi-factor authentication that is configured through the Administration Center, working together on Microsoft 365 services with non-members of your organization is completely secure.

Scenario number 3: Some of your users have no computer at all. Totally. Only a smartphone, tablet, and possibly access to public computers. Catastrophe? Not! Mobile versions of Word, Excel, PowerPoint, Outlook, and Teams are available for most mobile devices, and using a web browser, employees can open Office documents on the company's portals, SharePoint Online or SharePoint Server sites, automatically uploading Word, Excel, PowerPoint, and OneNote online components. - all you need to do is assign the appropriate licenses to them in Microsoft Administration Center 365.

With the means of managing mobile devices (including personal ones) and controlling the distribution of corporate data, you give users the ability to securely share files, control and monitor their actions, including Block sharing of files with unauthorized users - up to remote removal of corporate data from the device or complete cleaning in case of loss or dismissal of an employee.

Scenario number 4: you need to protect your business from any cyber threats - i.e. on the one hand, to ensure the integrity of operating systems and application software and their regular testing, and on the other hand, to protect all systems from external threats, including any types of attacks and exploits that have been, is and will be. The task seems intractable, but in reality it has already been solved - the ATP agent in Windows Defender not only knows how to quickly adapt to changing threats and deploy new defenses, but also helps to eliminate the consequences in order to reduce the damage from the attack and prevent re-infection.

What is your headache and how to cure it


Suppose you already use some hosting cloud server - for example, a mail server. In this case, congratulations - you have already moved to the cloud, but not completely. Now it will be logical to fully switch to cloud solutions, save the IT budget to support strategic business projects and provide employees with the latest work tools.

If we are talking about the implementation of the mail service, then the necessary solution is Microsoft Exchange Online - part of Microsoft 365, a cloud messaging service based on Microsoft Exchange Server. This is a full-featured solution for managing email, calendars, contacts and tasks for workstations, the web and mobile devices. Exchange Online provides organizations with a full range of messaging capabilities without the need for local server support.

Now let's take a sharper situation ‚Äî your company uses deeply outdated servers, such as Exchange Server 2007 mail server, Windows Small Business Server 2008, or, GroupWise or Lotus Notes. And all because, as Boromir said, you cannot just take and implement a new system. To install a new mail server, you will have to purchase a new hardware.

And why, exactly? After all, the "cloud" server Exchange Online does not require hardware updates.
