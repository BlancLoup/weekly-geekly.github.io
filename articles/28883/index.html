Carrier (was Funpidgin) - Rumors of my death are greatly exaggerated

Not so long ago , the group that created Funpidgin fork was separated from the Pidgin multi-client client development team. The separation was due to different views on the development of the user interface of the program. He personally attracted me by the presence of the assembly in .deb.
A little later, the project was renamed Carrier .
This client will love Pidgin users, who are not satisfied with the paucity of settings. It all started with the little things - the appearance of automatically changeable and unregulated input window. A heated argument flared up around her and a fork appeared as a result. After the release of version 2.4.2, released in the form of .deb and .rpm packages as well as the source for a long time there was no news. Even reactions to changes in the behavior of icq servers, violently rejecting connections from the "old" versions of clients did not follow.
However, as it turned out, the project is still too early to bury. Just yesterday, an update to version 2.4.3 appeared on the site. So far, only a win32 build is available (by the way, it was strongly complained about its absence for version 2.4.2) and source codes for fans of make && make install.

What's new in 2.4.3:
carrier:
* Synchronization with the release of Pidgin 2.4.3 (Troubleshooting connection problems with ICQ)
* Fixed release notifications plugin
* Added old smilies (<2.4.0)
* Fixed Carrier branding of Windows installer
libpurple:
* Yahoo! Japan now uses UTF-8 encoding, as well as official clients. Restored compatibility with web messenger (Yusuke Odate)
* Install avatars for Yahoo! it works now.
* Yahoo! protocol fixes concerning aliases
* Bug fixes in bonjour protocol
* Now for Yahoo! always use UTF-8 encoding (# 5973)
* Fixed crash with incorrect jabber id.
* Make the IRC "unknown message" debugging messages UTF-8 safe.
* Fixed connecting to ICQ
* Fixed a memory leak in jabber xforms.
Pidgin:
* Added plugin for submit button in win32 version
* Memory leak fixes
