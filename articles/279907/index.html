NetApp ONTAP Cloud: Amazon & Azure

The advantage of any cloud service is that it is the actual service. You do not bother where the equipment lives, who services it, etc. But this is also its dark side: who has access to your data, how many copies of your data are stored in backup copies and backup data centers and where? For a long time all these questions have repelled some large companies and government agencies.

Having solved these issues, many companies would be able to start using cloud services. To solve this problem, the ONTAP Cloud Cloud Storage System was developed - a virtual machine living in a public Amazon or Azure cloud with built-in encryption function. Now you own and manage your data and only you. Read more on the official website of NetApp and official documentation on the product itself and on OnCommand Cloud Manager .


From a management and interface point of view, this is a regular ONTAP with a few limitations, like the ONTAP Edge and ONTAP Select twin brothers in ONTAP Cloud there is no RAID-DP and FCP. But otherwise this is a full-fledged storage system, just living in a cloud. You can replicate your data from a productive FAS system to the cloud to create a DR site and development environment. And in case of an accident, switch to the backup site and start all or at least the most important services.

This is very convenient from a financial point of view: after the accident, you just before-order the necessary resources in the cloud and start your services, there is no need to buy equipment and maintain it on the second backup site.

High availability


ONTAP Cloud supports both mono-mode and HA mode, when two uplaens are located in different Availability zones to provide fault tolerance inside the cloud. Despite the fact that almost all cloud providers assure their customers of high availability of their data centers, multiple paths, component duplication, etc. all decent cloud providers also highly recommend backing up regularly. Somehow it does not fit in with one another, right? The fact is that all the same, no matter how cool inside the cloud, there are accidents. And there are such accidents when customers either temporarily do not have access to their data or lose data in whole or in part, for a certain period of time. There are examples of this, because of this cloud providers recommend backing up to the cloud. Thus, ensuring high availability of data within the cloud is a very demanded feature.

The High Availability feature allows you to ensure high data availability in the cloud by placing two nodes of the HA systems in two different Availability zones, which will allow you to survive in the event of a data center crash, automatically switch transparently and provide data access for applications using this data.


Disaster Recovery site in the cloud


A DR site is only needed when a replica is running and when it is necessary to switch to a DR site, all the rest of ONTAP can be turned off and save you money, because in public clouds AWS / Azure payment is hourly. Work schedules can be configured with ONTAP for Cloud to save budget.

Intra & Inter Cloud Replication


With SnapMirror technology, you can replicate data from ONTAP for Cloud to ONTAP Select , FAS, and NPS , as well as between several ONTAP for Cloud within one or even several public providers.


Cloning


Another useful feature of ONTAP is the ability to finely clone data without affecting the performance of the cloud storage system. This will allow you to deploy in the public cloud, in seconds, a lot of duplicates of your data for such tasks as testing, development, analytics, checking and cataloging backups, regardless of their size.



Cloud manager


Allows you to manage these features ONTAP Cloud:

  • Replicas between FAS, NPS , ONTAP Select and ONTAP for Cloud
  • Manages encryption keys for ONTAP Cloud
  • Displays the amount of computing resources consumed and storage in Amazon & Azure
  • Manage storage space: Create volums, aggregates, moons, file balls
  • Update ONTAP version



Encryption


Two types of Data-at-Rest encryption are supported: AWS or ONTAP. In the second case, you can store the encryption key in your home, which allows you to increase the security of your data and unlock data in the cloud using the KMIP protocol at the time of launching ONTAP.

Data Fabric Cloud Sync


Data Fabric is NetApp's development strategy and vision, and Cloud Sync is an implemented part of this strategy. Cloud Sync service is also related to ONTAP Cloud, as well as to any other platform based on ONTAP software. Data Fabric Cloud Sync allows you to synchronize NFS file sphere data in Amazon S3. The service can serve as an archive of data that can be used for analytical processing within Amazon. Data lying on the NFS ball on one of the platforms with ONTAP software is converted into objects and sent to the Amazon S3 storage. Further, data in Amazon S3 can be processed by analytical algorithms for extracting reports (such as MapReduce, RedShift).

I will give examples of the application of this service and data types for which analytical processing may be required:

  • Healthcare: Genomics, Pharmaceutical industry, patient records, disease analyst;
  • Oil and gas: Seismic analytics, machine analysis, forecasting the demand for goods and services;
  • Finance: Trading performance, loans and market risk management, fraud detection;
  • Automotive: Behavioral analytics, supply chain management systems, predictive analytics.


Protocols


CIFS (SMB) v2 / 3, NFS 3/4 / 4.1, pNFS, iSCSI are supported.



Like ONTAP Select, the ONTAP Cloud product has many useful features and integrations.

Licensing


There are two licensing schemes: when a customer pays for all services at Amazon / Azure. The second scheme, when the customer buys from the provider only computing and storage space, and the license for ONTAP Cloud software is purchased through NetApp.

findings


With ONTAP Cloud, you yourself own and manage your data in the cloud and get the opportunity to replicate your data to other data centers and rich ONTAP functionality. Deploying a backup data center will be cheaper in the cloud due to the fact that there is no need to maintain a backup site. Based on the replicated data, you can deploy development and testing due to the functionality of thin data cloning. Thin replication to the cloud will load network channels less and is an easy way to migrate your infrastructure to a public cloud.
