On Saturday, Kim Dotcom will present the file hosting service Mega



On Habr√© more than once and not two wrote about the new file-sharing service, which will replace Megaupload. It seems that the preparatory period is completed, and the file sharing service will start working already this week. In any case, Kim Dotcom, the author of the project, promises to launch this service in three days.

It is worth recalling that Mega will encrypt user data, including the transferred arrays of information, and what is stored on the server. According to Dotcom, all this will protect users of the service, and the service itself, from the prosecution of the authorities. Encryption will be done using 2048-bit keys. The generation of keys is random, produced as a result of mouse movements and pressing the buttons of the keyboard. There is also a specific file manager designed exclusively for the Mega service.

Among other things, the data will be distributed in the "cloud", so the loss of several servers (for example, as a result of seizure by the authorities) does not threaten users with data loss. Servers will be located in different countries in order to further reduce the likelihood of service disruption. Encryption keys will be stored in the cloud, too, and not on the user's computer.

The domain on which the resource will be located is regional, this is Mega.co.nz. If the service is really launched, as promised, it is likely that it will very quickly become popular. Here, the role is played by both the carried out information preparation and the security of the service.
