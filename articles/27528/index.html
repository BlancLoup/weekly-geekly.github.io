Setting the number of drop-down options in the navigation bar

With Firefox 3, the address bar has become intelligent, which is the automatic selection of drop-down options in accordance with the query string not only by URL, but also by tags, descriptions of bookmarks and Title of visited sites (thanks to hlomzik ). To configure the number of options, you need to do the following steps:
  1. Enter in the address bar about: config
  2. Promise you'll be careful :)
  3. In the filter string, enter browser.urlbar.maxRichResults
  4. Enter the required number


Removing the wrong options

Despite the fact that the navigation bar is intelligent, if you enter the wrong URL (for example, the page does not exist), the browser will remember it and will offer it in the future. To delete this option, you need to hover the cursor on it and press Shift + Delete Delete (thanks to telega2 ).

Outdated extensions

Firefox 3 includes functionality that some extensions used to take over. Therefore, the following extensions are no longer needed:
  1. Noswint
  2. Google Gears . The amendment, Firefox 3 does not include Google Gears, but supports the WHATWG, that is, it provides web applications with some interface for caching data and providing offline work. More information on this issue: Online and offline events and Offline resources in Firefox (thanks to jursovet !)
  3. Resize Search Box
  4. DownThemAll , Firefox 3 does not include all the functions of DownThemAll , but supports resuming when the connection is disconnected (thanks to Kain_Haart )
  5. Mailto: handlers for Gmail and other mail services


Restore the old navigation bar

Perhaps for someone a new ‚Äúintelligent‚Äù component of the navigation bar will seem redundant, it does not matter - to restore the classic panel, use the Oldbar extension. You can partially return the behavior of the old address bar without using the extension via about: config by setting the browser.urlbar.matchOnlyTyped parameter to true (thanks to amoral ).

Smart Bookmarks

Firefox 3 supports smart bookmarks - a dynamic list of URLs generated by a specific search criteria. To add such a bookmark, you need to select the addition of a normal bookmark and in the ‚ÄúLink‚Äù field specify the line in accordance with the list of parameters . The following are examples of bookmarks:

place: queryType = 1 & sort = 8 & maxResults = 15

Faster loading pages

To speed up page loading, you should enable support for multiple requests within a single connection (pipelining). To do this in about: config, you need to configure the following parameters:

network.http.pipelining - true
network.http.proxy.pipelining - true
network.http.pipelining.maxrequests - 8
nglayout.initialpaint.delay - 0 (right-click and create a new integer value with that name)

Full article on this topic with a description of the principles of work is here . Thank you green_hippo ! Added to the article so that everything was in one place.

To do this, in about: config, you need to change the values ‚Äã‚Äãof the browser.tabs.tabMinWidth parameter. vvchik offers a value of 20, which allows you to see 5 times more tabs than with the default settings.
