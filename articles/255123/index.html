Read and write speed

If something went wrong ...

Methods and assessment criteria


Let's try to evaluate drives by 9-point system. 4 criteria will be considered:

Removing memory chips
  • 0 points - without any problems
  • 1 point - possible difficulties
  • 2 points - possible under certain conditions

Read memory chip dump
  • 0 points - without any problems
  • 1 point - possible difficulties
  • 2 points - long / laborious, impossible

Performing pre-transformations
  • 0 points - without any problems
  • 1 point - possible difficulties (solvable)
  • 2 points - difficulties are possible (currently unsolvable)

Build the image
  • 0 points - without any problems
  • 1 point - possible difficulties (solvable, time consuming, long)
  • 2 points - difficulties are possible (unsolvable so far)

Accordingly, the lower the points in the total dial up the drive, the higher the probability of data recovery and the speed of the work to save these files.

Drive components


The structure of any flash drive includes a controller (microprocessor responsible for the distribution of user data) and memory chips (at least 1, but maybe several. Actually, the data in a specific form is stored on the memory chips). The task of the data recovery engineer from flash drives is to emulate the operation of a controller that has failed. Let's see what our drives consist of:

1. Kingston DataTraveler HyperX 3.0
  • Phison PS2251-01-L Controller
