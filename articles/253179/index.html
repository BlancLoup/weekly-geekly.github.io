Weekly io.js, March 13, 2015

io.js 1.5.1 and community events.

Release io.js 1.5.1



On Monday, March 9th, @rvagg released io.js v1.5.1 . A complete list of changes can be found on GitHub .



Major changes



  • tls : In this release, thanks to many commits, the problem with the previously announced memory leak in TLS was solved. Current tests indicate that there may still be any problems with memory leaks. Watch for full progress at # 1075 .
  • http : Solved the problem stated in joyent / node # 9348 and npm / npm # 7349 . Pending data was not fully processed on the 'error' event, resulting in an error in socket.destroy() . (Fedor Indutny) # 1103 .


Known Issues



  • There may still be a memory leak problem in TLS, details are # 1075 .
  • Some tests in Windows still fail. The correction of errors associated with them continues to be our priority task. See # 1005 .
  • A surrogate pair at the REPL may cause the # 690 terminal to hang.
  • Unable to build io.js as static library # 686 .
  • process.send() not synchronous, as described in the documentation, rollback introduced in 1.0.2, see # 760 and edit in # 774 .
  • Calling dns.setServers() during the execution of a DNS query may crash process # 894 .
