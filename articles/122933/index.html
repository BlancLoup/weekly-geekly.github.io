What will be discussed?


The event is devoted to the problem of optimizing computer games. You will learn about Intel's tools for analyzing performance and debugging graphics programs that should be in the arsenal of each game developer. Of course, the highlight of the program is Intel Graphics Performance Analyzers (Intel GPA) . The product was first introduced in 2009 at KRI, and since then it has always been popular with developers, thanks to a unique approach to optimizing graphics. Not so long ago, a new version of Intel GPA 4.0 was released, which includes 3 main modules:
  • System Analyzer HUD (Heads-Up Display) is a dashboard for performance monitoring and analysis. CPU / GPU / Direct3D metrics are displayed in real time over the game frame. This tool is the first step towards optimization. It allows you to determine the limited application performance - CPU or GPU;
  • Frame Analyzer is designed for detailed analysis and debugging of the captured frame, allows you to find the most difficult D3D API calls and eliminate bottlenecks;
  • Platform Analyzer is a tool for visualizing thread performance across all CPU cores.

At the webinar, we also talk about Intel Graphics Checker . This tool is provided as part of the Intel affiliate program and allows you to quickly assess the performance of the application, as well as get a detailed report with recommendations for optimizing the code. The new version 3.0 includes integration with Intel GPA. Thus, in addition to the FPS application, the tool allows you to get information about the efficiency of the use of the central and graphics processor.

Intel GPA tools and Intel Graphics Checker will provide you with real assistance in creating and optimizing graphics applications. Come in on the morning of July 1st to the webinar, and with live examples you will see how these tools can quickly identify bottlenecks and perform code optimization. In addition, you will learn about new benefits for members of the Intel affiliate program and how to get the tools for free.
