Russian translation of the speech by Alex Ionescu "Crazy attempt to rewrite Windows from scratch"

On November 12, 2013 , we published a video of a speech by Alex Ionescu (who is known to the Russian audience, primarily as a co-author of the Windows Internals series of books) on the ReactOS operating system. Unfortunately, at that time we did not have high-quality subtitles in English, much less a translation into Russian. But now, thanks to the help of volunteers, these omissions have been corrected. The Black_Fox user on translatedby.com created the correct English subtitles based on youtube automatic subtitles, and the Notabenoid translation community translated them into Russian.

In this conversation, Alex Ionescu, the lead kernel developer for the ReactOS project since 2004 (and recently returned from a long break) will talk about the current state of the project, which has already passed 60,000 revisions in the SVN repository. Alex will also cover some of the project‚Äôs objectives and the development and testing methodology, which is a large-scale event in such a project (an open-source project to implement the entire Windows OS from scratch!), In collaboration with other open-source projects (MinGW, Wine, Haku, etc.). d.).

Alex will tell you about the infrastructure that allows you to run such a large-scale project operating system (without corporate resources), as well as daily difficulties in working conditions of a very distributed team and a strong lack of knowledge about the Win32 device, which greatly complicates the recruitment of new participants. Finally, Alex will demonstrate the work of the OS, try the work of new games and applications, work on the Internet, etc., and, of course, show a couple of blue screens of death (BSOD).

Alex will also show how the new ReactOS assemblies can be built directly from Visual Studio, and how you can debug the kernel with the standard Microsoft WinDBG debugger, traversing the practical Windows kernel to debug problems with real drivers or applications. If time permits, Alex will touch on the question of reverse engineering of Windows functions or drivers for the purpose of understanding their operation logic, which can help with debugging problems outside the field of using ReactOS.
