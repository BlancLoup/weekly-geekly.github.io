Use new reports in the structured data dashboard to fix markup errors.

The level of training webmaster: medium or high

Since its launch last year, the Structured Data Dashboard has become one of the most requested features in Webmaster Tools. We continue to work on the simplification and convenience of finding errors in the markup.

Now, in the structured data dashboard, you can not only find out if Googlebot understands the markup on your site, but also determine where exactly the mistakes are made. New reports in Webmaster Tools are the result of our collaboration with site owners who participated in the June beta test. We took their feedback into account before launching a new version of the structured data dashboard.

In the new reports, each object represents one element of top-level structured data (nested objects are not counted), marked up using HTML code. Objects are grouped by data type, which are then sorted by the number of errors.

We added a separate graph of the number of errors on the right side of the dashboard so that you can see visually what changes on the website increase or decrease the number of certain errors.

To make our reports more detailed, we also updated our data processing systems, so for the first time, incomplete data may be displayed on the graph.

How to eliminate markup errors

  • To find out what caused the errors in a particular type of content, just click on it. A list of incorrect layout elements will appear. Using the top tabs, you can filter it by type of error.



  • The list will indicate where the markup does not comply with Google recommendations for various types of content . In our example, the startDate property is not specified for several events. There may also be no properties for nested content types, for example for reviews added to products, such a property is lowprice.

  • Click on the URL in the table to find out exactly what the markup errors are found when scanning the page. To instantly test your markup using the structured data validation tool , click the Check Data button. Often, pages of the same type of errors that can be eliminated at once, for example, by changing a parameter or pattern in the content management system.



  • Correct the errors found and run the structured data verification tool again. After re-scanning and processing the pages, the changes are also displayed in the structured data dashboard.

We hope that the new error reports will simplify the work with the structured data markup. In addition, in the coming months we will expand the list of detectable errors. In the meantime, you can share your opinion on the new feature and ask questions in the appropriate section of the forum for webmasters.
