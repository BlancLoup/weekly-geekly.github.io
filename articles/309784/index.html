Use Adobe Captivate for live-action role-playing games.

In the role-playing games of live action such as pavilion / cabinet games, or in quests on space themes (or any other technology-related) you can use computers for greater immersion and atmospheric.

For example, to create an office role-playing game based on the series FireFly, I needed to create a model of an onboard computer. I took my laptop, wrapped it in black cloth, covered all the keys with black tape, except letters, numbers and Enter, and unrolled the emulation of the spacecraft control system all over the screen.

To implement the idea and create emulation, the Adobe Captivate tool was very suitable.

  1. The sequence of actions was simple:
  2. Find with the help of hints the password to the onboard computer control system
  3. Enter the password in the input field and press the Enter key.
  4. If an input error occurs, display a window with a warning and the number of remaining attempts.
  5. If you enter the password correctly, open a window with a hint.
  6. Press the button to go to the coordinate input window.
  7. Enter latitude coordinates.
  8. Enter longitude coordinates.
  9. If one value is entered incorrectly, output the final warning message about danger.
  10. If the coordinates are entered correctly, display a message with a terrain map.



In fact, I needed to create a beautiful sequence of requests from the On-Board Computer System and windows in which I had to enter access codes and coordinates. Creation time was limited, and I did not bother with designer delights.



It unloaded the resulting movie in * .exe format, removing the scroll bar and the corresponding buttons, although the option in * .html would also be appropriate.

Captivate allows you to implement an algorithm with input errors, that is, I needed every time I tried to enter the password / code incorrectly to display a message about the number of remaining attempts and prompts.



Using the built-in capabilities of the tool, it is very convenient to put certain actions on the transition buttons, for example, returning to the main window or jumping to the slide.



In the Timeline field, the slide display time is set, and you can also fix the slide in a stationary state before pressing a certain key.

The game was held in 2014 and was a success. 4 runs were carried out.
