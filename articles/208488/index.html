How to improve the mobile version of the site. Recommendations and video tips

The level of training webmaster: medium or high

Wondering how to improve the mobile version of the website, and do not know where to start? Especially for you, we have prepared a list of recommendations that will help improve your mobile site. We accompanied some of the recommendations to well-known examples from practice and research results, and for some we even made several video tips (they can be watched with subtitles in Russian and several other languages). These tips and tricks will help you figure out how to correctly interpret data from Google Analytics and Webmaster Tools to improve the mobile version of your site.

Step 1. Make your mobile site truly user friendly.


A) Remove additional windows in versions for all mobile devices ( Google recommendations , useful article ).
  • Do not use JavaScript pop-ups that are difficult to close.
  • Remove all floating ads, especially advertising applications. Instead, use iOS 6+-style smart banners , side navigation, emails and other marketing tools.
  • Do not invite users to participate in the survey before they solve their problems.

B) Optimize the site for different devices.
  • Discard functions that require the use of plug-ins or video playback tools that may not be available on the user's device. For example, Adobe Flash Player is not available in the iPhone, as well as in Android 4.1 and later versions ( example from practice ).
  • Show tablet users either a version optimized for desktop computers, or - even better - a website for tablets ( research results ).
  • Give mobile phone users the ability to switch to a PC version. User selection should be applied throughout the session: do not force visitors to switch to the PC version after loading each page ( research results ).

Step 2. Help visitors get what they came for.


A) Optimize your site for scanning and indexing , and also make it more convenient for users who come from search engines ( practice example ).
  • Enable resource indexing ( CSS , JavaScript , etc.) in robots.txt .
  • Follow the tips on optimizing the mobile version of the site for search engines (depending on the method of implementing the mobile site):
    1. Responsive design : be sure to include a CSS @media. query CSS @media.
    2. Separate mobile version of the site : add the attributes rel=alternate media and rel=canonical , as well as the HTTP header Vary: User-Agent , so that Googlebot will immediately indicate the mobile URL in the search results
    3. Dynamic display : add the Vary: User-Agent HTTP header.
