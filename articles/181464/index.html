Django 1.6 alpha 1



The first alpha version of Django 1.6 has been released this week. This release is dedicated to the active member of the Django community - Malcolm Tredinnick, who died on March 17th.

The new version of the framework has significantly simplified the configuration file. Now django.contrib.admin enabled by default, and django.contrib.sites contrary, is disabled. In addition, locale detection and clickjack protection are now enabled by default.



In general, the new settings.py now looks like this: 

 """ Django settings for PROJECT_NAME project. SECRET_KEY = '***' # SECURITY WARNING: don't run with debug turned on in production! Other changes:
  • Support for persistent database connections
  • New test-runner that uses unittest2 functions
  • Time Zone Aggregations
  • Savepoint Support (SAVEPOINT) for SQLite
  • Added new model field for storing binary data
  • Added new form fields for working with GeoDjango
  • Added earliest method for QuerySet
  • Form widgets now use new HTML5 attributes
  • Now Pillow is the preferred solution for image processing. PIL support will be removed in Django 1.8
  • For QuerySet, the first() and last() methods are added, which return the first and last objects, respectively. If nothing is found, then None is returned.
  • django.contrib.comments is deprecated and will be removed in Django 1.8


View a more complete list of changes here: docs.djangoproject.com/en/dev/releases/1.6
