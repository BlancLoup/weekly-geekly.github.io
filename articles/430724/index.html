So, after you type in the address bar of the www.google.com browser, your computer sends a request to a local DNS server, and the job of finding the thing you need, returning the request and the answer is now assigned to it. The local server accesses the root server with a request to access the .com server, and the root server sends it to the .com server. That one checks if the local server is authorized for requests to google.com and sends it to the ns.google.com server. This is the normal DNS behavior that everyone expects from it.
