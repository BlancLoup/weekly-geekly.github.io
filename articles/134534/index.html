A lot of new about semantic markup from Yandex

Today, Yandex has several news from the world of semantic markup and data transfer about sites.

Let's start with the big one. Yandex became part of the Schema.org Working Group and is now involved in the development of semantic markup standards for the whole world. Help can now find a translation of the original manual on how to use Schema.org - Getting Started semantic markup, with a few additions.

Now about the modern. Yandex.Video robots have learned to recognize the HTML5 video specification tag. Now this markup is taken into account along with the rest. As in the case of schema.org, this tag allows you to mark several videos on the page.

A little bit about tasty. We started to support Schema.org markup for recipes. The owners of culinary sites just need to mark up the recipes on their resource using the Recipe scheme (http://schema.org/Recipe), and their snippets in the Yandex search results will automatically become as visual:





We also renewed support for the hCard microformat. Using this open format, you can tell Yandex where your website contains the address, phone number, time of your organization, as well as which Directory heading it should be attributed to. By the way, schema.org markup for organizations addresses provides similar opportunities. We use information about the contact details of companies not only for answering the main search, but also for searching organizations on Yandex.Maps.
Pay attention: now it is necessary to indicate the phone number of the organization in hCard, but the site address is not. In addition, in the field for the site address you can also specify, for example, the sites of the company's branches.
