What is ONTAP Cloud


The Cloud Storage Environment (CSE) on ONTAP Cloud implements the following features.

- Non-destructive, two-node support and high availability counteracts unplanned storage failures in the cloud.

- Used storage systems NAS and SAN, with support for NFS, CIF and iSCSI.

- The technologies of data deduplication, ‚Äúthin volumes‚Äù (thin provisioning) and data compression are used.

- NetApp Snapshot Zero-impact technology almost instantly backs up data without requiring additional storage resources.

- NetApp SnapMirror technology integrates a hybrid cloud environment by linking local FAS repositories to the ONTAP Cloud environment.

- Advanced data management tools (OnCommand Management Suite) and their monitoring (OnCommand Insight) are managed from OnCommand System Manager.

- Accessibility of applications is ensured by copying them through special ‚Äúavailability zones‚Äù.

ONTAP Cloud vs. Ebs


EBS storage can be thought of as similar to DAS storage for standard servers. In fact, EBS provides ‚Äúraw storage capacity‚Äù for Amazon EC2 servers.

EBS has some features and capabilities that are somewhat wider than what DAS typically offers, but they are still limited and do not contain any data replication tools.

They provide snapshots of the repository. These snapshots affect the storage performance, and they also create a complete replica of the storage in S3, for which the customer pays.

ONTAP Cloud Storage is a managed storage environment that uses EBS raw storage as the basis for its work. ONTAP Cloud takes over EBS storage, manages it using Data ONTAP storage OS and, besides the storage itself, provides the following additional features.

- Using multiple protocols (NFS, CIF, iSCSI).

- Data Protection (Data Protection) Snapshot copies, SnapMirror, SnapVault.

- Storage efficiency (Data efficiency) ‚îÄ Data deduplication, compression, thin provisioning. These features can reduce the number of EBS storage units you need to buy.

- Management ‚îÄ Unified Manager, System Manager, WFA, Insight, etc ...

- FlexClone technology.


How FlexClone Technology Works

NetApp Snapshots uses zero-copy technology. It involves operations in which the processor does not perform the task of copying data from one memory area to another. The term is used to describe technologies that help reduce the number of copies between intermediate buffers. See more here .

NetApp Snapshots have no effect on performance. Copies store only change data, minimizing storage usage.

Data protection in ONTAP Cloud


ONTAP Cloud includes SnapMirror replication software that moves data between Data ONTAP locations. Data can be moved from one availability zone to another.

For this, the client needs to have other Data ONTAP environments in the secondary availability area. This secondary location may be another location of the ONTAP Cloud, or it may be the location of a NetApp Private Storage.

SnapMirror can be used to move data between Data ONTAP platforms, including the ‚Äúdata center to cloud‚Äù, ‚Äúcloud to data center‚Äù and ‚Äúbetween clouds‚Äù (or ‚Äúbetween cloud regions‚Äù) directions.

OnCommand Cloud Manager



NetApp Cloud ONTAP How It Works


OnCommand Cloud Manager provides a simple wizard-based interface for deploying and monitoring Storage Virtual Machines in Cloud ONTAP and NetApp Private Storage

OnCommand Cloud Manager provides management of the storage environment in a hybrid cloud, including ONTAP Cloud and clustered platforms based on Data ONTAP.

It simplifies the operational management of the ‚Äúdata fabric‚Äù (Data Fabric), including the configuration, support and monitoring of each data storage node, ‚îÄ both virtual devices and real physical hardware.

Actually, OnCommand Cloud Manager implements the deployment environment for ONTAP Cloud, which offers installation with a few mouse clicks and automatically selects and assigns cloud resources for each case.

Deploying ONTAP Cloud, Cloud Manager uses AWS credentials to select the EC2 virtual machine and EBS storage that match the client's request.

Various types of AWS credentials include passwords (passwords), access keys (access keys), multifactor authentication (multi-factor authentication), paired keys (key pairs), certificates (certificates). See more here .

ONTAP Cloud can only be deployed through the Cloud Manager. It handles all actions performed to ensure that all AWS resources, ‚îÄ both computational and storage, are properly assigned to a specific ONTAP Cloud location. Attempting to deploy ONTAP Cloud by any other method will result in failure from the very beginning.

Cloud Manager is the central control point for the cloud-based hybrid environment and can manage ONTAP clustered environments, regardless of its location. It can work both in the AWS cloud and on the on-premise solution in the data center at the client‚Äôs site.

There are two main differences between the versions of ONTAP Cloud, and they depend on the way in which the software is received and installed. OnCommand Cloud Manager is available as AMI on the AWS Marketplace (the AMI version is deployed for the EC2 cloud), or can be downloaded from the NetApp support website. Cloud Manager is available to NetApp customers for free.

Cloud Manager can be used with any clustered Data ONTAP based platforms. It provides configuration, management and monitoring for this environment, but it requires the deployment of ONTAP Cloud and includes advanced features that apply only to the ONTAP Cloud (such as installation, acquisition of cloud resources, installation of a network to work in the cloud, etc.). Cloud Manager also manages the SnapMirror relationship between the ONTAP Cloud and on-premises FAS solutions.

ONTAP Cloud Purchase Models


ONTAP Cloud is available on the AWS Marketplace in several purchase options. It can be paid by the hour with a prepayment (pay-as-you-go), or you can buy a subscription from NetApp.

According to the hourly model, the purchase is made directly in the AWS Marketplace and consists of a fee for ONTAP Cloud software, support and AWS resources. In a subscription model, a customer buys a six-month or one-year subscription from a NetApp sales representative and then activates subscription and support through the Cloud Manager to access the ONTAP Cloud. This approach is called BYOL (Bring Your Own License). Amazon, in this case, bills only for using and ensuring the smooth operation of EC2 and EBS resources.

ONTAP Cloud can be run whenever it is needed. With an hourly scheme, the client pays only for the real time of his work in the ONTAP Cloud and can always disconnect if it is not necessary.

However, if you need to maintain your current data, payment is always charged, otherwise the data will be lost. That is, in the subscription model, the client pays for long-term data management, but only for the actual AWS resources used when ONTAP Cloud is running.
