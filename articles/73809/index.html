PowerShell 2.0 RTM is available for all versions of Windows

Windows PowerShell is a command-line scripting language designed for system administrators and automation. Built on the Microsoft .NET Framework, Windows PowerShell allows IT professionals and developers to control and automate the administration of Windows and applications.

Under the cat are new features that have been added to Windows PowerShell 2.0.


Remoting


Windows PowerShell 2.0 allows you to run a command on one or more remote computers from a single workstation on which Windows PowerShell is running. PowerShell remoting provides several connection options: interactive (1: 1), fan-out (1: many) and fan-in (many: 1 using the IIS hosting model).

Integrated Scripting Environment


PowerShell Integrated Scripting Environment (ISE) provides the ability to run commands, edit and debug scripts in the GUI. The main functions of ISE include syntax highlighting, selective execution, graphical debugging, Unicode support, and context-sensitive help.

Modules


Modules allow script developers and administrators to separate and organize their Windows PowerShell code into their own reusable elements. The code from the modules is executed in its own context and does not affect anything outside the module.

Advanced features


Advanced functions are functions with the same capabilities and behavior as cmdlets. The difference is that they are written entirely in the language of Windows PowerShell, and not compiled in C #.

Background tasks

Developments


This feature adds support for listening, redirecting, and managing system events to the Windows PowerShell infrastructure engine.

Globalization scripts


This new feature allows scripts on Windows PowerShell to display messages in the language that the user speaks and which is installed in the system.

Debugging scripts


New debugging features in Windows PowerShell have been added that will allow you to set breakpoints on lines, columns, variables, and commands and let you specify the action that you want to perform when the breakpoint is reached.

New cmdlets


Windows PowerShell 2.0 contains over 100 built-in cmdlets. These cmdlets, among other things, allow you to perform computer-dependent tasks, tasks on handling event logs and managing performance counters.

WinRM 2.0


WinRM is a Microsoft implementation of the WS-Management Protocol - a firewall-friendly SOAP standard that allows hardware and operating systems of various types to communicate. The WS-Management Protocol specification defines a common path for a system to access and exchange management information between an IT infrastructure.

WinRM 2.0 contains the following new features :

* WinRM Client Shell API provides the functionality to create and manage snap-ins (shells) and snap-in operations, commands and data streams on remote computers;

* WinRM Plug-in API provides functionality that allows the user to write plug-ins that implement an API to support resources or operations;

* WinRM 2.0 contains a hosting framework. Two hosting models are supported. The first is based on the IIS server, the second is based on the WinRM service;

* Bypassing associations allows the user to get instances of Associations classes using the standard filtering mechanism;

* WinRM 2.0 supports the delegation of user authority to multiple remote computers at once;

* WinRM 2.0 users can use Windows PowerShell cmdlets to manage the system;

* A special set of quotas has been added to WinRM that offers better QoS and server resource allocation for concurrent users. The WinRM quota set is based on the quota infrastructure implemented in IIS;
