How we use beta testing on google play

I will describe how we organized the testing and release of our Android Up5 application using the beta test feature on Google Play.
Why it is necessary to use beta testing on Gooplele Play, if on Android a test application can be installed from any source, the main thing in the security settings is to allow installation?

It is necessary to use and here's why. We are our first version, in fact the prototype and tested. Made the main page, laid out apk, wrote instructions on how to allow the installation. And we got a bunch of complaints about the fact that we are putting users on their favorite devices and we don‚Äôt know what. Even our IT friends, who seemed to understand the situation, blamed us.

The beta test feature on Google Play allows this issue, namely, the need for special settings to be excluded since the beta version of the application is downloaded via Google Play with important exceptions. First, applications will not be in search results, and secondly, the user rating, but at first they will not be positive at all, they will not be able to deliver you. Of course he will write a comment in the community, but this feedback will be useful, even if it is negative. This is the meaning of beta testing.

Your application on Google Play can have three states:
  • Alpha
  • Beta
  • Posted by

Each state can have its own file, which is good, because it allows you to organize the flow ‚Äúdevelopment - testing - publication‚Äù. It is important to remember that the versions are non-demountable. For example, I translated version 0.2 into beta, and then tried to return version 0.1 to beta, but this did not work out.

You, in order to organize testing, need to create either a Google group or a Google community. Moreover, it is not obvious, it is better to create separate communities or groups for alpha and beta. For example, you and your relatives take part in alpha testing, and you are inviting third-party, but interested users to beta testing.

It is necessary to warn users that it is impossible to fit into testing with one click. You will need to create an account on Google+, join the community, then click the "I want to become a tester" button and only then download and install the application. Not too easy, but we have users climbed this hill. By the way, this is a kind of protection from those who actually do not need your application.
