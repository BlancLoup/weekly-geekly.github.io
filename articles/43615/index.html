Virtual Movable Type, briefly about the main thing

Virtual Movable Type is a packaged and optimized for fast and convenient work assembly of a popular publishing platform. This build includes:
  • Movable type
  • Plugin for Movable Type Action Streams
  • Linux
  • Apache
  • Mysql
  • Memcache (initially disabled)
  • Fastcgi
  • Imagemagick
  • All required and optional Perl modules

This build was developed in conjunction with JumpBox and runs on the following virtual machines:
  • Parallels
  • VMWare
  • Amazon EC2
  • Microsoft Virtual Server

Who and what is this beast useful for?


This product can be useful to everyone, from beginners who want to become better acquainted with the Movable Type, and for professional users who need to work a lot with templates, quickly edit and test them.

The big disadvantage of this product is that it works in a virtual machine, which itself already takes a lot of resources from the computer. Therefore, I do not advise to put it on weak computers. Its big plus is the speed of work and the engine already fully configured and having no errors. Say, during installation on Denwer, you could run into a lot of problems: both during installation and after. And in Virtual Movable Type you will not find this.
