Choosing an OS for a home server

How it all began

It took me to build a small home server for the following tasks:
  1. Print server
  2. Backups of several home machines (a couple of laptops, one and a half desktop).
  3. Photo archive storage
  4. Music storage
  5. Document storage

For the future, it may be picked up a few more USB-devices, such as webcams and thermometers, but this is so, for the future is strong ...

It will not be a router - another computer is doing this. And this server will handle only internal needs. Plus, for complete happiness, you will hang on another phase (and ideally - also connected via wifi n, but this is also for the future).

Iron

The train looks like this: Intel's motherboard on 330 atoms (dual-core), gigabytes of RAM, Inwin 639 ITX-case, a pair of screws - 250 gigabytes and 640 gigabytes (this is temporary, then replaced by more). Such iron was chosen because of its small size. The case allows you to put only two hard drives, but it will suit me - much large volumes are not expected. Even the current 640 + 250 is enough for a long time. But iron is not interesting, they constantly write about him. And all about the same thing. :)

After I went through the open source, we went to proprietary options, but free (in my case):
  1. Windows XP Home (rolls distribution with the sticker). I will not say anything here. Only from poverty or complete non-support of the necessary hardware by Linux.   2. Windows Web Server 2008. I have 3 of them - MS distributed them in due time at every opportunity. Maybe it would have stopped on it, but WWS does not know how to share a printer, for example. Verified

Somewhere at this stage I remembered a product called Windows Home Server, promoted by MS just like an OS for home servers. After reading the description, I realized that this is almost what I need.

What in general is WHS: it is a server operating system from MS, based on Windows SBS 2003 SP2. That is, inside it is almost a full-fledged Windows Server with some additional features.

What I personally liked:
  • Gathering information about all home computers is initially only a complaint from the security center, but this is expanded with the help of plug-ins.
  • Automatic full backup of all home machines. That is, if the hard drive is covered, you can install a new one, boot from the CD, which will be transferred from the server backup and deploy it to the computer. To save space, identical files that are located on several computers are stored on the server in only one instance. Not very important data can not be backed up - there is an exception setting. Any svap-files, files of hibernation, etc. there are made immediately.
  • Single storage (all disks are combined with a single array, something like JBOD). But there is one very useful feature - the replacement of disks in the array (Drive Extender). That is, you can connect a new disk, turn it on into an array, then poke a ‚ÄúRemove from array‚Äù button on one of the old disks and all the information from it will be automatically pushed out to all other disks. It can be said, for this alone in my eyes, WHS has no competitors - maybe Linux and you can finish it up with the necessary functionality (dig in the direction of LVM, mhddfs, etc.), but this is all the time, plus usability.
  • The ability for important data to assign storage on two disks. Those. choose a daddy, click ‚ÄúDuplicate‚Äù in the properties - and it will be physically stored on two different disks. Those. upon departure of one of them information will not be lost. True backups are not duplicated. On the other hand, it rarely happens that both the original and the backup are covered at the same time.

There, of course, there is some other functionality, but in this case I needed this one.
