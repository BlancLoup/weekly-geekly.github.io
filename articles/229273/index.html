Tarantool User Group Meetup in Moscow

Next Saturday, July 19, we will meet Tarantool users in the Badoo office.

MEATUP.COM EVENT PAGE

Konstantin Osipov (Mail.ru) will talk about Tarantool 1.6, the new version of the server, in which many technical solutions were revised and a convenient, safe and still high-performance DBMS was made.
Tarantool is, first of all, a lock-free database that processes all transactions sequentially in one stream. Tarantool 1.6 is also a full-featured Lua Application Server with support for cooperative multitasking, non-blocking I / O, standard Lua packages and packages created specifically for Tarantool.

Anton Povarov (Badoo) will make a presentation ‚ÄúTarantool in Badoo: storing the history of visits‚Äù.
Every day on badoo.com, users browse about 100 million profiles of other users. We keep counters and a complete history of visits over the past 90 days, with some aggregation - this is about 5 billion events.
The system processing this data stream was created a long time ago and experienced several incarnations, getting closer to the database. At some point we decided to stop reinventing the wheel, abandoned the demons on C + sqlite, did not do it on mysql-yah, radishes and memeshs, but took and washed it on Tarantool.
Let us tell you why Tarantool, as a shardim, is replicable (everything is simple) and how smoothly this business was implemented on a live system without downtime.

It would be cool if someone else could talk about his use case. Write in the comments if there are interesting examples.

There will be a video of the reports.
