HP wants to teach us to the paper Internet?

Hewlett-Packard began testing a unique remote printing service for Cloudprint documents. The technology was developed by a small group of HP researchers for several months, and is now being tested online for the first time. The idea is very simple: in order not to suffer from viewing the mobile site on a small screen, the user orders the printing of the desired web page to the printer, and it is printed not from a pocket computer, but from a remote Cloudprint print server, sending it to a public printer nearest you in A4 format. Now on Google Maps a map of such public printers is being created.

As you know, one of the world leaders in the print market, Hewlett-Packard is now experiencing a certain crisis due to Web 2.0 technologies , which dramatically reduce the need for users to print documents. Statistics show that nowadays only 20% of home printouts are created from Microsoft Word, while more than half of the printouts are web pages, email and other Internet-related activity. Hewlett-Packard is seriously concerned that Web 2.0 services are completely unsuitable for printing. On the contrary, everything is done there in such a way that the user never left the online. Therefore, HP is actively working on new print technologies for the web.

When viewing a web page, a user of the Cloudprint service can order a document to print. In this case, the page is sent to the HP remote print server, where the prepress processing takes place and is saved in PDF format. The user receives a unique code for this document. Later, he can enter this code on the site Cloudprint - and download a beautiful document for printing.
