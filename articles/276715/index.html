3CX SBC - now with encryption and resiliency

Recently, a new version of the 3CX SBC (session boarder controller) has been released. First you need to remind what SBC is and why it is needed.



The most frequently encountered problem when connecting remote phones is a one-way voice or the lack of it, the reason for which can be:
  • Operator lock
  • NAT or worse double NAT
  • Firewall Limitations
  • Peculiar work of routers

3CX SBC builds a kind of VoIP-VPN to an IP PBX and proxies all traffic on one fixed port (5090 by default) and only one port needs to be opened for the correct passage of the call. The method is suitable for those cases when at a remote point it is not profitable to put a full 3CX.

3CX SBC is available in versions for Windows and Raspberry Pi and the update is available for both versions. The main feature is fault tolerance, i.e. SBC can monitor the status of the main PBX and switch to the backup.



Full list of new features


  • Fault tolerance
  • Call Encryption
  • Auto reconnect after channel recovery
  • More private DNS queries for name resolution

Distributions and documentation
