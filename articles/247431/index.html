NFV network function virtualization

Google and Microsoft have joined forces with telecommunications companies AT & T and British Telecommunications (BT) to produce the new generation of major packet data networks, starting with the BGP border gateway protocol . This is the main dynamic routing protocol, one of the main mechanisms for the functioning of the Internet. It is an essential component in routing traffic over IP networks, and is of great importance for the speed of deploying network services virtualization (Network Function Virtualization / NFV).



Now one of the most frequently discussed topics in the world of computer networks is the topic of NFV - a new, promising technology. It allows you to programmatically create such services that are now available only in the form of hardware solutions. Virtualization of network functions allows you to install services there, then and in the quantity that is required now and in this place. NFV was created with a focus on service providers, with knowledge of the features of the service providers and their most pressing problems.

Advantages of NFV:

  • allows you to flexibly and dynamically provide new services, while reducing capital and operating costs;
  • replacing equipment with standardized servers and network components does not tie operators to equipment suppliers;
  • reduces operational costs by simplifying monitoring and administration of operations (all network functions are transferred to a single virtualized infrastructure);
  • fast connection of new users to the network;
  • pays for the infrastructure of telecommunications companies.

When a service provider creates a connection with a new place there are several devices that must be installed on the network, this is, first of all, a managed router and a Carrier Ethernet demarcation device, which are crucial because they separate the customer‚Äôs network and the operator‚Äôs network. In addition, they add equipment that must be installed to monitor and control connections and traffic. In addition to a set of standard equipment already listed above, service providers are often forced to install a number of non-standard devices that are necessary to ensure the client‚Äôs business tasks. And thus, for the service provider, the purchase and service support of such equipment becomes very labor-intensive, especially since it is hardly possible to plan in advance the level of loading of this equipment. NFV helps to solve this problem due to the virtualization of network functions in software applications that can be run both on regular servers and on virtual machines running on these servers.



Virtualization technologies allow you to consider all the IT resources of the organization as a set of public services that can be grouped and re-combined to increase efficiency and rapid scaling. According to Light Reading, the new model was presented as a project on the Internet Engineering Task Force (IETF) - an open international community of designers, scientists, network operators and providers, created by IAB in 1986, which is engaged in the development of Internet protocols and architecture .

Commercial implementation of this project is planned for the near future. Google already uses a new model inside the company. Through a social network for GitHub developers, several companies have begun work on a project already developed by Google and contributed to it. Their task is to ensure the independence of network operators from suppliers, provide the ability to manage BGP configurations, group and combine the routing of software and hardware systems without a long data integration process. According to operators, GP plays an increasingly important role in building networks using software-defined network (SDN) and network services virtualization (NFV).



With NFV, the operator can, using just one network interface device to delimit traffic, allow all other functions to be located on the territory of the service provider, and use the SDN to simplify and manage this network for the client.

Telecommunications companies are interested in SDN and NFV, as it gives vendor independence, the ability to implement, easily plan and deploy networks using hardware and software that companies can choose. The use of ready-made standard equipment and virtualization tools for testing and deploying new services takes a few minutes, not months, as it was before.
