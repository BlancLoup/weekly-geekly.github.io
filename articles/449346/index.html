News from GitHub


  • # 14558 - Sergey Shlokov discovered that in the MODX database schema there is one interesting index that is essentially useless, and in some cases even harmful, and suggested removing it. Removing something is always perceived with caution, so the discussion on the appropriateness of this action continues on github. However, the evidence is very obvious and the garbage should definitely be removed from the system.
  • # 14559 - Ilya Utkin discovered a regression of changes, which is expressed in the missing animation of checkboxes on the resource page and corrected this misunderstanding.

Interesting notes and articles


  • Mat Dave Jones in a blog on modx.com wrote a good article about choosing a control system for the site (from open source solutions), which you can use to communicate with potential customers. Translated into Russian, the title reads ‚ÄúRead this before choosing an open source content management system , ‚Äù the note itself is in English.
  • Personal pages of MODX Revo users - Kirill Korovin wrote a note on his website on how to organize personal pages for users registered on the website. You should decide what to put on these pages yourself, but the general approach to implementation is described well.
  • Several paid applications have become free - Andrey Stepanenko decided to make several of his add-ons free. On the one hand, this may please someone, since there is no need to pay more money, but at the same time, these additions are now delivered as they are, without guaranteed technical support and development.
  • Under a note with the news about the closure of the MODX club Roman Voropayev shared an interesting opinion on the current status of MODX . No less interesting answer from Sergey Shlokov there. If you have something to say on this topic, you can join the discussion. It would be interesting to see the detailed opinions of people on this subject.
  • Translitor as an alternative to transliterating pseudonyms - Nikolay Savin proposed his improved class for more ‚Äúcorrect" transliteration of pseudonyms, the implementation of which was peeped in a similar solution from the world of Laravel. Similar behavior can be achieved with standard tools by changing the system settings, but it may be possible to install the package in one click by someone in an easier way.
  • The use of predefined templates for TinyMCE is a note about the capabilities of the TinyMCE editor, which is often used in MODX. Namely, how it is possible for the editor to prepare in advance the blank blocks, which at the time of editing can be selected and inserted into the document. Almost like ContentBlock, only simpler and free.

New and updated add-ons


  • modRetailCRM-2.5.0 - updated the package for integrating MODX with RetailCRM, which now supports full-fledged work with modifications from msOptionsPrice2, and also can unload goods with all properties in xml-format. Nikolai Savin, the author, wrote a short note about the update .
  • AdminTools-1.15.0 - Sergey Shlokov updated his component with a set of convenient tools for admin MODX, adding an indicator of unread messages there and correcting along the way several bugs inside.
  • Twilio-0.0.2-beta1 is a package for integration into MODX of Twilio service for sending SMS from YJ Tso (sepiariver). The author has posted documentation on use on his site .
  • instaFilters-1.0.0-pl is a terribly simple component that adds a filter to the image shown in the parameter, as in Instagram, but in pure CSS. Free, motivation and examples of use in the author's note .
  • mChimpX-2.1.0-pl1 - Anton Tarasov updated his component, which itself is a hook for FormIt, which allows you to sign a user on the Mailchimp mailing list at the time of sending the form. Mainly updated line breaks, added translations of some error messages and just fixes bugs.
  • msTimeStamp-1.0.1-beta - Andrei Stepanenko published his free add-on, which allows you to remove statuses of goods (new, special or popular) after some time. It works without the need to install tasks in the crontab. Read more in the author's note .
  • altHelper-1.0.1-beta - Pavel, after his note on solving the problem of automatically filling out alt-attributes in pictures , put together his decision in a free package. Use.
  • amoCRM-1.2.1-beta - Mikhail Voevodsky has released a large update package for integration with amoCRM. I wanted to list the innovations, but they are specific and in fact there are a lot of them. Especially since all the fixes are diligently listed in the changelog package.
  • msPre-2.2.20-pl - a large component update for mass editing of resources (or even products). Of the new features - selected resources and filter on them for faster catalog management and also the rapid creation of groups of products through the minishop. More details in the author's note .
