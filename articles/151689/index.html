Indie-Tracker Update

Updated Indie-Tracker - a simple task tracker for small development teams. The new version includes many small and large improvements in various aspects of the project.

Main function


The main change in this version is the addition of groups and the addition of a new working screen. The screens are divided functionally: the first screen - as in the previous version of the tracker - has the usual columns for observing the current progress of the tasks. The main task of the first screen is to see-remember what needs to be done now.



The second screen is for planning and filling. It is more convenient to create tasks, arrange them in groups and observe the overall progress. For example, if only 1 person is involved in the project, you can create groups like these: server, design, client, new functions, ideas for the future, and mistakes.



And if there are several people, it will be more convenient to divide tasks into groups of the type Vasya, Petya, Masha.



Hotkeys and Mouse

Added hot keys: without the help of the mouse, you can create tasks (Ctrl + N) and groups (Ctrl + G), as well as switch between screens (Ctrl + Tab).
The mouse support is also expanded: by clicking the middle key on the group header, the last one is closed, by clicking on the dimming screen - the pop-up window is closed, and by clicking the group name in the list of created groups, the corresponding column is opened / closed.

Save on the fly

Tasks are now saved on the fly, no need to wait for any responses, just work, and the program will take care that the data is synchronized with the server. A bit like the Google Docs.

Other improvements

Percent 80% rewritten server and rebuilt database, increased download speed
updated design
improved interface performance: tracker easily raises the project to a thousand tasks

Ideas for the future

There are plans for development in the following areas:
  1. Full keyboard support without using a mouse.
  2. More mouse control (context menus, auxiliary keys)
  3. Support for multiple projects and multiple users.
  4. Tag system


Questions


Points 1 and 2 are small and simple, I have no doubts, no questions. These are tasks for a snack. More interested in item number 3. Actually the question: how best to provide this functionality? For a more complete understanding of the issue I propose several possible solutions:
  • Each user can create multiple projects. Projects can be shared with other users.
  • Each user can be a member of his own team of one person and be invited to the teams of other users. Projects exist and are visible only within one team.


Basement


Finally, I wanted to remind you that the tracker does not pretend to be fat, the goal of the project is to provide a comfortable and simple software interface for managing tasks for small development teams that do not have the goal of counting man-hours and are not fooled by all this managerial.

The use of the tracker is possible for free, without restrictions. Register, try, use: indie-tracker.com .
