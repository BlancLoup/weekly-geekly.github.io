New image search interface: 36.5% more efficient



Search@Mail.Ru launched a new image search interface. Now it is available at the short address: http://images.mail.ru . The impetus for the changes were the results of our research, aimed at identifying the preferences and goals of users when searching for pictures.
We found out a lot of useful things - for example, we understood that most often pictures are looking for ...
  • See the place where soon going to go
  • Find photo examples, learn to take pictures.
  • Choose cards as a gift
  • Decide what to do shopping - from t-shirt to car
  • Create presentations, articles, wall newspapers
  • Just cheer yourself up
How did all this affect the design of the new image search interface?

We found that during the search for a suitable image, the user goes through three stages:

Let's see what you have here ...
First, the user evaluates whether the search results match his expectations. Therefore, as a rule, it is not limited to one page of results, but proceeds to the next.

As it turned out, according to the results of the analysis, users spend a lot of time on page navigation, because they have to aim at small numbers. To speed up the transition through the pages (and, therefore, the search itself) we replaced the page-by page with the large button ‚ÄúShow more ...‚Äù, when clicked, which has more than 20 additional images. Note that the previous pictures, while not disappearing - you can quickly return to them by scrolling to the top of the page.


Why haven't we done ‚Äúendless scrolling?‚Äù We realized that in this case people quickly scroll down, often not noticing the required results and increasing (and not reducing) the search time.

Faster, higher ... bigger!
Secondly, if the picture seemed appropriate, the user wants it to be larger. Therefore, we made it so that the size of the large image is automatically ‚Äúadjusted‚Äù to the size of the browser window.


We insure ...
Thirdly, among other things, the Search found that users often need to look at other options for pictures, while leaving the selected picture "in plain sight."

We did not disregard this need - now you can quickly continue the search by scrolling the mouse wheel, and the selected picture will be to the left of the search results. Of course, if necessary, you can quickly return to the list of pictures by simply clicking on the ‚ÄúHide Picture‚Äù link.

Growing up
Fourth, now we have 80% more pictures per web page - see for yourself:


The new interface is implemented using Ajax technology - it allows you to navigate without reloading the page - faster and more economically. Only those images that are currently visible on the screen are loaded - this saves traffic and minimizes the time required for the browser to display a new portion of images. It turns out a simple formula that every user can understand: less traffic -> less time to load -> less money.

In addition, we tried to compensate for the traditional cons of Ajax. For example, you can use the familiar ‚ÄúBack‚Äù and ‚ÄúForward‚Äù browser buttons and save a bookmark to your favorite picture in your favorites.

Plus 36.5% for everyone!
To test the new interface in action, Poisk@Mail.Ru conducted a comparative test: before and after the interface changes. During the study, users performed typical tasks, and we measured the time spent on their implementation. It turned out that using the new image search interface, users perform their tasks 36.5% faster.
More information about the results and testing methodology, you can find out in our community .
