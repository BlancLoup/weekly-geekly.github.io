We work with SkyDrive through Windows Explorer

I want to tell about one interesting program Gladinet cloud Desktop, in particular about its interaction with the service from Microsoft - SkyDrive.
Data service has already been mentioned on Habr√© - for example here .
The program itself can be downloaded offsite .

1. To start, download the program. There are two versions: Starter (free) and paid Professional. Starter includes 30 days of work in Pro mode. After installation in ‚ÄúMy computer‚Äù in the network location section, and in Russian Windows A new disk will appear on the network . In my case, it is called SkyDrive.



2. Go to the created disk and double-click on the label Click to mount [cloud storage]. The Add Virtual Folder Wizard opens. The program can work with several services, but at the moment we are interested in SkyDrive, so we choose it. Also enter the folder name.



3.The next step is to enter your Windows Live account details.



4. Further indicates whether we want to open the folder after creation and click Finish.



5. The program will create a folder structure on your disk as on your SkyDrive



Now you can use all the functions of SkyDrive, working as in Explorer. The maximum storage size is 25GB. Maximum file size 50MB

6. As an addition, you can make a shortcut to this folder in the "Send" menu. To do this, enter shell: sendto in the address bar of the explorer and create a shortcut to the SkyDrive folder inside it.



After these manipulations, you can send files to SkyDrive via the context menu.

UPD It turned out that the program clogs the 80th port and does not allow the Apache to work. The authors propose two solutions:
1. Transfer the Apache to another port
2. Run the program after Apache (for some reason there are no such problems)
