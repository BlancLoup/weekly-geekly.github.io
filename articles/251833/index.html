PHPCI updated to version 1.6



The PHPCI continuous integration system has grown to version 1.6 ( review , upgrade to 1.3 ), and I would like to talk about the changes that have occurred since version 1.3.

And so, the main list of changes in version 1.6:
  • SVN support has finally appeared
  • Reworked project summary widget in control panel to add information about several recent builds
  • For the Codeception and Behat plug-ins, widgets appeared in the UI
  • The system can now leave comments in merge requests on Github, noting errors found during the build
  • The Technical Debt plugin appeared (For fixing technical debt in the project: search for TODO, FIXME, etc. marks)
  • Many bug fixes and minor improvements.

As well as major changes released with versions 1.4 and 1.5 earlier:

  • Fully redesigned UI compared to version 1.3 (now Admin LTE is used for UI)
  • There was a plugin for Gulp (Javascript)
  • Added filtering of assemblies by branches on the project page
  • Added JSON web hook support for Github
  • There was a plugin for archiving in PHAR
  • Added the ability to silently install the system (for unattended installation)
  • The build status icon is aligned with Shields.io
  • Variable% BRANCH% added to plugin configuration
  • Added localization of the system into various languages ‚Äã‚Äã(at the moment the project is translated into English, Russian, Ukrainian, French, German, Danish, Dutch, Polish, Italian, Greek)
  • Added the ability to use the system without authentication in single user mode (for local use, use in corporate networks with its own authentication system, etc.)
  • There was a plugin for Slack notification
  • Added support for Merge requests for Gitlab.
  • A plugin for parsing PHP TAL templates has appeared.
  • Implemented Gitlab version 5 support.
  • Improved charts
  • And also again - fixed bugs and made improvements.

I would like to add from myself that I actively use PHPCI, and at the moment it almost completely meets my needs. The only major improvement that I lack is complete happiness (to give up TravisCI for my projects) is an assembly on different versions of PHP, but I believe that sooner or later this functionality will appear.
