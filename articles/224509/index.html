Rendering and viewing 3D objects


At this stage, we will create our first scene and add a couple of objects and a camera. Our first example will contain the following objects:
The plane is a two-dimensional rectangle that will serve as our main platform. It will appear as a gray rectangle in the middle of the scene.
The cube is a three-dimensional cube, which we will render in red.
Sphere - a three-dimensional sphere, which we will render in blue.
Camera - it defines what we will see in the output.
The axes are x, y and z. This is a useful debugging tool to see where objects are rendered.
First, let's see how it looks in the code, and then we will try to figure it out. 

 <script type="text/javascript"> $(function () { var scene = new THREE.Scene(); var camera = new THREE.PerspectiveCamera(45 , window.innerWidth / window.innerHeight , 0.1, 1000); var renderer = new THREE.WebGLRenderer(); renderer.setClearColorHex(0xEEEEEE); renderer.setSize(window.innerWidth, window.innerHeight); var axes = new THREE.AxisHelper( 20 ); scene.add(axes); var planeGeometry = new THREE.PlaneGeometry(60,20,1,1); var planeMaterial = new THREE.MeshBasicMaterial({color: 0xcccccc}); var plane = new THREE.Mesh(planeGeometry,planeMaterial); plane.rotation.x=-0.5*Math.PI; plane.position.x = 15; plane.position.y = 0; plane.position.z = 0; scene.add(plane); var cubeGeometry = new THREE.CubeGeometry(4,4,4); var cubeMaterial = new THREE.MeshBasicMaterial( {color: 0xff0000, wireframe: true}); var cube = new THREE.Mesh(cubeGeometry, cubeMaterial); cube.position.x = -4; cube.position.y = 3; cube.position.z = 0; scene.add(cube); var sphereGeometry = new THREE.SphereGeometry(4,20,20); var sphereMaterial = new THREE.MeshBasicMaterial( {color: 0x7777ff, wireframe: true}); var sphere = new THREE.Mesh(sphereGeometry,sphereMaterial); sphere.position.x = 20; sphere.position.y = 4; sphere.position.z = 2; scene.add(sphere); camera.position.x = -30; camera.position.y = 40; camera.position.z = 30; camera.lookAt(scene.position); $("#WebGL-output").append(renderer.domElement); renderer.render(scene, camera); });

If we open this example in the browser, we will see something similar to what we wanted, but still far from ideal.



Before we start making the scene more likeable, let's take a walk and see step by step what we did: 

 var scene = new THREE.Scene(); var camera = new THREE.PerspectiveCamera(45 , window.innerWidth / window.innerHeight , 0.1, 1000); var renderer = new THREE.WebGLRenderer(); renderer.setClearColorHex(0xEEEEEE); renderer.setSize(window.innerWidth, window.innerHeight);

In this example, we defined the scene, camera, and visualizer. A scene variable is a container that is used to store and track all the objects we want to display. The sphere and cube that we want to display will be added in this example a little later. In this snippet, we also created a camera variable. This variable defines what we will see when we render the scene. Next, we define the object of visualization (rendering). Rendering is responsible for the calculations, for how the scene will look in the browser in one or another perspective. We also created a WebGLRenderer object that will use the video card to render the scene.
Here, using the setClearColorHex() function, we set the renderer's background color to almost white (0XEEEEEE), and also set the size of the rendered scene using the setSize() function.
So far, we only had basic elements, such as an empty scene, render and camera . But, however, there is still nothing to visualize. The following code snippet adds auxiliary axes and a plane. 

 var axes = new THREE.AxisHelper( 20 ); scene.add(axes); var planeGeometry = new THREE.PlaneGeometry(60,20); var planeMaterial = new THREE.MeshBasicMaterial( {color: 0xcccccc}); var plane = new THREE.Mesh(planeGeometry,planeMaterial); plane.rotation.x = -0.5*Math.PI; plane.position.x = 15; plane.position.y = 0; plane.position.z = 0; scene.add(plane);

You can see that we created an ax object and used the scene.add() function to add axes to our scene. Now we can create a plane. This is done in two steps. First we determine that the plane will be displayed using new THREE . The code will be PlaneGeometry(60,20) . In this case, our plane will have a width of 60 and a height of 20. We should also tell Three.js what our plane should look like (for example, its color or transparency). In Three.js, we do this by creating a material object. For the first example, we will create a base material (using the MeshBasicMaterial() method) of color 0xcccccc. Next, we merge these two objects into one Mesh object called plane . Before we add a plane to the scene, we must put it in the correct position; we do this by first turning it 90 degrees around the x axis, and then we determine its position on the stage using the position property. Finally, we have to add this object to the scene, just as we did with the axes object.
The cube and sphere are added in the same way, but the value of the wireframe property is true , so let's move on to the final part of this example: 

 camera.position.x = -30; camera.position.y = 40; camera.position.z = 30; camera.lookAt(scene.position); $("#WebGL-output").append(renderer.domElement); renderer.render(scene, camera);

At this stage, all the elements that we wanted to visualize are added to the scene in their proper place. We have already said that the camera determines what will be displayed. In this piece of code, we position the camera using the x, y, and z values ‚Äã‚Äãof the position attribute and it starts hovering over our scene. To make sure the camera is looking at our objects, we use the lookAt() function to point to the center of our scene.

Adding materials, lighting and shadows


Adding new materials and lighting to Three.js is very simple and is done in much the same way as we did in the previous section. 

 var spotLight = new THREE.SpotLight( 0xffffff ); spotLight.position.set( -40, 60, -10 ); scene.add(spotLight );

The SpotLight() method illuminates our scene from the position of spotLight.position.set( -40, 60, -10 ) . If you render the scene again, you will notice that there is no difference with the previous example. The reason is that different materials react differently to light. The main material that we used in the previous example for objects (using the MeshBasicMaterial () method) does not react to light sources on the stage. They simply visualized objects in a specific color. Thus, we need to change the materials for our plane, sphere and cube as shown below: 

 var planeGeometry = new THREE.PlaneGeometry(60,20); var planeMaterial = new THREE.MeshLambertMaterial( {color: 0xffffff}); var plane = new THREE.Mesh(planeGeometry,planeMaterial); ... var cubeGeometry = new THREE.CubeGeometry(4,4,4); var cubeMaterial = new THREE.MeshLambertMaterial( {color: 0xff0000}); var cube = new THREE.Mesh(cubeGeometry, cubeMaterial); ... var sphereGeometry = new THREE.SphereGeometry(4,20,20); var sphereMaterial = new THREE.MeshLambertMaterial( {color: 0x7777ff}); var sphere = new THREE.Mesh(sphereGeometry,sphereMaterial);

In this section of the code, we changed the material property for our objects using the MeshLambertMaterial function. Three.js provides two materials that perceive light sources: MeshLambertMaterial and MeshPhongMaterial.
However, as shown in the following screenshot, this is still not what we are striving for:



The cube and the sphere turned out a little better, but so far there are still no shadows. Shadow rendering takes a lot of processing power and for this reason shadows are disabled by default in Three.js. But their inclusion is actually very simple. For shadows, we need to change the sources in several places, as shown in the following code snippet: 

 renderer.setClearColorHex(0xEEEEEE, 1.0); renderer.setSize(window.innerWidth, window.innerHeight); renderer.shadowMapEnabled = true;

The first thing we need to do is tell the render that we want to include shadows. You can do this by setting the shadowMapEnabled property to true . If you look at the result of this action, we will not notice anything. This happens because we have to explicitly specify which objects will cast shadows and on which objects it will fall. In our example, we want the sphere and the cube to cast a shadow on our plane. You can do this by setting the corresponding properties to true: 

 plane.receiveShadow = true; ... cube.castShadow = true; ... sphere.castShadow = true;

Now there is one more thing that you have to do in order for the shadows to appear. We need to determine which of the light sources will cause shadows. They can not do it all, this will be discussed in the next part, in our example we will use the SpotLight() method. Now we just have to set the correct property and finally visualize the shadows. 

 spotLight.castShadow = true;

Complicate our scene with animation


If we want to somehow revive our scene, then the first thing we need to do is find a way to re-redraw the scene at a certain interval. We can do this using the setInterval(function,interval) . With this function, we can define a function that will be called every 100 milliseconds. The problem with this function is that it does not take into account what happens in the browser. If you are viewing another tab, then this function will still be called every few milliseconds. In addition, the setInterval() method is not synchronized with screen redrawing. All this can lead to high CPU utilization and low efficiency.

Introduction to the requestAnimationFrame () method


Fortunately, modern browsers have a solution to the problems associated with the setInterval() function: the requestAnimationFrame () method. With this function you can specify a function that will be called at intervals defined by the browser. You just have to create a function that will handle rendering, as shown below: 

 function renderScene() { requestAnimationFrame(renderScene); renderer.render(scene, camera); }

In the renderScene () function, we call the requestAnimationFrame () method again to keep the animation going. The only thing we need to do is change the location of the call to the renderer.render() method after we have created the entire scene, we call the renderScene () function renderScene to start the animation: 

 ... $("#WebGL-output").append(renderer.domElement); renderScene();

If you run this code snippet, you will not see any changes from the previous example, because we haven‚Äôt yet animated anything.
