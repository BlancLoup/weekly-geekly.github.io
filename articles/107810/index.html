How to make PHP friendly with the Google Chrome console

Attention

This article contains information about the outdated version of PHP Console.

PHP Console 1.0





It will be a question of one marvelous extension for Google Chrome, which allows you to proxy error output and debugging messages from PHP to the Google Chrome console, as well as display them as pop-up popup notifications.

The PHP Console extension was written as a plugin for a fairly high-quality and functional error handler - Lagger . If the Lagger functionality is redundant for you, then you can use the compact PhpConsole class. A list of what it can do more:

‚úì There is no icon in the toolbar, all settings are made from the context menu
‚úì Proxies error / debug messages to the Google Chrome console.
‚úì Displays error / debugging messages in the form of temporary popup notifications.
‚úì The first notification is displayed within 3 seconds.
‚úì The display time of all other notifications is configured
‚úì The notification does not disappear if you click on it or hover the mouse
‚úì All notifications can be closed by clicking on the icon (x) to close the popup.
‚úì Displays the backtrace of the error call / exception in the console (click on Object)
‚úì Displays the backtrace of the error / exception call in the notification (click on the link # file_path)
‚úì Displays javascript errors in the form of temporary popup notifications.
‚úì Does not override the custom JavaScript error handler.
‚úì The error URL is clickable and opens the source code of the script / page.
‚úì Ignores repetitive errors.
‚úì Ignores internal errors in Google Chrome extensions.
‚úì Catch messages from scripts from any sources.

How is it better than the same FirePHP:
  1. Normally handles fatal errors.
  2. Traps messages in AJAX scripts and IFrames
  3. It has a very convenient function to display messages in pop-up notifications.
  4. Displays call backtrace
  5. Catches javascript errors
  6. Supports a more functional and highly configurable Lagger library
