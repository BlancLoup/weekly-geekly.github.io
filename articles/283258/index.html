[Peter] Meeting about OpenOnload: high-performance network stack for Linux

On Thursday, May 12, at 8:00 pm, a CodeFreeze meeting with Konstantin Ushakov , the OKTET Labs director, will take place at the Odnoklassniki St. Petersburg office. The topic of the meeting is Solarflare OpenOnload , a network stack for Linux, which reduces latency, processor load and increases the speed of packet transmission.



Solarflare OpenOnload is a high-performance network stack that drastically reduces latency, processor load, and increases packet speeds. This is achieved without any changes in your application itself, be it C / C ++ / Java or anything using the BSD Sockets API). Although the idea of ‚Äã‚Äãstacks working partially or completely in userland is not new, the radical difference of Onload is in the BSD Sockets API provision.

The report will affect the technical side of the implementation:
  • the sharing of resources between the core and the process;
  • packet routing;
  • polling mechanisms (poll, epoll, etc.);
  • socket caching

Konstantin will show why the simple idea underlying the idea is not so simple to implement (with examples of "unsuccessful" attempts). There will be numbers and there will be details of how the Linux kernel tries to achieve similar performance (including DPDK).
