Android x86 on Asus Eee PC 900

What for?


My aunt brought me a pink netbook, presented by her daughter, with the problem ‚Äúeverything slows down and does not work.‚Äù And I want to be able to climb on the Internet, watch movies and play simple toys. And so it happened that the netbook was equipped with a very slow SSD. And since the aunt has Android on the phone, it was decided to put it exactly - enough for the Internet.



Baseline


Asus Eee PC 900
Android-x86 4.0-RC1 Eee PC family (android-x86-4.0-RC1-eeepc.iso)

Installation


Installation is carried out according to the instructions
On the Asus Eee PC s101h, the installation went smoothly.
On the Asus Eee PC 900 in the BIOS, disable CardReader (you can enable it after installation).
At the end of the installation, the installer allows you to Create Fake SDCard or you can manually .







Network


Wi-Fi works out of the box.
Ethernet according to ReleaseNote 4.0-RC1 is not supported (no GUI).
However, craftsmen brought from the terminal: 

# netcfg eth0 up # netcfg eth0 dhcp # setprop net.dns1 8.8.8.8



Russian keyboard layout


For the Russian keyboard layout we put the Russian Keyboard , in the keyboard settings we specify to use an external keyboard.



Applications


With applications that use NDK and naturally do not have bundled libraries for x86, everything is very sad.





But in most cases there are analogues





Final refinement


Included are a cloud of applications unnecessary to the end user (benchmarks, Dev Tools, etc. for debugging and development), as well as applications related to the telephone part, which was turned off. 2D graphics acceleration was turned on in the settings, auto-rotation of the screen was turned off (after opening the netbook lid, the screen was turned to portrait mode and back to landscape mode did not return).





Total


Aunt from work happy. Gladly now clicks on the netbook in Lines.
