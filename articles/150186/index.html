Rules for declaring classes / methods / variables.

  • The source file must have the same name as the only public class in this file.
  • Each file can only belong to one package, but you can import (import) several classes
  • The package keyword is always higher than the import keyword, which, in turn, is always above the class declaration.
  • package and import are applied to all classes inside the file, regardless of their access level / names, etc.
  • Non-public classes can have any name that is not associated with the file name, if it does not contradict the rules for creating identifiers.


Class access modifiers.

  • There are three access modifiers : public, protected, private. And as many as four levels of access: public, protected, default, private. The default access level is determined by the absence of modifiers.
  • For external classes, only two access levels are available: public and default. This is logical, because the private class outside the file cannot be seen, and protected makes sense only for methods.
  • public classes are available from any package; default classes are available only from the same package they are in.
  • If class A is available to class B, then class B can:
    1. Create class A objects 
       A a = new A();
    2. Inherit class A through the keyword extends 
       class B extends A { }




Other modifiers for the class.

  • You can apply final modifiers (forbids inheriting a class), abstract (makes a class abstract) and strictfp (a class works with non-integer numbers) 
     final strictfp class FinalClass{ } class RockIsDead extends FinalClass{ //   }

  • A class cannot be both final and abstract (these are mutually exclusive modifiers), but strictfp can be used in any combination with final and abstract.
  • You cannot create objects of abstract classes.
  • Abstract classes can have methods with implementation, and abstract classes cannot have abstract methods. 
     abstract class Sweet { void dreams() { //  System.out.println("Ummm"); } abstract abstractDreams(); //   } class Fail { abstract void anotherFail(); //  }
  • A non-abstract class that inherits an abstract class must implement all the abstract methods of this class.


Interfaces

  • Interfaces are 100% abstract classes; they cannot contain non-abstract methods.
  • Interfaces tell the class what to do, but not how it will do it.
  • Interfaces can be implemented by any class.
  • Interface methods have access modifiers public and abstract by default.
  • An interface can have a variable that always has public static final modifiers. They can be omitted; a variable in the interface is always created with parameters that actually make it a static constant.
  • A non-abstract class that implements an interface must meet the following requirements:
    1. Implement all interface methods.
    2. Do not throw exceptions that are not declared in the interface.
    3. Use the same signature to implement all interface methods.
  • The class that implements the interface may be abstract. In this case, it can implement an arbitrary number of interface methods.
  • A class can inherit only one class, but it can implement many interfaces. Interfaces can inherit many interfaces, but cannot inherit classes, both abstract and ordinary.


Modifiers for class fields.

  • Class fields can have all four access levels.
  • One class can access the fields of another class in two ways: either through inheritance, or through the creation of an object.
  • If class A is not available class B, then the methods of class A are not available for class B.
  • public fields are accessible from any other class, protected through inheritance are available to any class, and through the creation of an object, only classes from the same package are available. The default fields are accessible in any way from only one package, and private fields are invisible to no class other than their own.
  • Access to the class fields without using a dot is possible only from the class itself.


Local variables.

  • Local variables do not have an access modifier.
  • The only modifier available for local variables is final.
  • Local variables, unlike class fields, are not initialized during compilation; you will have to do it by hand.


Other information about modifiers.

  • final methods cannot be overridden.
  • Abstract methods are declared with all the same buns as conventional methods, with the exception of the method body (there is not even empty braces).
  • The synchronized modifier is used only for a method or for blocks of code. 
     synchronized void hulkLomat(){ } void hulkStroit(){ synchronized (this){ //   } }
  • The native modifier is used only for methods.
  • The strictfp modifier is used only for classes or methods.


Methods with a variable number of variables (var-arg)

  • Syntax - variable_type ... variable_name 
     stupiedMethod (int... varArg) { } // stupiedMethod (int n, int... varArg) { } // stupiedMethod (int... varArg, int n) { } // stupiedMethod (int... varArg, int.. varArg1) { }
  • As you can see above, var-arg should always be the last argument.
  • Var-arg is used as regular arrays:

Enumerations.

  • Think of listings as special classes.
  • The types of constants in enums always match the name of the enumeration.
  • Enumerations can be declared inside or outside a class, but not inside a method.
  • If an enum is declared outside the class, then it cannot have the private access modifier.
  • Enumerations can have constructors, methods, variables, object constructors
  • SomeEnumName.Small (2) passes the int value 2 to the Small constructor
  • After listing, the semicolon is optional.
  • Cannot call enum constructor directly.


Static variables and methods.

  • Static methods do not have access to non-static variables.
  • To gain access to a static field or method, you do not need an instance of the class.


Arrays

  • Arrays can store primitive types or objects, but they themselves are always objects.
  • You can declare an array like this: int [] array, or you can like this: int array [].
  • You should always explicitly specify the size of the array when creating it.


miscellanea

  • Class fields can be final or transient (the transient modifier indicates that the object does not need to be serialized).
  • You can call local variables as well as the name of a class.
  • The final modifier does not mean the immutability of the object, it means the immutability of the reference to the object. In other words: declaring final ArrayList list = new ArrayList (); You can change the state of the object referenced by the list variable, but you cannot force the list to not refer to the created object.
