Significant progress of the experimental ARWINSS branch of the ReactOS operating system

The day before yesterday, Alexey Bragin ( reactos ), the coordinator of the ReactOS project, posted a message in the international section of the forum about his significant success in developing the experimental ARWINSS branch. Translation of text with a list of changes and screenshots under the cut.


As you already understood, sf.net has a new ISO image (42.6 MB) that is awaiting your comprehensive testing.

List of changes:
- The processing of standard objects has been moved to the user mode (there are fewer problems like ‚Äútrying to find an unknown descriptor‚Äù, but they still exist).
- Multiple changes in the mechanism for managing windows and device context, eliminating incorrect cropping and calculating the scope. The image on the screen has become clearer.
- Removed unnecessary in some cases, redrawing the entire screen (increased performance).
- Fixed various errors, for example, the use of released data (more stability, less critical failures).
- Fixed deployment of windows to full screen: the height of the taskbar is now taken into account when calculating the size of the window (for the proper deployment of the window and scroll).
- Fixed minimizing windows: the window is now hidden properly and the top-level windows are no longer minimized to the icon on the desktop.
- Fixed recovery of the window by clicking the button on the taskbar.
- Now use another, Windows-compatible way to draw a rectangle when selecting objects (fixed the appearance of strange purple lines instead of the selection rectangle).
