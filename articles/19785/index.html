Elephants are already here. Fast, reliable, powerful. PostgreSQL 8.3!

More than 200 developers, more than 300 patches, 15 months of hard work of developers and testers ... And now - the newest version of the best DBMS in the world is ready for use in industrial environments!

On February 4, 2008, the PostgreSQL Global Development Group (PostgreSQL Global Development Group) announced the long-awaited release of version 8.3 of the most advanced open source database, the release of which further strengthens PostgreSQL‚Äôs position as the most productive open source database system. Among the innovations relating to performance, it is worth highlighting:

* HOT (Heap Only Tuples)
* bgwriter process auto-tuning mechanism
* asynchronous transaction commit (Asynchronous Commit)
* Spread Checkpoints
* "Synchronized scans" (synchronized scans)
* additional reduction of disk space ("Var-Varlena")
* L2 cache protection
* reducing the speed of "speed wrap" transaction counter (Lazy XID)

PostgreSQL 8.3 also contains many interesting and attractive new products that developers and database administrators will appreciate. Here are some of these innovations:

* CSV logging
* SQL / XML (!)
* support MS Visual C ++
* ENUM types
* built-in Tsearch (!)
* SSPI & GSSAPI
* arrays of composite types
* pg_standby

This list is very short, this release contains many, many more. Look at the list of innovations and the matrix of possibilities , look through the description of the release , read the article in Russian . And be sure to download and try!
