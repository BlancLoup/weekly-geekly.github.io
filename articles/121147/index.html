BrowserDeps - different browsers for different connections

Good evening! It turned out that I use Opera Mini to surf through the mobile Internet and Firefox or Opera Mobile for Wi-Fi. I will explain this by saying that Opera Mini, due to the compression of traffic, including pictures, works quite well through slow connections. And if you consider that in St. Petersburg 3G is far from being caught everywhere and not at all as good as we would like, then you have to use slow mobile Internet everywhere where there are no hotspots. But if you're lucky enough to cling to the hotspot, then Opera Mini is no longer so good - through a proxy the traffic goes longer than directly, and the JavaScript does not work completely. This is where other browsers meet.

Previously, I had to choose a browser in which this link opens after each clicking on some link in the reader or twitter, and now I don‚Äôt need it anymore. And all because I had some free time and a huge desire not to choose this damn browser anymore.

For several evenings of reading the documentation (the first time I am writing under android), the first version was born, which can already be used. He named the first reading flown into the head - BrowserDeps, from the words Browser and Dependency. Quickly cooked up an icon, figured out git (there was a reason for this) and ... Now you can download it. I hope someone besides me will need it too :)

clickable

The principle of operation is quite simple. The application is registered in the system as a browser, you choose it as the default browser. And the next time when you click on the link, the application will start, check the connection and launch the necessary browser.
