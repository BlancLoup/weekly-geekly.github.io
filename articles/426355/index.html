Taylor Rows


Pros:

  • Highest accuracy
    This function, used to calculate the Sin value, can be used to calculate an infinitely accurate Sin value . The more iterations it undergoes, the more accurate the output is the value (in the hypothesis). In programming practice, it is worth considering rounding errors of calculations depending on the types of parameters used (double, float, decimal, etc.).
  • Calculates any angle
    As an argument, you can enter any value in the function, so there is no need to monitor the input parameters.
  • Independence
    It does not require preliminary calculations (as functions discussed below), and often is the basis on which faster functions are assembled.

Minuses:

  • Very low speed (4-10%)
    It requires a lot of iteration for accuracy to be close to the accuracy of Math.Sin, as a result of which it works 25 times slower than the standard function.
  • The greater the angle, the lower the accuracy.
    The greater the angle entered in the function, the more iterations needed to achieve the same accuracy as Math.Sin.

Original appearance (speed: 4%):

The standard function involves the calculation of factorials, as well as the exponentiation of each iteration.



Modified (speed: 10%):

The calculation of some powers can be reduced in a cycle (a * = aa;), and other factorials can be precomputed and put into an array, while changing the signs (+, -, +, ...) can be not raised to a power and also reducing their calculation using the previous values.

The result is the following code:

Polynomials


I encountered this method on the Internet, the author needed a quick search function Sin for doubles of lower accuracy (error <0.000 001) without using libraries of previously calculated values.

Pros:

  • High speed (9-84%)
    The initial polynomial unchanged showed a speed of 9% of the original Math.Sin, which is 10 times slower. Due to small changes, the speed increases dramatically to 84%, which is not bad if you close your eyes to accuracy.
  • No additional preliminary calculations and memory required.
    If above and below we need to make arrays of variables to speed up the calculations, then all the key coefficients were kindly calculated and put into the formula by the author himself in the form of constants.
  • Accuracy is higher than Mathf.Sin (float)

    0.8414709 56802368 - sin (1) (GPU, hlsl language);

Linear interpolation


This method is based on linear interpolation between the results of two records in an array.
The records are divided into mem_sin and mem_cos, they contain the pre-calculated results of the standard function Math.Sin and Math.Cos on a segment of input parameters from 0 to 0.25 * PI.

The principle of manipulation with an angle from 0 to 45 degrees does not differ from the improved version of the Taylor series, but it calls a function that finds ‚Äî between which two records there is an angle ‚Äî and finds a value between them.

Pros:

  • High speed (65%)
    Due to the simplicity of the interpolation algorithm, the speed reaches 65% of the speed of Math.Sin. I consider speed> 33% satisfactory.
  • Highest accuracy
    Example of a rare deviation case:
    0.255835595715180 - Math.Sin;

  • Fast leg

Minuses:

  • Requires memory space
