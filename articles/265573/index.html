Visual tab switching

With a large number of open tabs, it is sometimes difficult to understand which of them the browser switches to when using the Ctrl + Tab key combination. This is especially felt if the tabs are switched in the order of use, as it is set by default in the browser settings. To avoid this problem, we added the ability to visualize the switching process.

When you turn on this feature, you will see a list of thumbnails of open pages in the order in which they will be switched:


You can enable this feature in the browser settings, in the ‚ÄúTabs‚Äù section, ‚ÄúCycle tabs visually‚Äù. This is so far only a very initial stage of implementation, so we are waiting for your feedback and suggestions for further improvement of this function. This is especially true of the process of rendering page thumbnails - this is not always the case correctly. Please inform us about cases when such a bug can be fairly confidently reproduced.

Improved Cookies Manager

In previous builds, you could see in the settings a very simple Cookies Manager, which only allowed you to view their total number and delete everything at once. Now we have changed this useful tool, giving you the ability to view and delete Cookies individually, even if they are installed several from one website.


This improvement will be very useful for those who prefer to have the most complete control over the viewed content, not allowing site owners to abuse your trust.

Known Issues

  • Localization does not work - the interface is only in English
  • In this version, auto-update is disabled so that users who prefer their native interface language do not receive unpleasant surprises.
  • Tabs opened in the background cannot be switched using Ctrl + Tab
