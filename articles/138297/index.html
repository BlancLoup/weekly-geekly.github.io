Determining whether an IP address is in range

Introduction

After reading the article " Determining a country by IP: testing the speed of algorithms, " I would like to add another interesting way of finding an entry in the range of IP addresses.

The method consists in representing the range as a segment with ends [(-1, ip_range_begin), (1, ip_range_end)] and looking for a point (0, ip_address) on the given segment.

DB structure

Test table structure for storing IP ranges: 

CREATE TABLE `country_ip` ( `ID` int(11) NOT NULL AUTO_INCREMENT, `IPv4_beg` bigint(11) NOT NULL, `IPv4_end` bigint(11) NOT NULL, `IPv4_GeoRange` linestring NOT NULL, `Country_ID` SMALLINT(4) NOT NULL, PRIMARY KEY (`ID`), SPATIAL KEY `sk_range` (`IPv4_GeoRange`), INDEX `IPv4_beg` (`IPv4_beg`), INDEX `IPv4_end` (`IPv4_end`) ) ENGINE=MyISAM DEFAULT CHARSET=utf8 PACK_KEYS=0


Where IPv4_beg is the beginning of the range, IPv4_end is the end of the range, IPv4_GeoRange is a segment and Country_ID is a country identifier.

On the IPv4_GeoRange column, you need to create a spatial index SPATIAL KEY `sk_range` (` IPv4_GeoRange`) . This feature is available only for MyISAM.

The test is divided into two types of test:
  1. Number of requests within 10 seconds with the same IP address.
  2. Time spent on executing 1000 requests with random IP addresses.


In all tests, the cache is disabled (SQL_NO_CACHE).
