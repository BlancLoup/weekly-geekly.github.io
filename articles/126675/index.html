Grammar checker

Most text editing systems have a tool to automatically check spelling errors (when one or several letters are written incorrectly in a word; in English, speller ). Their principle of operation: the program analyzes each word in the text and searches for the same in the Database of all words and their various forms.

This text verification ensures that the words in the text will be spelled correctly (as in a dictionary), but does not protect against consistency errors and syntax errors in the sentence. For example, the sentence ‚ÄúI read an interesting magazine‚Äù is wrong, but the text editing system will not show the correct version: ‚ÄúI read an interesting magazine.‚Äù
The grammar checker in grammar checker helps to avoid such mistakes.

Created a lot of programs that check spelling errors. But so far (as far as I know) there are no programs that check grammatical errors in the sentence (for flexive / inflectional languages ‚Äã‚Äã- in which words have many forms); that is, checking for errors is not only spelling, but also syntax and syntax errors. Therefore, I would like to present a program that does it (as far as it turns out).



In 2003-2007, I wrote a program (three versions of it) that as quickly as possible analyzed the text (in Lithuanian) and issued corrections not only for spelling errors, but also corrected grammatical errors and communication errors in the sentence.
In January 2011, through Wikipedia, I came across habrahabr.ru, and then rg_software articles ‚ÄúNotes on NLP‚Äù and ‚ÄúNLP: spell checking - look inside‚Äù and met the phrase ‚Äúhow to create a pattern for the rule‚Äú the predicate should the same genus as the subject? ‚ÄùI thought that it was worth writing about my work - maybe it will be interesting for specialists and not indifferent.

The program works on the principle of a parser for programming languages: a large ‚Äúuniversal pattern / pattern of a sentence‚Äù is used, which describes all possible variants of matching the words in a sentence; all possible suggestions.
The parser is slightly modified for a human language (for more details, see the pdf-e description); In addition, in order for the program to offer a more successful (in her opinion) version of the input sentence - for this you need to consider all the forms of each word from the input sentence. For example, for the sentence ‚ÄúThis computer program recognizes simple text‚Äù it is supposed to consider 13.934.592 combinations of variants (24 * 24 * 12 * 7 * 24 * 12); This is done in about 5 seconds.



In addition to correcting grammar program:
* Supports the multivariance of words and phrases at all levels of the syntax tree of the sentence.
* Shows graphically the tree of the selected (revised) version of a sentence or a recognized part of it; The ‚Äútrees‚Äù of the universal sentence pattern, the ‚Äúwords‚Äù of the input sentence and their variants are also shown.
