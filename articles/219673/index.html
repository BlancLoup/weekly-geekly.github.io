Spreading the open innovation business model using the API

The API "keeps secrets" best


At the API days conference last December, @CyrilVart (@Fabernovel) gave an exhaustive presentation of "6 reasons why the API will change the look of your business" ("6 reasons why APIs are reshaping your business"). Let's take a closer look at it and, combining the information received with our own experience, we will evaluate what this new exciting area has to offer.

An API is a protocol that is used by software components as an interface to interact with each other. In other words, the API is a connecting element between functional components that connect with each other like blocks of the Lego constructor.

API still seems to you something wild and distant?



Think again, some APIs are pretty well known, we just don't call them that.

  • The Google API (including the Google Maps interface, a supporting landmark on a map of many web sites, or the Google Adsense interface, which allows advertisers to automatically upload text, images, videos, and multimedia advertising to Google‚Äôs web sites) requests 5 billion per day.



API for creating collective innovation


API promotes innovation. We placed the API at the center of a model of rapid innovation and modular design (Modular Design and Rapid Innovation model). It brought a number of useful results:
  • Acceptance: API-modules facilitate the adoption of innovations, transferring control to other participants in the process, allowing them to feel their involvement and set their own parameters based on your platform.
  • Collective creativity: the API does not impose anything on anyone, but on the contrary, it induces to create, this is the so-called ‚Äúset of design tools‚Äù, the practical embodiment of the principle ‚ÄúCreate with someone, not for someone.‚Äù
  • Flexibility: Due to the fact that these applications are created for the end user and the API components are scattered, the API development team is given complete freedom of action.
  • Speed: API forces innovation, allowing parallel code execution. The component can be used as a center that receives parallel connections from different applications.
  • Multifunctional development: the requests of various applications cooperate and expand the functions of the component within the mechanism of self-improvement of innovations. The progress of each participant benefits everyone.

APIs in common business models


In addition to stimulating innovation, Fabernovel mentions five key results of using the API in business:
  • New sources of income
  • New distribution channels
  • Expansion of partnerships
  • Fine tuning control (API does not open access to everything, but allows you to control a lot)
  • Organization flexibility (using internal APIs allows you to review the role of IT and speed up teamwork)

All these effective stimulants enable APIs to expand your business: an API is more than just a collection of components.



I picked up several iconic companies that managed to use the full potential of the API:
  • Expedia has opened the API for its partners so that they have access to orders, photos, search results, customer reviews: today Expedia has 10,000 companies and has annual revenues of $ 2 billion, 90% of which are derived from the API.
  • Based on an API that provides a range of CRM services in the cloud, Salesforce has built a powerful business model that is not tied to a physical point of sale and software installation in the field.
  • Netflix opened the API in 2008 to allow developers to use company resources (movie catalog, queue management, rental history) and create a creative business: today more than 800 devices can play Netfix content and about 20,000 developers use their API.
  • Facebook Connect allows users to log in to many other sites using their Facebook profile. Third-party developers can add the ‚ÄúLogin with Facebook‚Äù button using the Facebook Connect API. The Facebook Connect interface has standard registration data, with the help of which a universal ID is generated for the digital world and, moreover, it allows Facebook to build partnerships on a global scale, with a minimum of resources.
  • Amazon Web services offer a wide range of services (storage services, databases, computing facilities, servers, application services, deployment and management) available through a set of APIs. There is a fee for working with each of these services, which means that Amazon very closely follows the experience of using the service by each client. In 2011, the company earned $ 750 million.
  • Comcast, America‚Äôs largest cable television provider, has created a dozen internal APIs for teams to easily share data and solutions. To share resources through the internal API, you need only 30 minutes, and not a month, as it was before.
  • AT & T is now offering 79 APIs to third-party developers (Future Center Blog), and Orange has redesigned its sister site, Orange Partner , to show its API.
