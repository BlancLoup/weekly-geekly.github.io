After almost three weeks of drawing the layouts, integration and enhanced testing (yes, everyone was resting, and we were working!) A new version of open source application for automatic site acceleration was released - WEBO Site SpeedUp 0.9.5 ( more ). The current status is a public beta (i.e., most likely, the version contains some non-critical errors, but is completely ready for ‚Äúcombat‚Äù use). Under the cut more details and pictures


Some of the innovations have already been announced in the post about the alpha version , this is:
  • As you have already noticed, the product name has been changed to WEBO Site SpeedUp, so as not to conflict with Google Web (site) Optimizer. We have no competition with Google :)
  • Completely new interface, which has already collected a lot of positive feedback. Screenshots are also available here and here .
  • From the most delicious: it is now possible to quickly change sets of settings. By default, 3 sets are available (of course, you can create your own): safe (eliminates all potentially ‚Äúdangerous‚Äù settings), optimal (provides excellent acceleration, but can sometimes ‚Äúkill‚Äù the system) and extreme (well, the acceleration is maximum, but and the risk is very large, you need to customize).
  • The concept of the control panel has also been revised, and the most important parts of the application ‚Äúin three sentences‚Äù have been placed on it. Blocks can be hidden / revealed.
  • Added a separate page for cache files. The concept will be further refined and revised to make the interface more convenient.
  • Utilities for recursive gzip file compression and image optimization have been added (the latter only through smush, it, but punypng is on the way).
  • Added a separate page with information about the current system environment and its settings.
  • The choice of interface language has been added (if you are ready to translate the product into your native language other than Russian / Ukrainian, please write to us ).

+ some more new features
  • WEBO Site SpeedUp has its own API for optimization (in the new version, onBeforeOptimization, onAfterOptimization, onCache methods have been added to it). It can be used as a regular action, and one-time (for example, hanging indexes on the base when installing the application).
  • Now you can upgrade not only to the latest stable version, but also to an intermediate beta. In the process of adding the mechanism of "rollback" to a stable version, if the latest beta version is installed.
  • Improved work with .htaccess and detection of Apache modules on a large number of different environments.
  • Also one of the most anticipated improvements: the lang = ‚Äúwo‚Äù label in the page header is removed ( <!--WSS--> inserted in the end of the page instead).
  • Added the ability to ‚ÄúUse a single cache for all browsers‚Äù (it can help when using external caching mechanisms, but makes it impossible to use the cross-browser data technology: URI + mhtml).
  • Added the ability to cache static files located on external resources (either scripts or images for which normal caching headers are not exposed).
  • Added ‚Äúdelayed‚Äù loading for frames (so that they do not interfere with the main content of the page).
  • For native plug-ins (WordPress / Joomla!), Server caching has been improved (we will carry out additional performance tests before release 1.0).
  • Added the ability to exclude pages from server caching in the presence of certain cookies.
  • Added support for PrestaShop (1.2.5).

Licensing terms


WEBO Site SpeedUp (as well as Web Optimizer since version 0.6.0) has 3 different editions ( version comparison ):

  • Lightweight. Can be used on any sites. It has almost the same functionality as the non-commercial version.
  • Complete All that is in the product, and even more - 24/7 technical support.

Updates are free in all editions. All paid licenses have a limited duration (year). There is an opportunity to purchase an unlimited license ( please email us if this is your case ).
