Testing the collaboration of Windows 2012 R2, Traffic Inspector and Kaspersky Endpoint Security


And finally, a compatibility testing case conducted by Smart-Soft tech support.

For the purpose of the test, Windows 2012 R2, Traffic Inspector 3.0.1, Kaspersky Endpoint Security products were installed and traffic was run throughout the day using the HTTP Test tool. No problems were detected during the experiment.

Kaspersky Endpoint Security was tested on virtual machines using desktop (Windows 7, 8.1) and server (Windows 2012 R2) operating systems. Traffic Inspector 3.0.1 was installed on the server operating system, configured in gateway mode, configured clients, created traffic control rules. Client machines were configured to work through a proxy.

Kaspersky Endpoint Security for Business has been selected as antivirus protection. Before starting the installation, we downloaded the administration server distributions from the Kaspersky website: Kaspersky Endpoint Security for Windows for client machines and the Kaspersky Security Center management tool. Kaspersky Endpoint Security is installed with optimal default settings, so systems and data are protected immediately after its installation.

Subsequent configuration and protection management was carried out using a single console of Kaspersky Security Center, which simplifies the provision of control and protection of all workplaces. It is especially necessary to note the presence of the protection functionality of corporate mobile devices (smartphones, tablets). Using a single control center allows rational use of resources and contributes to greater software resilience to various types of threats.

Antivirus scanning technologies work at different levels of the operating system, effectively removing malware. The cloud network Kaspersky Security Network protects users from new threats in real time. At the same time, flexible monitoring tools allow you to ensure that only authorized devices are connected to a PC on the network.

Installing Kaspersky Endpoint Security did not affect the operation of TI and client machines. The program activity monitoring system did not block the TI module.

Load testing from client machines was performed using the utilities ¬´HTTP-Test¬ª, HttpTrafficGen.

We also measured the speed of the Internet connection before installing Kaspersky Endpoint Security and after. The measurement was performed using the speedtest.net speed testing site . We also used FTP servers located on the Internet, with obviously large distribution channels, for example:

To test the traffic between the client and the TI server, the iperf utility was used , which runs under all systems - free with open source and does not require installation under Windows. The program is quite accurate: the results coincide with each other in several dimensions. We ran it on the server from the TI and the client machine on both sides of the channel under test (on the one hand, in server mode, on the other, in client mode). The speed (channel bandwidth) is tested only in one direction - from the client to the server, that is, the client transmits data, and the server accepts. To test the channel bandwidth in the opposite direction, the server and the client are swapped at the ends of the channel. iperf can generate ‚Äúnowhere‚Äù traffic, for example, to a non-existent address. With its help, the channel or network interface was loaded to the required value.
