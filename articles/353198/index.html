SelectelTechDay: how it was


On April 5, the third SelectelTechDay conference was held, which brought together about 200 representatives of organizations from various fields of activity. The atmosphere of friendly communication, interesting reports and interactive demonstration of the capabilities of the new Selectel VMware Cloud service are just some of the events in which the guests of the event took part.

Therefore, traditionally, we organized tours of the server rooms and talked in detail about how everything works on the example of the Data Center Flower-2 .

Tatiana Svirko, the head of the engineering and technical department, told us about the parameters monitored by the specialists on duty at the engineering office, from where day-to-day monitoring of all data center systems is maintained.

New Selectel VMware Cloud service



On the day of the conference, a significant event took place - a new service, Selectel VMware Cloud, was introduced into commercial operation, a cloud solution that allows you to create a private and hybrid cloud based on VMware software and Intel hardware platform. The service is out of testing and made available to customers. Now you can order a virtual data center with payment for the actual consumption of resources.

The new service is aimed at customers who are already using VMware virtualization in their IT infrastructure, and is positioned to use according to 4 possible scenarios:

  • expansion of own data center to the cloud;
  • making work applications to the cloud;
  • cloud as a backup data center;
  • cloud for application development and testing.

All these scenarios are paid by Pay-as-you-go with hourly billing. The main focus was on enabling customers to build their own hybrid cloud based on the new service. This allows you to habitually use the existing infrastructure, if necessary, using the resources of Selectel for rapid scaling or redundancy.

Architecture


Cloud Selectel VMware Cloud deployed on the basis of data centers Dubrovka-3 (primary data center) and Flower-2 (backup data center). The decision contains the so-called arbitration servers installed in the TsDorochnaya -1 data center (primary arbiter) and Berzarina-1 (reserve arbitrator). Thus, arbitration servers constantly monitor the availability of the primary and backup data centers, initiating the process of restarting virtual machines if any of them is completely unavailable.


Alexander Krenev, development manager for network virtualization in Russia and the CIS, from VMware, in his report noted that currently it is the hyperconvergent architecture implemented in Selectel VMware Cloud that is most in demand on the market, allowing you to manage everything from one point and at once increase capacity when necessary.

He also told how to perform migration between the clouds and about the capabilities of the software network platform NSX and vSAN technology, which allows you to create ‚Äústretched‚Äù clusters for data storage.

Implementation


To ensure the high performance of the infrastructure, Intel solutions optimized for use in hyperconvergent architecture systems were used as a hardware platform:

  • Intel Xeon Gold 6140 is used as a processor;
  • the drives use a combination of Intel P4600 NVME disks (for cache) and Intel P3520 (for data).


The network part is implemented by hardware based on Brocade equipment, and a powerful VMware NSX tool is used as a software solution for network virtualization within virtual data centers. Its main advantage is the flexibility of configuration and ease of management of complex network configurations.
