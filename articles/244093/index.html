System hierarchy:




Image from www.teleparadigm.com

Main modules:


  • Accessibility monitoring
    Checks the availability of items and services, such as HTTP or DNS, for example.
  • Performance Management
    Tracks the quantitative indicators of any resources - interface utilization, processor load, element temperature.
  • Inventory (Resource / Inventory Management)
    Responsible for accounting for physical and logical network resources.
  • Trouble Ticketing
    Allows each problem to formalize, assign a responsible, determine the steps of the solution.
  • Fault Management
    It is a system for monitoring and managing alarms, which is designed to filter and correlate them in order to identify the root cause that generated the flow of interrelated alarms;
  • Order Fulfilment / Order Management
    A system that allows you to organize "orchestration" of orders, break up the whole process into a series of subprocesses, automate some of them.
  • Work force management
    This module formalizes all processes in the company. Now any event or action will have an owner and a specific set of states. For example, the replacement procedure for faulty equipment.
  • Fraud Management Systems (Fraud Management)
    Designed to prevent and prevent cases of unauthorized and unpaid use of services of telecom operators;
  • Service Quality Management (SLA Management)
    Provides online monitoring of services available to internal and external users;
  • Service Planning and Development Module
    Allows you to predict the development of events and simulate a variety of scenarios;
  • Security Management
    Provides access control to network resources;
  • Accounting Management Module
    Registers the use of various network resources.
