X-series high density solution


Huawei redesigned its high-density server line by adding the X6800 V3 model instead of the X6000 model. Thus, currently only 2 models are available:
1. X6800 - 4U, 8 nodes
2. X8000 - 44U, 80 nodes

I‚Äôll dwell on the new X6800 solution.



There are 3 types of nodes available to the user:
1. XH628 V3 Storage Node - double node high disk capacity. It has a hot-swap basket that holds 12 * 3.5 "disks + 2 * 2.5" disks installed separately from the basket;
2. XH622 V3 HPC (also known as Application Acceleration) Node is a double node for HPC, allowing installation of 2 full-size dual-slot PCIe (for example NVIDIA GPU or Xeon-Phi). Reduced floating-point performance with 4 nodes with GPGPU - 12 TFLOPS ;
3. XH620 V3 Compute Node is a single high performance node. Due to the increased density of the layout, the solution from 8 nodes has 16 E5-2600 V3 processors and 128 DDR4 slots in 4U.

It is worth noting that only single-type configurations are now available. Those. configuration with mixed node type is not available.

This solution will be particularly interesting in projects to build an HPC cluster or public clouds.

E-Series Blade Solution


Well, not to mention the blade solution Huawei E9000.

There is support for 16G FC, 10GE copper, FCoE, 56G FDR IB and 40 GE switches. But the converged switch 10GE + 8G FC remains the most popular one.

Fusioncube


And finally, I left a completely exotic and unknown solution.
Keeping up with the times, Huawei decided to keep up with their competitors and released their own hyper-convergent solution FusionCube.

As a hardware basis, X6800 or E9000 servers can be used. The virtualization environment is the Huawei FusionCompute own hypervisor (based on Xen). FusionStorage is used to virtualize disk capacity, and FusionManager is used as a management tool.

In the FusionCube X6800 configuration, scaling will occur at the chassis level, and in the FusionCube E9000 - at the blade level.

The success and relevance of this solution in our market is very ambiguous.
