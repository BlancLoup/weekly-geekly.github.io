Problems with fieldset in FF

Today I already stumbled upon two bugs rendering the fieldset in Firefox.

For fieldset, min-height doesn't work .

Bug five years ago. The fix appeared only five days ago. When it is included in the next build it is unclear how I understand for the current branch there can only be updates on security, not layout. (

The bug is fixed by setting min-height for the element inside the fieldset.


Absolutely positioned elements inside the fieldset with padding and relative do not count their coordinates from the content, not from the upper left corner of the block .

Those. move down and right by padding-top and padding-left.

Zasabmitil, the decision came up with the following:

We set styles only for for FF:
1. Problem padding is removed from the fieldset.
2. Set the appropriate margin for the internal elements.

Important: we cannot simply compensate the position on the left or top for an absolutely positioned element, because when this glitch in FF fixes the site will be displayed incorrectly. Therefore, we change the styles for the remaining elements so that they do not depend on this bug.
