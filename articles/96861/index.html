Update Kaspersky Rescue Disk

Perhaps everyone knows that with active infection, and especially file viruses, the most versatile and effective tool is to use the LiveCD from anti-virus vendors. Usually, this is a Linux based bootcCD with installed antivirus and databases, as well as a system for recognizing network settings and hardware. This allows you to connect to the Internet after the download and update the database while being in the LiveCD environment.

Unfortunately, there are cases when the infected machine is not connected to the Internet, or the connection settings are not recognized, require the installation of additional software, etc. Anti-virus companies update LiveCD images approximately every week, but taking into account the huge number of repack malware, it is often critical to have actual databases.

Now I want to talk about how to update the databases of Kaspersky Rescue Disk and re-create its image so that the actual information is recorded on the disk.

All actions must be performed on a deliberately uninfected machine connected to the Internet .

1. Download the Kaspersky Rescue CD image from here .

2. Download this utility here ( mirror ).

3. Unzip the utility archive into some empty folder. There also place the disk image downloaded in step 1.

4. Run diskupdt.cmd from the folder and wait for the utility to run.

5. The updated disk will be located in the folder with the addition of ‚Äúnew‚Äù in the file name. This image must be written to an empty blank disc, and then booted from it and perform a scan.

This utility works as Kaspersky Rescue CD version 10 (current at the moment), and with the old, eighth version. It is strongly recommended to use the current version, but if for some fantastic reason you don‚Äôt like it, you can update version 8 disk. The script distinguishes them by name: kav_rescue_10.iso and kav_rescue_8.iso , therefore, it is strongly not recommended to rename the downloaded original files . You can re-update previously updated images, renaming them to the above names, but it is extremely important not to confuse the numbers and not to name, for example, the eighth version of the tenth. In the opposite case, the drive will be broken, although the upgrade procedure may be successful.
