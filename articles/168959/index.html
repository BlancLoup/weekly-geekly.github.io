Bootstrap 2.3 released

It took a long time, friends. It has been about three months since the last update, but do not worry, today the wait has come to an end.
After numerous delays, including the fight against the flu, we are pleased to present you Bootstrap 2.3 .

And what's new?


Bootstrap 2.3 contains several innovations, as well as fixes and documentation improvements.
The most significant of them are:

Repository changes:
  • Local dependencies instead of global for builder and installer. Now for a quick start the easiest way is to run npm install
  • Upgrade to jQuery 1.9. No changes are required on your part, we have included the latest version of jQuery in our release.
  • Changelog moved to repository (was a separate wiki page)


New features and improvements:
  • Added indicator for carousel ( link ) Just add HTML and it will work :).
  • Tooltips added to tooltips. By default, it is still equal to insertAfter , but now you have the opportunity to specify where to insert tooltips (or popovers).
  • Poppers now use max-width instead of width , they have also become a bit wider (280px vs 240) and automatically hide the empty header with css :empty .
  • Improved smoothing in tooltips ( # 6713 )
  • Improved accessibility for links in all components. After merging with # 6441 state :hover merged with :focus . This applies to both normal links and buttons, drop-down lists, navigation bars, etc.
  • Added auxiliary CSS classes to hide content during printing.
  • The code for the input groups has been updated, now they are more close to the usual kommponentami forms. Added display: inline-block; Increased margin-bottom and added vertical-align: middle; matching styles <input>
  • Added a three-color gradient .horizontal-three-colors() with an example in CSS tests.
  • Added helper classes for text alignment .text-left , .text-center , .text-right
  • Added @ms-viewport , so IE10 can now use adaptive CSS for multi-screen mode.


Documentation changes:
  • Added a new example for navigation, justified
  • Added an example of a sticky footer with a fixed navigation bar.


As usual, you can see a more complete list of changes in the rubicon or in pull queries . In most cases, minor CSS corrections and errata corrections in the documentation are not reflected there.
