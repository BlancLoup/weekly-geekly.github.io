The task was to transfer the FF itself and the profile to a virtual disk, for the fastest possible work.

I will describe the home version as the most sophisticated.
By car (win 7) there is a virtual disk from DATARAM, 1 GB in size - just for such purposes - to speed up and partially encrypt.
The image of the virtual disk lies on the crypto (true type) disk.
Once a day, the image of a virtual disk is dropped into a backup. Also, the virtual disk is saved when you turn off the computer.

The very first option, which was made a year ago, is simply to transfer the folder with the profile and the folder with the firefox itself to the virtual disk, leaving ntfs links on it. The solution in 5 seconds, but not attracted by the fact that when working with files (for example, a cache with a bunch of small pictures), it will still constantly jerk the hard drive from which the path begins.
Well, constantly remember about the links - not good either.

Therefore, we make the most complete transfer:


Transfer of the executable part:
When installing new Firefox, everything is extremely easy - we immediately put it on a virtual disk.
When transferring - copy the folder " x: \ Program Files (x86) \ Mozilla Firefox \ " to our virtual disk, and just run firefox.exe from it. He will prompt us to become the default browser. We say "Yes" and rule the call labels where we have them (desktop, quick launch panel, etc.).
We pass through the registry and rule the way in several places. It seemed to me that most of the keys belong to outdated versions, and the most necessary thing was updated when we reassigned the default browser, but still I recommend checking the registry - you can search for the full " x: \ Program Files (x86) \ Mozilla Firefox " path.
