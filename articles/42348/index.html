Joomla and VirtueMart - a good solution for a small and medium online store

Our studio has completed the development of a new online gift store. To create a store site, we used a bunch of CMS Joomla and Virtuemart . I would like to dwell on the latter, since this product is deprived of the attention of the habrasoobshchestva, although many probably use it in development. So, what is good about VirtuMart, and what disadvantages can be noted in it:

1. This product is completely free, but it implements many functions necessary to create a full-fledged store. At least, I could not find any other free product with the same wide range of functionality.

2. Support for the Russian language is implemented very well.

3. This is an open source product with a fairly open structure, i.e. modifications, creation of themes, writing additional modules are implemented quite easily. Although there are some drawbacks, it seems that the old versions that have been transferred to VirtueMart when it worked only with Joomla version 1. In any case, not all standard Joomla 1.5 features for modifying the design templates are used here yet: the creation of the store theme is done separately from the Joomla template , and for some things you have to climb into the source code of the component.

4. Work speed. It is difficult, you can set and plus and minus. Of course, the script of an online store with a fairly large amount of goods (we now have about 600 positions) can no longer work out instantly. Therefore, we must not forget that the decent speed of the store on VirtuMart can be achieved only when caching is enabled.

5. SEO. In combination with the component sh404SEF for Joomla, our store works great. For all categories of goods and each product separately, CNCs are created with unique meta tags.
