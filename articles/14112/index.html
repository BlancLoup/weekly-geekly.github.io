GRDDL is now a W3C recommendation.

On September 11, the GRDDL infrastructure acquired the status of ‚ÄúW3C Recommendation‚Äù.

This is a bridge between existing XML and RDF dialects. Using GRDDL shifts the point of application of effort from the formation of RDF to the creation of algorithms for converting existing data into RDF.

Documents can be included in the GRDDL infrastructure in different ways. In general, for XML, you need to declare the namespace `grddl` and specify in the attribute of the root element the resource identifier of the corresponding conversion from XML to RDF (for example, the XSLT file). The mechanism also allows you to specify transformations for entire classes of XML documents (meaning classes by profile or namespace).
