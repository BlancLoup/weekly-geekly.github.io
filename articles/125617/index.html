Fuzzy search on the client and Soundex

Soundex is one of the algorithms for comparing two strings according to their sound. It was developed just under 100 years ago by Robert Russell and Margaret Odell. It is actively used in the USA when dictating surnames.

I have long been interested in using this algorithm and found a place for him to filter data on the client, or rather, to search for a hotel by name in the Ostrovok.ru project.

Task

On the Island, all found hotels are transmitted to the client and all filtering and sorting of the issue takes place in the browser. It was necessary to add a filter on the name of the hotel.


Solution and problems

I quickly discovered that by a simple coincidence, using indexOf, finding a Radisson hotel (Radisson? Redison? Redison?) Would be problematic. It was decided to add Daitch-Mokotoff Soundex (improved Soundex algorithm) to the exact coincidence, since this algorithm will leave only the most important: 

"Radisson" == "Redison" == "9346"


The first problem was the Russian language, since this algorithm is designed for the Latin alphabet. After a small search, a PHP script was found from the provocateur user, who previously translated Russian words into translit.

This solution was rewritten in JavaScript with a slight refinement of the rules. For example, I instead of JA became YA, since J is considered an essential letter for an algorithm. A crutch was also added for the words hotel, hotel, hotel.
