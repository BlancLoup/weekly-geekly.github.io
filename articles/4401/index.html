The official launch of the Live Search search engine has taken place.

Microsoft has announced the official launch of a new generation of search engine Live Search . It was previously known as MSN Search and has been open beta since March 2006.

Compared with the beta version in the official release, there are not so many improvements. At least, the developers do not say anything at all about improving search relevance, although the quality of search results, according to independent experts, has improved.

The main changes in the official release are related to the user interface. Now the search results show related requests (‚ÄúRelated Search‚Äù button on the right), like on the Ask.com search engine.

When searching for images, a larger number of reduced copies of images is shown, and their size changes in real time mode (there is a slider on top). On the right side of the screen, a special ‚ÄúNotebook‚Äù (Scratchpad) opens where you can drag any images from the search results to the mouse. They are stored there and form the user's personal collection. The idea is the same as in the Google Notebook app , but it is implemented more competently, without the need to install a plug-in.

By the official launch of the search engine, Microsoft has expanded its collection of satellite photos. The Local Search system has updated or added images of 25 American cities, so now satellite images cover the territory where about 30% of the American population lives.

In addition to all of the above, in the search for images implemented an endless scrolling, as in the usual search on the web. With this output, the search results are not paginated, but are gradually loaded as the user presses the PgDn button. Links to additional projects appeared on the main page of the search engine. including the Live Search Macros service to create your own personal search engine.

The main disadvantage of Live Search is the continuation of its own advantages. This is a very slow job, which is apparently explained by the excessive complexity of the elements of the graphical interface. The speed of displaying search results on the Live Search system is several times lower than that of Google or Yahoo. It is very likely that the problem is not only in the complexity of the graphical interface, but also in the speed of the servers themselves. This can be a significant drawback with a serious increase in server load.

In addition, the Live Search results still contain a lot of spam and irrelevant documents, but this problem is gradually being solved.
