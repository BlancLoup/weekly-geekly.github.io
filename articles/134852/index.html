Failure at DCU MSU

On the night of December 16-17, 2011, we had a serious failure of one of our repositories. As a result, about 100 virtual servers were affected, some of which were cloud-based.

Servers for which the backup and administration service was ordered were restored as soon as possible in accordance with the SLA ( http://www.parking.ru/documents/ ). Work is now underway to restore the storage and try to restore servers to which we do not have administrative access, among them were mainly Linux servers without integration components, which did not allow them to be backed up using the System Center Data Protection Manager.

ps If you have backup data, then we can restore your machine in the shortest possible time, without waiting for verification and restoration of the entire repository. For this you need to contact our technical support service info@parking.ru

UPD. The storage operation has been restored, but it is advisable to check your servers.
