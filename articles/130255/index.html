Version 1.5.5 released

Python 2.7, increase limits, Cloud SQL Preview, Google Storage for Developers, Premier Accounts

Major changes:
  • Premier Accounts. $ 500 (excluding other expenses) per month for improved support (relevant for serious projects).
  • Now you can (experimental) use Python 2.7 in applications that work with High Replication Datastore. (To activate, use runtime: python27 in app.yaml). New in Python 2.7
  • The maximum number of application files increased to 10,000.
  • Maximum upload file size increased to 32MB.
  • Frontend request execution time increased to 60 seconds.
  • Increased URLFetch limits (runtime up to 60 seconds, maximum POST request size up to 5MB).
  • Added support for Cross Group (XG) transactions for High Replication Datastore. Read more
  • Now you can (experimental) work with Google Storage for Developers from the App Engine. Read more
  • Prediction API has become part of the platform .
  • In the administrator console there is a schedule for monitoring paid instances.
  • Some API changes
  • Bugfixes
  • Announced test programs for Cloud SQL Preview, Full-text Search, Conversion API

Lists of changes:
  • Go
  • Java
  • Python
