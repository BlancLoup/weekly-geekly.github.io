GitHub changed tariffs

Now you need to pay for the number of users, not private repositories.


Service for hosting projects GitHub changed the pricing methods. The site provides unlimited space for creating public repositories of code and free projects. In each of the public projects can be an unlimited number of participants. These rules were before, these principles remain in force today.

GitHub is monetized by private repositories. In the past, organizations and private developers were asked to pay for the number of repositories. This caused criticism , forced to store old projects locally. Now on paid rates the number of private repositories is not limited in any way. In addition to the base rate, you need to pay for the size of the team.
