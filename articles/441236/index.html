Data collection


Those who want to see the results right away can skip this part.

As in the previous section , we use Python and the BeautifulSoap library to gather information about the article. With its help we will receive information about the time of publication and the title of the article. All other data have already been received from us, anyone can refer to the previous part.
The result was this function: 
def get_article_data(page_link): data_html = get_as_str(page_link) soup = BeautifulSoup(data_html, 'html.parser') str_log = "" # Get datetime t = soup.find("span", {"class": "post__time"}) if t is None: return str_log += t['data-time_published'] + ',' # Get title and link url = soup.find("meta", property="og:url") url_str = url["content"] if url else "-" title = soup.find("meta", property="og:title") title_str = title["content"] if title else "-" str_log += '{},"{}",'.format(url_str, title_str.encode('utf8')) # Extract other parameters (rating, etc) str_log += extract_as_csv(data_html) print(str_log) return str_log

The result of the function will be a string of the form '2007-06-12T12: 58Z, https: //habr.com/ru/en/post/35001/, "What is your attitude to plagiarism?", Votes: 0, votesplus: 0, votesmin: 0 , bookmarks: 0, views: 161, comments: 1 ', which is easy to save to a csv file.

Further, as they say, the matter of technology. Fortunately for us, all the articles on HabrÃ© are numbered through, so by typing it was easy to find the right date range (by the way, those who want to do â€œdigital archeologyâ€ can read the articles with the numbers https://habr.com/ru/post/1 / and https://habr.com/ru/post/2/ , published already in 2006). Then it remains only to run our parser in a loop, and leave it for the night (in order not to load the server, a pause was added between requests, so the process was not very fast).

There was a doubt about corporate blogs, but as it turned out, they have the same numbering, just when you open the link, a redirect occurs. So all the data was loaded completely, nothing was lost.

For the entire sample, we will make different requests, and see what happened. The technology and source code were described in the second part , those who wish can learn the technique in more detail.
