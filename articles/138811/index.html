unity3d as a development tool for three-dimensional flash applications



On Habr√© passed unnoticed event quite significant in the gamedev'a world - unity3d was updated to version 3.5 . Among other things, this release has the opportunity to compile projects under the Adobe Flash platform.
What does this give game developers and what opportunities does digital advertising offer?

About all of this to tell, at least briefly, will be completely superfluous. Moreover, you can still download the version with the exporter for flash absolutely free. With the exit from the ‚Äúdeveloper preview‚Äù, this feature will cost you some money.

Welcome to all the curious under habrakat!

It makes no sense to talk about the unity3d engine itself, there are enough articles on this topic on Habr√©. The release of version 3.5 was very welcome and brought a number of nice innovations.
By themselves, they are quite worthy of a separate story, I just briefly mention them:
  • new particle system shuriken: completely rewritten particles, with great potential;
  • own pathfinding, very fast and enjoyable;
  • light probes - pseudo light from the walls, greatly improves the picture and is fairly cheap in performance;
  • native support for rendering in HDR and gamma correction;
  • native LOD system;
  • Google Native Client support - among other things, making it possible to create projects under linux by launching them from google chrome

Well, the most interesting and interesting thing - an exporter appeared under Adobe Flash!



Flash, as a platform for browser games, the most popular in the world. No matter what anyone says, it is installed on the vast majority of client machines. Since this plugin is the most common, it is logical that it has the widest audience on the Internet.

In addition, existing popular playgrounds originally sharpened by flash. Such monsters as Kongregate and Armor Games, social networks - facebook, VKontakte, etc. They give a huge army of players and everyone has an Adobe player. Yes, here it can be argued that, they say, the same Kongregate places games on unity3d, and on social networks, products of this kind are no longer uncommon. But! Not everyone has a unity player and not everyone is in a hurry to install it for themselves.
A developer wants to have the widest possible audience.

Now about the technical side of things. Of course, as I mentioned, not everything is perfect.
I suggest to judge for yourself whether the list of features is sufficient. Here is a list from the developers site (in this place I thought for a long time and decided not to translate the terms into Russian, all one, many terms are untranslatable)

Supported by:
  • Lightmapping
  • Occlusion culling
  • Basic scripting
  • Editor scripting (JavaScript / C # / Boo). Note: for JavaScript, use #pragma strict.
  • Custom shaders
  • Animation / skinning
  • Basic audio features such as AudioSource / AudioListener
  • Physics
  • Navigation meshes
  • Baked substance textures
  • Playerprefs
  • UnityGUI, except for text input
  • Realtime shadows

Not supported:
  • Unity profiler
  • Asset bundles
  • Text input in UnityGUI
  • WWW classes. Note that you can write your own Adobe Networking APIs.
  • Raknet networking (if you need it, you can write it in Action Script 3 directly, using flash API)
  • Terrain
  • Cloth
  • Using VertexLit shaders in combination with:
    - Specular highlights
    - Spot lights
    - Emissive material color
  • Advanced audio features such as audio effects. Also pitch manipulation is not supported.
  • Deferred rendering
  • AnimationEvents that carry arguments

Partially supported:
  • Image Effects. Some work, some don't.
  • Not all parts of .NET scripting work (lambda expressions and LINQ are not supported, for example)
  • GUIText will have a dramatic impact on performance
  • Shuffken API
