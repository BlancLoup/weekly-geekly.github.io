Moving Google Chrome profile to another folder, nuances

It's no secret that for the Google Chrome browser, you can specify the profile folder manually using the command line. For example: 

"C:\Documents and Settings\bob\Local Settings\Application Data\Google\Chrome\Application\chrome.exe" --user-data-dir="S:\Profiles\bob"

It is useful (in my case, for example) when you need to store a profile on an encrypted partition (drive S :), which is connected as needed. This command can be used for a shortcut.

However, besides this, I use the following .reg file that I need to import into the registry. So that when you click on the links in the programs, Chrome is opened using the portable profile (and not the default profile): 

 Windows Registry Editor Version 5.00 [HKEY_CLASSES_ROOT\ChromeHTML\shell\open\command] @="\"C:\\Users\\me\\AppData\\Local\\Google\\Chrome\\Application\\chrome.exe\" --user-data-dir=\"S:\\Profiles\\bob\" -- \"%1\"" [HKEY_CLASSES_ROOT\http\shell\open\command] @="\"C:\\Users\\me\\AppData\\Local\\Google\\Chrome\\Application\\chrome.exe\" --user-data-dir=\"S:\\Profiles\\bob\" -- \"%1\"" [HKEY_CLASSES_ROOT\https\shell\open\command] @="\"C:\\Users\\me\\AppData\\Local\\Google\\Chrome\\Application\\chrome.exe\" --user-data-dir=\"S:\\Profiles\\bob\" -- \"%1\""

I advise you to save the contents of this .reg file, because When you update Google Chrome (in the case of major updates at a minimum), it knocks associations to the default profile in the registry.

Ps. It is assumed that Google Chrome is used as the default browser in the system.
