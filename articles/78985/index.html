Convenient work with tabs in VS2008

When working with a large number of files, a situation arises when all the tabs no longer fit into the space allocated to them from above. Then you either start scrolling the drop-down list with files in which for some reason the mouse wheel does not work, or search for the desired file in Solution Explorer. There are, of course, such assistants as ReSharper, but not at all. There is one undocumented trick that will help keep you nerves.

To do this, create a new DWord Value parameter in the registry:
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\VisualStudio\9.0]
"UseMRUDocOrdering"=dword:00000001
After that, the tab with the active document will automatically move to the left edge of the window. Thanks to this, all the tabs with which you work will always be at hand. Studio can not be restarted.
