What's new


  • Added full support for mhtml (at the same level as in DURIS ). Due to the optional separation of CSS files and loading on the DOMContentLoaded event DOMContentLoaded technology of ‚Äúsimple‚Äù CSS Sprites is now available to everyone. Moreover, recent research ( sirus will not lie) showed that Aptimize uses the perfect wrong approach in this area. This suggests that Web Optimizer in this case works much better than its competitor.
  • Added the ability to cache everything your heart desires through the use of a lightweight PHP proxy for static requests (since they are hard cached on the client, then with a large number of regular visitors there will be no additional load on the server). The technology is fully debugged and is used in cases where it is not possible to write the rules in .htaccess .
  • Significantly improved application performance. Now the processing of an HTML document for issuing to a user can take as little as 1 ms (the average time to create a page usually varies around 200-500 ms). For this purpose, three new settings have been added to balance the integrity check of the cache and the performance of the application.
  • Now Web Optimizer is available as a plugin for WordPress , Joomla !, a module for Drupal (6.x and 5.x) and a separate web application .
  • Processing of conditional comments is added and the cache for modern browsers and main versions of IE is divided. Now the download under IE has become even faster without affecting other users.
  • Significantly improved and stabilized checking the capabilities of the system before applying the rules for Apache / emulating them through the capabilities of Web Optimizer. Now, even for CGI environments, it is possible to correctly use Apache modules.
  • Improved compression for IE. In particular, it is now possible to force the use of deflate before gzip for IE6 and IE7.
  • Added the ability to merge files into body (can be resource intensive for large HTML documents).
  • The Web Optimizer logo displayed on the site can now be arbitrarily configured (it is possible to select a picture, styles for its location and to set text for a simple link).
  • The hotly discussed label ( lang="wo" ) in the title can now be turned off (but this is only available in the full version).
  • Added the ability to distribute pictures not only to the hosts of the main site, but also using a similar system of another site (i.e. pictures called from a host other than the current one, can also be distributed to speed up their loading).
  • Improved application behavior when it is not possible to use curl . In this case, dynamic and external files are simply excluded from processing.
  • For closed testing of the site, the ability to set the login / password for HTTP Basic Authorization in the configuration has been added (so that the processing of dynamic files takes place correctly).
  • Added several improvements to the logic of creating CSS Sprites. In particular, the processing of the background color has been improved and the size of the image has been slightly reduced when combining icons.
  • Improved the work of "unobtrusive" JavaScript. Several new counters and informers have been added, and now Google AdWords is calculating the height of the block, which makes it possible to exclude page ‚Äújumps‚Äù when loading ads.
