Create a resume on LaTeX - how and why?

I prepare many documents in LaTeX, not in Word. And by the time when I decide on the next place of work, I summarize what was done, and, so as not to be lost, I record them in the documents that make up the framework of my CV . In my personal conviction, carefulness in creating documents for a future employer is needed not so much for communication with Eychara, as for understanding the way you are going and the direction in which you decide to move on. So why did I choose LaTeX for my resume?

A small introduction to the topic for those who do not know what LaTeX is. This is a processor, which accepts a markup language as input, giving PDF with an output document at the output. The markup language tells the processor what is in the document what, where are the pictures and why, tables, pictures, links to them, table of contents, index, footnotes, etc. are automatically printed. The ideal tool for collaboration, for complex large documents, with graphs and formulas, which can make serious changes in the middle half an hour before printing, and you need time. Can also be rendered separately style, like CSS in HTML. The author focuses on the content, and when publishing, the document styles are applied where the author‚Äôs article is included. For these reasons, all the math and physics journals are only put into LaTeX.

It is convenient for me that I can include the source of the document under the version control system and make "branches" for various needs - that is, to establish joint work on the document in the same way as it is done on the program code. It is convenient for me that in order to replace the scheme in several documents, it is enough for me to simply export the new version to the same file name on disk and with the next PDF assembly a new version will be pulled. If it turns out to be 1 cm longer, then LaTeX most likely correctly turns everything up to the end of the document, replaces all links, does not allow dangling lines, etc.

Having corrected one line in the text, I ‚Äúcompile‚Äù an extended version of the summary or a shortened one. LaTeX is a great technology when:
  1. several documents have common blocks and I want to edit them in one place, and not in each document,
  2. I want to make a conditional assembly of the document - for example, do not include some set of paragraphs / sections for a specific version of the document, or include additional ones. At the same time be sure that all links, numbering, table of contents and so on. do not "break"
  3. a lot of people are working on their pieces and at the last minute you need to take into account their latest versions,
  4. the document is very complex (with footnotes, with cross-references, pictures, tables, formulas),
  5. there are external data that must be collected in PDF (for example, CSV),
  6. I do not want or do not need or do not have the time / energy / desire to think about the design, but I don‚Äôt want to give the ‚Äúsemi-finished product‚Äù either. LaTeX will make sure that the document looks stylish, and the author‚Äôs business is about content.
