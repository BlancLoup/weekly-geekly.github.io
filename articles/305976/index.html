Technical part

By the level of carbon dioxide, we can determine how many people are in the room and their activity. That is, CO2 data contains much more valuable information than just stuffy / fresh. For example, while on vacation, I can see from a mobile application that people have entered the apartment. At a minimum, this is a reason to figure out whether relatives came to water the flowers or the attackers entered. To measure the instrument are optical CO2 sensors. So far they are not ‚ÄúThionian‚Äù, but we are already working on our own.

‚ÄúThe physics here are simple and famous. The sensor has a light source, it shines at a certain wavelength. Carbon dioxide absorbs this light. There is also a receiver, which fixes everything. Accordingly, the more carbon dioxide, the less light. This signal is what determines everything. ‚Äù (c) Mikhail Amelkin



The installation process of the smart microclimate system is as follows:

1. Install compact ventilation: Tion O2 MAC breather (other models can also be used, but for this you need to replace the ‚Äúbrains‚Äù - this can be done by our customer service). Installation of the breather takes about an hour, for ready repair, without dirt and complicated work. The device forcibly supplies fresh air from the street, cleans and warms up - it will be possible not to open the windows, and also to forget about noise, dust and drafts in the apartment.

2. Install the MagicAir base station in a convenient place and connect it to any USB port or 220V network. We set up radio communication with the breather and set up Internet access via Wi-Fi. Sensors inside the base station will measure air quality and send data to the cloud server. The history of data is saved and the user can see the dynamics of humidity, CO2, temperature in the context of the rooms for the period on the smartphone screen. And this is not just a statistic with a beautiful schedule - based on the dynamics of indicators, you can rebuild the air conditioning system in the apartment and optimize the costs of maintenance of equipment and electricity.

3. We indicate in the mobile application (at the moment the application MagicAir is available in the AppStore and Google Play) the standard level of CO2 in the room. If this level is exceeded (for example, when people appear in the room), the base station will choose the optimal breather mode, and when the CO2 level decreases, the system will automatically reduce the ventilation power or turn off the breather altogether. This, by the way, is also energy efficient.

‚ÄúNow the base station controls only the breather, but by the end of the year we plan to expand the system so that we can connect devices from other manufacturers. To do this, we will release a separate gadget - which will communicate with the base station over the air and simulate the signals of an infrared remote control in order to control the climate technology. First of all, there will be air conditioners; it is also planned to add a valve to the system for radiators, exhaust fans, cleaners, etc. ‚Äù (c) Mikhail Amelkin

MagicAir system architecture



In the same room is the base station. It is tied to the user's personal profile on the cloud server. The base station processor decides what to do with the breather, air conditioning, and other actuators. Commands are determined based on user settings.

In the remaining rooms are located peripheral sensors. They communicate with the base station over the RF channel. The connection of the base station with Tion actuators is also over the RF channel through the built-in radio module. Communication with devices from third-party manufacturers - through the IR module, which can be purchased separately. The IR module itself is also a peripheral device that communicates with the base station through the same RF module. Communication backend and base station via Wi-Fi, communication backend and frontend through any communication channel through which the smartphone goes to the Internet: Wi-Fi, 3G, etc ...

A personal user profile is stored on the cloud server, all microclimate parameters, information about the system architecture and the state of all executive devices are collected. Here big data is analyzed, correlations are tracked for learning computational algorithms.

Control of actuators from a smartphone is carried out indirectly through the backend throughout the chain: front - internet - back - internet - Wi-Fi - RF.

This is how Tion's MagicAir system looks like. Use on health!
