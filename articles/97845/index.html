Multitasking in iOS4. Not so smooth

One of the "goodies" of Apple's mobile operating system, which users of iDevice have been waiting for, turned out to be a bit raw. It's about the multitasking function, which many lacked. Of course, the principle of operation of this function is implemented much better than in other mobile operating systems, but the end user is likely to find some inconvenience.


Inconvenience number 1
The first inconvenience (it was already mentioned on Habre) was that the special tray of multitasking, among others, included shortcuts of those applications that have not yet acquired the necessary "demon" of multitasking. This means that when you double-click on the Home button, you will see shortcuts for all applications that have ever been opened. The buffer of the launched programs will be constantly filled with necessary and unnecessary for work in multitasking mode.
My solution is inconvenience # 1
Of course, it is unrealistic and ugly, for such a function to add a separate hard button on the phone, which would completely close the application, and not send it to the buffer. But you can get by with a little blood if the applications get a display exit button. Although I don‚Äôt like this idea myself, it would somehow help in solving the problem of buffer overcrowding. Inconvenience number 2
Although the second inconvenience arises from the first, but sometimes there is a need to close all applications in the buffer. However, in the current firmware version, you can only manually close applications one by one.
My inconvenience solution number 2
You can in the same buffer make a red label with a large cross to close all applications at once. But this does not seem to be exactly Apple style. Therefore, it would not be bad to make it possible to close all applications at once by shaking the phone with the buffer open:


Inconvenience number 3
In part, this inconvenience is caused by the presence of the two previous ones, but since it was a question of shortcomings, it means that we should mention all of them. It lies in the fact that to manually close several applications in a row, sometimes you do not get on the left upper part of the shortcut where the close icon is located. Actually, why only the left corner reacts to pressing-closing? After all, the rest of the label in this case remains out of work ...
