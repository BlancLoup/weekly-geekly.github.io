New release of open hypervisor Xen 4.1

After 11 months of development, a new release of the open hypervisor Xen has been released.
Incomplete list of changes:

A new stack of tools XL, already almost similar in functionality to the old XM / Xend
Prototype of a new scheduler, better coping with delays sensitive tasks
Processor pools
Large system support (> 255 CPU)
X86 AVX support
New memory access API (simplifies the integration of security modules)
Greater stability declared with new regression tests

And yes, in portage - already there, ebuilds do not have to wait;)
