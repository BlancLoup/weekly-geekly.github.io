Tags for files on computer

It has long been troubled by the problem, which lies in the fact that on the screw a lot of all sorts of data, and everything seems to be necessary, and this business turns into a landfill, to find in which something is harder and harder. Now I have a screw for 300GB and I do not think that in the near future I will buy another place, because then the dump will simply become unreal.

Naturally there were attempts to somehow streamline what is - to collect all the pictures in one place, all the music in another, all the films in the third, and so on.

But this whole harmonious system breaks down when you realize that for example a music file belongs to several categories at the same time: it belongs to the OST of a movie, it is also electronic music, and a calm track. That is, if you approach the question, scattering everything into folders, then it should simultaneously be in folders:
data / mp3 / ost
data / mp3 / electronic
data / mp3 / calm
data / films / FilmTitle / ost

One comrade suggested that I should not bathe and just put the copies in the right place. This is terrible! At this rate, the dump will turn into complete chaos!

Overall, it would be logical to project the principles of content organization in our beloved web 2.0 to your everyday needs.

All you need is a program that is embedded in the shell and allows you to tag any file with tags, write information about them somewhere in your database, somehow conveniently and unobtrusively show it (for example, in an additional Explorer panel), organize tags into categories and Most importantly, search for files by them.

Maybe the miracle has already happened and such a program has been in existence for a long time?
