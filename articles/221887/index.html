ERP development on tryton: Part 2

This part covers the structure of naming methods, working with users and groups, installing additional modules.

Part 1: Install tryotnd, query structure, work with the database
Part 2: Method Structure, Working with Users and Groups, Installing Modules
Part 3: Work with modules of the company (company), contractors (party), product (product) and warehouse (stock).

0. The structure of the name of the method


We received the list of server methods, now we will understand how they are built and what they are responsible for.
The names of the methods are built quite simply and have the following form:
% primary_class_name%.% class_name%.% method_name%.

Here only the tryton developers redefine the names of the classes, basically so -% module_name%.% Class_name%, less often% module_name%.% Class_name%.% Logical_class%. For example: model.res.user.create and model.stock.shipment.in.create.

Total, method names have the following form:
% primary_class_name%.% module_name%.% class_name%.% logical_subclass%.% method_name%.

1. Main classes


Tryton has 5 main classes:
  • model - ORM class for communication with the database.
  • report - class for generating reports in different formats, for example odt or pdf.
  • wizard - a class that implements a finite state machine that describes a series of steps and states.
  • common - class for working with databases and anonymous methods.
  • system - class for working with system server calls.


The common and system methods were discussed in the previous section. About the report and wizard methods there will be separate articles, in this part we will consider model.

2.2. Module ir. Installation of additional modules

At the ir module of the functional above the roof, in this part we will consider only the functionality concerning the installation of additional modules.

To install extra. it is necessary to download the module, unpack it into trytond / modules and rename the unpacked folder into the module name. On the official site they offer to download from PyPi , after unpacking we get the folder trytond_% module_name% -% module_version% rename it leaving only% module_name%. Studying the source code, I came across a link from the repository of the tryton modules in one of setup.py, here it‚Äôs more convenient to download. When unpacking, we get the% module_name% -% module_revision% folder, again leaving only% module_name%.

Below is the API for working with modules. Although I would recommend installing the modules through the gtk client. Also in the API, there was a method for removing modules, but you cannot delete a module through the gtk client, and on the developers irc channel it is written that remote modules are a very complicated and non-trivial procedure. So use the removal of modules with great care.
