And once again about 320 million passwords


Troy Hunt promises that he will not use your password in any way and his service is absolutely reliable.
AlexanderS : News in a year: ‚ÄúThe base with 500 million unique passwords was published‚Äù

Jokes jokes, but what if you really check the password without passing it? You can perform a binary search using JS with server requests based on the record number.

320 million passwords fit into 29 requests, and if there is no password in the database, the server will not receive its hash in any way - all work is done on the client side. But 29 requests take me about three seconds. Therefore, you can immediately transfer to the page 2048 hashes that cover the first 11 steps of the search, and each subsequent request will give another 64 hashes, thereby covering six steps at once. Then to verify the password you will need only three requests.
