Opera 10.60 Alpha 1: Speed, eye-candy, and bug fixing

A new build of your favorite browser has been released with an impressive list of changes.

We are preparing for the release of version 10.60, and we are aiming to synchronize the speed of development of the distribution on all platforms. This alpha will give you an idea of ‚Äã‚Äãeven more performance, improving the interface (UI), and of course about the work on the bugs.


Performance !

Our developers do not sleep and again increased productivity. You will notice this in the process of working with JavaScript, DOM and CSS.

UI enhancement

In addition to the visual settings, you have already seen improvements in Speed-dial, but we are also working on the interface. Note the ‚ÄúMenu‚Äù text on the O button for your convenience. Also for thumbnails of all internal tabs added their own icons.


In Windows 7, the thumbnail view on the taskbar is now disabled by default, this was done because of the inconvenience of navigation with a large number of open tabs. But you can enable this in the settings (opera: config # UserPrefs | UseWindows7TaskbarThumbnails).

Carbon-neutral?

After the main work on the Opera 10.50 framework and replacing Carbon with Cocoa, we try to get rid of the Carbon code as much as possible. Some things are now loaded dynamically to improve performance.

Also for Mac users, remember that Java will only work if the latest updates for Mac OS 10.5 and 10.6 are installed. Java for Mac OS 10.4 is not supported.

Unix

For Unix users, this obviously means that there will be no version 10.5x. Nevertheless, we strive for synchronization and release 10.60 on all platforms, including Unix, will be simultaneous (mid-June).
