TurumPurum - musical soc. network


Dear Habravchane, I want to introduce you to my brainchild - musical soc. network called Turum-Purum .
Last week I already showed it, but it turned out that the assembly was still raw, there were a lot of bugs and I closed the article.
Thanks to those of you who tirelessly found mistakes, deficiencies and offered innovations, I finally made a completely stable version.


So, first about the positioning and purpose of the project. I wanted to create something like a habr, but with a musical theme, because sites of this type are not yet (in any case, I have not seen such).

Main functionality


Basically coincides with Habra. The main differences are the emphasis on the musical component of the project. Here, for example, what a member profile looks like:


And changing such a profile is very simple:




For a profile, a separate domain is allocated, so it is very easy to send a link to your profile.

I would like to note one innovation: the user rating system, in its construction, it differs sharply from the generally accepted ones, so what is the point:
the user has 3 characteristics: sound level (UZ), battery charge (BZ) and rating. Now more about each.

Sound Level


Roughly speaking, this is user level. This characteristic affects the user accessible site functionality. For example, after level 4, captcha ceases to appear when adding comments, and after 7 the user can change the rating of other users. The sound level is partially increased when performing various actions.

Battery charge


This characteristic is also responsible for the available functions, but from a different angle - this is something like mana when casting spells in a game. Those. Need to create a collective blog? - must be at least 50 units of charge. The battery charge is restored to a certain number every hour.

User rating


Characteristic showing the attitude of other users to this. When leaving the minus begins to adversely affect the Battery Charge.

In sum, all three characteristics make it very convenient to exist on the site.

Briefly about blogs


- Blogs are collective and personal.
- Each collective blog has an administrator and moderators appointed by the administrator.
- You can specify the scope of each moderator (topics, comments, settings, black list).
- In any blog, you can enter the system of invites - done with one click in the admin panel of the blog.

RSS feeds


- for any blog, taking into account the section (silent topics, loud topics, new topics);
- for the main page (loud, new);
- for the user's feed (blogs that the user has joined);

Rest


- built-in system encyclopedia, which can edit all project participants.
- integration with last.fm - output of the last listened tracks

Development plans


- make a more functional user part, with the ability to search for people of similar tastes.
