Nevertheless, the problem of third-party dependencies is there, and it needs to be solved somehow. And it costs equally in any ecosystem where third-party dependencies are used. Main features


The description is taken from the project site , we make a discount on the marketing bullshit, there are repetitions in the original. Personal impressions will be lower.

Search


  • Search for vulnerabilities in Javascript, Ruby, Python, Scala and Java
  • Checking GitHub Repositories
  • Checking open source packages before use
  • Uses its own database of vulnerabilities Snyk


Monitoring


  • Application dependency checking
  • Integration into CI processes
  • Warnings about new vulnerabilities in real time
  • Application Support on AWS Lambda and Heroku


Correction


  • Update or fix vulnerable dependencies
  • Automatic pull requests from Snyk in the gihub repository on Node.js and Ruby
  • Pull Requests with selected fixes
  • Interactive Assistant for quick fixes
  • Notification of new vulnerabilities that affect your projects
  • Notifications in Slack and by email about vulnerabilities and fixes
  • Understandable Uzizymosti

Prevention


  • Snyk can check pull requests that can make your applications on Node.js, Ruby, Python, Scala and Java vulnerable
  • You can use Snyk in CI for automated tests.
  • You can customize the desired security level for your needs and priorities.


Integration


  • Automatic monitoring of your github repositories
  • Adding Snyk to CI Processes
  • Flexible policies customizable to your team


Teamwork


  • Use Snyk Organizations for teamwork.
  • Administrator and Member Roles
  • Anyone on the team can search and fix vulnerabilities.
  • Only the right people will be notified of new vulnerabilities.


Training


  • Subscribe to the vulnerability base to learn about them in time.
  • Learn about how to exploit vulnerabilities and how to fix them.

In addition, you can test the package without installation, right on the site - and get a clear report like this . And, of course, you can get a beautiful badge like this:

Using


Once you have installed snyk, it just works. So far I have tried it only on Node.JS projects, and some unpleasant aftertaste remains. At a minimum, I noticed that in the absence or in the empty node_modules directory, the check considers that there are no dependencies, and this means that everything is fine in the project. The fact that you could carry them by hand, they could not be fully established, and so on - is not taken into account. But in general, the report turned out to be quite good - although in fact it all comes down to keeping the dependencies up to date. Which is very obvious and without a special tool. However, we all know that it is not always possible to ‚Äújust pick up and upgrade‚Äù to a new major version, and the presence of a critical vulnerability, which the monitoring tool warned about in time, can serve as a good incentive not to postpone it for a long tomorrow.
