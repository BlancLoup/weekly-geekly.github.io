How does the first example work?
The operation of addition is left-associative - the analysis of the arguments begins from left to right. Moving in this way, the parser sees an expression in which there are two operations ‚Äî addition and postincrement ‚Äî for a postincrement, the priority is higher; therefore, it is calculated first by returning ‚Äú1‚Äù as the value and increasing the variable by one. Then the operation of addition is calculated, adding the resulting unit with a two (since the post-increment increased the value of the variable by one). It turns out "three".

In a similar way, multiplication is processed along with addition: 2 + 2 * 2 = 6, not 8, because multiplication has a higher priority.

In the second case, everything happens in a similar way, but a little differently - the parser, processing the left-associative addition, takes the first two arguments, adds them, gets a ‚Äútwo‚Äù, moves on, sees the two, the addition and post-increment of the variable. The post-increment is more priority, it calculates it earlier, returning ‚Äúone‚Äù to addition, the value of the variable increases, but no one uses it - the numbers ‚Äú2‚Äù (from the previous addition) and ‚Äú1‚Äù (returned post-increment) are added. It turns out "three".
