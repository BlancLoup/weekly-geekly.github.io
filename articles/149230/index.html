The Bricks - UI framework



Bricks is a free set of user interface components. It is designed to simplify the creation of design and prototyping of the future site. This set will allow you to save time in the development of web elements and will be a wonderful basis for creating a website from design to layout. You no longer need to spend time drawing numerous forms, as they are already included in this set. Just move the component into your project, change its style and use. It is simple, fast and elegant.

Each Bricks component can be customized and edited. The component folder contains various visual variations of these components. You can adjust the rounding radius yourself, select any style you like, or choose an element variant when you hover or click.

Let's take, for example, a simple input field. At first glance, this is a fairly simple UI element, but it is not. The input field is a kind of symphony, which contains many different options, for example, the type of field when an error occurs, when the operation is successful, when the cursor is hovering, when you click on it. And if suddenly the client decided to replace rectangular shapes with shapes with rounded corners? Bricks will help solve this problem, because each input field is a component that has many settings.



Folders with texts that are used in the component are highlighted in blue. If you want to change the titles or description, you can find them in the directory ". Content.

You can also create your own set of styles for components, which will help you save time and use them in future projects.

Bricks is the core of the framework. In the near future we will release to it cool add-ons containing additional components. These additions will be divided into categories such as forms, elements, e-commerce and words.

Our plans


In the near future we are going to release 4 add-ons for the framework with hundreds of new components. We also plan to update its core by adding new elements to it. In order for the set to exist and be updated, we need your help. We will be happy if you tell us what it has shortcomings. You can also make your own components or add configurations to existing ones and send it all to us. If we like them, we will include them in the standard Bricks assembly.

Currently, Bricks is presented for designers in the form of PSD files for Photoshop. We plan to impose elements of the set and additions, thereby creating a full-fledged framework for developers.
