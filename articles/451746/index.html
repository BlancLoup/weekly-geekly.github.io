MegaSlerm for Kubernetes engineers and architects


After 2 weeks, the Kubernetes Intensives will start: Slurm-4 for those who meet k8s and MegaSlorm for k8s engineers and architects.


At Slurm-4, there are the last 10 places in the hall. Those who want to learn k8s at a basic level are enough.


For Ops, who gets to know Kubernetes, starting a cluster and closing the application is a good result. Dev requests have even less: to understand how to optimize an application for work in a cluster.


Engineers and architects have tasks at a different level:


  • is it possible to run the database in k8s, do we need to do this, that we win and lose from such an architecture;
  • which approaches to deployment are obviously not suitable for k8s;
  • what kind of security threats does k8s carry (for example, a developer can easily get admin access) and how to prevent them;
  • what infrastructure automation schemes does k8s allow and how to learn how to find and use them.

If you have already reached this level, a lot of interesting things await you at Megaslum.
