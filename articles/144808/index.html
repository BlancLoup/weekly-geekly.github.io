1. "Development Platform for High-Load Web Service: Debugging Tools and Scalability" (Alexander Demidov, 1C-Bitrix)

Vertical scaling capabilities (by increasing memory, CPU, disks of an existing server) of a web project are always limited by physical characteristics. And sooner or later, the developer is faced with the task of implementing horizontal scaling - distributing the project to several servers. A traditional website is bad for this purpose - most often, scaling means that the project code is refined. In the report, we will describe how to provide scalability directly in the development platform, thereby making life easier for web developers. In addition, we will consider which tools for debugging, monitoring, and searching for potential bottlenecks can be used directly within the site, as they can complement external services.

  • unlimited scalability;
  • the ability to obtain any number of resources at the moment when they are required, upon request, automatically;

  • savings are much more important than ‚Äúfair‚Äù consumption pricing;
  • scaling often does not work as expected and is not achieved by simply changing the hosting;

  • ‚ÄúResources by request‚Äù is a half-truth: not all the necessary resources can be requested and received;

The report focuses on adjusting popular myths about clouds, describing how to achieve cost-effectiveness, performance and fault tolerance, and listing what is extremely difficult to achieve.
