Custom scripts in Opera Mobile for android

It's no secret that for android there are quite a lot of Internet browsers, each has its pros and cons, you have to keep five of them in your phone just in case. It so happened that Opera Mobile was always the main browser for me, well, I got used to it since the days of Symbian, even made a special Opera for the android with support for extensions, as for the desktop version, in the mobile Opera you can also enable support for custom scripts (UserJS ) , it is done quite simply through the settings in opera: config . First you need to decide where our scripts will be loaded from, for myself I decided that I would allow them to load from the UserJS folder, created such a folder in the phone and put some scripts in there, this is the way I got to the folder: / sdcard / Opera / UserJS , then you need to activate support for user scripts in Opera, type opera: config # UserPrefs | UserJavaScript in the address bar, open the settings, tick the box User JavaScript and in the User JavaScript File field set the path to the folder with scripts in my case: / mnt / sdcard / Opera / UserJS, Nazhmaem bottom "Save" button and re apuskaem Opera. Here ubuntuone.com/4Vk16aVjrDTcvaAV1Bat7r I uploaded several scripts for checking, a script for convenient reading bash.Im , a couple of scripts for Habr (HabrAjax and HabrPercentageRing), a script that clears the Captcha from DepositFiles and a timer before loading the file, I added a larger amount, I already use a larger amount. after the tap to which the file will start immediately, additional settings will be required in opera: config , in the address line we write: opera: config # UserPrefs | CustomUser-Agent , then in the Custom User-Agent field you need to register DepositFiles / FileManager 0.9.206 , and if you still have YB / 3.5.1 added in the space after the space, then from file sharing andex The people will also disappear captcha and the file download will begin immediately.
