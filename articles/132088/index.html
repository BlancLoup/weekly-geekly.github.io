Matryoshka principle

To experiment, presented below, I plow this comment . I decided to beat the photojob, which was presented there.

Therefore, I will have Windows 3.11 inside Windows 98 inside Windows XP inside Windows 7 .

Ps. This experiment was done solely to test the possibility of running virtualization inside virtualization.

How to achieve the goal.


I will not specifically describe in detail how to do everything in steps. Because everything has long been painted and there is nothing new in the process. And screenshots of the installation of each product can be found on the Internet.

Therefore briefly:

1. In Windows 7, run the component Windows XP Mode. In reality, Microsoft Virtual PC runs with Windows XP preinstalled.

2. Now in running Windows XP, install Oracle VM VirtualBox. This is the only virtual machine from the top three popular VMs (VMWare, MS Virtual PC, VirtualBox), which has worked inside the virtual machine. QEMU, Bochs I did not try.
Now we create a virtual machine and install Windows 98. VirtualBox does not have drivers for Windows 9x. As a result, we had to install a universal vesa driver for the video. Not the best option, but suitable for my purposes.

3. Inside Windows 98, install DOSBox. When running DOSBox, we admire the vesa driver glitches. Double-pressing the Alt + Enter combination saves the day. Next, install and configure Windows 3.11.

As we see, both virtualization and emulation were used to solve the problem.

Upd: In the comments suggested that due to the nature of the work of the Windows 98 kernel begins a heavy load on the processor. To solve this problem, you need to install a special driver in Windows 98.
