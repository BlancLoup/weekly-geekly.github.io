Microsoft will open the source code of FoxPro DBMS

Microsoft will publish the source code for key Visual FoxPro components. At the same time, the company will not continue to develop the legendary database management system after version 9.0.

The opening of the source is carried out in the framework of the project CodePlex , which was created specifically for these purposes in June last year.

The company guarantees support for users of Visual FoxPro until 2015, including standard support until January 2010, and then - according to a special plan.

The team of creators of FoxPro will be scattered across other departments of Microsoft. In fact, the company closes this project and publishes source codes only so that independent programmers can get at least some chance in the future to keep the FoxPro systems working.

Independent programmers themselves are well aware that they were buried alive. FoxPro is an excellent DBMS, where you can still perform many tasks, but in the absence of support it is extremely risky to use this technology. Therefore, everyone is looking for a way out of this situation. For example, you can start rewriting programs for Visual Studio (.Net) and / or SQL Server right now. You can find help on this issue on the VFPConversion.com site in English or on the FoxPro Club in Russian.
