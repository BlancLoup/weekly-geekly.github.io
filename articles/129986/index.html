Habra Informer v.0.1

Most recently, I started working on the habra-informerorm , in order to conveniently follow the updates on the site and not be distracted once again.
A number of works were carried out in which it was implemented:
- 3 streams (monitoring of new articles, behind a sandbox and for QA).
- Fixed some bugs.
- Now the script immediately displays the title, the number of comments and who left the topic.

The same work was carried out further, thanks to the drawn design from AcidSlayer, we began work on the basis of the database (sqlite) and made out using QML. But due to a number of reasons (the result led to the inconvenience of users, for example, dependencies on the declarative library), we had to not use this solution in this intermediate version, but the work was not stopped and was being done in this direction.

In future

So it remains in the plans:
- Implementation on QML
- Using DB
- Authorization and opening functionality with it
- improved tracking and notification of updates (not only new records)
- Also, users were invited to implement broadcasting from the informer directly to the site (commenting, creating new articles, etc.), but this functionality remains in the plans.

I would be happy for any advice, recommendations and ideas.
