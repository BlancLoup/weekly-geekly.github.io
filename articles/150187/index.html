Sophisticated authorization methods

Instead of the preface


At every step, everything is now simplified and simplified, authorization is becoming less noticeable and more reliable. In this topic, I tried to fantasize about complex authorizations.

Many input fields for entering email addresses



On the authorization form is a number of input fields associated with the grid.
The login key is a bunch: email + key.
For example:
pupkin@mail.ru
F2C1
Login is recorded in cell F2, the address of the mail server with the dog is in cell C1.

Bank cells



The authorization page is a large number of bank cells with numbers.
When registering - choose a free cell (no lock) and enter your email address there.
Our key is our email, and the number of the cell that we chose is unknown except for us.
(It is clear that for millions of users this method is not very suitable, but still it has the right to life)

Browser resources


An extension is put in the browser that will know which tabs we have open (I'm not sure that extensions can do this) .
The key to enter the site is a bunch of email and a set of open tabs (for example: Habr, Bash and Yandex Weather).
Alternatively, instead of tabs, you can use a set of browser bookmarks.

Chess



In this version, the key is the position in chess. Without logins and emails, positions in chess and so much.
When registering, the user moves the pieces, exposing only to him a known position in chess. Removes extra pieces from the board, the remaining moves.
For authorization, you just need to remember your original position.
