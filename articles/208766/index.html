Dropbox crash


Today, January 11, 2014, many users began to receive errors when working with the Dropbox service.
The error manifests itself in the form of a service response error 500 when trying to perform various operations.
Personally, I noticed this on the example of failures in the work of my iOS application using the Dropbox API.


In my case, this was reflected in the fact that it was not possible to delete files from the application folder. In addition, when I tried to enter the account settings on the site, I received 500 Error.

There is a topic on the Dropbox support forum that discusses the crash
Many people write that the files uploaded to the cloud were damaged.

In this thread, one of the users indicated that he was helped by rolling back the desktop version of the Dropbox client from version 2.6 to version 2.4

UPD: Found a link in which Dropbox employees unsubscribe about the progress of work: Dropbox Status Update
