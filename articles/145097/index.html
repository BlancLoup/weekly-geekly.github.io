We listen to music from Vkontakte through Amarok - V.2.0

Hello! I have not written articles for more than two years, and now, finally, free time has appeared. You probably do not remember my topic 2 years ago - Listen to music from Vkontakte through Amarok . Let me remind the essence - then I wrote a plugin for KDE-shnogo Amarok`a, which allowed to search and listen to music from Vkontakte. If you are still interested, I ask under the cat (carefully - pictures!).

Time passed, bugs were fixed, the plugin gained popularity and consistently ranks among the top 10 applications on kde-apps.org, but I always want something more ...
And now, after a few free evenings, I present to your attention an updated improved version of the vk_search 2.0 plugin.
So, what's new:
  • Screwed OAuth, thanks to which now the main feature has appeared - your music and your friends
  • The search is also present - just enter the desired artist in the search field
  • Well, by the way - the code was completely rewritten and now the latest version of api vk.com is used


When you first start you will be asked to allow access to data - for this you need to enter your login and password:

You should not be afraid - this is a browser, your data will not leak anywhere + neither cookies, nor anything else is saved (therefore, you have to log in as soon as the token expires). Besides - you can see the source code yourself and make sure that there is no bad thing there;)

Well, a couple of screenshots for clarity:






The plugin is available both via the Amarok settings - Scripts - Download, and on the website kde-apps.org: vk_search and on the githaba: vk_search

UPD: Upgraded plugin - now authorization "eternal" + music is translated into status.
UPD2: If the plugin does not work for you and it does not appear in the menu, install the libqtwebkit-dev package (for Ubuntu, specify the name for your system). Also, there is a problem when working with the mplayer backend - the tracks are not played and Amarok crashes.
