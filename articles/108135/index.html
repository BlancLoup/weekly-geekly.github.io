Tips and tricks in Opera Mail: response templates and filter synchronization

I'll tell you about the two settings in the Opera's mail client, which can only be found in the config files.

1. It is convenient to insert two Newlans at the beginning of the response template (otherwise the cursor will appear right at the beginning of the quote). Example:

This is convenient if you answer at the top of the letter. So do Gmail and Yandex, I think, so it is more convenient for most.

2. You can make the contents of the filters in the mail synchronized via IMAP (like Labels, which are synchronized without additional settings). Those. if you add a letter manually to the filter in one opera, it will be added to the same filter and in all others working with the same box.
This is convenient if:
  • You use the mailbox in the opera on multiple computers
  • You use several versions of the opera at the same time (stable and beta).
  • You want to be independent of your folder with the settings of the opera (so that, in which case, to raise everything from scratch)

How to do it all - under the cut.



In both cases, you will have to edit the config files manually.
First you need to define a mail directory. to do this, go to opera: about and see the Mail directory.
Now close the Opera and wait until it disappears from the process.
Go to the Mail directory.

1. Edit the response template.
Edit accounts.ini
We are looking for lines that begin with Reply =
There will be several such lines (by the number of accounts +1)
By default, the setting looks like Reply = On%: Date :,% f wrote:
To add two newlines ... everything is logical, correct: Reply = \ n \ nOn%: Date :,% f wrote:
You can also correct the template for the forward: Forward = ..., but personally it suits me.

2. Sync filters
Editing index.ini
We are looking for the string: Name = <filter name>
Below there is a line "Keyword =". Those. keyword is empty. As soon as we write a non-empty keyword here, messages added to the filter will be marked on the IMAP server. If in another opera the filter is marked with the same keyword, it will be synchronized.
So: we put the same keyword on the filter on all operas. I choose a keyword that matches the filter name, but I cannot guarantee that it will work with spaces.
After assigning a keyword to add a message to the filter, the opera will connect to the server. This is clearly seen at the bottom of the mail panel.
Important: edit the filter is best immediately after creation. If there were already letters in the filter, then after editing they will not be noted on the server. In this case, you can move messages to another filter (temporarily), and then back.
Only those filters that are filled in manually make sense. Automatic filters will work this way.
Interesting fact: the internal implementation of labels is absolutely identical to filters.
