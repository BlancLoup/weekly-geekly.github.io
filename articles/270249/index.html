New Intercepter-NG released [Android Edition] 1.6

Today is a small holiday for fans to expand the functionality of their phones and other gadgets running on android. New Intercepter-NG [Android Edition] released under version 1.6.



To be honest, initially no major improvements were planned. The main idea was to rebuild the code to work on Android 5. Actually, this was the main wish of users over the past year, but as is usually the case, the appetite comes with eating, so plans changed during the play.

The current version works on the 5th and 4th versions of the android and theoretically should work even on 2/3 (but no such tests were performed). After the launch, you can see the cosmetic changes: icons in colors from the original windows version and support for portrait mode, which, surprisingly, turned out to be very usable.



In the settings you can change the font size, which will affect the text output in all modes (except for scanning).

Scanning engine was borrowed from the latest versions of desktop interceptor. Due to this, more details about the OS versions are displayed, and network names are resolved via NetBios and Bonjour.

In cookie interception mode, you can see the full contents of cookies. The most delicious of the new products is the introduction of the SSLStrip into the mobile version of the desktop along with the Cookie Killer. In addition, the Forced Download feature has been added, which appeared this summer in Intercepter-NG 0.9.10, which allows you to force the download of the selected file through the browser of the victim.

The Cookier Killer and Forced Download functions can be turned on / off dynamically in the settings, without restarting the attack, the only condition is that the attack should have been launched with SSLStrip.

Poisoning of selected clients occurs instantly, there is no need to wait as it was in the previous version.

Also made many minor improvements and fixes. Feedback and suggestions can be sent to the mail, forum or the corresponding branch on w3bsit3-dns.com .
