Magical Transformation: A wireless access point or router becomes a hypertext Fidonet node

This news directly concerns the owners of Asus RT-N16, Asus RT-N12, Asus RT-N10, Asus WL-500gP, Asus WL-500W, D-Link DIR-320 devices . It has long been a secret that all of them (after one or another flashing) can be used as a Linux MIPS server. On this topic there is a forum wl500g.info , and even at Habrahabr we saw reports and recipes of half a dozen bloggers who set up a print server on this hardware, a file server with HTTP and BitTorrent access , a WiMAX router , increased RAM, and even soldered a hard drive .

The next step in this direction was made by a fidoshnik named Alexander Ryzhov (2: 5019 / 40.5), who collected for the same architecture all those free programs that are included in the fidoip package (the same package that I mentioned here in January last year in connection with the advent of support for it hypertext Fidonet).

It turned out the Fidonet knot, which is able to work around the clock, without making excessive noise, without consuming excess electricity, without at all interfering with the household of Fidoshnik. After that, if necessary, you can go to it via SSH - and read and write fidopochta.

All the necessary software and installation instructions can be easily found on the fidoip wiki at SourceForge.
