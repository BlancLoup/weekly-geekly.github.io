For the server side of the application, a little-known framework, modestly named di, was chosen. In this software, the (H) MVC pattern is implemented in the classical form:

Those. any request from the user will be redirected to the Model through the dispatcher, it will in turn analyze and process it, prepare the system for operation and transfer control to the required controller. Which, in turn, interprets the data and gives it to the user through the presentation. If necessary, the controller can also change the state of the Model.
Based on the above, the system uses the following file architecture:
  • controller (place for controllers)
    • Controller.php (default controller)
  • model (heart of the project)
    • core (place for classes and libraries)
    • data (settings files)
    • include (space for third-party libraries)
    • index.php (file that will handle user requests)
  • view (place to store templates for presentation)
  • .htaccess (plays the role of manager)
