Makhost: how to return your domain (except. RU)

If the makhost distributes hosting, then here is my article. (I wrote a draft, I waited how it would end, and I forgot.) The life story about how to regain control of a domain registered through Makhost . Return domains RU has already been discussed here , and I will tell you about the passage of a similar quest with foreign domains.

Prehistory

My sites lived on Makhost until April 16th. When the second shutdown occurred, I moved the sites to another hosting. At the same time I decided to control the domains directly, through the registrar admin panel. Two RU domains were transferred easily: I registered on Webnames, asked Makhost to transfer control to my account, which was done. There was one more .MOBI domain, fortunately there was no existing site on it.

Level 1

Webnames said they needed an authorization code, which you can get from Makhost or from a registrar. I turned to Makhost, an employee with a rare name Pulp Flup replied . The dialogue blew the brain:
- How to get an authorization code for the domain ****. Mobi?
- We do not issue the code, you can manage the domain through us.
- How can I manage a domain through you without using hosting?
- It is necessary to use hosting services.
We'll have to contact the registrar ...

Level 2

Using DomainTools, I found out that the registrar is DirectI. We could not immediately find a feedback form, the company has several projects, each project has its own website ... As it turned out, customer support can be contacted here . Using the knowledge of English and Google Translate, I wrote something like the following: ‚ÄúHow can I find the authorization code for a domain ****. Mobi? The domain is registered through Makhost, they refuse to report the code. ‚ÄùImmediately they sent a password to the support system. An hour passed, another ... there was no answer.
About two hours later, a letter arrived from Makhost with an authorization code (by the way, from another employee). A little later, a message appeared in the DirectI support system stating that my message was transferred to some department (I did not remember the name in long and in English). And then the code itself was written there too. Identity confirmation was not asked, apparently it was enough that the e-mail address in the domain data and the one specified during the call match.

Level 3

In the Webnames admin area there is a ‚ÄúTransfer Domain‚Äù item. There I indicated the domain name, entered the authorization code. After that, you must pay for the renewal of the domain for at least a year, then a letter comes to the domain administrator's e-mail asking if it‚Äôs really necessary to transfer the domain. Naturally, I agree (just waiting for this).
Then comes another link - "if you want to refuse the transfer, click here." The link is valid for five days. As soon as they expire, the domain is transferred. Hooray!

Instructions + result

  1. Ask Makhost authorization code (authcode). If they refuse, we go further.
  2. We register on Webnames (or any other registrar that supports your domains).
  3. We determine the registrar with the help of DomainTools, we write to him that Makhost is scoundrels, they do not give the code.
  4. Get the code, transfer and extend the domain.
Bottom line: Makhost is trying in every way to keep customers, sometimes using the most filthy ways. The quest to move to another hosting is not so easy to perform, but sometimes it is easier than it seems, and certainly worth the effort.
