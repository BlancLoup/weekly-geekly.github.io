New version of the sniffer 0x4553-Intercepter 0.8 + 0x4553-NAT

The long-awaited update of the Intercepter sniffer, in addition to minor fixes, DHCP server mode was enabled.

It has the minimum required functionality and gives the computers on the network the specified settings, faster than the existing DHCP server does. If a legitimate server suddenly has time to issue settings earlier, then Intercepter
will send a special response to the client, after which the client will resume the settings request procedure.

The main purpose of this mode is the hidden delivery of a false network configuration to intercept traffic.



Stealth is to use a non-existent MAC and IP address, as well as to issue a nonexistent gateway, which in turn will route the packets, which will allow to intercept and process data.

Also, from this point on, 0x4553-NAT is part of Intercepter.
This is symmetric NAT, which translates packets from Ethernet to Ethernet, and from Ethernet to PPPoE.

They are supported by the 'stealth' mode for collaboration with DHCPD spoiler.

Read the help file before use!
