Debug Mail - an easy to use smtp server for developers



Debug Mail is a service that provides a mail server that intercepts and stores all emails passing through it. The addresses of the sender and recipient do not matter, the actual sending of letters does not occur.

To connect the service, it is enough to register the SMTP server settings received after registration in your development environment. After that, all email traffic will be processed using Debug Mail.

Key features:

  • Real-time update of the list of letters in the web-interface. New emails are displayed immediately after the email is received by the SMTP server.
  • Create multiple projects to group letters by him.
  • Inviting colleagues to collaborate projects.
  • Quickly search for messages, group them by title and date received.
  • View headers, html, text, plain letter versions.
  • Adding comments to the html content of the letter through the service coment.me .
  • Opening access to specific letters on a short link (including for unregistered users).

The service is in the public beta. We will be grateful for any feedback - judge strictly!

For those who are interested in how the service was developed, I recommend reading the post about the Debug Mail server architecture .
