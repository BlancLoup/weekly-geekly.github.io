Free translation close to the original .

Attention! There was a new build , which, it turns out, not everyone noticed. I ask this topic to be outdated.

Accelerated process.


This spring we are doing a little differently. First, we staffed our team of developers of the desktop version of the browser with a large number of first-class specialists. This means that now we are moving faster than ever. Secondly, as you know, we are conducting a number of significant changes to the integration into the platform, which will give significant improvements for all three desktop platforms. The current calculations for the remaining work on Opera 10.50 show that we will achieve the quality of the final product on Windows earlier than on other platforms. Thus, we decided to allow Windows users not to wait for the completion of work on versions for other platforms and release the final version for them a bit earlier. In the near future, you will see how we reach the beta stage for Windows, while Mac and Linux will reach the beta stage around the time that Opera 10.50 on Windows is already close to release. Obviously, when the Windows version reaches the final stage, we will focus our attention on bringing the version for other platforms to the same quality and as soon as possible. We also expect this to be an exception to our operating principles, meaning that in the future we will release final versions for all platforms at one time.

The return of the "penguin" and widgets for the desktop


This build contains a kernel update with a large number of regressions corrected. Many functions in 10.50 become more stable or work again as in version 10.10. Also, widgets can again work as standalone applications, as was previously implemented in alpha version 10.20. And finally, the Linux builds are back!
