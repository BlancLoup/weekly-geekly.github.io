Acronis True Image for Mac - why and for whom?

Good afternoon, dear inhabitants of Habr! Today we proudly present our new product, Acronis True Image for Mac , an application that protects your Macs from data loss.

Actually, as the name implies, this is a product for the Mac platform. And at the moment the most frequent question that we encounter is why and for whom we have made this product.

Why and for whom did we make this product?



By releasing Acronis True Image for Windows, we have recently been increasingly confronted with requests for Mac support from our clients. It is understandable with the success of Apple over the past few years.



Yes, in Mac OS X there is Time Machine, but if a person used Acronis True Image, then, of course, he wants to have the same solution under Mac OS X. That is, the first group of our clients are our existing clients with mixed environments .
The second group of clients are people who are not satisfied with how embedded Time Machine works. Let's dwell on this in more detail.

Time Machine Setup

Time Machine makes backup copies once an hour, and it is not so easy to change. Yes, for most people reading this post and using Mac OS X, this is not difficult, but a simple user will hardly cope with this quickly. So the ability to clearly plan backups is a plus.

Incremental copies of files

Time Machine does not do incremental backups very well. That is, they are there, but at the file level. If the file has changed, then it is all entirely and will be added to the backup. This is especially painful for virtual machines. In fact, using TM in this case is almost impossible. The place where you fold your copies gets clogged up very quickly.

Virtual Machine Support

When it comes to virtual machines, we support backing up a Parallels virtual machine. That is, even if you have a Parallels virtual machine running, a backup copy of your disk will also contain it, and in a consistent state. From this copy, you can restore the virtual machine without losing data and ‚Äúboot‚Äù. Why only Parallels? Mostly because Parallels dominates the market.

Cloud Backup

Cloud Support. We can make backups to Cloud. Yes, the first copy will take time, it is better to put it at night, for example. But in the future everything will be much faster. Mac OS X users are, on average, more mobile than Windows users, and therefore we consider it very important to have access to our backup copies and, most importantly, continue to make these copies on the go.

Data compression

Time Machine does not compress data during backup. We squeeze.

SMB support

We can make backups on the SMB ball (AFP, of course, too). For a mixed Windows ‚Äì Mac environment, this is also a noticeable plus.

As a result, in the current decision, we tried to focus on the most reliable data storage, taking into account the interests of the target users. Using Acronis True Image for Mac, you can quite effectively follow the rule of three copies in different places (one copy must be deleted). One copy is on your machine, the second is on a local USB drive or NAS, the third is in the Cloud.

To be completely honest, let's talk about the current limitations of our product. Since any commercial development is limited in time, we also had quite tight deadlines. Therefore, it was decided to make the minimum required set of functionality in order to enter the market and begin to receive feedback from users.
