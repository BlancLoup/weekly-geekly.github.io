All announcements of TechEd Europe 2014: partners, clouds and news for IT pro

On October 28, TechEd Europe 2014 started the conference, during which several key announcements were made. New offers from Microsoft partners, new technologies for building IT infrastructure were announced and new and updated Microsoft Azure cloud services were introduced.


Below are the new announcements of technology and all the news of the TechEd Europe 2014 conference.

In the framework of the plenary report, the news and announcements presented earlier were also announced: the new high-performance virtual machines of the G series and the new high-performance storage Azure Storage Premium. Read about these announcements here .

New offers from partners


We announce the launch of the new Azure Marketplace - places where partners, customers and simple developers can post, monetize or search and place ready-made solutions in the Azure cloud with a few clicks.

We are pleased to welcome solutions from Kaspersky Lab , NGINX Plus ( press release ), Veeam and others, such as SAP, Cloudera, DataStax, CommVault, Trend Micro, Core OS, Cassandra clusters from DataStax distributed from Riak databases, among the first partners of Azure Marketplace. Basho and hundreds of others.

Partners can post their solutions with different prices and licenses. Users will pay for purchased solutions as they are used, paying only what they consume. Already, you can find solutions for every taste from more than 3,000 presented on the Azure Marketplace !

New technologies for building IT infrastructure


New Azure Operational Insights service (log analytics service, server fleet monitoring and data center)



For IT professionals presented a new service Azure Operational Insights . This cloud service will allow you to centrally collect, combine, compare and visualize the data of your computer park. The service will allow you to integrate with your data center and manage the capacity of its resources, monitor server updates and configuration changes, review work environment configurations and processes.

Azure networking features

For the needs of building a cloud infrastructure, many updates of the network functions of Microsoft Azure were presented . Below is a list and brief description:

  • Network Security Groups - now you can create and use groups and network security rules to control incoming and outgoing VM traffic or VM groups in a subnet. Groups and rules are controlled independently of the virtual machines themselves;



  • Multi-NIC Support - now you can create, configure and use multiple virtual network cards in a VM;



  • Forced Tunneling - now you can redirect all Internet traffic that is born in cloud services via corporate networks and VPN for additional monitoring and auditing;



  • ExpressRoute Enhancements - ExpressRoute users can now share a single connection point for multiple Azure subscriptions. Plus, any virtual network can be connected to several ExpressRoute networks;

  • New VPN Gateway dimensions - for solving high load tasks when building hybrid solutions, more efficient Azure VPN gateway gateways were introduced that will speed up data exchange when working with ExpressRoute and Site-to-Site VPN gateways;

  • Operational and audit logs for VNet Gateways and ExpressRoute - now you can analyze and monitor the operation logs of the Virtual Network Gateways and ExpressRoute networks. The Azure portal now has the ability to display all the data needed for analysis of logs and API calls, as well as scheduled infrastructure updates;

  • Virtual Network Gateway policies - now you can manage the encryption of tunnels between virtual networks. There are 3DES, AES128, AES256 and Null encryption options available, as well as the inclusion of the Perfect Forward Secrecy (PFS) option for IPsec / IKE gateways;

  • Source IP Affinity - Azure load balancer now supports Source IP Affinity (also known as session affinity or client IP affinity). Now you can balance traffic based on a bundle of two parameters (Source-IP, Destination-IP) or three parameters (Source-IP, Destination-IP and Protocol);

  • Traffic Manager nested policies - now you can create nested policies to manage Traffic Manager traffic for more complex load balancing scenarios. Detailed announcement ;



  • Internal Load Balancer, Reserved and Instance IP addresses for VMs on the new portal management - now you can configure the internal load balancer settings, as well as IP reservation on the new Azure portal.

New and updated cloud services


New Azure Batch service (HPC tasks and clusters on demand)



Introduced a new service Azure Batch , which allows you to easily perform HPC and loaded batch tasks in the cloud when scaling from tens to tens of thousands of virtual machines on demand.

As part of the plenary report, Mark Russinovich demonstrated with a real example how an extension for the popular 3D tool Blender allows using the Azure Batch to select 37 virtual machines directly from the editor for calculating complex graphics and accelerate the rendering of the scene dozens of times.

Azure Batch will allow your to solve the following tasks: in the shortest possible time, process data packets by allocating as much power as you need on request, build and receive clusters in the cloud on demand, scale hundreds of times, perform tasks on a schedule and from queues, scale on demand and automatically , get it all in the form of a ready-made service at no cost to the infrastructure.

Announcement of the new service in the official blog. Learn more about the new service on the official page . Sample Python code for Azure Batch is available on GitHub .

New service Azure Data Factory (orchestration by data streams in the cloud)



A new service Azure Data Factory is presented, which offers customers a single mechanism for managing data movements, orchestrating and monitoring various data sources, including SQL Server, Azure Blobs, Tables, SQL Database.

Developers can use Azure Data Factory to build efficient data workflows that aggregate, aggregate, and transform data from local, cloud, or remote services.

Azure Automation GA Release (Infrastructure Deployment Automation Service)

Azure Automation enables Azure administrators and users to create, deploy, monitor, and maintain resources in Azure's cloud environment using scalable and flexible workflow tools and scripts.



Azure Automation is based on PowerShell Workflows (runbooks) for describing all administration and automation processes for deployment. You can start creating rabucks from scratch or take one of hundreds of ready-made rabucks from the gallery on the Azure portal.

Useful Azure Automation resources: blogs , documentation , forum , User Voice .

Azure Event Hubs GA (Cloud IoT service for connecting thousands of devices and processing millions of events)



The Azure Event Hubs service, previously available in the preview stage, is now commercially launched. The service allows you to receive and process millions of incoming messages in real time and deliver them to cloud applications. The service is focused on the Internet of things and connected devices, allowing you to combine thousands of devices and millions of events.

A detailed description of working with the service can be found here . New Microsoft IoT blog.
