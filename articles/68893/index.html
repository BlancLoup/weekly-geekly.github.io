Neutrino Atomic Edition 0.8.7

What it is?

Neutrino AE is a fork of the open source Neutrino CMS , the main highlight of which is the minimalism of undemanding to resources and hosting capabilities. Development has been conducted in parallel for the past year and a half, and technologies from one edition periodically jump to another. The license is GPLv3. The archive with CMS weighs very little, the kernel runs on bare php without additional libraries, does not require a database (the information is stored in text files), is fast and has a low server load.

What is the difference?

The main difference between the Neutrino AE and the classic version is that the atomizer is positioned as a human-friendly system. It is easier to install and configure, as well as its basic kit includes the minimum required set of the most popular and useful plug-ins. Classic Neutrino comes in bare form, and all the additional features, except for creating pages / blog entries and commenting on them, have to be added with additional plugins, or added independently.

Neutrino AE features

At the moment, in Atomics, the following functionality is implemented:
  • creating articles (by default they are presented as blog entries)
    • Visual editor nicEdit (implemented as a plugin, included in the basic distribution)
    • taxonomy (tags to records)
  • commenting articles (optional, disabled in the settings)
    • pre-moderation (optional, disabled in settings)
    • bbcode support in comments (optional, disabled in settings)
    • support for wiki markup in the comments (optional, disabled in the settings)
    • captcha to protect against spam bots (implemented as a plugin, included in the basic distribution, disabled by default)
  • phptemplate template-making (several public themes have already been created, you can find them on the forum
  • Sophisticated plug -in system - all functions of the kernel are connected to the system hooks, allowing you to completely redefine the functionality.
  • Setting up the site through a web interface
    • Plug-in management
    • Managing blocks on the site (implemented as a plug-in blocks, included in the basic delivery)
    • Setting up information about the site and editing the standard settings (commenting, the number of entries in the tape, etc., implemented as a plug-in settings, included in the base. Delivery).

What's new in 0.8.7

All system functions are integrated with the hook system , so it is possible to implement any additional functionality in plugins. Regarding flexibility, Neutrino AE is now not inferior to such popular solutions as Drupal or Wordpress - just write plugins. Also, the block system on the site has been significantly reworked. With the introduction of a set of regions, site owners no longer need to go into the code to display an additional block. All this is done through the admin panel of the plugin blocks . It also indicates in which part of the web page to place the created block, and the visual editor makes the creation of static html-blocks simple to disgrace.
