Jabber instead of ICQ


We want to introduce you to another server function that is related to the client and in some way affects privacy. Its goal is to allow you to communicate using the Jabber protocol as much as possible, and using the ICQ protocol as little as possible.
In 9034, if you search for ICQ contacts or Mail.ru account on the QIP protocol tab , then a request will be made to the server of metacontacts, and if the user of the required number has a qvip account, then instead of ICQ, his JID qip .ru will be issued.



For those who lead a double (and more :) ) life on the network (and among QIP users, alas, there are those who have so much fun), this function is unpleasant, so the setting ‚ÄúDisable JID issuance when searching for my ICQ‚Äù is provided. The server function, so that it can be disabled on the personal page of qip.ru/settings and in the QIP-Jabber protocol settings. The webpage has not yet been updated, but soon this tick will appear there.


The function is introduced only for domains included in QIP Apps, for isolated mode and for Jabbers other than QIP-Jabber, the function is missing. For domains participating in the partner program also works, but only if the account is the main QIP Infium account.

During the creation of QIP Infium 9034, quite a lot of changes have occurred in the program's functionality. Covering all is not possible, and not necessary. We will try to present the main ones:
  • Improved stability. The plugins that made a mistake now turn off automatically without causing the program to crash.
  • Compatible smilies. As part of the idea of unifying smiles, the first step has been taken. Added a lot of new smiles (Kolobok Big Pack) By clicking the right mouse button to select, all smiles are shown, and when you click the left, only your favorites, pre-set by you. In the incoming text displays all the smiles. Thus, now you can add any smiles from any client, and use your usual set to send. Everything is documented on the wiki .




  • Significantly redesigned contact list view. Now the user can determine whether he wants to see the avatars of the interlocutors or not, to set the positions of the elements of metacontacts. These actions are available to any user using the ExStyle_Editor program. The menu function Settings-Contacts list is activated. You understood correctly - in fact, this is the first option in a switch that is not enabled by default. For the sake of experiment :)
