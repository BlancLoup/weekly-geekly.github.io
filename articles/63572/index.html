Firefox 3.5, Linux and non-original size pictures

Hello dear habrareyudi. I have been using FF 3.5 for a long time, starting with the deepest alpha versions, but this happened mostly at work, where I have Windows XP on duty. Imagine my surprise when, under the newly released Fedora 11, instead of images on some sites, I began to observe black squares.
Pictures and the continuation of the story under the cut.

As an example:

With the help of inexhaustible inquisitiveness, it was found that this effect occurs when displaying pictures of non-original size.

img src=http://path/to/picture/something.jpg width=xxx ....


In addition, experimentally it was found that:

  • .Jpg and .png are precisely affected by this effect.
  • This problem is not with color profiles and it cannot be solved by the remarkable man Twangel mentioned in the recent habratopic and all those who participated in the discussion.
  • It works like the original distribution version there and the binary build downloaded from offsite.
  • The problem arises regardless of where the picture is located, on the same host as the webpage, or on a remote server.
