The proprietary code of the exFAT driver is illegally published under the GPL.

After leaking the source code of the proprietary implementation of the exFAT driver from Samsung, the code for this driver was distributed over the network. The GPL-violations.org mailing list draws attention to an outrageous fact . Someone took the code, erased all references to the original Samsung license from there - and published it on Github under the GPL license.

Worse, this guy (or girl) does not respond to comments and is rude in the comments .

I am a programmer, not a lawyer. Here's the code for you, decide for yourself what to do with it, at your discretion, ‚Äùthe user rxrz, author of the repository, writes rudely in the comments.

In fact, GPL-violations.org is monitoring the violation of GPL patents when someone takes a free code and illegally uses it in proprietary projects. In this case, it turned out exactly the opposite. After leaking a proprietary code, someone put it on Github under the GPL. Such things can also be fought, because they can cast a shadow on the OSS reputation and put the Open Source community in a bad light.

A spokesperson for GPL-violations.org has already reported the incident to Samsung. In the next few hours, you can expect to send a DMCA request - and the driver code will be removed from the Github site.
