If you decide to implement an ERP system

We must try to follow three simple rules if you want a successful implementation.

Minimum development


It is not strange, but it is software development that is the weakest part of ERP projects. No matter how beautiful you wrote, you never know in advance whether it will work in real life or not. Good programs are not written the first time. The more in the development project, the higher the risk of failure. Software development is like creating a design. It seems to have agreed on everything, but when I got the result, you understand that it is not that. Of course, there is only one way to reduce the amount of development - to take a product, where you have already developed and work what you need.

Development requires debugging. This means that after the development there may be problems at the start of the system (something does not work, something does not work, something has been forgotten about, etc.). This process reduces user loyalty, which will then be extremely difficult to raise. Therefore, you need to start to implement with some kind of basic, ready, reinforced concrete working, proven functionality that does not require any development. Put - works.

Maximum employee engagement


The more employees are involved in the project, the greater the chance that everything will be fine. If in the project someone is spinning like a squirrel in a wheel, and the rest, picking a stick in his teeth, are watching this fascinating attraction, then there will be no result. However, this rule applies not only to ERP-systems, but also to all other large-scale projects.

Quick results


The project should give quick and concrete practical results. Some part of the system must be started as soon as possible. What - it's up to you with a consultant. If the project lasts a year and no one uses the system, then everyone gets used to the fact that the ERP-system in your company is needed in order not to use it. Everyone just gets used to the situation and does not want to change it. Fuse, which was initially beginning to fade. The system should be started when there is a fuse and desire, and not in a year.
