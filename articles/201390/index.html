Distribution of elephants: Beeline gave free access to its subscribers to the Beeline WI-FI network via the EAP-SIM protocol



Many, when entering cafes, restaurants, airports, noticed the presence of the Beeline WI-FI WPA network - This is the paid network of the former Golden Telecom, previously it was only possible to access it by paying for or receiving a service as part of a package.

But starting from October 25-26, all users of the Beeline mobile network received free automatic access to this Wi-Fi network.
In other words - went to the network coverage area, if WI-FI is enabled on the device - automatically connected to the network.

(Not) Big spoon of tar


Currently, automatic access only works on iOS devices.

Why only on iOS, how is it implemented?


To set iOS operation parameters, * .ipcc files are used - they are created by the operator and then distributed to Apple companies across all devices.

Yes, yes it is this file that blocks LTE operation in Russia.

It can specify a lot of interesting SSID access point and authentication method.

After the official launch of iPhone sales in Russia, the operator updated this file - ipcc, including auto-connection to access points there.

But there are questions


1) Is there a tariff when connecting to this access point? (If the Hot Spot 2.0 standard is used then it is possible to implement)
2) Is there subscriber authentication or is the general password sewn up (Which would be very strange)
In other words - how the AAA mechanism is implemented
