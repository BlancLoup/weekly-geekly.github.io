Say "No" to the non-free Microsoft Office format as an ISO standard!

I ask ISO national members to vote against the ratification of the Microsoft Office OpenXML (OOXML) specification as an ISO standard for the following reasons:

1. There is already an ISO26300 Open Document Format (ODF) standard: additional standards increase costs for business, government, and citizens;

2. There is no compatible implementation of the OOXML specifications: Microsoft Office 2007 saves files in a special format that is different from the OOXML specification;

3. The specification is not complete, for example, there is no description of autoSpaceLikeWord95 and useWord97LineBreakRules;

4. More than 10% of examples in the specification do not correspond to XML;

5. There is no complete guarantee that software that fully complies with OOXML specifications will not suffer from patents or patent deductions;

6. The specification conflicts with other standards, such as ISO 8601 (Date and Time Representation), ISO 639 (Name and Language Representation Codes) or ISO / IEC 10118-3 (Cryptographic Hash);

7. There is an error in the spreadsheet format that does not allow entering dates before 1900. This error applies to both the specification and Microsoft Excel 2000, XP, 2003 or 2007;

8. The specification was created without engaging the experience and expertise of interested parties (manufacturers, sellers, buyers, users and regulators), exclusively from Microsoft;
