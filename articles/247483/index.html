Dnipropetrovsk Discussion Meetup: CQRS Problem / Solution

On Tuesday, January 13 , the Dnipropetrovsk office of Cyclum organizes a meeting dedicated to the architectural principle of CQRS (Command and Query Responsibility Segregation) .



CQRS (Command and Query Responsibility Segregation) is an analogue of DDD (Domain-driven design) in distributed systems. It is based on the CQS (Command Query Separation) pattern. CQS is as follows: any method in the application is either a query (Query) that returns data, or a command (Command) that modifies the data. The CQRS pattern is quite efficient to use with the Event Sourcing approach, the idea of ‚Äã‚Äãwhich is to record every system change in the repository (EventStore).
