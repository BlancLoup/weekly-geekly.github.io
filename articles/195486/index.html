Meet SQL Server 14

Under this general theme, the next seminar of the Russian SQL Server User Group will take place on October 9 at the Microsoft Technology Center, Russia, Moscow, ul. Lesnaya 5, 6th floor.

The program will feature reports:

In the new release of SQL Server, a buffer storage extension mechanism (data porridge) - Buffer pool extension to a fast storage medium (SSD) is implemented, which significantly speeds up the OLTP processing of the database load. Consider the possibilities and limitations of this technology.

About speaker:
Dmitry Kostylev began working with Microsoft SQL Server in 2000. The main professional activity is the development and support of both the databases themselves and large projects intensively using relational repositories. Currently, under his control, one of the knowledge bases about files in Kaspersky Lab is working and developing.

Columnstore indices in SQL Server 2014, Dmitry Pilyugin (60 min.)

With the advent of SQL Serever 2012, a new type of index became available - Columnstore. In SQL Serever 2014, the functionality of the Columnstore indices has been significantly expanded, many limitations have been removed. The report will discuss the main improvements regarding the work with the Columnstore indices in SQL Serever 2014.

About speaker:
He has been working with SQL Server since 2004, mainly engaged in developing solutions in the field of corporate information systems. He is currently the head of the database team of TNS Gallup AdFact. From time to time he speaks at technical conferences and events, the main and favorite subject is the query optimizer and things one way or another related to him.

Overview of In-Memory OLTP technology (Hekaton) in SQL Server 2014 CTP1, Sergey Olontsev (60 min.)

The report will discuss one of the most anticipated opportunities in the new SQL Server 2014, previously known under the code name Hekaton (translated from Greek means the number 100). In-Memory OLTP is a new component built into the SQL Server engine and provides the ability to several times increase the performance of OLTP queries due to the fact that tables with data will be placed directly in memory. Additionally, it will be possible to compile stored procedures into machine code for even greater performance gains. This component was created with the expectation of high parallelism of operations with minimal locks.

About speaker:
Sergey Olontsev (Microsoft SQL Server MVP, Microsoft Certified Master) started working seriously with Microsoft SQL Server more than 7 years ago, and during this time he had to deal with a very wide range of tasks. Predominantly specializes in administration, ETL solutions, performance optimization, diagnostics and solving all kinds of problems.
