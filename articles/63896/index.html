Save on GPRS traffic using Opera Mini Proxy

Not so long ago, I was to travel on business to a neighboring region. And, since working affairs demanded constant use of the Internet, I had to think about how to save on expensive GPRS roaming traffic (UPD: the method is also suitable for downloading prona without the knowledge of parents bypassing the limitations of corporate firewalls).

The first thing that came to mind (thanks, Habr) - use Opera Turbo . Let me remind you briefly: Turbo is a new beta version of Opera browser that allows you to drive traffic through their servers, which compress html and repack the images in jpeg with a high degree of compression (in addition, flash-videos are replaced with their screenshots with the ability to download the original by clicking mouse), while maintaining all the markup of the document, and all the scripts work as expected.

Using Opera Turbo allows you to reduce the cost of traffic by 2-3 times (experimentally verified on viewing Wikipedia pages). Good? Good but not enough. Let's try to squeeze even more.

The next step was an attempt to use Opera Mini on the desktop . As you know, opera mini also works through the server‚Äôs compressing traffic, but this time the html pages are repackaged into the internal format of the opera mini obml , the markup changes ‚Äúfor a mobile device‚Äù, and javascript works only selectively (images are also compressed).

The traffic saving is much more tangible, compression is about 8 times (in the same example with Wikipedia). Still, the mini interface is not intended for desktop use. Is it possible to solve this problem? There is.

The feature is called Opera Mini Proxy (or Opera Mini Proxy Daemon, ompd). This program is a local proxy server, which is written in the browser settings and sends all traffic through the Opera Mini servers, decoding the obml returned by them into the usual html on the fly.

Details:

- Run - ompd.exe, it does not give anything and just remains to hang "in the background", servicing requests.
- All settings - via the web interface, localhost: 8080 (the same parameters must be specified in the browser, as the address and port of the proxy server).

Important settings:

- First, on the settings page, click ‚ÄúCreate new OM Account‚Äù (after entering the desired username), and select the created account as a working one in the ‚ÄúOM account (cookies, etc.)‚Äù field.
- In the field "HTML page width" set to the width of the physical screen or the width of the window
browser.
- In the field ‚Äúmemory size in kb‚Äù we put 6000 (this affects how the server will
divide the page into blocks loaded 1 time. In 6000 home page Habra with
pictures placed without cutting).
- In the field ‚Äúscreen width‚Äù set 180 (more = more beautiful, but the pictures
proportionally will be more, and compression is less. In my opinion - 180 in
just right).
- ‚ÄúOM protocol version (2.xx or 3.xx)‚Äù I set to 3, because faster places and

Another important point is the work with pictures (setting ‚Äúimage quality‚Äù):

"Text" - without pictures.
"Low / high" - with pictures, determines the degree of compression.
Under the picture, the program usually adds a direct link to the original picture, but this link does not always work (for example, sometimes it doesn‚Äôt work in Wikipedia).

Everything, now you can use your favorite browser, compressing data through Opera Mini servers. I remind you that compression is approximately 8 times that in money equivalent (according to MTS daily tariffs) - about 7,500 rubles per gigabyte.
