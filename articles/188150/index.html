Proper adaptive alignment of the site header

Often, seemingly simple layout tasks require a complex structure of HTML markup and the use of CSS tricks. Centering items or aligning content can be very tedious. One of these tasks is alignment of the elements of the upper part of the site so that the logo is on the left and menu items are on the right. You can use float and position: absolute, and for vertical alignment you can add margin and padding to different elements. It seems to be nothing complicated. But if the site should be correctly displayed on mobile devices, many problems arise.



The following is a concise way to solve this problem.

HTML markup is as simple as possible: 

<header> <h1>Super Bad</h1> <nav><a>First Link</a><a>Second Link</a><a>Third Link</a></nav> </header>


The height of the header is fixed, add text-align: justify , for child elements: 

 header { text-align: justify; letter-spacing: 1px; height: 8em; padding: 2em 10%; background: #2c3e50; color: #fff; }


Add display: inline-block for all nav elements so that you can arrange them one after another: 

 header h1, header nav { display: inline-block; }


In order for the text-align: justify attribute to work as we want, you need to use a small trick with pseudo-elements, which was found in the article Perfectly justified CSS grid technique using inline-block by Jelmer de Maat: 

 header::after { content: ''; display: inline-block; width: 100%; }




The result was a horizontal alignment, without using float and position: absolute . Now you need to align the elements vertically. When using vertical-align, for nav elements there will be a dependence on the height of the parent block - the header. And this is not very correct. Examples of using vertical-align: top and vertical-align: middle on jsbin. The following is perhaps the most convenient method of vertical alignment.

Use pseudo-elements again. using the example from the article Centering in the Unknown , mentioned by Micha≈Ç Czernow: 

 header h1 { height: 100%; } header h1::before { content: ''; display: inline-block; vertical-align: middle; height: 100%; }

The result is what you need:



It remains to solve two problems: correct display with a large amount of text in the header and adaptability. If the site title is too long, the layout will start moving down:



Use the pseudo-element trick on the header :
CSS code 
 header { text-align: justify; height: 15em; padding: 2em 5%; background: #2c3e50; color: #fff; } header::after { content: ''; display: inline-block; width: 100%; } header > div, header nav, header div h1 { display: inline-block; vertical-align: middle; } header > div { width: 50%; height: 100%; text-align: left; } header > div::before { content: ''; display: inline-block; vertical-align: middle; height: 100%; }



It looks much better:



We now turn to adaptability. There are several ways to solve this problem, you can simply not set the height of the cap, and all the internal elements will be adaptive to the height. It does not require a second pseudo-element trick, a live example on jsbin.
CSS code 
 header { text-align: justify; padding: 2em 5%; background: #2c3e50; color: #fff; } header::after { content: ''; display: inline-block; width: 100%; } header h1, header nav { display: inline-block; vertical-align: middle; } header h1 { width: 50%; text-align: left; padding-top: 0.5em; } header nav { padding-top: 1em; }



If you need to set the height of the header, you will have to use the second pseudo-element trick, and add a media query for screens of different sizes: 

 @media screen and (max-width: 820px){ header { height: auto; } header > div, header > div h1, header nav { height: auto; width: auto; display: block; text-align: center; } }


The result is adaptive and on mobile devices looks like this:
