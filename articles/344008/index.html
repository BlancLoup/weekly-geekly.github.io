Let's look at the convolution formula. But first, what do we want to see in the formula, what should it reflect? Let's turn to Wikipedia :
‚ÄúIn the convolutional neural network, only a limited matrix of small weights is used in the convolution operation, which is‚Äú moved ‚Äùacross the entire processed layer (at the very beginning, directly in the input image), forming an activation signal for the neuron of the next layer with a similar position after each shift. That is, for different neurons of the output layer, the same weights matrix is ‚Äã‚Äãused, which is also called the convolution kernel ... Then the next layer resulting from the convolution operation with such weights matrix shows the presence of this feature in the processed layer and its coordinates, forming the so-called feature map (English feature map). ‚Äù

So the convolution formula should show the ‚Äúmotion‚Äù of the nucleus. wl by input image or feature map yl‚àí1 . This is what the following formula shows:

xijl= suma=‚àí infty+ infty sumb=‚àí infty+ inftywabl cdoty(i cdotsa)(j cdotsb)l‚àí1+bl qquad foralli in(0,...,N) enspace forallj in(0,...,M)

Here subscripts i , j , a , b Are the indices of the elements in the matrices, and s - the size of the step of convolution (stride).

Superscripts l and l‚àí1 - These are indices of network layers.
xl‚àí1 - the output of some previous function, or the input image of the network
yl‚àí1 - this xl‚àí1 after passing the activation function (for example, relu or sigmoid; the item about the activation function will be a little later)
wl - convolution kernel
bl - bias or offset (absent in the picture above)
xl - The result of the operation of convolution. That is, operations take place separately for each element. ij matrices xl whose dimension (N,M) .

Having dealt with the parameter designations and understanding the basic principle of the convolution operation (the details of the convolution formula will become clear towards the end of the article, so you may need to come back here later), you can go on to the important nuance of this formula and the operation itself: the core has a central element . Moreover, the central element may not necessarily be in the center (where is the center of the matrix two by two pixels?), But in general in any cell of the core - which one you want. For example, the central element of the kernel in the animation above is at position (1.1), and the relative convolution operation of this element occurs. So, what is the central element?

Central element of the core


So, the indexing of core elements occurs depending on the location of the central element. In fact, the central element defines the beginning of the ‚Äúcoordinate axis‚Äù of the convolution kernel. Look at the picture below, on the left, the core with the central element in the zero row and zero column, on the right - in the first row and first column:
See how indexing has changed? And yes, I say that in the second core, the central element ‚Äúmoved‚Äù to the first row of the first column, meaning that this happened relative to the numbering of the first core. In fact, the indices of the central element are always equal (0,0), and for convenience of further exposition, I will say the ‚Äúold‚Äù coordinates, speaking of the position of the central element in the upper left corner, and the ‚Äúnew‚Äù coordinates - about the position of the central element in any elsewhere in the kernel, using the numbering of the ‚Äúold‚Äù coordinate grid (for example, (1,1) for the right convolution kernel in the figure above).

But, going back to the convolution formula, what does it mean - the sum from minus infinity to plus infinity? After all, the core itself has quite specific dimensions and it does not have an infinite number of elements. I found different spellings of the formula, for example,  sumu=‚àíaa sumv=‚àíbb ( here and here ). I also found variants with infinity, as in the variant at the beginning of the article ( here and here ). But the latter seemed to me a more ‚Äúcommon‚Äù case.

The minus in the convolution kernel formula is a consequence of the location of the central element. We should ‚Äúgo through‚Äù all possible existing elements, and we can start from minus infinity. Or from minus a and b . If an element by these indices is not defined for a given kernel, then multiplication occurs by zero, and in fact operations start not from minus infinity, but from the position of the central element multiplied by minus one (in the numbering of the ‚Äúold‚Äù coordinates). And the operation will end not at plus infinity, but at the difference between the number of core elements along the axis under consideration minus the index of the central element (again, in the numbering of the ‚Äúold‚Äù coordinate axis). And the last value of the resulting range is not included (since the indexation is from zero).

In words, it may sound hard, and probably best to see how this can be done in python. Below is the code for calculating the indices for the ‚Äúnew‚Äù axis of the core, depending on the selected central element:

I will cite once again the convolution formula:

Activation functions


Below are the formulas of activation functions that can be used in a future model. In fact, this is just a ‚Äútransformation‚Äù xl at yl in this way: yl=f(xl) . The activation function allows you to make the network non-linear , and if we did not use the activation functions (then it would turn out that yl=xl ) or would use a linear function, then it does not matter how many layers there would be in the network: they could all be replaced with a single layer with a linear activation function.

So, ReLU:

fReLU=max(0,x)

And sigmoid:

fsigmoid= frac11+e‚àíx


A sigmoid is used only if there are no more than two classes (for the classification task): the output of the model will be a number from zero (first class) to one (second class). For a larger number of classes, in order for the model output to reflect the probability of these classes (and the sum of the probabilities over the network outputs equals one), softmax is used. The function looks simple, but there will be certain difficulties in calculating the formula for backprop.

fsoftmax=yil=f(xil)= fracexil sumk=0nexkl

Where n - this is the number of classes.

Loss function


The final stage of the network is a function that assesses the quality of work of the entire model. The loss function is at the very end, after all layers of the network. It can look like this:

E= sumi=0n frac12(yitruth‚àíyil)2

Where n Is the number of classes yl - model output, and ytruth - right answers.

 frac12 here it is only needed to reduce the formula during backpropagation of an error over the network. Can be removed  frac12 and nothing fundamentally changes.
After reading this article I decided to use cross-entropy:

E=‚àí sumi=0nyitruthln(yil)



The structure of the future model


Now, having analyzed the main layers of the network, we can present an approximate view of the future model:

  1. A function that extracts the following image / batch from the dataset for training;
  2. The first layer of the convolutional network, which accepts an image as an input, sends an attribute map to the output;
  3. The makspool layer, which reduces the dimension of feature maps;
  4. The second layer of the convolutional network accepts the maps obtained at the previous step, and at the output gives other feature maps;
  5. Adding the maps obtained in the previous step to one vector;
  6. The first layer of a fully meshed network takes a vector, performs calculations that give values ‚Äã‚Äãfor the hidden fully meshed layer;
  7. The second layer of a fully connected network, the number of output neurons of which is equal to the number of classes in the dataset used;
  8. The output of the entire model is fed to the loss function, which compares the predicted value with the true value, and calculates the difference between these values.

The resulting loss function is a kind of quantitative ‚Äúfine,‚Äù which can be viewed as a measure of the quality of the prediction model. We will use this value for teaching the model using backpropagation - back propagation of an error. The formulas that use this error and ‚Äúpull‚Äù it through all the layers to update the parameters and train the model will be covered in the next part of the article.
