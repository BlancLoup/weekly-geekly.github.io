News and Releases


  • PHP 7.4.0 alpha 1 - The first alpha version of the new PHP branch has been released! Ahead of the semi-annual testing cycle , the final release is expected in late November.

    The number of new features is impressive: typed properties arrow functions , FFI, preloading and much more! A list of all changes to the UPGRADING document.
    The review of new features can be viewed in habraprevod New in PHP 7.4 or in Nikita Popov's report on PHP Russia .
  • DevConf 2019 (Moscow, June 21) - Jubilee Tenth DevConf. By tradition, reports are selected by vote. Look at the podcast Five minutes PHP number 54 .

Php internals


  • [RFC] Allow throwing exceptions from __toString () - Offer accepted unanimously.
  • [RFC] Numeric Literal Separator - The sentence crossed the threshold on voting, and therefore in PHP 7.4 you can use the underscore symbol as a separator in the numbers: 

    $i = 197_823_459; // 197823459
  • [RFC] Nullsafe Calls - The old proposal is again discussed in the internals. It considers the possibility of adding a new operator ?-> , which would allow making safe chains of method calls, in the case where the return value may be null : 

     $o?->mayFail1()?->mayFail2()?->mayFail3()?->mayFail4();

    There is a similar proposal in ECMAScript , and in Hack this feature is already implemented .
  • [RFC] Alternative "use" syntax for Closures - The author of the proposal considers the use syntax in closures not very convenient and offers to transfer it to the function body:

    Hidden text 
     //   $closure = function ( ArgumentType $argument1, ArgumentType $argument2, ArgumentType $argument3, ArgumentType $argument4 ) use ($importVariable1, &$importVariable2, $importVariable3, &$importVariable4): ReturnType { // ... }; //    $closure = function ( ArgumentType $argument1, ArgumentType $argument2, ArgumentType $argument3, ArgumentType $argument4 ): ReturnType { use $importVariable1, &$importVariable2; use $importVariable3, &$importVariable4; // ... };
