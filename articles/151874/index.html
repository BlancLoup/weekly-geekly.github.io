KYOCERA made friends with the MFP with Android smartphones



As befits a decent modern company, Kyocera has developed its own Android application - ‚ÄúKYOCERA Mobile Print‚Äù (Google Play). I did not really pay attention to him, because I do not use the smartphone at work so much , and the very fact of the existence of software is not surprising - having mobile applications in the arsenal has now become a necessity. But, recently and by chance, I discovered that the application not only prints - it can scan. Such a ‚Äúdiscovery‚Äù changed the alignment, because On a smartphone, I often photograph documents (several pages per day on average), occasionally I have problems with poor readability of the resulting images, and therefore, the appeared alternative to the built-in camera immediately translated my indifferent attitude to the program into an interested one.



Under the cut - a short story with pictures about ¬´KYOCERA Mobile Print¬ª.



Installing from Google Play is standard, after it we see a quite informative icon in the list of applications:



When starting, we get to the main menu, with a list of available operations:
  • Photo print
  • Printing of documents
  • Print Web Pages
  • Scanning

The application is not Russified, but the hacker's instinct will help to easily deal with the buttons.

I will demonstrate with the function of scanning and subsequent printing, commenting on the actions. I will try to do this briefly, and in the end I will summarize the subjective result.

So:



If you select "Scan" - the program will ask you to specify the device. As you can see, you can search for available devices on the network, or specify the IP directly. Previously used devices are stored in the Recently-list (a hint that the word "usability" is not alien to the developers).



The grid found KYOCERA TASKalfa 3500i (this is our guest Wi-Fi network, with a guest multifunctional device)

For scanning I use the booklet:



The next window is the scan device settings:



The program remembers them for the future. The list is quite large:
  • File name
  • File type (JPEG, PDF)
  • Orentation
  • Scan position
  • Original size (Letter to B6)
  • Quality (Text, Photo, mix)
  • Color (Full, Grayscale, B & W)
  • Resolution (200-600 dpi)
  • Job accounting (on-off)
  • User login (on-off)

The scanned image is immediately displayed on the screen of the phone, and you can save it, or cancel it if something is wrong.



Now you need to print the result. Select a print from the main menu and specify the source of the file to print:



(Notice that the snapshot from the camera can be immediately sent to print?)

Print settings are also mass, and they are also remembered:
  • Copies
  • Orentation
  • Duplex
  • Color (or B & W)
  • Print size (Letter to B6)
  • Pages per sheet
  • Print quality
  • Page range
  • Staple
  • Punch
  • Job accounting (On-Off)
  • User login (On-Off)

In the end - I get the right print (do not pay attention to the b / w - TASKalfa 3500i does not print in color)



I did not encounter any complexities, everything works quickly, and the interface is intuitive.

I will linger on an additional point: for printing the web, the program has implemented its own browser, for me it is a rather original solution. It is fully functional, supports Flash and JavaScript:



An open browser page can be sent to print to a printer, or sent via e-mail. I will clarify - sending by e-mail will open the send menu from Android, and there you will be able to choose from your applications not only e-mail, but also something else suitable (Evernote, for example, or Skype)



Perhaps the process is all.

What we have in the end:

I really liked the app. Even trying to put myself in the place of a person who is not associated with the company - I see no reason for quibbles. The speed of work, interface, design - everything is at the height and corresponds to the solid name Kyocera. The functionality covers basic user requirements. Printing the web with its own built-in browser is great, I don‚Äôt know how elegant it is in terms of programming, but the solution works.
