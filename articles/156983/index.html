AWS: The second generation of standard instances and the next price reduction!

Hi habrasoobschestvo

So in order.

The second generation of standard instances on Amazon EC2 / VPC


The list of available servers on Amazon EC2 / VPC has new names M3 Extra Large Instance and M3 Double Extra Large Instance . These are 2 new types of standard instances. Why standard? Because all standard instances have a balance of memory - CPU. There is no significant advantage in one direction or another, as for example in High Memory or High CPU types.

So what kind of cars does AWS offer? I ask under the cat.

M3 Extra Large Instance

  • 15 GiB memory
  • 13 EC2 Compute Units (4 virtual cores with 3.25 EC2 Compute Units each)
  • EBS storage only
  • 64-bit platform
  • I / O Performance: Moderate
  • EBS-Optimized Available: No
  • API name: m3.xlarge

M3 Double Extra Large Instance

  • 30 GiB memory
  • 26 EC2 Compute Units (8 virtual cores with 3.25 EC2 Compute Units each)
  • EBS storage only
  • 64-bit platform
  • I / O Performance: High
  • EBS-Optimized Available: No
  • API name: m3.2xlarge

Yes, quite good shapes. New instances will allow even more flexibility to build the system architecture and more efficiently spend money on it. Learn more about these and other types here: Amazon EC2 Instance Types .

Price reduction


And once again, AWS again lowers prices in Virginia and Oregon. On average, prices fell by 19% affected types:
  • M1 Small Instance
  • M1 Small Instance
  • M1 Large Instance
  • M1 Extra Large Instance

Personally, but in my experience, these types of instasos are used as often as possible, and these regions are among the most attractive in the price category and proximity to the main market - the USA. Read more about pricing here: Amazon EC2 Pricing .

I am very happy to repeat again and again that AWS is the flagship of the market not only in terms of the functionality of cloud services, but also in price policy.
