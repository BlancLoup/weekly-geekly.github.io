1 - Do not forget the `var` keyword when you assign a value to a variable

The assignment of a variable that was not previously declared will result in this variable being created in global space. Avoid this.

2 - Use `===` instead of `==`:
Using `== (! =)` Performs type conversion if necessary. While `===` does not do this conversion. The
the operator compares not only the values, but also the type that resolves faster than `==`. Examples: 
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ [10] === 10 // is false [10] == 10 // is true '10' == 10 // is true '10' === 10 // is false [] == 0 // is true [] === 0 // is false '' == false // is true but true == "a" is false '' === false // is false ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


3 - `undefined, null, 0, false, NaN, ''`:
All of the above will always be a lie in logical expressions.

4 - Do not forget `;`:
Using `;` to complete expressions is good practice. You will not be warned if you miss it, as the JavaScript parser will insert it for you.

As in the previous example with an empty array, to truncate the array, use the `length` property:

25 - Floating point problem: 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 0.1 + 0.2 === 0.3 // is false 9007199254740992 + 1 // is equal to 9007199254740992 9007199254740992 + 2 // is equal to 9007199254740994 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Why it happens? 0.1 + 0.2 equals 0.30000000000000004. What you need to know is that all numbers in JavaScript with a floating point inside are represented as 64 bits in binary form according to the IEEE 754 standard.
If you want more examples, check out this blog .
You can use the `toFixed ()` and `toPrecision ()` methods to avoid such problems.

32 - Avoid the `eval ()` or `Function` constructor:
Using the `eval` or` Function` constructor is an expensive operation since every time they cause the source code to be converted into javascript engine. 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var func1 = new Function(functionCode); var func2 = eval(functionCode); ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


33 - Avoid `with` (Good stuff):
Using `with` defines a variable in global space. If any variable has the same name, then it will create certain problems, as overwriting the value may occur.

34 - Avoid for-in looping for arrays:
Instead‚Ä¶ 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var sum = 0; for (var i in arrayNumbers) { sum += arrayNumbers[i]; } ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

... this is better ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var sum = 0; for (var i = 0, len = arrayNumbers.length; i < len; i++) { sum += arrayNumbers[i]; } ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The bonus assignment for `i` and` len` will be made only once since this is the first expression of the loop construction. It is faster than ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ for (var i = 0; i < arrayNumbers.length; i++) ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Why? The length of the array `arrayNumbers` is calculated at each iteration of the loop.

35 - Pass functions, not strings for `setTimeout ()` and `setInterval ()`:
If you pass strings to `setTimeout ()` or `setInterval ()`, they will be processed in the same way as with `eval`, this is slow.

Instead of using ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ setInterval('doSomethingPeriodically()', 1000); setTimeOut('doSomethingAfterFiveSeconds()', 5000); ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

... use ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ setInterval(doSomethingPeriodically, 1000); setTimeOut(doSomethingAfterFiveSeconds, 5000); ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


36 - Use `switch / case` instead of a series of` if / else`:
Using `switch / case` works quickly when there are more than two conditions, it is more elegant and better for organizing code. Avoid this
expressions when there are more than 10 conditional expressions.

Using a `switch / case` expression with a numerical range makes the following trick possible.

40 - Avoid using `try-catch-finally` inside loops:
`try-catch-finally` creates a new variable in the current execution space each time an exception is thrown and it is associated with the variable.

Instead of this‚Ä¶

... use it ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var object = ['foo', 'bar'], i; try { for (i = 0, len = object.length; i <len; i++) { // do something that throws an exception } } catch (e) { // handle exception } ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

You may lose the connection if XHR takes too long (for example, network problems), use `setTimeout ()` with calls
Xhr. 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var xhr = new XMLHttpRequest (); xhr.onreadystatechange = function () { if (this.readyState == 4) { clearTimeout(timeout); // do something with response data } } var timeout = setTimeout( function () { xhr.abort(); // call error callback }, 60*1000 /* timeout after a minute */ ); xhr.open('GET', url, true); xhr.send(); ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

As a bonus, you should avoid synchronous Ajax requests.

42 - Deal with WebSocket timeout
Normally, when a connection is established, the server can reset the connection by timeout after 30 seconds of inactivity. The firewall may also, after a certain period, reset the connection on timeout due to inactivity.
This problem can be solved by sending empty messages to the server. To do this, you should add two functions to your code: one to maintain the connection and the other to cancel this support.
Using this trick, you will manage the timeout.

Adding `timerID` ... 
 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ var timerID = 0; function keepAlive() { var timeout = 15000; if (webSocket.readyState == webSocket.OPEN) { webSocket.send(''); } timerId = setTimeout(keepAlive, timeout); } function cancelKeepAlive() { if (timerId) { cancelTimeout(timerId); } } ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

`keepAlive ()` should be added to the end of the `onOpen ()` method for webSocket
connections and `cancelKeepAlive ()` to the end of the `onClose ()` method.
