Data protection from non-physical damage. Rollback rx

Preface / Disclaimer


I recently met the Rollback RX software and was extremely surprised by the lack of at least some informative description of its work in Russian, I want to try to change this situation and talk about the program, its capabilities, advantages, disadvantages and principle of operation. About the program


What is he doing

It protects data from the snapshot from being changed, that is, it makes snapshots of the section and allows you to return to it at any time. Also allows you to mount any snapshot as a virtual disk to extract individual data without rolling back to it. Saves from viruses, playful hands and an inquiring mind.

How does

In fact, the program becomes a layer between the hard disk and the OS, keeps the content indexes current at the time of the snapshot creation and prohibits changing this content. Due to this principle of operation, pictures are taken almost instantly and do not perform unnecessary operations with data.

Reasoning


Benefits

- The speed of creating a snapshot (in fact, the program simply copies the index and locks the data for overwriting)
- A small overhead charge both on productivity and on memory (data are not duplicated)
- Theoretical independence of images from exposure from the system

disadvantages

- It makes impossible the low-level backup of the protected partition (the same acronym)
- Keeps snapshots only on the same section from which they are made
- May cause uneven wear of the carrier when storing multiple images at once.

Vague moments

- It is not clear whether it will, in principle, work under Win 10, the version adapted for it is promised only in January
- I didn‚Äôt work with raids and scsi disks before, I don‚Äôt know how the picture is now
- Reliability: on the one hand, the principle itself seems reliable, but its concrete implementation 4 years ago confused one security specialist
- It is not very clear how well the program works with SSD and how much it can accelerate wear when storing multiple shots

Tips / thoughts

- If there is such an opportunity, then any temporary files should be transferred to an unprotected partition, especially with regard to the paging file, and it is probably better to simply disable hibernation (although it is not so critical when storing a small number of images)
- Also the problem of uneven wear can help solve the storage of the minimum number of shots
- The program does not create miracles and use it on partitions with a constant flow of files (for example, on those partitions where you download torrents and where you constantly rub them from) it makes no sense
- The protection of the system partition and the storage of 2 snapshots of it: the original and last
- Do not forget that the program does not backup your data to external media, which means it does not save from physical influences and it is worth keeping some other software for this case
