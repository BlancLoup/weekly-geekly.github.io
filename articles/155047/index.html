Which one would like to see Sailfish OS from Jolla?

After the announcement that Jolla will present the interface and the SDK of its new Sailfish operating system on November 21, changes have occurred in the company's management. Its CEO Jussi Hurmola left his post to focus directly on the development of a new operating system, and Mark Dillon, who was one of the chief engineers of the Maemo & MeeGo project, and also held several key positions at Nokia at different times: Configuration Management at Nokia, S40 CoreSW Integrator at Nokia, Symbian Build Manager / Team Leader at Nokia, Configuration Manager at Nokia.



Waiting for the introduction of OS Sailfish, I tried to formulate my user wishes for the functionality and interface of the new OSes. The list is clearly not exhaustive and certainly subjective. I expect that the comments will substantially complement or correct it.



1. So that the user himself can assign the mode of operation of this or that application in the background.

2. So that the user decides whether he will have an open or closed system (in fact, this is already implemented in MeeGo Harmattan, I would just like this functionality to be preserved).

3. For the device to work for at least 20 hours with maximum load (at the same time, I personally do not care how thick it will be - within reasonable limits, of course).

4. For OSes to support applications for all existing OSes - on the principle of virtual machines, for example, the hardware part already allows this. Application support from the Android Play Market, it seems to be already announced.

5. So that the kinetic recharge is implemented in the OS, that is, that the smartphone is recharged from oscillation, while it is in its pocket, for example.

6. So that the interface of the OS does not require Hadwarn control buttons on the device (that is, in this sense, that a certain continuity with respect to N9 is preserved).

7. For OSes to support the x86 architecture, that is, so that it, or its derivatives, can be installed equally on smartphones and tablets, and on laptops and desktops.

8. So that OS Sailfish firmwares are released for dozens of flagship devices for Android from different manufacturers. In general, this development of OS Sailfish, in my opinion, should be one of the main ones.

9. So that the interface implements not only full-fledged multitasking, but also multi-window mode, which implies opening several applications on one desktop.

10 In order for OS to implement deep integration with one (or several) of cloud services, allowing to synchronize any data.
