Microsoft StorSimple virtual array. Part 2

In our previous post Microsoft Azure StorSimple: easy access to the hybrid cloud. Part 1, we told you why Microsoft StorSimple is the best solution for those who are just starting to work with hybrid cloud storage. Companies that have entered into an Enterprise Agreement with the ability to use Microsoft Azure, were able to access the virtual version in March: the Microsoft StorSimple virtual array. It can also be used in a VMware and Hyper-V environment. We offer you a small excursion from the question-answer series on this solution.

For whom is the StorSimple virtual array the ideal solution?


Microsoft's StorSimple virtual array is an excellent solution for companies operating in a distributed IT infrastructure, which must always remain flexible. In many companies, IT departments seek to control the backup and recovery process, as well as perform disaster recovery centrally. Most often, long-term investment in hardware is not the optimal solution, especially if the company is at the stage of dynamic growth. This is where the StorSimple virtual array comes into play.

Its benefits are obvious. This solution is:
  • adaptive
  • cost effective
  • extremely flexible
  • easy to manage

And:
  • provides a number of key features and applications, including work in the Microsoft Azure cloud environment.


What does this solution look like from a technical point of view?


First of all, it must be remembered that, unlike its older brother, the StorSimple hardware device, the StorSimple virtual array is not only available for the iSCSI protocol, but also works with the SMB protocol. This means that it can easily start working with already existing file services, and at the same time provide exactly the same advantages as its predecessor.

Any DFS methodologies that the company applies are easily combined with the StorSimple virtual array.
When combined with the Microsoft Azure cloud environment, the StorSimple virtual array offers additional features:

  • Electronic storage for backup Backup Vault, integrated into Azure, makes it possible to back up and restore data stored in the virtual array StorSimple
  • In case of an accident, the transition to the backup array can be made by simply pressing the key.
  • If necessary, this procedure can be performed automatically by using the Azure Automation service, if required.
  • The following functions performed by the elder brother are also available in this virtual array:
    • Local Volume Binding
    • Authorization
    • Encryption
  • Connections are established exclusively according to the ‚Äúfrom internal to external‚Äù scheme, which the IT security staff will undoubtedly appreciate.

Requirements


The StorSimple virtual array only requires the following virtualization platforms:
  • Hyper-V (2008, 2012, 2012 R2) or
  • VMware (5.5 and later)
In addition, the VM must meet the following performance requirements:
  • 8 GB RAM
  • 4 cores
  • 500 GB of local disk space for the operating system and cache

What architecture can I use?


Azure blob storage is an essential element of the architecture. In order to create volumes in the container storage, as well as the volume of the volume containers themselves and the volumes there, users need to register an Azure storage account. The storage account sets up redundancy settings (for example, geo-redundant storage (GRS), locally redundant storage (LRS)) in order to determine the availability of BLOB storage. The virtual array uses this storage to store the appropriate data blocks.

The StorSimple management tool is the main management tool used to manage all StorSimple installations. It can only be accessed through a web browser, thus eliminating the need to install any local controls.



Microsoft Azure cloud also provides other high-performance services. Thus, data that is currently stored in the cloud will be constantly available. The data is stored in the so-called cloud images in the Azure Cloud cloud environment, which provides access to data using a virtual device. This feature is extremely convenient if the data needs to be processed in another application, for example, in Hadoop.

Possible use cases


The StorSimple virtual array is mainly used as a file solution in branch offices of companies that do not have their own IT staff in the field. This environment cannot function without the central control provided by the StorSimple control. There is no reason not to use an additional VM along with the virtual array if the virtualized branch server is already in place.

Summarize


Microsoft's virtual array is an incredibly simple solution. Moreover, the StorSimple virtual array is a hybrid solution that miraculously serves as a springboard for the cloud environment.
