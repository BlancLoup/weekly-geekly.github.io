Yandex.Search for a site: good news for site owners based on CMS

Earlier we wrote about pinger plugins for popular CMS, which allow indexing a site on demand.

The plugin installed in the CMS monitors changes on the site and generates indexing requests when creating or modifying documents. This allows you to make the search results on the site more ‚Äúfresh‚Äù, that is, to promptly index new documents. We remind you that indexing on demand as part of the Yandex.Search service for a site improves only the search on the site and does not affect the indexing of the site by the main Yandex robot.

Today, the list is complemented by a plugin for the TYPO3 control system. Total additions exist for 5 popular CMS: Wordpress, Joomla !, Drupal, ModX, TYPO3.
You can download plugins and read instructions on this page , but now this is not the only way.

Today it is possible to download plugins already configured for your search form. Simply select the desired CMS from the list and get the already customized distribution. This is convenient: it remains only to download the distribution file and enable the plugin in the content management system settings.



For those who need more IP addresses to configure additional indexing, it is now possible to specify up to 10 IP addresses in the plugin settings. You can specify addresses as IPv4, and IPv6.
