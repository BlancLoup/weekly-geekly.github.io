rescue initrd in cloud selective

News in one line: a rescue initrd was added to the list of available external kernels when booting the virtual machine, which contains tools for recovering the virtual machine.

In case of configuration errors (not the kernel, remote partition table, strange file system manipulations, forgotten password, errors in boot script configurations, etc.) we added an initrd with a more or less full shell (full set of busybox), plus a set of utilities to restore / repair the machine.

Rescue initrd is available in the list of external cores for download.



Initrd content


  • Full busybox (including network management, dd, wget, etc.)
  • bash (with all due respect to busybox, it‚Äôs not very comfortable as a shell)
  • testdisk - partition recovery
  • photorec - recover files from badly damaged filesystems (different types, not just photos, the name is not the most successful)
  • full curl
  • vi and nano
  • Utilities for resize and create ext2 / 3/4, vfat, reiser
  • All utilities for working with lvm
  • fdisk


Network


Important point: when booting into rescue initrd, nothing happens automatically (including network configuration). In connection with the ban on the use of foreign IP addresses, you can register any address, but only the one that is allowed for the machine will work (indicated in the tab header with the virtual machine).

This can be done either by the old method via ifconfig, or via ip. Just in case: the mask is 255.255.255.0, the gateway is the IP address of the virtual machine, whose last digit is replaced by .1.

Transition to the virtual machine file system


The infiltrate-root / dev / mapper / vmXXXX_system-root command will allow you to be in the file system bypassing the login process and not running any scripts from it. Particularly useful with a forgotten password, a randomly made chmod 777 -R / etc / * (do not do this!), Corrupted / etc / inittab, etc.
