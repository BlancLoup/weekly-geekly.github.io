Sun Fire V100 and Solaris 10 11/06

I have always been more concerned about SUN products, but attempts to put Solaris 10 11/06 make me more and more doubt their responsibility.

And so try the number one setup over the network. To install over the network I used two articles www.znark.com/tech/solarisinstall.html and www.sun.com/bigadmin/content/submitted/setup_dhcp.jsp

After some manipulations, Solaris began to load. But ... when loading reached cpu and on this the case was stalled. Although at the same time, Spark continued to turn to linux via nfs, but it was no further. Well, I thought it might be a combination of the linux 2.4.x kernel and nfs-utils. I put a similar bundle on another server already with the 2.6.x core. It became even funnier. The system has stopped finding the kernel. And it stopped mounting the file system. This was followed by a second attempt.

Attempt number two. Install from DVD. Since the drive in the Spark CD and runs on the phase of the moon, I took a regular DVD drive. And here miracles begin. The drive is recognized by the system, but there is no disk with Solaris 10 11/06, it says there are no tags and that's it.

Well, now the actual question. Is everything so good with Solaris? And why practically everything that can be sharpened only under the Sun environment and the debug output is so small?

UPD: Made openboot find the DVD drive. But however not trivial.
