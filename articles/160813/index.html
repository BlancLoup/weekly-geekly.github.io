Repost Board: run nanoCAD free 3.5 / 3.7 under Linux using Wine

Usually we do not repost other people's messages, but this is an exception to the rules ...


We are so often asked if nanoCAD will work under Linux, that we automatically have maximum interest for this message on the forum :-) So when the forum received a message from the user with the nickname PITRider with practical tips on how to run nanoCAD under Linux, we decided to share it in the ‚Äúas is‚Äù mode with the maximum number of readers.

PITRider writes:
‚ÄúTo work in nanoCAD 3.5 or 3.7 under WINE, the following is required:
  1. WINE 1.4.1 version (stable) - under 1.5 it works crookedly (broken icons and non-working file dialogs!)
  2. The architecture of the bottle WINE win32 - relevant for OS x86_64. To create such a bottle you need to run once: WINEARCH=win32 winecfg
  3. Install via winetricks dotnet35 (or dotnet35sp1), if with your hands you need to delete the mono from the registry ... winetricks does it itself
  4. Install ODBC Driver - jet40 mdac28

My sequence of actions in 2 lines:
WINEARCH=win32 WINEPREFIX=/path/to/wine winetricks -q dotnet35sp1 jet40 mdac28
WINEPREFIX=/path/to/wine wine /path/to/NC37.exe
