x3650 M4 - do you need HD?

The x3650 M4 server has long been and still remains an IBM sales hit. It seemed that in the segment of small 2-processor servers no radical changes were foreseen, never change the winning team. But the ‚Äúbig brother‚Äù x3650M4, a model with an HD (High Density) index, entered the market, and questions immediately arose. New advantages for comparable money or is it still the best enemy of the good?

Basic model and HD index


Obviously, it is necessary to compare x3650M4 HD, first of all, with the base model. The differences are clearly visible in the table compiled according to IBM specifications at the beginning of December 2013.



The list of characteristics in the table is not complete, but it is justified. The main difference is in the storage subsystem.

Storage: flexible configuration


Actually, x3650M4 HD is called a storage server, which is not surprising, given the opportunity to increase storage capacity to 41.6 TB against 25 TB from the ‚Äúyounger brother‚Äù. Compared to the base model, the number of hard drive bays has dramatically increased in the new server: 24 2.5 ‚Äùdrive bays are located on the front panel.



Alternatively, you can install 16 HDD and 16 SSD. And in terms of configuration flexibility, this is a very valuable feature. Here it is already possible to structure storage resources on the principle of fast / capacious.



This is the server with 16 installed 2.5 ‚Äùhard drives and 16 1.8‚Äù solid-state drives - this configuration of the storage subsystem, if you use car jargon, can be called ‚Äúfull stuffing‚Äù. And all drives available from the front panel support hot swapping.



Another 2 compartments are located on the back side of the server. An optional RAID adapter allows the system to boot from these two disks, which helps to completely isolate the OS from production data.



RAID


Serious storage subsystem - so you can expect different from the standard set of options for RAID. Indeed, in addition to the integrated (without occupying a PCI slot) RAID level 0/1, it is possible to use the hardware ServeRAID 5210e and increase the data transfer rate from 6 Gbit / s to 12 Gbit / s. Standard with ServeRAID 5210e updates are available to levels:
- RAID 5/50 without cache
- RAID 5/50 1GB with cache
- RAID 5/50 4GB (expected in Q1 2014) / 2GB / 1GB with flash

In addition, with the FoD feature (Feature on Demand), an upgrade to RAID level 6/60 is available.

Independent zone cooling


As soon as the system offers the opportunity to increase the number of disks and increase the possible load level, the question of cooling immediately arises. Agree, in the 3650 M4 HD, this question should be particularly acute. The server uses a 2-zone cooling system: 4 fans operate independently of each other, serving, relatively speaking, 2 temperature corridors, separate for each processor.



Here again, can not do without the automotive analogy. Imagine a dual-zone climate control that provides optimal cooling in different parts of the cabin. How much more efficient is this air conditioning system! And, of course, all 4 fans support hot swapping.

Control


In the administration of the system here without surprises. We can say that the set of tools is traditional, customary and proven to be effective. IBM Systems Director and IMM2 (Integrated Management Module). In combination with the traditional Light Path Diagnostics LED diagnostics panel in System x servers and the PFA failure prediction system.



Application scenarios


If the model is x3650 M4, positioned as a server to support cloud environments, virtualization, applications for teamwork and, of course, infrastructure applications, then x3650 M4 HD, in addition, is suitable for serious work with large amounts of data. And primarily to support business intelligence and data extraction, transformation and loading (ETL) operations at high speeds, which requires both significant computing power and large storage capacity.

To whom such a combination of characteristics can be beneficial? It is hardly possible to somehow limit the range of potential interested parties. Not only government agencies that have to work with huge amounts of data, large corporations and their branches, but also companies from the SMB sector, which today are increasingly working with their SQL databases in virtualized and cloud environments. In addition, a powerful server with a serious storage subsystem will be a good solution for working with video. The volume of video data in corporate environments is growing rapidly. The recordings of surveillance cameras, instructional videos, video presentations, commercials - all this is not only for large enterprises, but also for companies that are rather modest in scale.

Summing up all the possibilities offered by the x3650 M4 HD server, we can confidently say that it will be at least as popular as the base model. Very good computing power and scalable storage subsystem - all in a 2U chassis.
