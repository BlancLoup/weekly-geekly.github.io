Part One - Advertising



Imagine that you no longer need to wait for the compilation. Imagine that your application retains its state, and after compilation it does not restart, but the code that you wrote will be updated in the running application. Imagine that you save your code in your favorite editor and right there, instantly, your code is already in a running application. Imagine that you saved the skin of the interface of your application in Photoshop and right there, in a split second, it was updated on the screen of the running application. Imagine that you once installed and launched an application on iPad in development mode, and the application on the iPad is updated directly on the device in real time. The fact that you can create as many copies of the working application on different computers and devices as the whole team can test at the same time? Submitted? Great? Dreamed about it? And, drumming, it happened, it all became a reality. Meet the COLT!


COLT. Code Orchestra Livecoding Tool. This is a small program that tracks changes in the resources of your application (code, pictures, sound, etc.) and, at a file change event, runs a special, incremental compilation that will create a small file that contains only the changes you made (in differences from the usual practice, when the Flash / AIR application collects everything again). And will deliver to the working application.

You can change files from IDE, you can take changes from version control system. You can simply ‚Äúshare‚Äù the folder for the designer, give the network address of your application, and the designer will live to change the files in the folder and see the changes right away.

You can track code changes through event subscriptions and create new libraries, approaches and revolutionary tools for game and application development. You will definitely have new ideas on how to improve your work!

Never has flash / air development been so fast and productive. Feel the new buzz from flash and mobile development with COLT.
