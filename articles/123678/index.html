New Firefox Aurora 7.0a2 - reduced memory consumption, improved performance and a bit small things



A new version of Firefox Aurora 7.0a2 arrived . And with such changes that they can not be told.

The first, and, in my opinion, the most important thing is to reduce memory consumption by 30% or more , improving responsiveness and speeding up the launch. Removed the leaks in the JS engine, still conjured, and got this magical result.

I just recently my laptop for the sake of FF upgraded from 2 + 1 Gb of memory to 2 + 2 Gb. Because eclipse drove ~ 700mb steadily, FF 5.0.2 ate from 0.7 to 1.2Gb (it could have been more, but I restarted it to help it ‚Äúcleanse the soul‚Äù). And if you run also VirtualBox, in general, you could drain the water.

Now FF for several hours fit into 500Mb. T.ch. 30% may be true. Although there is one nuance - all the extensions fell back, and they could affect memory. But as I remember from the experiments, there seemed to be little influence.

Of the other improvements:
text-overflow: ellipsis; - FF is now able to cut the text delivering at the end of "...", as if hinting that it is longer than it might seem. Cheers-cheers-cheers.

Acceleration JS parser is a trifle, but nice.

Navigation Timing - you can detect how long a particular user action has taken on the page (such as navigationStart, connectStart / End, responseStart / End, domLoading / Complete). Potential statistics of the speed of the site at the client.

There are changes in the DOM - non-standard methods and properties for File are removed, such as getAsBinary, FileSize, etc.

In the address bar, the ‚Äúhttp: //‚Äù prefix is ‚Äã‚Äãno longer displayed. Not familiar, but logical.

Firefox Sync now instantly updates bookmarks and passwords.

If you already use FF Aurora - within a day or two you will receive an offer to upgrade. If not, but you want to - switch the channel to Windows, add a PPA to Ubuntu, in Mac you also see the channel.

Update . Firebug is not compatible yet, even 1.8b5 does not work under the new FF yet. But I think this problem will be solved in a day or two.

Update2. Thank you lafayette : Firebug 1.8b5 works on 7.0. Just install the Add-on Compatibility Reporter plugin.

Update3. Thanks Yakof :

One of the most visually noticeable changes in the seventh version is the rendering of small fonts, including the system one for Win7 Segoi UI using GDI Classic instead of DirectWite. So, goodbye blurry fonts!

In order for this to work correctly, the ffonts for which GDI Classic will be used (Arial, Consolas, Courier New, Microsoft Sans Serif, Segoe UI, Tahoma, Trebuchet MS, Verdana by default), and the parameter gfx.font_rendering.cleartype_params.force_gdi_classic_max_size indicate the size of the font to which Direct Write is disabled (default 15).

The second noticeable update is the inclusion of the new Azure graphics processing engine, which gives a really serious increase in graphics tests (although it only works in Win Vista and 7). You can check, for example, at the same SpeedReading - my fox confidently goes around even IE, under which these tests were actually sharpened

In Ubunt, I did not notice any changes associated with rendering fonts. I am happy for the winery.
