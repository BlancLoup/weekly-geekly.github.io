Excursion from version 1.6.1 to 1.6.5

Much time has passed since the publication of the news about the release of Google App Engine 1.6.0 . And by the way there were 5 updates, each of which brought a lot of tasty and interesting. I will try to fill this gap.


The description contains basic and big changes and does not contain a list of fixed bugs and other implementation details, otherwise the article would have turned out to be just huge. Full list here .

Version 1.6.1

Release date: December 13, 2011

Changes to the platform:
  • Appeared the so-called Frontend Instance Classes . Now, by default, your scripts are launched not on a pseudo-machine, but on a pseudo-machine, more resembling an instance from Amazon. You can (for some money) by default for your scripts "create" more powerful instances;
  • High Replication Datastore (HRD) Migration Tool has been released . This application, as the name suggests, allows you to transfer data to HRD via a web interface, controlling the migration time, the migration date, etc. Those. fully manage the migration process.

New APIs:
  • The Conversion API is released , which allows you to convert different formats into each other;
  • Logs Reader API is released which allows you to process and visualize logs.


Version 1.6.2

Release date: January 31, 2012

Experimental implementation of backups and restoration of them for Datastore . An extremely convenient innovation that eliminated the need to write your migration scripts with serialization and deserialization.

At last Django in App Engine was friends with Cloud SQL.

The Channel API has learned to limit the lifetime of a connection key.

And Python API for the Blobstore has learned how to make asynchronous calls for all types of operations on data.

Version 1.6.3

Release date: February 28, 2012

Changes to the platform:
  • Appeared Traffic Splitting , which allows you to decide what percentage of traffic to which version of your application will go. Separation can be done by looking more closely at what user has come and redirecting him either by IP or by cookie. Thus, for different countries you can define different versions of your application;
  • Added protection against spoofing via e-mail. Now all emails that are sent either from user accounts or google apps for domain are automatically signed with a DKIM signature .


Changes in the control panel:
In addition to some changes in pricing, you can now:
  • Manually shut down instances directly in the control panel;
  • Logs Viewer now contains a link to the instance that served the request. Of course, while the instance is active.


Version 1.6.4

Release date: March 27, 2012

A lot of everything :)
  • The lifetime of the Logs can now be determined independently;
  • Now in the control panel you can see the statistics on Datastore indexes ;
  • Now you can migrate Blobstore to tools for HRD;
  • Datastore backups can now be made directly to Google Cloud Storage ;
  • Memcache viewer appeared , which allows you to see the statistics in general and check some record by setting its key in particular;
  • Work with objects from Google Cloud Storage as if they were Blobstore objects;
  • NDB for Python has been released.


After 2 days Go is released, and its full support in App Engine will appear with the next version.

Version 1.6.5

Release date: April 24, 2012

  • Datastore has learned how to perform these requests:
    SELECT Property1, Property2 FROM MyEntity ORDER BY Property3 ;
  • Entity group (group of records) received integer (autoincrementable) identifiers;
  • Datastore Admin now allows you to restore individual records from backups;
  • The decorator @ db.transactional (Python) now supports concurrent transactions;
  • Images API accessed Google Cloud Storage;
  • Appeared Request Headers (the most delicious!), Which are geographically oriented. So far, only Region, City and LatLng headers .
