Gears will no longer be in Chrome

Google finally decided to put an end to its Gears plugin. The search giant finally decided to remove this feature from their Chrome browser. Aaron Bugman, a member of the Gears team, said in a blog post that there will be no new releases for Gears. Gears will be removed from Chrome in version 12. It should be noted that Gears was launched by Google with a lot of hype in 2007. One of the main features of Gears was its ability to get web applications running offline or when network connectivity is too slow.



Most web browsers already provide Gears functionality.


It should be noted that the functionality that Gears has provided is now available in most web browsers. In addition, Google has implemented a cache of applications that will allow the launch of a web application, even if the user is not online. In addition, the browser architecture has changed a lot since 2007.

Google, using its own Chrome browser, which has gained immense popularity in the last couple of years, has gained the platform they need to create their own web-based functionality on the market. Google also quickly came out with several versions of the browser. Hints at the end of Gears began as early as December 2009, when Google talked about its preference for the HTML5 standard over Gears. With an API for geolocation, notifications, and several other features in Chrome, the importance of Gears has greatly diminished.
