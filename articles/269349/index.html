New animation tabs in the assembly Vivaldi 1.0.303.23

Hello!

Today we posted another build in which several important changes were made. First of all, we have significantly reworked the code responsible for dragging tabs. In addition, we have done a lot of work with thumbnail pages for bookmarks and Express panels. Immediately answer a possible question. No, these changes were not made since the last build - by today, the code of these functions turned out to be ready for public testing, which we propose to do. Now - a little more about tabs.

Improved tabbed browsing


This feature is almost completely redesigned. Now all problems associated with dragging tabs should become a thing of the past. Additionally, we improved the animation, which, if desired, is disabled in the settings. Of course, there are still some minor shortcomings (read about this in the section on known problems), but in general, it seems to us, everything looks and works quite well.

Sketches of bookmarks and express panels

Here is a small improvement: if you do not have a thumbnail of a page in a tab or on the Express panel, then it will be created automatically when you open this link.

Known Issues

On some weak machines, the animation of the tabs is slow. It is recommended to disable this feature.
When laterally placing multiple tabs, their size is too small.
Scrollbars appear on the tab bar when the browser window is resized in Mac 10.11.
