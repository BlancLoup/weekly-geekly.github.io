YouTube on torrents

P2P YouTube - a new extension from Torrent Stream , for browsers Google Chrome; Mozilla Firefox and Opera, which makes it possible to play videos on YouTube through torrents!

In addition to the main designated function, this extension provides other features that may also be useful and interesting.

Want to know more about features and expansion options?

Description of features and capabilities:

Bittorrent accelerator is the full bittorrent protocol support for any video you are watching on You Tube website. All videos on YouTube you can now watch through torrents and receive a video stream not only from one of the remote YouTube servers, but from all users who have watched this video or are watching it through this extension. This will provide you with a more stable and reliable stream! The principle of operation is one - the more users will use this solution, the more efficiently it will work and everyone will have a stable stream at the maximum speed for its Internet channel!

The libvlc-based web player (an excellent alternative to Flash!) - now you can watch YouTube videos through the world's most functional web player. / imho Learn more: How to control the player .

Attention! When you start a playlist, autoplay of all the clips it contains will occur regardless of whether this option is turned on or off.

And now, some technical details about how streaming via torrents on YouTube works:

  • After the first viewing of the video by any of the users, a torrent file is created that is attached to the url of the video page
  • When this video is launched by another user, the playback will already go directly through the torrent file, which also additionally includes http-sid with the server support address going directly from YouTube (all this happens on the fly and in the background).
  • Support for the stream from other peers occurs both during the video playback by them and after it, while the user stores this data in the cache. This is a very significant advantage over the RTMFP protocol, which is still more suitable for live broadcasts, allowing you to support the stream from other users only at the time of viewing, limited to the amount of data of a certain time range.
