It is somehow strange to see poor html in posts on an IT resource, when the entire progressive world uses various wiki-like markup. What blog to ask, I do not know, so here I ask. Do you plan to introduce any Markdown for articles, plus a program code highlighter? html is clearly redundant for this purpose.

Update based on a comment below. The meaning of semantic markup of the text (and the wiki markup is often just that) is that at least two hares are killed immediately:

* first, semantic markup allows you to generate anything from the text, and html requires additional efforts, for example, with a global design change (to put it simply, we immediately get anything from the markup, but from html, we will not get anything at all)
* secondly, the markup allows you to use a single style for all posts: indents, quotes, lists - _all_ looks uniform.
* thirdly, the markup is easy to read and easy to write, as an example you can see the comment editor at [Stack Overflow] (stackoverflow.com)
* fourthly, textual markup makes it easy to compare versions of documents, for habr it is irrelevant, but nonetheless.
