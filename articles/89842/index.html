About multitasking and copy-paste


I look at the modern market of mobile operating systems and it becomes a little sad. All forces rush into creating beautiful interfaces, others create thousands of unnecessary applications, and sometimes they forget about the implementation of essential functions of the first vital necessity. More precisely do not realize consciously. Of these functions, I can notice multitasking and copy-paste, or rather its implementation and the presence of a clipboard in general.

Recently I learned that after giving up normal multitasking in Windows Mobile 7, they also refused the clipboard, thereby preventing the user from copying / pasting anything between applications. Here is the question: WHY IS SUCH A COMMUNICATOR?


In the iPhone OS there is no multitasking, I don‚Äôt know whether they will copy it. In Android OS, there are both multitasking and copy-paste, but which ones !!! Before buying HTC Hero, I read a lot of information about the implementation of multitasking in Android. I immediately realized that there would be no familiar implementation (like Alt + Tab), but on the forums they talked almost about the revolution, they said that it was hard to describe it, I had to try it myself. In general, Android "reasonably" manages the RAM of the device in which it is placed. Android itself decides which processes have the honor and privilege to be started, which should be paused, and which should be unloaded. In fact, everything is very sad and not convenient. Being in the application, you never know what will happen to him after clicking on the house. A long clamping of the house (in Symbian it shows running applications) displays the last used applications. That is, even if you left the application, when the house is clamped, it will be displayed. And to see exactly the running applications, you need to install a third-party task manager. By the way, the task manager does not intercept the long clamping of the house, it is necessary to open it every time as a normal program. Kopipast implemented even worse, about him, I already wrote in a note about the cons of Android.

So today we have the following picture: these two very important functions are correctly implemented only in Symbian OS, which I see every day hundreds of times. But on the approach of Symbian ^ 3. I really hope that there will not be the largest bummer!
