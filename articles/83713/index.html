Version 1.0 released

Those who are following the development of the Open Source project Web Optimizer (which will soon be a year) are already up to date with the events of the past few days - a stable version 1.0 of the application is installed to automatically speed up websites - WEBO Site SpeedUp .

Download the latest version: www.webogroup.com/en/home/download

Under the cut - some details about the new version and plans for the near future.

New interface


Regarding past versions of the Web Optimizer application (0.6.7 and earlier), the interface was created from scratch. Now all the work with the application is carried out through the control panel.


Significantly redesigned application configuration mechanism. First of all, debugging mode became available (when WEBO Site SpeedUp works only if there is a corresponding Cookie or GET parameter on the way). Secondly, configurations (groups of settings) are introduced, which can be switched or even transferred from site to site.


The settings themselves remained virtually unchanged.


And thirdly, a part of the system settings (paths, access to the site) are transferred to the system state (the page is usually loaded for a long time in connection with checking all the system capabilities)

Here (as in the corresponding block on the Control Panel), a list of comments on hosting (mainly non-working Apache modules) is displayed. A lot of questions have already arisen on this list - we will refine the hint system for each item and will soon include it in the interface.

The cache also has its own page.


You can try the new interface in action at the online demo: www.webogroup.com/ru/home/demo

We conducted preliminary usability testing, part of the improvements in its results will be added to the product. If you have any suggestions about the product interface, you can write them in the comments or contact us .

Licensing policy


WEBO Site SpeedUp is now available in three versions.
  • Non-commercial version. Can be used without restriction on non-profit-making (non-profit) sites. The settings are enough for the initial acceleration of the site, but there are a number of limitations on performance.
  • Light version. Designed for commercial sites that do not need the functionality of the full version, but the acceleration of the client part is required. The performance of this version is slightly higher than that of non-commercial.
  • Full version. All settings and maximum performance.

If you are not sure whether the full version is suitable for you, we can order a trial key for it for several days. Write us for this .

The current version of WEBO Site SpeedUp is a new version of the Web Optimizer application. Therefore, if you have a license key for Web Optimizer, then it will be suitable for WEBO Site SpeedUp.

Future plans


A number of our users understand the purpose of the product - speeding up the client part of the site - and use it for its intended purpose. But for most, site acceleration is server optimization. In this regard, effective and flexible server caching mechanisms (for WordPress and Joomla!) Will be added in the next versions, which will work faster than their counterparts and provide more flexible configuration (for example, deleting pages from the cache when sending comments).

For all other CMS, server caching is already enabled in a separate web application, which can significantly reduce server load.

In addition, of course, we are introducing and we will implement all user improvements announced in the official bug tracker . If you have rational suggestions, add them to this list.

Performance studies


Some time ago, an online performance optimization blog on WordPress, located on a simple shared hosting - www.speedingupwebsite.com was launched . Based on its results, we will most likely improve both the WEBO Site SpeedUp functionality and develop a checklist for such blogs: what is critical and what is not from the point of view of such optimization.

In the future, we plan to expand this series of articles on Joomla! and Drupal.

We will also continue to publish successful experience in implementing performance optimization technologies, which lead to an increase in site efficiency (for example, the history of www.halfwaytoconcord.com and www.hworld.ru ).
