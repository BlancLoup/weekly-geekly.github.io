Runnable: search engine code with its execution in the VM



A former Amazon employee launched a search engine on the source code Runnable.com , which has a unique feature: it not only searches for code, but also executes it in the EC2 virtual machine right in the search results. Moreover, you can personally make changes to the code - and run it again to view the results.

Finding the right source code fragment is one of the main tasks that arises during project development. Why write for the second time what has already been written before us? This logic is well understood in large corporations: both Google, Amazon, and Yandex oblige employees to publish all the code they write on the corporate network to avoid duplicates.

But simply publishing the code for everyone does not solve the problem, because this code is too much. In the same "Yandex", the developers still continue to do two or three versions of the same library (according to bobuk 'a), what can we say about larger companies.

For example, the Ohloh search engine provides more than 20.6 billion lines of publicly available code, and the Programmable Web search engine has more than 10,000 publicly available APIs.

The Runnable search engine allows you to more quickly understand all this wealth, testing PHP, JavaScript, Node.js and Ruby on Rails code right there on the spot. It all runs in the Amazon cloud.
