Jolla Releases SailfishOS SDK for MOSOS and Windows

Last week, Jolla released the SailfishOS SDK for MACOS and Windows. From the beginning of March, the SDK was available for Linux, on the basis of which porting applications from other mobile platforms to SailfishOS had already begun. In this topic, I decided to collect the most interesting videos related to SailfishOS and the development for it, and also asked me to share my opinion on the SDK SailfishOS developer of the popular twitter client for MeeGo Harmattan.

I asked my opinion on the SDK SailfishOS to share the developer of the popular Rocket twitter client for MeeGo Harmattan, Ural Khamitov . In addition to the MeeGo Harmattan SDK, he worked in the SDK for Symbian 9.x, QtSDK for Symbian ^ 3, Android SDK, Cascades from BlackBerry. I asked him a few questions:

- What are the distinguishing features of the SailfishOS SDK?
Sailfish SDK, like Nokia QtSDK, uses QtCreator as its development environment. The only difference is in the emulator. Nokia had a QEMU brake emulator and a simulator that didn‚Äôt do much. In Sailfish, VirtualBox is used for emulation.

- How many applications can be launched simultaneously?
I launched 9 empty ‚ÄúHelloWorld‚Äù, when starting the 10th the emulator starts to slow down wildly. Checked three times, the result is the same. Perhaps this will be fixed, or the restriction of 9 simultaneously running applications will remain.

- How many functional elements are available in a minimized form of the application?
There are no restrictions on the number, but the fact is that the icons go in a row and if there are more than 4 of them, then we get such a mess that it will be difficult to get on the icon.

- What's up with the browser? it was not shown in any demo video, and in the theme for the MiGow flashed the Internet Explorer icon?
The emulator has just one demo application in which various UI elements are presented. There are no more programs.

- If you convert Rocket, how difficult is it, how long will it take?
At a quick glance I appreciated that the UI will have to be rewritten from scratch, but in the core, which in C ++ will need to be fixed, not very much.

- Can I start developing mobile apps with the SailfishOS SDK? Do I need preparation for development?
I would not recommend starting development with Sailfish for several reasons. Firstly, there are no devices for testing, and secondly, the SDK is still very raw and there is very little documentation, we can say that it is not there at all.

- What functionality is expected on the notification screen? Is integration possible like Rocket in MeeGo?
There is a certain Feed, an analogue from MeeGo, but there it is empty. I think that the integration of its applications will be possible, but this is only a guess.

- How intuitive did you find the interface? Is Harmattan clearer for a simple user or is it really the same?
Talking about the interface early, but I think you can get used quickly.

- What are the advantages of the OS itself, you would note, both from the point of view of the developer, and from the point of view of the user?
Talking about it is also very early, as such, the OS is not there yet. This is just an empty emulator, with the ability to run your programs. No more.
