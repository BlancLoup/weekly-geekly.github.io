Customization and buns

Everything is configured intuitively through the web interface. We connect the lanyard to the LAN port, go to 192.168.0.1 and ...

To begin with, in Network-> WAN / LAN Number, we indicate the number of WAN interfaces. I put "for the future" all 4 at once!
We press Save and after some thought and restart - we have 4 WAN interfaces! The hamster is dancing in joy and dancing to the jig, the Toad, who was trying to choke, waving his paw and walking away.

On Network-> WAN - select the port number of the WAN, insert the cord from the provider and drive in the settings. The settings are minimal.
As a minus - you can not call the interfaces in its own way for the memory where what provider.
We save, select the next WAN port, insert the cord of the next provider, drive in the settings ...

If the provider is paranoid and it links the account to the MAC address - there is a MAC Clone page: you can set your MAC for each of the WAN interfaces, if you no longer need it, you can return a specific interface to your native MAC. And what is brilliant is the ‚ÄúYour PC's MAC Address‚Äù field, the ‚ÄúClone You Mac to ...‚Äù button and the drop-down list for 4 WAN: no need to search and copy with pens!

On the page Load Balance - you can set the balancing on the machine or manually, as a percentage. It also shows the current load on the channels.

DDNS is also not forgotten: it is presented here by as many as 4 providers, including the eternal DynDNS.
And the beauty is that you can set separately for each WAN port !!!
