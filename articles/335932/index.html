Test Management for JIRA and its application in software development



At the start of the project, many ask the question "Where to keep test documentation?" .
In this article, I would like to tell you about the Test Management for JIRA tool (in this case, the plug-in is considered as a tool) in order to conveniently, efficiently and efficiently build and manage a test design project.

Why should we pay attention to this tool? Because he:

- Conveniently installed, supported
Installation takes place through the section managing plugins in JIRA

- There is a connection with JIRA
Yes, at first this will not surprise you, but the trick is that it comes as a plug-in for JIRA and does not need to be separately integrated with other test management tools.

How will this improve my work?

The analyst creates the task - the tester attaches the test cases to the task - the developer, without authorization to any test management system, immediately sees the test case and the steps to it that he needs to go through before transferring the functionality to the test.

This will not only improve your work, but also speed it up.

- Beautiful, clear reports in the form of charts
Not only testers can make reports, but the whole team.

- Adding and deleting your status to the test case, plans, wounds and results :)
- Drawing a check sheet to the task of developing
- Prioritization and label distribution in each test case
It will help, for example, when regressing to find all the necessary tests for label - regress

- Adding environments
If necessary, run a specific test case on a specific environment

- Creating a common test case with a description of the tests and attaching it to the necessary scenarios
It is used when the result can be obtained in different ways.

- Ability to integrate with continuous integration system
Automatic fixation of a successfully or unsuccessfully passed test. Used through REST API .
