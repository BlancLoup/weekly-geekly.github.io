NGINX 1.8 and 1.9 release

We are pleased to announce the release of new releases of NGINX 1.8 for a stable (stable) branch and NGINX 1.9 for the main (mainline) branch, within which the development of new functionality will continue. New release branches replace 1.6 and 1.7 respectively.

Over the last year, 1.7 versions were released in 12 versions, and all the new features that appeared in them are now available in 1.8, including:

  • SSL verification of backend certificates - It is designed to protect connections to upstream servers from MITM attacks ;
  • Logging to syslog - Allows you to transfer logs over the network via UDP messages;
  • Thread pools for performing blocking I / O operations - Increase NGINX performance in situations where locking on disk reading is unavoidable;
  • New load balancing method ‚Äúhash‚Äù - Allows you to select a server from a group based on the hash of an arbitrary variable;
  • Unbuffered reception of the request body - Allows you to send a request to the backend immediately, without waiting for the end of receiving the request body entirely.

A complete list of innovations over the past year can be found in the change log for 1.8.

Numbering Features


Renumbering branches is a key point of our annual process, when the next stable branch (functionality is frozen) branches off from the main branch (development of new functionality). A few weeks ago there was a separation of 1.8 from the main as a new stable, after which the main 1.7 changed the version to 1.9. We no longer support 1.6 (the previous stable branch) and do not develop anything in 1.7.

In the understanding of NGINX, stable (1.8) means that the feature set does not change and only fixes critical errors fall into this thread. Stable does not necessarily mean greater reliability and fewer errors. In fact, the main branch (1.9) is considered as more reliable, since it includes a much larger number of fixes, and not just critical ones, which later also flow into the stable one. On the other hand, changes in the stable branch are not likely to affect any third-party modules, whereas in the main this can happen when adding new functionality.

What to expect from 1.9?


Among the first features that were added in 1.9 are: TCP Balancing, allowing users to load balance any applications running on top of TCP, as well as support upstream blocks in shared memory, which improves the consistency of balancing and identifying problems between multiple workflows. Later this year, support for the HTTP / 2 protocol and dynamic loading of modules is also planned. But this is not all, a lot of work is being done and many other great opportunities can be expected in the framework of the new branch.

What about NGINX Plus?


NGINX Plus is a commercial version of NGINX with enhanced functionality . NGINX Plus is based on the main branch and usually new releases are released quarterly . New features from the main branch flow into NGINX Plus and are released after thorough internal testing, as well as baptism in the framework of the main branch releases.
