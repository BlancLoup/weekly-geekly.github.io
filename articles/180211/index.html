Mageia 3 released



Mageia is a Linux distribution, founded in 2010 by former Mandriva employees. The distribution was created as a fork Mandriva and has much in common with its ancestor. Unlike Mandriva, this is a non-profit distribution and is developed by the community. Distribution is in second place in popularity on the site distrowatch.com.

The main working environment is KDE, but there are versions with GNOME. There is a graphical configurator that allows you to configure the system without having to go to the console. For fans to play, too, there is something.

New versions of the distribution are released about once every 9 months. For those who need rolling-release - there is a Cauldron branch that is being developed, which is constantly updated with new versions of programs.

Today came the third version of this distribution. The issue is dedicated to the memory of Evgeny Dodonov.

Innovations


Let's see what the developers have prepared for us a new one.

Basic system:

  • Moving programs from the root directory to / usr. The practice of placing part of the system utilities has historically developed, for example, utilities were needed to mount the same / usr. Now it is no longer relevant, because / usr is mounted using intramfs and there is no need to keep some utilities in the root. Read more .
  • The kernel has been updated to version 3.8.
  • Added test support for GRUB2, the default is the first version of GRUB.
  • Systemd version 195
  • Reworked package groups in the program installer.
  • For packets, XZ compression is used.
  • The program installer now shows a general progress indicator for the installation.
  • Updated logo, wallpaper and theme.

Additional Information


Distribution repository is divided into three groups:

  • Core - free software
  • Non-free - non-free software
  • Tainted - free software that may have problems with patents (codecs, etc.)


By default, only Core is enabled. You can connect the rest and update the system, after which all necessary codecs will be installed, more complete versions of some libraries will be installed, etc.
