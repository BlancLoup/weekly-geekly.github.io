News and Releases


  • PHP Specification - Despite its popularity and already a solid age, PHP has never had a formal specification. But fortunately, times are changing and the HHVM team has presented to the public the draft PHP specification . The document is based on PHP 5.6 syntax and does not include a SPL description. Now subsequent versions of the interpreter, PHP-NG, as well as alternative implementations like HHVM can provide full compatibility. The specification itself will become an integral part of the language development process.
    Not without a little criticism .
  • The next PHP version will be called PHP 7 - Voting on the controversial issue ended with a result of 24 against 58. Some people believe that PHP 7 can be confused with PH7 , an alternative implementation of the interpreter. And of course, a great post from Phill Sturgeon about the endless controversy that has now become history.
  • PHP 5.6.0RC3 - This is the third release candidate for the new version of PHP and another one is planned. The final release should be expected by September.
  • PHP releases 5.4.31 and 5.5.15 - Updates on current branches.
  • HHVM 3.2 "Outkast"

Php


  • RFC: Abstract Syntax Tree - Another massive offer from Nikita Popov. It is proposed to use an abstract syntax tree as an intermediate structure during the compilation process. Such a solution will, on the one hand, make the parser and compiler more comprehensible and easy to maintain, and on the other hand, will make it possible to implement previously impossible syntaxes, such as [$a, $b, $c] = $array instead of list() . As for performance, this solution is slightly faster, but will require more memory.
