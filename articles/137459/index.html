PHP deobfuscation


It just so happened that the other day I got one PHP script I needed, but obfuscated .
For some reason, it did not work at all. I have been writing in PHP for a long time, and I really like to catch non-standard situations in scripts, especially when there are no errors in the logs, and the script simply does not fulfill its intended duties, it‚Äôs itching to decipher. The last straw was the fact that the author of this script was not online to help solve my problem. The script, by the way, was bought by my friend, in fact, he asked for help.

The purpose of this article is to show the principle of decryption of encrypted scripts, for example, to decipher an intruder-filled virus in PHP .



Code analysis



Here is one of these scripts , opening such a script in Notepad, we will see:


To begin with, we divide it into lines:

and save to file encoded_script.php

It can be seen that the code is divided into 4 parts. In each, eval is executed.
For the convenience of decoding eval, we use the PHP extension, Evalhook. It was written by Stefan Esser for which he thanks a lot. With it, you can decrypt in several passes, for example, monsters encoded via www.php-crypt.com with eval, gzuncompress, base64_decode, etc. functions.

Evalhook installation



Required:
  1. PHP> = v5.2
  2. php-devel
  3. PHP Zend Optimizer


Download the source archive 
fetch http://php-security.org/downloads/evalhook-0.1.tar.gz


Unpack and build an extension for PHP 
 tar xvfz evalhook-0.1.tar.gz cd evalhook phpize ./configure make sudo make install


The extension is ready, now you can dynamically load it into the console and watch the encrypted script. 
 php -d extension=evalhook.so encoded_script.php


where encoded_script.php is an encoded file.
To get the source code, simply press ‚ÄúY‚Äù several times during the deobfuscation process. Each new Y will decode the next eval. Well, arm the console and fight!
