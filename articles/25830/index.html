Update Windows XP and user rights.

Now investigated an interesting situation about which I report to the habrosocommunity. The purpose of this note is to find out about the occurrence of similar situations, and, of course, to warn the reader.
On one of the working machines, Windows XP was updated, it was updated firmly from the local update server (all updates, in theory, are checked on a test machine, but then something was watched). After the required reboot, the problem appeared: all domain users who have administrator rights on this machine lost these rights. Local administrators remained intact.

Going under the local account, rolled back the updates, returned the rights, began to look for the cause. That is, of course, it was clear that the reason for some of the updates - but which of the several dozen established? I installed updates one by one, overloaded the car and watched what was going on.
The result was somewhat paradoxical: the situation with the removal of rights was repeated after the update 816093 (Security Update for the Microsoft Virtual Machine). It surprised me because it has been updated for several years now, and there have never been such problems before.
Therefore, I conclude: this update may cause problems not on its own, but in conjunction with any update that was released the other day.
Has anyone encountered anything like this lately?
