Application Testing with Fuzzing

Everyone who develops in php, and indeed in any other programming language with fuzzy typing, faced a similar problem: 
PHP Notice: Undefined index: test in /var/www/Testing.php on line 171 PHP Notice: Undefined index: test2 in /var/www/Testing.php on line 171

Or even this: 
 PHP Fatal error: Cannot use object of type stdClass as array in /var/www/Testing.php on line 171


In the course of developing one large project, I had to start testing written methods for correctly applied parameters.
In my case, I tested the class and the contract for it, but the ideology of testing is the same.
Suppose there is a function that is used to taking an input parameter array, to process some elements.
For example: 
 PHP Warning: mysql_affected_rows() expects parameter 1 to be resource, string given in /var/www/Testing.php on line 177

I wrote a simple class that helps in testing functions, it generates random data, starting from int by pumping null.
Here is his code:
