Start beta testing Handy.CMS 3.1 for developers

Start testing beta version of content management system Handy.CMS 3.1 for developers. Register online: handycms.ru/forum
One of the advantages of the system for developers is the possibility of developing their own modules through the built-in functions of the system.



Features Handy.CMS 3.1


Handy.CMS is a harmony of convenience and functionality.

  • AJAX-based administrative interface
  • Caching not only the public, but also the administrative part of the site, which significantly speeds up the work
  • Expanded system of differentiation of access rights not only to modules, but also to individual elements
  • Hierarchical Administrator Management
  • Flexible work with a template engine - the ability to install any template engine (default template template Smarty)
  • Ability for developers to create their own system modules using Handy.CMS API
  • SEO-pack: CNC (when developing modules, an external developer can set the address format himself), meta & keywords
  • Ability to work with the system through a system of special labels and macros
  • The ability to install the system on the design of any complexity, the system does not put restrictions
  • Ability to create, move, edit and delete sections of any nesting level
  • Ability to create different language versions of the system
  • Database backup system
  • Automatic update system

Requirements for system operation:
- OS 32-bit
- PHP 5.2.1 and higher
- Apache
- MySQL 4 and above
- PHP modules: Curl, GD2, simpleXML
- Zend Optimizer 3.3+
