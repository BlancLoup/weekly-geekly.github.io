GoblinX 3.0

Many people know that GNOME is not included in the Slackware distribution. There are a number of ways to use slabar and sit under a gnome ‚Äî for example, take some third-party builds ( GSB , GWare or Dolphine, and so on) or try to build a gnome yourself, using, for example, GARNOME, as the GNOME developers themselves recommend on gnome.org .

Dance with a tambourine, of course, a delightful occupation, and give a sense of self-confidence in the event of their successful completion, but what about housewives who are accustomed to the gnome and want to sit under slackberry? They can use one of the GoblinX distribution options ( project page ). Release 3.0 was just released recently.

GoblinX is positioned primarily as live-cd, that is, does not require installation on the hard disk. Available in four variations: g.Standart - with KDE as the desktop environment, g.Noblin - with GNOME, g.Mini - with Xfce and g.Micro - with Fluxbox.

Country of origin - Brazil.

I downloaded the g.Noblin image and ran it on the virtual machine to look. You can choose different boot options - just a gnome, a console, and a netbook interface. tty1 is beautifully designed, but for some reason you need to enter a password for root, although it is right there indicated.



By the way, an interesting observation - if you start the gnome from the console from under the root with the gog command, then it works for some reason faster than if you start the gnome right away by selecting the appropriate menu item when booting :) In the distribution, like in any other liv-sd , crammed a huge number of packages for all occasions. And painted beautifully, do not say anything.



But there is no office. I was interested in the possibility of installing GoblinX on a hard disk. For these purposes, you can use the included installer with a graphical interface.



The interface slowed down and slowly rendered. The installer copied the entire bunch of packages included in the distribution kit, including itself, onto the hard drive, and at the same time did, although I did not ask, the user goblinx, under which he logs in by default.
There is a graphical package manager Gslapt.
