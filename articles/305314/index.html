It seems that Github is again unavailable in Russia ... and other sites



On July 7, 2016, many Github users from Russia began to complain about interruptions in access to the site.

Here is a list of some problem addresses:
  • raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
  • 151.101.36.133
  • help.github.com
  • github.io
  • pages.github.com
  • gist.github.com


Presumably, the roots of the problem grow from IP 151.101.36.133 - this is the address of the content delivery network for Github. This address is included in the blocking list by the decision of the Oktyabrsky District Court of Stavropol.


The way to solve the problem is to register an alternative IP address in the hosts (there are several Github ones, only one is blocked).
UPD . The problem affects not only the resources on the github domains, but also others, for example docs.botframework.com/en-us
