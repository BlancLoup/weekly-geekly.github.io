Skype 4.0 Beta 2 for Windows - criticism was not in vain

This is already the second news on the topic today, but here I will try to describe in more detail the changes that have affected Skype 4.0 in the transition from the first beta version to the second. The most important thing that causes respect is the reaction of the developers to the opinions of users. After all, the new interface has caused a real storm of protests and wishes. The main ones were taken into account in this release.

Actually news from Skypeclub.ru :

‚ÄúOn October 1, Skype released the second beta version of the Skype 4.0 client (release 4.0.0.166), which took into account the numerous suggestions of users received after the release of Skype 4.0 Beta 1 .

The main user complaints after the release of Skype 4.0 Beta 1 were presented to the new user interface of the program, which became much wider and did not allow displaying the main Skype window as a familiar contact list. The second beta makes it easy to switch the new interface to the familiar display, which is now called Compact view or ‚ÄúCompact view mode‚Äù . The work of the client in compact mode is practically no different from the usual Skype 3.8 and earlier versions and allows you to position the window in a narrow strip with the edge of the screen.

The second important change affected notifications of incoming chat messages and other Skype events. Now they should become more visible, and skipping a new message will not be as easy as in the first beta. If the dialog box is not active, each new message is accompanied by a pop-up hint, and a change in the Skype icon in the system tray. The icon changes its color and displays the number of unread messages.

An important change for those wishing to have more than one version of Skype running on a computer is the ability to start and login to the program from the command line .

Also in this version: plug-ins for Internet Explorer and Firefox are updated, language interfaces are updated, support for Vista Core Audio is improved, work of some API is corrected.
