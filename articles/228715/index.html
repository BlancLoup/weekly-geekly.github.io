Microsoft buys UnityVS - Unity3D debugger for Visual Studio

Good news for all game developers using Unity3D. Microsoft has announced the purchase of the company SyntaxTree . This company is notable for the fact that it is developing a plugin for Visual Studio, which allows you to debug Unity applications in VisualStudio itself. Indeed, without this plugin, developers have to be content with IDE MonoDevelop. In general, an extremely convenient and useful thing when developing for Unity.

I think the vast majority of habrazhiteli knows about Unity. This is a very powerful cross-platform gaming framework with adequate pricing policy (more recently). However, there are no ideal things, and Unity is an example. Out of the box, Unity allows you to debug only in MonoDevelop and period.

In general, MonoDevelop is not the worst IDE, however Visual Studio is a more mature product, it has a car and a small cart of plug-ins, and often it is a much more familiar tool. That is why many (including myself) prefer Visual Studio. This crowd of people would like not to part with their favorite instrument. Especially for them, SyntaxTree wrote a plug-in for Visual Studio, which allows you to fully work with Unity code in this IDE.
