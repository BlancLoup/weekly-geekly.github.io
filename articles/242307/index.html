Open Terminal Client OTC-110 - news "November 2014"

news


Cosmetic changes

  • correct shutdown when pressing the sleep or alt-ctrl-del button if there is no session connection
  • shutdown is accompanied by the appropriate inscriptions
  • output of the kernel messages delivered to the second console (ctrl-alt-f2)
  • Added switching to debug mode (press del several times in the settings). In this case, before connecting and see with what parameters the remote session is started.
  • NAND check added (in debug mode in About), takes a lot of time and runs on console 9
  • can connect to VPN - PPTP settings are available
  • unexpected use of the DS-110 board - as a player for transportation videos. The usage scenario is approximately the following: we synchronize (GSM modem) videos and playlist, play according to the playlist, send mail when updating. The resolution of the clips is 720p, a larger size requires large volumes when synchronizing and the use of certain codecs in the preparation of clips.
  • as a result of the previous news: the GSM modem is now working (the Huawei E1550 model is tested on the Beeline operator), to work, you need to switch the ethernet interface to dynamic mode, disconnect the ethernet cable and connect the modem, then turn on the device. The modem must be switched to the modem only mode, without CD emulation, flash drives or anything else.
  • at the client's request, touchscreen support was added to the kernel (more details opensourceclient.org/forum/viewtopic.php?f=14&t=40 ), the kernel had to be rebuilt (more details opensourceclient.org/forum/viewtopic.php?f=14&t=39 ). At the same time introduced bootlogo. Unfortunately, the client did not provide any details, but he was very pleased - they did something for some exhibition, the exhibition should have finished by the deadlines, there is no feedback yet
  • ssh server is disabled by default; it is enabled in the Security section.
  • FreeRDP showed up when redirecting the COM port - ‚Äúladder-effect‚Äù (link to git ), should be solved while the solution is being searched for - if there is a solution to this problem - please contact us
  • to the previous point: while we were looking for a solution we collected the latest beta of FreeRDP right on the board - support for two monitors in the RDP session is not far off
  • To the previous point: the syntax of the FreeRDP version 1.1 launch line has been greatly changed - the shell has been redesigned, it is impossible to return back
  • now the shell is a regular package for debian, and not the result of the work of checkinstall, for this it was necessary to streamline and brush a lot
  • as a result of the previous news: almost ready to put the shell source code on github
