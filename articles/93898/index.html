Amazon Web Services Offers Failover DBMS



The cloud hosting service provider Amazon Web Services today unveiled a new feature Multi-Availability Zone (Multi-AZ) , which expands the functionality of the cloud-based DBMS Relational Database Service (RDS). Using the new feature, users can increase the availability of their own databases by automatically configuring the base to replicate the master data.

The backup database can be connected at any time at the request of the client, for example, at the time of failure of the main database server. Physically Multi-Availability Zone is located in a data center, remote from the main servers of Amazon Web Services. Amazon says that Multi-Availability Zone users will not need skills to configure and administer databases at the engine level, since the service is as automated as possible and replicates real-time data completely autonomously.

The company's message states that using the Multi-Availability Zone makes sense not only in the event of disasters, but also when the main services go to scheduled maintenance or when a serious load is placed on the main server.

Some information for those who do not know. Amazon RDS is a web service that makes it easier to install, operate, and scale relational databases in the cloud. Amazon RDS provides you with everything you need to work with the MySQL database and at the same time takes control of the administration and provides a cheap service on the AWS cloud. To learn more about technology, visit the product page on Amazon.
