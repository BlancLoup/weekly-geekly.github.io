1C: Enterprise 8.2 in Ubuntu 11.10 with a local HASP key

upd. vdovin_ds writes :
1 8.3, linux. wine
So perhaps the content of the article is outdated.

I was tasked with installing the latest version of 1C on Ubuntu. By the end of the first day of reading the forums, I learned the whole complexity of being. By the end of the second - earned!

The Internet is full of various information about this, but most of it is irrelevant: it describes the installation of the server part of 1C, the installation of an outdated version or versions with a network key.

At first I tried to install 1C on a "clean" Wine version 1.3 (including the USB patch ), but dancing with a tambourine around the computer did not lead to positive results, so I had to switch to WINE @ Etersoft . Actually, step by step installation instructions under the cut.
To begin with, a fresh image of Ubuntu 11.10 was installed on the computer, and the following were copied to it:
  • 1C: Enterprise 8.2.14.540
  • Configuration "Accounting for Ukraine", Version 1.2.2
  • WINE @ Etersoft 2.0 Local


After that, wine-etersoft_2.0.0-eter0.2ubuntu_i386.deb and wine-etersoft-local_2.0.0-eter2ubuntu_i386.deb are installed on the computer. The second package contains a version of WINE @ Etersoft, which is allowed to use for non-commercial purposes. For organizing a separate Linux workplace for an accountant, lawyer, estimator, etc. you should purchase a single-user version of the WINE @ Etersoft Network.

After a successful installation, run the command wine --update in the console . This will create a .wine folder in the current user's home directory and install the necessary files there. You should also put the wine-etersoft.lic file from Etersoft after registration in this folder.

Then run setup.exe from the 1C distribution kit and install using the default settings. At the end, when the installer prompts you to install the protection driver, uncheck the box next to this item - the driver for working with the USB dongle will be enabled later.



After successful installation, if necessary, the ‚ÄúAccounting for Ukraine‚Äù configuration or a similar set of templates is installed. You can install it in the default folder, it will be specified in the 1C settings later.

Next, you should insert into the computer a USB dongle that comes with the 1C distribution. Since we only need his work in the Wine emulator, we do not install the official drivers for it , but instead open the Wine settings and enable its support:



It is also recommended in the settings of the graphics settings to prevent the window manager from controlling the windows:



Now we run 1C: Enterprise using the link that appeared on the desktop and set the path to the configuration templates directory:



After that we can create a new database or import an existing one. To convert 1C databases of older versions, after adding it to the list of information bases, double-click on the "Configurator" button. After it is launched, it will automatically offer to convert the database into a new format (it is recommended that you first make a backup of the database).



When you create a new database after clicking on the "Done" button 1C after some deliberation freezes. However, to our happiness, the database manages to be created, so it‚Äôs enough to open the console and execute the sudo killall wineserver in it, then start the applications again and add the already created database.



To start the 1C: Enterprise thick client, click the Configurator button in the infobase selection window, and then in the similar window that appears, click the 1C: Enterprise button. Otherwise, regardless of the settings, the 1C thin client is launched. The reason for this strange behavior could not be determined.



After the initial filling of the information database, we get a workable copy of 1C: Enterprise under Linux.



I hope this instruction will help someone save time, which can always be spent on more entertaining classes;) All with the first Friday of this winter!
