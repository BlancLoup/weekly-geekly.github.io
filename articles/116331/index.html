Analysis of the code showed that it contains not one, but as many as two password strength analyzers - the current, fairly simplified, and the old more profound.

Let's start with the current one, according to which the analyzer is working now. In the second part will be the analysis of the old.
  • Password strength is given in the range [0; 4] depending on the calculated ‚Äúbit strength‚Äù bits

  • To assess the "bit resistance" formula is used 
    bits = log(charset)*(length/log(2))
    Where
    • bits - bit resistance
    • log - natural logarithm
    • length - password length
    • charset - the total size of the sets for each of the types below, if they are present in the string:
      • small English letters [ abcdefghijklmnopqrstuvwxyz ];
      • capital English letters [ABCDEFGHIJKLMNOPQRSTUVWXYZ] ;
      • special characters [~ `! @ # $% ^ & * () -_ + ="] ;
      • numbers [1234567890] ;
      • other characters
