If I need to download the deviantART gallery, YouTube channel, VKontakte playlist, etc., then I use ...

There are urges to write a softphone: you enter the URL, it offers the appropriate options, what to download (gallery, favorites, friends friends, friends friends favorites), where to download (how to stuff folders, maybe how to write to the database), how to download, when to download etc. Placing data on the site (where there is a block of information on the page), the relationship between them (the graph - what is associated with what), etc. are set declaratively (XPath, RegEx, etc. in the hierarchy) and partly by scripts (Lua, JS, etc.). I want a universal tool: on the one hand, so that there is a deep understanding of the structure of the site, like specialized rockers; on the other hand, to have the power of universal rocking settings. I create a survey in order to test the ground: someone needs it or not.
