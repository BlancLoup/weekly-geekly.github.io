New indicators of the percentage of download on the tabs of the future Firefox tabs

In the twentieth of May, the future Firefox code ( remember : most likely , it will eventually be called Firefox 4, and not just Firefox 3.7) was changed, visually appearing as progress bars , showing the conditional percentage of page loading on tabs, relevant pages. Also, the tabs will be equipped with closing buttons that appear only when you hover the mouse over the spine (for the rest of the time, this space will be given to the page title):



In Bagzilla, these changes correspond to bug 549061 (to which another pair of screenshots is attached: [ 1 ], [ 2 ] - showing the tabs in different Windows themes); The Burning Edge blog talks about the appearance of this innovation in the Firefox nightly build of the 23rd , which is also discussed on the MozillaZine forum .
You can set yourself this build if you want to feel the innovation, but follow the usual precautions (starting from under the new user profile , etc.), especially taking into account the short-lasting bugs mentioned in the assembly (self-resetting of the toolbar settings to the default state) etc.).
Users of the Tab Mix Plus extensions don't have to worry, because they already have all these useful features of tab design.
