Porting MIPSfpga to other cards and integrating peripherals into the system. Part 3

In the first part, I described cmoda7 as an example of how to port MIPSfpga ( Porting MIPSfpga to other cards and integrating peripherals into the system. Part 1 ) on FPGA boards other than those already ported, such as basys3, nexys4, nexys4_ddr from Xilinx, as well as de0, de0_cv, de0_nano, de1, DE1, de10_lite, de2_115, DE2-115 from Altera (Intel), in the second part how to integrate the Pmod KYPD keyboard ( Porting MIPSfpga to other boards and integrating peripherals into the system. Part 2 ).

In this part, we add to the MIPSfpga-plus a built-in ADC, and the popular LCD from Nokia 5100.
From the previous parts, we can conclude that the integration of the periphery in MIPSFPGA consists of five main steps:
  • Add interface module to communicate with peripherals (i2c, spi, etc.).
  • The connection of the input / output ports of the module with the bus AHB-Lite.
  • Assigning addresses of signals connected device.
  • Adding constraints to the physical contacts of the board.
  • Writing a program for MIPS processor.


Connection of the built-in cmoda7 ADC



As I said earlier, the cmodA7 board has a built-in ADC, pin 15 and 16 are used as analog inputs of the FPGA module. The operating range of the built-in ADC is from 0-1V, therefore, an external circuit is used to increase the input voltage to 3.3V.


This circuit allows the XACD module to accurately measure any voltage between 0 V and 3.3 V (relative to GND). To work with the ADC in Vivado there is a block of IP (intellectual property) Xilinx, with which you can simply integrate it into our MIPSfpga system.

Nokia 5100 LCD Display Integration


In the network, such an LCD display among arduinschikov is in great demand, we can say they gave him a chance for a second life by applying it in various projects. Therefore, it was decided to try to connect it to the MIPS processor as a practice.
I will describe the following actions briefly and essentially as all steps are described in detail above.

The display is controlled via SPI interface, the display is a slave device. However, instead of the usual four control lines, there are only three. These are the CLK clocking lines, the SCE chip selection, and the MOSI input data. The MISO exit line is missing. This leads to the need to apply special management techniques, more on this later. In the Nokia 5110 there is also an additional information / command line - D / CÃÖ. Each byte transmitted to the display can be interpreted as a command or information byte, depending on the level on the D / CÃÖ line.
