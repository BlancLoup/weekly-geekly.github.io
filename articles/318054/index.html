RFC prototype HTTP status codes for developer errors (range 7XX)

The guys at Railscamp came to the conclusion that the current HTTP specification is incomplete: because developers have many ways to "shoot themselves in the foot" in the code, but there is no formalized way to communicate the nature of the error to the end user :)

An RFC was proposed with the following status codes (HTTP status codes) reporting the reasons for the error. These codes are expected to be included in HTTP specifications in the 7xx range.

Codes occupy the entire range from 701 to 799 and are divided into 10 groups. Here are some of them:

  • 707 - Can't quit vi
  • 724 - This line should be unreachable
  • 725 - It works on my machine
  • 726 - It's a feature, not a bug.
  • 735 - Fucking IE
  • 767 - Drunk
  • 780 - Project owner not responding

All codes are available here . Enjoy reading!
