DevConf 2014 - will be held this Saturday. New Storage section - will reveal all the secrets of data storage


June 14 (Saturday) DevConf 2014 will be held in Moscow
For the first time in Russia, we managed to gather the authors of MySQL , PostgreSQL , Percona Server , Sphinx , Tarantool and experts NeoJ4 , BigData , HandlerSocket , NoSQL in a separate thread DevConf :: Storage - in fact, this is a separate conference on modern databases.

DevConf 2014 Program :: Storage


  • Cool as in Google. Sphinx-based search services
    Vladimir Fedorkov. The last twelve years working in the field of high loads.
  • NoSQL attacks: JSON functions in MySQL server.
    Sveta (svetasmirnova) Smirnova, author of JSON functions for MySQL and the MySQL Troubleshooting book, as well as MySQL technical support engineer since 2006, Oracle.
  • Schema-less PostgreSQL
    Oleg Bartunov, major contributor PostgreSQL, Researcher at the Sternberg State Astronomical Institute, Moscow State University.
  • RDF Nootropics for BigData
    Leonid Yuriev is a leading system architect at Peter-Service R & D. Previously seen at the Natalia Kaspersky Innovation Center and Infowatch.
  • Problems using MySQL on modern equipment
    Alexey Kopytov, a Percona developer since 2010, has been involved in the development of Percona Server and Percona XtraBackup. Develops MySQL since 2004, worked in MySQL AB, as well as in Sun Microsystems and Oracle.
  • Optimizing the performance of a large legacy PostgreSQL database
    Alexander Chistyakov works as chief engineer at Git in Sky
  • Tell me, graph ... Or about using NeoJ4 in a web project
    Alexander Novikov, Lead Developer at T-Platform
  • How NoSQL works
    Andrei Aksyonov, still doing Sphinx.
    If you already know what B + tree, Fractal tree, LSM, SSTable, columnar storage, Bloom filter, read / write amplification are, what the No letters in the word NoSQL mean, how miraculously the stonewall bases press data 100+ times, and why the file on the disk is also NoSQL, only small and angry, you have absolutely nothing to do on this report
  • Tarantool 1.6 Primer
    Kostya Osipov, developing Tarantool
  • Infrastructure for MySQL. Clouds against rusty servers.
    Vladimir Fedorkov. The last twelve years working in the field of high loads.
  • We deal with locks in MySQL Server.
    Dmitry Lenev, Oracle employee, has been developing MySQL Server since 2003
  • MySQL + HandlerSocket = NoSQL
    Averin Sergey - I lead various projects in the company Badoo


See you at DevConf 2014!
