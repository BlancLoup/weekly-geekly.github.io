Alexey Taktarov - Animations in the world of states




‚ÄúWe cannot use setTimeout for animation. Because setTimeout will not guarantee that your function will actually work in the specified period of time. This can lead to effects such as frame blending. That is, you will think that your animation will come in 16 milliseconds, which roughly corresponds to 60 frames per second, but in fact it will work more, and this debt will accumulate and accumulate.

Fortunately, the browser has a modern API that allows you to deal with this problem, and it is called requestAnimationFrame. This is a function that ensures that the callback will work at a convenient time for the browser scheduler. At the same time I want to note that it can also work at irregular intervals. If you want to use requestAnimationFrame, I highly recommend looking at which browsers you are targeting. ‚Äù
