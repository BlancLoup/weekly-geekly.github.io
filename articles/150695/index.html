Tracking changes in the properties of the Js object using dirtyFlag from the KoLite package


To track changes in the ViewModel object, dirtyFlag , which is part of the KoLite library, can help, which in turn extends the knockout capabilities by adding new features:

  • dirtyFlag - designed to track changes in both the individual and in the aggregate properties of the ViewModel
  • asyncCommand - designed to execute asynchronous commands
  • activity - designed to indicate activity. Used with asyncCommand

An example of using dirtyFlag to detect changes in a ViewModel object



For example, take the following ViewModel object: 

function Vm(){ var user = { name : ko.observable('Alexey'), email : ko.observable('alexey@mail.me'), address : ko.observable('Moskow st. Petrov 12'), awards : ko.observableArray([new Award('silver','The best speaker'), new Award('gold','Brain Storm #2')]) }, dirtyFlag = ko.DirtyFlag(user); return { user : user, dirtyFlag : dirtyFlag, modelStatus : ko.computed(function(){ return dirtyFlag().isDirty() ? "Changes detected" : "No changes"; }), clearDirty : function(){ dirtyFlag().reset(); } } };


This ViewModel uses the dirtyFlag property, which will track changes in values ‚Äã‚Äãwithin the compound user property. In order to indicate what exactly the user property will be used for tracking, the following construction is used: ko.DirtyFlag(user); . After binding the model, dirtyFlag will track changes in the user property. In order to get the data ‚Äúdirty‚Äù status you need to call dirtyFlag().isDirty() . isDirty () returns true if changes are detected and false otherwise. If the user has made changes and then returned the data to its original state, then isDirty returns false, since it preserves the initial state of the properties being analyzed (that is, a copy of the object data). This is clearly seen when looking at the dirtyFlag code.
