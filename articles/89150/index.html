Perst - High Performance OOBD

Perst - An open source, object-oriented embedded database


McObject high-performance object-oriented embedded database.

Supported Platforms

  • J2SE 1.4 and above
  • Android (Open Handset Alliance)
  • J2ME MIDP 2.0 / CLDC 1.1
  • .NET Framework (1.0, 2.0, 3.0, 3.5)
  • .NET Compact Framework (1.0, 2.0)
  • Mono

Main features

  • Stores data directly in Java and .NET objects, excluding the conversion time required by relational and object-oriented databases.
  • Compact (build for .NET only weighs a little over 500 kb)
  • Transaction support
  • Easily used development tools (below is the code by which you can evaluate it)
  • It is possible to use a SQL-like query language (for .NET 3.5, you can use LINQ)
  • Export to XML
  • Replication
  • Full-text index (indexing speed certainly doesn‚Äôt compare with Sphinx or Lucene, but still it will be useful for some tasks)
  • Base encryption using RC4 algorithm
  • Fast DB recovery system after failures (without using log-files)
  • The rest of the possibilities you can find on the Perst specification page .

Conclusion


Perst is really fast and compact OODB, which is suitable not only for your PC application, but also for mobile devices.
But can it be applied on sites you ask? The answer will be as follows: the tutorial claims support for a multi-client application using ThreadTransaction and the database property perst.multiclient.support, but I was not able to achieve high performance (10 threads create 10 records of 21 seconds each).
