ExtJS: version 2.1

A fairly substantial in scale and innovations update should please everyone who uses ExtJS in their developments. In version 2.1, the framework acquired new components, as well as the AIR version of the local API help.

The incomplete list of innovations is as follows:
  • Full REST support
  • Added Ext.StatusBar Component and Samples
  • Added Ext.Slider Component and Samples
  • Added Example to Demonstrate Remote Loading of Component Configs
  • Added Grid Filtering Sample
  • Added Layout Browser Sample
  • Added Spotlight Sample

A complete list, including an exploded list of bug fixes, can be seen in the ExtJS download section , where links to download the entire package as well as the local API help are located.

Personally, I had noticed some oddities in the behavior of the new version, such as: the appearance of a horizontal scroll in FF2 when displaying dialog boxes (apparently due to a splash). Also, I could not find an example for the StatusBar component (it is accessible from the file system, but not through the page of all examples). The Layout Browser example will work (as it always happened with help) only being downloaded from a host and not open directly from the local disk.

Of all the innovations, the StatusBar and Slider are very useful, as well as the remote download of the configs component, which opens up great opportunities for writing powerful, retaining their state applications.

UPD by dkrnl : A very important point is the change of license for GPLv3.

UPD : Post about the update in the official blog: improve performance, review components and preview ExtJS v3.0.
