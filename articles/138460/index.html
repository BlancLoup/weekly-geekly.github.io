Recover deleted mail to mail.ru (and others)

Attention!
If you found this page from a search engine, and you do not have a saved copy of all emails, or if you have never used your email program, and only the web interface mail.ru, you cannot restore mail! Please do not email me.

Hello!
Have you ever seen an empty mailbox after you set up your pop3 email client and forgot to tick ‚Äústore mail on server‚Äù? It happened to me.
So, you have mail in the mail client, there is no mail on the server. Everything would be fine, the letters themselves were preserved, but the mail on the server was very necessary, and I began to think.

Option 1: Check the cart on mail.ru


Naturally, there were no letters in the basket. Found an entry in the FAQ, which stated that it is impossible to recover letters deleted in this way.

Option 2: send mail to yourself via smtp with fake headers


Did not work. And, I think, with the time of the letters, nothing could be done. Perhaps somewhere this option will work.

And here I remember about ...

Option 3: Use the function of collecting mail from other servers


Mail.ru, as well as other mail services, has a function for collecting mail from your other mailboxes. Since we have mail in the mail client, I found the folder where the letters are stored (in eml format), picked up the pop3 server (used the Small HTTP Server, set up in a minute), indicated to the server a folder with letters, tried to pick it up as the email client. Unfortunately, I have not received a single letter. Raised smtp server, sent a letter. To give my pop3 server a letter, it was required to remove the leading zero from the file name and remove the extension. I did this with all the files and told mail.ru to pick up all the letters from my server.
An hour later, all the letters were on the server as they were before deletion (only all were ‚Äúunread‚Äù).
I hope this is useful to someone.
