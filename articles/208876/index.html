Some theory


Today, there are many different classes of error-correcting codes, differing from each other in structure, purpose, efficiency, coding / decoding algorithm, and others.
In one approach, codes can be divided into two groups: block , in which encoding and decoding are performed within a single block of bits and tree , in which processing occurs continuously, without dividing the message into blocks.

The decoding of the block code can be accomplished by hard or soft decision making. In decoding with a hard decision, a value is assigned to each accepted bit in the demodulator
0 or 1. A soft decision decoder accepts not only a binary value of 1 or 0, but also a confidence value associated with a given bit. The decoder will use this soft information, and at the output we get the same hard decision.
In short, the vector from the value 0 or 1 comes from the channel to the input of the hard decoder. The vector from the values ‚Äã‚Äã(-‚àû; + ‚àû) comes to the input of the soft decoder. The use of soft decisions gives more opportunities, as it provides a wider maneuver to correct the arising interference.
Chase's algorithm is a block code with a soft solution .


Chase Algorithm


Chase's algorithm is an understandable and relatively efficient error correction (interference) algorithm in the transmitted message. The main idea of ‚Äã‚Äãthe algorithm is to generate an array of code words that will contain the word closest to the received sequence. The algorithm is based on the fact that if a word decoded by a conventional hard decision contains an error, then one of the ‚Äúnearest‚Äù code words to it most likely matches the transmitted one.

4. We introduce some noise and get the vector r = [-1.02, -1.1, -2, 1.95, 0.98, -2.34, -0.73, 1.97].


5. We use the second Chase algorithm . It is characterized by the following method of generating an array of check code words:

  • Choose from the received vector r p least reliable bits, that is, p bits of the smallest absolute value. The last bit in the vector is not taken into account, since it checks parity and can be calculated on the basis of other bits;
  • Choose the number v = <p;
  • Generate an array of helper vectors h . Consisting of vectors of long p, with all possible combinations of bits with a value of 1, the number of which in one vector should be no more than v, and bits with a value of 0 in the remaining positions (see example below);
  • Generate an array of check vectors e . According to the following rules:
    • Each vector must be one bit shorter than the vector r;
    • The array of vectors must be the same length as the array of auxiliary vectors generated in the previous step;
    • Each vector must consist of zero bits, except for the positions of the least reliable bits selected in the first step. Bits from the array of auxiliary vectors h should be located at these positions.
