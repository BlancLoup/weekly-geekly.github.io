Turbo is now in Opera's main code.

Today, Opera developers have announced the availability of the new Opera 10 assembly. The main innovations of this version are: new location of installation files and profile files, new spell checker, crash log generation system ported to Linux version, as well as Opera Turbo included in Opera main code.

For those who are not yet in the know, let me remind you that Opera Turbo is a function that appeared in the past experimental build of Opera 10, the essence of which is to speed up the loading of pages due to pre-compression of pages and images. To do this, use the same server that is used to work Opera Mini.

What is new in this build?
  • New, experimental Opera Turbo mode: automatic. In this mode, Opera monitors the quality of the connection, and if it decides that the connection is slow, Turbo is activated. If Opera decides that the connection is fast enough, Turbo mode will be turned off. Of course, you can choose whether to use this mode or not.
  • Fixed errors with loading images
  • Gmail bugs fixed
  • Fixed display channels for a subscription to rss
  • Fixed error with entering Cyrillic characters that appeared in the past experimental build of Linux version
  • Improved performance
  • New spell checker. Now hunspell is used as an engine, which seems to be smarter and more compatible with different dictionaries.
  • Now the Linux version of Opera generates a crash log in the event of a crash. Developers are asked to send them such logs. Due to this, a number of errors have already been fixed, now we can expect stability improvements in the Linux version.
  • In order to unify, the names of configuration files and some paths have been changed. Now, regardless of the system used, all configuration files have the same name.
Due to new changes the following errors are possible:
  • The disappearance of all site icons from the cache
  • Cleaning speeddial
  • Loss of ad blocker rules
  • Spell check may not work. You need to move the dictionary files to the new places. In Windows: C: \ Program Files \ Opera 10 \ locale \ ru \, in Linux: / usr / share / opera / locale / ru / (where ru is the dictionary language designation).
Also found that in this build, a number of Yandex services do not work in Turbo mode, for example, mail.yandex.ru.

Attention! Be sure to back up the folder with the configuration of Opera
