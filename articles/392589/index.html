pyLCI - front end for Raspberry Pi and other Linux devices

Hello!



Today I want to show you the project I have been working on for the last couple of years and finally release the first usable version. This is pyLCI - Python-based Linux Control Interface , an external interface for computers with Linux, which allows you to interact with the system through symbolic displays and buttons. With the help of applications written for this interface, you can perform a large number of tasks related to the administration and configuration of the system, as well as run various scripts and execute commands. And all this is independent of the monitor with the keyboard!

And also my interface:
  • Cheap
  • Plain
  • Easily expandable
  • Universal

What can you use this interface for?



  • Connect to wireless and wired networks, get information about network connections and configure them
  • Connecting Bluetooth devices
  • Media Player Control and Sound Tuning
  • Scanning / printing documents, taking photos with the camera
  • Manage running services
  • Data backup
  • GPS and navigation
  • Pentesting


Where can I use this interface?



  • Raspberry Pi and other SBC for Linux. A wonderful thing, since there is no need for a second computer for many routine and sometimes otherwise inaccessible tasks. Connect to WiFi without UART and keyboard? No problem. Run the shutdown command in order not to spoil the SD card? Also. Run some script? Yes please.
  • Home servers and routers. Viewing the list of DHCP clients on this is quite convenient. Turn off the WiFi access point or request a new address via DHCP? Why for this climb in the web interface? Also, for example, you can easily block sites on the router on the list when you sit down to work . You can also backup external media on a team, scan documents, and a whole lot ‚Äî just make pyLCI wrappers for your scripts.
  • HTPC. Conveniently, to quickly change the volume of the music, flush / switch the track, pause playback, or connect Bluetooth speakers.
  • Stationary computers. Especially if Raspberry Pi = is used as a stationary computer =) Even if not, this is quite a good addition to changing the system settings.
  • Laptops and tablets for Linux ... No, well, no one forbids ;-)



Recently, I decided to make pyLCI more accessible for everyone to use, and as a result of my work, I present you version 1.0 .

What does she know out of the box?


  • Connect to wireless networks (more info here)
  • View IP / MAC interface addresses
  • Change the volume
  • Switch tracks in the console player (as an example, I use MOCP).
  • Scan I2C bus for devices
  • Shut down and restart computer


While this is a fairly small list, but it is very easy to fill up. pyLCI is a kind of framework that can be easily extended with ‚Äúapplications‚Äù in Python to add one or another function to it. It provides these applications with I / O devices that can be used to communicate with the user, as well as a set of basic UI elements ‚Äî such as a menu item. Thus, when writing an application, you can not concentrate on the hardware or the nuances of the UI logic (such as nested menus) and fully devote yourself to solving the main goal =)

Of the disadvantages:



  1. The entire system at the moment is one process, including applications. In this regard, and the possible number of settings that the system should be able to change, it is easier for the average user to start it under the root than to configure all the necessary permissions. If there is a need, I can list which groups / permissions are needed for specific applications and methods of connecting IO.
  2. I have written installation scripts only for Debian / Raspbian. I would be grateful if someone could help with adding support for other distributions in install.sh, config.sh and update.sh.
  3. Also, there is no script for sysvinit - at the moment everything is sharpened under systemd. If anyone has a suitable proven script for rework, please share =)
  4. If you give the pyLCI keyboard, it captures it all.
