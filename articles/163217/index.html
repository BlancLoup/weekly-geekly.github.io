Qt 5.0.0 is out!

They still had time!
Literally less than an hour ago, the new owner of Qt Framework, Digia, announced the release of the long-awaited release of Qt 5.0.





Qt 5.0


Qt 5.0 is a global update from the Qt 4.x series, which has been developed since 2005. It took a lot of time and effort to adapt Qt for the future.

Qt 5.0 is distributed as a full binary SDK package for Linux (64 and 32 bit), Mac OS X (10.7 and 10.8) and Windows.
And also includes Qt Creator 2.6 IDE, examples, and full documentation.

The video below should give a good first insight into the new features and features of Qt 5.0.

This video was made using a real-world capture program written entirely in Qt 5 using Qt Quick, OpenGL and WebKit.

Compatibility


Qt 5 is the successor to Qt 4, and every effort has been made to maximize compatibility with Qt 4.x.

A lot of things were fixed in the internal architecture, and Qt 5 was more modular, compact and fast, but compatibility was preserved so that most applications could be translated to Qt 5 with just a few changes and recompilation of the code. This, of course, means that Qt Widgets are fully supported and are an integral part of Qt 5. Qt Creator, which is the largest application developed by the Qt Project, is a great example of this. A few weeks ago, Qt Creator 2.6 was released, compiled with Qt 4.8. The Qt 5 SDK contains the same Qt Creator 2.6, but compiled with Qt 5.

What's next


There are a few things that will be continued in Qt 5.0.x: there are a few bugs that need to be fixed; There are also no binary builds for MinGW and MSVC 2012 compilers.
It is planned to release the first Qt 5.0.1 patch release at the end of January.

The team aims to release version 5.1 in late spring 2013, which will be aimed at improving the stability and refinement of such modules as Qt 3D and Qt Sensors. There is also hope that Tech Preview versions of Qt for Android and iOS will appear at the same time.
