Solution of the cross-platform problem (Development of the 1C Enterprise platform)

In the modern computer world, with an abundance of various operating systems, almost every self-respecting developer produces his own software products for the maximum possible number of them, of course, considering the profitability.
In this small note I want to tell you what 1C did, with its most important product, 1C Enterprise.

We assume that ce began with version 1C of the company 7.7. It runs perfectly and works under Windows, because under him and written. Many people use this version to this day. There are successful projects to launch this system under Linux, but under the wine emulator, BUT no one will ever give a 100% guarantee that it will also work as well as under Windows.

Later version 1C Enterprise 8.0 appeared after it 8.1. This version can already work in a client-server version, i.e. Database server + 1C Enterprise server + Client machines . And both the Database Server and Server 1C Enterprise can work not only under Windows control, but also Linux. And as a database, you can use not only MSSQL but also Postgresql and IBM DB2. But still, the client part runs only under Windows. Of course, once again there was an opportunity to use wine, but to be honest, I did not like the stability of this implementation.

But after a while 1C released a new version of 1C Enterprise 1C 8.2, and in my opinion solved the problem of platforms very nicely. Now 1C Enterprise can work through a browser. Each of you can verify this by following the link http://v8.1c.ru/demo-ma/ . In my opinion, it was impossible to come up with a prettier one. Theoretically, you can now run 1C under any OS (Windows, Linux, Mac X OS) and at least on a cell phone or other device that has a browser.

Of course, the interface of the program has changed a lot, someone says that it became very inconvenient, but I note it was also said when 1C Enterprise 8 appeared, but over time people got used and found it in something even more convenient than its predecessor.
