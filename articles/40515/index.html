About fla-source

On duty, I often have to rake someone else's source code with animations. And sometimes I just come to the wild horror of what is happening on the timeline and in the library. so I want to share with you with some rules for formatting frames and organizing symbols, which I derived from my experience.

1. Never use key frames without extreme necessity , especially if it is a layer with a background that does not change throughout the entire movie. Well, why, one wonders, do the last frame key? If some changes are made in the first frame, then almost always the last keyframe is forgotten. When extending the background on the timeline, you need to use static frames (hot key f5).

2. Never leave unfilled frames in the middle of a clip. If subsequent frames do not need to be filled, drive them empty. otherwise, it may lead to animation errors.

3. Never apply motion animation to graphic objects and text without first converting them to symbols. This is most annoying, because the result is a large number of graphic characters (with the names Tween), which are terribly inconvenient to work with!
If you created a text that you plan to animate. do not be lazy to convert it into a symbol, then it will be much more convenient for you to work with it.

4. Let sane names of characters and layers. Well, at least only characters! Sometimes it takes a lot of time to search for a particular character in the library.

5. Sort characters, music and bitmaps into folders. This is just plain convenient! Especially if the number of items in the library exceeds a couple hundred.

6. Delete all unused characters from the library. For this there is even a special command ‚Äúselect unused items‚Äù. The only exceptions are the elements that are planned to be used in further work.
