qutIM 0.3: Through thorns to the stars!

Funny day, Habrachelovek!

After exactly a year of active development, we are all those who participated in the development of qutIM , we are glad to present you the release version 0.3 under the codename Nolwe.
Over the past year, a huge number of bugs were fixed, missing features were added and some plugins were reworked.





General changes



Github!

The qutIM code repository has moved from gitorious.org to GitHub .
This was done with the intention that now each line of code passes a review, which allows you to find errors at an earlier stage. (And it's just more convenient to live with the wonderful Tomahawk )
In total, since the transition to git, 5189 commits have been made, 26 people have been commited to the branch.

Translations

The qutIM translations are now available on translations.launchpad.net/qutim , and this means that anyone can participate in the qutIM translation!
Now the most complete translations are translations into: Russian, Bulgarian, German and English. A translation into Czech, Ukrainian and Arabic is slowly but surely developing. Chinese is in the very germ.

Release Schedule


I would also like to say that qutIM has switched to another release model. Yes! Now you do not have to wait for many months before the new update comes to your stable distribution.
Now non-serious changes lead to a change in the last digit. Such mini-releases will be released once a week. (0.0.0.x)
When a large number of changes accumulate, the second digit is incremented from the end. (0.0..0)
When the API is changed, the second digit is incremented.
For especially large changes, a major number changes.

Pictograms

As you have noticed, the pictogram has changed in qutIM thanks to the artist, carefully drawing the pictograms of statuses, protocols and other artwork.



Meal'n'Real!



The release includes the following protocols:
  • Jreen based XMPP / Jabber.
  • ICQ
  • IRC
  • Mail.ru
  • Vkontakte

As well as all the protocols from the library libpurple (which is more than 10), although this library in the ranks of developers is not popular and errors are possible.





As previously reported, in qutIM everything is implemented through a plug-in system.
In qutIM there are several contact list implementations (Default and Tory), chat forms, engines for displaying message logs (WebKit View with support for Adium themes, ultra-light Text View and experimental QML View), settings windows and much more.
This provides a huge scope for customization qutIM to fit your needs.

Separately, I would like to mention the possibility of full pre-tuning of qutIM for installing administrators of ready-made versions (absolutely all settings take default values ‚Äã‚Äãfrom system files, which opens up tremendous opportunities for system administrators).

Plugins

As for the plug-ins, there are a lot of changes.

  • Updater - At the moment, this plugin is needed in order to update client icons (there are quite a lot of them) without updating qutIM itself.
  • Highlighter - A recent plugin that allows you to customize qutIM highlighting for certain words using regular expressions.

  • Weather - weather forecast reincarnation
  • BlogImprover - easy integration with blog services like Juick.com and Psto.net.
  • UrlPreview - A preview of clips and audio through the corresponding tags in html5 has been added to UrlPreview.

  • ConnectionManager - the logic of work was reworked when the Internet connection was lost, at last qutIM again behaves transparently in these extreme situations!


What are your plans?



Well, after the release, you can tell about the plans that will be implemented in the very near future.
I want to remind that now releases will be more often :)

  • And much more
