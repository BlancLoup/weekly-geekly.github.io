Admin Section:

I did not notice any special changes here. Everything is the same as in version 5 of Zimbra. Pretty familiar and easy interface. The only thing I noticed was a few errors in the translation of the admin panel ... but these are minor details.



There are also problems with displaying the page structure in Internet Explorer 8. Everything is in order in Firefox.



Service Classes and Domain Policies.

Here I would like to clarify the functionality of these sections. Actually, like in Exchange, you can create group policies here that can be assigned either to all users or to some users. These policies allow you to simplify the configuration of mailboxes.



A service class is a policy that provides detailed mailbox settings. For example, you can restrict all users to use tasks, chat ... or you can assign everyone a common font for writing letters. Well, and so on.



And in the domain properties, you can configure only the technical settings of the boxes of this domain ...
I advise before creating users, first create your own (or change the ready) service class, and only then add users.

Servers section

Here you can configure the mail server services, as well as some protocols.

Using Zimbra in a corporate environment:

Here I would like to elaborate on the use of Zimbra in the company.
In general, Zimbra has almost all the functionality of MS Exchange Server. There is a check for mail viruses out of the box, there are general calendars, common tasks and so on.

The functionality of the joint work itself is implemented very well. For example, each user can share any mail folder (for example, Inbox) for any other user.



Or you can chat in a corporate chat with users - friends, so as not to send 100 letters per day.

But against this background, many unfinished trifles emerge. I can not list them all ... but I am sure that the developers will fix everything.

I look forward to new versions of Zimbra Desktop, because It seems a bit unmodified ... there is no multi-user support ... no chat rooms, zymelets installed on the server are not installed in the client, and so on.
