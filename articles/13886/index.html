What's new for developers to expect from Flash Media Server 3?

  1. Improved performance . The number of parallel streams for Flash Media Server 3 (hereinafter simply FMS 3) will increase by 200 percent compared with the previous version. This improvement will help reduce the cost for each individual thread, since parallel threads can run more on a single server. The improvement does not depend on the server platform, whether it is Windows or Linux Red Hat 4. You will learn more detailed information on server platforms and other necessary characteristics after the product goes on sale.
  2. Streaming support for H.264 and AAC . FMS will support the standards H.264 and HE-AAC, as video-audio in Flash Player, AIR and Adobe Media Player. This has already been written .
  3. New security features . The RTMP protocol is now equipped with a newer and more secure encryption mode, which provides more powerful protection of media content. This protocol will be called RTMPE. Similar to SSL (RTMPS), developers can use it as an additional security option. In addition, a new (more precisely, one of the new) SWF Verification feature will help protect SWF from being illegally used, copied or placed in other places.
  4. Improved support for live video . FMS 3 will support scalable and high-quality live video and audio created in Flash Media Encoder. And all this will work on one FMS-server.
  5. Mobile video support in Flash Lite 3 . FMS 3 supports recording and live streaming in the Flash Lite 3 platform. Now you can watch any video (for example, a video from YouTube) on any mobile device that supports Flash Lite 3.
  6. Compatible with the help line of the Flash Platform. Help systems and documentation of the FMS, as much as possible adjusted to be supported in Flex and Flash CS 3
  7. New plug- in architecture . FMS 3 increases its extensibility by introducing a powerful C ++ API that will help third-party developers to increase server functionality to fit their needs.
  8. Publish to another server . This is a great opportunity to help developers publish live video (audio) on another FMS server or SDN.

<o: p> </ o: p> I already use FMS 2, can I upgrade to FMS 3? <o: p> </ o: p>
Yes, of course, you can upgrade FMS 2 to FMS 3. Also, for those who still use Flash Communication Server 1.x, for upgrading, you need to upgrade this version to FMS 2 and only after that upgrade to FMS 3.

Can I run FMS 3 on any Linux version? <O: p> </ o: p>
FMS 3 officially supports Windows Server 2003 (all builds), as well as Linux Red Hat 4. In addition, FMS 3 (application development version) will support Windows XP.

<o: p> </ o: p> What versions of Flash Player will support FMS 3? <o: p> </ o: p>
FMS 3 can perform a regular connection and work with streams, starting with Flash Player 6. But, in order to use new features, such as secure RTMP, SWF Verification and H.264 support, the user needs to update the player to Flash Player 9 (the latest build Flash Player, you can always find on Adobe Labs )

Will FMS 3 work with AIR? <O: p> </ o: p>
Yes, of course it will. You can use AIR in conjunction with FMS 3 to develop desktop applications. You can create video-audio chat rooms (just like regular chat rooms), instant messaging programs. It all depends on your imagination.

Will my applications developed for FMS 2 work in FMS 3? <O: p> </ o: p>
Yes, they will, regardless of whether you have changed the client part or the server part.

Does FMS 3 support ActionScript 3? <O: p> </ o: p>
FMS 3 supports ActionScript 3, but only as a client application. Services (applications) you will need to develop using ActionScript 1. FMS 3 will introduce support for AMF3, so that users can exchange bitwise data (bitmap data ‚Äî that is, it can be pictures, online drawers (for example, as an option ) or any other bitmap data).

<o: p> </ o: p> Can I run FMS 3 on 64-bit Windows or Linux systems? <o: p> </ o: p>
FMS 3 supports 64-bit systems, as well as 32-bit systems. This means that FMS 3 can work with files that are larger than 2-x GB. <o: p> </ o: p>


What additional utilities come with FMS 3? <O: p> </ o: p>
Two new utilities are included in the purchase along with FMS 3. FMSCheck is a program that starts via the command line and checks FMS 3 for operability from under the current system. You can use this program in your scripts, for example, to get the current status of FMS 3.
FLVCheck - the program will check the structure of the FLV file and determine whether it can be launched in streaming mode. The program also fully supports the validation of the MPEG structure. FLVCheck (Windows version) can be found on Adobe Labs right now.

<o: p> </ o: p> Will FMS 3 support redirect, like HTTP-302? <o: p> </ o: p>
Yes, and this is one of the new features of FMS 3 - support for redirects.

FMS 3 fully supports IPV6.

Yes, you can develop applications where live and live streams dynamically switch. <O: p> </ o: p>
