Opera 12 - get full control over the browser

However, enough about external changes - in any case, you yourself will see them without third-party hints. It is much more interesting to see what has changed in the internal part of the browser. And here the list is very extensive. Here are just a dozen of the most interesting:

Previously, when trying to launch the browser with the restoration of all the tabs opened in the last session, there were obvious ‚Äúbrakes‚Äù - Opera diligently downloaded all the content at the same time. Now this process has changed: the active tabs are loaded first and those that are often viewed first. Now it does not matter how many tabs will open at the same time - the speed will still be high.
Separate processes for plugins

Parallel data loading
In Opera 12, the process of loading pages has become more optimized, allowing you to load scripts, styles, and images into several streams simultaneously. This greatly speeds up the work with "heavy" pages.
Improved performance on secure connections
While loading secure pages (https), the browser has to perform many of the procedures necessary to ensure a secure connection. This process is different on different servers and depends on many factors, including the type of security certificates used or the different versions and methods of implementing TLS security protocols. In Opera 12, we tried to analyze and optimize many of these procedures, so it was possible to significantly speed up the loading of protected pages in most cases.

Improved extension support

Now the 64-bit version of Opera 12 is available for all platforms. One note: due to the fact that 64-bit applications are more voracious in terms of using RAM than their 32-bit versions, by default we continue to offer 32-bit version of Opera for Windows.
