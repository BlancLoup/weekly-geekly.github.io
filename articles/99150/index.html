Relatively new features in GMail labs: partial removal of the read mark and manual update of POP accounts

Mass bonjour.

UPD : Community scolded and reported that the features are not new. However, not everyone is aware of them - so I leave the post in place with the addition of ‚Äúrelatively‚Äù in the title.

Today I once again found that sometimes looking into GMail labs is still useful. This time, not so globally as in the past , but it can also be useful to someone. So, two new and, in my opinion, useful additions in labs:

Mark Unread From Here


Everything is simple - now you can mark it as an unread part of the thread instead of the whole thread as a whole. Choose a specific message, go to the menu in its upper right corner and select ‚ÄúMark Unread From Here‚Äù there (I'm not sure how it will be in the Russian version of the gmail). As a result, when you return to the thread all the following messages will be deployed. It is also not possible (I confess, rather, desires) to test, but there is a chance that this will make only some of the messages in IMAP unread, because before, when working with GMail partially via the web, partly by an IMAP client, it was terribly annoying that mark as unread on the web the interface marked unread all the messages in the thread instead of, say, the last one.

Refresh POP accounts


But this recently, I just discovered a shortage - the ability to manually start checking POP-mailboxes, from which my mail in the GMail account is collected. Well, or rather, it was possible to do this by going to the mail collector settings and clicking "download" there - but it was very inconvenient. Now you can have a link "Refresh" in the top menu and pull out its ring in case of [emergency] need.

Such is a brief overview, I hope someone will come in handy. Merci for attention.
