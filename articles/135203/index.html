Django 1.4 alpha 1 released


The developers of the magnificent framework decided to please us before the New Year - and they released 1.4 alpha version!

Approximate release schedule versions:
  • Week since January 30, 2012: first beta release.
  • Week from February 27, 2012: first release candidate.
  • Week from March 5, 2012: final release.

Immediately note that Python 2.4 is no longer supported . Minimum - version 2.5.

A complete list of what appeared new can be viewed on the official website , and I will write a little of what I liked.
  • Support query SELECT FOR UPDATE .
  • Improved password hashing: PBKDF2 (or BCrypt ) algorithm is now used instead of old SHA1.
  • HTML5 DocType in admin interface.
  • Sort by several columns in the admin panel.
  • The old FormWizard from django.contrib.formtools.wizard has been replaced with a completely new one .
  • reverse_lazy () - the lazy version of django.core.urlresolvers.reverse ()
  • Internationalization for urlpatterns via i18n_patterns () will make life easier for multilingual sites. More on this .
  • Support for * args and ** kwargs in simple_tag , inclusion_tag and new assignment_tag .
  • The truncatechars template filter for trimming strings to a given length.
  • CachedStaticFilesStorage backend for caching static files that turns css / style.css into something like this: css / style.314d22ab3153.css
  • CSRF enhancements that add a decorator for ajax requests, protection against PUT and DELETE, and other tasty buns .
  • Updated manage.py code
  • ... and many small changes added.
