For myself there I found a very useful clip utility:

Everyone knows since the days of DOS that you can override the output console, writing, for example, example.exe >> 1.txt , then all that example will output to the console (the stream stdout) will be written to the file 1.txt . It turns out you can use this command c: \ dir | clip , then, everything that will be displayed by the dir command (or rather only stdout, namely the contents of this directory) will be written to the clipboard.
