Jump Start May 20: Data Center Virtualization using Windows Server 2012 R2 and System Center 2012 R2. Deploying Virtual Machines and Services

We bring to your attention a series of online events dedicated to data center virtualization using Windows Server 2012 R2 and System Center 2012 R2. As part of these online events, the key Windows Server and System Center technologies used to deploy, manage, and maintain the physical and virtual data center infrastructure are discussed in detail. If you are using Hyper-V in Windows Server 2008 / R2, you will see a huge amount of changes that have occurred in the Microsoft hypervisor with the release of Windows Server 2012 / R2. If your organization uses VMware, you can evaluate the capabilities of the latest version of Hyper-V, as well as compare the technologies of Windows Server and System Center with VMware vCenter and vCloud Suite solutions.


On May 20, in the third final online event of the series, we will take a closer look at network virtualization technology (Hyper-V Network Virtualization, HNV), from general operating principles to setting up virtualized networks and configuring the HNV gateway.
In addition, we will discuss the steps to create a private cloud in System Center 2012 R2 Virtual Machine Manager and the use of roles to quota the resources of the created clouds. You will see how you can use private cloud resources using the App Controller-based self-service portal. We will analyze in detail the structure of the Service Template and its application for deploying and updating applications. Finally, you will get acquainted with the capabilities of System Center 2012 R2 for managing hybrid structures when some of the objects and services are located in the local network, and some in the Microsoft Azure public cloud.
