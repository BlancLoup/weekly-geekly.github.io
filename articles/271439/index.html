In order to start developing on Unity under OS X, you need the following things (here you can begin to bend your fingers):
1) A computer running OS X. Any of the latest versions is potentially suitable.
2) Unity 5. Lives here - unity3d.com/ru/get-unity/download . In the installation there is nothing difficult, after downloading the installer, just agree with all the proposed points.
3) If you are completely new, then read a series of articles about creating a platformer - you can start reading from here

4) Next step - download Visual Studio Code - code.visualstudio.com . Code is a cross-platform editor, which appeared as a result of the following output: Visual Studio on OS X and Linux porting is long and expensive, but to have a high-quality lightweight editor that has all the editor features from Studio is very good and convenient. Soon the editor will become open-source (http://habrahabr.ru/company/microsoft/blog/271199/), and will have to be content with the installer. However, the installation of VSCode, as in the case of Unity, does not cause much trouble and passes quickly enough. As a result, you get a full-fledged C # editor, who is able to IntelliSence, code highlighting and other nice things, inherited from the ‚Äúolder sister‚Äù - the studio.



5) Next we need a plugin, without which nothing happens - github.com/dotBunny/VSCode . He is the one who allows you to make Code the default editor in Unity and correctly configure debugging mechanisms, IntelliSence and everything else. The debugger for Unity, by the way, works only in OS X so far, so you and I are very lucky. If you do not use git, you can use the quick link to download the archive with the plugin.

After loading the plugin, transfer the ‚ÄúPlugins \ Editor \ dotBunny‚Äù folder to the Assets folder of your project. If you don‚Äôt have the Plugins folder yet, create it. Right now.
