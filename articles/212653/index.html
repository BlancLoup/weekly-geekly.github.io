Decentralization: What services already exist?



Decentralization is the absence of a single center of control and a single point of failure.
Undoubtedly, at the present time, many projects are trying to translate certain services from the Client-Server principle to the Client-Client principle.

Let's see what we already have, and what we lack.

This post will be interesting to people who do not follow the emergence of new decentralized services in the network, you can briefly read about each of them.

Some of the important principles used in decentralized projects are things like Zero-config and zero trust .

In other words, the service can be considered completely decentralized, if to launch it, it is enough just to start the application without further entering any data for connection. Zero trust means that an attack of the MITM type should in principle not be feasible due to the implementation of the protocol. (The network will automatically reject the malicious node - DHT, the network can function even if all the nodes except you and the person to whom you transfer are captured - I2P, you can put the SORM on which the porridge is in place of the data, although it is physically connected to you directly to the channel break (CJDNS)

What requirements should the services meet that would be believed and used as completely decentralized?


- OpenSource
- Zero-Config
- MITM protection at the protocol level
- Low entry threshold (how to use and why it is needed - it is understandable for a simple user and you can describe it in one sentence)
- Fully decentralized operation algorithm (except for the first bootstrap)

Overlay nets

  • I2p
    Read more

    I2P is an anonymous, self-organizing distributed network that uses a modified DHT Kademlia, but differs in that it stores the hash addresses of network nodes, AES encrypted IP addresses, as well as public encryption keys, and the Network database connections are also encrypted. The network provides applications with a simple transport mechanism for anonymously and securely forwarding messages to each other.

    Although the I2P network is strictly focused on determining the packet transmission path, thanks to the Streaming lib library, their delivery is also implemented in the originally specified sequence without errors, losses and duplication, which makes it possible to use IP-telephony, Internet radio, IP-TV, I2P video conferencing and other streaming protocols and services.

    Inside the I2P network runs its own directory of sites, electronic libraries, as well as torrent trackers. In addition, there are gates for accessing the I2P network directly from the Internet, created specifically for users who, for various reasons, cannot install the ‚ÄúInvisible Internet Project‚Äù software on a computer.

  • TOR
    Read more

    Tor (abbr. From the English. The Onion Router) - free software for the implementation of the second generation of the so-called "onion routing". This is a proxy server system that allows you to establish an anonymous network connection, protected from listening. Considered as an anonymous network of virtual tunnels, providing data transmission in an encrypted form. Written primarily in the C, C ++ and Python programming languages. As of the beginning of 2014, it has more than 324,000 lines of program code, not counting comments.

    With Tor, users can remain anonymous online when they visit websites, publish materials, send messages, and work with other applications that use the TCP protocol. Anonymization of traffic is provided through the use of a distributed network of servers (nodes - "nodes"), called multi-layer routers. Tor also provides protection against traffic analysis mechanisms that threaten not only the user's online anonymity, but also the confidentiality of commercial secrets, business contacts, etc. Tor operates on-line routers with network levels, allowing for anonymous outgoing connections and hidden anonymous services] .

    BitTorrent (literally, ‚Äúbitstream‚Äù) is a peering (P2P) network protocol for cooperative file sharing over the Internet.

    The protocol was created by Bram Cohen, who wrote the first torrent-client ‚ÄúBitTorrent‚Äù in Python on April 4, 2001. The launch of the first version took place on July 2, 2001.

  • Bitmessage
    Read more

    Bitmessage is an open source cryptographic messenger using a decentralized P2P network. The Bitmessage network is similar to the Bitcoin network, in that it relies on cryptography and sacrifices some convenience for the sake of security and decentralization, but it is intended for the task of sending messages, and not monetary transactions. The network received a sharp surge in popularity [1] after Edward Snowden's disclosures about the secret tracking system for Internet users PRISM.
    Currently, the Bitmessage network processes several thousand private messages per day.

Search engines

  • Yacy
    Read more

    aCy (from the English. Yet another Cyberspace, Another Cyberspace; in tune with the English. Ya see) is a freely distributed decentralized search engine built on the principle of peer-to-peer networks (P2P). There are versions for Windows, Linux, MacOSX. The main program module, written in Java, operates on several thousand computers (as of 2011) of YaCy network members.

    Each project participant independently explores the Internet, analyzing and indexing the pages found, and adds the index results to a common database (the so-called index), which is shared by all YaCy users on the P2P principle.

    Unlike partially distributed search engines, the YaCy network is characterized by a decentralized architecture. All nodes are equivalent and there is no central server. The program can work in the mode of a search robot or as a local proxy server, indexing the pages visited by the user (this ignores pages with signs of personal data ‚Äî cookies, etc.).

    Access to search functions is provided by a locally running web server, which generates a search page with a field for entering search queries and returns search results in the same format as other popular search engines.

    Now in Namecoin domains are registered only in the zone .bit, for which the namespace "d /" is used (for example, the record of the domain "bitcointalk.bit" uses the name "d / bitcointalk"). The power of a distributed computer network ensures that two identical names do not appear and that your name-value correspondence (domain-address in a particular case) cannot be assigned and changed by any unauthorized person.
