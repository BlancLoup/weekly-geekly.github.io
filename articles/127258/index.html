How to report bugs?


So, if you think you have found a mistake, report it to our CUBRID JIRA tracker at http://jira.cubrid.org , whose interface is very intuitive. To do this you need to log in to the site under your login. If you do not already have an account, you can register . Registration is simple and does not require activation by e-mail.



As soon as you log in, you will see a page with the latest entries in the tracker. Click the "+ Create Issue" link in the upper right corner of the page and select the project ( CUBRID Tools or CUBRID Web Apps ) and the request type ( Bug ).



In the opened request form fill in the following fields:
  • Summary (enter a title that clearly describes the error found)
  • Component (CUBRID Manager, Migration Tool, Query Browser, or WebShell)
  • Reporter (make sure your username is listed here)
  • Environment (version of the tool itself, as well as the operating system, PHP or browser, if used)
  • Description (describe the error you found in detail, including step-by-step instructions, so that we can reproduce it)
  • Attachment (if there are snapshots or code files, attach them here)
  • Labels (enter the bug-bash label; this will make it easier for us to search for all requests created by participants during this event; be sure not to forget to enable it.)

Before creating a request, please make sure that no one reported this error. To see the list of created requests, click on the drop-down arrow next to ‚ÄúProjects‚Äù in the main menu and select the required project (CUBRID Tools or CUBRID Web Apps).



After opening the project page, select the program for which you are looking for requests (in the tracker is called as a ‚Äúcomponent‚Äù).



You will see the entire list of queries created for the selected component. Additionally, if you are interested, you can watch how much and what other participants of this contest create. For this, we created a special filter on our tracker. Please use it.

Where can I download?


Tools and Web applications CUBRID you download from the following resources.

Images for virtual machines

You can download one of several images that already have CUBRID, CUBRID Tools, Apache, PHP, etc. installed. This option is very convenient for those who do not want to install programs themselves. You can download virtual images from the official site.

CUBRID software kits

If you want to download all the programs in one archive, we have prepared them uploaded to Sourceforge.net. Choose your OS and download the appropriate set. Each of them contains CUBRID itself, as well as CUBRID Manager, CUBRID Migration Tool, CUBRID Query Browser, CUBRID WebShell, and CUBRID driver for PHP. The archive also has a README file, which explains how to install the tools.
  • Windows x86
  • Windows x64
  • Linux i386
  • Linux x86_64

Setup files

If you prefer to download and install programs selectively, you can go to the main pages of programs and from there download everything you need.
  • CUBRID for Windows

  • CUBRID Migration Tool

  • CUBRID PHP Driver
