VMware bought Zimbra

Well then, Yahoo! finally got rid of the asset , which was her like the fifth wheel, and VMware continues to build its own stack of applications. Very logical and correct move, moreover, relatively inexpensive. When VMware bought Java developer SpringSource in August for almost half a billion, it was less clear. Companies really need to diverge from Miscrosoft: VMware virtual machines are mainly used to run .NET applications, and with the release of Hyper-V, VMware has problems. Java business development would allow VMware to diversify, but SpringSource alone is not enough for that. However, VMware, as we see, decided not just to go into a dull defense, but rather aggressively climbed into the original patrimony of MS - postal and collaborative services. Zimbra, of course, has specific customers, mainly education, but EMC / VMware itself is a heavy truck that can easily compete with MS. So now Zimbra will have a new start, already in the corporate market.

In general, very interesting ideas emerge for this year in the field of virtualization and all kinds of as-a-service.
