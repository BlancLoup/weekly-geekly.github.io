Copy to clipboard in FireFox 3.5 and IE8

With the advent of the 10th flash, FF3.5 and IE8, the magic button ‚ÄúCopy to clipboard‚Äù stopped working on many sites. I stopped working for security reasons (so that when you go to a website, you don‚Äôt have a left link or advertising text in the buffer, or the attackers didn‚Äôt use this feature for other purposes).

Now EkshonSkript allows you to copy to the clipboard only when the user acts, that is, when you click on the clip (and the clip can be made transparent;). This is what jhuckaby used when creating his Zero Clipboard script.

View demo:

  • Simple debug test
  • Multiple use
  • HTML sors copy

The script works in all modern browsers.
Tested in FF3.5, IE8, IE8 (mod7), Chrome3, Opera10b2, Safari4. Support flash 9 and 10.

The bottom line:


After we use JS, initialize the script, specify the path to ZeroClipboard.swf and create the client for copying, we can safely buffer all the information to the clipboard by clicking.

But the script is not limited to this, it has wonderful properties (methods):
  • sticking to the DOM element (Gluing) - that is to say, we specified the element's id, glue the flash drive over it (it copies to the buffer), which automatically adjusts to the size of the element;
  • css effects - the script highlights the element on which the button is suspended, creating subclasses of "hover", "active" (ala pseudo). You can change them to your own in CSS;
  • the ability to set your own presentation is in case you need to create a custom rectangle, and you don‚Äôt need to stick a flash drive anywhere;
  • and he can also steer with standard events (onLoad, onMouseOver, onMouseOut, onMouseDown, onMouseUp, onComplete).
