New conference for anyone interested in JavaScript



Hello.

On June 5, together with SPb Frontend, we will hold a big JavaScript conference in all its manifestations in St. Petersburg: client and server, library and self-old, old-school and avant-garde.

The conference is called HolyJS (not bad, right?).

Conference site: http://holyjs.ru

Reports will be about 20 , parallel halls - 3 .

Confirmed reports include:
  • A well-known technical blogger, Clojure fan and DataScript author Nikita tonsky Prokopov will give a talk on how to organize work with the state and data of the application in conditions of distribution, poor communication and complex structure;
  • ASP.NET Legend Dino Esposito will explain how to accurately (smarter than through the user agent) define client devices in order to adapt UI / UX for them;
  • The head of front-end development at Avito and the author of basis.js Roman Dvornov will give a talk on CSSO - a tool for CSS minification;
  • Victor Grishchenko will tell about Swarm - a tool for data synchronization between different devices;
  • Denis Mishunov will talk about psychological and technical methods and examples that allow you to control the user's perception and help make your website ‚Äúfast‚Äù in user understanding.




Among the topics that are currently being worked out:
  • Architecture of modern JS applications
  • Functional JS Programming
  • Optimization and Performance Issues
  • Node.js: best practices, performance, memory management
  • JS and ECMAScript specification
  • Practice of using ES6 and ES7
  • Client server synchronization
  • Application Testing
  • Developer Tools

This is not a complete list of our members, but we understand that there is only one day, and only 20 reports.
