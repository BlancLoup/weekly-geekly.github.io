Release MODx 2.2

In Revolution 2.2, the manager was improved, media content sources appeared, static elements and a fresh look at the creation of news feeds and blog entries was implemented.

MODX Revolution 2.2 currently claims to be one of the biggest releases. We spent a tremendous amount of time on the work of improving the Manager and the administrative interface so that their aesthetics and performance would bring pleasure to use.

Version 2.2 also introduces many major features: media content sources allow you to choose where to store files, including Amazon S3 or potentially something like Dropbox (need some work); static elements allow developers to store site elements in the file system; and custom resource types open up the possibility of going beyond ‚Äúone-size-fits-all‚Äù documents and creating custom types such as blog entries, events, biographies based on the type of content (?).

Even more attractive, Revolution 2.2 makes a powerful new addition to blogging, news publishing, and other publishing tasks called Articles . If you felt that creating a news site or a blog took too much time, Articles for Revolution 2.2 will remove the obstacles to your old way of using familiar blogging tools. Articles has built-in import functionality for WordPress and Blogger posts, and for other services this feature will be implemented in the near future. Take a look at Articles or try it in action with MODX Revolution 2.2.

Due to the large number of changes and innovations, we strongly recommend that you read the update instructions before upgrading your live website to Revolution 2.2. Upgrading from version 2.1.x to 2.2-pl should go without complications.

The main novelties of version 2.2:
  • Media Content Sources: You are no longer limited by your local server. Media sources allow you to merge a tree of files with anything ‚Äî a separate directory, Amazon S3 storage, etc.
  • Static elements: store all static elements (Chunks, Snippets, Templates, etc.) in the file system or somewhere else with the ability to edit them with anything.
  • Custom panels: create and organize Manager welcome screens, and create and connect new Panel widgets.
  • Major user interface improvements, incl. cleaner look and faster loading times.
  • Added import / export access policies and their templates for developers and site administrators. This means that you can now customize them all at once for later reuse on other sites and projects.
  • Script installation or installation from the command line for those who want to continue to be a geek.
  • Many internal improvements to the Manager make it easier and improve the possibilities for its expansion.
  • Added the ability to create custom types of resources - now not only the only default type, you can have types for different purposes such as blog entry, gallery, forum, event, etc.
  • Added search to speed up finding any documents by keyword - right at the top of the resource tree, no more scanning and hunting for these old articles!
  • Improved parser performance and performance, added more than 600 unit tests, other kernel optimizations.
  • More than 160 resolved tasks, tons of bug fixes!
