Criticism: "Configuring Firefox: remove hangs and devour memory"

It is not clear: why should I put this or that key in one way or another? In addition, some key values ‚Äã‚Äãare questionable. Therefore, I decided to bring some clarity.
So...

  1. browser.sessionhistory.max_total_viewer = 0 . This value will lead to the fact that none of the visited pages will not be cached, and this, in turn, will lead to reloading the page from the server to which you decided to return again, for example, by clicking Back.
  2. network.http.pipelining = true and network.http.proxy.pipelining = true may well increase the speed of loading pages, as they allow you to send requests to the server without waiting for answers to previous requests, but network.http.pipelining.maxrequests = 30 - nonsense, since this parameter can take values ‚Äã‚Äãfrom 1 to 8 (see http://kb.mozillazine.org/Network.http.pipelining.maxrequests here).

  4. network.dns.disableIPv6 = false ‚Äî perform attempts to resolve names to IPv6 addresses. Under Linux, this value is set by default. It would probably be more correct to set it to true to disable such an attempt.

  7. ui.submenuDelay = 0 - does not affect the page loading speed in any way, as this parameter controls the delay between hovering the pointer over the browser submenu and displaying the contents of the submenu. UI..2A )
