Simple image comparison using php

The algorithm may not be new, and is not ideal , but, surprisingly, it works. No graphs and correlations.

To begin with, it is worth noting that the comparison is very approximate (at least at this stage), from time to time two completely different photos turn out to be similar by 60%, and also hard modifications (turns, inversion, cropping) are not taken into account - for this you need to bring and bring. For me personally, two parameters turned out to be the most important: speed , independence from image resolution and the ability to compare ‚Äúimages‚Äù right in the database.

Principle

  • Open the original image
  • Scale it to the size of the mask (in my case it‚Äôs 20 by 20, I don‚Äôt need big sizes - a comparison is approximate, it‚Äôs quite possible to make a mask and 10 by 10).
  • Calculate the main color of the mask.
  • We create an array, where the value is a key of type af2 (1,2 is the coordinates, as in naval combat. 2 is the discrepancy with the main brightness).
  • We generate a string - a key.
  • Compare two lines by relevance.

In this case, there is a version that works through GD with PNG images.
// -
function getimageid($image)
{
//
$size=getimagesize($image);

//
$image=imagecreatefrompng($image);

//
$zone=imagecreate(20,20);

//
imagecopyresized($zone,$image,0,0,0,0,20,20,$size[0],$size[1]);

//
$colormap=array();

//
$average=0;

//
$result=array();

//
for ($x=0;$x<20;$x++)
for ($y=0;$y<20;$y++)
{
$color=imagecolorat($zone,$x,$y);
$color=imagecolorsforindex($zone,$color);

// Ryotsuke
$colormap[$x][$y]= 0.212671 * $color[ 'red' ] + 0.715160 * $color[ 'green' ] + 0.072169 * $color[ 'blue' ];

$average += $colormap[$x][$y];
}

//
$average /= 400;

//
for ($x=0;$x<20;$x++)
for ($y=0;$y<20;$y++)
$result[]=($x<10?$x:chr($x+97)).($y<10?$y:chr($y+97)).round(2*$colormap[$x][$y]/$average);

//
return join( ' ' ,$result);
}
