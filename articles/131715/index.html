MonoDevelop 2.8.2 released

The second update to release 2.8 of the development environment for .Net, Mono, MonoTouch and MonoDroid platforms has been released. All innovations are related to development for Android and iOS. Also debugger optimized a little.
In addition, it seems that the code editor has been brought to mind. After half an hour of bullying, I could not get him to make mistakes (a week ago, a similar procedure revealed the inability to save the file without a code, but with comments).

Full list of changes can be read here . Review release 2.8 on the Habr√©, if someone has not read, you can see here .


The latest version for Windows and MacOS X can be downloaded from the official site . There are no packages for Linux yet, but I think they will appear there for openSuSE and NLE.

Debian users (you will need to install mono from the experimental repository) and Ubuntu can optionally use my ppa , where you can find both this release and the package rebuilt with each commit in the github repository.
In addition to this release, there is a corrected binding to GTK. The essence of the edits is that they are compiled under .NET 2.0 and allow you to compile code for this version of .NET, which is not the case with the original Ubuntu repositories. Otherwise, they are absolutely identical. There is also a binding to GTK3 from the github repository. They put and work "side-by-side".
