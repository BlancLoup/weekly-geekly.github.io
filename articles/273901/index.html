We join the tox community or install the node in 5 minutes

Greetings, oh greedy reader, priest and fighter for justice on the Internet! In our weapons replenishment, it is called tox. Now I will show you,% username%, its assembly and disassembly.



There is such a wonderful system called TOX. Here is their official website, but the profile on github.

About tox have already been written many times, but I will list the main advantages: DHT - there is no dependency on the central server, at least one node is enough, encryption on the client - Vasya writes to Masha, telegram Pasha will not read anything.


There are three installation options:
  1. manual assembly and configuration, with a description of the stages (discussed in the article below)
  2. automatic assembly and configuration with the script proposed by ksenobayt (new version)
  3. installation from the packages found by Darka (the ‚Äúdefault‚Äù configuration. An example of how to edit the config can be found in this article)


So, to begin with, download toxcore from the github: https://github.com/irungentoo/toxcore/archive/master.zip , unpack.

Installation is painted in the official documentation:
  • How to run a Bootstrap Node (wiki)
  • Compiling (github, INSTALL.md)
  • Instructions (github, README.md from bootstrap_daemon)

It is on the basis of the above pages that this tutorial was written and the node was launched.

Add the tox-bootstrapd user, under which everything will work, and copy the configuration file: 

 useradd --home-dir /var/lib/tox-bootstrapd --create-home --system --shell /sbin/nologin --comment "TOX DHT bootstrap daemon" --user-group tox-bootstrapd chmod 700 /var/lib/tox-bootstrapd cp tox-bootstrapd.conf /etc/tox-bootstrapd.conf cp other/bootstrap_daemon/tox-bootstrapd.conf /etc/tox-bootstrapd.conf


Edit this file: 

 vi /etc/tox-bootstrapd.conf


enable_lan_discovery = true - we have a server, and it‚Äôs in DC (or even a enable_lan_discovery = true on it!). So it makes no sense to start the node in LAN, change true to false .
tcp_relay_ports = [443, 3389, 33445] - ports to which our node responds. That is, if you port = 33445 default node port = 12345 to port = 12345 instead of port = 33445 , then your node will work on port 12345 and serve ports 443, 3389 and 33445, with enable_tcp_relay = true . In my case, I left enable_tcp_relay = true and indicated tcp_relay_ports = [12345, 33445] .

enable_motd = true - motd (Message Of The Day), if true returns the first 255 characters from motd , that is, if you have motd = "Vive le TOX" , then the subject who requested the motd will receive Vive le TOX
