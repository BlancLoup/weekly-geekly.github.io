For a full broadcast, we need: Gateway, digitizing the audio signal, and transmitting it to the server. And the server that this audio stream will distribute to everyone.

Step two, configure the gateway

Gateway that transmits an audio stream to a server. In this capacity, a regular computer will be used, which one hell will not turn off anyway. I would be happy to use it as a server, but, I'm afraid not you keep it not so much that habraeffekt, but at least a dozen listeners at a time - the ascending channel is very thin.
Software - SAM Broadcaster.

I'm too lazy to describe its setting, and there is nothing complicated there. Specify the microphone input as a source, set the flow parameters, and register the server address. Stop. What server?

Step three, start the server

We go to amazon (however, any server with access to the shell will do, but Amazon has good hourly rates and an easy opportunity to increase capacity), and create a new Instance.
Again, AWS setup descriptions are beyond the scope of this article. A good guide, in which everything is chewed, is here .

It also describes how to install ddclient to access the server by its normal name, not by IP. I advise you to do it right away, and to drive in the sftp and ssh clients the name is already.
After that, we need to install the Icecast server, which will distribute the audio stream.
Ubuntu such ubuntu, apt-get install icecast2 and all. The setting is described here.
