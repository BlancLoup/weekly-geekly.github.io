This approach is used by many, and so are we. But now I want to offer you one option that made my nervous system a little stronger. There are no commands, no waiting, I opened the telegram, I chose the right chat and ... everything! The data is already there!

It's simple, one method will save us:

editMessageText

Guess? Yes that's right:

  1. Create a new chat and add the "right people" and one bot to it.
  2. We estimate how many metrics we need to output. Using the sendMessage method, we create the necessary number of messages, saving somewhere the message id from the response of this command.
  3. We update metrics for crown / other magic by simply editing the message using the editMessageText method, somewhere in our backend.
  4. ??????
  5. Profit!

In fact, a message can be one thing, and all metrics can be divided through line breaks, I just like to separate them by separate messages. For example, in the last message I display the last update time, to understand that the data is relevant.
The great thing is that the telegram will update everything after editing the message, and if the phone is connected to the Internet you will always receive information as quickly as possible.

Somewhere outside the city, it is sometimes almost the only way to have relevant data.

And do not forget to add the "right people" before you create a message, since the updated data will not be available to them.
