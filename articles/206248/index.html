Bitcoin try to integrate with torrents

The idea is as follows: the author of some content, when creating a torrent file, gets the opportunity to specify his Bitcoin wallet, and those who have downloaded the torrent in their clients will see the corresponding icon, clicking on which will allow the author to thank him with a certain amount in cryptocurrency. In fact, it‚Äôs just that the Bitcoin wallet installed in the system will open or go to the website where you can copy the wallet number or scan its QR code for the subsequent payment.

Thus, the FrostWite torrent client developers are trying to use Bitcoin popularity, as well as with the maximum degree of convenience to organize the possibility of donate - and, as you can see, the content does not necessarily have to be licensed, but simply popular, which should to a certain extent please not only the original content authors, and those who would like to preserve their anonymity.

Technically, this is just about adding a record of this type: 

{ 'announce': 'http://tracker.site1.com/announce', 'info': { 'name': 'directoryName', 'piece length': 262144, 'files': [ {'path': ['111.txt'], 'length': 111}, {'path': ['222.txt'], 'length': 222} ], 'pieces': '6a8af7eda90ba9f851831073c48ea6b7b7e9feeb...8a43d9d965a47f75488d3fb47d2c586337a20b9f' }, 'bitcoin': 'bitcoin:175tWpb8K1S7NmH4Zx6rewF9WQrcZv245W?label=Content-Owner&message=support%20my%20content&amount=0.004' }


The developers' plans include the possibility of adding some lightweight Bitcoin wallet directly to their client, in order not to have to perform additional gestures for payment. Moreover, it is planned to organize the process of monetizing content so that the author can indicate the number of people who have to pay for the distribution before it can be really downloaded. How exactly will be implemented the last opportunity is not said.

To ensure technical compatibility with existing torrent clients, the ability to add Bitcoin metadata to torrent files should become the official part of the BitTorrent protocol - this issue is already discussed on the BitTorrent Enhancement Proposal mailing lists.
