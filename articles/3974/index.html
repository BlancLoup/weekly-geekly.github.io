New to Haiku (SVN)

Changes from r19800-r19900

Development of AHCI support has begun. Hardware cache reset support for SCSI has been added. USB stack scalability problems have been resolvedThe following file systems have been ported:
  • GoogleFS
  • Nfs
  • NTFS
Changes in the details

After a few small fixes, NetPositive began to serve the first signs of (networked) life under Haiku. After the network stack fixes that have resolved performance problems, IPv4 and ICMP should work faster. NetServer now starts at boot.

Owners of SATA drives should thank Marcus (Marcus). He began work on the AHCI module. At the moment, the module is already present in the Haiku experimental images by default and can detect connected SATA drives. In addition, BGA added hardware cache reset support for SCSI devices in Haiku, based on its previous work for yellowTab.

After a large number of corrections made by Michael Lotz USB stack should become more stable to use. Disconnecting and reconnecting devices should no longer cause a stack crash, as drivers are notified of these events.

GoogleFS was ported from BeOS to Haiku. Using the usual file search for Google queries will allow you to create many original applications.

Among other new products, porting nfs to Haiku is worth noting. Although support for mounting is not working yet, but development is going on quite actively and soon this possibility will appear.

And the last. Thanks to Troeglazov Gerasimu, support for the NTFS file system was ported to Haiku (the implementation of ntfs-3g was used as the basis). So now there is the possibility of working with Windows drives. Note that recording support is currently disabled, for reasons of data integrity.
