Search engine optimization. Part 1 - internal factors

Recommendations on html-code, layout and a little - on copywriting. Actual for February 2009

This is not just another list of ‚Äú101 tips on how to achieve an ideal in that‚Äù, which is full on the Internet. This is "insider information" SEO-company, leading more than 600 projects. Lists what we pay attention to when starting search engine optimization. Of course, not everything can be realized (for example, the site's engine does not allow something, but somewhere the points contradict (!) To each other). Nothing wrong. If more than half of the recommendations can be implemented, then this is already very good!

Immediately let's agree on agreement or disagreement with the list. The goal of our company is Top precisely in Yandex. (Google and Rambler most often also increase - because many tips of a general nature). Yandex's algorithms are closed, but they can be tested by experiments. Our recommendations are what we were able to probe (actual in January 2009). You could have the opposite result. And this contradiction needs to be correctly understood: these are simply the results of various experiments on the black box.

PS Promotion is automatic (600 handles are difficult to move). All checks are automatic. The robot is weaker than a human, therefore it is difficult to check some items. For example, closing of some information with the display: none style or the similarity of text color and background color.

  1. Main link: TITLE tag, H1-H3 tags, STRONG tag, title attribute in IMG tag. There should be key phrases in normal form. The rest of the text available for indexing should contain at least 2-3 key phrases in normal or indirect form.
  2. The amount of text available for indexing is at least 600 characters (not bytes). When calculating cut text relating to the design, navigation, etc.
  3. In the code, NOINDEX blocks are undesirable, links have undesirable rel = "nofollow" blocks.
  4. After the closing HTML tag there should be nothing.
  5. The text is desirable to have a unique one. Check on copyscape.com or immediately in Yandex.
  6. The text should be without typos, without ‚ÄúAlbanian‚Äù slang, without replacing Russian with similar Latin letters.
  7. TITLE should not contain unjustified special characters, like ^, ~, *, +, \, | etc.
  8. TITLE should not contain more than 15 words. Please note: a contradiction with clause 1 may occur if you promote one page for a large number of requests.
  9. H1-H3 tags should not be too close to the end of the document or to the closing HTML tag.
  10. External links should not be too close to the end of the document or to the closing HTML tag (easily computed basement reference block).
  11. In the text being indexed, illustrations are desirable (of course, with key phrases in the title attribute). In this case, the pictures should be visible to the user, and not hidden in the size of 1x1 pixels.
  12. If possible, the URL of the page should contain the key phrase in translit or in English. For wiki engines, you can use the key phrase in Russian in the URL.
  13. There should be no text merging with the background. (A manual check is desirable.)
  14. An existing robots.txt file should not close too many areas on the site.
  15. If the site has good traffic, then place the counters with freely available statistics.
  16. It is advisable not to use flash to store content (despite all the assurances that it is well indexed). About frames in general we are not talking.
  17. Correct server responses for pages: appropriate use of codes 200, 403, 404. Having the correct value last-modified.
  18. The percentage of key phrases in the text being indexed should not be more than 15. We recommend much less, but there is no exact data. But it is generally accepted that a lot is harmful.
  19. The last rule, which is reasonable to make the first and even zero. The current page presence in the index of Yandex, Google and Rambler. If the page you have created for a long time is not indexed, then perhaps the site has a bad SEO karma. Optimization efforts may be in vain. Problems can be very specific and will require vast experience to solve them. For automatic operation of such a site is not suitable.
