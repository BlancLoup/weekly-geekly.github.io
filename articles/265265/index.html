YouTrack 6.5 - Bug Tracker for the whole team

Good day, Habr!

I hasten to tell you that a new version of our bug tracker has just been released - YouTrack 6.5!



In the new version, we focused on improving UIX, completely redesigning the administrative part of the product, added several important integrations and simplified import from Jira to one click.



What is new in YouTrack 6.5?


  1. Integration with JetBrains command products
  2. Integration with BitBucket and GitLab
  3. Import from Jira in one click
  4. New design administrative part
  5. Project Creation Wizard


Improvements

  1. Improved GitHub Integration
  2. New tab VCS changes
  3. Support brunch in TeamCity
  4. Improved Dashboard
  5. Workflow language enhancements
  6. Improved mailbox integration
  7. Bug-fix versions are now included in the Stand-alone server license.
  8. @username with embedded notifications

And now a little more


Integration with JetBrains command products


Most recently, we wrote about the release of a new product Hub 1.0, an account management system and access rights, which is the single entry point for all JetBrains team products, including YouTrack, Upsource and TeamCity.
YouTrack 6.5 can be connected to an external Hub (by default, YouTrack version 6 and higher comes with a built-in Hub). Hub also allows you to connect Upsource (code review tool) to it, to get a single entry point to both products, a common base of user accounts, groups, access rights, a unified dashboard with widgets from YouTrack and Upsource, as well as integration between products , available by default.
Read more about the Hub and the integration between YouTrack and Upsource here . TeamCity will also soon join the team with a special plug-in, which is scheduled for release in autumn 2015. At the moment, you can connect only the stand-alone version of YouTrack to the Hub, but we are already working on supporting the cloud version.

Integration with BitBucket and GitLab

Now you can set up native integration with Bitbucket or GitLab, which allows you to:

  • Apply any commands from YouTrack directly from the comment to the commit, mentioning the ID of the desired task. All changes to the task will be applied on behalf of the author of the commit.
  • Use commit hashes in YouTrack tasks to directly link to Bitbucket / GitLab commits.
  • See all VCS changes related to the task in a separate tab.
  • Filter tasks that contain changes to the VCS using the special has: {vcs changes} query.



Please note that in YouTrack 6.5 only integration with gitlab.com is available. Integration with GitLab CE and GitLab EE will be added in one of the next minor versions.

Import from Jira in one click

Now you can import your projects from Jira to YouTrack with all of their history, comments, a list of observers, voices, customized fields, attachments, authors, performers, and even added one-click hours worked. To do this, you just need to add a link to Jira and enter your username and password. YouTrack will automatically create groups for task authors and performers from Jira, of course, if you have the necessary rights in Jira projects. You can read more about importing from Jira in our blog post (in English). Please note that import is supported from Jira version 5+.

New design administrative part

We completely reworked the UI of the administrative part of YouTrack to simplify the process of setting up and administering the tracker for new users. For existing users, some new solutions may not seem obvious. We welcome your feedback and comments on the new UI administration of projects, custom fields, workflow, integrations, etc.

Project Creation Wizard

To create a project in YouTrack, you just need to fill in a couple of fields and add, if necessary, your teammates. YouTrack will automatically create a team for your new project, add it as the value of the field Artists and give the team the necessary rights to work on the tasks in the project. You can read more about the project team here (documentation in English).

Also note the new project list. A search with filtering by project name, groups, etc. has been added to it. Now it is easier to edit the project, and you can add a group or an individual user to the project in one click.



Improvements



GitHub Integration

YouTrack has long supported native integration with GitHub, similar to the new one with GitLab and Bitbucket. In the new version, you can link one GitHub repository with several projects in YouTrack. All changes from GitHub are also visible on a separate tab ‚ÄúVCS Changes‚Äù. Speaking of her ...

New tab VCS changes

Changes from all version control systems are now visible on the special tab VCS Changes. You can also filter all tasks into changes in VCS using the has: {vcs changes} query.

Improved Dashboard

Now you can stretch or narrow widgets, and also open widgets on the whole screen by double clicking on the widget. We also improved the search on the Dashboard, added saved searches and tags.


Support brunch in TeamCity

In version 6.5, support for feature branches in TeamCity has been added to YouTrack.

Workflow improvements

  • Now you can track changes not only to tasks, but also comments to them.
  • Scheduled rules or triggered by certain events are now launched on behalf of the special YouTrack Workflow user.


Improved mailbox integration

Using YouTrack as a helpdesk? Then YouTrack 6.5 has something useful:
  • New variable $ {reply-to} has appeared.
  • If the user has several email addresses, now they will be associated with one profile.
  • The content of the email header in square brackets can be ignored.
  • Now it is possible to disable a specific rule.


Bug-fix versions included in Stand-alone server license

Starting from version 6.5, all minor versions 6.5.x are included in the Stand-alone license. That is, if your subscription was active at the time of release of version 6.5, then you will receive all minor bug-fix versions for free, even if your subscription expires at the time of their release.

@Username with embedded notifications

Now notifications about a mention of a user name in comments (@username) are built in system notifications. Notifications can be configured in the user profile, and they are no longer part of a special workflow.

The next batch of servers will be updated next Monday. We plan to complete the transition to the new version within 6 weeks. If you can not wait to try version 6.5, write to our team those. support, and we will include you in the next upgrade.
