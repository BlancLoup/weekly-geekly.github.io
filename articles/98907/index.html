Open XMPP testing

All attempts by the brainfucker to delay the launch of open testing have failed for another week. Therefore, we bring to the attention of the community an alpha version of the XMPP protocol support. We hope to have time to collect feedback and detect errors in order to prepare for the official release.



QIP Miranda Pidgin Adium Other


To connect to the XMPP VKontakte server, you need to use the following data:

Account: username@vk.com (short name or idxxxx)
Password: password from VKontakte profile

We added several commands that can be entered directly during the chat with the other person:

\ history - shows the history of the correspondence (using the space, you can specify the number of displayed messages);
\ delete - deletes the last message;
\ en - translates further entered text into English (Google Translate);
\ ru - translates further entered text into Russian.

The server supports TLS encryption, SASL authentication and the following extensions: XEP-0030, XEP-0092, XEP-0012, XEP-0202, XEP-0199, XEP-0158, XEP-0054, XEP-0085. S2S support ( ability to communicate with users of other XMPP servers ) is not yet available.

The statuses from the client are automatically exported to the VKontakte profile. That is, users who have configured the export of VK status to Twitter can maintain a microblog by updating the status in the client. The first testers disagreed about the need to export all statuses from the client. There is an idea to send only those that begin, for example, with a certain symbol.

We are waiting for feedback from active users of XMPP.

UPD Due to problems with encryption, as a result of which some users could not connect, it was temporarily disabled. In some clients it is necessary to allow in settings to connect to unencrypted connections. Temporarily.
