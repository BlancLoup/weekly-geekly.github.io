Vulnerability allowing to hear the interlocutor with the microphone turned off



Vulnerability allows you to activate the microphone turned off the interlocutor without his knowledge. In this case, the microphone icon will remain crossed out.

The error is in the function of automatic restoration of communication in the event of a break in the group call mode. Tested on Windows versions 5.5.0.124, 5.6, BETA 5.7.0.123.

Steps to reproduce the error:
  1. Call or receive a call.
  2. The interlocutor mjut microphone.
  3. Turn the call into a group by typing in the chat / add echo123
  4. Turning off the Internet and waiting for the red message "Hold on while we try to get the call back".
  5. We connect the Internet and wait for the call to be restored.
  6. We hear the interlocutor, despite the fact that he has a microphone icon looks crossed out
