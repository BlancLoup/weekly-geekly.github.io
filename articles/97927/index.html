Zingaya - click-to-call widget on your site



Hello! We are pleased to announce that the Zingaya service we have been working on for the last year has finally been launched.
Currently, there are a large number of ways to communicate with site visitors - from forms that need to be filled out and sent to live chat and callback widgets. All these methods are used and in some cases really are the best choice, but we have set ourselves the task of creating a more convenient and understandable method for the visitor of the site, which also ensures its anonymity.

About the service


So, Zingaya is a web service that provides a new channel for communicating with site visitors by placing the widget code (similar to the youtube embed code) on a web page. Using a widget, a site visitor can call the widget owner in one click. The call from the widget is redirected to the preset landline or mobile numbers, to Skype or IP telephony using the SIP protocol. The caller needs a microphone.




Technology


The client part is presented in the form of a widget created using Flash. Now we will explain why this technology was chosen at the moment, until the prophets, predicting the demise of Flash and the realm of HTML5, came to a halt. There were several reasons:
  • The prevalence of technology. In Flash, this figure is about 98% of all computers connected to the Internet.
  • Ability to work with a microphone and the availability of data transfer protocols in real time.
  • Simplicity for the caller so that it would not be necessary to allow the browser to download and run some applet ten times.
  • Cross platform

Silverlight is not yet suitable because of points 1, 4 and partially 2. HTML5 is not suitable because of points 1 and 2. Well, Java is not suitable because of point 3 and partially 1.
Many may wonder why we think that we are so unique. In fact, analogues exist, including even in the opensource world (red5phone), but none of them can offer support for the new RTMFP protocol and active echo cancellation (in case the caller has a laptop and does not have a headset). We use a proprietary media server that supports the things listed above, and is also highly stable and productive. We have been working on it for more than a year and have studied this issue very well: the experience with our previous project flashphone / flaphone was not in vain.




We started the service right now, as we decided that the sound quality reached the required level. For business, various useful functions are supported, such as recording conversations, working hours, voice mail, linking the widget to the site name (to protect it from being placed on another resource), filter countries from which you can call using the widget.

Examples of using


For whom did we do this service? For two categories of users. The first are active network users who want to always be available. The widget can be placed, for example, on your blog or on a personal page.
The second category is almost all companies that have a website. The widget, for example, can be configured to call the call center of the company. This usually increases the conversion of users into buyers (for e-commerce) and increases user loyalty, as it is sometimes more convenient to communicate with a company representative with a voice.
You can test the widget right now in our blog or create your own after registration .
