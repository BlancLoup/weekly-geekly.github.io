Tuning the Opera menu

Hi% username%! I heard many times how people asked to remove the built-in mail or other features and did it several times myself, to which every time everyone who tried was refused. But it doesn’t prevent them from independently removing from the field of view all that we don’t like?
First, some basic information:
  • The standard menu configuration file is stored in C: \ Program Files \ Opera \ ui \ standard_menu.ini (toolbars in C: \ Program Files \ Opera \ ui \ standard_toolbar.ini);
  • In Windows 7, the user menu configuration files are located in C: \ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \ menu \ (respectively for toolbars in C: \ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \ toolbar);
  • The final profile is formed by imposing custom settings on the standard ones. you can not override only the necessary part;
  • You can select the current style in the “General Settings” (Ctrl + F12) -> “Advanced” -> “Panels”. I recommend before any manipulations to create a copy of the edited profile and only change it ;

The updated configuration (now does not depend on the version of the browser and will not be erased when updating). To install, unzip the archive to the% AppData% / Opera folder (for Windows 7 in C: \ Users \ Andrey \ AppData \ Roaming \ Opera \ Opera \)
Some changes:
  • The “Exit”, “Opera Unite” and “Synchronization” items are removed from the menu (the last 2 are duplicated with the button in the status bar);
  • The item “Show menu” has been moved to “Settings” -> “Quick settings”;
  • The button "Mail and chat ..." moved to "Settings";
  • “Design” is transferred to “Settings”;
  • The "Full Screen" button has been added to the "View" toolbar on the status panel;
  • Removed submenu responsible for widgets. Instead, you can use “Settings” -> “Manage widgets”;
  • Development tools are moved from the “Page” to the root of the menu;
  • With the “Page” submenu, everything related to copy-paste text has been removed; this functionality is available through the context menu via PCM;
  • In the submenu "Page" moved print settings;
  • The context menus are significantly reduced by removing little-used items (“Set as desktop background”, etc.)
