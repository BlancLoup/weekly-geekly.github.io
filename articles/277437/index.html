Microsoft fixed multiple bugs in the system components of Windows 8.1 and Server 2012

Microsoft has released 14 updates for Windows 8.1 and Server 2012, which fix bugs in their various components, including kernel executables, system drivers, and other system components. Several updates also concern Windows 7 SP1, for example, update KB3118401 has been released for these operating systems, which delivers so-called updates for them. Universal C Runtime (Windows 10 Universal CRT). It appeared in Windows 10 and without this installed application update, compiled with its support, will not be able to run on these operating systems.



Another update KB3126033 fixes a bug in the Remote Desktop Protocol (RDP) service. The error is related to the use of the Restricted Admin mode when connecting to a remote computer, and the connecting party may receive an error message. The system file Winlogon.exe is updated, the system process of which is responsible for the Windows login process.

The KB3128650 update ( Access to COM + role-based security is denied in Windows Server 2012 R2 ) fixes a bug in an API function called GetEffectiveRightsFromAcl . This feature is frequently used in Windows Server 2012 and can lead to access denial errors during operations with SIDs. Such critical system files as the Winload.efi EFI firmware file, as well as the Ntdll.dll and Ntoskrnl.exe kernel files are subject to updating.

Update KB3130896 ( System crashes with Stop error 0x00000139 in Windows 8.1 and Windows Server 2012 R2 ) fixes bugs in the network file server driver Srv.sys, which cause the BSOD screen with code 0x139 on the server when one or more clients attempt to connect to the network printer with using the SMBv1 protocol.

Update KB3133924 ( "Code 10 Device Cannot Start" error for EHCI USB Controller devices in Device Manager in Windows Server 2012 R2 ) fixes bugs in various Windows USB drivers that lead to an error message when connecting to an EHCI USB controller on a PC. The update is addressed to the drivers Usbd.sys, Usbehci.sys, Usbhub.sys, Usbohci.sys, Usbport.sys, Usbuhci.sys.

The KB3134812 update ( FSRM GUI in Windows Server 2012 R2 ) fixes a bug in the Srmgui.dll system library that prevents the normal installation of file server settings using the File Server Resource Manager (FSRM) GUI.

The KB3134815 update fixes a bug in the CryptDuplicateKey API function of the Rsaenh.dll cryptographic library, which prevents the client from normal operation when creating a duplicate key with a 40-bit RC2 cipher length.

The KB3121261 update ( LUN reset in Windows Server 2012 R2 ) is addressed to the Classpnp.sys and Storport.sys disk class drivers. The error leads to a violation of the process of replicating server data.
