Qubes 2 based on Xen hypervisor released

A few days ago, Qubes 2 was released, aimed at maximum security in use. At the core of Qubes is the idea of ‚Äã‚Äãrunning separate virtual machines to ensure the isolation of user applications. Xen is used as a virtualization system.



All Qubes user applications are categorized. A separate virtual machine is created for each category and its own execution priority is assigned. As a result of this, even potentially vulnerable applications cannot access the main subsystems that operate in strict isolation from the user.



All virtual machines have their own graphical server, but the user works in a single working environment. KDE and xfce4 are currently supported. An interesting feature is the support of Windows as one of the user domains. This means that now windows applications will run in the same style as the rest, as transparently as possible for the user.



To install Qubes 2, you need a 64-bit Intel / AMD processor, at least 4GB of RAM, and 32GB of free disk space. It is also recommended * the presence of SSD, support for VT-X / AMD-V, support for VT-d / AMD-IOMMU, and a graphics card from Intel. Video cards from nVidia / ATI may have problems. A portable version of Qubes 2 for USB-drives is available.



In general, the system looks quite interesting. Extensive opportunities for isolation of the environment and a significant simplification of the overall configuration is an excellent choice for experiments.
