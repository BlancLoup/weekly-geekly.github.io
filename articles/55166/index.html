Subversion 1.6 and TortoiseSVN 1.6 released



What's new :
  • Improved authentication data management (saving authentication data to KWallet or Gnome Keyring, saving keys to SSL certificates, asking for confirmation before storing passwords in an unencrypted form)
  • New URL format - support relative root paths (root relative URLs)
  • Svn: externals extension: support for svn: externals files
  • Definition of tree conflicts - conflicts at the directory structure level
  • Improvements in the way data is stored - the repository will take up less disk space
  • New Python-bindings
  • Improved online conflict resolution in command-line client
  • The --set parameter of the update command supports the new exclude value, which allows you to exclude certain branches from the working copy.
  • Svnserve logging support
  • The new HTTP URI syntax allows you to access specific revisions: 
      http: // host / repos / path? [p = PEG] [& r = REV]
  • Enhancements to the on-line client: the log command now supports several -r and -c parameters in one call; new parameter --trust-server-cert
  • API changes
  • Bugs fixed
