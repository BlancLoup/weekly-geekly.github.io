Microsoft Crm 4.0 WebServices Error

Problem:
In MS CRM 4.0, I encountered the problem of placing my site in the virtual directory of the MS CRM site.

The following error was issued:
Microsoft.Crm.WebServices.Crm2007.MultipleOrganizationSoapHeaderAuthenticationProvider, Microsoft.Crm.WebServices, Version = 4.0.0.0, Culture = neutral, PublicKeyToken = 31bf3856ad364e35 'does not exist.
Parameter name: Microsoft.Crm.WebServices.Crm2007.MultipleOrganizationSoapHeaderAuthenticationProvider, Microsoft.Crm.WebServices, Version = 4.0.0.0, Culture = neutral, PublicKeyToken = 31bf3856ad364e35

Decision:
It was found on the Dynamics forum.
The solution is completely delusional (it is not clear why we should do this), but it was the only way that we managed to launch our website. It is necessary to register the Microsoft.Crm.WebServices.dll library on the MS CRM server (in% of the CRM web installation path% / bin path)

gacutil / i Microsoft.Crm.WebServices.dll

PS If you have not found gacutil.exe, then in Explorer you can simply drag the necessary DLL into the WINDOWS \ assembly folder.
