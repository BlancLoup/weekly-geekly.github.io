Co-ownership and domain management

In any web project, the domain plays a key role and is one of the most valuable things.
If a startup has more than one owner, then an unpleasant question always arises on whom to issue a domain, since domain registrars do not offer the possibility of establishing several owners in a normal way.

And today, in the information age, even the majority of lawyers have no idea that there is a domain, how to properly arrange it for an enterprise, who should manage it, and ‚Äúwhere does it have a button‚Äù.
And this question is more acute at the initial stage of a startup, where the extra costs for an enterprise are useless, so what can be done in this case?

Alternatively, you can write receipts, enter into contracts and then, if necessary, try to defend your rights in court with these documents.

But it is much better not to lose control from the beginning, and I‚Äôll tell you about it later.

In this article I want to share with you my personal experience of an unusual solution to this issue, to whom it is interesting - I ask for cat.

Task

The situation is as follows: after registering a domain, whoever has a password for managing a domain, he is the owner, that is, he can change any information, including and about the owner.
Task: to split the authority to manage the domain.
A bad decision is to give control to everything; who first wants one and will change the password.
A reliable solution is to divide the password between the owners, in accordance with the size of shares in the project.
For example, there are 3 owners with equal shares, each comes up with their own six-digit password.
Thus, it turns out the situation when something can be changed only by a collective decision, and if there is no such decision, then everything will remain as it was originally intended. Problem solved.

There are nuances

In the absence of one of the participants in the scheme, the missing piece of the password can be recovered only by going through its password using the other known parts.

This method is suitable for most registrars and domains of the first level, where management is carried out by login and password, for example, godaddy.com.

What is the email, phone, address and full name?
So that no one individually could use this data to restore the password to the account, block the contact information in the same way:
Email - you need to register the mailbox on gmail and share the password on the same principle.
Phone - enter a non-existent number, for example, that there were not enough numbers.
In the full name, you can simply enter the names of all owners, or only last names, or only first names using a hyphen.

Which DNS to specify, and which hosting to choose?
Try to approach this issue very carefully, using a similar principle of access sharing. It is better to use DNS and hosting with the same provider and separate passwords to the DNS panel and passwords for access to the server. But remember that this choice can remain for a long time.

Conclusion

In any case, each project is individual and has its own technological characteristics, which, of course, must be considered when using this model.

Again, you need to understand that people work everywhere, everywhere there is a human factor,
so remember to always use common sense when making such decisions.
