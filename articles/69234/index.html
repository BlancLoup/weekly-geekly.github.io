Fuzzle CMS 2.6 - create Flash-sites without Flash

I would like to acquaint users with the development of our project Fuzzle CMS to create managed Flash-sites about which I already wrote about 4 months ago .

1. We tried to develop SEO-direction - the names and addresses of the pages became smaller, the ability to set TITLE, KEYWORDS and DESCRIERTION for each page was added. Developed connecting Google Analytics and other counters.

2. We added the ability to arrange editable blocks not only on every page of the site, but also at the level of the site design as a whole. Design can be loaded not only from SWF, but also from PNG and JPG-files, and create a menu by placing blocks. In the end, it turned out that you can create a Flash site without using Adobe Flash at all .

3. We launched the extension directory and fully released the API for developers.

4. We worked on usability and fixed many bugs, including those mentioned by Habr users, for which we thank them very much:
- created a page manager, as previously it was not entirely clear how to create them ( proc )
- decided a bug with reloading pages in Safari 4 and Chrome ( Scala )

Well, they took into account many other things, of course.

In addition, since we repeatedly thought that the price of the system was quite high, then for those who seriously consider Fuzzle CMS from the standpoint of developing sites on it , we wanted to make the following suggestion:
During the next week (September 11-18) we offer a 90% discount, i.e. the cost of the system will be 1850 p. Those who wish are required to contact me by mail and send the domain name for which the license will be issued. The number of licenses offered is limited.

For those who want to try first, as always, there is a local full-featured demo version, a demo site on the Internet and a 10-day test license for any domain ( all of the demo are here )

Good luck to everyone in development, and I am waiting for your questions!
