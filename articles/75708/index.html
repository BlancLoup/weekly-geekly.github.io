Ubuntu 9.10 and the disappearing menu

The topic promises to be quite short, I hope it will help someone.

For several months I have been using Ubuntu. After the release, I switched to 9.10, and a couple of days ago there was a problem. Not to say that everything is absolutely terrible but unpleasant. The tab with applications disappeared, which seemed to me rather strange.

Same problem?

First of all I tried to solve the problem through the ‚ÄúChange menu‚Äù gnome, but I could not start it. Got into Google, found a couple of posts with a similar problem, but no solution was found in them. But prompted where are the menu settings.

Namely, '/home/%username%/.config/menus'.

I only found empty files there. Well, it is not surprising that an empty menu is generated from empty files and there is nothing to show.

The most ingenious solution that occurred to me to simply demolish the above folder. It helped. I just had to edit the menu for a new one, and then I understood where the problem came from, the menu editor hangs as soon as I try to edit the links added to the menu through Wine, after which the problem repeats.

The output was found in manual editing from under sudo files in the '/ etc / xdg / menus' folder.

That's all, I hope to see you soon and not only in the sandbox;)
