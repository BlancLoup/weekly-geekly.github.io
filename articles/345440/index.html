3CX V15.5 SP3 alpha and 3CX Session Border Controller Update

At the end of the year, we were not idle and prepared several updates that you can test if you suddenly become bored during the holidays. Updates two - alpha-version 3CX v15.5 SP3 and update 3CX SBC (Session Border Controller).

3CX V15.5 SP3 Alpha


The update 3CX v15.5 SP3 alpha, released just before the Catholic Christmas, the following features are implemented:

  • IPv6 protocol support ( dual stack )
  • Support for .NET Core 2.0, .NET Standard 2.0 and Windows Net framework 4.6.1
  • System SSL / TLS protocols are now enabled by default.
  • Web conferencing enhancements
  • Special account for hosting administrator

We remind you that you should not use this version on work systems. Besides:

  • This version cannot be upgraded to beta or release.
  • Configuration backups of this version will not work in newer versions.
  • Technical support for this version is also not available.

3CX Session Border Controller


3CX SBC is a very popular utility (server) from 3CX, used to provide better communication, security of calls and traffic saving. Or simply for a quick and convenient connection of office IP-phones to the cloud installation of the 3CX PBX on Google, OVH, your own free 3CX hosting or on your cloud infrastructure.



The latest update 3CX SBC significantly increases server performance. Firstly, the number of RTP ports by default is increased from 8 to 32. Secondly, the SBC service is now automatically restarted when an emergency stop occurs. Other improvements and improvements:

  • The SBC service waits for initialization until it receives an IP address. Now there is no need to organize a connection via RDP or SSH in order to restart the service manually.
  • Disabled auto-restart service at 3:00 at night.
  • On Mondays, SBC updates are automatically checked.
  • Merged repositories of the system 3CX and SBC. Now PBX 3CX, SBC Debian 9 and SBC Raspberry Pi use a single repository for installation and updates
  • Timestamps have been added to the log file to quickly find problems.
  • Raspberry Pi supports up to 20 IP phones on the client side of the cloud PBX

Change Log .

The update is already working on Debian 9 and Raspberry Pi. Debian 8 users must remain at the previous SBC version until the OS is upgraded to Debian 9.
