Network settings from FreeRadius via DHCP


The task came to adjust the issuance of IP addresses to subscribers. Conditions of the problem:

  • We will not give a separate server for authorization - you will manage;)
  • Subscribers should receive network settings via DHCP
  • The network is heterogeneous. These are PON equipment, and ordinary switches with configured Option 82 and WiFi bases with dots
  • If data do not fall under any of the conditions for issuing IP - you must issue IP from the "guest" network

From the good: there is still a server on FreeBSD that can ‚Äúwork‚Äù, but it is ‚Äúfar away‚Äù;), not ‚Äúright on this network.‚Äù

There is also a wonderful Mikrotik device. The general scheme of the network is approximately as follows:



After a little thought, it was decided to use FreeRadius subscribers to issue network settings. In principle, the scheme is normal: we turn on the DHCP server on Microtick, on it the Radius Client. Configuring a bunch of DHCP server -> Radius Client -> Radius server.

It seems to be not difficult. But! The devil is in the details. Namely:

  • When authorizing PON OLT according to this scheme, a request arrives on FreeRadius with a User-Name equal to the MAC address of the head station, Agent-Circuit-Id equal to the MAC PON Onu and a blank password.
  • When authorizing with switches with option 82, a request is received on FreeRadius with an empty User-Name equal to the subscriber's device's MAC and the Agent-Circuit-Id and Agent-Remote-Id additional attributes containing, respectively, the relay switch's MAC and the port to which the subscriber is connected.
  • Some subscribers from WiFI points are authorized via PAP-CHAP protocols
  • Some subscribers with WIFI points are authorized with a User-Name equal to the MAC address of the WIFI point, without a password.

Historical background: what is "Option 82" in DHCP?

These are additional options for DHCP that allow you to send additional information, for example, in the Agent-Circuit-Id and Agent-Remote-Id fields. Usually used to transmit the relay switch MAC address and port to which the subscriber is connected. In the case of PON equipment or WIFI base stations, the Agent-Circuit-Id field does not carry any useful information (there is no subscriber port). In this case, the general scheme of the work of DHCP in this case is as follows:



Step by step this scheme works like this:

  1. Subscriber equipment makes a DHCP broadcast request for network settings
  2. A device (for example, a switch, a WiFi base station or PON) to which the subscriber equipment directly connects "intercepts" this packet and changes it, incorporating additional options 82 and Relay agent IP address into it, and sends it further along the network.
  3. The DHCP server accepts the request, generates a response and sends it to the relay device.
  4. Relay device forwards the response packet to the subscriber device

So simply all this of course does not work, we need the appropriate configuration of network equipment.
