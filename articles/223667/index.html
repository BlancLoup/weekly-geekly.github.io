Mail.Ru for business: everything you wanted and did not hesitate to ask



You may remember how we announced the Mail.Ru for Business project a few months ago .

If any of you missed, Mail.Ru for Business is a free service that allows you to create up to 5,000 corporate addresses in a domain and gives users access to the full range of Mail.Ru Mail resources. At the same time, we provide reliable protection against spam and viruses and offer unlimited mailbox volume. Mail can be used through third-party clients that support IMAP, POP3 / SMTP protocols - Outlook, The Bat, as well as through mobile applications for all popular platforms. Along with the mail, other tools are also available to users, such as the Cloud for virtual data storage, the Calendar for organizing work, and the Agent for instant messaging with colleagues.

Login by mail.mydomain.tld and through the form on the website

At the time of launch, we provided access through the main Mail.Ru page, as well as through a special page on biz.mail.ru. Now you can set up logging in to Mail for your users using the mail subdomain (or any other of your choice) and through the form on your website.

To log in through a subdomain you will need to create a CNAME record on a subdomain with the value biz.mail.ru.

API for connecting a domain and creating users

Together with colleagues from Reg.ru, Setup.ru, Webnames.ru, we implemented the ability to connect a domain with one button directly from the partners interface. This feature has become available with the opening of our API in public access. In addition to connecting a domain, any API client will be able to create users on the domain, and, therefore, provide its clients with mail functionality with mail account management from one familiar place.
The usage documentation is located at https://biz.mail.ru/developer . Connecting to the API and getting the keys while in manual mode - if you want to try, write here in a personal or to the address o.parinov@corp.mail.ru.

Box and Domain Aliases

Most often, aliases are used as shorter versions of the box name or for temporary use on questionable services.

For each domain user, we can configure up to five mailbox aliases. Letters sent to the alias address will be delivered to the main address. Also, by sending a letter from the Mail web interface, you can select the alias address as the sender.

We also provided the ability to add an alias to an entire domain . If you have several domains for the same purpose (for example, the same domains in different zones), you can duplicate mail acceptance functionality. To do this, select the primary domain, connect it to the system, and then add additional ones in the section "Domain Aliases". Then even in case of an error in the perception of an address by ear, the letter will arrive to the correct addressee - to the appropriate box in another domain.

Migration from other mail services

Literally from the moment of launch, many users asked us a very reasonable question: ‚ÄúHow can I pass on to you and not lose everything acquired by overwork?‚Äù. Of course, we understood how important the archive of correspondence is. Therefore, we have developed a migration system - the ability to configure the collection of mail from the old mail server for new mailboxes.

When you create a new mailbox, you can install the start of the collector by confirming the use of the same password or by setting the previous password - and then all the letters of this mailbox will be transferred from the old server. The settings and the inclusion of the transfer capability are made in the "Migration" section. More details can be found here .

We offered the easiest and fastest way - adding users from a CSV file, which contains at least the login and password of the box. Subsequently, we also added the ability to set first and last names (third and fourth columns), a list of incoming groups separated by commas (fifth column) and a password from the old server to enable migration (sixth column). Signature DKIM

Signature DKIM allows you to confirm the authenticity of the sender of the letter and increases the "deliverability" of letters. Communication with the sender is established using a digital signature, which is then verified on the recipient's side. Most mail services, including Mail.Ru, use the result of the DKIM scan to filter spam.

Now we have the ability to install your own DKIM-signature for letters sent from your domain. DKIM signatures are configured in the Server Status section ‚Äî all you need to do is set the necessary TXT record in the DNS editor of your domain. (By the way, in one of the following posts we plan to tell you more about how we did it).

User Activity Report

Every action of the corporate user in the Mail is recorded and recorded. Now these records are available to the administrator in the domain management interface on the user page. The administrator will now always be able to view the history of events taking place with the box and notice undesirable activity in time.
