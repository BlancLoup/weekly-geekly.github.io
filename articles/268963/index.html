HTML5 Video and Audio support in new Vivaldi 1.0.300.5 browser build

Hello!

Today's assembly is in some sense special. But not because of the assembly number, which passed for the 300 mark, but for a completely different reason. In particular, in this version of the browser, the long-awaited support for proprietary codecs HTML5 MP4 (H.264 / MPEG-4AVC & AAC) Video and MP3 Audio finally appeared. This will allow to solve a lot of problems when working with websites and services that actively use these technologies. Now about this in more detail below.

HTML5 Video and Audio support

From now on, Vivaldi browser supports a wide range of media formats. Thus, if you have previously experienced some difficulties while working with websites and services, which are abundant in media content, now these problems should not be. If you still encounter some difficulties - let us know about it so that we can fix them before the beta version, which is already very, very soon.

Media support in Vivaldi has been tested in recent versions of Windows and MacOSX, so some difficulties are possible in earlier versions of these operating systems due to the absence of some important components. Ubuntu Linux users need to install the chromium-codecs-ffmpeg-extra library and reboot the system so that Vivaldi can use the new features. For users of other distributions, we have prepared small instructions , following which you will be able to use the new features of Vivaldi.
