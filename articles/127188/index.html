Java hosting - GlassFish 3.1.1 in Jelastic Cloud

The latest update of the cloud platform Jelastic is out of schedule and is mainly aimed at supporting GlassFish. Honestly, the release of the intermediate release is due to numerous requests for support of this server. Naturally, we wanted to please our users. Oracle developers appreciated our initiative and even wrote about it in a corporate blog .

To create an environment with GlassFish, simply select it in the drop-down list in the dialog box for creating a new environment. After creating the environment, you can deploy your application through the deployment manager in Jelastic or in the standard way through the GlassFish admin area. To access the standard GlassFish admin panel - open your environment and just add the port: 4848. You will be taken to the admin panel of the server. In the future, we will make a more convenient transition on the button.

GlassFish is more voracious in memory when compared to Tomcat. But this is not strange, since it is a more technologically complex and more functional server. Of course, there are still questions on the integration of GlassFish into Jelastic, which require our attention. But I am sure that with your help these issues will be solved much faster.
