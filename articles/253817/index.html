Weekly build Vivaldi 1.0.135.2

Hello!

Today we present the new weekly build of Vivaldi browser at number 1.0.135.2 with important changes. The main news is that the browser core has been updated to version Chromium 41. You can read the full list of changes in this kernel here .

Plus, we started the implementation of the browser auto-update system, in this assembly it will be available so far only to users of the 32-bit version of the browser for Windows (however, users of the Linux-version can update the browser for a long time through the prepared repository). Work on this task is still far from completion, but the process, as they say, has begun.

Further, importing data from other browsers is another important area we have been doing this week. At the moment, data import is possible only from the Chrome browser, bookmarks and passwords are available for importing. But we must warn that for some Windows 8.1 users, the import of passwords may not be available. But we fixed the error leading to crash when importing data. We, of course, will continue to work on this feature in the coming weeks.

The bookmarks bar is still a priority. We added the ability to display only the icons of sites on this panel, and also improved the speed of the browser when working with the bookmarks panel. Also slightly corrected folder icons. We are waiting for your comments and suggestions on the bookmark bar, because in fact the work here is only at the very beginning and far from complete.

Problems that we already know and are working to fix them:

  • Broke mouse gestures
  • Non-Chrome Bookmarks Import Does Not Work
  • Importing bookmarks from Chrome does not work when Chrome is running
  • Vivaldi crash when opening a new tab, if the tabs are located on the side
  • Context menus on pages sometimes do not work the first time.


And everything else is just great :)
