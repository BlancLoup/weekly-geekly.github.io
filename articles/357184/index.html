Mass hacking FirstVDS

At least 3000 sites per 1000 servers are compromised at the super user level.

A few days ago, we discovered that the files on our sites were unauthorized and changed. It turned out - hacking. It seems to be nothing interesting, but as a result of a small investigation, we managed to find hundreds of hacked servers on the FirstVDS network.

The main purpose of the attacker is to advertise the casino "Vulkan". From infected sites, it creates link networks with each other that link to advertising web pages, thereby affecting search engine algorithms. These pages lure bots for requests like ‚Äúthe best casino on the Internet‚Äù, and when a user goes to such a page, they are transferred to the casino. The work of the original site, as a rule, is not completely disturbed, and therefore problems remain unnoticed by administrators of most servers.

This adversely affects search engine optimization sites. Search engines reduce the position of infected sites and advertising budgets fly into the pipe. In addition, hacked servers operate in a mode of abnormal load, which affects their stability.

Discovered servers are completely compromised. There are no guarantees that files and data are not stolen, or there is no loophole in place to keep control of the server in the future, after the sites have been treated.

Incident detection


In the root folders of our sites appeared other people's scripts. It was surprising that the owner of the files was not the user of the apache web server (which is typical for hacking the site through vulnerabilities in its scripts), but a super-root user. 

# ls -al .cli.php -rwxrwxrwx 1 root root 8375 Sep 2 07:15 .cli.php

First, we eliminated the leakage of keys and root password by checking the server access logs via ssh. They showed that at the time of modifying the site files there were no open ssh sessions. It became clear that the vulnerability should be searched either in programs running as root, or in FirstVDS technical support, since Access to the file system is possible through ISPManager and from the parent server through VDSManager.

Investigation


The source code of the malware has the URL of the head (that is, the control center of the malware). It is used for self-updating, loading data for link-link generation, checking for infection, as well as some other uninteresting functions. There is a web shell.

Domain domain head 0x31.ru. Bots are controlled from Germany (51.255.108.194) and from Krasnoyarsk (171.33.253.168).



Since vulnerability remained unknown to us, we studied other sites that form the link mass. This revealed about 1600 different sites on 540 unique IP addresses.
