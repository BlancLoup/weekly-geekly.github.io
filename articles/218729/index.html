Strongly Typed SignalSpy to Test Qt Applications

When writing unit tests as a rule of good tone, it is considered to check class invariants through an open class interface. In the case of Qt, things are a bit more complicated, since member functions can potentially send signals that go outside of objects and are thus part of an open interface. For these purposes, the QtTestLib module has a useful QSignalSpy class that monitors a certain signal from the object being tested and scrupulously keeps the protocol, how many times and with what values ‚Äã‚Äãthis signal was called.
