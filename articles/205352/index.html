Fujitsu ETERNUS CS800 S4 - backup deduplication system

There is a constant increase in the volume of data that needs to be backed up, caused by the accumulation of information, an increase in the number of servers and other computing devices, and an increase in the number of users. The consequence of this process is a lack of space on existing storage systems and an increase in the cost of backup storage. Given the above features, the best solution is to back up to disks with deduplication and the possibility of subsequent copying to tape. Deduplication in ETERNUS CS800 S4


The deduplication process in this complete Fujitsu solution is as follows. First, the data is divided into blocks of variable length, based on data-dependent separators:



After that, each unique block is assigned its own unique signature and a special signature list is compiled, which contains a list of all unique blocks, as well as their sequence in the original data. Next, the unique blocks and signature sheets are recorded on hard disks with subsequent compression (archiving) of the recorded data. Thus, the required disk space for data storage can sometimes be reduced by an impressive 95%.

Fujitsu ETERNUS CS800 S4 usage scenarios


Fujitsu ETERNUS CS800 S4 storage systems can be connected to an enterprise infrastructure using four protocols - NFS, CIFS, OST and VTL. Depending on the protocol used, CS800 acts as a Network Attached Storage (NAS), Vurtual Tape Storage or Open Storage.

Network attached storage

Backup to NAS CS800 is carried out using NFS and CIFS protocols.



The CIFS (Common Internet File System) protocol provides access to files hosted on remote devices in a Windows environment. Fujitsu ETERNUS CS800 S4 supports user authentication through both ACL (Access Control Lists) and ADS (Active Directory Services).

The Network File System (NFS) protocol was originally developed by Sun Microsystems and allows all network users to access shared files stored on computers of various types. NFS provides access to shared files through an interface called the Virtual File System (VFS), which runs on top of TCP / IP. In the case of using this protocol, users can manage shared files as if they were stored on a local hard disk. This protocol is commonly used on UNIX and Linux networks, but can also be used with Windows networks.

Virtual tape storage


The Fujitsu ETERNUS CS800 S4 solution can be used in an enterprise network as a Virtual Tape Library (VTL). In this case, its disk space for backup applications is represented as LTO or DLT tape cartridges. Thus, data is stored on hard disks through an interface that looks like a tape library with cartridges, drives, and a cartridge changer. And the ability to configure multiple VTL interfaces of various types on ETERNUS CS800 S4 at once makes it easy to integrate it into an existing data processing environment.

In this mode, the CS800 S4 solution can emulate various tape drives, including IBM and HP models (LTO-1, LTO-2, LTO-3, LTO-4 or LTO-5), as well as emulate Fujitsu CS-TL tape libraries, ADIC Scalar i2000 and ADIC Scalar i500.

Open storage


Also in the enterprise network, Fujitsu ETERNUS CS800 S4 can be represented as a storage server for a Symantec NetBackup or Backup Exec media server using a special protocol from Symantec. In this case, the storage server consists of logical storage units (LSU), which are similar to directories in the NAS or tape cartridges in the VTL.

To work in this mode, you must have Symantec NetBackup 6.5.3 (or later) or Symantec Backup Exec 2010 (or later) and an OST plug-in on the media server. In this case, the so-called SPEED mode is used, which is characterized by distributed deduplication. Those. the data is deduplicated on the media server, after which only unique data blocks are copied to the ETERNUS CS800 S4, thereby reducing the load on the network and speeding up the backup process.

Path-to-tape


Often there may be a situation where the data must be stored on a removable medium for long-term storage. In this case, the Path-to-Tape (PTT) functionality will be very useful. It allows you to save data stored on Fujitsu ETERNUS CS800 S4 to tape libraries that are connected to it via the FC port. In this case, copying takes place directly from the CS800 to tape libraries without the need to use any additional device: data is sent to the tape in its original form. Such a backup allows, if necessary, to restore information from the tapes without the participation of CS800 S4. Note that PTT supports a number of backup applications, including:
  • Symantec NetBackup
  • Symantec Backup Exec
  • Secure Backup Oracle
  • Atempo Time Navigator
  • EMC Networker

Simpana Media Agent


The Fujitsu ETERNUS CS800 S4 solution is also compatible with CommVault Simpana software. In this case, the Simpana Media Agent is installed directly on the solution considered in this article.



This allows the server client to write data directly to the CS800 S4 without involving the Simpana server in the process, thereby reducing the load on the network. It also supports copying data to tape and restoring it from tape to another media agent without being involved in the ETERNUS CS800 S4 process. Also, the Simpana Media Agent installed on the CS800 S4 supports replication of data with similar devices.
