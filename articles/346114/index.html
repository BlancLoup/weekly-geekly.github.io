Meltdown: not only affects performance

Security updates


Microsoft has released security updates for Windows operating systems that fix critical vulnerabilities in Intel, AMD, and ARM processors, which fix the Meltdown and Specter vulnerabilities that have been revealed recently. Patches can lead to poor performance on affected systems and more. Below are 2 screenshots and explanations for them.

Rapid test on the example of 7-Zip


Before installing security update KB4056890 (version 1607)


After installing security update KB4056890 (version 1607)


In the first test, we get 8946 MIPS and 3:53 min, in the second, 8558 MIPS and 4:06 min. The difference is 4.3%. We lose in performance for the sake of security (due to the fault of "excellent" Intel engineers) if we dig deeper, it becomes clear that it is not only in performance.

Before installing the patch, the processor temperature at the peak was 72 degrees, after installation it was already 86 degrees. The time between measurements is 35 minutes. As a result, we get not only less productive but also more hot CPU (10-15% on average).

Why is the processor warmer?


Perhaps when working, he often drops speculative instructions from his cache.

It's all?


Not. This is just the tip of the iceberg, this is what is already known today:

  • decrease in CPU performance by 5-30%
  • performance degradation with read / write operations
  • increased TDP 10-15% (CPU, HDD)
  • 10-30% increase in electricity bills
  • security issues around the world on billions of devices
  • lost profits and other losses

And what awaits us is unknown. Meltdown, Specter, IOHIDeous - a good start to 2018.
