My view on GTD or how to synchronize my tasks on several platforms in the absence of the Internet

Prehistory

I work in the state structure. Access to the Internet from my work computer is prohibited, so the use of convenient (for me) Microsoft Exchange and google services became unavailable. Yes, and synchronize the tasks of this environment is not very easy for Android.
The very need to create a unified environment with projects and plans appeared not so long ago, when a couple more directions were added to my main activity. The volume and level of assignments has increased, and quality reductions cannot be allowed. In addition, permanent patrols are detached from the working atmosphere, while solving problems ‚Äúon the go‚Äù has always been a priority.

So, the task

1. Requires a task manager that works on multiple platforms (Win, Android)
2. Synchronization should occur without an internet connection.

I started searching for a solution from the android market (now Goggle play)

Unfortunately, all applications with a version for win refused to either synchronize without connecting to the network. Then I began to think that there could be no other option besides manual duplication. But I came across an application called Due today.

Here are descriptions of the feature set from the forum w3bsit3-dns.com

Features:
-simple and fast scheduler of cases and projects
- built on GTD ideology
- synchronization with ToodleDo.com and through it with the Desktop version !!!
-projects (with the possibility of investing some projects in others)
-creation of a group of subordinates and the delegation of tasks via mail (registration of subordinates on Toodledo is required)
-creation of "floating" tasks that are visible in all lists until their completion
-creation of tasks as google calendar events
- assignment of task reminders
-Reminder daily for overdue tasks
- recurring tasks with a given interval
-the ability to attach a project to the program start screen
- filtering tasks by context and tag, by delegation, by status
-Filtering by Next Week and task breakdown by day (what's missing in MLO)
Widgets (6 different)

The main advantage was the ability to save to the phone's memory card and the ability to recover this data in the Desktop version. That is, all information is stored in the file dackup.txt, and then easily disclosed on the working computer. This way I get actual tasks on both devices.
The downside is that when you add a task or its attributes, you have to save and restore. But this did not become a problem for me: I spend all the changes in the phone version, and update them a couple of times a day on a working machine.
Otherwise, in the absence of the Internet, this type of synchronization turned out to be optimal.
