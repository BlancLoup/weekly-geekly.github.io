Firefox 29+ interface and customization

As you know, the 29th version of the Firefox browser received a new interface called Australis for its shell (tabs, search bar, navigation buttons and bookmarks; removing Addon Bar and replacing it with a pop-up under the ‚Äútools‚Äù button (three horizontal lines, a common mark for many web interfaces)). Consider what we have lost today from the interface control levers that still existed in the form of several browser extensions.

Of course, for developers of interface extensions, this was not unexpected - for two versions, innovations are announced and appear in alpha versions, then they are worked out in beta. This is a good 3 months for the opportunity to prepare in advance (but most of the extensions are supported by enthusiasts who can not always allocate free time for this). It is possible to partially rollback the interface to previous versions by installing the official Classic Theme Restorer extension .

If you look at what users of the new version complain the most:

  • disappearance of the reaction to a double click on an empty tab-bar (the default was to create a new tab);
  • tabs that have increased in area in most OS;
  • placing the reset button in the address bar, a number of other buttons "not in their places";
  • I do not like to change the interface all the time ... well, it already was.


Such changes to the interface relatively successfully make up for the Firefox extensions to work with the browser interface. Relatively - because part of their functions, according to the facts at the beginning of the use of the new 29th version, does not work, although most of the main extensions do not have catastrophic losses in the functions.

What extensions continue to help manage the interface?


The good news for supporters of custom interfaces is that, although there are losses of functions, all the major extensions (there are several of them, the list below) continue to work. The difference from the 28th version is that all the existing functions were previously working and were also interfered with, and now some of them have lost influence on the new interface.
