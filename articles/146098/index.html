Cisco Call Manager and Skype connect

For one project, it was decided to test Skype Connect for coexistence with Call Manager. At the same time, figure out how this works and whether it is possible to use this entire bundle in production to reduce telephony bills.

Version used CCM 6.1.X
First you need to register in Skype Manager and correspondingly put money into the account. All this is detailed on the Skype site and I will not focus on this. In addition to one moment, the money must be added to the account in the Skype manager itself, otherwise they will fall directly on the Skype account and will not be available in Skype Manager. To use credit cards, account must be authenticated. I requested $ 10 from pay pall for test needs.

After the money appeared on the account go to Skype Connect and create a SIP Profile.
We choose the Name, the number of channels we need (here we need money in the account) and get the data for authentication.

To make outgoing calls, you must also put money on the created SIP Profile.

Now we have everything (SIP User, Password and money in the account) for making calls using the Skype network.

We are starting to configure CCM. In the process of setting, I had to communicate with skype, which said that for skype connect it is necessary to purchase CUCM and there are also such rumors on the Internet, I didn‚Äôt believe them and pulled the main thing out of them that the alarm goes through UDP, not TCP. This was a revelation to me.

By crawling through the menus and questions to the experts, it turned out where to change the Outgoing Transport Type and then a working configuration was created in a couple of hours with tcpdump.

Create a new SIP Security Profile. As in the screenshot. In place of the X.509 Subject Name, I entered the username obtained in Skype Connect, but probably it is not necessary.


Create a SIP Realm. I myself saw Realm tcpdump and this is an internal IP CCM. User and Digest Credentials which we received in Skype Connect.


Next, create a SIP trunk. Caller ID DN - User received in Skype Connect.


Create a Route Pattern and call. The number should be e.164.

2) MTP is not needed
