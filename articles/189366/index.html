MODX EVO store - the first repository for MODX EVOLUTION


Getting to the development


The first step was to create a package installer. It took a couple of hours, since in fact all the code has long been present in MODX in the install folder, here you can read about it.


Well, after that is a matter of technology. After discussion with Bumkaka, they identified what is prioritized and set to work. Decided to do on the basis of MODX EVOLUTION.

And now, 3 weeks after the emergence of the idea, we are ready to present you our small startup.


So far, the functionality is not very great and there are a lot of plans for improvements. But I would like to dwell in more detail on some points that distinguish our repository even against the backdrop of a counterpart for MODX REVO. So from words to deeds:

Your and only your personal repository


The general repository is good, but surely each developer has a lot of his own work, which is either too valuable or not yet finalized for publication in the community, well, or simply there is no desire to share them. Here in MODX REVO for this you need to write your repository. We have made it possible to place our work with us and use them at our discretion.

Many people will have reasonable questions right away: what about security, but if I post my package, who guarantees me that it will not leak into the network and all the work for the cat, etc., etc.? The answer to this question will be very simple: we do not store your decisions on our server.

After some thought, it was decided to keep only the link to the package, which makes it possible to store packages anywhere, for example on Dropbox, Yandex Disk, on your server or anywhere else, opening access to them only while using MODX-STORE.

It also makes it possible to easily update packages, both personal and public, for example, using GitHub. Actually all the solutions that we have shared repositories are already stored here: https://github.com/modx-store/
This will give everyone the opportunity to participate in the support of their favorite extensions, as well as always use the latest versions.

How things work


To start using MODX-STORE, you need to do only 2 steps:
  • register at extras.evolution-cms.com for a personal repository
  • install the Store module on MODX EVO 1.0.5 or higher or install my assembly where the Store is already present


Future plans


  • First of all, we plan to implement the possibility of placing paid solutions in a common repository.
  • And all the other plans will be adjusted after receiving feedback from the MODX community and the first results of the repository.
