Qqt - syntactic sugar for Qt

Sources In comparison with html, it is more difficult to create new windows on Qt - you need to specify the type of the window- ‚Äútag‚Äù, and create a new variable, fill in separate fields. In an attempt to solve this problem, the Qqt library arose. For example, you can disregard the order and the number of parameters when creating a QAction: 
Action act(tr("Text"), QIcon(":/iconPath"), tr("tooltip"), QKeySequence("Ctrl+f")); // Action act(QIcon(), QKeySequence(), tr("text"), tr("tooltip")); //      : PushButton btn(QIcon(), tr("text"), tr("toooltip")...); PushButton btn(QKeySequence(), ...; //    : Action a; a.text = ""; a.icon = QIcon(); a.toolTip = "";

In the example above, the Action variable calls new QAction, stores a pointer to the original, has a type conversion operator on QAction *, and this pointer can be used in normal work ‚Äî for example, in a call to QObject :: connect. If variables from this library are declared inside a class, then usual calls to new in the constructor are also not required, you can immediately begin filling in the necessary properties.

Simplification is observed in work with layouts (see documentation). As a result, it is much more convenient for me not to use the GUI editor, but to write the code right away - and there is no need to switch between the code editor and the GUI.

The BSD license, the code is rarely updated - the source code in the company is stored on another version control system, and it is uploaded to git only for legalization in the BSD license. After the appearance of Qt under VS 2013, the library will be transferred to the use of variadic templates instead of the current parameter listing.
