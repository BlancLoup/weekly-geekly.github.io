Chrome will help with habraeffekt?

Habraeffekt (leproeffekt, slashdot effect,% sitename% effect, etc.) occurs when a link to another, usually less popular resource is published on a popular resource, as a result tens, hundreds of thousands of users send a huge number of requests to the site server to kneel and ask for mercy. Sometimes habraeffekt even can be regarded as a DDos-attack insidious enemies of modernization . With this unpleasant situation, Google itself has decided to fight.



Starting from version 12, Google Chrome has gained a new feature. Google called it ‚ÄúHTTP Throttling‚Äù . It is activated if the browser receives at least 4 server errors from the URL that you are trying to open. If the site does not open, Chrome believes that the server is either unavailable or is under a DDoS attack. As a result, the browser blocks access to the site for a ‚Äúshort period of time‚Äù. Then the request to the site is repeated and, if the site does not become better, the interval ‚Äúincreases exponentially‚Äù. At first, requests will be blocked for 0.7 seconds, then the interval will increase and the maximum 900 seconds (15 minutes).

In Chrome 12, HTTP throttling is disabled by default.
