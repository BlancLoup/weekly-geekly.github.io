Update Vivaldi 1.4 for Linux - the hunt for flash plugin



Hello! Today, an unusual update is not a test build, but an updated stable version of Vivaldi 1.4 for Linux. The reason for this update (now to version 1.4.589.41) was a slight change in the operation of the Vivaldi browser with the Adobe Pepper Flash Plugin (PPAPI). We have expanded the number of directories in which Vivaldi will look for the plugin installed in the system. Thus, we hope that Flash content will work for more of our users.

And yes, this update is only for Linux users. You can wait until the update comes from the repository or download the installation packages directly from our site .

How it works. During installation, Vivaldi browser checks for the location of the plug-in from the official package provided by Canonical partners for Ubuntu Linux. Additionally, if you have Chrome installed in your operating system, Vivaldi can use the plugin that comes with this browser. If both versions of the plugin are available, Vivaldi will detect the newest one and use it.

Ubuntu linux


If you are using Ubuntu, you can use the following commands in the terminal to connect the Canonical partner repositories and install the Adobe flash plugin from them: 

sudo add-apt-repository "deb http://archive.canonical.com/ubuntu `sed -n '/UBUNTU_CODENAME/s/.*=//p' /etc/os-release` partner" sudo apt update sudo apt install adobe-flashplugin

After rebooting Vivaldi (if it was working at the time of the update), you should get a working copy of the flash plugin in the browser.

Other Linux Distributions


Follow the documentation of your distribution to install the package that provides the official Pepper / PPAPI Flash from Adobe. If, after installing such a plugin, you still do not see it in the Vivaldi browser, let us know in the comments under this article the name and version of your distribution, as well as the path where it was placed after installing the flash plugin (if you know it).

You can also try to use this script if there is no Adobe Pepper Flash package in your distribution.

We remind you that the weekly assemblies are test, so do not forget to backup important data. You can report any errors found at the same address .
