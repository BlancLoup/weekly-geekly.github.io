ZendCon'09


Before the conference, there were tutorials - the opening of the PHP session on IBM I5 was for me. IBM teamed up with Zend to bring PHP to mainframes . If you leave aside the i5 ecosystem, then for everyone else this can also have significant consequences, especially if IBM helps develop the core of the language. What's new in this build


Impressed by talking to users at ZendCon, we started optimizing. We used such heavyweights as ShugarCRM and Magento as ‚Äústress tests‚Äù (we wanted to try out Bitrix, but did not find an open source version). Literally the first profiling sessions showed that type inference produced too much garbage in memory, and it could lead to looping (and freezing) - now these problems have been fixed. I would like to say at once that this is only the initial optimization and we plan to seriously pursue this further. * Very * I want to hear feedback from all who upgrade. If you have specific problems - write in any way. I personally process all feedback.

New features:
  • improved PHPDOC generation
  • navigation / search for uses for global variables
  • recognizing class members based on name for those cases where type inference is not possible
  • SFTP support
  • autocompletion after the new keyword
  • namespaces are displayed in the file structure tree
  • quicfix (Alt-Enter) to define unknown variables with var

Many bug fixes, details in the blog and tracker

Unfortunately, there are known problems.
  • The most serious is degradation: multi-line literals in this assembly are incorrectly highlighted as containing the WI-494 error
  • Several namespace XXX declarations; (with ";") in one file will be mistakenly recognized as embedded WI-497
  • renaming a variable removes the $ sign from the WI-500 name

We will try to release the update as soon as possible. Wait for bug fixes, optimization and of course new features :)
