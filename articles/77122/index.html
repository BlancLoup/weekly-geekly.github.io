Making fonts in KDE 4 beautiful

By default, in my distribution (OpenSuse 11.2), the fonts in KDE 4 looked really bad. Honestly, I even felt a little hurt when one Mac driver came up and said: ‚ÄúLinux, Linux, even normal fonts cannot do‚Äù.
However, the situation can be very easily corrected. In the end, everything will look like this .


So what do we need? In fact, there is no magic here. To do this, you need only one set of fonts - Droid Fonts. You can download it either as a droid-fonts package, or on the official website . If you used the second method, you will need to install the fonts. This can be done through the installation of fonts in the KDE settings.
The Droid Fonts suite was developed by Ascender Corporation specifically for Google Android.

Well, now we have the fonts installed, let's configure them. Go to the font configuration in KDE and set the fonts. I did this for myself:
  • General - Droid Sans 10;
  • Fixed Width - Droid Sans Mono 10;
  • Small - Droid Sans 8;
  • Toolbar - Droid Sans 8 (Bold);
  • Menu - Droid Sans 10;
  • Window Title - Droid Sans 10 (Bold);
  • Taskbar - Droid Sans 10;
  • Desktop - Droid Sans 10.


Now you need to adjust anti-aliasing. First, enable anti-aliasing, for this set the value Enabled. Now we will configure it, click the ‚Äúconfigure‚Äù button and set the smoothing parameters. Enable sub-pixel rendering, if you have an LCD monitor. Hinting style myself I did full. Experimenting with these settings , find the best indicators for your monitor and eyes (and taste).

That's all. Everything is very easy. And also put all the Force Font DPI parameter in the font settings window to 96 DPI. This will avoid different font rendering in different applications.
Feels like working has become really more enjoyable.
Good luck to you.
And don't forget - Have a lot of fun!
