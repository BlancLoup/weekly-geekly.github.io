djbdns

It's no secret that bind has long ceased to be the de facto standard in the world of dns servers, both as caching and authoritative for zones.
Recently, the most popular is the development of the respected Mr. DJ Bernstein - djbdns


Like all software products of Bernstein, djbdns is distinguished by high security and reliability by design, easy setup, powerful functionality, excellent performance and low hardware requirements.

It should be said that for newbies in the world * nix or those who have never encountered the software released by Bernstein, the setting may seem to be not trivial and confusing.
However, it should be said that having understood the proposed concept, configuring djbdns becomes simple and enjoyable.

djbdns can be used as a caching server for desktop stations, reputable zone servers, or recursive caching servers.

djbdn is reliable enough, there is experience in using it both on a local caching server and on production servers.

I would like to pay particular attention to the format of zone configuration files.
Instead of a complex bind-style format, a simple delimiter-style format is used, which provides a simpler configuration with both hands and scripts.

Basic documentation is available at: cr.yp.to/djbdns.html
There are mailing lists in which you can find answers to your questions.

If you are faced with the task of setting up a dns server, try djbdns, in any case, at least you will receive a new useful experience.
