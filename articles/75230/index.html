Magnet links are intended to indicate the resources available for downloading via the p2p network. Such links basically identify files not by their location or name, but by the content, more precisely, by the hash code. For torrents, they are:

magnet:? xt = urn: btih: <info-hash> & dn = & tr = <tracker-url>

where info-hash is unique data about the uploaded file in hexadecimal encoding, xt is a required parameter, dn is the display name that the client uses when receiving metadata. tr - the parameter describing the address of the tracker. If the torrent belongs to several trackers, then each is registered in the magnet. The parameters bn and tr are not necessary, and in general the magnet may contain a hash of the file. If the tracker is not specified, the client must use DHT to get a list of peers.
