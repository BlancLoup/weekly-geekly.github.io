Critical vulnerability Windows 7 x64: BSOD and system compromise



A well-known hacker, webDEViL, posted on his Twitter message that a specially crafted web page (the point is clear from the screenshot) containing an iframe and viewed by Safari causes a BSOD error in the system and allows an attacker to execute arbitrary code on the system under attack.

The presence of the vulnerability is caused by an error in the win32k.sys file, which has already been repeatedly noted as the culprit of critical errors in Win XP. Successful operation leads to code execution with system-level privileges.

The vulnerability is confirmed by the 64-bit version of Windows 7, however, other systems may also be compromised.

The patch does not currently exist.
