Microsoft showed Project Astoria: what it will look like using Android code in Windows applications


On the Microsoft website, in the Universal Windows Platform Bridges section, the announcement of the Project Astoria software tool, designed to transfer Android applications to Windows 10, appeared. A similar announcement with the sonorous title Project Islandwood was made for iOS as well.

However, for Android, unlike general applications in the iOS section that Xcode projects can be imported into Visual Studio and assemble Windows applications with minimal ObjectiveC code changes, the project engineers showed a short video with a cursory description of the Project Astoria features.

As you can see from the video, the Android application will be analyzed for compatibility, and as a result of this analysis, typical Android user actions will be translated into similar ones for Windows 10. As an example, a rather simple act of sharing some content on social networks is given. What exactly will happen to the java-code in this case is still unclear. To work with the Windows SDK, you can use both Android Studio and Eclipse without switching to Visual Studio.

In order to understand exactly how much code can be reused and whether it contains Google-specific tools, you will need to use the App Analysis tool. The site will need to download the APK file, after analyzing which the developer will see what exactly he is recommended to do for compatibility with Windows 10. The video shows this point.

In addition, they say in Redmond that they made sure that the transition from Google services to Windows services looked literally like replacing one line of code with another. It is claimed that all tools like advertisements or in-app purchases will be transferred to the Windows environment with minimal effort.

The status of Project Astoria is not even alpha yet, but if you want, you can sign a special Testing Agreement and send your team an APK file that will be used during development.
