White IDE - RAD Browser Development Environment

Hi, I want to present one interesting development of the studio in which I work. And as a leading developer I will cover it :)


The first version was not entirely successful, because of its heavy weight and some non-logical. In the second version, I revised the architecture of both the project structure and the component model. The bulky scriptaculous was originally removed, but this does not mean that it is completely removed. Also from version 2, the xAjax framework was removed, I replaced it with my light class, for comfortable XHR communication.

At the beginning of the development of version 2, I planned to rewrite Prototype by myself, removing all unnecessary methods, but as it turned out, there were none. In any case, I would encounter a Prototype, so it was decided not to build my bike. There are also other innovations, including:
  • Dynamic namespace.
  • Inclusion of FirePHP in the project.
  • Support for many browsers (Internet Explorer 7+, FireFox 2+, Opera 9.5+, Safari 3+, Chrome)
  • Component Templates.
  • Delayed and batch loading.
  • Coverage of (yet) a large number of JavaScript frameworks.
