As I have already mentioned, money is taken for everything separately) For example, for memory - specifically for the amount of megabytes consumed at the current time. Optimization

Selectel gives administrators a completely new optimization plane - at a price. Now it is advantageous to configure the server for dynamic memory consumption, limited at the top (for example, so that the number of Apache processes is from 1 to 8-16 + launching all sorts of Ruby on demand) + you need to be careful not to load the disk too much. In general, my opinion is extremely positive - here you are sure that you will not be "asked to leave" for too high a load - but you can spend a lot of money in the case of a non-optimized application.
