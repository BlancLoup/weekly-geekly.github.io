Firefox 3.0.2 Released

Firefox (3.0.2) is released, but one password bug is noticed.

UPD: Under the cut described what kind of bug and how it can be fixed.

Among the list of fixes:
  • Fixed several security vulnerabilities .
  • Fixed several stability issues.
  • Released build in Sinhala and Slovenian .
  • For testing purposes, beta builds have been released in the following languages: Bengali, Galician, Icelandic, Kannada, Marathi, Telugu, Thai and Hindi.
  • Fixed several minor issues related to the display of certain web pages.
  • Fixed several issues with themes that affected RTL localization.
  • Fixed an issue where some users who configured toolbars did not have the Back and Forward buttons ( bug 426026 )
  • Firefox 3.0.2 adds new root certificates that use Extended Validation (EV).
  • On certain IDN sites, the password manager incorrectly substituted saved logins and passwords into the fields.
  • Fixed several hangs and crashes that occur when using screen readers.
  • Updated Russian spell check dictionary. Also in the dictionary added support for the letter "E".
  • Several Mac-specific issues have been fixed:
    • In some cases, the hot keys stopped working.
    • Indian, Chinese, Korean, and Japanese characters could not be entered (using IME) in the Flash object text fields ( bug 357670 )
    • Firefox 3.0.1 did not work if the user profile was stored in the AFP directory ( bug 417037 )

  • For information on previous changes, read the Firefox 3.0.1 release notes .


It would seem all is well and good. But many have noticed a bug with saved passwords. When upgrading from previous versions, some users do not save and do not display old passwords (this is due to non-Latin characters in the database of stored passwords).
There are several ways to get rid of this:
  1. Put Firefox 3.0.1 over 3.0.2
  2. Go to the profile and find the signons3.txt file there, open it and convert the encoding to UTF-8 (it used to be ANSI) also on the Mozilla Russia forum, it is advised to delete the lines with "curc"
