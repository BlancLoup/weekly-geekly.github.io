Joker 2015 Java Conference Announcement

Hello!

Exactly 3 months later, on October 16-17 in St. Petersburg, we will hold a Joker 2015 conference. The best Java speakers from Russia, Europe and the USA, current and fashionable topics, hundreds of programmers from all over the Russian-speaking Java world - this will be Joker 2015.



What awaits us all at Joker 2015? Let's go under the cut, I'll tell you everything there.


Last year joker

Last Joker was held in St. Petersburg, too, in mid-October. It was the fourth big Java conference from JUG.ru. In the grid of Joker 2014 there were 38 reports . Of these, 36 - in the main program and 2 keynote. There were quite a few participants: together with the speakers, there were more than 700 people.



This video tells in detail what last year's Joker was:



Now it's time to talk more about Joker 2015.

Format
It will remain the same: reports of 60 minutes, two days, three parallel tracks. Of the new ones, we will extend the breaks a little so that the participants have time to communicate normally with the speakers and with each other, drink coffee, go for a smoke, etc. etc. Secondly, we will experiment with closing - at the end of each day, we will gather all the participants in one hall and make closing keynote. Thus, in the current grid we have 28 reports - 4 keynotes and 24 sectional ones.
Unlike last time, all keynotes will be technical!



Speakers and Topics
With whom we have already managed to agree:




Alexey Shipilev is a performance expert from Oracle, the face of the Java SE Performance team, a brilliant speaker. Repeatedly spoke both in Russia and abroad, consistently included in the top speakers at any conference at which he speaks. His numerous reports in Russian are available on the new site JUG.ru. Alexey is now thinking about what exactly to make a report. Most likely, it will be a report about updates in the class String in based on its current projects: Compact Strings , Indify String Concat .




Martin Thompson (Martin Thompson) - an expert on high-performance systems, the author of the famous framework Disruptor , now more working on Aeron, high-performance transport for messages. Martin will make two presentations: one about the 20-year evolution of Java through the performance lens of the platform, and the second about the problems of building fast (in the sense of low-latency) data exchange systems between threads (in the original - ‚Äúchallenges of doing low-latency message exchange between threads in memory ").
