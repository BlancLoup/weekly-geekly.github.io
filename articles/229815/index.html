Available for download WordPress 4.0 Beta 1


WordPress 4.0 Beta 1 is available for download!

The official release is scheduled for the end of August of this year.

In the new release there is a new interface for finding and installing plug-ins, the library library interface has been improved. Also there was a choice of language during installation. The record editor has significantly improved: a preview of the embedded media files in the editor, such as video, the editor window stroke, has been added. These and other improvements are more detailed under the cut. Also welcome discussion in the comments.


How to try?



To test WordPress 4.0 Beta 1, you can install the Beta Tester plugin or download the distribution here .

What updates await us?



  • Preview of embedded objects . As you know WordPress supports embedding objects from third-party services. For example, we all know how to insert a video from YouTube. But to check how it will look always had to view the recording. Now the embedded object will be loaded in the visual editor.
  • Improved work with the media library. The media library has not changed for several years. This did not even change the responsive design of the WordPress admin panel. Now the developers got to her. It will have a grid view and will work on Ajax, that is, without reloading pages.
  • Plugin installation changes. Similar to the media library in the installation of plugins from the wordpress repository, the design has improved. More information about plugins. The interface is built on the card approach. What can I say, it turned out much more informative.
  • When installing a new blog there is a choice of installation language . Not a major change for me. Anyway, I put the Russian version, so as not to drag the extra translation files. I wonder if they will be put automatically or not.
  • Automatic scrolling editor. When writing a long post in the WP editor, it was really annoying to move the screen cursor up to use one button or another. Now this problem is solved, although it still requires testing. In some browsers they write that there are glitches.
  • Updating the configuration of topics A large number of settings always scares the user. In order not to overload the user with unnecessary settings in the configurator, the widgets are moved to a separate panel


Found a bug?


Naturally in the beta will be errors, then it is beta. Putting it on production is highly discouraged. If you find a bug, report it to the developers via the beta testing forum .

In the comments, I propose to share my impressions of WP 4.0.
