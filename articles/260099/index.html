Admin tools for monitoring IP PBX 3CX Phone System (Part 2)

Part 1

Logs

This is probably the most necessary tool for analysis.
You can customize the level of display logs. Errors are minimal. Detailed - maximum.

There is a rather convenient filter for subscribers, trunks and service numbers (IVR, groups, etc.). In addition, you can filter in calls. Each call that enters the system is assigned a unique ID by which it can be tracked.



1. Setting the level of detail of events was transferred to the general log field, which is undoubtedly more convenient. Moreover, when changing the level, it is no longer necessary to restart the 3CX Phone System service. Events are displayed dynamically depending on the options selected.
2. A filter for subscribers appeared, i.e. Now you can sort the calls belonging to a specific extension.
3. Filter by call - allows you to track a specific call by ID in the system. The identifiers themselves are displayed on the main field as Call [C: 19] (label 4 in the figure).

Windows logs

In the 3CX interface itself, the display of the standard Windows log is duplicated. There is not much information there, mainly statuses and errors that are issued by PBX services.



3CX Binary Log Veiwer

Long-term logs are stored as binary files, and for viewing, download the 3CX Binary Log Veiwer utility in the Updates section.



You can run the utility from the folder C: \ Program Files \ 3CX PhoneSystem \ Bin \ Support \ 3CXBinLogViewer



The principle of the utility is quite simple - choose the period, the level of logs, the necessary tags and click Apply. In the lower window displays the desired fragment of the log.
\


By default, Log Veiwer displays information page by page, 10,000 records per page. Log output level is adjustable.



Of utility

Export to text: In the Tools> Export to text menu, you can export the filtered information to a text file. All records from all pages are exported, even those that are not displayed in this coin. If you want to export all the data, simply remove the filters.
Command Line: 3CX Log Viewer can also be launched from the command line using the following parameters:
3cxBinLogViewer.exe <WAY TO LOGO> <WAY TO TEXT EXPORT FILE>
If you run 3cxBinLogViewer.exe <Path to Logos> , 3CX Log Viewer will load the logs at the specified path. For example - 3cxBinLogViewer.exe c: \ 3cxlogs.
If you type 3cxBinLogViewer.exe <WAY TO LOGO> <WAY TO TEXT EXPORT FILE>
3CX Log Viewer will upload binary files to text. For example - 3cxBinLogViewer.exe c: \ 3cxlogs c: \ 3cxlogs \ txtlog.txt.
Finally, it should be noted that the level of logs and storage duration is configured in the system.
Binary files are divided into 50 MB and cyclically overwritten if it is worth the storage period.
