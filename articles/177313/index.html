Configure 2X ApplicationServer XG and publish Windows applications for Android, iOS, Linux, Windows, Mac and browsers with HTML5 support



Hello! In this article, we will look at how to configure 2X ApplicationServer XG on a single terminal server, Windows Server 2008/2012 Standard, and provide access for users from mobile devices: Android and iOS, and workstations: Windows, Linux and Mac. We will also consider how to launch a trial version of the HTML5 gateway and provide connectivity via Internet browsers with support for HTML5 technology: Google Chrome, FireFox, Internet Explorer, etc.

2X ApplicationServer XG is designed for delivery from a central server of virtual desktops and applications. This will provide you with continuous access to data, resource-based load balancing, and complete network transparency for administrators. The free version (ASXGFREE) is available for permanent use and has no expiration date. Free versions are for testing and general reference only.

First you need to download and install 2X ApplicationServer XG on a terminal server - Windows Server 2008/2012 Standard, for this we download the installation file from the link below:
2X ApplicationServer XG

Before installing 2X ApplicationServer XG, we make sure that RDP services are running and you have free CALs (user licenses), or a trial version of Remote Desktop Services is running. Since the port 80 is used as a standard for the gateway, if it is in use for example an IIS server, you will be warned about this when installing 2X ApplicationServer XG. You can safely continue the installation, the gateway port can be changed later. If port 80 is used for the gateway, make sure that the firewall is configured correctly and there is remote access to TCP port 80, if there is another port in use, for example TCP port 82, then this port must be opened for access.

Install the program, when installing, select the full package - 2X ApplicationServer XG. After installation, you will be offered Wizard mode, setting up with the help of the ‚Äúvisual assistant‚Äù, you can continue or refuse and set up 2X ApplicationServer XG manually.

If the standard RDP port (TCP 3389) has been changed, then we need to change the settings of the agent of the terminal server. To do this, open the settings of your terminal server: Properties> Agent Settings, change the port to the desired RDP port.



If port 80 is in use by another service, as mentioned earlier, you need to change the port of the gateway (access port) TCP 80 to another. Open Gateways> Your gateway> Properties> Network, and change the 2X Secure Client Gateway port to the desired one, click "Apply".





The next step is to publish the application, as an example, consider the publication of the application "Notepad". Go to the section Publication> Add> Application> Terminal Server> Application> in the "Address" field, point to "Notepad" in the windows / system32 folder> click "Finish" and then "Apple".





After the server part is configured, you need to install 2X Client, to do this, download the necessary client from the 2X Software website and configure the 2X Connection connection. To do this, select> create 2X connection> enter server IP address> username> password> domain (if used)> change the port (in case port 80 was changed in the gateway settings)> save information and connect. If everything was installed correctly, you can start using your Windows application from any mobile device or workstation on which 2X client is installed.



After the basic configuration, we can proceed to setting up a trial version of HTML5 gateway. To do this, go to the terminal server, where 2X ApplicationServer XG was installed. We check that the main gateway redirects HTML requests on TCP port 81, for this we go to the 2X ApplicationServer XG console> Terminal Servers> Gateways> Your gateway> Properties> Advanced.
