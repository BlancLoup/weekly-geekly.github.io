Own repository for MODX Revolution

I will not write much. This is more like news than a manual. Although the question of own repositories for MODX Revolution is a mystery covered in darkness (if anyone has seen ready-made solutions, please give a link).

In general, I created a package that allows you to create your own repositories for MODX Revolution.

So far, not everything has been added (in particular, the number of downloads is not yet considered, and the private mode of operation is not implemented with access via the API key), but in general the package is already quite working and quite suitable for personal purposes.

This solution is especially useful for web-studios and freelancers.

Details in the video clip.


UPD: The promised video with the creation of packages using the component PackMan from splittingred . Be sure to look to the end, as the video demonstrates the creation of updates.


To install the package, add my repository http://rest.modxstore.ru/extras/ and download from it. By the way, there are a couple of interesting solutions there, such as phpTemplates (allows using pure php code in static MODX templates, turning them into controllers) and modxSmarty (allowing Smarty to be used in the front end with support for MODX elements at the Smarty level).

PS If any web studios are interested in individual solutions based on all this, write in a personal.
