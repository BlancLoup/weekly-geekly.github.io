Microsoft fixed multiple bugs in the system components of Windows 8.1 and Server 2012

Microsoft has released 22 updates for Windows 8.1 and Server 2012, which close various bugs in the drivers and system components of the OS. Update KB3115224 improves the stability of virtual machines running on Windows Server 2012. As part of the update, EFI firmware files called Winload.efi, Bootmgr.efi, as well as the kernel Ntoskrnl.exe are subject to correction. Another update, KB3134785, fixes a memory leak bug in the Rpcss.dll remote procedure call library on Windows Server 2012 R2 and Windows 8.1.



Windows Explorer (Explorer) is also subject to update, KB3136019 update fixes a bug in the Mfmp4srcsnk.dll library, which is used by Explorer to play MPEG-4 video files. For Windows Explorer, another update with the identifier KB3138602 is also addressed , which fixes the system hang-up bug when the OS boots, as well as the bug of incorrect work with the context menu of files.

The KB3141074 update ( Stop error when it is shared VHDX file is accessed in Windows Server 2012 R2-based Hyper-V guest ) fixes a bug in the Hyper-V virtualization driver called vhdmp.sys, which resulted in a BSOD with code 0x00000001 (APC_INDEX_MISMATCH).

Update KB3140250 fixes a bug in the Volsnap.sys driver that caused incorrect handling of the MinDiffAreaFileSize registry value on Windows Server 2012.

Update KB3140234 fixes a bug in the Rasl2tp.sys driver that caused the BSOD with the STOP code 0x0000009F. A blue screen appears when a user disconnects a computer running Windows 8.1 and Server 2012 with an active L2TP VPN connection.

The KB3140219 update fixes a bug in the Ipmidrv.sys driver that caused the BSOD with the code 0x00000133 after installing the hotfix of fix 3061460 on Windows Server 2012.

Update KB3139219 fixes a bug in the Storport.sys driver, which caused the computer to turn blue screen of death with STOP code 0x1E when the computer was restarted or shut down.
