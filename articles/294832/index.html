Compiling a cold database: importing data from 2GIS to Bitrix24



Now we are not talking about the Bitrix24 cloud service, but about the box version, which was previously called the Corporate Portal: Bitrix24, and now it's just Bitrix24. During the work of the ‚ÄúLead Generator‚Äù service , there were also questions about similar functionality for the boxed version. Then there was nothing ready and, frankly, could not even promise when we implement similar functionality for the boxed version.

Having summed up the pros and cons to a common denominator, they decided to do what they did. Below is information about the method of compiling the base, about the pros and cons of the developed scheme, as well as instructions for users of the Bitrix24 box.

Short about the method


For those CRMs for which there is no native module, a scheme has been implemented for unloading samples of companies sorted by months. For example, to build a customer base in the cloud version of Bitrix24, you need to install a special native module with which to search and download data by company. About how this is done is written in the article: " Immediately collect customer base. "

In the case of the boxed version, everything is much simpler at the initial stages - making up the base, but there are some drawbacks, but more on that later.

You find the companies you need by any keywords or by activity, import and then simply download the ready-made sample in CSV format, which you later upload to your Bitrix24.

Advantages and disadvantages


Of the benefits:
  • No need to install additional modules.
  • No need to mess with the access key.
  • There is no need to constantly go to different browser windows, the whole sampling / base process takes place in one window on one page.
  • Very simple wizard with detailed explanations.
  • All imported data is stored in a secure storage and is always available for reloading.
  • Faster work.


Of the minuses:
  • There is no possibility to import a directory of activities.


As you can see, there are much more advantages, and we decided to donate with a reference book of the fields of activity in favor of better and faster service for our clients.

Instructions for use


Registration / Authorization

We go to the site www.lead4crm.ru and click on the "Login" button, there we choose any convenient authorization method from social services. After the first authorization, you can link other social accounts in your personal account, if you have such, so that next time you can use any existing one.

Sometimes it happens that we log in via Yandex (or any other), after which it is forgotten which account was logged in since the user uses in Yandex (or any other) several accounts. In this case, it is better to write down your personal number so that you can further contact the support service. Or it is possible to simply understand by yourself whether authorization with such an account was previously, if authorization was previously, then Yandex will not ask permission to transfer personal data.



Creating a sample file

Once in your account, you will need to replenish the balance and if you need to select a tariff plan, select it.

After crediting funds to the personal account, you can begin to create a sample. Why use the sample creation wizard is very simple:

1. Select the CRM version: ‚ÄúBitrix24 Boxed Version‚Äù
2. Select the desired city. The system will automatically substitute your city if it is in the 2GIS directory.
3. Select the type of search. In the wizard, everything is written in detail what is needed for what.
4. Click the Import Companies button to open the search dialog.
5. After you import the companies you need, close the search dialog and download the sample for the current month.



Upload sample to CRM Bitrix24

Now you can go to CRM Bitrix24. Open your portal and go to the ‚ÄúCRM - Companies‚Äù section, and there in the header of the companies table, click on the settings button and select the item: ‚ÄúCompany import‚Äù.

Next, select the downloaded sample file and set the following parameter as required:
  • Data file encoding: Windows-1251


The remaining parameters should remain the default. Responsible can specify any, not necessarily yourself.

In the ‚ÄúDuplicate control‚Äù step, specify what should be done with duplicates.
