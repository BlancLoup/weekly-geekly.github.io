Tandem office and mobile telephony. How we developed FMC

FMC (fixed mobile convergence) is a new technological solution at the junction of different types of communication networks: fixed and mobile. With its help, any organization can create a single network of office and mobile phones with an overall plan of short numbering. The technology makes it possible to call up directly for short internal numbers to employees outside offices in different regions of the country or the world without using a real PBX.

Using FMC in a company leads to simplified business processes of organizations, coupled with the ability to optimize costs. According to preliminary calculations, the savings on one SIM-card will be up to 30% due to the reduction of dial-up time and low IP-telephony rates.

Of the additional opportunities for organizations, it is worth noting the ability to set different rules for handling calls inside the workplace and record incoming and outgoing calls from mobile and landline phones. Thus, all the problems that arise in the organization when using corporate tariffs for mobile communications and office PBX are removed.

FMC in MTT Business


The FMC project is an additional method for receiving and initiating voice calls from the Virtual PBX. This happens without using the Internet, only through GSM radio channels. At the same time, FMC is a part of the Virtual PBX service and is not connected outside of it. FMC is also provided only using the MTT SIM associated with the Virtual PBX.

FMC includes all the functions of a virtual PBX for the city and mobile phones of the company, without requiring an Internet connection. For the client, connecting to the FMC service means accessing the opportunity to start a new workplace of type VATS - SIM FMC. In the administrator‚Äôs personal account it becomes possible to specify the SIM FMC as one of the options for dialing to the subscriber.

Mobile Internet is available to the subscriber immediately after registration in the network of the cellular operator and activation in the Virtual PBX, while on a limited list of sites, such as mtt.ru, business.mtt.ru, etc. without billing. Mobile Internet with no speed limit is possible to connect exclusively with packages.

Business API


The following infrastructure elements of the MTT company participated in the implementation of the SIM Card (FMC):

  • Billing platform - for billing calls.
  • WebAPI - for the implementation of the functions for connecting the number and provisioning on the mobile component of the network.
  • BusinessAPI - in part of the implementation of the business logic of connecting / managing services.
  • Infrastructure elements of the mobile network (SRF, STP, etc.).

Service implementation scheme at the Business-API level




For FMC, it was necessary to refine the Business-API. In particular, the following functions were added to the API:

  • receiving the price for the FMC service;
  • receive a list of tariffs for FMC numbers (for selection);
  • receiving the price according to the tariff;
  • add / remove FMC services;
  • add / delete / change FMC number to the VATS client;
  • get a list of FMC numbers (with activation status).

Workplace was also transformed into a collection of entities - WPsi (sip_id), WPsf (softphone), WPsim (FMC). In the future, other types of jobs will be added, which will provide flexibility in setting up redirects and adding new types. To separate workplace entities, the type parameter is added to the structure, which can take values ‚Äã‚Äã- sip, softphone, sim (hereinafter other types).

For the possibility of combining entities into groups for parallel and mixed types of redirection, the wp_group type was introduced, which is a grouping structure within the workplace.

To connect DEF numbers, the existing number connection functions and the installation of the designated number are used. When introducing DEF numbers into MTT-Business as part of the FMC project, they are not registered directly on the SIM cards, but simply act as an identifier for outgoing calls and they will also be used to redirect incoming calls to the appropriate employees or groups.

In the Business-API, a function is added to generate a verification code and assign it to a specific drug. Generation and verification of the code is carried out through the client‚Äôs personal account, as well as via SMS messages.
