Layout with eyes closed

The customer gives the task for the layout, and he is always interested in 2 main questions: cost and time. A beginner coder cannot evaluate the work himself and, as a rule, looks at how much others ask for it. An experienced typesetter knows his price and estimates the layout based on the time spent.
But how to determine the exact time to develop, because the client wants to know him before he chooses the artist. What determines the layout time?


1. Professional experience of the maker-up.


Definitely it can be said that an experienced typesetter will typeset faster, because he has already done many elements of the layout more than once and he only needs time to type the code. But here you can save time, if instead of writing code from scratch to use the framework of functional blocks.

It should be noted that in terms of layout time it is very difficult to say something about the professional level of the layout designer. This indicator must be taken in conjunction with the quality indicator of the code, its semantics. By the way, I noticed one interesting point - the semantic layout takes less time. This is explained by the fact that it has more meaning and less code.

Very often, the customer forgets that "this is not the final version of the design" or "the menu should be drop-down, it is not drawn on the layout, but it should be done in the style of the site."
If there is a TK, read it very carefully and clarify all doubts about the functioning of the elements.
It is most convenient for me to compile TZ in a dialogue with a customer asking questions that interest me. 3. Experiments


An inexperienced typewriter assesses the layout layout time in days, experienced - in hours. This happens because in the layout you need to experiment a lot. As long as you do not write the code with your pens and make sure that it works correctly in all browsers, it is difficult to make clear time forecasts.
If you have not done any elements in practice, but you know that in theory they are precisely made, then it is better to take time to develop them with a margin.
If there are any elements in the layout that you have never encountered either in practice or in theory, then you need to think carefully before agreeing to such an order.
