Universal admin

Good evening everyone!

I want to tell you about my product, and at the same time unobtrusively ask readers: how much can such a product be in demand. Somehow it should be called (well, at least in this article), so I came up with the code name for it: "Universal admin".

So, first of all, what this universal admin panel is for.

Probably, if you are a web developer, then you sometimes (and maybe often) have such a problem:
there is a certain site (self-written or on the basis of CMS) which takes data from a DB. These data should be periodically edited, and not to you, but to a person who has little understanding in web technologies.
And the admin of your site is uncomfortable. Or convenient, but there are scary buttons: "delete", "blow up the site", "blow up the planet." Or maybe there are no admins at all.

Universal admin - one of the ways to solve this problem. It can adapt to almost any structure and allows you to edit the data in the most convenient way.

It consists of two parts: the ROOT interface, the administrator (moderator) interface .

The ROOT interface allows you to define HOW and WHAT data you will edit.
That is, choose tables, columns, rights for them (view / edit / delete / search / sort), set up an internal data type for columns, define relationships between tables. There are also all sorts of filters, default values ‚Äã‚Äãand other settings for the database / tables / fields.

The internal data type affects how the fields will look when viewed, how they will be edited. Each data type has its own settings.
Here are the standard internal data types:
  • text (edited via text field / embedded HTML editor)
  • list of values ‚Äã‚Äã(edited via checkbox, radiobutton, select)
  • date Time
  • logical type

Through the administrator interface, you can edit (add / edit / delete) data.
In addition to standard features, the interface has several tools that allow you to:
  • highlight many records
  • quickly change one field without opening the edit form
  • "Fill in" the values ‚Äã‚Äãof many fields

The system has the possibility of extensions , and in particular:
  • extension of the internal data type (if you want to view and edit the field in a special form)
  • add special action on records (e.g. publication, ban)

PS At the moment, not even all the described functions work, so I will not post the source code. At first, only MySQL will be supported. The system is written in PHP =)
UPDATE! "Universal Admin" is only a code name. The asbestos versatility (a panacea for all problems!) Does not pretend.
UPDATE! This system does not claim to compete with PhpMyAdmin. PhpMyAdmin is primarily aimed at managing the structure , my system is at data management .
