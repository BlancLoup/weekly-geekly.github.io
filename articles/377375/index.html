Google Code is closing

In the corporate blog of the search giant, the head of the Open Source division Chris DiBona reports that the Google Code hosting will be completely closed on January 25, 2016. The main reason for closing is the emergence of better alternatives, such as GitHub or BitBucket. Most significant projects have moved there, and the Google Code administration team is forced to deal with the abuse and spam remaining.

Closing will take place in three stages. Starting today, the ability to create new projects will be disabled. On August 24, the site will be read-only, without the ability to leave comments and requests. Finally, on January 25 of next year, Google Code will be completely terminated.

Up to this point, developers will have two opportunities to dispose of their code. The first is to simply download the archives to your computer. As a second opportunity, Google prepared a master of importing Google Code projects to GitHub. The Export to GitHub button also appeared in the project interface. The BitBucket and SourceForge import guide is here .

The last time Google restricted the use of its hosting code in May 2013. Then it was announced that file hosting for new projects will be closed due to abuse. Instead, developers were advised to upgrade to Google Drive.
