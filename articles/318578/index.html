3CX V15 SP4 - New Features and Release Candidate Versions for Linux

Dear friends! First of all, we apologize for the delay in the publication of articles, which happened solely for technical reasons. We have done everything necessary to ensure that such delays do not recur. We continue to publish informational articles and technical manuals for you in the same mode!


Holidays are coming, and we have prepared for you a gift - release 3CX v15 Service Pack 4, filled with new features and important fixes. A 3CX V15 Linux Edition received the status of Release Candidate!


The main improvements in the new release are: automatic tuning of DECT and FXS devices, allowing you to configure a DECT base station with multiple terminals (handsets) with a few mouse clicks - each SIP number is assigned to each terminal; A new reporting module and numerous fixes when generating reports. In the new release, we also reduced the functional gap between the Windows and Linux versions of the system - the Linux version has a fax server, fault tolerance system and support for external user directories.


The update, as usual, is installed via the 3CX Interface Update section.


New opportunities


  • Simple automatic setup of DECT base stations with multiple DECT terminals. Each terminal is assigned an individual 3CX extension number. Setup Guide will be published soon.
  • Log into the user's personal portal directly from the 3CX management console
  • New reporting module for Linux
  • The ability to export call logs to a CSV file
  • Improved fault tolerance system performance
  • Fault tolerance appeared in the version for Linux
  • Added support for G729 voice codec in 3CX clients
  • New fax server for Windows and Linux with IPv6 support
  • Added additional first-level domains to select the FQDN of the 3CX server name
  • Country and time zone selection added to management console
  • Added integration with Office 365 Directory Service for Linux version. Now you can sync Office 365 contacts with your 3CX address book.
  • Additional security option - blocking calls to satellite telephone prefixes
  • Automatic configuration of a 3CX entity (extensions, trunks, PBX parameters) using a template file (answer file). More information is available here .

Fixes


  • Numerous bug fixes in the 3CX V15 kernel for Linux - putting it on Release Candidate status
  • Fixed a bug in the Management console service, which sometimes started too long. Because of this, the system interface at the entrance gave an error.
  • Added a message about the need to update the browser cache when updating the management console interface
  • Fixed a bug where the Queues / Agents and Queue Productivity Survey statistics did not converge.
  • The calls that were wedged in showed an inaccurate overall call duration.
  • Added rounding call cost
  • Fixed other minor report generation errors.
  • Fixed a bug that occurred when you need to specify a user IP address and port in the SIP Contact field in the trunk settings
  • Backup now saves the list of blocked prefixes (countries)
