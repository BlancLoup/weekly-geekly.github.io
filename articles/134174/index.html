Processing the archive of video records in the LiveVideoArc virtual machine

If you have a lot of identical video records in the same type of format (for example, Dvd discs) and you want to easily convert and compress them, then you are here. I want to tell you about my project that will help you to do this.

LiveVideoArc - this is a meat grinder, to the input of which various files arrive (downloaded via ftp), and after processing on the output, you download compressed xvid files mpeg4.
I will try to answer possible questions:
  1. This program?
    Not. This is a whole Linux distribution, with add-on scripts to handle
  2. How is this better than another universal converter?
    We can process any files and customize processing options.
  3. And what happens if a new codec and compression algorithm appears?
    Update the program through the OpenSuse repository and get new codecs.
  4. It's free?
    Yes. And it is available under the open license GPL3, which allows us to copy, upgrade the assembly for ourselves and distribute this assembly.



The assembly was performed in SuseStudio and published in SuseGallery for any operations with it.
So we need to process the video. We load this assembly (in a virtual machine or on real free hardware), then configure the network using yast2. Connect via ftp to livevideoarc and upload video * .VOB in folders / home / linux / inbox Number. The signal to start processing is the file VIDEO_TS.VOB. As soon as it appears in the inbox, processing begins.
The uniqueness of this project is also in the fact that the sound passes normalization (amplification of a quiet sound signal to the maximum). Also in the new version, a 1280x1024 folder has been added to process .AVI files recorded from the computer screen.
If you have a bright head, there is a desire and opportunities, you can send me sample scripts for processing in other formats that may be included in future versions of the project (for example, converting to mkv, mov, ipad, iphone format, etc.).
The project has existed since April 2010 and is actively developing.
