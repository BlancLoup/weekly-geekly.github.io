Dynamic generation of individual characters in an escape sequence

When working with character objects (control sequences, & -connections, escape sequences, mnemonics, etc.) in my case Escape-sequences of the type "\ xnn", it became necessary to form them dynamically. Such a task may arise, for example, when parsing a malicious code from an attacked site.



Actually the problem - the following code does not work: 

$inx = '50'; $str = "\x".$inx; $str .= "\x51"; var_dump($str); //  \x50Q


When parsing, I found out that this code works: 

 $str = "\x50"; $str .= "\x51"; var_dump($str); //  PQ

And the code in single quotes does not work at all, since the sequence is not interpreted at all: 

 $str = '\x50'; $str .= '\x51'; var_dump($str); //  \x50\x51

Formulation of the problem


Now the task can be formulated as follows: a proper interpretation of a string containing a dynamically generated sequence of type "\ xnn", whose characters are composed of fragments, is necessary.

In this case, two operations are performed ‚Äî the concatenation and interpretation of character objects. And they are executed in strict sequence. First, the interpretation of character objects, then concatenation, but not vice versa.

Thus, in the first example, one of the encrypted characters of the sequence is divided into fragments.

"\ x51" is converted to the character "Q", and for the string "\ x" php searches for the next 2 characters, does not find them, and when combining the parts, outputs them to the string with 4 characters. Such a string is no longer a command for conversion for the php interpreter.

In the second example, php replaces the contents of double quotes with characters and "sticks together" them into a single string.

Decision


Since it was not possible to force php to first glue the characters together and then interpret them, we try to get the string as it is and replace the non-interpreted sequences with ready-made characters.

My searches for the built-in or finished function of converting a sequence of type "\ xnn" into characters did not lead to a distinct result. ‚ÄúThey say that he was looking bad‚Äù (V. Tsoi). As a result, this problem was solved using the strtr () function. Here, the non-intrigued sequences '\ x50' are replaced with the convertible "\ x50". The only difference is in quotes.
