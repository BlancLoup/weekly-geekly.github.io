Excel Add-in Guide for Beginners

Hello% username%!

Recently, I had a duty to write a small Excel add-in. By the way, before that, my experience with MS Office from under .NET was reduced to simple utilities that generated reports in Excel using Office Primary Interop Assemblies. And when the office bought Aspose.NET licenses, interaction with the office via COM was forgotten like a bad dream. So, starting to understand the topic, I was unpleasantly surprised by the lack of sane Russian-language tutorials for beginners. Now I want to fill this gap. As an example, we will create an add-in, which, at the press of a button, generates on the current sheet a list of the days of the current month and highlights the days off. Interested please under the cat!

Instead of the preface


I want to make a reservation in advance that I myself am not a professional in the field of development for MS Office, and here there will probably be people who criticize my decision to smithereens. Well, I will be glad to any constructive criticism. Also, this article is not a comprehensive guide to writing Excel Add-ins, it does not set a goal to describe in detail the Excel Add-ins architecture or the contents of the Microsoft.Office.Core namespace. This is an introductory article, which, I hope, will help the newcomer to orient himself in a new topic for him and guide him through the process of creating an add-in that does something meaningful. So let's get started!

Cooking solution


Open a studio and create a new project like Excel 2010 Add-in. Let's call it SampleAddIn:



If everything worked out right for you (and what could have happened wrong here? ..), you will see a solution with the following structure:



The only file that the studio generated for us is ThisAddIn.cs: 

public partial class ThisAddIn { private void ThisAddIn_Startup(object sender, System.EventArgs e) { } private void ThisAddIn_Shutdown(object sender, System.EventArgs e) { } #region VSTO generated code /// <summary> /// Required method for Designer support - do not modify /// the contents of this method with the code editor. /// </summary> private void InternalStartup() { this.Startup += new System.EventHandler(ThisAddIn_Startup); this.Shutdown += new System.EventHandler(ThisAddIn_Shutdown); } #endregion }


Everything is simple here - we just register event handlers for starting and ending our add-in. Now, naturally, our Add-in does nothing, but you can already start it as a regular project, by pressing F5. What is especially nice, at this stage you can not bother deployment, because Studio itself launches Excel with the necessary security settings and pushes our Add-in to it. If everything worked out, then when you start Excel you will see this inscription:

 public partial class SampleRibbon { private void SampleRibbon_Load(object sender, RibbonUIEventArgs e) { } private void button1_Click(object sender, RibbonControlEventArgs e) { MessageBox.Show("Hello, world!"); } }


Let's try to run our project. If you did everything correctly, then Excel has a new tab with a button that, when clicked, a message box appears.

We pick a little deeper


At the moment, we are faced with two problems: first, if the ThisAddIn class has an Application field that gives access to the running instance of Excel, then there is nothing like this in the Ribbon class, and therefore it is not clear how to interact with Excel sheets. Secondly, as we all know, mixing mapping logic and business logic is an excellent blank for a good portion of spaghetti in your code. Therefore, we will change the code so as to get rid of these two problems, and at the same time we will understand the ‚Äúmagic‚Äù that has embedded our Ribbon into Excel without any action on our part.

The CreateRibbonExtensibilityObject method from the AddInBase class, from which our ThisAddIn class is actually inherited, is responsible for creating custom feeds in Add-in. Thank God, this method was prudently made virtual, so overloading it will not be a problem. To begin with, add the following code to the ThisAddIn class: 

 protected override Microsoft.Office.Core.IRibbonExtensibility CreateRibbonExtensibilityObject() { return null; }


And run the project. Voila! The tab has disappeared. Now change the SampleRibbon class: 

 public partial class SampleRibbon { public event Action ButtonClicked; private void SampleRibbon_Load(object sender, RibbonUIEventArgs e) { } private void button1_Click(object sender, RibbonControlEventArgs e) { if (ButtonClicked != null) ButtonClicked(); } }


And the ThisAddIn class: 

 protected override Microsoft.Office.Core.IRibbonExtensibility CreateRibbonExtensibilityObject() { var ribbon = new SampleRibbon(); ribbon.ButtonClicked += ribbon_ButtonClicked; return Globals.Factory.GetRibbonFactory().CreateRibbonManager(new IRibbonExtension[] { ribbon }); } private void ribbon_ButtonClicked() { MessageBox.Show("Hello, world!"); }


Run the project. If you did everything correctly, the tab reappeared, and at the touch of a button, a message box appears as before.

We work with cells


Now the time has come for that, for the sake of which all this fuss was generated - the generation of the table. To begin with, let's create a couple of helper methods for styling cells: 

 public static void MarkBold(dynamic border) { border.Weight = XlBorderWeight.xlMedium; }


This method makes the cell border bold. By the way, pay attention to the argument type border - dynamic - starting with .NET 4.0 this is a very convenient way of working with COM objects! If suddenly you do not know, I recommend reading, for example, with this article. For our convenience, we will create a couple more methods: 

 public enum Border { Left = 1, Right = 2, Top = 3, Bottom = 4 } private void MarkCell(dynamic cell, DateTime day) { if (day.DayOfWeek == DayOfWeek.Saturday) { MarkBold(cell.Borders[Border.Left]); MarkBold(cell.Borders[Border.Top]); MarkBold(cell.Borders[Border.Bottom]); } if (day.DayOfWeek == DayOfWeek.Sunday) { MarkBold(cell.Borders[Border.Right]); MarkBold(cell.Borders[Border.Top]); MarkBold(cell.Borders[Border.Bottom]); } cell.Columns[1].ColumnWidth = 4; }


Now we will write the code to get the list of days of the month: 

 private static IEnumerable<DateTime> GetDaysOfCurrentMonth() { var today = DateTime.Today; var dayIter = new DateTime(today.Year, today.Month, 1); while (dayIter.Month == today.Month) { yield return dayIter; dayIter = dayIter.AddDays(1); } }


And put it all together: 

 private void ribbon_ButtonClicked() { GenerateTable(Application.ActiveWorkbook.ActiveSheet, 1, 1); } private void GenerateTable(dynamic wSheet, int row, int column) { var currentMonth = GetDaysOfCurrentMonth().ToArray(); for (int i = 0; i < currentMonth.Length; i++) { wSheet.Cells[row, column + i] = currentMonth[i].Day; MarkCell(wSheet.Cells[row, column + i], currentMonth[i]); } }
