Cross-browser beautiful input type = "number"

The purpose of the post: share with developers the decision to use the cross-browser "input type =" number "" or learn about the best solutions. The main problem is that this type of input `a is displayed differently in different browsers, and it is practically impossible to customize it via css. This solution allows you to easily arrange the arrows in accordance with the design, as well as display them in those browsers that do not support the native ‚Äúinput type =" number "".

At the moment, only one parameter is passed to the plugin - this is the selector to which the js code is applied: 

jQuery(function ($) { $('input[type=number]').iLightInputNumber(); });


For example, take three input`a with different attributes: 

 <input type="number" id="input01" step="100" min="0" class="input1" placeholder="Your budget"> <input type="number" min="1" max="99" class="input2" placeholder="1"> <input type="number" min="1" class="input3 input4" value="1">


Working example

Checked: opera, chrome, ff, ie9 +
