Our colleagues are actively involved in the life of the Google community: they arrange technical meetings and large conferences, promote the latest technologies, and simply write a pretty darn good code. Therefore, DataArt representatives are not the first to send a message to the main Google event in the world - the I / O conference. Android N is already familiar to everyone who was interested, and many have already tried Android N Preview. Android Wear 2.0



The main revolutionary new product of Google I / O for me was the new version of Android Wear 2.0. This version was the first full wearable-operating system, independent and independent of the phone. What previously needed to be done using the DataItems and Messages API can now be directly implemented using HTTP REST requests directly from the clock. In addition to WiFi, the new watch versions will support a SIM card with 3G / LTE. Added support for two more input methods out of the box - now, besides voice input, you can use handwriting input and a mini keyboard. Google has officially published an API for implementing Complications - they are already present in many Watch Faces, but now their implementation is standardized, which significantly speeds up development.




There is one more good news - now Google Fit recognizes exercises that a person performs in a watch, for example, lifting dumbbells for biceps or deadlifts. Thus, the collection of statistics went far beyond the simple analysis of walking / running.



Android Instant Apps . A new unique technology introduced by Google - launch applications without pre-installation.



Android Studio 2.2 Preview - a preview of the new IDE version includes many improvements, the main ones being the radically redesigned visual editor with support for Constraints, an improved Instant Run that works even faster, and built-in support for the SDK, including. Firebase SDK.



Firebase is one of the most trending products of this Google I / O. It is positioned as a universal tool for building dynamic databases, authentication, online storage, etc.

Initially, I was somewhat upset about how few sessions were devoted directly to IoT. There were only two of them, but IoT experts were promised a small surprise at keynote. This surprise was Google Home, a smart device that allows voice to communicate with the new Google assistant. You can ask him to place an order or postpone a meeting. The device can also control other smart devices in the house: you can, for example, turn on the lights or music in another room. The device itself is small, and its appearance can be chosen to your taste. The Google Home market will be released this year. In fact, this is a competitor to Amazon Echo, which is already on sale a little earlier.


Very interesting, in my opinion, was the presentation about Instant apps. The essence of the idea is that now we have the opportunity to use the application without installing it! The user can get a link to the content, pay for parking or just try the application before installation. Instead of installing a full-fledged application, its light version is loaded, in which the main functionality is available, including the ability to make payments. The full application can be easily installed directly from the Instant app. You can get the link in a variety of ways: using Physical Web, QR code, nfc, just plain text, etc. It‚Äôs even more interesting that they promised support from Android 4.1+ (API 16) and higher. This is a great solution in a world where smart devices and services are becoming more and more. We will be able to use almost everything that surrounds us without the need to install applications, and companies and developers will receive a new way of delivering content and services to users.



Also pleased with the announcements of the updated Firebase and Android Wear 2. Firebase received a bunch of new features, some of which are also free. This is a great solution for the Internet of things and not only. A wearables can now work autonomously, without a phone, and run separate standalone applications. Also Android wear 2 received new input methods.



There is good news for us, the developers, another Android studio update. Can't wait to try.
