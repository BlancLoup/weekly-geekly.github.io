What would you like to see in the new version of Visual Studio

At the moment, the Visual Studio development team is actively developing a future version. And you have the opportunity to influence what functionality will be included in the new product using the Microsoft Connect website.
You can express new ideas on the User Voice site or just vote. In the next planning phase, they will be carefully analyzed and perhaps some of them will be included in the new version of Visual Studio.

We are looking forward to your feedback at visualstudio.uservoice.com

To date, about half a thousand wishes have already been made. In the top 15 are the following:

  • Develop the product but return the VS interface written in the native code.
  • Improve Visual Studio Performance
  • Debugging Lambda Expressions
  • Create a 64x Bit Version of Visual Studio
  • Ability to rename already created projects
  • Improve WPF Performance
  • Provide a centralized rights management site for TFS, Sharepoint and Reporting Services
  • Ability to change the project template for already created projects
  • Ability to transfer a project between collections of projects in TFS
  • Eliminate 260 character file name limits
  • Excel and Microsoft Test Manager Integration
  • Ability to build configuration dependencies in TFS Build
  • Make two IDE options - Native for Native programmers and Managed for Managed programmers.
  • Extend Edit & Continue for 64x bit code and other options in which this feature is not working now

There are many very interesting wishes in the top 100, and if it is important for you that they enter the final version of the product, do not be lazy and vote for them.
