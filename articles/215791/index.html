PPS Actually, the FAQ, which I saw out of the corner of my eye among the comments on posts about I2P
Q : How to make the I2P console not only accessible through localhost?
A : In clients.config should be like this: 
 clientApp.0.args=7657 0.0.0.0 ./webapps/ clientApp.0.main=net.i2p.router.web.RouterConsoleRunner clientApp.0.name=I2P Router Console clientApp.0.onBoot=true clientApp.0.startOnLoad=true

Here instead of 127.0.0.1 0.0.0.0 is inscribed - binding on all interfaces.

Q : How to make it possible to use I2P through a router running on another computer?
A : In i2ptunnel.config change tunnel.0.interface=127.0.0.1 to tunnel.0.interface=0.0.0.0 for a non-SSL (plain HTTP) proxy, or tunnel.6.interface=127.0.0.1 to tunnel.6.interface=0.0.0.0 if you are using HTTPS-proxy.

Q : Where are all these files - i2ptunnel.config , clients.config , webapps.config , and so on?
A : Usually - in the user's home directory. For Unix-like OS, OS / 2 and other nevindov these files are likely to appear in ~/.i2p after the first launch of the router. For Windows, the most likely location is %AppData%\i2p .

Q : What is the port of the router console?
A : 7657

Q : On which port does the gateway open in Irc2P?
A : 6668. Easy to remember - standard IRC + 1 port.
