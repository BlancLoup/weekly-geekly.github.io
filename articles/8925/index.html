FireFox - One profile for different OS

For example, I certainly coped with the question of synchronizing data and programs for Windows ‚Äî I carry software (mostly handicrafts from the PortableApps ) with me on a flash drive ... It wouldn‚Äôt be a woman‚Äôs trouble ‚Äî a woman bought a pig. I, of course, is not a woman, but I left my home machine without Windy and completely moved to live under Linux Gentoo.

The first problem of the switcher is the search for the usual software for the new platform. In my case, there were almost no difficulties - the cross-platform KeePass applications for storing passwords and FireFox with Thunderbird were first installed from ports.

It should be noted that I could not completely abandon Windows, because I simply have to work in Photoshop and cannot do anything about it. Even Gimp is not an alternative. So, on my old desktop computer, Windows, Photoshop is still installed, and Windows flash software is running. How to synchronize the database of these programs? The first time I tried to copy user profiles from a flash drive when mounting a device and back to a USB flash drive, when unmounting it. It seems to be all good, but only sometimes you stupidly forget a flash drive to mount or unmount, and during synchronization in this way it is fraught with data loss. As it turned out, there is a more elegant, but less trivial solution.


So:
1. Determine where the general profile will be stored. In my case, this is a flash drive, in the case of using two operating systems on one computer, this should be a directory on the FAT partition. Why FAT, I think, is understandable. Just in case, accessing the FAT partition for Linux is quite simple, but in order to gain access to the ext / reiserfs partition from under Windows, additional gestures will be needed.

2. Copy the folder with the active profile in the selected directory. By default, the Windows profile folders can be found at C: / Documents and Settings / user / Application Data / Mozilla / Firefox / Profiles / xxxxxxxx.default, and in Linux /home/username/.mozilla/firefox/

3. Run FireFox with a special parameter. On Windows, Win + R firefox.exe -profilemanager On Linux, firefox -profilemanager.


4. Select the option to create a profile, in the appeared window specify its name. The leading dog breeders recommend calling the profile a default user, I did not listen to the recommendations and the heavenly punishment did not overtake me ...


5. Click * Choose folder * and specify the path to the directory with the profile created in step 2. Finish.


That's all. Users of Portable versions - do not rush to delete the old profile! Personally, I could not launch PortableFirefox with the necessary parameter, so in Linux I pointed the way directly to the USB flash drive in / FirefoxPortable / Data / profile /

By the way, all this is relevant for Thunderbird, of course.
