Scrambls - protection of your intellectual property and encryption in social networks

Hello, dear habrazhiteli! Many in our time are interested in the security of their personal data, and especially correspondence. And this is not surprising, given the scandal with the NSA and other factors.

There is also a certain problem with social networks and cloud services: nothing else belongs to us , that is, our posts in social networks are their property, not ours, as our data in cloud storages do not belong to us.

And how to get out of this situation? The guys from Scrambls seem to have solved this problem already!

So what does scrambls do? It encrypts the text entered by the user before sending it to the server (for example, when posting a comment in FB), and when it detects an encrypted fragment on the page, it decrypts the text and displays it in its original form. In this case, the user himself chooses whether everyone can decrypt his record, or only selected users.

Illustrative example:

[scrambls} dfeJD44Mybb —ò—ï—ó–á—ñ—ñ—ñ {] - text that is visible only to people from the group I created

How it works



To work with scrambls, you need to install a browser plugin and log in to their website. When you enter into the text of the text, surrounded by double dogs, the plugin encrypts the message with the private key. When the markers [scrambls} and {] are found on the page, the plugin decrypts everything in between. In this case, all encryption operations occur on the client, and the server on which the ciphertext lies lies only transmitting data. When encrypting, of course, you can select a group that is allowed to view the text.

The visually decrypted text on the page is highlighted with a special background; when you hover the mouse over a message, a hint is displayed indicating the group for which the message is intended.



It is also a very good feature - changing the group of the encrypted message without re-encryption. This is possible due to the uniqueness of the key for each message.

If you do not have a plug-in, or if the scramble message came through a regular instant messenger, you can use the online encoder / decoder.

By the way, they have not only browser plug-ins, but also a mobile version for iOS (allows you to work with Twitter) and the client Scrumbls for files, which allows you to encrypt files and folders before sending them to the cloud. The client for files is available only under Windows, so I could not test it.
