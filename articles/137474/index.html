Qt SDK 1.2 Software Development Kit Released

We are pleased to announce the release of the Qt SDK 1.2 application development package. And it is interesting not only because the release date and time coincides with its version: the package was released at 12:12 of the first of February of 2012 ( 12:12 1.2.12 ).

The new SDK includes the latest version of Qt Creator 2.4.1 and the Qt 4.8 library for desktop and embedded Windows, Mac, Linux / X11 (which appeared separately in December 2011). In short, the new SDK 1.2 contains improvements for the target assemblies of Symbian and MeeGo Harmattan 1.2, more than 20 new Qt Mobility examples in Qt Creator, and the ability to easily integrate the In-App Purchasing API into a Qt application that allows developers to make purchases inside their own mobile Qt- applications.

Key benefits of the updated SDK list


Qt Creator 2.4.1


Qt Creator 2.4.1 is a patch that contains some improvements and fixes for Qt Creator 2.4:
  • Updates to C ++ and QML editors, such as syntax highlighting in QML and improved static code checks in QML;
  • Improved code refactoring utilities;
  • Ready-to-reuse ‚Äúschemas‚Äù for programming styles (C ++ and QML) between projects.

Qt 4.8 for desktop and embedded Windows, Mac and Linux / X11

  • Abstraction of the Qt platform, which makes it even easier to port QtGui to a new windowing system;
  • Qt Quick improvements, including the ability to embed OpenGL shader effects in Qt Quick applications using QML Shader addons;
  • Upgrading Qt Webkit version 2.2.1 from the WebKit project, including a large number of improvements in HTML, CSS and JavaScript technologies;
  • Multi-threaded OpenGL makes it even easier and safer for multiple threads to display OpenGL from more than one stream at a time.

New Qt Mobility Examples

  • The updated version of the Qt SDK contains more than 20 examples of Qt Mobility.

In-App Purchasing

In-App Purchasing allows developers to provide additional (premium) content for purchases within games or applications, for example, new items / levels / functionality, as well as try and buy application updates right inside the application with the appropriate interface.

This feature can provide additional sources of revenue from Qt mobile applications. The In-App Purchasing API beta was previously available for self-installation, and now this component can be added when setting up the Qt SDK by simply using the SDK configuration utility.

Using in-app purchasing in applications can significantly increase the revenue of mobile developers. With the strong continued growth in Qt applications downloads in the Nokia Store, as well as the growth of the Nokia Store itself ( about 11 million downloads per day ), Qt continues to be a good choice for mobile application developers for more than 155 million Nokia Qt- enabled phones .
