Algorithm and advantages of CMS automatic update

We have repeatedly met complaints about difficulties from users of different cms arising in the process of updating system versions - and not always difficulties are caused by a lack of user experience. Paradoxically, these problems are only aggravated in the case of an automatic update (if it is available at all). Although, according to the developers, this tool, on the contrary, should simplify the work of users, sifting out all the possible tinsel of this process.

Let's try to understand the installation algorithm of the latest version, it is common to most site management systems.
  • Check the upgrade option - whether the new version of your CMS is available.
  • Backup files are updated automatically. It makes no sense to save a copy of all the content, because it will take up too much space on the hosting.
  • It is worth remembering that the update process itself takes some time, during which the site may not be available to users. But it largely depends on the CMS. Therefore, we recommend not to carry out any work with the admin panel, also do not change the system files in the process.
  • What to do in case of an error on the way to auto-update? If the error is so serious that the system does not fix it on its own, then the user is asked to recover from the reserve.

ImageCMS users can now also keep the system up-to-date without any extra effort. Why? Because from the version of ImageCMS 4.5 we launched an automatic update, and all the latest functionality will be available for use as quickly as possible.

The key features of automatic updating of versions of ImageCMS are outlined in the publication . We do not blindly update all files - only those in which changes were made change. It also affects the speed of the update process ‚Äî your users may not even notice.
