Perfect Pechkin

The lack of an ideal email client is one of the most severe headaches in the daily electronic document management.

Here I tried to formulate the rules and features that I personally would be comfortable with. If you know what advantages existing customers have - write. Perhaps it is here that we will create the image of an ideal mail client, and there - who knows - and we will find an opportunity to realize it!

Tags
I can not without tags. Gmail has been using them for a long time, but for some reason, the Gromopird still believes that I can do without them. Ideally, I would like to be able to mark each letter with an unlimited number of labels, search and sort by them, group into groups, create and save (!) Filters on them, and also be able to attach tags to contacts from the address book.


Actions
As in Photoshop, save certain actions and sequences, hooking the output of one script to the input of another, and also tie in the actions of the client with external programs. In Bat, such a chip exists, but I never learned it. In my opinion, it is too inconveniently implemented.

Base
I would like to have a built-in synchronization system of the mail database. Let me reserve a database on a third-party secure server from time to time - this will protect me from the fall of the main mailbox and the loss of mail. Plus - if I need to work on someone else's computer - quickly remove all traces of activity (again, without losing data)

Search
Approximate to the search in the latest macos with the preservation of results, filters and navigation through the list found. With a visual option for all and separately for advanced :)
