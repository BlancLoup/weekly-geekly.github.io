CirPark Platform - software and hardware platform for parking management


CirPark Platform - software and engineering infrastructure based on CirPark Scada software (Supervisory Control And Data Acquisition) - dispatch control and data collection.


The picture is clickable

And so CirPark Platform is:

  • Intelligent parking navigation system - iPark ;
  • LED parking light system - LEDPark
  • Electric car charging stations for indoor and outdoor parking facilities - EVPark
  • A system for monitoring electricity consumption by chargers and any other energy consumers;
  • Vehicle search system

CirPark Scada software package designed for collecting, processing, displaying and archiving information from an object (system) and real-time control.

As part of parking management, CirPark Scada works with parking controllers and allows you to receive, display, transmit, process information from the following equipment:

  • Parking sensors
  • Vehicle counting sensors
  • Lighting Controllers
  • Electric vehicle charging stations
  • Allows you to make plans for parking facilities (floor by floor, division into zones, linking to the terrain, and so on)
  • Create visualization screens for the operator
  • Keep statistics
  • Customize equipment operation reports
  • To form the logic of the controllers and sensors
  • Display alarms and equipment faults

Connection to the server part of CirPark Scada is possible, both through the web-interface, and with the help of an application for Windows OS. The open XML API opens wide possibilities for integration of third-party solutions into CirPark Scada, for example, payment-access systems.

The iPark system uses CDU-TCP-PARK parking hubs as control equipment. Hubs are universal, they are also used for sensors of the metering of traffic flow, in lighting control systems and charging stations for electric vehicles. They have 4 digital inputs and 4 output relays, the equipment is connected via RS-485 interface, there is also an Ethernet port.

Data through special hubs are transmitted to the server, which registers them, processes and displays to the parking operator using the CirPark Scada software.

Traffic accounting system


The traffic flow accounting system can be implemented in iPark based on three types of sensors, which make it possible to accurately take into account the traffic flow in a parking facility.

Induction Loop Sensors - 1-channel and 2-channel induction loop controllers. Work on the principle of inductance. In the roadway laid several turns of the wire connected to the sensor. Under voltage from the sensor, the wire acts as an inductance coil in the electromagnetic field, the machine, running into the location of the wire, changes the inductance, this change is detected by the sensor, the relay closes, the system reads the presence of the machine. 2-channel controllers allow you to track the direction of movement. The sensor detects only the ingress of metal in the circuit, respectively, does not respond to people.

Infrared and ultrasonic sensors of a stream of cars, go in couple the receiver - the transmitter. All sensors have versions for external and internal installation. Work on the principle of infrared or ultrasonic barrier. The transmitter emits a wave, the receiver accepts it. Breaking the barrier with any object is fixed by the system. The disadvantage of such sensors is that the sensor does not recognize what the barrier broke, the car it was, a man or a cat.

Pros - easy to install and use, does not require additional costs for laying the wires in the roadway.

The data from the sensors through the controllers and hubs are transmitted to the server accounting machine flow, which, in turn, after processing displays the latest information on parking displays and scoreboards. The traffic flow accounting system is autonomous and can be installed at the facility separately from the navigation system in the parking lot. The central controller with the pre-installed software CirPark Scada automates the accounting and control of the transport flow of any object.
