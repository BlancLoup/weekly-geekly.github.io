Fixed a bug related to the inability to use Cyrillic in IMAP folder names

The release of Zimbra 8.8.12 clouded over for users from Russia as an annoying bug. A number of Zimbra users experienced difficulties with IMAP folders, which contained Cyrillic characters in their names. We are pleased to announce that the Zimbra development team has successfully solved this problem.



Literally the other day, the Zimbra development team released a patch that fixed a bug with handling non-Latin characters. This means that everyone who postponed the Zimbra update to the new version can now safely install Zimbra 8.8.12, and those who have already installed the latest version of the software solution can, after the update, again call the IMAP folders with the Cyrillic names after installing the patch.

The patch installation process will be slightly different, depending on whether you are using a .deb or .rpm Linux distribution. For example, in the case of Ubuntu, the servers will be updated from the official Zimbra repository, which is added to the list of application sources of the operating system during the installation of the Zimbra Collaboration Suite.

So, in Ubuntu, if you have all the Zimbra components installed on one server, just enter the command sudo apt-get update && apt-get upgrade , so that the package manager will find the most recent versions of the Zimbra components and install them on their own. After the installation process is complete, you will need to switch to the zimbra user and restart the Zimbra Collaboration Suite services using the zmcontrol restart command. In the case of a multi-server installation, this procedure will have to be done on each of the servers.

In RHEL or the CentOS-based update, the update will occur according to approximately the same scenario, but due to the use of a different package manager, the commands entered will differ slightly:

su
yum check-update
yum update
su - zimbra
zmcontrol restart

As in the case of Ubuntu, with a multi-server installation of Zimbra, you will need to repeat this process on each of the Zimbra infrastructure nodes.
