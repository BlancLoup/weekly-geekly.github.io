Zabbix: LLD monitoring of iron under Windows on PowerShell

Changelog

The time has come for me to assemble my bike to monitor the physical state of Windows hardware. I could not find a complete solution or at least more or less working since my acquaintance with Zabbix, which is more than 3 years. And even more so that it was ... elegant or something. Personally, even in such things I want to see slimness and maximum functionality. That is why only LLD and PowerShell are discussed below. And of course, only free software.

So, monitoring of what will be done:

  • SMART disks (information, general condition and individual indicators)
  • Temperature, voltage, speed coolers ( your choice )

And it will all look something like this:



Totally needed:

  1. smartmontools
  2. Openhardwaremonitor
  3. NSSM
  4. 2 PS scripts below
  5. template

Template


Under the spoiler is the current template. Just save the content in xml format and import it into your Zabbix.

The template was created in Zabbix version 3.2, it may work in earlier versions. The keys are standard and are of the form ZScript [script_name, parameter1, .., parameterN] . Parameters are passed directly to the script itself.

I hope the template turned out as simple and clear as possible.

PS scripts


Below are the necessary scripts. Inside, both the detection and the request of individual elements are already located. Work tested on Windows from XP SP3 to 2016. Of course, the decision with the Execution Policy is yours.

windows.hdd.ps1 -> For smartmontools
# 2017/04/12 AcidVenom v3
# Zabbix
param($1,$2)
#
# : discovery
if ($1 -eq "discovery") {
try {
$items = smartctl-nc --scan-open | where {$_ -match "/dev/sd"}
write-host -NoNewline "{"
write-host -NoNewline "`"data`":["
$n = 0
foreach ($obj in $items) {
if ((smartctl-nc -i $obj.substring(0,8) | where {$_ -match "SMART support is: Enabled"}) -ne $null) {
$n = $n + 1
If ($n -gt 1) {write-host -NoNewline ","}
$line = "{`"{#DISKID}`":`"" + ($obj.substring(5,3)) + "`"}"
write-host -NoNewline $line
}
}
write-host -NoNewline "]"
write-host -NoNewline "}"
}
catch {exit}
}
#
# :
else {
try {
if ($2 -eq "status") {
$obj = smartctl-nc -H /dev/$1 | where {$_ -match "result:"}
$obj = $obj.substring(50)
}
elseif ($2 -eq "model") {
$obj = smartctl-nc -i /dev/$1 | where {$_ -match "Device Model:"}
$obj = $obj.substring(18)
}
elseif ($2 -eq "family") {
$obj = smartctl-nc -i /dev/$1 | where {$_ -match "Model Family:"}
$obj = $obj.substring(18)
}
elseif ($2 -eq "fw") {
$obj = smartctl-nc -i /dev/$1 | where {$_ -match "Firmware Version:"}
$obj = $obj.substring(18)
}
elseif ($2 -eq "serial") {
$obj = smartctl-nc -i /dev/$1 | where {$_ -match "Serial Number:"}
$obj = $obj.substring(18)
}
elseif ($2 -eq "capacity") {
$obj = smartctl-nc -i /dev/$1 | where {$_ -match "User Capacity:"}
if ($obj -match "User Capacity:") {
$obj = $obj.Substring($obj.IndexOf("[")+1,$obj.IndexOf("]")-$obj.IndexOf("[")-1)
}
else {
$obj = "N/A"
}}
# SMART
#
else {
$obj = smartctl-nc -A /dev/$1 | where {$_ -match "^ *$2"}
try {$obj = $obj.Substring(87,$obj.IndexOf("(")-87)}
catch {$obj = $obj.Substring(87)}
}}
catch {$obj = ""}
Write-Output $obj
}

Only disks with enabled SMART are detected. Parameters are requested from the "RAW_VALUE" column. Want to monitor another parameter? Just enter his number. By default, brackets and their contents are discarded. If the drive you need does not give the parameter, then an empty field is returned.

windows.hard.ps1 -> for OHM
# 2017/02/13 AcidVenom v2
# Zabbix
param($1,$2)
# , ,
# : discovery temperature/voltage/fan
if ($1 -eq "discovery") {
$items = Get-WmiObject -Namespace Root\OpenHardwareMonitor -Class sensor | Where-Object {$_.SensorType -eq "$2" -and $_.Name -notmatch "#|VBAT" -and $_.Parent -notmatch "hdd"}
write-host -NoNewline "{"
write-host -NoNewline "`"data`":["
$n = 0
foreach ($obj in $items) {
$n = $n + 1
$line = "{`"{#ID}`":`"" + $obj.InstanceId + "`", `"{#NAME}`":`"" + $obj.Name + "`"}"
If ($n -lt $items.Count) { $line = "$line,"}
write-host -NoNewline $line
}
write-host -NoNewline "]"
write-host -NoNewline "}"
}
#
# : VBAT
elseif ($1 -eq "VBAT") {
$obj = (Get-WmiObject -Namespace Root\OpenHardwareMonitor -Class sensor | Where-Object {$_.Name -eq "$1"}).Value -replace(",",".")
Write-Host -NoNewline $obj
}
# InstaceId discovery
#
else {
if ((Get-WmiObject -Namespace Root\OpenHardwareMonitor -Class sensor | Where-Object {$_.InstanceId -eq "$1"}).SensorType -eq "Voltage") {
$obj = (Get-WmiObject -Namespace Root\OpenHardwareMonitor -Class sensor | Where-Object {$_.InstanceId -eq "$1"}).Value -replace(",",".")}
else {$obj = (Get-WmiObject -Namespace Root\OpenHardwareMonitor -Class sensor | Where-Object {$_.InstanceId -eq "$1"}).Value -replace(",.*","")}
Write-Host -NoNewline $obj
}

By default, the script does not detect the names of sensors that have # . For what it is needed, see below.

Also reserved is one sensor name - Vbat. This is the voltage of the BIOS battery. Its value is rendered into a separate element for a trigger (triggered if less than 2.9V).

The temperatures and revolutions of the coolers at the output are integer, the voltages are transmitted as is.

Theoretically, it is possible to detect other indicators (loads, frequencies, ..). Use the corresponding second parameter in the key for this. For example: ZScript [hard, discovery, load] .

Zabbix agent preparation


For maximum unification, I use the following and only parameter for my scripts: 

UserParameter=ZScript[*],powershell C:\System\Zabbix\Scripts\windows.$1.ps1 "$2" "$3"

This gives maximum flexibility and uniformity. If you need to add more parameters for any scripts, you can simply add variables in quotes. This will not affect current running scripts. I use the windows prefix just in case, it's so convenient for me to store templates for guaranteed identification.

smartmontools


To monitor the status of disks, smartmontools is used (version 6.5 at the time of this writing). When installing the software, the smartctl-nc utility is installed by default - it is used in the script to facilitate the life of your server. You also need to set the bin folder in variable environments.



We simply add the path to the end of the Path variable separated by a semicolon. Default is C: \ Program Files \ smartmontools \ bin



Openhardwaremonitor


For sensors, OHM is used. Software is free with open source. Installation is trivial, but to complete the work you need to run the program as a service. For such things there is NSSM, I would still advise downloading the latest build . And here you can see the syntax.
The promised choice of sensors is ensured by the fact that the names of the sensors can be changed ! Change the names to readable (they will be used in the names of the elements), and comment on unnecessary sensors.

Do not forget to restart the program / service to use the rename.

The result was a good monitoring tool. Those interested can hang their triggers on the desired prototype elements. Or completely rework the template.
