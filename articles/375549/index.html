Firefox 35: video calls between browsers and other innovations



On January 13, Mozilla uploaded a new version of Firefox with several interesting innovations to its FTP server. Most users did not even notice the update, but now they have simple cross-browser video chats.

The function was actively started testing in Firefox Beta . It is based on an API called WebRTC, with its help you can implement video calls and file sharing natively, without the need to install any browser plug-ins or extensions. The overwhelming number of Internet users has the support of WebRTC, so the new product can be widely distributed.

The service is called Firefox Hello, its icon looks like a smiling textbook. Hello is implemented simply: if the interlocutor does not have a Firefox account, then after creating the chat room, you are invited to share a link to it through something else. At the same time, it is not necessary to open the link in Firefox browser, video chat works from Chrome and Opera.

If both interlocutors have a Firefox account, then you do not need to send the link, you just need to add each other to the contact list. Two people can participate in the video chat. Conversations can be renamed, users are invited to save them for cataloging future communication.

Mozilla plans to expand the project and add several features: screenshots and some collaboration tools. Firefox Hello works with Telefonica‚Äôs TokBox OpenTok service.

Also, the new version significantly changed the look of the search line, now you can access other search engines in one click, without changing the default. But this implementation has caused a significant rejection of users accustomed to the old mind.

To do it as it was before, you need to change the browser.search.showOneOffButtons parameter from true to false . To do this, open a new tab, type about:config in the address bar, press Enter and find the parameter by its name. After the change, you need to restart the browser.

Other major changes include Mozilla Firefox version 35:

  • MP4 support on Mac OS X (Snow Leopard 10.6 and higher).
  • The built-in PDF.js viewer has been updated to version 1.0.907.
  • An app store beta is open. Link to it added to the menu.
  • Mosaic rendering in OS X.
  • The Share function has been changed, now after its inclusion you can share a link to the current page in social networking services in two clicks.
  • Added support for CSS Font Loading API.
  • Now CSS filters are enabled by default.
  • Implemented API Resource Timing .
  • Added the ability to inspect ::before and ::after .
