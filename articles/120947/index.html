Email marketing: plain / text

Good day,

In continuation of the topic I would like to focus on such a part of email newsletters as the plain / text version.

First of all, what is plain / text? Wikipedia says:
Text data (also text format) is a sequence of characters in a computer, in the strict sense of the term, corresponding to the text / plain MIME type.

Under the cat we will discuss how to effectively and efficiently conduct mailing lists for recipients of letters in this format.

By the way


It is annoying that not all mailing services allow you to create combined mailings. In most cases, you are given only a single HTML editor or field for source code in which you can create your template. But the same mailchimp and pardot have the necessary functionality to distribute user preferences and send them in the required format.

Browser View Link


In one of his topics, comrade dudeonthehorse wrote that a link to viewing a letter in a browser is an unnecessary thing that only shows the lack of professionalism of the mailing list designer. This is, in principle, the case, but only applies to the HTML version of the list. In the case of plain / text, everything is different. After all, it is extremely logical to show the user at the very beginning of the letter a line of this type:
HTML %url%
Or, if your audience lags somewhat behind this wording, you can rephrase it like this:
, . , ---> (%url%)


Content


This is different than with the HTML version, because in it you can operate with images, headings, links and text selections to attract the user to individual elements of the letter. In the case of a text version of these goodies, we cannot see, which unambiguously leads to a shortage of attention of the addressee. The only solution would be to reduce the text and the number of paragraphs without losing the basic meaning for faster perception of information.

But even here we have several tricks for highlighting areas:
