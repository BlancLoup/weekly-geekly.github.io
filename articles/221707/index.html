Remote access to accepted payments and project statistics by API in Funding.To


Despite the fact that Funding.to is a standardized solution that, thanks to our widget, is easily integrated into any resource , we strive to ensure that our users have complete freedom to reflect data related to fundraising. In order for you to process information about the project conditions and payments accepted, we have implemented the ability to use API tools.



How it works?

A request using the GET or POST method allows you to get data that displays the current state of your project, while it is possible to create an automated system that will create requests and process the received data, displaying them on the site. This system allows you to create a visualization of the progress of your project in real time, in addition, you can use the information about the payments that have been made within your project.

The API provides access to key project data. On request you can get the following information:
- The final amount - the amount that is set as a target in the terms of the project;
- Date of completion of the project;
- The amount collected is the funds that were credited to the project account at the time of the request;
- Payment information, which includes the date of payment, the amount deposited, the method of payment, the payer's e-mail address, the payer's comment.



Why API?

In our case, the API allows you to use a single standard to transfer information about the project and its participants. This approach makes the solution universal and independent of the criteria for browser and platform. Unified data that is submitted in the prescribed form can be used by any relevant software.
Despite the fact that much of the information about the project displays our standard widget, the use of requests by means of API allows you to create a unique website design. The standardized data from which a response to a query is formed can be used as the basis for creating visual data.

What is the significance of such an application of API in crowdfunding?

From our point of view, the key to successful fundraising through collective funding is a unique and visual presentation of the project that will interest potential investors. Obviously, the underlying factor that determines the effectiveness of a collection is the idea underlying it, but even with a great idea, insufficient elaboration of the visual presentation of your project will result in the investor simply not paying attention to you.

Developing an individual design to reflect information about your project can be one of the main components of its successful implementation. An original and complete presentation of information on the fundraising process will help you turn your investors and visitors into project fans, who with great interest will watch how the necessary amount is gradually collected or the project funding is formed.

In addition, you can use the results of the query to organize the presentation of your project on other resources, track the progress of fundraising without visiting its page and other operations based on remote access. Pays.To offers you all the tools you need for crowdfunding, and provides you with everything you need to create a unique project that can best represent your idea and project.
