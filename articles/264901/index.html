Vscale: cloud servers



We have launched a new service called Vscale . Its users can create virtual servers in the cloud and manage them through a convenient graphical interface. The service is equipped with a simple API that allows you to automate management and scaling.


What is Vscale?



The name Vscale is an abbreviation for vector scaling - vector scaling. We use this term to describe the capabilities of a modern virtual infrastructure, on the basis of which you can place projects with an architecture of any complexity. They can be scaled both vertically and horizontally. Moreover, both types of scaling can be combined in a single project. The metaphor of a vector representing a generalized type of scaling across the entire infrastructure is well suited to describe this practice.

Features and Benefits



Among the advantages of the new service, first of all I would like to note:

  • Modern hardware platform. Vscale is based on a high-performance hardware platform: Intel Xeon E5v3 processors, DDR4 memory, server-class SSD drives. Because of this, we can guarantee high speed sites and applications.
  • High speed deployment. The process of creating a new server due to server SSD takes from 20 to 50 seconds.
  • Flexible pricing policy. Hourly charges can reduce the cost of testing applications and change the tariff plan in any period of time.
  • Convenient control panel. Working on a new service, we sought to make its control panel as convenient and intuitive as possible.
  • A simple API that simplifies the interaction of developers with the infrastructure.


Operating Systems



Currently the following operating systems are available for installation:

  • Ubuntu 14.04;
  • Debian 8;
  • CentOS 7.


Over time, we plan to supplement this list with other distributions and images with already installed software. If you have any suggestions and suggestions on this subject - write, we will try to implement. Warning questions - Windows will not.

Virtualization system



KVM is used as the virtualization system in the new service. We chose KVM because it has the following advantages:

  • Independence of resources. Each virtual server is allocated a separate area in memory and on the hard disk. This allows you to avoid "overcell" resources (a situation where the hoster sells more resources than is actually available).
  • The ability to use modified kernels of operating systems. Thanks to this KVM feature, we will be able to offer users a wide range of distributions and ready-made images with pre-installed software.


Features of billing



Tariffing in Vscale is carried out on an hourly basis. At the same time, we have fixed monthly rates.
The fact is that funds are deducted from the balance every hour, but the settlement period is 672 hours (= 28 days). After the limit is reached, the write-offs are not made until the end of the current month.

Separately, it should be said about the cost of the last hour. The cost of the cheapest configuration is 29 kopecks per hour, or 200 rubles per month. If we proceed from the above figures, the hourly rate should be 200 / (24 * 28) = 0, 2976 rubles (29, 76 kopecks). For reasons of convenience, we round off the cost per hour. The balance to the round figure is the cost of the last hour. If you decide to switch to a more productive configuration, the deductions from the balance will be carried out at a new rate from the moment of transition.
Please note that the funds are deducted even if the server is turned off: the system resources still remain reserved for you. To stop writing, the server will need to be removed.

The cost of all configurations includes a limited amount of free traffic. The traffic is calculated on the basis of actual consumption. If the monthly package is exceeded, a fee of 0.5 rubles per gigabyte is charged.

Locations



At the moment, all servers are in our data center in St. Petersburg, but in the future it will be possible to install servers in Moscow.
We plan to host servers in other locations. If you have any suggestions, write!

Network



All servers are connected to the network at a speed of 1 Gbit / s.
The servers are connected to the same network as the Selectc clients. Because of this, we can guarantee good ping and excellent network connectivity, including with Tier-1 operators.
