Three more online Mail.Ru Group courses in open mode



From December 5 to December 15, our Mail.Ru Group Education Department has opened several more courses recorded on the Stepik platform based on lessons in the Technopark without deadlines and deadlines. They can be held in a convenient mode for you without the risk of losing points. Even doing not cater, you will have a chance to get a certificate with a high score.

Since December 5, the following courses have appeared in open access:

  • Multi-threaded programming in C / C ++ . The course examines the principle of multi-threaded programming in the context of the development of network and high-load systems. The core of the course is interprocess communication (IPC) and thread synchronization training. Having completed the course, you will be able to develop your own web server, which will allow you to work out the skills you have gained.
  • Hadoop. System for processing large amounts of data . The course is devoted to methods of processing large amounts of data (BigData) using the Hadoop system. After completing the course, you will gain knowledge of basic storage methods and methods for processing large volumes of data, understand the principles of operation of distributed systems in the context of the Hadoop framework, and learn practical application development skills using the MapReduce software model.

Since December 15, a course on Web technologies has been opened, about which we recently wrote a separate article . In addition, we already wrote that we have the following online courses in open access:

  • Security analysis of web projects - a crash course on techniques for auditing security of web projects.
  • Developing a web service in Java, part 1 and part 2 , in which you first learn the basics of developing a server-side web application, and then develop a web server to an industrial level.

If there are too many courses, and there is not much time for this, then you can take a quiz on which course suits you best! In fact, this test is a part of our joint project with Newtonew for young professionals, but it can also be decided by experienced developers. But you shouldn't trust him too much. One way or another, we invite you to participate !
