The Sandbox has appeared in the Yandex.Direct API.

New software versions can now be tested on a test environment without the risk of damaging active advertising campaigns.

The sandbox mimics the Yandex.Direct API, but is completely isolated from the real data. Created ads are not displayed anywhere, and funds withdrawn and credited are virtual.

However, campaigns simulate a complete set of states:
‚Ä¢ The campaign is created with restrictions in the Directive on the number of ads and phrases;
‚Ä¢ Promotional materials are virtual moderation and activation;
‚Ä¢ Available transfer of funds between campaigns;
‚Ä¢ Virtual money is spent, the campaign stops when they are exhausted;
‚Ä¢ Statistical reports, although they contain conditional data, but in structure coincide with these reports.

Actions that are available in the API only with a positive balance in the Sandbox are available without actually making a deposit. This makes the Sandbox a full-featured environment for debugging key application functions such as working with a cost per click and statistics operations.

The sandbox is available to all users with access to the API. Regardless of the actual role of the user in Yandex.Direct, in the Sandbox both client and agent roles are available, which allows independent developers to create applications for agencies without having access to real agent accounts. To get started, you need to create (initialize) the Sandbox using the special page in the API section of the Yandex.Dreface Web Interface.

Detailed information on working with the Sandbox is available in the Yandex.Direct API documentation .
