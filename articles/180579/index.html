Responsive Email Design, or How to read a letter on the fridge

For us, as for most social networks, email-mailings are very important. One of the interesting and complex technical tasks is the correct display of letters on mobile devices. Now, about 23% of Badoo emails open to them, and this number is constantly increasing. Or maybe someone is reading our letter right now on their Internet fridge.
Today we will tell you what rules to follow in the development of e-mails, how to preserve their usability on mobile devices, and also share our tricks accumulated in the process.

Requirements


What problems have we encountered and what are the requirements for our letters:

  • Emails must be correctly displayed in many different email clients and browsers;
  • they should be optimized for various devices on iOS and Andriod platforms (smartphones with large, medium and small screen resolutions; tablet computers);
  • need mandatory support for Outlook 2003/2007/2010, because it occupies a large segment of use;
  • editing components that are used in another 50 letter templates;
  • it is necessary to create universal blocks that will adapt to the situation, since in letters, data has a dynamic structure (for example, letters are translated into 44 languages, the sizes of blocks, images, text size change in them).

First you need to determine which platforms the project targets. Over the past year, the percentage of opening our letters on mobile devices has grown from 15% to almost 23%.


But do not forget about desktop email clients, such as Outlook, which opens approximately 18% of all emails sent by us.

    Media queries


    The main tool for optimizing emails for mobile devices is Media queries. Below is a table of correspondence with Media queries and the most common mobile devices:
    iPhone 3g, 3gs iPhone 4, 4s iPhone 5 320 * 480 480 * 800 720 * 1280
    max-device-width: 480px
    -webkit-device-pixel-ratio: 1
    -webkit-device-pixel-ratio: 1.5
    -webkit-device-pixel-ratio: 2

    But what about email clients on mobile devices that do not support media queries? For example, the Gmail client for Android and iOS platforms removes the entire style tag from the letter. The following rule applies here, which is key throughout the development of letters: elements with a fixed width of more than 320 pixels cannot be used. In order for the letter to be "rubber", it is better to use relative units of measurement.

    All classes are named with a double underscore. So we solve a known problem in Yahoo, which supports CSS classes, but does not support Media queries. It uses classes for the mobile version of the web, which we do not need at all. Yahoo rewrites the original class names to their own. For example: m__btn to m_btn, that is, it mistakenly replaces the double underscore with one. Thus, the class names in the style tag do not match the names in the class attributes.

    Background Images in Outlook 2007/2010


    Interestingly, Outlook 2007/2010 is much inferior in HTML and CSS support to its predecessor Outlook 2003. For example, background images are not supported in them. The exception is the use of the background property on the body: 

     <body background=""> ... </body>

    But what about the other elements in the letter? The solution is, we can use VML: 

     <!--[if gte mso 10]> <v:rect style="width: ; height: ;" strokecolor="none"> <v:fill type="tile" src=""></v:fill> </v:rect> <v:shape style="position: absolute; width: ; height: ;"> <![endif]--> ... <!--[if gte mso 10]> </v:shape> <![endif]-->

    As a result, we get a block with a given width and height, which may contain text.
