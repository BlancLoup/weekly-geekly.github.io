Microsoft will allow to run Linux in Windows Azure

In the spring of this year, Microsoft plans to launch in a test mode (Community Technology Preview) the ability to run virtual machines in its Windows Azure cloud service, and among the operating systems whose images can be deployed in the "cloud" they also call Linux.

At the moment, the peculiarity of the work of virtual machines in the "cloud" from Microsoft is that they can not save their state - in other words, when you turn off or restart, the virtual machine loses all its data. Until recently, despite numerous requests from business users, Microsoft did not add this capability to Azure, relying on the fact that third-party applications themselves will provide this functionality. This factor, according to the source, was the reason that the ‚Äúcloud‚Äù of the Redmond company was not considered an acceptable alternative to Amazon Web Services or Google App Engine.

Now, the situation will change: virtual machines will be able to save their data using the capabilities of the cloud service themselves, thus greatly facilitating the work of programmers. In addition, Azure, as already mentioned, will be able to deploy their own Linux images, as well as run SQL Server and SharePoint applications. At the same time, the main rival of Microsoft in the field of cloud computing, Amazon EC2 has long provided the ability to run Windows and Linux on its own.
