Revolution begins today

After several years of development, thousands of hours and more than a million dollars of our own funds, we are proud to present the release of MODx Revoution 2.0.

What's new?


MODx Revolution is a brand new MODx. It is the very essence of what makes MODx a great platform for creating various websites and rich applications (RIA), allowing you to do more and do it better.

MODx Revolution 2.0 has become even more powerful. It can serve high-traffic sites, and also allows you to manage multi-sites and sub-domains from one installation of the system.

The list of new features and enhancements in MODx Revolution is too large to list each of them here. Therefore, we will voice only those that, in our opinion, you will love. You can learn more by trying MODx yourself or by going to the list of changes .
  • MODx Revolution 2.0 can be used to create much larger sites. It boasts better performance thanks to a new caching system with the ability to fine tune parameters. In addition, you can replace regular caching with large-scale caching, for example, Memcached.
  • Now you can expand or customize any part of the MODx kernel, including System Preferences, Users, Language, Sessions, and much more. This opens up the possibility of safely expanding the basic functions and the ability to integrate with third-party applications, while maintaining convenient product support.
  • Developers can create separate extensions (add-ons) or even whole MODx distributions using transportable packages (Transport Packages).
  • You can set different views of your site based on almost any connection. Linking opens up a simple way to natively create multi-sites, subdomains, or even perform a secure launch of the MODx kernel outside the root directory.
  • Developers gained access to better error tracking and debugging through kernel tracking. The system allows error detection at various levels and selected external targets, including ECHO, HTML, and FILE. You can also use it in add-ons to audit job reports, error reports, debugging, or other needs.
  • MODx unified simplified tags now support multi-line calls to simplify code readability during development, call cached tags using other cached tags, and connect native and custom modifiers to any element: Snippet, Chunk, TV, etc., similar to PHx in MODx Evolution, but much more efficient.
  • The new MODx manager has been developed using the MODx API, ExtJS (Sencha) and Smarty templates to demonstrate the capabilities of the new kernel and API.
  • Users and managers can now be grouped with detailed permissions and permissions. The user system can be extended to allow external authentication with the ability to store user data via an API. The system of users is completely new - all rights are now reliably delegated using the ABAC (Attribute Based Access Control) security model.
  • Property sets allow you to change any properties for calling a snippet from anywhere or even the entire site from a single section in the Manager. Create property sets, assign them and manage them from a single section.
  • Properties are available for all types of items including Chunks, TV, Templates and Plugins.
  • Manage your MODx vocabulary and customize it - language meanings for things like shortcuts, event notifications, and more for local, industrial, and regional terminology, and all this is the Dictionary Manager.
  • Install add-ons and custom MODx packages using Package Manager. Open the package manager and look for available add-ons or even use your own add-on repository.
  • Manage files in the file system (even outside the root directory) using the Static Resource Manager.
  • Create exact copies of other resources in any desired places in the site tree using Symlink Manager.
