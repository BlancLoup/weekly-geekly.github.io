Razor - a new presentation engine in ASP.NET


One of the projects that my team was working on was the new presentation engine for ASP.NET.

ASP.NET MVC has always supported the concept of a ‚Äúview engine‚Äù; in fact, it is a replaceable module that implements the choice of different template syntax. Today, the standard presentation engine for ASP.NET MVC uses similar files as ASP.NET Web Forms - .aspx / .ascx / .master. There are other popular ASP.NET MVC view engines - Spark and NHaml .

The new presentation engine we are working on is optimized for generating HTML code, focusing on the template code. The code name for this engine is ‚ÄúRazor‚Äù, the first beta version will be available soon.

Development goals


During the way we developed the prototype and evaluated the Razor, we identified several main objectives:
  • Compact, Expressive and Flexible : Razor reduces the ‚Äúkeyboard load‚Äù required to work in the file, provides a fast and flexible programming process. Unlike most template syntaxes, you do not need to interrupt code writing in order to explicitly define server blocks of code in HTML. The parser is smart enough to do it for you, based on the code. You get a compact and expressive syntax that is easy and fun to type.
  • Easy to learn : Razor is easy to learn and allows you to become more productive with a minimum amount of effort in a short time. You use all existing programming language skills and HTML.
  • Not a new language : We deliberately chose a path that is not based on the new imperative language for Razor. Instead, we allowed developers to use existing C # / VB skills (or other language) and provided a markup syntax for a template that gives you a terrific process of building HTML, leaving the choice of language to you.
  • Works in any text editor : Razor does not require a specific tool and allows you to be productive in any old text editor (it works great even in a notebook)
  • Intellisense support : Despite the fact that Razor was developed with regard to the lack of binding to a specific tool or code editor, it will have excellent support in Visual Studio. Visual Studio 2010 and Visual Web Developer 2010 will soon receive updates for intellisense.
  • Unit testing : The new implementation of the engine will support the ability to perform unit tests on views (without requiring a controller or a web server, you can place it in any unit test project ‚Äî there is no need for special application domains )
We spent the last few months developing applications using this technology, conducted usability studies with various volunteers, including non-.NET developers groups. Feedback from people was excellent.

Choice and Flexibility


The best feature in ASP.NET is to replace literally any part of the code. If a particular mechanism does not meet your needs, you can easily replace it with your own implementation.

With the next version of ASP.NET MVC, a new dialog box ‚ÄúAdd-> View‚Äù will appear, which will allow you to choose the syntax for the new presentation template file. It allows you to easily select any available view engine installed on a local machine:



Razor will be listed, as it will be integrated directly into ASP.NET MVC. All supporting methods and models will be available in both Razor and .ASPX presentation engines.

You will have the opportunity to combine and compare presentation templates written using different engines within one application or site. For example, you can create some views using .aspx, others with .cshtml or .vbhtml (Razor file extensions for C # and VB, respectively), and others with Spark or NHaml. Moreover, you can have a view template on one engine using a partial view written on another engine. You are the captain and choose what you need.
