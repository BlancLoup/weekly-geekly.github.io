Big site update, API for uploading your own files and a lot of new content.

First of all, thank you all so much for the large number of comments we received last time. We carefully analyzed your criticism of the site and suggestions and much has been redone, redone. Details under the cut:





- Introduced the top bar with a common player. Now a fixed top bar with a single player for albums and playlists.
- Introduced global album and track functionality
When you hover over an album or track, a special button appears that opens a menu with all the available actions for an album or track: share with friends, add a playlist, or buy an album / track.
- The current listening queue has appeared
When you start listening to an album or playlist, the track list will be added to the queue, which makes it easy to control playback even when you continue surfing the site and leave from the album page or playlist. The same queue stores the history of auditions for the current session.
- Sharing appeared, not only playlists, but also albums and tracks, and not only on Facebook, but also on Twitter, LiveJournal, and also through the embed-code for the blog. You can listen to the shared album or track without registering on the site.
- Altered the logic of buying content, it became easier and faster. For users who have tied their bank card to the service, a one-click purchase is available.
- Written new video player with progressive download.
- Made a new design for the album page and playlist
- Redone search results page. Now you can see all the content that is in our catalog on one page.
- Made saving the listening state. If you listened to something and closed the tab with the site, then when you return to the site you will automatically continue to listen to music from the moment of disconnection.

And we are starting to test a new service for users, - uploading your own music files to Fidel servers.

Especially for you, we present an open API for uploading your own multimedia files to Fidel.

The API can be downloaded from the link from our developer forum after quick registration. devforum.fidel.ru/default.aspx?g=posts&m=1#post1

API features and available actions for downloaded files.

You can download music files of any format. After uploading to our servers, the downloaded tracks will be available for listening in a special playlist of the user called ‚ÄúDownloads‚Äù (for streaming purposes, all tracks are converted by us to mp3 with a different bitrate).
If in the future you want to take your files back to the original format, then when loading you need to use the API function to store the original file on the server.

You can upload only one file in one package, but the number of packages sent for users is unlimited.

For downloadable files, there is one small restriction on use on the Fidel service:

- If we do not have rights to the downloadable track (or we did not recognize it as a track to which we have rights), then it will be available only for your personal listening. We have no restrictions on the number of downloaded files or on the download speed. So use, try and comment.
