Indexing Android application content is now available to all developers.

Level of training webmaster: any

You have not only a website, but also an Android application? Now you can connect one to the other so that users of smartphones and tablets can easily find the content of the application right in the Google search results.

Links to the content of the application in the search results help you quickly find your content and more often launch your application to those who already have it installed. Now you can show users the most appropriate content in the most appropriate form. By linking specific pages of the site with the relevant content of the application, you get the opportunity to influence whether your users go to the site or to the application.



Hundreds of application developers are already using application indexing . Among them are the first developers from Russia, such as Afisha. At the Google I / O conference, we presented many new features that simplify the placement of links to the site in the application, the linking of the site pages with the application settings, tracking performance and possible errors.

Practical implementation is very easy

We have greatly simplified the indexing of application content. If your application supports "external linking" with HTTP, follow these steps:

  1. Add support for web content links to the application.
  2. Link the site with the application
  3. Done!


Having indexed your URLs, we will identify the links between the application and the site and we can begin to show links to the application content in the search results on smartphones.

We are able to locate and index links to the content of your application, but it would be useful to also publish them. Publication is simply necessary if the application supports only a special scheme for creating such links. You can publish links in one of the following ways:

  • Add a <link rel = alternate> element to the <head> tag of the relevant web pages or to a Sitemap and specify the application URI in it. Read more about how to do this on our developer site.
  • Use the App Indexing API


In addition, a new tool for finding errors related to indexing application content has appeared in Webmaster Tools. The new section displays the type of error detected for the site-application pair, as well as examples of problematic application URIs.



We provide detailed instructions for correcting errors, including a QR code with a link to the content of the application, so that you can quickly open and view it on your smartphone or tablet. Finally, so that you do not miss anything, if errors occur, we will send you notifications.



Try to provide your application's content for indexing. If you have any questions, please contact the webmaster help forum , and, of course, continue the discussion of interesting examples and implementation details in our webmaster community .

By Maria Moeva, Webmaster Trends Analyst
