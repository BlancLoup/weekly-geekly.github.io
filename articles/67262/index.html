We are looking for feedback: Scold and give tips for the user interface of the disk shell

Over the past few years, I happened to make disks for Microsoft Russia several times. These were the most different discs, where the main idea is a shell for viewing the disc content. As the basis of the user interface, I took a weblog model, where there is content, there is a tag system for finding the desired content, as well as additional links.

The first disc was made back in August of 2007 - a webcast for students. The code was written by guys from Logrus (they are not only involved in localization;)), and since that time the disk has been used several times.

More precisely, the shell code.

This code was used for the disc of the ReMIX 2008 conference and the Platforma 2009 conference. This time I have the task to make a new disc, and there is an opinion that the user interface of the disc is very incomprehensible and complicated.

Naturally, I have my own opinion, the author, after all, but in the words of opponents there is logic, in connection with which I want to bring the design of the disc to discussion.

Under the cut there is a screenshot of the disk design and a link to a simple archive, by downloading which, you can launch the shell to one of these disks.
