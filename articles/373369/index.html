There is no "coolest type of memory" - there are suitable and not suitable for equipment types


Modern servers use DDR3 and DDR4 memory, but not the whole row, but the following varieties:

  • Unbuffered ECC DIMM is what experts call UDIMM. Typical memory modules of up to 8 GB each are the most budgetary option in servers and workstations. From ‚Äújust RAM‚Äù for home PCs, it is characterized by the presence of an ECC scheme that detects and corrects errors in individual bits of memory.

  • Registered DIMM is the same register memory that de facto has become the hallmark of middle-class servers and workstations. The most common option in a wide retail. Modules up to 32 GB each, slightly slower than non-registered, but noticeably more reliable. It always supports ECC and is more practical than Unbuffered, because it scales much better (number of modules per channel).

  • Load Reduced DIMM or LRDIMM - ‚Äúlast squeak‚Äù (2013 and newer) modes in server platforms. Compatible with RDIMM connectors, but arranged differently - modules have a memory buffer for transferring data from the host to DRAM, that is, the LRDIMM takes over some of the functions of the memory controller, and the controller works in serial mode. By reducing the electrical load, the memory works either faster than RDIMM with a higher clock frequency, or at a much higher capacity than its predecessors at the same frequency. Monitoring tools have also become better.

It is most appreciated for the opportunity to pile up an impressive (9-12 modules per processor) number of modules and thus achieve the maximum amount of RAM in the server.

In multiprocessor systems, the memory must be configured in the same way. The maximum speed depends on the processor, because the memory controllers are now located just under the CPU lid.

  • NVDIMM - a mixture of horror and hedgehog of usual RAM and SSD, hybrid non-volatile memory . Fits into standard DIMM connectors and is compatible, but comes with a standalone power supply. Does not lose data in case of power supply failure. While not too popular, but already used in cloud computing.
