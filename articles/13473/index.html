Flash An old dream, the end of the context menu?

Remember how we used to dream of turning off the context menu within the browser, with the once-hated About Macromedia / Adobe FlashPlayer. It was a long time ago and everyone had long since accepted the fate, fortunately, at that time the ContextMenu class appeared, at the time holding back the righteous rage of the flashers.

But we needed to remove this menu completely. Some people say, why do you need to disable the context menu for swf-videos? There are several reasons for this:

Games The Flash games industry is quite extensive and here we mean non-executable applications, where killing a menu takes a couple of lines of code, namely killing the context menu within the browser. In many games, the right-slit is used quite actively and flash-game developers have reluctantly refused this opportunity.

User Experience - yes, if you have a remote control for two buttons - this is better than using one left. Who knows, maybe in some time I will come up with a unique interface where all the mouse buttons will be used to the full.

RIA - here without any comments. We need the right mouse button!

And now, we can say YES to its context menu in our AS3 and Flex applications. Someone Paulius Uza created a test right-click script and asks the world community for help. We will test its script on various browsers and operating systems, as it is possible soon we will have another opportunity and more extensive freedom of action in the development of our own applications.
