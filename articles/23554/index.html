Idea for a startup: IdeaExchange

As a product manager of one of the Parallels products, it is very important for me to correctly prioritize for a variety of features, changes, improvements and, frankly, bugs that come to me daily from the support department, forum, sales, top management and other channels.
What should be done now and immediately, and what can wait? How to properly distribute the efforts of developers in the current release, so that users are as satisfied as possible? These are the questions that arise every day for me and not only for me, but also for all products / services that are designed to meet the needs of their users to the maximum.
The obvious way to solve it is to give users the opportunity to vote for or against this or that feature.

What is new in this idea you ask? Everyone talks and writes about it.

I am looking for a good / convenient tool that will allow:
1. Create new feature requests, enhancements, bugs - OpenID authentication required
2. Display a list of requests, sorted by votes (popular), last (new), last commented
3. Search for queries by keywords
4. Vote for / against the request
5. Comment on requests
6. Moderate requests
7. Mark in which release this feature is planned.
8. To note that the feature is implemented in such a version of the product.
9. Watch which features will be included in the specified release.
10. In general, view the roadmap product for upcoming versions
11. Display statistics on requests / comments
The variant of implementation can be viewed at: ideas.salesforce.com

The tool is needed as a separately installed web application, or as a SaaS application that fits into the design of the company's corporate website.

Tell me if there are any ready-made implementations of this functionality on the market, or is you ready to assist in promoting this startup.
