The continuation of the epic with a USB stack


We still managed to break the ice, we still found several testers during the last stage and are ready to move on.

Developer Vadim Galyant poured an experimental ReactOS image with updated components of the USB stack into the cloud. We need your tests again. Urgently! It only depends on you whether the developments will fall into the general branch before release 0.4.4. There is very little time left, literally a couple of days.

‚Üí Link

Features of the image:

0. This is a LiveCD, run from media without installation.
1. For tests of real hardware, it is necessary to replace all three usb drivers (u | o | e) hci with Windows drivers (XP SP3 or 2003 SP2).
2. Supports booting from a flash drive.
3. There is an important nuance - flash drives during operation cannot be removed AT ALL - this is tantamount to disabling the hard IDE hard drive. The consequences will be fatal for data on a flash drive and running system. In theory, and add to a hot flash drive is not worth it, too.
4. For the purity of tests, it is advisable to completely disable all available sound cards on the tested computer.

Before experimenting with booting from a flash drive, it is advisable to boot from the CD-ROM first and make sure that all USB devices are seen and working properly. And then test boot from USB.

If you are ready to test just booting from a flash drive, then use 1 point of this instruction and the Rufus program.

HDD, card readers, drives, printers, scanners, modems use BULK transmissions. Therefore, if the flash drives work, then the rest of the devices from this list should work. The truth in ReactOS is no support for all types of devices at a higher level.

Mice, keyboards, joysticks use INTERRUPT transmissions.

Multimedia devices use ISOCHRONOUS transmissions (in this image there is no support at all!).

Ask any questions directly to Vadim and send him reports on the results of image testing.

We are waiting for results!
