Delta Chat - decentralized messenger on top of email

Is your email out of date? Which messenger should be used to avoid getting into a difficult situation and not losing touch with friends? Is it possible to use old technology in a new way?

Some brilliant ideas are so simple that afterwards there is always bewilderment, why did something so obvious not come into common use earlier? Well, if this has not happened so far, now - in the era of a new wave of censorship, government blocking of Internet resources and the constant threat of surveillance - it's probably time to rethink the approach to e-mailing ...

I present to you the project DeltaChat




Delta Chat is a free decentralized messenger for Android, iOS, Linux, Mac, operating on top of the oldest stack of E-mail protocols. A Windows version is under development.

The key features of DeltaChat are the following:

  • Delta Chat does not have a single server or infrastructure ‚Äî the application connects to any email server you choose;
  • Distributed communication with other domains through the federation of mail servers;
  • the recipient can be any email owner, even if he hasn‚Äôt yet installed Delta Chat;
  • if possible, the Autocrypt mechanism is used for encrypting correspondence, messages are encrypted automatically (and a lock is shown to the user);
  • the display of read marks and delivery status and quick notifications function thanks to the Push-IMAP protocol ;
  • to protect against spam, by default, only messages from the contact list are displayed;
  • fully open source GPLv3 licensed application and standards-based protocols

FAQ


Does Delta Chat support images, videos and other attachments?

Yes. In addition to plain text, all email attachments are displayed as separate messages. Outgoing messages will receive attachments as needed automatically.

Does Delta Chat support HTML emails?

Yes. If the incoming email lacks the plain text part, we convert the HTML emails into plain text ourselves. Outgoing messages always use plain text.

Does Delta Chat support pass-through encryption?

Yes. Delta Chat implements the Autocrypt Level 1 standard and can thus provide end-to-end E2E message encryption with other Autocrypt-enabled applications.

What do I need to do to enable end-to-end encryption?
Nothing.

Delta Chat applications (and other Autocrypt-compatible email applications) exchange the keys necessary for end-to-end encryption automatically at the moment that the first messages are sent. After that, all subsequent messages are automatically encrypted. If one of the chat partners is using an email application that is not compatible with Autocrypt, the messages are not encrypted until an application compatible with Autocrypt is available again.

If you want to deactivate end-to-end encryption, use the appropriate setting in the ‚ÄúSettings / Advanced Settings‚Äù section.

How to check encryption?

If a small lock is displayed next to the message, this means that the message is encrypted with end-to-end encryption and sent from this sender, and your reply will also be protected with end-to-end E2E encryption.

If the lock is not displayed, the message is protected only by the transport level of encryption from the application to the mail server. because you or the sender have disabled pass-through encryption or the sender is using an application that does not support pass-through encryption.

How can I check the sender?

The user profile displays additional information:

You can tap ‚ÄúQR invite code‚Äù on Android, and then use the ‚Äúscan QR-code‚Äù button on another device to scan this code. If both devices are connected to the network, they will create a chat channel with each other (if it does not already exist), and the encryption keys will be checked.

To diagnose end-to-end encryption, Delta Chat shows two fingerprints of keys. If they are the same on your device and the device of your chat partner, then the connection is safe.
