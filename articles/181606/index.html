Prepare a sleigh in the summer, and a cart in the winter. Copy space grows stronger . So it's time to prepare for difficult times, and take care of creating an alternative Windows-compatible operating system. Therefore, the ReactOS project team proudly presents the next release:

ReactOS 0.3.15



The most significant innovations of this release:

  • A ‚Äúracially faithful‚Äù implementation of USB support for the mouse, keyboard, and storage devices.
  • Rewritten session management subsystem.
  • Alternate ReactOS Memory Management Module took on almost all the responsibilities of memory management.
  • Updated UniATA driver with SATA-1/2 / AHCI support.
  • Advance support for debugging ReactOS components using WinDbg.
  • Virtual PC (and potentially Hyper V) is returned to the list of supported virtualization platforms (except for a bug with a video driver ).
  • Various improvements due to the introduction of the AutoHotKey based test suite.
  • Bugfixes on the results of scanning utility Driver Verifier built-in drivers distribution.

At last


In addition to the traditional versions of the release distribution, you will find a hybrid disk image used to demonstrate ReactOS at LinuxTag. This image includes some additional USB fixes that provide more reliable initialization and device discovery, as well as the default Lautus theme. Please note that support for themes has not yet been fully implemented and the use of Lautus in a hybrid disk for LinuxTag was made primarily for demonstration purposes.

PS Unfortunately, significant improvements in the work of console applications , as well as the most recent version of the UniATA driver, were not included in the release for several reasons. But do not worry, you can easily test them in ReactOS " night builds ".
