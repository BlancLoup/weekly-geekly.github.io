One of these contests has already been mentioned at Habr√© - this is the Baltic Science and Engineering Competition .

Sample project number 3: Modification of Dalvik VM for beta testing of Android applications with the ability to play custom actions

  3. Inspired by the compatriot project about Back-In-Time debugger. I decided to do something similar for Android applications.
  4. Made a modification of Dalvik VM (java virtual machine, used in Android), in which the code recording system events and user actions is automatically inserted into the bytecode of the application under test.
  5. I developed a ‚Äúplayer‚Äù that allows you to ‚Äúre-play‚Äù the life of an application created by a beta tester using the special log file created in this way. In this case, of course, it is possible to use a debugger.

Specificity of scientific projects, unlike programming contests


It is extremely important to understand the fundamental difference between the Baltic competition and others like it from all sorts of programming contests.
At programming contests:
  • A task that is already 100% solved before you by ‚Äúsmart uncles‚Äù is given;
  • The task needs to be solved in 10 minutes, the schoolchild should think very quickly, quickly ‚Äúput out the fires‚Äù;
  • The quality of the code does not matter. The main thing is to send a file with a task for verification (well, the one that is ‚Äútask3NewNewNewFinallyWorks.cpp‚Äù) and get the cherished ‚ÄúAccepted‚Äù;
  • Only the basic elements of the language are used: arrays, strings, a pair of standard algorithms.

At the Baltic Competition:
  • An open end problem is given. No one really knows what will happen in the end and whether it will turn out at all;
  • The task can not be solved in 10 minutes. The task is difficult and, as a rule, a year is done;
  • The result of the student is a full-featured software product. A schoolboy is forced to learn to write the project code correctly, to think through the architecture correctly;
  • A schoolboy ‚Äúsmokes‚Äù 1000 manas to understand complex technologies, and in 90% of cases all this is in English.
