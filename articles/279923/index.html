Automatic completion of JS-code from the base Stack Overflow

A couple of weeks ago, Stack Overflow laid out in open access a dump of all content published on sites on the Stack Exchange network for 2014-2016, in XML format. There all questions and answers, comments and ratings: only 33.8 GB in the archive.

Web application developer Emil Schutte has found a useful application for this dump. He extracted from there code fragments that fit the criteria (approved responses with more than 50 votes and the "javascript" tag) and indexed them.



This is how the program Autocomplete from Stack Overflow appeared . If you write something in the form, then it searches for a suitable fragment in the base Stack Overflow - and automatically offers it.

It can be expected that the auto-completion function is implemented for other programming languages. And not only on the basis of Stack Overflow, but also on the code of various open projects.

Stack Overflow is a popular programming questions and answers system developed in 2008. Now there are 11.3 million published questions , 18.4 million answers and 160 million comments, edits and votes. Every day, 8 million developers visit the site to find a solution to a problem. The authors of the site themselves estimate that if each visit to the programmer saved 30 minutes of time, the total savings will be in the region of 1.85 billion hours.
