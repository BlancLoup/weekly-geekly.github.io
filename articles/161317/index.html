Backup (RK) of data of IBM SAP HANA by means of IBM Tivoli Storage Manager

What is IBM SAP HANA

SAP HANA - High Performance Analytic Appliance. It is a modern platform for business applications and real-time analytics with in-memory queries.

IBM has prepared a balanced platform for solving problems implemented by SAP using In-memory computing technology. The IBM SAP HANA complex consists of computational blocks that are flexibly scaled and contain GPFS software ‚Äî a high-performance file system with disk sharing that provides high fault tolerance.

How is SAP HANA backup organized by standard tools?

Backup is used to protect data from errors when working with disks.

SAP HANA backup includes:
  • RC for file system (external mounted drives)
  • RK containing a replica of data from the ERP, simulation data
  • RC is carried out using the ‚ÄúSAP in-memory computing studio‚Äù or using SQL scripts via the command line
  • For RK configuration files, they are manually copied to another storage device.
  • RK log for SAP HANA version 1.0 is not available

Calculation of space on disks for the Republic of Kazakhstan is based on the used RAM for SAP HANA + 10-50% reserve. Only used space will be reserved.

Consider implementing a backup system for SAP HANA with a volume of 3.5 TB

As shown in Figure 1, the SAP HANA complex in question consists of 8 nodes. There are 7 productive nodes, one of which is allocated as a master node, and the 8th Standby node for implementing the high availability function within a single cluster.

Figure 1 - SAP HANA Cluster



It should be noted that the IBM SAP HANA solution was built using the GPRS cluster, and does not require maintenance and the availability of a disk array for the operation of the complex itself.

The master installs the Tivoli Storage Manager and Tivoli Storage manager for SAN backup client, it is also recommended to install them on the Standby node to eliminate the single point of failure.

At the initial stage, the data is uploaded by means of HANA Data Studio to a file system outside the SAP HANA cluster, the implementation of this feature is described in the SAP documentation.

For the organization of long-term storage of backup and archive copies, we connect a disk system to the IBM SAP HANA complex. As a result, using DataStudio on disk storage we get ‚Äúflat‚Äù files for further copying with Tivoli tools.

The backup client collects data from the disk system and initiates data writing to the tape. To reduce the load from the productive server from the backup process, it is proposed to use the Lan-Free client. For this purpose, FC adapters have been added to the master node and standby cluster node. The IBM Storwize V7000 disk system was proposed as a disk system that can satisfy customer requirements for the volumes of stored data and ensure high access speed.

Solution components:
  • Tape library
  • Disk System - Storwize V7000
  • Backup server
  • SAN switches
  • Software - Tivoli Storage manager Extended Edition
  • Software - Tivoli Storage manager for SAN

Conclusion
