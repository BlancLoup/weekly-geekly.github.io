Running KDE 4 under Win

Crawling across the expanses of the Internet, I came across the fact that KDE is ported under Win and at the moment you can already install the Plasma shell. Let no effects and incredible glitches, but possible.

I decided to put myself, taste it, see the result in pictures yourself.



There is an option to start the installation immediately from the network, but after several unsuccessful experiments, I decided to secure myself and first selected the Download Only mode, and then the local installation.

So let's start:

Stage One - Download


First we need the kde installer for wines. As developers write:
  • Go to http://www.winkde.org/pub/kde/ports/win32/installer/
  • Download the latest version of the installer and save it to disk, for example in C: \ KDE4

Microsoft Visual C ++ 2005 SP1 Redistributable Package (x86) libraries are still needed

Next you need to download the components:


After that, we are offered to choose the directory where to install KDE, and then the mode in which to install.
Because we need unstable version 4.1.96 then choose

Unstable - because the plasma shell is not yet complete, and cannot be attributed to stable.

Then they suggest choosing the path - where to put the files.


And choose the type of network connection. Then the server - from which to load the packages. I stopped in German, with him I had the highest download speed.


After that, the installer loads the package lists and offers to choose the version.


And then the packages themselves.


After the list of dependencies and download, will give a window on the completion of the work.


Next, go to stage 2 - installation



Run the installer again and select the local installation mode, remove the tick.


Choose if you ask the folder where you downloaded the packages, choose

Stage 3 - Launch



Register variables according to the manual on the site.






We try to start something:
Try running some Qt application in the bin folder, for example linguist.exe
If you can, try running any KDE application such as kwrite.exe.


If everything is ok, then try to run the plasma. For some unknown reason, for me, just starting up resulted in APPCRASH, but how the shell started everything.
