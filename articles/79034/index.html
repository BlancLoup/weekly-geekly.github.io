WEBO Site SpeedUp alpha version

WEBO Site SpeedUp is a product for integrated site acceleration. Applies all technologies of client (and partially server) optimization. The average acceleration is 2.5 times.

What is the salt? Salt in the interface. It is completely rewritten. From scratch. We tried to take into account all the wishes of users and think over each (maybe not all are yet - this is just an alpha) trifle. Now to deal with the acceleration of the site should be easier and more understandable.

For the past few months, we have been processing all the impressions from users and have carefully tested every detail, discussed and watched.

Control panel



Now on the main screen contains all the functions that may be useful. You can immediately see what state the application is in, what cache size, what other measures you can take to speed up and what problems this server environment has.

File cache



On the cache screen, you can view information on all created files, as well as quickly recreate the cache.

Optimization tools



One of the most "delicious" parts of the new application (fully available in the alpha version). It allows both to create gz-versions of files (for static compression), and to optimize images through smush.it.

Now all the functions available in Web Optimizer are regrouped for greater convenience and are available on separate application screens.

Unfortunately, in the alpha version, we did not have time to prepare a new significant improvement: the use of various configurations on the fly. Wait next year.

This interface (with minor changes) will be ported for all supported CMS (images are for standalone applications, the same appearance is planned for WordPress, Joomla !, Drupal, Bitrix system plugins).

Heredity


Since updates for Web Optimizer are free in all editions, to install WEBO Site SpeedUp you will only need to upgrade (to version 1.0, when it appears, or now to beta version). All license keys are suitable for the new application.
The alpha version has been tested so far only under modern browsers (IE8, Firefox, Safari / Chrome, Opera - yes, in the last Opera alpha, the interface looks especially good because of the border-radius :). In other browsers artifacts and interface malfunction are possible. We are working on troubleshooting.

The main response that we expect to receive is how comfortable the new interface is for you, the users. How well is it suitable for daily work with the application. What is good in it and what is not very good. Comments are welcome.
