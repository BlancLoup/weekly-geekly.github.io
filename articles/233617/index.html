Personal storage of html-fragments for amateurs and professionals

Personal pages that host widgets that are selected and customized for personal needs have long been outside the main trend of web services development.

On the one hand, personal page services exist and delight their loyal users; the best example is the My Yahoo service.

On the other hand, such services stop working. The closure of the iGoogle service was announced in the summer of 2012, and on November 1, 2013 all personal pages of the users of this service were permanently deleted.

In general, one can state zero interest of investors and entrepreneurs in the creation of such services. It seems to be a good idea to provide the user with only what he needs on one page encounters quite natural difficulties:

  1. The relative complexity of the process of placing new widgets: you need to go to a special place, choose what you need, and place your favorites on a personal page.
  2. With a large number of widgets offered, the lack of diversity.
  3. Tightness in the strict framework of the possible functionality - only possible that is approved by the administration.
  4. With a large number of custom widgets (tens, hundreds) is difficult, almost impossible to manage.

In TheOnlyPage web service, the concept of widgets as components built according to a predetermined scheme was rethought and it was decided to remove all restrictions and use an arbitrary html-code .

In a previous publication, it was said about the functional content of online note services. The TheOnlyPage web service was considered , the most ascetic option of presenting notes is the bare text. A kind of notes in pure form.

The directly opposite option is to save widgets represented as arbitrary fragments of html-code . All web design tools can be used.

For marketing reasons, in order not to confuse the average user, it was decided to call such html-fragments , Images . Although visual objects displayed using arbitrary html-code, of course, are not limited to images.

Some details of the proposed solution.


Earlier, we discussed the selection of a successful visual metaphor for the bookmark service and its implementation in TheOnlyPage web service.

Naturally, the pictures in the TheOnlyPage service are presented to users in the same pattern as bookmarks and notes . Top: search bar; in the left navigation pane: the last saved selections by text fragments and color labels; occupies a central place: a list of images ordered by viewing / editing / creating images.
Just click on the title of the image - opens it and allows you to see its contents.


Double click on the line containing the image title opens the edit form:



To create a picture, just press the New Picture button, paste the html code into the Code field, enter the picture caption in the Signature field and click the Save button.

If there is a desire to see how the browser is processing the specified html-code - click the View button.

Thus, a professional html-coding can create special solutions for themselves. Fans still need to copy-paste any informer codes, pictures or videos, or use a special bookmarklet to collect html fragments from other sites.

Bookmarklet usage example


A bookmarklet for creating new bookmarks, notes and pictures can be set by simply dragging a special link to your browser‚Äôs bookmarks bar. You can read more about installing a bookmarklet in the online documentation .

For example, reading the most popular publication on habrahabr.ru we wanted to save the video from this article.

Simply click on the TheOnlyPage bookmark link and, if necessary, go to the Pictures tab. The service will automatically determine which object to save, based on the size and location on the page, and will suggest the most noticeable for saving:



In the Code field, the html-code of the saved video will be automatically substituted, as well as the address of the page from which the media object was taken.
In the Signature field will be the title of the page.
By clicking the View button, you can see how the picture will look. You need to preview the saved images to make sure that the saved object will be correctly displayed in TheOnlyPage .



After pressing the button Create a new image will be created and saved as a fragment of the usual html-code.

When using a bookmarklet , there are three ways to determine what to save as a picture.
  1. If something was selected on the page, then the html-code of the selected objects will be offered for saving.
  2. If nothing is selected on the page, and the page belongs to one of the popular sites that specialize in the presentation of media objects, such as YouTube‚Äôs video hosting service or slideshare hosting presentations, TheOnlyPage will automatically build an HTML code fragment intended for the presentation of the corresponding media object.
  3. If nothing is selected on the page, and there is no special mechanism for constructing a picture for this site, then the html-code of the media object occupying the dominant position on the page will be used.

As you can see, the average user will cope with the task of storing the media fragments they like in their private collection. But a professional who understands html-markup will do this with the knowledge of what is happening and, possibly, will correct the stored html-code in their own interests.
