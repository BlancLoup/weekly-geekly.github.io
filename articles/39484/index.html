New Opera-9.6 - new features

Today, the first public beta version of Opera-9.6 browser has been released. You can confidently say that your favorite browser has become even faster, more qualitative in terms of support for modern web technologies (Acid3 = 85), and also traditionally introduced several new functions. These include:

Expansion Opera Link - now you can synchronize not only bookmarks, Quick Start, notes and the contents of a personal panel - they also added a list of search resources and a history of entered addresses.

Previewing news feeds - when you subscribe to receive news feeds, you can now preview their contents.

Changes in Opera Mail:

Low Bandwidth Mode is a new feature that allows you to minimize the consumption of traffic by the M2 mail client. For the IMAP protocol, this means that only messages will be synchronized, and all attachments will be downloaded only at the request of the user. In case of using the POP protocol, only the first hundred lines of the message will be loaded, the rest will be loaded when opening the letter.

Track / Ignore topics or contacts (Follow / Ignore) - this feature allows you to better navigate among the messages received daily, simplifying and speeding up access to really important letters.

Go to thread (Go to Thread) - this feature allows you to view all messages that relate only to the specified conversation thread.

However, in addition to adding new features, developers continue to improve existing ones. In particular, the new beta browser contains the following changes:

Interface
- Opera now remembers the position of the bookmarks bar after restart;
- Added new settings option opera: config # UserPrefs | ShowBookmarksInAddressfieldAutocompletion, which allows to exclude from the drop-down list of completion of the list of bookmarks;
- Setting the opera: config # TransferWindow | KeepEntriesDays option to 0 allows you to delete the download history when restarting Opera;
- A special marker appeared that allows you to track the edges of the previously viewed part of the page while scrolling. You can enable it by setting the opera: config # UserPrefs | EnableScrollMarker option to 1 (the marker will be displayed only if you have reached the end or the beginning of the document) or 2 (the marker will be displayed at each scrolling);
- Web sites with authorization are now saved in the list of entered addresses;
- Opera has learned to stop loading pages with an iframe if the user decided to close it or stop the download;
- The document property is removed from the iframe object for compatibility with Gecko, WebKit and Acid3 test;
- The dialog box to protect against fake sites appears when working through a proxy;

Mail, News, Chat
- In the IMAP account configuration dialog box, the option to disable attachment download has been added;
- Fixed opening and saving images in the mail using the standard context menu of image processing;
- All main mail views (with the exception of "All messages") are made selectable (even if you kill, you still do not understand what the essence is);

Displaying pages and processing scripts
- Fixed the problem of dynamically adding instructions for processing XML-tables;
- Added support for the caller function property (more details - here );
