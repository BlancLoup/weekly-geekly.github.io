Our VoIP startup Octopus


It is time to introduce our startup Octopusline to the general public.
Several introductory ones so that a disinterested audience would not regret the time spent reading: this is not a social network, not an aggregator, this is not a mashup with google maps, no gloss web, it works while on windows and it is not free. In general, it is not fashionable :).
We are a small group of people, working under the wing of a large company, having complete freedom of action and evaluated by result.

We have made a communication system (PBX and Call Center) based on SIP, which should not be bought. PBX can be configured via the browser, and soft calls can be made via SIP. PBXs usually do not buy home, so this is a product for business.

In order to understand what will be discussed further, you need to know what the SIP protocol is. You can read an article on Habr√© about SIP , you can search for information yourself, or to understand what is written below.

SIP is a session initialization protocol that works in IP networks using very familiar and understandable methods and helps to manage video and voice streams. For example, you launched a SIP server on the TELE.RU domain and created a KARL user there. Your SIP address in this case SIP: KARL@TELE.RU - looks like an email. Another user got himself an account on the server MULTI.RU. His address looks like this: SIP: ALEX@MULTI.RU.

In order to establish a session with ALEX, KARL sends a request to your sip server ‚ÄúI want to talk to SIP: ALEX@MULTI.RU. SIP server TELE.RU using DNS finds the server MULTI.RU, he finds out whether the ALEX user is online, his IP address, and sends the information back to the server TELE.RU. Then the two servers agree on what and how they will transmit (voice for example or video) and give SIP commands to users' devices (softphones, IP phones) to start a conversation.

And now the most important thing is that SIP has penetrated traditional telephony, and operators one by one begin to offer SIP phone numbers to the market. SIP phone number looks like a normal city and in fact converts the signals of traditional telephony into IP traffic managed by SIP. In practice, it is a box in which, on the one hand, the telephone wire is plugged, and on the other, the Internet. You, as a user, will have access to this number via the Internet, using any softphone or IP phone. For access you will be given the address of the operator's SIP server, a login and password to it.

What we offer in the West is not new, it is called Hosted PBX or Virtual PBX, and is in steady demand. You simply redirect IP traffic from your SIP number to our PBX and manage all settings through the web interface. Logically, you can clarify the analogy with the services of virtual hosting - many simply do not need their own server (or IP PBX), located on the territory of the company or the data center. It is expensive to purchase and maintain. A virtual hosting (or a virtual IP PBX) offers all the same, but on the basis of small volumes and cheap. At the same time using SIP is implemented simply a huge number of functions that covers any conventional PBX.

What is the system and what has been done. A SIP server, a whole bunch of telephony functions, a web-based management interface with separate access rights and a win softphone were implemented.
