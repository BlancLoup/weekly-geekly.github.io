Github Releases: Release Releases

Github developers have implemented a new Releases feature for easy software distribution to end users. Entering the Releases section, the user can always find the latest version of the program, the changelog and the full version history. Link to releases posted on the main page of the project.



Along with each release, text notes, links to the binary file and source code are published.



For numbering versions it is recommended to use semantic numbering .



Window to create a new release. By specifying the version number and release description, you can attach separate files: for example, binary distribution, minified scripts, program documentation in PDF, etc.



If they also roll out their package manager and payment system, then they will have an ideal ecosystem for distributing applications. Who knows in which direction Github will evolve. With a recent investment of $ 100 million, they can only do what their heart desires.
