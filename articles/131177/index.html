Sms


About sms informing users asked and wrote for a long time. At the moment, smki are tied to almost any event that can occur with a car, be it speeding or just opening the door. SMS is not tied to the balance on the device's SIM card, but is tied to the personal account in the system, from which the SMS fee is charged. This service works in almost all countries of the world where there are mobile operators. The whole event processing cycle lies on the server side, which means that it is not necessary to be constantly online, when an alarm event (or a programmed action) is received, the SMS itself will inform the operator or the host. One of the features of working with SMS is that it is possible to assign information from digital sensors by transmitting information from them in a text message (changing the temperature in the refrigerator or changing the level of fuel in the tank).



SMS to driver


In the system, a driver or several drivers are tied to each vehicle, but only one is assigned during the flight. Each driver has a mobile phone, which in turn is tied to his account. Now the operator (dispatcher) can send messages (about changing the route, for example) directly from the monitoring service, dialing it and send it via the dispatch form. Further, this function will grow into communication with the driver using a navigator or PDA, but at the moment it is the most convenient way.
