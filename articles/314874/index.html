Decompiling Java methods on a productive application under load - myth or reality?

Testing is undoubtedly one of the whales on which application development stands. As with any characteristic whale, testing can be fouled with bugs and does not stop for a long time. Sit back. Let's talk about OverOps - a solution for detecting errors in the operation of applications in Java, Scala, Clojure and Groovy. The essence of OverOps work is as follows:

1) when you start the JVM, the OverOps agent starts next to it;
2) the agent is able to monitor the occurrence of exceptions in the code (both processed and not processed);
3) the agent at the time of the exception removes the heapdump and imposes it on the decompiled bytecode.

As a result, you can see what values ‚Äã‚Äãthe variables took until the time the exception occurred. When an agent is running, the vendor claims its maximum overhead is 3%.
