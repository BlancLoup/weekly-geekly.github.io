Desktop Backup Systems

There is:
Number of Files: 283309
Total Size: 62.92 GB
Backup Type: Incremental
Source: Network Drive (LAN)
Storage location: local computer (Windows), from which the copy program starts
Task: daily incremental back-up of data, storage of copies in the last 2 weeks.

At first, I wanted to find some kind of free solution. But I could not find anything worthwhile ... Therefore, I switched to commercial systems.


Backup Manager - an overview of this program can already be found here , so I will not go into details. Let me just say that due to the lack of the possibility of incremental backup from it, I had to give up right away.

Genie Backup Manager - worked for a month, one of the best in functionality programs. Recently, in the creation process, it falls out tightly, generating a very strange Windows error (it simply falls out). It does not create any logs in this case, I could not find the reasons on the forums. In the process of backup ate about 1GB of RAM. Backups are created in the form of a single file with the extension gbp, which cannot be read anymore, a lot of space is allocated on the disk by default for some of their internal directories (Cataloging), which are disabled in the settings (gigabytes). Pretty slow in terms of work. The system is paid. There are several types of backups: full, incremental, differential, mirroring. Can do self-extracting backups.

ApBackUp is a rather simple program, it shoved all the files into one archive. It worked perfectly with a small amount of data, when I backup a large one, I think about something for an hour, then it shows some strange error, creates an empty archive and that's it. Making work could not.

Handy Backup is very convenient. simple and clear with the Russian interface program. It works perfectly, it does not reflect after pressing the start button, BUT the more files it has already processed, the more it takes up space in the operator. When processing the specified task for 35% in the memory already hung about 3GB of this program. It can create as one archive of the entire backup (no more than 2GB), or just keep the folder structure on your disk, each file in which will either be original or as a separate archive. It is able to complete and incremental backup. In my case, after 8 hours of execution, I made 50% of the full backup, after which I wrote the following:
ERROR:> 03:54:19 Unknown Win32 error. [C5670E02]
STATUS:> 03:54:19 Backup operation finished on Thursday 09/18/2008 3:54:19
2008-09-18 03:54:19 SESSION ENDED

Making work could not.

What do we have in the end? At the moment I do not have a working system. I really hope for Genie Backup Manager, with whom I am experimenting now, in order to understand his problems, for he is the best one presented here.
