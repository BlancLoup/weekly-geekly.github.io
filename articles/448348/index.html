Elements: theoretical part


An element (object) is a unit: material, abstract or conditional.

Containers and objects


The elements are divided into: container (K), container object (KO) and object (O).


  1. Container (K) - an element that can contain other elements - container objects (KO) or objects (O). The container itself cannot be inside of anything.
  2. Container Object (KO) is an element that can contain other elements - container objects (KO) and objects (O), and at the same time it can be inside a container object (KO) or container (K).
  3. An object (O) is an element that cannot contain other elements within itself, but can itself be inside a container object (QO) or container (K).

Relativity


Each of the elements can be considered separately:

  1. As a Container, cutting off information about everything inside of which it can be, making it original, in relation to which we will consider everything else. For example: a book, when we take it for a Container, it doesn‚Äôt matter that it is in a room or in a closet on a shelf, only it‚Äôs important and what‚Äôs inside it (sheets, pages, text, chapters, paragraphs, sentences, words , syllables, letters, symbols).

  2. As a container object. Then it is taken into account, and what is inside the element, and what is inside itself.

  3. An object. When we consider an element as O, then we directly go to its type and properties, while not forgetting about the element within which it can be. If we turn to the consideration of the book as an object, then we begin to consider its parameters: length, thickness, width, author, title, etc.

Interactions


Elements can interact with each other, even though each of them may be in a different branch of the tree (DOM). The interaction between elements and element types occurs within a specific pattern.




Interactions are otherwise called intersections (cross).

Take the two elements E1 and E2. When crossing, E1 and E2 may have some influence on each other:

  1. E1 has an effect on E2.
  2. E2 has an effect on E1.
  3. E1 and E2 have equal or no influence on each other.

If we consider the example with the address given above, then if Ivan Ivanovich is taken as E1, and his apartment is E2, then the first interaction option will suit us, since Ivan Ivanovich can sell his apartment. The apartment does not have an impact on Ivanovich.

And if for E1 we take the Worker, and for E2, the work, then option 3, since The employee works and he gets paid for it, and the employee can be fired from his job or he can quit.
