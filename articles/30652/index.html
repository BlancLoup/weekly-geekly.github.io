1C-Bitrix: Media Player

A few weeks ago, we announced to our partners our intention to license and release components for playing video and audio content.

The entire solution was named ‚Äú1C-Bitrix: Media Player‚Äù and has already been released as a component in the ‚ÄúStructure Management‚Äù module and entered into the ‚Äú1C-Bitrix: Site Management‚Äù edition without exception.

Corporate users do not really want to embed videos from Youtube or other projects into their sites. Yes, and customer needs have greatly expanded ...

In general, we believe that playing video and audio is already a mandatory task of any modern site and our clients and partners should have a universal and functional component to solve these problems.

So, what we can now and what you have.

We implemented a universal media player, which automatically, depending on the data format, is represented by the Flash or Silverlight component.



The following media content formats are supported:

Flash:

* FLV7 video
* FLV8 video
* H.264 video
* Youtube video
* MP3 audio
* AAC audio
* Jpg images
* Gif images
* PNG images

Silverlight:

* WMV1: Windows Media Video 7
* WMV2: Windows Media Video 8
* WMV3: Windows Media Video 9
* WMVA: Windows Media Video Advanced Profile, non-VC-1
* WMVC1: Windows Media Video Advanced Profile, VC-1
* WMA 7: Windows Media Audio 7
* WMA 8: Windows Media Audio 8
* WMA 9: Windows Media Audio 9
* MP3: ISO / MPEG Layer-3

Playlist Formats:

* Asx
* ATOM + Media
* RSS + iTunes
* RSS + Media
* SMIL
* Xspf

Implemented many parameters that allow you to customize the player to your needs, including, for example, specify a direct link for downloading video that will be displayed directly in the player, or specify the author's sign that will be superimposed on the displayed video.

This is the first component built on the principle of the minimum number of settings in the basic mode and a large number of parameters in the advanced mode of operation. So, in the simplified mode, in order to insert a video, it is enough to specify only the URL or the path to the file and the size of the displayed player, the image that is displayed before the start of playback, and some playback settings.

In the advanced mode, the component provides a large number of settings, including customization of the appearance with different settings for different types of players.

Flesh player supports swf skins. The delivery comes with two standard skins. Additional ones can be downloaded from the Internet (for example, here) or created by yourself

WMA-player does not support skins, but allows you to change the composition and color gamut of controls.

For a Flash player, work with the playlist is built into the Flash player itself.
For a WMV player, this feature is implemented using the API player and Javascript.

You can also configure where the playlist will be displayed - right or bottom.

The source of the video can be any URL, including videos from YouTube.

Well, accordingly, the media player will be used in all the modules of the product: in blogs, in the forum, info-blocks, in the visual editor ...
