Microsoft Project 2010 - poking around little by little

I would like to propose a solution to a simple, but frequently encountered problem:

In addition to project activities in our life there are plenty of time-consuming unsystematic tasks (order business cards, speak at a seminar, meet a potential candidate, etc.). It is impossible to keep their records as separate projects; not keeping records is a very big risk of overlapping schedule and processing.

Plus, such tasks are often managed by secretaries and personal assistants for whom complex project planning tools are not available.

Outlook integration - SharePoint - Project 2010 allows you to work wonders with task management.

First configure:

1) Let's create a special site in SharePoint 2010 for working with Project Server 2010 data





2) Configure it to synchronize with Outlook using the command on the contextual tape



3) In the Project Server, based on this list, create a Task entry.



Voila!

1) Now we can entrust the secretary to create tasks in Outlook -



2) The secretary or we ourselves can control all tasks on the portal (synchronization happens automatically in both directions!)



... and by the way, now it is possible in SharePoint to customize the relationship of tasks if necessary.



And at any time we can synchronize information with Project Server 2010



All our tasks from Outlook will be correctly reflected in Project Server



And we will be able to see online the planned download of ourselves and all our colleagues in total for all projects and individual tasks.
