C ++ IDE from JetBrains: when?

After our successful April Fool's jokes, we are regularly asked when we will release C ++ IDE. It is clear that everyone who writes under Linux has been waiting for such a development environment for a long time, and the alternative available to Windows and Mac OS will not hurt anyone.

C ++ IDE is a separate product, it is based on the IntelliJ platform, like our other IDEs. C ++ support in ReSharper is a completely different topic, and we'll write about it separately (not today).



We were sent a cloud of requests for a private preview . We are sincerely and very pleasantly impressed. And so we write this post. Although the private preview is not yet ready for delivery to courageous and ready to touch his colleagues, we want to tell what will be in it. And it will be soon, we expect that in March we will start receiving the first reviews from some of you.

The first successful developers are already selected, in whose hands the private preview will fall in the first place. The principle of selection is simple: it is known that in a private preview it should work well, and that it is not yet fully polished. When completing the form, the signatories to the sample indicated their interests and the nature of their projects. Those whose interests successfully coincided with the already working in the IDE features, we chose. Soon they will receive a long-awaited link to download and check how well their luck smiled.

By the way, it‚Äôs not too late to register for a private preview , but we do not guarantee that all those who register will receive the very first private preview. Do not blame me if this build bypasses someone.

Here is a list of the most interesting of what will be in the near private build:

  • CMake + GCC + GDB7.
  • Multi-platform support: Linux, OS X, Windows (Cygwin, MinGW).
  • C ++ 11 support (including lambdas, string literals, templates with a variable number of parameters (variadic templates), decltype, auto, etc.)
  • Code analysis and many, many tips (quick-fixes).
  • Refactors (extract method / variable / typedef / define /, etc., change signature, rename, safe delete, etc.).
  • Code generation and templates.
  • Preprocessor support (i.e. all the usual convenient features work with macros).


Clang / LLDB support will appear in subsequent EAP builds.
