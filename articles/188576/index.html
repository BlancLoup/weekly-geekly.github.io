Avaya Aura Communication Manager: Automatic Dialing

Sometimes it is impossible to reach a subscriber. And patience is only enough for a few calls, and then everything is forgotten and as a result, many issues remain unresolved. Some administrators of Avaya telephone stations do not even suspect the possibility of automatic dialing and answer questions from users about such a possibility that this is impossible to do, and some implement this option on third-party software.

Next, we consider how the possibility of automatic dialing at the Avaya Aura Communication Manager stations is implemented.
Attention! There are no pictures at all!


Technology work


The auto callback option was implemented in 4 more release of Communication Manager. Earlier software releases did not support this function, this was done by telephones and not at all as the PBX does. The Automatic Callback function is activated by the subscriber during a call: the subscriber makes a call to the phone, say, who is busy and presses a button or dials the service code and then hangs up. The PBX monitors the status of the previously called subscriber and as soon as the subscriber‚Äôs phone becomes available, the PBX calls back the caller and when he picks up the phone makes a call to the called party.

As for not answering, everything is not so colorful here. If you cannot reach the subscriber who is not answering and enable the autodial function, the PBX will wait for any action from the subscriber (for example, picking up the handset or enabling / disabling the service function). After the called subscriber has shown his activity, the PBX immediately calls you and connects with the called subscriber. But it happens that a person after an absence on the spot can come and not touch the phone literally for hours, and all this time you will wait for the PBX to call you.

And yes, when the PBX, after realizing that the subscriber is available, will call you with his phone number.

But it is absolutely necessary to notice one very important thing! This option will work either only inside the PBX itself, but may also work through trunks if the PBX connection is made using the Q-SIG service protocol, which is extremely rare in Russia. It is also worth noting that this functionality will not work through T1 trunks.


Customization


Assigning a service code for auto dialing

To assign a service code (FAC) to the auto redial function:
  1. Enter settings: change feature-access-codes
  2. Fill in your free codes with the fields Automatic Callback Activation and Deactivation.
  3. Profit ...


Set the non-response interval

It is necessary to specify how many times the PBX will call the caller until it cancels the autodial function enabled by it. This is necessary so that the process does not become infinite, if the caller suddenly left forever, but forgot to cancel the automatic redial.
  1. Enter settings: change system-parameters features
  2. Specify the number of calls in the Automatic Callback - No Answer Timeout Interval (rings) parameter, after which the automatic redial feature will be canceled.
  3. Profit ...


Assigning a service button for autodial

To assign a button for auto dial:
  1. Enter the subscriber card: change station n , where n is the extension number.
  2. Add an auto-cback entry to one of the logical buttons in the Button Assignments section.
  3. You can repeat 2 points a few more times, thus it is possible to expand at the same time to put several subscribers on the automatic redial.
  4. Repeat from 1 to 2 points for all phones on which you need to implement the function of automatic redial.
  5. ... Infinite profit!


ATTENTION! It is assumed that all phones, on which there will be an automatic redial button, are assigned the appropriate COS , in which the Auto Callback option will be enabled.
