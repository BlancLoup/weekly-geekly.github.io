Definition of terms in the contract for the development of the site

In the process of interaction between the web-studio and the client, situations often arise when important issues within the project depend on the understanding and interpretation of technical terms. For studio workers, the definitions of most of the terms that one has to operate in conversations with a client seem obvious, and sometimes it seems that for a client ordering a website it should be no less obvious. However, in life a client may not know the definitions at all (just use words without understanding their meaning), interpret them in their own way, or (especially in case of disputes) use the interpretation from the first available source, including any available person who understands. "

In due time having swallowed on this basis of problems, now the text of any our contract begins with definition of terms. At once I will make a reservation that we used not vocabulary, not encyclopedic and not exact technical definitions, but adapted to our taste under the sauce of the legal style of presentation. Legally, it is in fact not so important that an exact correspondence, as much as the coercion of the definition itself in the text of the document, in order to rely on it in the form in which it is presented.

Here I quote a part of our contract, which defines the technical and production terms used in the process of developing the site (in the text of the contract and the terms of reference in particular). I hope someone can be useful, criticism and alternative opinions are welcome.

Definition of terms


  1. A data transfer protocol is a set of agreements that defines the exchange of data between different objects.
  2. Server - a software package for a computer designed to store, process, and provide access to information via data transfer protocols through an informational dialogue ‚Äî processing incoming information request messages and generating response messages to them.
  3. Network protocol is a data transfer protocol intended for making connections and exchanging data between two or more network nodes connected to a network via communication channels.
  4. Client is a computer software package designed to receive and use the services provided by servers by exchanging information messages with servers using data transfer protocols by computer users.
  5. A web server (web server, HTTP server) is a server that uses the HTTP network protocol.
  6. A web client (web client, HTTP client, browser, web browser) is a client using the HTTP network protocol.
  7. Domain name (domain name) is a unique standardized identifier in the symbolic form of an information resource of a communication node on the Internet.
  8. Website (web site, web site) is an information system created on the basis of a web site of a communication center identified by a domain name, intended for posting information on the Internet, means of its visual design and interaction with users via web clients.
  9. Hosting (hosting services, hosting site) - a range of services provided according to the organizational and technical regulations agreed with the service provider for providing access to servers running on the hardware and technical base of the service provider's communication centers, their capabilities, including server administration.
  10. Site hosting (site hosting site) - hosting, which includes the use of a web server to host a site, site mapping to a domain name and access to it from the Internet and other related services.
  11. Site visitor - a user interacting with the site through a web client.
  12. Information material - information in text, graphic, audio, video, photo, and also in other form, intended for placement on the site.
  13. Content Management System is a software package designed to manage website information materials, providing the user with the necessary functions for creating, editing, storing, publishing information, regulating user access rights to the website and other functions based on the server software of the site hosting platform.
  14. Website design is a unique graphic design of the website and ways of presenting information presented to website visitors by their web clients.
  15. A web page (web page) is an electronic text document in the HTML standard format (SHTML, XHTML, etc.) containing semantic, design and service information parts, information on the markup of its parts and, if necessary, client program code and other functional and design elements, or information about their location and use, in the appropriate formats (JavaScript, CSS and others) provided for by the standard of the page format used.
  16. Static webpage - webpage as a file.
  17. Static website - a website consisting of a set of static documents (web pages and other files) and transmitted to web clients in unchanged form when they are requested by web clients for identifiers defined by their web server (for example, relative file paths and files).
  18. Dynamic website - a website that has a dynamic formation of responses (of various formats, including web pages) generated by a web server as a result of the execution of a program code (for example, a content management system).
  19. Website Management System Website Template is a document template based on a web page containing a layout and a special markup code for the placement of semantic parts by the site management system used to format documents of a dynamic website in the form of web pages.
  20. The layout of the web page of the site - a static image that reflects the design of the web page of the site.
  21. Layout of web pages of a site (layout of a site) is the process of creating web pages based on available layouts of web pages of a site and then transforming them, if necessary, into templates of web pages of a content management system.
