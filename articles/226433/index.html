Php


  • RFC: Uniform Variable Syntax - A proposal from Nikita Popov, which involves the introduction of a single syntax of variables and calls in PHP. So, inaccessible constructions will become possible: $foo()['bar'](); $foo::$bar::$baz; foo()(); $foo->bar()() $foo()['bar'](); $foo::$bar::$baz; foo()(); $foo->bar()() $foo()['bar'](); $foo::$bar::$baz; foo()(); $foo->bar()() , as well as operations with any expressions in brackets, for example (function() { ... })() and others.
  • RFC: Bare Name Array Literal - It is proposed to make quotes optional in declaring the keys and values ‚Äã‚Äãof arrays by analogy with the declaration of objects in JavaScript.
  • RFC: Bare Name Array Dereference - Continuing to the previous one, in this sentence, the author wants to shorten the syntax of the call to the elements of the nested array, and instead of $myArray['foobar']['andSoOn']['moreKeys'] use something like $myArray:>foobar:>andSoOn:>moreKeys .
