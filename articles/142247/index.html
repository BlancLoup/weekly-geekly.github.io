The Pirate Bay: and .torrent files have not gone anywhere

When my ¬µTorrent once again refused to show the contents of the torrent after inserting an insignificant magnet link from Pirate Bay, I wondered which of the .torrent files were deleted. At first glance, it seems that after the appearance of the tenth step, the file disappears, and only the magnet link remains. However, what will happen if the number of seeders decreases? Can a generous pirate soul allow precious pirated content to disappear?

Go to the search for "Windows" and look at the dead tail of the results. We find the torrent "Windows 8 Watermark FIX" without a single Sid and Licher. We look at the link to the torrent file:

http://torrents.thepiratebay.se/7076716/Windows_8_Watermark_FIX_(updated)_-_SS64084.7076716.TPB.tor*ent

The structure is noted:

http://torrents.thepiratebay.se/ {ID} / {NAME} . {ID} .TPB.tor*ent

We return to the search , we find the popular torrent "Windows 7 Loader, Activator" with 1478 sidahs and 14 leechers. There are no links to the torrent file, but there is a name and there is an ID. Substitute:

http://torrents.thepiratebay.se/6780109/Windows_7_Loader,Activator_v2.0.6_Reloaded_-_DAZ_[Team_Rjaa].6780109.TPB.tor*ent

And - lo and behold! - file is downloaded. Despite the fact that he was "removed." We experiment further and understand that the format is really:

http://torrents.thepiratebay.se/ {ID} / {ANYTHING}

That is, the link is enough to specify the ID:

http://torrents.thepiratebay.se/6780109

It remains only to write a user script, so as not to get these actions up manually.

Interestingly, on The Pirate Bay specifically left a child's hole, to deliver at least the inconvenience of downloading? And I wonder how it will help protect against militant copywriters, because the only change is the disappearance of the link from the web muzzle.
