What's new in Quest vWorkspace 7.2 Beta 2?

Quest just the other day, Quest announced a new beta version of its vDI solution vWorksapce - 7.2 (build 409) (beta 2). What is new in this build? Look closer!
Compared to Beta 1, the following appeared:
1. vWorkspace Reporting - allows you to create reports on the activity of users and administrative staff in real time, and sections for the past tense. Very useful and necessary feature, not my opinion.
2. Full and official support for VMware vSphere 4.1 hypervisor - although the previous version of vWorkspace supported work with vSphere 4.1, but not officially :)
3. Integrated updates vWorkspace 7.1 MR1.

However, one should not forget that all features of Beta 1 have been preserved in the current release, namely:
1. Quick Start Wizard - completely redesigned vWorskpace configuration wizard for pilot, test or industrial operation.
2. Updated product installer .
3. PoweShell support - that's it! - support for cmdlets to automate vWorkspace management. Personally, I have been waiting for this feature for a long time - after all, now the admin's routine can be automated!
4. Support for a new product licensing model - it is now possible to license without sending VMAC to Quest to automate a product, which will certainly simplify system operation.
5. Improved error reporting system ‚Äî a modified mechanism that can now offer solutions to various problems.
6. Virtual Desktop Failover Mode - if persistent VM is not available, then a temporary VM from the same group (pool) of VM is deployed to the user.
