Office as Platform: publish the SharePoint application in the Office Store

This is a continuation of a series of articles on the development of cloud applications for Office 365 SharePoint Online. In this article in the cycle, Dmitry Plotnikov, a SharePoint consultant, talks about publishing ready-made SharePoint applications to the Office Store. All articles in the column ‚ÄúOffice as Platform‚Äù you can always find at the link #officeplatform - Vladimir Yunev.

In this article I will talk about working with the app store. In the app store, there are extensions for Excel, Outlook, PowerPoint, and so on, but this article only deals with SharePoint.

Application Installation


Compared to the on-premises version of SharePoint, Office 365 users do not need to customize anything. The store is available at a direct link http://store.office.com .

The installation process itself is extremely simple, just select the collection of sites where you will deploy the application of your choice.



And in the end, be sure to click on Trust It.



To remove an installed application, use the corresponding button from the context menu.



Publication


To publish applications, use the Sales Dashboard. After creating an account, the validation process will occur, and proceed to publishing applications.

In the first step, select the type of application.



The next step provides general information about the application, including the name, logos, link to help and so on.





Next, specify the country and the corresponding name and description.



The following indicates the availability of applications for countries



in the last step, select a pricing model
