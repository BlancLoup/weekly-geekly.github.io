VLC 2.1 released

VideoLAN has announced the release of the free video player VLC 2.1 (Rincewind) - the first major release of VLC in a long time.

The list of changes is significant: a port has been implemented on mobile platforms, many new formats are supported, devices, metadata, even more improvements in the reproduction of old formats, and preparations have begun to support new codecs. Fixed about a thousand bugs, more than 7000 commits from 140 volunteer developers were made to the main branch.

1. Rewritten audio and audio module processing engine : better sound and sound device control, correct support for multi-channel 5.1, 6.1 and 7.1 formats, new audio output capabilities in Vista, Android, iOS, OpenBSD and OSSv4, several new sound filters / effects, increased frequency sampling, accuracy, real-time configuration in the audio engine, many new supported audio formats with metadata.



2. New codecs and hardware decoding : support for video codecs G2M4, MSS1, MSS2, TSCC2, CDXL, Ut, VBLE, audio codecs Ulead DV, Indeo Audio Coder, RealAudio Lossless, subtitles SCTE-27 and EIA-608, hardware decoding through QuickSyncVideo under Windows , via VDPAU under GNU / Linux, through MediaCodec under Android, through VDADecoder under OS X.



3. New video subsystem : OpenGL ES output via OpenGL ES, support for color conversion shaders in glsl for Android and iOS, support for deinterlacing for greater color depth in the XYZ color space, new anaglyph filter for 3D, support for 4K.

4. Support for new formats : fragmented MP4, Wave / RF64, improved support for tags with metadata and covers for Ogg, AVI, MP4 and MKV, support for sound FLAC, Atrac, ADPCM, DV Type 1, 12-bit DV in AVI containers, extended support for AVI, MKV and MJPEG streams, better recording in AVI and MKV formats, audiofingerprinting in AcoustID format.



5. For anime fans : new downmixer from MKV / Flac 6.1 to 5.1 format, correct YUV-> RGB color matrix in OpenGL shaders, improved search support and improved speed in MKV, editorial support in MKV, better support for subtitles and meta data in MKV , various improvements in subtitles ass.



6. New input devices : from screen in OS X Lion and later versions, support for Microsoft Smooth Streaming, new RTMP module using libavformat, support for VNC / rfb and Remote Desktop in viewing mode, significant improvements in playback methods with Blu-Ray, Dash, v4l2 and HTTP.

7. For mobile devices : port on Android from 2.1 to 4.3, on ARMv6, ARMv7, x86 and MIPS, new port on iOS from 5 to 7, on all iPad and iPhone after 3GS, partial port on WinRT, for Windows 8, 8.1 and WP8, optimized output via OpenGL ES, improvements in decoders, encoders, and renderer OpenMAX IL, new formats and interfaces for audio and video output for mobile devices.

8. For developers : libVLC and most modules are now distributed under LGPLv2.1 +.



The libVLC media framework can be used in most applications, the kits for the libVLC SDK developers, improved libVLC APIs have appeared, and the VLC web plugins now support playing in windowless mode.
