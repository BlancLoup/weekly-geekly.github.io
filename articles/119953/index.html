1C-Bitrix in the Scalaxy panel

Good time of day, colleagues!

This time, the company Oversun shares great news with all Bitrix developers: now the CMS 1C-Bitrix image is available in the panel, working on CentOS 5.5

As a product, we took a turnkey solution from the company 1C - RPM-package bitrix env 2.0 , so all the documentation and instructions for the products of Bitrix are relevant for admin.panel Oversan-Skalaksi .

In addition, we have made several changes to the product:

a The standard Bitrix env 2.0 package has its own scaling script /etc/init.d/bvat, which configures the software when the machine starts up, depending on the size of the RAM. This script was slightly corrected by us so that with a virtual machine size of more than 1GB of RAM, Apache could fork up to 50 processes (by default, this value is 10).

b . We also connected the Scalaxy.repo repository, from which the scalaxy-guest-tools package was installed, which is currently empty and does nothing except checking the version of the package and the update, if necessary. In the future, this package is designed to configure the software inside the virtual machine for scaling.

With this package, we can perform any actions on all machines running from this image. If you want to remove this package, you need to connect to the machine via ssh, see the ‚ÄúBitrix virtual appliance version 2.0‚Äù prompt, then press Ctrl + C to enter the shell and execute the following commands:

# yum -y remove scalaxy-guest-tools
# yum clean all
# rm -f /etc/yum.repos.d/scalaxy.repo
# rm -f /etc/cron.hourly/scalaxy_tools_update
