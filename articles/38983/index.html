Accelerate with Gears

What is Gears?

If you don‚Äôt go deep into details, this is a program that downloads static site files to your computer, for example, icons, design pictures, JS scripts or styles. As a rule, this significantly increases the speed of work with the site. It also allows you to work with an online application that is essentially offline, the main thing is to have at least a short-term access to the web.

A vivid example of this is the admin engine of the blog engine - WordPress, version 2.6.

There will be no pieces of code, with detailed tutorials, just some general information. I dug up this information both on the official website of Gears and on the Russian Internet, I was interested, so I decided to share it with you.


Install this program (or rather, add-on to the browser) is easy, just go to the official page .
Here are the system requirements:

Windows
Windows XP / Vista -> Firefox 1.5+ and Internet Explorer 6.0+

Windows mobile
Windows Mobile 5+ -> Internet Explorer Mobile 4.01+

Mac
Mac OS X 10.4+ -> Firefox 1.5+ and Safari 3+ (I could not install Gears under windows vista, maybe temporary difficulties?)

Linux
Linux -> Firefox 1.5+
32-bit OS (64-bit not supported)

As you can see - Opera, unfortunately, goes to the forest. But this, I believe, is a temporary phenomenon. Exactly like this - that sites that are already ready to work with this technology - once and twice. One of the first who decided to fix this is the WordPress development team.

As promised - I will give some statistics on WP.

I didn‚Äôt want to start using this technology for a long time, but I decided to try at work, where the Internet connection speed does not exceed 250K. As a result, the loaded WP admin panel began to fly very quickly. All scripts, all icons and styles are loaded instantly. And this is not a joke, really momentarily, only content is loaded. Sorry for the face of the blog does not apply.

Now this is not so much an admin as an application that hangs in my separate label, and is loaded instantly.

Naturally, you now ask how this differs from the usual browser cache. The main difference is that Gears ___ statics of the site, and you work quickly and without brakes, when the browser thinks about some icon on a godforsaken page.

By itself, Gears even more ruins the border that lies between the online application and offline. Moreover, with the development of technology and its spread, it can destroy it completely.

A person can sit and work all day in full analogue of Excel, not even suspecting that he works with Google Docs. Already, some browsers allow you to create application shortcuts, in fact, opening a browser window without any identifying marks. Just a window with the interface that the programmer wrote and invented the designer.

And now you can work with such "labels" using Gears. It is phenomenally fast and very convenient. Even with short-term problems with the Internet.

I also installed Gears on IEm on my PDA - I really don‚Äôt like it, but the speed of working with a blog is more than through mini Opera.

The technology is multifaceted, it allows you not only to keep statics on your hard disk, but also to work with the site in several streams. Saving statics on the locale is rather a prelude, a nice bonus to the main features.

A striking example of this is YouTube. With Gears installed, you have the opportunity to choose an unlimited number of files to download, and go to sleep - in the morning everything will be on YT servers, isn‚Äôt it convenient?

Moreover, the relatively recent versions of the API for Google Gears allow (so far, unfortunately in the US) using cellular networks or public WiFi, you can find yourself on the planet, find restaurants that are nearby, or see which parks are nearby. And while GPS is absolutely not needed.

Briefly, what Gears is is a technology that allows you to create unique (if not impossible, who knows), until now, web applications, providing tremendous speed and convenience to the end user.

The user may have an unstable communication channel - however this is not a hindrance. As they shouted at some forum - Dialapschiki - rejoice!

Data, over time, can become a lot - also not a hindrance, SQLite acts as a repository. You need to change the car, for example because of the breakdown of the old laptop - just install Gears and copy the database from the catalog to the catalog.

However, now it is impossible to rely too much on this technology - it is rather raw, and it will be filed more than once.
