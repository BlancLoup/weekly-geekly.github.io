How is the Jigsaw ?!

I think everyone is interested in the state of the JigSaw project. The modularization built into the Java platform, a potential replacement for other systems, functionality that was delayed until JDK 9, although initially assumed in JDK 8, etc. etc.

In short: Jigsaw prototype, take 2

More information about the discussion can be found here:
mail.openjdk.java.net/pipermail/jigsaw-dev/2013-August/thread.html
mail.openjdk.java.net/pipermail/jigsaw-dev/2013-September/thread.html

The very essence of what is happening:
1. Mark put the community to the knowledge that the project will be developed anew.
2. Mark promised (more) a detailed document on the proposed requirements and the implementation of the Jigsaw.
3. This has not happened yet.



All subsequent correspondence on alias boils down to the fact that:
1. It's time to go.
2. Already do not need: JDK 8 profiles is enough.
3. Suggestions for functionality (quite a bit).
4. Compilation time dependencies and execution time dependencies are different things.
5. Breaking rt.jar into parts and creating a modular system are different tasks.
6. Maven and OSGI are not perfect.

When will the information !?
Very soon.

The easiest way to get it first is to follow the information on the alias. I am also sure that we will definitely draw attention to the fact.
