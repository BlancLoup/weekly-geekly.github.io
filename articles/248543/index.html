Systemd author - Lennart Pottering game

While around the introduction of systemd in various distributions inflame serious passions, even to leave the posts on the board, as was the case with Debian, some developers instead of fruitless and heated fights by correspondence are engaged in useful work.

Someone introduces systemd, someone dissenting does fork, and someone - the game.



One of those who disagree with the systemd - Gethyn ‚ÄúXylemon‚Äù ThomasQuail made the game based on the popular one at the time when it was decided to scold Bill Gates - XBill.



It is posted on the developer‚Äôs github and is called XLennart. As you already understood, this is a parody game in which the player is required to prevent Lennart Pottering from systemd from planting on distributions and operating systems.

The game comes with open source and it is free. How free is not entirely clear, since it is not indicated under which license it is distributed. But this does not prevent us from paying more attention to it.
The player, being in a role of the system administrator, and someone (very similar to Lennart), tries to destroy his computers. The little man runs around the game space level and infects computers with the SystenD virus, which very much resembles one known initialization system.



The task of the player is to prevent this. If the little man gets to the computer, he will try to change the installed operating system. The game ends if there is no more than one working computer left. With each level the difficulty increases. There are computers, networked. In this case, the infected computer begins to transmit the virus over the wire to the other computers on the network. To interrupt the data transfer (it is displayed in the lights running along the wires in the direction of the new computer) using a bucket of water, which is located in the upper right corner.

The status bar below contains information:

  • The number of computers running on their systems initialization / shutdown / systenD
  • Level
  • Number of points




Running game
After the archive with source codes is downloaded and unpacked, let's start compiling.
There are no special system requirements for the application and it is collected in three steps:
  1. ./configure
  2. make
  3. make install


After installation, run the xlennart application from the terminal.

For openBSD there is a port openports.se/games/xlennart
And on the OpenBSD 5.7 coming out soon, the game can be installed via the package manager
pkg_add xlennart

Launch parameters
-l Run the specified level in n.
--gui - Specify a specific frontend. May be gtk, motif and anthena
--size - The size of the playing field. Default 400x400
-v - Display version information and exit
-h - Help output and exit
