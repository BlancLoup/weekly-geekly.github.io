What can it be used for? Of course, for cloning visual components from a template. I would prefer to leave the advantages and disadvantages of this approach to the gurus.

Usage example: 
 procedure TForm1.Button1Click(Sender: TObject); var Clone: TPanel; begin Clone:=TPanel(TClonable(Panel1).Clone(Self, [])); Clone.Top:=Panel1.Top+Panel1.Height; end;


Description of the Clone TClonable method:
function Clone (const AOwner: TComponent; aExcept: array of string): TComponent;
  • AOwner : TComponent - the new owner of the component being cloned
  • aExcept: array of string - an array of strings containing property names (meaning PPropList) to be excluded when copying
  • Result - a link to a new object of the TComponent class representing a copy of the original object, the Name property is empty


In my opinion, the implementation provides a convenient way to copy components: with the provided copy mechanism for child objects, which may contain TWinControl heirs; reassignment of events; the ability to exclude unnecessary properties.
