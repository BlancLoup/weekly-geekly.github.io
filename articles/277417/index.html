In ReactOS 0.4, a lot of new things are waiting for you compared to the previous release 0.3.17, for example:

  • ext2 \ 3 \ 4 file system support for reading and writing;
  • support for reading from the NTFS file system;
  • updated desktop shell explorer.exe;
  • Support for most existing virtual machines as a guest system (with some limitations);
  • support for automounting shared folders for Virtual Box and VMWare;
  • the inclusion of the mbedTLS TLS library;
  • improvements in the work on the "official" laptop Dell Latitude D531 ;

Known limitations and problems:


The recommended amount of RAM for testing is 1024-3078 mb. For its work, ReactOS only has 64 mb of RAM , but due to the peculiarity of the memory manager and file system drivers, at some points in time, depending on the software used, it may require much more. This happens if you download or unpack files of considerable size. Hence the recommendation to use the amount of RAM that is equal to or exceeds the size of the file being downloaded or unpacked.

The fix under commit number 70646 for problems related to the activation context , unfortunately, was not released. But it, nevertheless, is already in the nightly builds and will definitely fall into the next release 0.4.1, the preparation of which is already well underway.

The Intel 21140 Based PCI Fast Ethernet Adapter in VirtualPC and Hyper-V requires a separate driver, which is not included in the package. Download it here .

If Virtual Box is used, version 5.0.4 or higher is strongly recommended for the correct operation of guest add-ons.
