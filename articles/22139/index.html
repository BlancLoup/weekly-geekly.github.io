Really working development management system

Initial data:
development of a multi-module non-trivial web resource without a detailed problem statement prior to the start of development.

Problem:
We have to use a lot of different accounting systems for project documentation and problem statement, because there are many requirements, but there is no satisfying system for all of them.


pros

Google docs
1. Ability to bind comments to the source text of an entry
3. A handy editor for formatting a document.

Jira
2. Update Notifications
4. Personalization of the task (binding to the account in the system)

Basecamp
2
6. Ease of perception of tasks of medium and large size.

Wiki
5. Ability to create a flexible knowledge base structure

What I want to see:
A system that has all of the above features - allowing:
  • set the URL and place of the document in the hierarchy in the knowledge base,
  • leave comments tied to a place in the source text of the task,
  • see the task in 1 of 2 views: source text and text with comments,
  • notify colleagues about updates (for example, when exiting an editing task),
  • form tickets for parts of a task or a whole task.


As a result of this approach, flexibility is given. The final modified text of the task in any case can not be obtained without editing based on the comments - but you can either see the original task in its entirety, or the final commented pieces of the task.
