Auto provisioning for Fanvil BW210P



In our office, Fanvil devices are used for IP telephony. How justified the choice in favor of cheaper Chinese products will be shown by time, and I will tell you about how I implemented the auto provisioning mechanism on Fanvil BW210P models.

For those who are not familiar with this term, but somehow miraculously read it to this place, I will explain that auto provisioning is a mechanism that allows you to centrally manage the settings of IP phones.

What it looks like.
The phone connects to the FTP server, finds its configuration file, checks the version of the config file on the server with its own. If the number is different, then the config file is applied.

The process uses four files:
dsc.exe is the dsCrypt utility. It is used to encrypt ready-made configs. (Windows utility. Unfortunately, neither aescrypt, nor aespipe, nor other Linux utilities could create encryption that is understandable for the telephone. If someone knows those and tells about them, I will be grateful.)
genconf.sh is a script that generates a ready config.
key.txt - file with encryption key. The key consists of 64 HEX characters.
users - this can be said base phones. Contains mac device address, sip number and password.

Pure-FTPd server is deployed on Ubuntu 10.04.
Because The Windows utility is used, I had to install wine.
