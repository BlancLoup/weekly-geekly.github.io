Backup for Zimbra Collaboration Suite

Unfortunately, in the free version of Zimbra Collaboration Suite there is no built-in data backup mechanism. System administrators in companies where they decided to implement this solution have to invent solutions themselves. The most commonly used scripts, which in time make full backup copies of data. For small companies, this is, of course, a way out, but when the number of users reaches at least 50, the storage of archives with the contents of their mailboxes begins to take an inadmissible amount of space. Plus data recovery becomes a very long and problematic operation. Need another solution!



At the moment, Zextras Backup is the only data backup solution for Zimbra with different recovery options: from single-element to full disaster recovery. All recovery modes are transparent to the user and 100% independent of the OS of the source and target servers, architecture and version of Zimbra. Zextras Backup also allows you to take a complete snapshot of the mail system and store it anywhere.



Zextras Backup provides easy management of all backup settings, including checking the integrity of backups using Zextras Administration Zimlet. Zextras Backup supports compression and deduplication, thereby providing high savings on server space. The storage size becomes on average 70% less than the current amount of data. You can also exclude from the reservation extensible service classes (Classes of Service) and thus save even more disk space.

In addition to working in real time, Zextras Backup has an ‚Äúexternal backup‚Äù function that allows you to save self-consistent snapshots of data on remote servers, which provides additional security.

Data recovery in Zextras Backup always maintains the consistency of shared folders, so they do not need to be reinstalled after recovery. User mailboxes are always available for work, all recovery modes are completely transparent, which reduces the load on the user to a minimum.

6 recovery modes


Six recovery modes in Zextras Backup will help at any time to restore the necessary data, from one element to the whole domain.

Restore from external archive


Restore all data and settings or import data from an external backup using the Zextras migration tool. Recovery is possible from the continuous backup folder or an ‚Äúexternal source‚Äù.

Recover Items


Mailbox item recovery by Item ID parameter.

Recover new account


Restore all mailbox data and settings exactly as they were at a certain point in time, creating a new mailbox hidden from GAL with the same items and folders.

Recover deleted data


Recover any items deleted from the mailbox in a certain period of time and in a specific folder.

Recover deleted account


Restoring a deleted account is the same as it was at the time of deletion.

BLOB recovery


Recover corrupted BLOB data using the cross-reference recovery mode in the Zimbra database and Zextras Backup data.



Redundancy Algorithms


Zextras Backup works on the basis of atomic continuous algorithms using the deduplication and compression system. Thanks to atomic write algorithms, all data in the backup storage is always 100% consistent and can be transferred to storage devices, such as tape drives and optical disks, even during backup system operation.

Atomic algorithms are designed to work with individual data and with separate subsequent operations. Therefore, if some operations make a mistake, then the whole operation does not continue, but returns to its original state. For example, if an error occurs in a backup, it does not affect other operations.

The old ways Zimbra backups took all the files and compressed them in a single archive. If there were problems while writing this file, the file was corrupted.
