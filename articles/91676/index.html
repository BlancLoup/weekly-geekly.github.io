New version of App Engine SDK - 1.3.3 released


Mostly in this version errors are fixed, but nevertheless there are additions, including one rather interesting (about SQLite).

Changes for Python


  • New experimental opportunity to use SQLite as a datastore simulator engine for local development. In theory, this should increase performance when testing locally on large amounts of data.
  • Ability to implement your property in db.Expando
  • Fixed a bug that occurred when trying to set a position in the query (offset) greater than the number of query results ( http://code.google.com/p/googleappengine/issues/detail?id=2875 ).
  • Fixed a bug that prevented seeing values ‚Äã‚Äãlike ByteString in the Development Console ( http://code.google.com/p/googleappengine/issues/detail?id=1176 ).

Java changes


  • Two new system properties have been added: com.google.appengine.application.id and com.google.appengine.application.version
  • Now before the HardDeadlineExceededError error is thrown, the DeadlineExceededException exception is always thrown first.
  • Reduced the likelihood of ‚ÄúToo many URLMap‚Äù errors when uploading complex applications.
  • Fixed a bug that caused QuotaService.getCpuTimeInMegaCycles () to return the number of cycles, not millions of cycles.
  • Fixed a bug due to which the behavior of jsp was different during development and after posting ( http://code.google.com/p/googleappengine/issues/detail?id=3022 ).
  • Fixed a bug that occurred when uploading applications with .tag files ( http://code.google.com/p/googleappengine/issues/detail?id=2902 ).
