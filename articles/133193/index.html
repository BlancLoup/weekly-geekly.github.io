DATA STORAGE



MySQL , why MySQL ? We are not talking about other repositories, because the classic web project was then built on MySQL. And it would seem, when we have a problem with the database and it would be good now to cancel the hard drives and put everything in HEAP. This is bad because memcached is not a data warehouse. He is not obliged to remember what you put in him, he can delete the key at any time, even if it has not been cached. The most important thing is everything in memory.
