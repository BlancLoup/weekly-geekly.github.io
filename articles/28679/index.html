Parsing a GET request in PHP (jokes for automatic parsing)

Today I decided to figure out how PHP defines the key in the $ _GET array for parameters received by the GET method, respectively. To be honest, I did not expect to see such illogicality in the work of this mechanism. Although in general, why it happened, it is clear ...


So, immediately to the practice. How do you think the $ _GET line will display a line like this: "? Aa [bb] [cc = 11"? In principle, the options are not so few, among them are logical, and not very. As a result, in PHP we get the following array keys: $ _GET ['aa'] ['bb'] = 11. Where did the "cc" go? What will give "? Aa [bb] cc = 11"? Same thing ($ _GET ['aa'] ['bb'] = 11).
