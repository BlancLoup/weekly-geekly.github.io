Markdown, Dropbox and Hosting / Blogging platform for perfectionists

Migrated from Q & A

I want to talk about the idea that I have been rooting for some time and even took the trouble to prescribe it in some detail.

The task of this post: to find like-minded people and professionals who will help to implement it. Well, or realize that it makes little sense, and I should do more productive things.

Idea in one sentence:

mdbox.com - Web Service for creating and managing simple websites and blogs using Dropbox.

Anyone who loves Markdown and Dropbox, please under the cat

How it works:
  1. User logs in to mdbox.com under his Dropbox login.
  2. MDBox Dropbox App is installed on its Dropbox account, as a result of which the Apps/MDBox folder is created Apps/MDBox
  3. The user chooses his subdomain name, for example jdoe.mdbox.com
  4. After that, all files with the .md extension in the Apps/MDBox are available as HTML pages. For example: Apps/MDBox/about.md becomes jdoe.mdbox.com/about.html
  5. .md files must contain content in Markdown format.
  6. Subfolders in Apps/MDBox become available as path elements. For example: Apps/MDBox/blog becomes jdoe.mdbox.com/blog . A special file index.md in each subfolder is given as index.html for this path.
  7. .md files can be edited in any text editor on any computer, tablet and even phone
  8. Also, at mdbox.com itself, a web version of the Markdown specialized text editor with syntax highlighting is available and, ideally, with WYSIWYG editing capability.
  9. The user can configure his own domain for any folder. For example: Apps/MDBox/blog -> jdoeblog.com
  10. The user can also set up automatic publishing of files to other services. For example: wordpress, blogger, tumblr, livejournal, habrahabr, etc ...
