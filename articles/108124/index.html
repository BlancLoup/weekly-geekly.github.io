samba3 + win7: normal flight

Recently there was a task in the NT4 domain based on SAMBA to include 7s.
Moreover, it was necessary to transfer the existing database of users and computers.

On Habr√©, the topic is not covered, however, good people have already posted their own update experience on the Internet. Samba is also with openldap backend. In general, I confirm - everything works with a bang, the update has passed without problems.

I will deposit my 5 kopecks.
So, for CentOS 5 rpm you can download it here:
ftp.sernet.de/pub/samba/3.5/centos/5/i386 (I used 3.5.6)
In addition to the Win7-Samba3DomainMember.reg file, you also need to install a hotfix so that no errors are displayed. Link here .

I needed these packages here:
samba3-client-3.5.6-43.el5
libwbclient0-3.5.6-43.el5
samba3-3.5.6-43.el5
samba3-winbind-3.5.6-43.el5 (only if you need winbind)

Additionally (to support smb: // addresses in the system):
libsmbclient0-3.5.6-43.el5

I remind you that you need a version of Windows 7 Professional, Ultimate or Enterprise to be included in the domain.
If you have already installed Win7 and set up an account, then User Profile Wizard can be very useful. It will help the current session with the user domain. Tested-works.
