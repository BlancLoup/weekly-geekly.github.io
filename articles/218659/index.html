Chromecast has received new features - Support for "live" broadcasts

After writing a review article about the Chromecast capabilities , as well as the new possibilities for using the device after the release of the SDK, it took quite a long time. Development of applications for HC is very active and almost every day there are new applications that support the transfer of information to Khromkast.
So, the main news of the day: Chromcast now allows you to broadcast a "live" and private video.

And although on the official support page the information was updated only on the second item, current broadcasts can be sent to Khromkast without problems. At the moment, the broadcast can only be run from a computer using the official extension to google chrome , nothing is said about support for android, but everyone is waiting for the application to be updated. People also write about the possibility of broadcasting from i-devices:

If you want to find out how to start broadcasting from a phone or tablet right now, without waiting for application updates, I ask for a cat.

Autocast


So for broadcast, we need Tasker , the AutoCast plugin, which came out of beta status this weekend and allows you to stream videos to the TV through chromecast, photos from the gallery as a slideshow, photos + videos from the gallery, show over the photos and videos notifications coming to your phone , voice anything at the time of the photo / video, use custom pages as a screensaver and a lot of interesting things, which I will talk about some other time. We also need the AutoShare plugin, which allows you to interact with the programs through the standard Share / Send / Share menu. Since Free versions of a version are usually limited to 5 characters in teams, I advise you to use the full versions of these great applications.
For mobile apps, live streaming is currently banned, and if you go to youtube from the Chrome or Chrome Beta browser and enter live for example, we‚Äôll only get a list of recordings of all kinds of live concerts, since it will be determined that you are logged out from a mobile device and substitute m.youtube.com into the address bar. In this case, the filter will be cut.


If you click "menu - Full version", then the full version of the page will be loaded accordingly and in the filters you can select the item "On the Air" and current broadcasts will be available to us, which can be sent via Chromcast to the TV.

A list of some current and planned broadcasts by category can be viewed on the Live channel. which is also available in the mobile version of the browser.

AutoShare and AutoCast Setup

Everything is configured quite easily:
Open AutoShare, select Manage Commands, click + and enter the command that should appear in the menu. I have this cast video.

Next, open the tasker. Add a condition - State - Plugin - Autoshare. Next, click the pencil, mark the Event Behaviour and select Cast video from the Command Filter. (then click back, back)

For the action choose Plugin - AutoCast Other App - pencil. In the Cast Device field, select a device (if it is one, then only the ‚Äúfirst found‚Äù will be available, if there are several, then you must first find them in the application itself. Then the names will be displayed in the list and you can configure commands for different devices). Paste% astext into the YouTube Url field (this variable corresponds to the link that is transmitted when you click on the share menu). In the Command field, select Play, put a check on the Control Other App Service and you can still Persistent Notification.
Back, back. Everything.

Now, if we click on the page with the menu translation - Send and select AutoShare Command from the list, we will see a list of commands from AutoShare. If the command is configured only one, then it will work by default. The link will be transmitted to Khromkast and you can watch a live broadcast.

Also, using the application, you can control playback (including voice using another AutoVoice plug-in), launch playlists from YouTube (I do this with pebble 3 clicks), etc., but this is another story ...
