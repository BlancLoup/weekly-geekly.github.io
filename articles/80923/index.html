Firefox Addon for KWallet integration

As a KDE and Firefox user, I always felt uncomfortable due to the fact that saved Firefox passwords are stored in Firefox itself, and KDE passwords in KWallet. In this connection, I signed up for a bug in Bugzilla @ Mozilla about the need for integration .
And here comes a savior - Argentinean developer Guillermo Molina wrote the corresponding addon - KDE Wallet password integration , which allows you to store Firefox passwords in KWallet.

Initially, I discovered an error, due to which non-ASCII data from non-UTF pages were not correctly stored, which was reported to the add-on author. To his credit, the bug was fixed during the day and I did not find any other problems, everything works as it should :)
According to the discussion on addons.mozilla.org, the addon should work on both 32 and 64 bit systems (I have 32, everything worked fine and works)

I would also like to make a small warning. I'm not sure that already saved passwords are correctly recorded in kwallet when installing an addon, since I use Weave for password synchronization, and at the first update all data was received and recorded in KWallet (by the way, there are no problems with such double synchronization of Weave-KWallet). Therefore, in order to avoid losing the data acquired by unsustainable labor, I advise you to backup all the same, because addon is still experimental.

We try, about the noted problems we unsubscribe on the add-on page .

PS If someone can, post to the Fire Fox blog.

UPD: Murz reports that the new, second, version of KDE Wallet password integration 2.0 began to work fine with Firefox Sync, so now you can safely use this plugin without fear that the passwords will no longer sync, as it was with version 1.x.
